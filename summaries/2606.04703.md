## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）智能体的持续学习与自我进化。具体聚焦于“经验内化”（Experience Internalization）这一将交互经验转化为模型参数化能力的过程。
- **问题（Problem）**：现有经验内化方法在单次迭代中表现良好，但在多轮迭代的持续学习场景下，会出现性能逐步崩溃（progressive capability collapse）而非持续提升的问题。
- **方法（Method）**：系统性地从三个关键维度（经验粒度、经验注入模式、内化机制）分析性能崩溃的原因，并提出一个结合“原则级经验”、“逐步注入”和“离策略上下文蒸馏”的稳健配方。
- **创新（Innovation）**：
    1.  首次系统性地揭示了多轮迭代经验内化中的性能崩溃问题。
    2.  提出了一个三维分析框架，深入剖析了导致该问题的根本原因。
    3.  通过实验验证了离策略上下文蒸馏在多轮内化中的稳定性优势，挑战了当前主流的“在策略”范式。
- **意义（Significance）**：为构建能够持续、稳定地从自身经验中学习和进化的LLM智能体提供了关键的理论洞见和实用的工程指导，推动了从“一次性学习”到“持续自我进化”的范式转变。
- **一句话总结**：本文揭示了现有LLM智能体经验内化方法在多轮迭代中性能崩溃的根源，并提出了一个结合原则级经验、逐步注入和离策略蒸馏的稳健配方，实现了可持续的自我进化。

## 1. 研究动机（Motivation）

- **核心问题**：当前LLM智能体的“经验内化”方法，如“在策略上下文蒸馏”（on-policy context-distillation），虽然在单次内化中能显著提升性能，但在多轮迭代的“持续学习”或“自我进化”过程中，性能不仅不会累积提升，反而会逐步下降，甚至低于基线模型（如图1所示）。
- **重要性**：真正的智能体需要能够像人类一样，从过去的交互中学习，并将这种能力内化，从而在未来的任务中表现得更好。这种“自我进化”能力是构建自主、自适应AI系统的基石。如果内化方法无法在多次迭代中持续工作，那么“持续学习”的愿景就无法实现。
- **挑战**：为什么多轮迭代会导致性能崩溃？是经验本身的问题，还是内化过程的问题？作者认为，失败的原因并非单一，而是与经验如何被表示、如何被注入到教师模型、以及如何被蒸馏到学生模型这三个环节密切相关。现有研究缺乏对这种多轮迭代失败模式的系统性分析和理解。

## 2. 创新点（Innovation）

本文的核心创新在于**系统性地诊断并解决了多轮迭代经验内化中的性能崩溃问题**。具体创新点如下：

1.  **问题发现与定义**：首次明确指出并实验验证了现有经验内化方法在多轮迭代下的“能力崩溃”现象，而非简单的性能瓶颈。这为后续研究设定了一个明确且重要的新问题。
2.  **三维分析框架**：提出了一个新颖的分析框架，将经验内化过程解构为三个关键维度：
    - **经验粒度（Experience Granularity）**：区分了“实例级经验”（instance-level）和“原则级经验”（principle-level）。
    - **经验注入模式（Experience Injection Pattern）**：区分了“全局注入”（global injection）和“逐步注入”（step-wise injection）。
    - **内化机制（Internalization Regime）**：对比了“在策略上下文蒸馏”（on-policy）和“离策略上下文蒸馏”（off-policy）。
    这个框架不仅解释了崩溃的原因，也为设计更稳健的方法提供了清晰的指导。
3.  **稳健配方的提出**：基于上述分析，提出了一个简单但有效的配方：**原则级经验 + 逐步注入 + 离策略上下文蒸馏**。该配方在多个基准测试上证明了其能够稳定地维持多轮迭代的性能提升。
4.  **对主流范式的挑战**：通过严谨的实验，证明了在持续学习场景下，“离策略”蒸馏比当前主流的“在策略”蒸馏更稳定、更有效。这一发现挑战了该领域的普遍认知，并可能引导未来研究方向的转变。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心架构是一个迭代的“经验内化”循环，用于实现LLM智能体的自我进化。其流程如下：

1.  **交互与经验收集**：当前策略 $\pi_{\theta^{(k)}}$（学生模型）与环境交互，产生一系列轨迹 $D^{(k)}$。
2.  **经验提取与抽象**：将轨迹 $D^{(k)}$ 总结为自然语言形式的经验池 $E^{(k)}$。根据“经验粒度”维度，可以提取为“实例级”或“原则级”经验。
3.  **教师模型构建**：同一个策略 $\pi_{\theta^{(k)}}$ 在注入经验池 $E^{(k)}$ 后，扮演“经验感知教师”的角色。根据“经验注入模式”，教师模型可以接收全局经验或逐步选择的经验。
4.  **蒸馏与内化**：通过“上下文蒸馏”技术，将教师模型的行为蒸馏到下一个版本的“无经验学生” $\pi_{\theta^{(k+1)}}$ 中。根据“内化机制”，蒸馏可以在“在策略”（学生生成的轨迹上）或“离策略”（教师生成的轨迹上）进行。
5.  **迭代**：将 $\pi_{\theta^{(k+1)}}$ 作为新的学生模型，重复步骤1-4，形成闭环。

### 3.2 关键算法与数学推导

论文形式化了两种核心的内化机制：

- **离策略上下文蒸馏（Off-policy Context-Distillation）**：
    教师模型 $\pi_T$ 生成轨迹 $H$，学生模型 $\pi_\theta$ 通过最小化前向KL散度来学习匹配教师的输出分布。
    $$L_{\text{off}}(\theta) = \mathbb{E}_{H \sim \pi_T} \left[ \sum_{t=1}^{T} D_{KL}(p_t \parallel q_t) \right]$$
    其中，$p_t = \pi_T(\cdot | h_{t-1}, \mathcal{E}_t)$ 是教师分布，$q_t = \pi_\theta(\cdot | h_{t-1})$ 是学生分布。

- **在策略上下文蒸馏（On-policy Context-Distillation）**：
    学生模型 $\pi_\theta$ 生成轨迹 $H$，教师模型 $\pi_T$ 在这些学生诱导的状态上提供监督。学生通过最小化反向KL散度来学习。
    $$L_{\text{on}}(\theta) = \mathbb{E}_{H \sim \pi_\theta} \left[ \sum_{t=1}^{T} D_{KL}(q_t \parallel p_t) \right]$$

- **全局注入 vs. 逐步注入**：
    全局注入下，教师分布为 $p_t^{\text{glob}} = \pi_T(\cdot | h_{t-1}, c_{\text{glob}})$，其中 $c_{\text{glob}} = [x; \mathcal{E}^{(k)}]$ 是固定的全局经验上下文。
    逐步注入下，教师分布为 $p_t^{\text{step}} = \pi_T(\cdot | h_{t-1}, \mathcal{E}_t^{\text{step}})$，其中 $\mathcal{E}_t^{\text{step}} = R_\phi(h_{t-1}, \mathcal{E}^{(k)})$ 是由选择器 $R_\phi$ 根据当前历史 $h_{t-1}$ 动态选择的经验。

### 3.3 模型、训练与数据集

- **模型**：主要使用 **Qwen3-4B-Instruct-2507** 和 **Qwen3-8B** 作为学生模型。教师模型是学生模型本身加上经验上下文。经验提取和选择使用 **DeepSeek-V4**（默认）或学生模型自身（Qwen self-generated setting）。
- **训练数据**：从五个公开的网页推理QA数据集（WebWalkerQA-silver, DeepDive, WebShaper, WebDancer, SailorFog-QA）中构建了一个包含15K个样本的训练语料库。
- **评估基准**：
    - **WebWalkerQA**：作为领域内（in-domain）基准。
    - **GAIA-Text-103** 和 **BrowseComp-ZH**：作为领域外（out-of-domain）基准。
- **训练细节**：使用 `verl` 框架，学习率 $1 \times 10^{-5}$，batch size 128，在8块NVIDIA A800 GPU上训练5个epoch。推理时温度设为0.7，最大交互步数 $T_{\text{max}} = 100$，上下文窗口为32,768 tokens。

### 3.4 实验设置与结果

实验围绕三个维度展开，核心结果如下：

1.  **经验粒度（图2）**：
    - **实例级经验**：在第一轮迭代中带来短暂提升，但在后续迭代中性能迅速下降，甚至低于基线。
    - **原则级经验**：在所有迭代中都能提供更稳定、更持久的性能增益。
    - **结论**：原则级经验通过抽象出可复用的策略，避免了实例级经验中的轨迹特异性细节带来的过拟合和错误累积。

2.  **经验注入模式（图3, 表1, 表2）**：
    - **全局注入**：性能提升短暂，在多轮迭代中迅速退化。分析发现，全局注入会导致严重的“过早回答”（premature-answer）问题（表2：63.82%），模型在未进行任何工具调用前就直接给出答案。
    - **逐步注入**：在单次和多次迭代中均表现更优，能有效维持性能并避免过早回答（表2：0%）。
    - **结论**：逐步注入通过将经验与当前决策状态对齐，提供了更精准、更相关的监督信号，尤其是在需要多步工具调用的长程任务中至关重要。

3.  **内化机制（图5, 表3）**：
    - **在策略蒸馏**：虽然单次迭代有提升，但多轮迭代后性能下降。原因是教师模型在学生诱导的“错误”状态上进行局部修正，无法提供连贯的、经验引导的完整轨迹示范。
    - **离策略蒸馏**：在所有迭代中均能保持稳定甚至持续提升。教师模型生成的轨迹是连贯且成功的，为学生提供了更清晰、更一致的“经验引导行为”示范。
    - **结论**：离策略蒸馏提供了更稳定、更高效的监督信号，并且避免了在策略蒸馏中观察到的轨迹长度膨胀问题（表3：学生模型轨迹平均长度从2.5膨胀到21.9）。

4.  **最终配方（图6）**：结合原则级经验、逐步注入和离策略蒸馏的最终设置，在所有基准测试上均能稳定地维持多轮迭代的性能提升，并保留了模型利用经验池的能力。

### 3.5 技术细节

- **经验提取**：使用DeepSeek-V4或Qwen模型，将成功的轨迹总结为自然语言经验。原则级经验通过提示词要求模型提取“可复用的策略、决策规则和失败模式”。
- **经验选择**：对于逐步注入，使用一个LLM选择器 $R_\phi$，根据当前交互历史 $h_{t-1}$ 从经验池 $\mathcal{E}^{(k)}$ 中检索最相关的1-2条经验。
- **拒绝采样**：在离策略蒸馏中，教师模型会生成多条轨迹，只保留那些成功完成任务（获得高奖励）的轨迹用于训练学生模型，以确保训练数据的质量。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了LLM智能体在持续学习场景下，经验内化方法无法稳定工作的核心问题。
    - 提供了一个系统性的诊断框架，揭示了性能崩溃的深层原因，而非仅仅提出一个“黑盒”方法。
    - 提出了一个简单、有效、可复现的稳健配方，为实际工程应用提供了直接指导。
- **对未来研究/应用的影响**：
    - **研究范式转变**：挑战了“在策略蒸馏”在经验内化中的主导地位，可能引导未来研究更多地关注“离策略”或混合策略。
    - **工程指导**：为构建能够自主进化的AI Agent（如网页助手、研究助理）提供了关键的设计原则：使用原则级经验、进行逐步注入、采用离策略蒸馏。
    - **新问题提出**：为后续研究打开了新的方向，例如如何更好地自动生成原则级经验、如何设计更优的经验选择器、如何结合在策略和离策略的优势等。
- **局限与开放问题**：
    - **领域泛化**：实验主要集中在网页推理任务上，结论是否适用于其他领域（如代码生成、机器人控制）和语言需要进一步验证。
    - **其他影响因素**：除了三个维度外，经验池大小、选择器质量、过滤标准等因素也可能影响稳定性，本文未深入探讨。
    - **安全与偏见**：更稳定的内化也可能固化模型中的错误、偏见或不安全行为，尤其是在自我进化的闭环中。实际部署需要加入人工监督和过滤机制。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“经验内化”类比**：可以将其类比为学生学习的过程。
    - **实例级经验**：就像学生死记硬背了一道具体的数学题（例如“小明有3个苹果，小红有5个，一共几个？”）。下次遇到“小明有10个橘子，小红有2个”时，他可能就不会了。
    - **原则级经验**：就像学生理解了“加法”这个原则。他学会了“把两个集合的数量合在一起”这个可复用的策略，无论题目中的物品和数字如何变化，他都能解决。
    - **全局注入**：就像考试时，老师把整本数学书放在你旁边，但你需要自己从几百页里找到当前这道题对应的公式。这效率低下，且容易分心。
    - **逐步注入**：就像一位聪明的助教，在你做第一题时，只把“加法”的公式卡片递给你；当你做第二题（关于面积）时，再把“面积公式”的卡片递给你。这极大地提高了效率和准确性。
    - **在策略蒸馏**：就像一个学生自己先做了一遍题（可能做得不好，很多步骤是错的），然后老师来给他批改，指出他错在哪里。老师只能纠正错误，但无法展示一个从头到尾都正确的解题过程。
    - **离策略蒸馏**：就像老师先完美地做了一遍题，然后把这份完美的解题步骤作为标准答案给学生看，让学生模仿学习。学生看到的是一个连贯、正确、高效的完整过程。

- **为什么离策略更好？**：在策略蒸馏中，学生模型自己生成的轨迹可能已经“跑偏”了，教师模型在一个错误的起点上做局部修正，效果有限。而离策略蒸馏中，教师模型从一开始就“带路”，生成一条正确的路径，学生模型直接学习这条正确的路径，自然更稳定、更高效。
