## 0. 概览（Concise Summary）

- **主题（Topic）**：自动驾驶领域的视觉-语言模型（VLM）知识蒸馏。
- **问题（Problem）**：大型VLM在自动驾驶中面临高GPU内存消耗和高推理延迟的问题，而小型VLM性能不足。传统的监督微调（SFT）和知识蒸馏方法未能有效解决能力差距，且缺乏针对自动驾驶任务的结构化蒸馏策略。
- **方法（Method）**：提出Drive-KD框架，将自动驾驶能力分解为“感知-推理-规划”三元组，并为每个能力设计专门的单教师蒸馏策略。在此基础上，构建一个多教师蒸馏框架，并引入非对称梯度投影（AGP）来缓解跨能力梯度冲突。
- **创新（Innovation）**：
    1.  首次对自动驾驶VLM的蒸馏设计进行系统性研究，确定了最佳的蒸馏层和信号（注意力）。
    2.  提出了基于能力分解的单教师蒸馏配方。
    3.  构建了多教师蒸馏框架，并创新性地提出AGP方法来解决多目标优化中的梯度冲突。
- **意义（Significance）**：实现了小型VLM在自动驾驶任务上的性能大幅提升，使其在效率和性能上超越了大型模型（如InternVL3-78B和GPT-5.1），为资源受限的边缘设备部署高性能自动驾驶VLM提供了可行路径。
- **一句话总结**：本文通过将自动驾驶能力分解并设计针对性的多教师知识蒸馏框架与非对称梯度投影，成功训练出高效且性能卓越的小型VLM，在DriveBench基准上超越了参数量大数十倍的模型。

## 1. 研究动机（Motivation）

- **解决的问题**：大型VLM（如InternVL3-78B）在自动驾驶中性能强大，但其巨大的模型尺寸导致高GPU内存消耗和低推理吞吐量，难以满足自动驾驶对实时性和边缘部署的需求。相反，小型VLM虽然高效，但其在复杂的驾驶视觉问答（VQA）任务上能力不足。
- **为什么重要**：自动驾驶是安全关键型任务，需要高可靠性和低延迟。在车辆上部署大型模型不切实际。因此，如何将大型模型的知识高效地迁移到小型模型，使其在保持高性能的同时实现高效部署，是推动VLM在自动驾驶中实际应用的关键。
- **挑战在哪里**：
    1.  **蒸馏信号选择**：现有的知识蒸馏方法缺乏对自动驾驶任务中蒸馏信号（如输出层、隐藏层、注意力层）的系统性研究，不知道在何处施加监督最有效。
    2.  **能力分解与依赖**：自动驾驶涉及感知、推理、规划等多个相互依赖的能力。现有工作常将它们视为并行目标，忽略了其顺序依赖性，导致蒸馏效果不佳。
    3.  **多目标优化冲突**：当联合优化多个蒸馏目标时，简单的加权求和会导致梯度冲突，即提升一个能力可能会损害另一个能力。这在安全关键的自动驾驶中是不可接受的。

## 2. 创新点（Innovation）

1.  **系统性蒸馏设计研究**：首次对自动驾驶VLM的蒸馏设计进行了系统性的预研究。通过分析层间相似性、能力内一致性、隐藏状态与注意力的离散度，以及输出分布对齐的局限性，为后续的蒸馏策略提供了理论依据和设计原则。
2.  **能力分解与单教师蒸馏配方**：将自动驾驶能力分解为“感知-推理-规划”的顺序三元组。基于预研究发现，为每个能力设计了专门的单教师蒸馏配方：
    - **感知**：蒸馏第1层的文本到视觉交叉注意力。
    - **推理**：蒸馏中间层的全注意力，并采用层组匹配策略。
    - **规划**：蒸馏倒数第二层的文本到视觉交叉注意力。
3.  **多教师蒸馏框架与非对称梯度投影（AGP）**：将三个单教师模型统一到一个多教师蒸馏框架中。为了解决多目标优化中的梯度冲突，创新性地提出了AGP方法：
    - **第一阶段（能力内）**：在每个能力内部，将硬标签监督梯度作为锚点，蒸馏梯度作为跟随者。只移除跟随者中与锚点冲突的部分，确保蒸馏不会损害基本的学习目标。
    - **第二阶段（能力间）**：将三个能力合并后的梯度进行成对投影，以缓解不同能力之间的梯度冲突，并通过随机排序避免对特定能力的偏向。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Drive-KD的整体架构如图1所示，包含三个主要部分：

1.  **教师模型**：三个大型VLM，分别专门用于蒸馏感知、推理和规划能力。每个教师模型在特定层提供注意力图作为蒸馏信号。
2.  **学生模型**：一个小型VLM，其结构与教师模型类似但规模更小。学生模型接收多模态输入（图像+文本提示），并生成输出。
3.  **蒸馏过程**：学生模型同时从三个教师那里学习。对于每个训练批次，根据其所属的能力类别（感知、推理或规划），使用对应的教师注意力图作为软标签，并结合硬标签监督损失进行优化。AGP模块用于处理多目标梯度冲突。

**图1：Drive-KD多教师蒸馏框架。** 三个教师分别蒸馏不同学生层的注意力：感知/规划使用第一层/倒数第二层的交叉模态注意力；推理使用中间层的分组匹配全注意力。学生也接收输出层的硬标签监督。应用AGP来缓解跨能力梯度冲突。

### 3.2 关键算法与数学推导

**1. 蒸馏信号选择（预研究）**

- **层选择**：通过两个视角选择蒸馏层。
    - **表示视角**：使用相邻层余弦相似度 $S(\ell)$ 和层内视觉-文本余弦相似度 $C(\ell)$。
        $$S(\ell) = \mathbb{E}_{b,i} \left[ \frac{\mathbf{h}^{(\ell)}_{b,i} \cdot \mathbf{h}^{(\ell+1)}_{b,i}}{\|\mathbf{h}^{(\ell)}_{b,i}\|_2 \|\mathbf{h}^{(\ell+1)}_{b,i}\|_2} \right]$$
        $$C^{(\ell)} = \mathbb{E}_{b} \left[ \frac{\bar{\mathbf{v}}^{(\ell)}_b \cdot \bar{\mathbf{t}}^{(\ell)}_b}{\|\bar{\mathbf{v}}^{(\ell)}_b\|_2 \|\bar{\mathbf{t}}^{(\ell)}_b\|_2} \right]$$
        其中 $\bar{\mathbf{v}}^{(\ell)}_b$ 和 $\bar{\mathbf{t}}^{(\ell)}_b$ 分别是第 $\ell$ 层视觉和文本token的均值池化向量。
    - **能力视角**：使用能力内一致性 $\text{intra}^{(\ell)}_c$，衡量同一能力下不同图像的特征中心之间的平均余弦相似度。
        $$\text{intra}^{(\ell)}_c = \frac{2}{G(G-1)} \sum_{i<j} \cos\left( \mathbf{c}^{(\ell)}_{c, \text{img}_i}, \mathbf{c}^{(\ell)}_{c, \text{img}_j} \right)$$

- **信号选择**：比较隐藏状态和注意力的离散度，发现注意力更稳定，因此选择注意力作为蒸馏信号。
    - 隐藏状态离散度：$D_{\text{hid}}(i, j) = 1 - \cos(\mathbf{u}_i, \mathbf{u}_j)$
    - 注意力离散度：$D_{\text{att}}(i, j) = 1 - \cos(\text{vec}(\tilde{\mathbf{A}}_i), \text{vec}(\tilde{\mathbf{A}}_j))$

- **输出分布对齐的局限性**：通过分析教师模型在驾驶数据上的置信度（如广义边际 $\Delta^{(\tau)}_{t, \text{gen}}$），发现其输出比通用数据更分散、置信度更低，因此不适合作为蒸馏信号。

**2. 单教师蒸馏损失**

- **硬标签监督损失**：标准的交叉熵损失。
    $$L_{\text{sup}} = -\frac{1}{|Y_{\text{ans}}|} \sum_{t \in Y_{\text{ans}}} \log p_\theta(y_t | x, y_{<t})$$

- **注意力蒸馏损失**：最小化学生和教师注意力图之间的均方误差（MSE）。
    - **全注意力蒸馏**：
        $$L_{\text{full att}}^{(\ell)} = \text{mean}_{b,i} \left[ \text{mean}_j \left( \bar{A}^{(\ell)}_{s, b, i, j} - \bar{A}^{(\ell)}_{t, b, i, j} \right)^2 \right]$$
    - **文本到视觉注意力蒸馏**：
        $$L_{t-v \text{ att}}^{(\ell)} = \text{mean}_b \left[ \frac{1}{|T_b| |V_b|} \sum_{i \in T_b} \sum_{j \in V_b} \left( \bar{A}^{(\ell)}_{s, b, i, j} - \bar{A}^{(\ell)}_{t, b, i, j} \right)^2 \right]$$

**3. 多教师蒸馏与非对称梯度投影（AGP）**

- **总损失**：
    $$L = w_{\text{ce}} L_{\text{sup}} + \sum_{t \in \{p, r, pl\}} \pi_{c, t} w_t L_t$$

- **AGP 第一阶段（能力内）**：
    - 定义锚点梯度 $\mathbf{g}_{\text{sup}} = \nabla_\theta (w_{\text{ce}} L_{\text{sup}})$ 和跟随者梯度 $\mathbf{g}_c = \nabla_\theta (\sum_{t} \pi_{c, t} w_t L_t)$。
    - 如果 $\mathbf{g}_f^\top \mathbf{g}_a^{(c)} < 0$，则移除冲突部分：
        $$\mathbf{g}_f \leftarrow \mathbf{g}_f - \frac{\mathbf{g}_f^\top \mathbf{g}_a^{(c)}}{\|\mathbf{g}_a^{(c)}\|_2^2} \mathbf{g}_a^{(c)}$$
    - 合并：$\mathbf{g}^{(c)} = \mathbf{g}_a^{(c)} + \mathbf{g}_f$

- **AGP 第二阶段（能力间）**：
    - 对三个能力合并后的梯度 $\mathbf{g}^{(p)}, \mathbf{g}^{(r)}, \mathbf{g}^{(pl)}$ 进行随机排序的成对投影。
    - 如果 $\tilde{\mathbf{g}}^{(c_i)^\top} \mathbf{g}^{(c_j)} < 0$，则：
        $$\tilde{\mathbf{g}}^{(c_i)} \leftarrow \tilde{\mathbf{g}}^{(c_i)} - \frac{\tilde{\mathbf{g}}^{(c_i)^\top} \mathbf{g}^{(c_j)}}{\|\mathbf{g}^{(c_j)}\|_2^2} \mathbf{g}^{(c_j)}$$
    - 最终梯度：$\mathbf{g} = \tilde{\mathbf{g}}^{(p)} + \tilde{\mathbf{g}}^{(r)} + \tilde{\mathbf{g}}^{(pl)}$

### 3.3 模型、训练与数据集

- **模型**：
    - **教师-学生对**：主要实验使用 InternVL3-8B → InternVL3-1B 和 Qwen2.5-VL-7B-Instruct → Qwen2.5-VL-3B-Instruct。
    - **模型规模扩展实验**：使用 InternVL3 系列，教师规模为 {8B, 14B, 38B}，学生规模为 {1B, 2B}。
- **数据集**：
    - **训练集**：一个包含10,500个人工标注的驾驶蒸馏数据集，包含单视图和多视图VQA。数据集按感知、推理、规划三种能力进行了划分。
    - **评估集**：DriveBench，用于评估模型在感知、推理和规划三个维度的能力。
- **训练**：使用标准的监督微调（SFT）作为基线，并与提出的单教师和多教师蒸馏方法进行比较。

### 3.4 实验设置与结果

- **评估指标**：在DriveBench上报告感知、推理、规划三个维度的能力得分（%）和平均得分。同时报告部署指标：峰值GPU内存（GB）、生成吞吐量（tokens/s）和首token延迟（s）。
- **主要结果（表1）**：
    - **单教师蒸馏**：显著优于预训练基线。例如，InternVL3-1B的平均得分从25.53提升至43.47。在规划维度上，蒸馏后的学生模型（52.97）已能与GPT-5.1（51.94）竞争。
    - **多教师蒸馏（Drive-KD）**：进一步提升了性能。InternVL3-1B的平均得分达到44.05，规划得分达到55.51，超越了GPT-5.1。
    - **效率对比**：蒸馏后的InternVL3-1B（4.1GB内存，45.7 tok/s）相比预训练的InternVL3-78B（171.6GB内存，4.0 tok/s），在性能更优的同时，内存减少了约42倍，吞吐量提升了约11.4倍。
- **推理-规划相关性分析（表2）**：Drive-KD的学生模型在图像级别的推理和规划得分之间显示出正相关性（Pearson ~0.3），而GPT-5.1的相关性接近于零。这表明Drive-KD不仅提升了规划分数，还增强了同场景下推理到规划的一致性。
- **模型规模扩展（表3）**：Drive-KD在不同规模的教师-学生对上都有效。更大的教师不一定总能带来更好的学生，最佳教师大小取决于学生容量。
- **消融实验（表4）**：
    - 输出分布对齐（CE+KL）会降低性能。
    - 隐藏状态蒸馏不如注意力蒸馏有效。
    - 在多教师训练中，不使用梯度投影（Multi-teacher）或使用简单投影（G1, G2）效果不佳，而AGP（Ours multi-teacher）取得了最佳的整体性能，尤其是在规划维度。

### 3.5 技术细节

- **层组匹配**：对于推理能力，教师和学生模型的层数可能不同。采用滑动窗口的层组匹配策略，将学生的一个中间层与教师的一组相邻层进行匹配，取教师组内注意力的均值作为目标。
- **在线损失重加权**：在单教师蒸馏中，使用基于梯度幅度的在线损失重加权来稳定多损失训练。
- **教师混合权重**：在多教师蒸馏中，对于属于能力c的批次，使用固定权重 $\pi_{c,t}$ 来混合三个教师的蒸馏损失。例如，对于感知批次，感知教师的权重最高。
- **AGP的随机排序**：在第二阶段，对三个能力梯度进行随机排序，以避免算法对特定能力产生系统性偏向。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **理论贡献**：首次对自动驾驶VLM的蒸馏设计进行了系统性研究，为蒸馏层和信号的选择提供了理论指导，并指出了输出分布对齐在此领域的局限性。
    2.  **方法贡献**：提出了一个创新的、基于能力分解的多教师蒸馏框架（Drive-KD），并设计了非对称梯度投影（AGP）来有效解决多目标优化中的梯度冲突问题。
    3.  **实践贡献**：证明了通过精心设计的蒸馏，小型VLM可以在自动驾驶任务上超越参数量大数十倍的模型，实现了性能与效率的卓越平衡。这为在资源受限的车辆平台上部署高性能VLM提供了可行方案。

- **对未来研究/应用的影响**：
    - **推动高效自动驾驶VLM发展**：为开发更小、更快、更可靠的自动驾驶VLM提供了新范式。
    - **提升系统可解释性**：“感知-推理-规划”的三元组分解比传统的端到端“黑盒”模型更具可解释性。
    - **赋能下游系统**：许多语言引导的驾驶和视觉-语言-动作（VLA）系统都依赖于VLM。Drive-KD通过提升上游VLM的能力和效率，为构建更可靠的闭环系统奠定了基础。

- **局限与开放问题**：
    - **输入模态限制**：当前框架仅处理图像和文本，未利用视频、点云等时序和几何信息。
    - **推理能力瓶颈**：将复杂的多步推理迁移到小模型仍然是最困难的挑战。
    - **评估局限**：结果基于开环的DriveBench评估，不能保证在真实世界或闭环仿真中的性能。作者强调任何蒸馏模型在部署前都必须经过严格的闭环测试和长尾场景测试。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：知识蒸馏就像老师教学生。**
    - **传统蒸馏**：老师（大模型）只告诉学生（小模型）最终答案（输出分布）。但答案可能不准确或太复杂，学生学不好。
    - **Drive-KD**：老师不仅告诉学生答案，还教他解题思路。对于“感知”题，老师会展示“看第一行公式”（第1层注意力）；对于“推理”题，老师会展示“看中间步骤的草稿”（中间层注意力）；对于“规划”题，老师会展示“看最后一步的总结”（倒数第二层注意力）。这样学生学得更扎实。

- **类比：非对称梯度投影（AGP）就像一个聪明的团队协调员。**
    - **问题**：三个老师（感知、推理、规划）同时给学生布置作业，他们的要求有时会冲突（例如，感知老师强调“看清路标”，推理老师强调“思考路标含义”，这两个任务的学习方向可能相反）。
    - **AGP第一阶段（能力内）**：协调员先看每个老师自己的要求。如果某个老师的要求（蒸馏梯度）与“完成作业”这个基本要求（硬标签监督梯度）冲突，协调员会删掉冲突的部分，确保基本作业能完成。
    - **AGP第二阶段（能力间）**：协调员再看三个老师之间的要求。如果感知老师的要求和规划老师的要求冲突，
