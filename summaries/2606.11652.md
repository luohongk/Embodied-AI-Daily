## 0. 概览（Concise Summary）

- **主题（Topic）**：多模态小语言模型（SLM）智能体的工具调用（Tool Calling）能力优化。
- **问题（Problem）**：现有的强化学习（RL）方法在训练多模态小语言模型进行工具调用时，面临两大挑战：1）多模态任务中，工具使用路径多样，难以用单一的、基于精确匹配的奖励信号（如答案是否正确）来有效评估；2）这种稀疏的二元奖励信号无法为模型提供关于中间决策过程（如选择哪个工具）的丰富指导，导致小模型难以有效学习。
- **方法（Method）**：提出了一种名为**输入归因感知策略优化（IAPO）** 的强化学习算法。该方法通过计算学生模型和教师模型在输入组件（如工具定义、用户查询）上的**积分梯度（Integrated Gradients, IG）** 得分，来衡量模型在做出工具调用决策时对不同输入部分的关注度。然后，利用两者IG得分分布的KL散度作为**密集的、过程级的惩罚项**，与最终结果奖励相结合，共同指导策略优化。
- **创新（Innovation）**：1）首次将**输入归因**（Input Attribution）作为密集的监督信号引入到多模态小语言模型的工具调用强化学习训练中。2）提出了基于积分梯度的归因得分计算方法，用于量化不同输入组件对工具调用决策的贡献。3）设计了IAPO算法，通过对齐学生和教师的归因模式，实现了知识蒸馏式的过程级指导。
- **意义（Significance）**：该方法显著提升了小模型（Qwen2.5-VL-3B）在视觉问答任务中调用工具的准确性和鲁棒性，平均准确率提升超过3%。它解决了小模型在复杂多模态任务中难以从稀疏奖励中学习有效工具使用策略的关键问题，为训练更高效、更可靠的轻量级多模态智能体提供了新思路。
- **一句话总结**：本文提出IAPO算法，通过让学生模型模仿教师模型在输入上的归因模式，为小模型的多模态工具调用提供了密集的过程级奖励信号，从而显著提升了其工具选择的准确性和任务表现。

## 1. 研究动机（Motivation）

- **问题与空白**：将强化学习应用于大语言模型（LLM）智能体的工具调用训练已取得进展，但这些方法主要针对**纯文本**和**大模型**。当迁移到**多模态小语言模型（SLM）** 时，现有方法存在根本性缺陷。
    1.  **奖励信号不适用**：现有方法（如ToRL, TRM）的奖励设计依赖于对工具调用轨迹的**精确匹配**或**格式检查**。然而，在多模态任务中，解决同一个问题可能有多种有效的工具调用路径（例如，高亮、遮罩或画框都能突出显示图表区域）。这种“唯一正确答案”的假设不成立，导致奖励信号不准确。
    2.  **奖励信号稀疏且脆弱**：基于最终答案正确与否的二元奖励（0或1）是稀疏的。对于小模型而言，它无法提供关于“为什么选错工具”或“如何改进决策过程”的指导。模型可能偶然选对工具并答对问题，但并未真正学会正确的工具选择策略，导致训练不稳定且泛化能力差。
- **重要性**：小模型因其低成本和低延迟，在实际部署中具有巨大潜力。然而，它们推理和决策能力较弱，更需要精细的、过程级的指导信号。解决多模态SLM的工具调用问题，是推动轻量级、实用化智能体发展的关键。
- **挑战**：如何设计一个既能反映工具调用正确性，又能提供密集、过程级反馈的奖励信号？作者发现，模型在做出正确工具调用时，其内部对输入（特别是工具定义）的**归因**（即输入各部分对输出的贡献度）与错误调用时存在显著差异。这为设计基于内部知识的奖励信号提供了可能。

## 2. 创新点（Innovation）

- **核心创新**：提出了**输入归因感知策略优化（IAPO）** 算法。其核心思想是利用一个更强的教师模型，通过**知识蒸馏**的方式，将教师模型在工具调用决策时的“注意力模式”（即输入归因）迁移给学生模型。
- **具体技术贡献**：
    1.  **基于积分梯度的归因得分**：首次将积分梯度（IG）方法应用于多模态工具调用场景，量化了输入中不同语义块（如工具定义、用户查询、图像）对生成工具调用序列的贡献度。这提供了一个可解释的、反映模型内部决策过程的信号。
    2.  **归因对齐惩罚项**：设计了新的奖励函数，将传统的基于最终结果的奖励（$R_{out}$）与一个**归因对齐惩罚项（$P_A$）** 相结合。$P_A$ 通过计算学生模型和教师模型在输入归因分布上的KL散度来衡量两者决策过程的差异。惩罚学生模型，使其归因模式向教师模型靠拢。
    3.  **两阶段训练流程**：提出一个实用的两阶段训练流程：第一阶段使用标准的GRPO进行“冷启动”，让模型具备基本的工具调用能力；第二阶段再使用IAPO进行精细的归因对齐训练，提升工具选择的准确性。
- **为何奏效**：该方法将稀疏的、结果导向的奖励信号，转化为密集的、过程导向的指导信号。通过强制学生模型模仿教师模型的“思考方式”（即关注哪些输入信息），而不是仅仅模仿其“行为结果”（即调用了哪个工具），从而更有效地传递了工具选择的策略知识，尤其适合能力较弱的小模型。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

IAPO的整体架构如图1所示，主要包含以下步骤：

1.  **策略展开（Policy Rollout）**：学生策略模型（Policy LLM）根据输入（图像I和提示x）生成工具调用序列 $y'$，并执行工具得到编辑后的图像 $I'$，然后基于新的视觉输入生成最终答案 $y$。
2.  **输入归因计算**：对于生成的工具调用序列 $y'$，分别计算学生模型和教师模型的输入归因得分。输入提示 $x$ 被划分为K个语义块（如工具定义T、用户查询R、纯文本P、图像占位符I），如图2所示。
3.  **归因对齐惩罚**：计算学生和教师归因得分分布之间的KL散度，作为惩罚项 $P_A$。
4.  **IAPO奖励计算**：将最终结果奖励 $R_{out}$ 与归因惩罚项 $P_A$ 结合，形成最终的IAPO奖励 $R_{IAPO} = R_{out} - \lambda P_A$。
5.  **策略优化**：使用标准的GRPO算法，基于IAPO奖励更新学生策略模型。

### 3.2 关键算法与数学推导

**1. 积分梯度（IG）得分计算：**

- **目标**：量化输入中每个语义块 $b_k$ 对生成工具调用序列 $y'$ 的贡献。
- **步骤**：
    1.  **输入表示**：输入经过多模态分词器后，得到嵌入矩阵 $E = [e_1, ..., e_S] \in \mathbb{R}^{S \times d}$，其中 $S$ 是输入总长度，$d$ 是隐藏维度。每个语义块 $b_k$ 对应一组token索引 $\Omega_k$。
    2.  **归因目标**：定义工具调用序列的对数概率为归因目标：
        $$s(E) = \log \pi_\theta(y' | E) = \sum_{t=1}^{L'} \log \pi_\theta(y'_t | y'_{<t}, E)$$
        其中 $L'$ 是工具调用序列的长度。
    3.  **梯度计算**：计算目标 $s(E)$ 相对于每个输入token嵌入 $e_i$ 的梯度：
        $$g_i = \frac{\partial s(E)}{\partial e_i}, \quad i = 1, ..., S$$
    4.  **Token级归因**：使用“梯度乘以嵌入”的公式计算每个token的归因得分：
        $$o_i = \| g_i \odot e_i \|_2$$
        其中 $\odot$ 表示逐元素乘法。这个得分衡量了第 $i$ 个token对增加工具调用序列概率的贡献。
    5.  **块级归因**：将属于同一个语义块 $b_k$ 的所有token的归因得分求和，得到块级得分 $O_k = \sum_{i \in \Omega_k} o_i$。

**2. IAPO奖励与优化：**

- **归因对齐惩罚**：将学生和教师的块级归因得分向量 $\mathbf{a} = \{O_a^1, ..., O_a^K\}$ 和 $\mathbf{b} = \{O_b^1, ..., O_b^K\}$ 归一化为概率分布 $\hat{\mathbf{a}}$ 和 $\hat{\mathbf{b}}$。惩罚项定义为两者的KL散度：
    $$P_A^i = D_{KL}(\hat{\mathbf{b}}_i \parallel \hat{\mathbf{a}}_i)$$
- **IAPO奖励**：对于第 $i$ 个响应，其最终奖励为：
    $$R_{IAPO}^i = R_{out}^i - \lambda P_A^i$$
    其中 $\lambda$ 是控制惩罚强度的超参数。
- **优势函数与策略优化**：在GRPO框架内，对一组 $G$ 个响应计算组内相对优势：
    $$A_{IAPO}^i = \frac{R_{IAPO}^i - \text{mean}(\{R_{IAPO}^j\}_{j=1}^G)}{\text{std}(\{R_{IAPO}^j\}_{j=1}^G) + \epsilon}$$
    然后将此优势广播到响应中的所有token，用于标准的GRPO裁剪代理目标函数中，从而更新策略。

### 3.3 模型、训练与数据集

- **模型**：
    - **学生模型（策略）**：Qwen2.5-VL-3B-Instruct。
    - **教师模型**：使用标准GRPO在相同训练集上训练好的Qwen2.5-VL-7B-Instruct模型。
- **训练**：
    - **框架**：VeRL框架。
    - **优化器**：AdamW。
    - **流程**：两阶段训练。第一阶段是短期的冷启动GRPO训练，让模型学会基本调用工具。第二阶段是IAPO主训练阶段。
- **数据集**：
    - **任务**：图表和表格的视觉问答（VQA）。
    - **训练集**：来自ChartQA（图表）和VWTQ、VWTQ_syn、VTabFact（表格）的70%数据。
    - **测试集**：CharXiv、ChartQA（水平/垂直）、VWTQ、VWTQ_syn、VTabFact的剩余30%数据。
- **工具集**：6个视觉编辑工具，分为三类（高亮、遮罩、画框），分别作用于列或行。

### 3.4 实验设置与结果

- **基线方法**：
    - **现成模型**：GPT-4o, Qwen2.5-VL-3B（无RL训练）。
    - **VTool-R1**：使用标准GRPO的多模态工具调用基线。
    - **TRM***：使用GPT-4o作为验证器来检查工具调用正确性。
    - **ToRL***：在最终答案奖励基础上，对失败的工具执行施加-0.5的惩罚。
- **主要结果（表1）**：
    - IAPO在6个测试集上的平均准确率为**55.9%**，显著高于所有基线方法。
    - 相比VTool-R1（52.9%），IAPO提升了**3%**。
    - 相比TRM*（53.4%）和ToRL*（53.8%），IAPO也表现更优，证明了其奖励设计的有效性。
    - 在更具挑战性的**CharXiv**（OOD测试集）上，IAPO的优势更为明显（26.1% vs. 23.1-24.1%），表明其泛化能力更强。
- **案例分析（图3）**：
    - 展示了标准GRPO训练后，3B模型仍存在大量工具误用（30/256）。
    - 验证了IG得分与工具调用正确性高度相关：正确工具的IG得分在训练中逐渐升高，并持续高于错误工具。
    - 7B教师模型的工具误用更少（4/256），且IG得分区分度更高，证明了使用更强教师模型的合理性。
- **训练动态分析（图4）**：
    - IAPO的工具调用率（Tool-call ratio）迅速接近1.0并保持稳定，而VTool-R1则较低且不稳定。
    - IAPO的工具调用成功率（Tool-call success rate）在训练后期急剧上升并接近100%，远超VTool-R1。这表明IAPO先鼓励模型“敢于”使用工具，再通过归因对齐指导其“正确”使用工具。

### 3.5 技术细节

- **提示词分解**：将系统提示词分解为有序的语义块，包括纯文本上下文（P）、用户查询（R）、图像占位符（I）和工具定义（T），如图2所示。这种分解是计算块级归因的基础。
- **两轮交互**：采用两轮交互设置。第一轮生成工具调用并执行，第二轮基于修改后的图像进行推理并给出最终答案。
- **教师模型选择**：消融实验（表2）表明，使用同一模型系列（Qwen2.5-VL）且经过RL训练的7B模型作为教师效果最好。使用更先进但不同系列的模型（如Qwen3-VL-8B）效果反而不佳，可能是由于内部机制差异导致归因模式不兼容。
- **超参数 $\lambda$**：消融实验（表3）显示，较小的惩罚系数（$\lambda=0.1$）效果最佳，过大的系数（$\lambda=1$）会损害性能。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **解决了多模态SLM工具调用的训练难题**：针对现有RL方法奖励信号稀疏、不适用于多模态场景的问题，提出了一个有效的解决方案。
    2.  **引入了新的训练范式**：将“输入归因”作为过程级监督信号，为模型训练提供了更丰富的指导，超越了仅依赖最终结果的传统方法。
    3.  **提供了实用的算法**：IAPO算法设计清晰，两阶段训练流程实用，实验证明了其在多个基准上的有效性。
- **对未来研究/应用的影响**：
    - **算法层面**：IAPO为设计更精细的、基于模型内部状态的RL奖励函数开辟了新方向。未来可以探索其他类型的归因方法或内部表征。
    - **模型层面**：该方法特别适用于小模型，有望推动轻量级、高效的多模态智能体在资源受限场景（如移动设备、边缘计算）中的部署。
    - **应用层面**：在需要复杂视觉推理和工具交互的任务中（如自动化数据分析、图表解读、辅助设计），IAPO训练的智能体将更加可靠和准确。
- **局限与开放问题**：
    - **计算开销**：计算IG得分需要额外的反向传播，增加了训练的计算成本。未来可以探索更高效的归因近似方法。
    - **教师模型依赖**：性能依赖于一个强大的教师模型。如何在没有强教师的情况下进行自训练或弱监督学习是一个开放问题。
    - **多轮交互**：当前方法限于两轮交互。扩展到更复杂的多轮工具使用场景是未来的一个方向。
    - **归因方法的局限性**：IG方法本身也有其局限性，如对积分路径的选择敏感。更鲁棒的归因方法可能进一步提升性能。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“输入归因”？**
    - **通俗解释**：想象一个学生（SLM）在解一道数学题。题目中包含了公式、数字和文字描述。学生最终写下了答案。输入归因就是去分析，这个答案的得出，**主要归功于题目中的哪一部分**？是那个关键公式，还是某个具体数字，或者是文字描述中的提示？如果学生归因正确（比如，正确使用了公式），那么他大概率做对了。如果归因错误（比如，忽略公式，只关注了一个无关数字），那么他很可能做错了。
    - **在本文中**：模型需要决定使用哪个工具（比如“高亮列”还是“遮罩行”）。输入归因就是看，模型在决定调用“高亮列”这个工具时，它的“注意力”主要放在了提示词中的哪个部分。如果它主要关注了“高亮列”这个工具的定义，并且这个工具是正确的，那么IG得分就高。如果它错误地关注了“遮罩行”的定义，IG得分就低。

- **为什么KL散度可以作为惩罚？**
    - **通俗解释**：KL散度衡量的是两个概率分布的“距离”或“差异”。假设教师模型认为“工具A定义”的归因得分是0.8，“工具B定义”是0.2。而学生模型认为“工具A定义”是0.3，“工具B定义”是0.7。那么学生和教师的归因分布差异就很大，KL散度值就高。IAPO会惩罚这种高差异，鼓励学生模型去模仿教师模型的归因模式，即也认为“工具A定义”更重要。这就像老师告诉学生：“你看，我解题时主要看的是这个公式，你也应该多关注它。”

- **类比：学骑自行车**
    - **标准GRPO（稀疏奖励）**：教练只告诉你最终结果“你骑到了终点，奖励你一块糖；你摔倒了，没有糖”。你完全不知道是因为你蹬得太快、龙头没扶稳还是转弯角度
