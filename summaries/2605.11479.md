## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人操作策略的离线评估问题，特别是针对稀疏奖励和有限长度轨迹的场景。
- **问题（Problem）**：在离线评估操作策略时，存在三个核心挑战：1）奖励稀疏，仅提供任务是否完成的二元信号；2）评估轨迹长度有限，导致截断偏差（truncation bias），即无法区分策略失败和人为截断；3）策略常表现出恢复行为，导致任务进展非单调，使得基于累积奖励的标准价值函数估计方法（如TD(0)、蒙特卡洛）不可靠。
- **方法（Method）**：提出一个基于“折扣活性（Discounted Liveness）”的离线策略评估框架。该框架将策略评估重新定义为任务完成问题，并学习一个能反映状态到目标集语义距离的价值函数。方法分为两个阶段：1）利用成功轨迹中的状态计算可靠的价值锚点；2）通过自举（Bootstrapping）机制，将这些锚点值传播到其他状态（尤其是被截断轨迹中的状态），从而修正截断偏差。
- **创新（Innovation）**：1）首次将操作策略的离线评估形式化为一个折扣活性问题，而非传统的累积奖励问题。2）提出一个两阶段的自举评估算法，通过“最小化（min）”算子有效缓解有限长度离线数据中的截断偏差，并能捕捉非单调的任务进展。
- **意义（Significance）**：为现代基于视觉-语言-动作模型（VLA）和扩散策略的通用操作策略提供了一种更可靠、更能反映真实任务进展的离线评估方法。这对于策略的迭代改进、部署前的安全性评估以及策略的在线调整至关重要。
- **一句话总结**：本文提出了一种基于折扣活性公式的离线策略评估方法，通过将价值函数定义为到目标集的语义距离，并利用自举机制修正截断偏差，从而在稀疏奖励和有限轨迹下更准确地评估操作策略的性能。

## 1. 研究动机（Motivation）

- **解决的问题**：现代机器人操作策略（如VLA模型）的离线评估问题。
- **为什么重要**：可靠的策略评估是机器人策略开发与部署流程中的核心环节。它用于指导策略的迭代改进、评估部署就绪度以及进行策略的在线调整。没有准确的评估，就无法判断策略的真实能力，也无法安全地将策略部署到现实世界。
- **挑战在哪里**：
    1.  **稀疏奖励（Sparse Rewards）**：现实世界的操作任务通常只提供任务是否完成的二元信号（成功/失败），缺乏密集的、能反映每一步进展的奖励信号。这使得基于累积奖励的传统强化学习方法难以学习到有意义的梯度。
    2.  **截断偏差（Truncation Bias）**：实际评估中，轨迹长度是有限的（例如，由于人为超时）。这导致无法区分“策略失败（无法完成任务）”和“策略被截断（如果给更多时间，策略本可以成功）”。在离线数据中，所有被截断的轨迹都会被标记为失败，这会导致价值函数系统性低估策略的真实性能。
    3.  **非单调任务进展（Non-monotonic Task Progression）**：现实中的操作策略经常表现出恢复行为（例如，物体滑落后重新抓取）。这意味着任务进展不是单调的，状态到目标的“距离”会先变近（成功抓取），然后变远（物体滑落），再变近（重新抓取）。传统的价值函数假设累积奖励单调增加，无法捕捉这种动态变化。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **问题形式化创新**：将离线策略评估重新定义为**折扣活性问题（Discounted Liveness Problem）**。价值函数不再表示“期望累积奖励”，而是表示状态到目标集的**语义距离**。一个状态的价值越低，表示它离成功完成越近。这个定义天然地适合处理稀疏奖励和非单调进展。
    2.  **算法创新**：提出一个**两阶段自举评估框架（Two-Stage Bootstrapped Evaluation Framework）**。
        - **第一阶段**：识别并计算“锚点状态”的价值。这些状态是成功轨迹中所有状态的后继状态，其价值可以通过折扣活性贝尔曼算子精确计算，因为它们有明确的成功终点。
        - **第二阶段**：使用第一阶段计算出的锚点价值来**自举（Bootstrapping）** 其他状态的价值，特别是那些来自被截断轨迹的状态。核心机制是使用 `min` 算子：`V(s) = (1-γ) + γ * min{l(s), V(s')}`。当一个被截断轨迹中的状态与一个已知的锚点状态在特征空间上相似时，`min` 算子会用锚点状态的价值覆盖掉默认的悲观失败价值（1），从而将成功信号反向传播到失败轨迹中。
- **与现有方案的不同**：
    - 与**TD(0)和蒙特卡洛（MC）** 不同，它们依赖于累积稀疏奖励，无法处理截断偏差和非单调进展。
    - 与**分布式蒙特卡洛（MC-D）** 不同，它虽然能学习回报分布，但同样受限于截断偏差。
    - 本文的方法通过重新定义价值函数和引入自举机制，直接解决了上述根本性问题。
- **为什么能奏效**：`min` 算子起到了“修正滤波器”的作用。它允许乐观的、基于成功经验的价值（来自锚点）覆盖悲观的、基于截断假设的价值（来自超时标签）。这使得价值函数能够从成功轨迹中学习，并“原谅”那些虽然被截断但实际有成功潜力的状态，从而更准确地反映策略的真实能力。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出的方法是一个两阶段框架，用于从离线数据中学习价值函数。

1.  **第一阶段：计算锚点价值（Anchor Value Computation）**
    - 从数据集中筛选出所有成功轨迹（`D'`）。
    - 对于这些成功轨迹中的每一个状态，使用折扣活性贝尔曼算子（公式7）从后往前计算其价值。这些价值被称为“well-defined”价值，因为它们基于观察到的成功结果。
    - 对于数据集中所有其他状态（来自失败轨迹），其价值被初始化为一个悲观值 `V(s) = 1`（表示需要无限步才能成功）。

2.  **第二阶段：自举与价值传播（Bootstrapping and Value Propagation）**
    - 使用第一阶段得到的 `V(s)` 来更新目标函数 `l(s)`。对于状态 `s`，新的目标函数不再是二元稀疏奖励，而是 `l(s) = V(s)`。
    - 然后，对整个数据集应用折扣活性贝尔曼算子（公式11）：`Ṽ(s) = (1-γ) + γ * min{ V(s), Ṽ(s') }`。
    - **核心机制**：`min` 算子。当一个状态 `s` 的下一状态 `s'` 的价值 `Ṽ(s')` 比 `s` 自身的锚点价值 `V(s)` 更小（即更接近成功）时，`min` 算子会选择 `Ṽ(s')`。这允许成功信号从锚点状态向后传播到其前驱状态，即使这些前驱状态位于被截断的轨迹中。反之，如果 `V(s)` 更小，则保留悲观估计。这个过程迭代进行，直到收敛。

### 3.2 关键算法与数学推导

- **活性价值函数定义（公式2）**：
  $$V^{\pi}(s) = \mathbb{E}_{\xi \sim \Xi^{\pi, P}_s} \left[ \min_{k \in \{0,1,...,N\}} l(\xi(k)) \right]$$
  其中 $l(s)$ 是目标函数，$l(s) \le 0$ 表示状态 $s$ 在目标集 $L$ 内。该公式定义了从状态 $s$ 出发的轨迹中，所能达到的“最差”目标值（即离目标最远的点）的期望。

- **折扣活性贝尔曼算子（公式7）**：
  $$\tilde{V}^{\pi}(s) = (1-\gamma) + \gamma \min\{l(s), \tilde{V}^{\pi}(s')\}$$
  这是本文的核心算子。它通过引入折扣因子 $\gamma$ 将价值与“步数”联系起来。$(1-\gamma)$ 项确保了价值在 $[1, -1]$ 范围内。`min` 算子确保了价值函数是保守的（悲观估计），因为它总是取当前状态和下一状态价值中的较小者。

- **目标函数定义（公式8）**：
  $$l(s) = \begin{cases} 1 & \text{if } s \text{ is a not goal state} \\ -1 & \text{if } s \text{ is a goal state} \end{cases}$$
  这是一个简单的二元稀疏奖励函数。

- **自举后的贝尔曼算子（公式11）**：
  $$\tilde{V}^{\pi}(s) = (1-\gamma) + \gamma \min\{ \bar{V}^{\pi}(s), \tilde{V}^{\pi}(s')\}$$
  其中 $\bar{V}^{\pi}(s)$ 是第一阶段计算出的锚点价值。这个公式是第二阶段的核心。

- **命题1**：证明了公式2定义的价值函数满足一个不等式关系，并且任何满足等式关系的固定点都是真实价值的保守（悲观）估计。
- **命题2**：证明了折扣后的贝尔曼算子（公式7）是一个压缩映射，保证了迭代求解的收敛性。

### 3.3 模型、训练与数据集

- **模型**：价值函数由一个神经网络参数化。网络的输入是图像观测的SigLIP2嵌入和本体感受状态的拼接。
- **训练**：使用优先经验回放（Prioritized Experience Replay）进行训练。超参数见附录C。
- **数据集**：
    - **LIBERO-Spatial**：模拟环境，π0 VLA模型，200训练/100测试轨迹，50%成功率。
    - **Square (Robomimic)**：模拟环境，扩散策略，200训练/100测试轨迹，50%成功率。
    - **Cloth Folding**：真实硬件（Franka Panda），人类遥操作演示，150训练/20测试轨迹，50%成功率。

### 3.4 实验设置与结果

- **基线方法**：Ours-NB（无自举的消融）、TD(0)、MC、MC-D。
- **评估指标**：
    - **成功指标（Success Metric）**：在成功轨迹的最后N步中，模型预测的“到成功的步数”小于N的帧数百分比。越高越好。
    - **失败指标（Failure Metric）**：在失败轨迹中，模型预测的“到成功的步数”超过任务标准步数Ns的帧数百分比。越高越好。
    - **复合指标（Composite Metric）**：成功和失败指标的算术平均值。
- **主要结果（图3）**：
    - 在所有三个任务上，本文方法（Ours）在**成功指标**和**复合指标**上均显著优于所有基线。
    - 在**失败指标**上，本文方法略逊于部分基线，但差距不大。
    - 这表明本文方法在提升对成功轨迹的识别能力上效果显著，同时保持了较好的失败识别能力。
- **消融实验**：
    - **自举机制（图3）**：Ours-NB在所有指标上均不如Ours，证明了自举机制的有效性。
    - **数据集大小（图5）**：随着数据集增大，所有方法性能提升，但本文方法始终优于基线，且在数据量小时优势更明显。

### 3.5 技术细节

- **状态重叠与自举**：在连续状态空间中，两个状态不可能完全相同。自举机制依赖于神经网络学习到的特征空间中的相似性。如果来自失败轨迹的状态与来自成功轨迹的锚点状态在特征空间上足够接近，`min` 算子就会触发，将成功信号传播过去。这相当于隐式地构建了一个状态相似性图。
- **`min` 算子的作用**：`min` 算子确保了价值函数始终是当前状态和下一状态价值中的较小者。这有两个作用：1）它保证了价值函数的保守性（悲观性），不会高估策略性能；2）它允许成功信号从后向前传播，因为如果下一状态价值更低（更接近成功），它会“拉低”当前状态的价值。

## 4. 意义与影响（Significance and Impact）

- **解决了什么问题**：解决了在稀疏奖励、有限轨迹长度和非单调任务进展这三个现实挑战下，对机器人操作策略进行可靠离线评估的问题。特别是，它显著减少了由人为超时引起的截断偏差。
- **贡献**：
    1.  提出了一种新的策略评估问题形式化方法（折扣活性）。
    2.  提出了一种能有效缓解截断偏差的两阶段自举算法。
    3.  在模拟和真实硬件任务上验证了方法的有效性，证明了其相对于经典方法的优越性。
- **对未来研究/应用的影响**：
    - 为评估日益复杂的通用机器人策略（如VLA模型）提供了更可靠的工具。
    - 其“活性”视角可能启发其他领域（如自动驾驶、安全关键系统）中类似问题的解决。
    - 自举机制的思想可以推广到其他需要从稀疏、截断数据中学习的问题。
- **局限与开放问题**：
    - **过度乐观**：自举机制可能导致对某些失败轨迹的过度乐观估计，特别是当失败轨迹中的状态与成功轨迹中的状态在特征空间上非常相似时。
    - **分布外依赖**：方法的性能高度依赖于数据集的质量和覆盖范围，在训练分布外可能不可靠。
    - **对无恢复行为的策略效果下降**：当策略本身不展示恢复行为时，自举机制的收益会降低。
    - **未来方向**：如何为状态相似性定义结构化边界以控制过度乐观；如何提高数据效率；如何评估不同策略架构对价值函数性能的影响。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：一个“永不放弃”的考试**
    - **传统方法（TD/MC）**：就像一个老师只看学生最终交卷时的分数（成功/失败）。如果学生因为时间到了（截断）没写完，老师就认为他完全不会。这忽略了学生在考试过程中可能已经接近正确答案，只是需要更多时间。
    - **本文方法**：就像一个老师不仅看最终结果，还看学生整个答题过程。老师会标记出学生已经写对的步骤（锚点状态）。然后，即使一个学生没写完，老师也会看他写的步骤是否与之前某个成功学生的步骤相似。如果相似，老师就会认为这个学生“本可以”成功，并给他一个更积极的评价。`min` 算子就像是老师的一个“纠错”机制，它会用“成功学生”的得分来覆盖“时间到”的惩罚。

- **“活性（Liveness）”是什么？**
    - 在控制理论中，“活性”问题问的是“系统最终能否到达某个目标状态？”。
    - 本文将其用于评估，问的是“从这个状态出发，策略最终能多接近目标？”。
    - 价值函数 `V(s)` 就是对这个“接近程度”的量化。`V(s) = -1` 表示“已经在目标上”，`V(s) = 1` 表示“遥不可及”。中间的数值则表示“还需要多少步”。

- **`min` 算子如何工作？**
    - 想象一条从状态A到B再到C的路径，C是成功状态（`V(C) = -1`）。
    - 传统方法：如果路径被截断在B，`V(B)` 会被设为1（失败）。
    - 本文方法：首先，从C反向计算，`V(B) = (1-γ) + γ * V(C) = 1 - 2γ`，这是一个小于1的值。然后，对于状态A，`V(A) = (1-γ) + γ * min{V(A的锚点值), V(B)}`。如果A的锚点值是1（悲观），`min` 算子会选择 `V(B)`，从而 `V(A)` 也被更新为一个小于1的值。这样，成功的信号就从C传播到了A和B。
