## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）智能体的技能学习与重用。具体属于LLM智能体的记忆增强与程序化决策领域。
- **问题（Problem）**：LLM智能体在决策时通常依赖“即时推理”，即使在重复场景中也会从头开始推理，导致计算冗余、不稳定，且无法有效重用过往经验。现有的记忆方法（如检索增强生成）大多属于“情景记忆”，存储的是历史记录，智能体仍需进行大量推理来理解和使用这些记录，效率低下。
- **方法（Method）**：提出**Skill-Pro**框架。该框架将交互经验转化为可重用的“程序化技能”（Procedural Skills），并引入**非参数化PPO**（Non-Parametric PPO）机制，在不更新LLM参数的前提下，通过“语义梯度”生成技能候选，并通过“PPO门控”进行验证，最后通过“在线评分”维护一个高质量、紧凑的技能池。
- **创新（Innovation）**：1) 形式化了**Skill-MDP**，将被动的情景叙事转化为主动的、可执行的程序化技能。2) 提出了**非参数化PPO**，将PPO的信任域优化思想应用于非参数化的技能演化，实现了无需权重更新的技能学习与优化。3) 实现了**极致的记忆压缩**和**跨任务、跨智能体的高重用率**。
- **意义（Significance）**：证明了高质量的“程序化记忆”比原始的“情景记忆”在长期自主性上更高效。为构建能够自主积累和演化程序化知识、无需频繁参数更新的自进化AI智能体提供了新范式。
- **一句话总结**：Skill-Pro通过非参数化PPO，让LLM智能体像人类一样，从交互经验中自主学习和提炼可重用的“程序化技能”，实现了高效、稳定且高度压缩的经验重用。

## 1. 研究动机（Motivation）

- **问题**：当前LLM智能体在解决序列决策问题时，普遍采用“即时推理”（on-the-fly reasoning）模式。即使在面对相似或重复的场景，它们也会重新进行完整的推理过程，而不是利用过往的成功经验。这导致了巨大的计算冗余和错误累积风险。
- **重要性**：解决这个问题对于提升LLM智能体的**效率**（减少计算开销）、**稳定性**（减少错误）和**长期自主性**（能够从经验中学习和成长）至关重要。一个不能从经验中学习的智能体，其能力是受限的，无法在复杂、长期的任务中持续表现良好。
- **挑战**：作者指出了建立可重用程序化记忆的三个核心障碍：
    1.  **可执行性（C1: Executability）**：交互经验通常是描述过去事件的被动“情景叙事”（如“我做了A，然后B发生了”），而不是可以直接在运行时实例化的主动“决策程序”（如“当情况X发生时，执行步骤A、B、C”）。
    2.  **可重用性（C2: Reusability）**：如何确保存储的程序（技能）在未来任务中能被可靠地调用，并带来一致的性能提升，而不是产生负面效果。
    3.  **非参数化优化（C3: Non-Parametric Optimization）**：如何在**不更新LLM参数**的前提下，从经验中学习和优化这些技能，从而避免参数化方法（如微调）带来的高成本、灾难性遗忘和通用能力下降等问题。

## 2. 创新点（Innovation）

- **核心创新1：程序化技能形式化（Procedural Skill Formalization）**：提出了**Skill-MDP**（技能增强的马尔可夫决策过程），将LLM智能体的决策过程从基于情景叙事的模式，转变为基于可重用程序化技能的模式。技能被形式化为一个三元组 `ω = ⟨I_ω, π_ω, β_ω⟩`，分别定义了激活条件、执行程序和终止条件，确保了技能的可执行性。
- **核心创新2：非参数化PPO（Non-Parametric PPO）**：这是本文最关键的创新。它借鉴了PPO算法的信任域思想，但将其应用于**非参数化的技能文本优化**，而不是参数化的神经网络权重更新。它包含两个关键子模块：
    - **语义梯度（Semantic Gradients）**：从交互轨迹中提取自然语言形式的“梯度”，指示如何修改技能的激活、执行和终止条件。这相当于在文本空间中进行“梯度上升”。
    - **PPO门控（PPO Gate）**：一个基于PPO裁剪代理目标函数的验证机制。它通过计算候选技能在历史轨迹上的“反事实优势”，来评估其是否比旧技能更优，从而过滤掉不可靠或幻觉产生的技能候选，保证了技能演化的稳定性。
- **核心创新3：在线评分维护（Score-Based Maintenance）**：引入一个在线评分机制，根据技能在实际使用中带来的平均优势增益来动态评估其价值。通过定期修剪低分或冗余的技能，维持了一个紧凑、高质量、持续进化的技能池，实现了极致的记忆压缩。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Skill-Pro的整体架构如图2所示，主要分为三个部分：

1.  **Skill-MDP（左侧）**：智能体的决策过程被建模为一个Skill-MDP。在每个决策点，智能体首先根据当前状态 `s_t` 和技能池 `Ω`，通过一个技能选择策略 `µ` 选择一个技能 `ω`。然后，一个冻结的LLM（`π_LLM`）根据当前状态和选中的技能 `ω` 生成原始动作 `a_t`。技能会持续执行多个时间步，直到其终止条件 `β_ω` 被满足。执行完一个完整的轨迹后，轨迹数据 `T` 被存入缓冲区。
2.  **程序化技能（中间）**：技能池中的每个技能都是一个结构化的自然语言程序，包含激活条件、执行步骤和终止条件。技能池通过“精炼”、“生成”和“基于分数的修剪”进行动态管理。
3.  **非参数化PPO（右侧）**：这是技能演化的核心引擎，分为两步：
    - **① 语义梯度**：从一批轨迹中，通过“事后归因”为每个轨迹计算一个结构化语义梯度 `g_i`，然后聚合这些梯度得到稳定的批次级梯度 `ḡ_ω`，并用它来更新旧技能 `ω`，生成候选新技能 `ω'`。
    - **② PPO信任域验证**：使用PPO门控对每个候选技能 `ω'` 进行评估。只有通过了PPO门控验证（即 `J(ω') > 0`）的最佳候选技能，才会被加入到技能池中。

### 3.2 关键算法与数学推导

1.  **Skill-MDP形式化**：
    - 定义：`M_Ω = (S, A, Ω, P, R, γ)`
    - 分层策略：`π_Ω(ω_t, a_t | s_t) = µ(ω_t | s_t, Ω) * π_LLM(a_t | s_t, ω_t)`
    - 优化目标：`max_{π_Ω} E[ Σ_{t=0}^{T} γ^t r_t ]`

2.  **非参数化PPO**：
    - **语义梯度**：
        - 单轨迹梯度：`g_i = ∇_sem(τ_i, ω) = <g_i^(I), g_i^(π), g_i^(β)>`
        - 批次聚合：`ḡ_ω = Aggregate({g_i}_{i=1}^B)`
        - 技能更新：`ω' = ω ⊕ ḡ_ω` (⊕ 表示LLM驱动的文本更新操作)
    - **PPO门控**：
        - 重要性比率：`ρ_t(ω') = π_LLM(a_t | s_t, ω') / π_LLM(a_t | s_t, ω)`
        - 优势估计：`Â_t = G_t - R̄`，其中 `G_t` 是回报，`R̄` 是运行基线。
        - PPO门控函数（验证指标）：
          $$L^{CLIP}(ω') = \hat{E}_{\tau \sim B} \left[ \frac{1}{|\tau|} \sum_{t \in \tau} \min \left( \rho_t(ω') \hat{A}_t, \text{clip}(\rho_t(ω'), 1-\epsilon, 1+\epsilon) \hat{A}_t \right) \right]$$
        - 最佳候选选择：`ω_new = arg max_{ω'} J(ω')`，其中 `J(ω') ≜ L^{CLIP}(ω')`，且要求 `J(ω_new) > 0`。

3.  **在线评分维护**：
    - 技能 `ω` 在轨迹 `τ` 中的增益：`G(ω; τ) = (1/|T_ω(τ)|) * Σ_{t ∈ T_ω(τ)} \tilde{r}_t`，其中 `\tilde{r}_t = r_t - \bar{r}` 是优势信号。
    - 在线评分更新：`Score_{b+1} = G_{b+1} / max(1, N_{b+1})`，其中 `G` 是累积增益，`N` 是调用次数。
    - 修剪规则：移除 `Score(ω) ≤ 0` 的技能和冗余技能，若超出容量则按分数升序修剪。

### 3.3 模型、训练与数据集

- **模型**：使用**冻结的LLM**作为动作策略 `π_LLM`。在TextArena上，使用**Gemma-2-9B**作为学习技能的模型，并在Gemma-3-4B、Qwen3-32B、LLaMA-3.3-70B-Instruct等不同模型上评估技能的可迁移性。在ALFWorld上，使用**Qwen3-32B**。
- **训练**：训练过程是技能池的演化过程。智能体在环境中交互，收集轨迹，然后通过非参数化PPO（语义梯度 + PPO门控）生成和验证新技能，并通过在线评分维护技能池。**LLM本身的参数完全不更新**。
- **数据集/环境**：
    - **ALFWorld**：一个文本和视觉结合的具身智能环境，用于评估跨任务记忆重用。
    - **TextArena**：一个多轮序列决策基准，其中的**Mastermind-v0**游戏被用作主要实验环境，包含不同难度等级（Base, Hard, Extreme），用于评估域内和跨任务重用。
    - **BFCL v4**：伯克利函数调用排行榜，用于评估泛化性。

### 3.4 实验设置与结果

- **实验设置**：主要回答三个研究问题（RQ1-RQ3）。对比基线包括多种记忆增强方法（RAG, Expel, A-MEM, AWM, G-Memory）和推理方法（ReAct, CoT）。评估指标包括**重用率**（域内、跨任务、跨智能体）、**性能**（成功率和平均回报）和**效率**（存储成本、推理成本）。
- **主要结果（RQ1）**：
    - **表1**：Skill-Pro在所有场景下的**重用率**（0.925）远超所有基线（最高0.349）。同时，其**存储成本**极低（仅816 tokens），而基线方法动辄数十万甚至上百万tokens。推理成本（∆ Prompt Tokens/Step）也最低。
    - **表2**：在**性能**方面，Skill-Pro在ALFWorld和Mastermind-v0的几乎所有难度级别上都取得了最佳或接近最佳的成绩，尤其是在ALFWorld的OOD（分布外）任务上达到了0.909的成功率，证明了其强大的泛化能力。
- **消融实验（RQ2）**：
    - **表3**：对Skill-Pro的各个组件进行消融。
        - `w/o Skill`：性能大幅下降（-36.0%），证明技能是核心。
        - `w/o NP-PPO`：重用率和性能均下降，证明演化机制的重要性。
        - `w/o SG`：PPO门控通过率下降30.2%，证明语义梯度对生成高质量候选至关重要。
        - `w/o PPO Gate`：所有候选都被接受，导致训练不稳定（图3），性能下降25.2%，在线评分几乎为0。
        - `w/o Score (FIFO)`：性能下降最严重（-27.6%），在线评分为负，证明基于分数的维护对于保留优秀技能、淘汰劣质技能至关重要。
- **演化与重用分析（RQ3）**：
    - **图4**：展示了两个代表性技能的演化谱系，清晰地显示了技能如何被反复精炼、分支和修剪，最终形成高效、紧凑的技能。
    - **图5**：展示了技能在不同LLM智能体和不同任务难度下的选择分布。结果表明，技能选择模式在不同智能体和难度下保持稳定，证明了Skill-Pro提炼了任务的核心逻辑，具有良好的泛化性。

### 3.5 技术细节

- **技能选择策略 `µ`**：论文实现了两种简单策略：1) **基于相似度**：选择激活条件与当前状态最相似的技能。2) **基于价值**：先选出Top-K个相似技能，再从中选择估计价值 `Q(s, ω)` 最高的技能。论文强调其重点在于技能演化，选择策略可以替换为更高级的方法。
- **语义梯度的生成**：通过LLM分析轨迹，进行“事后归因”。例如，如果技能执行失败，LLM会分析是激活条件不准确（不该激活时激活了）、执行步骤有误还是终止条件设置不当，并给出具体的文本修改建议。
- **PPO门控的计算**：使用历史轨迹和候选技能 `ω'` 重新计算动作概率，得到重要性比率 `ρ_t`。优势 `Â_t` 使用回报减去运行基线来估计，避免了训练一个独立的价值网络。`L^{CLIP}` 的计算完全在文本和数值层面进行，不涉及任何梯度反向传播。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了LLM智能体**经验重用效率低下**的根本问题，将被动的情景记忆提升为主动的程序化记忆。
    - 提出了**非参数化PPO**这一新颖的优化范式，证明了在文本空间中，通过语义梯度和信任域验证，可以像优化神经网络一样优化程序化知识，且无需更新模型参数。
    - 实现了**极致的记忆压缩**和**卓越的跨场景泛化能力**，为构建高效、可扩展的长期自主智能体提供了可行方案。

- **对未来研究/应用的影响**：
    - **AI智能体**：为构建能够持续学习和自我进化的AI智能体提供了核心机制。智能体可以在不进行昂贵微调的情况下，通过交互不断积累和优化自己的“技能库”。
    - **机器人学**：机器人可以从模拟或真实世界的交互中学习可重用的操作技能，并泛化到新的环境和任务中。
    - **软件自动化**：AI可以学习并优化复杂的软件工作流，形成可重用的“技能”，提高自动化效率。
    - **人机协作**：人类可以审查和编辑智能体学到的技能，实现人机协同的知识积累。

- **局限与开放问题**（作者提及）：
    - 当前的技能是**显式**的自然语言表示，而人类的程序化记忆是**隐式**的。未来工作将探索将技能内化为隐式执行模块，以更接近人类智能。
    - 技能选择策略 `µ` 相对简单，未来可以研究更高级的、基于强化学习的选择策略。
    - 语义梯度的生成依赖于LLM的“事后归因”能力，其质量可能受限于LLM本身的推理能力。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：学习骑自行车**
    - **传统LLM智能体（情景记忆）**：就像一个每次骑车前都要回忆一遍《自行车操作手册》的新手。他需要阅读“如何上车”、“如何保持平衡”、“如何转弯”等所有步骤，然后一步一步执行，非常慢且容易出错。
    - **Skill-Pro智能体（程序化记忆）**：就像一个已经学会骑车的老手。他的大脑中已经形成了“骑车”这个**程序化技能**。当他看到自行车时，激活条件（“需要出行”）被触发，执行程序（“跨上车、踩踏板、控制方向”）自动运行，终止条件（“到达目的地”）满足后停止。整个过程流畅、高效，无需有意识地回忆每个步骤。

- **非参数化PPO的通俗解释**：
    - **传统PPO（参数化）**：想象你在训练一个舞者（神经网络）跳舞。你通过调整他身体的每个关节角度（参数）来优化舞步。每次调整都很细微，但需要大量的计算。
    - **非参数化PPO（Skill-Pro）**：想象你是一个舞蹈教练，在指导一个舞者（冻结的LLM）改进一套**舞谱（技能）**。你看了他的表演（轨迹），然后给出文字反馈：“你转圈时手臂应该举得更高，结尾的姿势可以更舒展。”（**语义梯度**）。然后，你让舞者按照新舞谱试跳一次，并评估：“嗯，新舞谱确实比旧的好，没有摔倒，动作更流畅。”（**PPO门控**）。你不需要去调整舞者的肌肉，只需要修改舞谱上的文字。这就是非参数化PPO。

- **PPO门控的作用**：
    - 语义梯度是由LLM生成的，它可能会“脑补”出一些不切实际的修改（幻觉）。PPO门控就像一个**安全审查员**。它会拿着新技能去“复盘”过去的成功和失败案例（历史轨迹），计算如果当时用了新技能，结果会不会更好。只有那些在“复盘”中表现更好的新技能，才会被批准加入技能库。这防止了糟糕的修改破坏智能体已有的能力。
