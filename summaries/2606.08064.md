## 0. 概览（Concise Summary）

- **主题（Topic）**：多智能体强化学习（MARL）、人形机器人控制、合作运动技能、Sim-to-Real迁移。
- **问题（Problem）**：如何让多个人形机器人通过协作，完成一项需要高度协调和实时适应的动态任务——合作长绳跳。该任务要求两个摇绳机器人协同控制一根柔性绳索，并适应不同跳跃者的节奏，现有方法多聚焦于单智能体或无交互场景，难以解决此类多智能体紧密协作问题。
- **方法（Method）**：提出一个名为 **Marope** 的层次化多智能体强化学习框架。该框架包含三个训练阶段：1）**低层**：使用MAPPO算法训练一个去中心化的绳索操控策略，让两个摇绳机器人学会协同摇绳；2）**高层**：训练一个中心化的调度策略，根据跳跃者的节奏动态调整摇绳指令，实现同步；3）**多样性发现**：通过一个基于IPM（积分概率度量）的多样性内在奖励，训练出多种不同风格的跳跃策略，用于增强高层调度策略的泛化能力。
- **创新（Innovation）**：
    1.  首次将多智能体强化学习应用于多个人形机器人的合作长绳跳任务。
    2.  提出层次化框架，将复杂的合作任务解耦为可复用的低层操控技能和自适应的高层协调策略。
    3.  通过多样性内在目标自动发现多样化的跳跃者行为，显著提升了系统对未知合作者的适应能力。
- **意义（Significance）**：将人形机器人运动研究从单智能体、无交互的技能（如跑步、跳跃）拓展到需要多智能体紧密协作的动态场景，为人形机器人在更复杂的社会性任务（如集体表演、协同搬运）中的应用奠定了基础。
- **一句话总结**：本文提出了 Marope 框架，通过层次化多智能体强化学习和多样性策略发现，首次实现了人形机器人在仿真和真实世界中稳定、鲁棒的合作长绳跳。

## 1. 研究动机（Motivation）

- **问题与空白**：人形机器人运动控制研究已取得显著进展，如跑步、跳舞、跑酷等。然而，这些研究大多集中在**单智能体**或**无交互**的任务上。长绳跳是一项典型的**多智能体合作**任务，需要两个摇绳者通过全身运动协同产生连贯的绳索运动，并与跳跃者精确同步。现有方法无法直接解决此类问题。
- **为什么重要**：长绳跳是一项广受欢迎的社会性运动，实现人形机器人参与其中，不仅能验证多智能体协同控制的先进算法，也为未来人形机器人在娱乐、体育、救援等需要团队协作的复杂场景中的应用提供了技术探索。
- **核心挑战**：
    1.  **多智能体协同**：两个摇绳机器人需要基于部分观测，通过全身控制产生协调一致的绳索运动，同时保持自身平衡。
    2.  **柔性物体操控**：绳索是可变形的欠驱动物体，其形态无法直接控制。简单的轨迹回放或开环控制无法应对绳索复杂的动力学特性。
    3.  **精确时序同步**：成功的跳跃依赖于绳索旋转相位与跳跃者起跳相位的精确匹配，微小的相位误差就会导致失败。
    4.  **适应多样性**：在开放世界中，跳跃者的风格（如频率、高度、滞空时间）是未知且多样的，系统需要具备强大的泛化能力来适应不同的合作者。

## 2. 创新点（Innovation）

- **关键创新**：提出了 **Marope** 框架，这是一个用于多个人形机器人合作长绳跳的层次化强化学习框架。
- **与现有方案的不同**：
    - **层次化解耦**：不同于端到端地学习整个任务，Marope 将任务分解为“低层绳索操控”和“高层节奏协调”两个层次。低层策略学习一个可复用的“摇绳”技能，高层策略则学习如何“使用”这个技能来配合跳跃者。这种解耦降低了学习难度，并提高了技能的复用性。
    - **紧凑指令空间**：低层策略的指令不是复杂的绳索轨迹，而是一个紧凑的命令空间（旋转中心、角速度、宽度等），这大大简化了学习问题。
    - **多样性驱动的泛化**：为了应对未知跳跃者，Marope 不是简单地收集更多数据，而是**主动**通过一个基于IPM的内在奖励，训练出一个能产生多种多样跳跃风格的策略。然后，用这些多样化的“虚拟玩家”来训练高层调度策略，使其学会适应各种风格，而不是过拟合于单一风格。
- **为什么能奏效**：
    - 层次化结构让每个子问题更简单、更聚焦。
    - 紧凑指令空间抓住了摇绳任务的核心变量，避免了高维轨迹带来的复杂性。
    - 多样性发现机制是一种有效的数据增强策略，它迫使高层策略学习到一种更鲁棒、更通用的协调策略，而不是记忆特定模式。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Marope 的整体架构如图1所示，是一个三阶段训练流程：

1.  **阶段一：去中心化合作绳索操控（Decentralized Cooperative Rope Manipulation）**
    -   **目标**：预训练一个低层策略 $\pi_{\text{manip}}$，使两个摇绳机器人能够根据指令 $G_{\text{manip}}$ 协同摇绳。
    -   **方法**：采用**集中式训练、分散式执行（CTDE）** 范式的 MAPPO 算法。每个机器人根据其局部观测（包括指令、绳索形态、自身状态）输出动作，但在训练时可以使用全局信息。
    -   **指令设计**：指令 $G_{\text{manip}} = [G_{\text{move}}, G_{\text{rope}}]$，其中 $G_{\text{move}} = [\hat{v}_{\text{lin}}^x, \hat{v}_{\text{lin}}^y, \hat{v}_{\text{ang}}^z]$ 控制摇绳中心的移动，$G_{\text{rope}} = [\hat{h}, \hat{w}, \hat{\omega}]$ 控制绳索的旋转高度、宽度和角速度。这种紧凑设计是关键。

2.  **阶段二：高层调度策略（High-level Scheduling Policy）**
    -   **目标**：训练一个中心化的调度策略 $\pi_{\text{sched}}$，根据跳跃者的节奏动态调整低层策略的指令 $G_{\text{manip}}$，实现同步。
    -   **方法**：$\pi_{\text{sched}}$ 接收跳跃者的节奏信号 $G_{\text{rhythm}}$、绳索状态和跳跃者位置作为输入，输出调整后的 $G_{\text{manip}}$。其核心奖励是让绳索旋转相位 $\bar{\theta}_{\text{rope}}$ 跟踪目标相位 $\hat{\theta}_{\text{rope}}$，该目标相位与跳跃者的跳跃相位 $\phi$ 对齐。

3.  **阶段三：多样化跳跃者发现（Diverse Player Discovery）**
    -   **目标**：生成多种不同风格的跳跃策略，用于增强高层调度策略的泛化能力。
    -   **方法**：训练一个以潜变量 $z$ 为条件的跳跃策略 $\pi_{\text{jump}}(a_t | o_t^{\text{jump}}, z)$。通过最大化一个基于**积分概率度量（IPM）** 的多样性内在奖励，鼓励不同 $z$ 对应的策略产生不同的运动状态分布。这些多样化的策略随后作为“对手”与高层调度策略进行合作训练。

### 3.2 关键算法与数学推导

1.  **绳索操控建模**：
    定义旋转中心 $p_c$ 和旋转轴 $e_r$：
    $$p_c = \Pi_{xy}\left(\frac{p_{\text{base}}^1 + p_{\text{base}}^2}{2}\right) + [0, 0, \hat{h}]^\top$$
    $$e_r = \frac{\Pi_{xy}(p_{\text{base}}^2 - p_{\text{base}}^1)}{\|\Pi_{xy}(p_{\text{base}}^2 - p_{\text{base}}^1)\|_2}$$
    其中 $\Pi_{xy}$ 是水平投影算子，$\hat{h}$ 是目标旋转高度。目标角速度 $\hat{\omega} = \hat{\omega} \cdot e_r$。

2.  **绳索旋转相位计算**：
    为了量化绳索的旋转状态，定义旋转相位 $\bar{\theta}_{\text{rope}}$：
    $$\bar{\theta}_{\text{rope}} = \frac{1}{2\pi N} \sum_{i=1}^N \text{atan2}\left(\text{sgn}(\langle \bar{\omega}, e_r \rangle) \xi_i^y, -\xi_i^z\right)$$
    其中 $\bar{\omega}$ 是通过最小二乘法估计的绳索瞬时角速度，$\xi_i$ 是将绳索点 $p_{\text{rope}}[i]$ 投影到以 $e_r$ 为轴的圆柱坐标系下的坐标。

3.  **多样性内在目标**：
    为了发现多样化的跳跃行为，引入一个基于IPM的多样性目标：
    $$\max_{\pi_{\text{jump}}} \gamma_{\mathcal{F}}\left(p(x, z), p(x)p(z)\right) = \sup_{f \in \mathcal{F}} \mathbb{E}_{z \sim p(z), x \sim \pi_{\text{jump}}(z)} \left[ f(x, z) - \mathbb{E}_{\tilde{z} \sim p(z)} f(x, \tilde{z}) \right]$$
    其中 $x$ 是描述策略访问状态的特征变量（如末端执行器位姿），$z$ 是潜变量，$\mathcal{F}$ 是1-Lipschitz连续函数集。该目标等价于最大化状态分布 $p(x|z)$ 和先验分布 $p(x)$ 之间的Wasserstein-1距离。
    实际实现中，用一个可学习的度量模型 $\phi$ 来近似这个上确界：
    $$\max_{\phi} \mathbb{E}_{z \sim p(z), x \sim \pi_{\text{jump}}(z)} \left[ \phi(x, z) - \mathbb{E}_{\tilde{z} \sim p(z)} \phi(x, \tilde{z}) \right]$$
    $\phi$ 的Lipschitz连续性通过谱归一化（Spectral Normalization）保证。多样性内在奖励 $r_t^{\text{div}} = r_t^{\text{task}} + \beta \phi(x_t, z)$ 用于优化跳跃策略。

### 3.3 模型、训练与数据集

- **模型**：
    - 所有策略（$\pi_{\text{manip}}, \pi_{\text{sched}}, \pi_{\text{jump}}$）和价值网络都是多层感知机（MLP）。
    - $\pi_{\text{manip}}$ 和 $\pi_{\text{jump}}$ 输出高斯分布的动作。
    - $\pi_{\text{sched}}$ 输出Beta分布的动作，以更好地处理有界指令。
- **训练**：
    - **算法**：低层使用 MAPPO，高层和跳跃策略使用 PPO。
    - **框架**：基于 **Isaac Lab** 仿真器进行大规模并行训练。
    - **阶段**：如算法1所示，分三个阶段依次训练：跳跃策略（含多样性）→ 绳索操控策略 → 高层调度策略。训练高层调度时，冻结前两个策略。
- **数据集**：无外部数据集。所有数据通过策略与环境交互在线生成。

### 3.4 实验设置与结果

- **实验设置**：
    - **机器人**：宇树科技 Unitree G1 人形机器人（29个自由度）。
    - **仿真环境**：Isaac Lab，绳索由N个刚性胶囊通过球形关节连接模拟。
    - **真实世界**：使用光学动作捕捉（MoCap）系统获取绳索、机器人和玩家的位置信息。
- **实验结果**：
    1.  **绳索操控（表1）**：Marope 在旋转跟踪误差、宽度跟踪误差、移动跟踪误差和稳定性指标上全面优于基线方法（单智能体、开环、无分段采样）。这证明了去中心化MARL和紧凑指令空间的有效性。
    2.  **玩家协调（表2）**：Marope 在重叠率、相位跟踪误差、玩家跟踪误差和完成率上显著优于无调度策略和无玩家多样性的基线。这证明了高层调度策略和多样性发现机制的必要性。
    3.  **可视化分析（图2）**：
        - 图2a的PCA可视化显示，引入多样性目标后，跳跃策略访问的状态空间（末端执行器位姿）显著扩大，证明了其发现多样行为的能力。
        - 图2b的相位跟踪图显示，绳索旋转相位能够紧密跟踪跳跃者的跳跃相位，证明了高层调度策略的有效同步能力。
    4.  **真实世界部署（图3）**：成功在四种真实场景中部署：机器人-机器人摇绳、机器人-人摇绳、机器人为人摇绳、机器人为机器人摇绳。这证明了方法的鲁棒性和Sim-to-Real迁移能力。

### 3.5 技术细节

- **观测对称性**：为了处理两个摇绳机器人的对称性，在指令中加入了 `sign(i)` 项，使角速度方向相对于机器人自身是对称的。
- **绳索观测**：从绳索上随机采样m个点的位置作为观测，并在训练时随机化采样索引，以增强对真实世界传感器噪声的鲁棒性。
- **Sim-to-Real**：采用了广泛的域随机化技术，包括绳索物理参数（长度、密度、刚度）、机器人物理参数（质量、摩擦、质心）以及观测噪声，以确保策略能成功迁移到真实世界。
- **课程学习**：在训练多样性跳跃策略时，引入课程机制，只有当任务奖励达到一定阈值后才启用多样性内在奖励，防止多样性目标干扰基本跳跃技能的学习。

    - 提出了一个通用的层次化MARL框架，可以推广到其他需要多智能体协调和适应动态环境的任务。
    - 提出了一种基于IPM的多样性发现方法，有效提升了策略在开放世界中对未知合作伙伴的泛化能力。
- **对未来研究/应用的影响**：
    - 为人形机器人在体育、娱乐、协作生产等领域的应用开辟了新道路。

## 5. 澄清与简化（Clarifications and Simplifications）

- **层次化框架类比**：可以把 Marope 想象成一个乐队。低层策略（$\pi_{\text{manip}}$）是乐手，他们经过训练，可以熟练地演奏各自的乐器（摇绳）。高层策略（$\pi_{\text{sched}}$）是指挥，他根据主唱（跳跃者）的节奏，指挥乐手们如何演奏（调整摇绳的速度和位置）。多样性发现则像是让乐队练习与不同风格的主唱合作，从而在任何主唱加入时都能完美配合。
- **紧凑指令空间**：想象一下，要描述一个摇绳动作，你可以详细描述每个时刻每个关节的角度（高维轨迹），也可以简单地说“以每秒2圈的速度，在1米高的位置，以1.5米的宽度摇绳”。后者就是紧凑指令空间，它抓住了任务的核心，而忽略了无关的细节，使得学习更容易。
- **IPM多样性**：可以理解为，我们想让不同潜变量 $z$ 对应的跳跃者“跳得不一样”。IPM就像一个裁判，它专门寻找两个跳跃者之间“最不一样”的地方（比如一个跳得高，一个跳得快），并奖励这种差异。这样，我们就能自动得到各种风格的跳跃者，而不是手动去设计。
- **CTDE**：在训练摇绳机器人时，它们可以互相看到对方的观测和意图（集中式训练），从而学会更好地配合。但在实际执行时，每个机器人只能看到自己的传感器信息（分散式执行），这更符合真实情况。
