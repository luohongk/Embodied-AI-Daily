## 0. 概览（Concise Summary）

- **主题（Topic）**：3D 视觉-语言模型（3D VLM），在线（Online）3D 场景理解，流式视频处理。
- **问题（Problem）**：现有的 3D 大语言模型（3D LMMs）都是离线（Offline）工作的，需要完整的 3D 场景或预定义的视频片段，无法在实时流式视频中进行交互和空间推理。同时，缺乏用于训练在线 3D 语言模型的大规模流式数据。
- **方法（Method）**：提出了 **Stream3D-VLM**，这是首个仅依赖流式视频的在线 3D 空间理解模型。其核心包括：1）利用 LLM 的自回归目标将流式控制建模为下一个词元预测问题；2）提出**视觉-空间特征融合（VSFI）** 模块，将流式 3D 重建模型（StreamVGGT）提供的增量几何先验注入视觉流；3）提出**几何自适应体素压缩（GAVC）** 模块，根据 3D 结构动态压缩视觉词元，降低推理延迟。
- **创新（Innovation）**：1）首次将 3D VLM 从离线扩展到在线流式场景；2）提出 VSFI 模块实现几何与语义特征的在线融合；3）提出 GAVC 模块实现基于 3D 结构的动态词元压缩；4）构建了包含超 100 万 QA 对的大规模流式 3D-语言数据集 Stream3D-1M 和包含 29 个任务的基准 Stream3D-Bench。
- **意义（Significance）**：这项工作填补了 3D LMM 在实时、在线应用（如机器人、AR/VR）中的空白，证明了在流式视频中实现强大 3D 空间推理的可行性，并为该领域提供了数据、基准和方法基础。
- **一句话总结**：本文提出了 Stream3D-VLM，通过增量式几何先验注入和自适应词元压缩，首次实现了在流式视频中进行实时、高效的在线 3D 空间理解与对话。

## 1. 研究动机（Motivation）

- **解决的问题**：现有 3D 大语言模型（3D LMMs）存在一个根本性的局限：它们都是**离线（Offline）** 的。无论是依赖点云、网格等 3D 输入的方法，还是近期依赖 RGB 视频的方法，都需要在模型开始推理和交互之前，获取完整的 3D 场景观测或预定义好的视频片段。这与现实世界中的机器人、AR/VR 眼镜等应用场景严重不符，这些场景要求模型能够从连续的视频流中实时感知和理解 3D 空间，并在任意时刻响应用户的查询。
- **为什么重要**：实现在线 3D 空间理解是推动 3D LMM 走向实际应用的关键一步。它能让机器人实时导航和操作，让 AR 眼镜提供即时、上下文相关的空间信息，从而极大地提升智能体在真实世界中的交互能力和实用性。
- **挑战在哪里**：
    1.  **实时交互**：模型需要自主决定“何时”回答，而不是在每个时间步都生成输出，这需要一种新的流式控制机制。
    2.  **连续 3D 感知**：如何在不依赖昂贵的 3D 传感器（如激光雷达）的情况下，从流式 RGB 视频中持续、增量地提取并融合 3D 几何信息，是一个核心挑战。
    3.  **长上下文效率**：在线推理意味着模型需要处理不断增长的视频帧序列，导致视觉词元数量激增，带来巨大的计算开销和延迟，难以满足实时性要求。
    4.  **数据匮乏**：缺乏大规模、带有精确时间戳的流式 3D-语言数据来训练这类模型。

## 2. 创新点（Innovation）

论文提出了三个关键创新点：

1.  **自回归流式控制机制**：不同于传统模型需要反复处理历史上下文来判断是否回答，本文创新地将“何时回答”这个决策问题，转化为 LLM 原生自回归训练目标中的“下一个词元预测”问题。通过引入 `<SEP>`（继续）和 `<END>`（回答）两个特殊决策词元，模型可以自主地在处理视频帧时决定是保持沉默（生成 `<SEP>`）还是触发回答（生成 `<END>`）。这使得模型能够跳过冗余帧，实现高效的流式交互。

2.  **视觉-空间特征融合（VSFI）模块**：为了在不依赖显式 3D 输入的情况下赋予模型 3D 感知能力，本文提出了 VSFI 模块。该模块利用一个流式前馈 3D 重建模型（StreamVGGT）从视频流中**增量式**地提取与时间对齐的几何先验（包括潜在几何特征和相机参数）。然后，通过交叉注意力机制，将这些几何先验作为键（Key）和值（Value）注入到由标准视觉编码器提取的 2D 视觉特征（作为查询 Query）中，从而生成几何增强的视觉词元。这种设计既摆脱了对稀疏 3D 数据的依赖，又实现了可扩展的 2D 视频训练。

3.  **几何自适应体素压缩（GAVC）模块**：为了解决在线推理中长上下文带来的视觉冗余和延迟问题，本文提出了 GAVC 模块。这是一个即插即用的模块，它利用从 StreamVGGT 获得的深度图和相机参数，将 2D 视觉词元反投影到 3D 空间，形成 3D 体素。然后，通过**空间 K-Means 聚类**将空间上邻近的体素动态地分组，并使用**双重注意力聚合**机制（同时考虑特征相似性和空间邻近性）来聚合每个簇内的特征。这种方法能够根据 3D 结构动态压缩词元，在保留结构完整性的同时显著减少冗余和延迟。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Stream3D-VLM 的整体架构如图 4 所示，其核心是一个处理流式视频的端到端框架。

1.  **输入处理**：流式视频帧按时间顺序输入。每一帧首先通过一个标准的**视觉编码器**（Vision Encoder）提取 2D 视觉词元 $H^{2D}_t$。同时，该帧被送入一个**空间编码器**（Spatial Encoder，即 StreamVGGT），增量式地提取潜在几何词元 $G_t$ 和相机词元 $c_t$。

2.  **特征融合（VSFI）**：几何词元和相机词元通过一个轻量级 MLP 投影到视觉词元的嵌入空间，得到 $H^{3D}_t$。然后，在 VSFI 模块中，以 $H^{2D}_t$ 为查询（Query），$H^{3D}_t$ 为键（Key）和值（Value）进行交叉注意力计算，生成几何增强的视觉词元 $H^f_t$。

3.  **词元压缩（GAVC）**：增强后的视觉词元 $H^f_t$ 被送入 GAVC 模块。该模块利用空间编码器预测的深度图和相机参数，将每个词元反投影到 3D 空间，形成带有 3D 坐标的体素。然后通过空间 K-Means 聚类和双重注意力聚合，将 $N$ 个词元压缩为 $K$ 个（$K < N$）更紧凑的体素特征。

4.  **流式控制与生成**：压缩后的视觉词元与文本词元（用户查询）一起输入到**大语言模型（LLM）** 中。LLM 通过其自回归机制，在流式决策词元（`<SEP>` 或 `<END>`）上进行预测，决定是继续处理下一帧还是开始生成回答。当生成 `<END>` 后，模型开始生成最终的文本回答。

### 3.2 关键算法与数学推导

1.  **流式控制建模**：
    将流式控制视为下一个词元预测问题。训练目标由两部分组成：
    - **流式决策损失** $L_{stream}$：监督特殊词元 `<SEP>` 和 `<END>` 的预测。
    - **标准语言建模损失** $L_{LM}$：监督回答文本的生成。
    总损失函数为：
    $$L = \lambda L_{stream} + L_{LM}$$
    其中，$L_{stream} = \frac{1}{|D|} \sum_{t \in D} CE(p_\theta(y_t | y_{<t}), y_t)$，$D$ 是决策词元集合，$CE$ 是交叉熵损失。

2.  **视觉-空间特征融合（VSFI）**：
    几何增强后的视觉词元 $H^f_t$ 通过交叉注意力计算得到：
    $$H^f_t = \text{softmax}\left( \frac{(W_Q H^{2D}_t)(W_K H^{3D}_t)^\top}{\sqrt{d_k}} \right) (W_V H^{3D}_t)$$
    其中 $W_Q, W_K, W_V$ 是可学习的投影矩阵，$d_k$ 是键的维度。最后通过残差连接保留原始语义：$H^f_t \leftarrow H^f_t + H^{2D}_t$。

3.  **几何自适应体素压缩（GAVC）**：
    - **3D 体素构建**：给定深度图 $D_t$ 和相机内外参 $(K_t, E_t)$，将 2D 图像坐标 $(u_j, v_j)$ 反投影到 3D 位置 $p_{t,j}$：
      $$p_{t,j} = E_t^{-1} \left( D_t(u_j, v_j) K_t^{-1} [u_j, v_j, 1]^\top \right)$$
      然后通过位置编码得到 3D 体素：$v_{t,j} = H^f_{t,j} + PE(p_{t,j})$。
    - **动态聚类**：使用空间 K-Means 将体素集 $V_t = \{(v_{t,j}, p_{t,j})\}_{j=1}^N$ 划分为 $K$ 个簇：
      $$\{C_k\}_{k=1}^K = \text{KMeans}(\{p | (v,p) \in V_t\}, K)$$
    - **双重注意力聚合**：对于每个簇 $C_k$，计算中心特征 $\bar{v}_k$ 和中心坐标 $\bar{p}_k$。然后为每个体素 $v_j$ 计算特征权重 $s^f_j$ 和空间权重 $s^p_j$：
      $$s^f_j = \cos(v_j, \bar{v}_k), \quad s^p_j = \exp\left(-\frac{\|p_j - \bar{p}_k\|^2}{2\sigma_k^2}\right)$$
      最终聚合特征为 $v'_k = \sum_{j \in C_k} w_j v_j$，其中 $w_j = \alpha s^f_j + (1-\alpha) s^p_j$。

### 3.3 模型、训练与数据集

- **模型**：基于 Qwen2.5-VL-3B/7B 作为 LLM 主干，集成 StreamVGGT-1B 作为空间编码器。视觉编码器和空间编码器在训练时冻结，VSFI 模块和 LLM 主干完全可训练。
- **训练**：采用统一的端到端多任务指令微调范式，在混合数据集上训练一个 epoch。优化器为 AdamW，权重衰减 0.03。
- **数据集**：
    - **Stream3D-1M**：作者构建的大规模流式 3D 时空 QA 数据集，包含超过 **100 万** QA 对，来源于 **5.2k** 个 3D 扫描场景（ScanNet, ScanNet++, ARKitScenes）。数据生成流程如图 3 所示，通过规则生成和 VLM 验证结合的方式，覆盖了 5 种认知能力和 3 种时间交互模式。
    - **Stream3D-Bench**：用于评估在线 3D 空间理解的基准，包含 **10,037** 个高质量样本，覆盖 **518** 个视频和 **29** 个任务。其评估粒度包括过去、现在和未来，并要求模型具备响应时间精度。

### 3.4 实验设置与结果

- **评估基准**：Stream3D-Bench（在线），VSI-Bench（离线空间推理），ScanRefer（视觉定位），ScanQA（问答），Scan2Cap（密集描述）。
- **主要结果**：
    - **Stream3D-Bench（表 2）**：Stream3D-VLM-8B 在所有指标（数值、多选、开放式答案）上大幅超越所有基线模型，包括 GPT-4o、GPT-5 等商业模型以及各种开源模型。其回答-时间精度（Answer-Timing Acc.）达到 86.7%，TTFT 仅 62ms，端到端延迟仅 0.39s，显著优于其他方法。
    - **VSI-Bench（表 3）**：尽管是为在线场景设计，Stream3D-VLM-8B 在离线空间推理基准上也取得了 **65.9%** 的平均准确率，超越了所有开源模型和商业模型 Gemini-2.5 Pro，证明了其强大的 3D 空间理解能力。
    - **ScanQA, ScanRefer, Scan2Cap（表 4）**：在传统的 3D 场景理解任务中，Stream3D-VLM 同样取得了最优或接近最优的结果，表明其在线能力并未牺牲离线性能。

### 3.5 技术细节

- **推理流程**：视频以 1 FPS 的帧率流式输入。模型逐帧处理，利用 KV 缓存加速解码，无需显式拼接历史帧。
- **数据生成细节**：核心标注引擎（Core Annotation Engine）计算每帧的元数据，包括物体可见性（通过深度感知遮挡和时序阈值）、相机运动学（路径长度、位移、旋转）和几何测量（距离、方位角）。
- **GAVC 效率**：空间 K-Means 聚类在 GPU 上并行执行，引入的额外延迟可忽略不计。如图 5 所示，即使在 25% 的词元保留率下，GAVC 仍能保持有竞争力的准确率，同时大幅降低延迟。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **解决了在线 3D 空间理解的根本性难题**：首次将 3D LMM 从离线推向在线，使其能够处理流式视频并实时交互。
    2.  **提供了高效的技术方案**：提出的 VSFI 和 GAVC 模块分别解决了在线场景下的几何信息注入和长上下文效率问题，为后续研究提供了可借鉴的范式。
    3.  **填补了数据与基准的空白**：构建的 Stream3D-1M 数据集和 Stream3D-Bench 基准为该领域的研究提供了宝贵的资源和评估标准，将极大推动该方向的发展。

- **对未来研究/应用的影响**：
    - **机器人领域**：使机器人能够像人类一样，在移动过程中实时感知环境，进行导航、抓取和操作。
    - **AR/VR 领域**：为 AR 眼镜提供实时、智能的空间助手，可以回答“我身后的桌子有多大？”、“帮我找一下我的钥匙”等问题。
    - **具身智能**：为构建能够在真实世界中自主探索和交互的通用智能体迈出了关键一步。

- **局限与开放问题**：
    - 论文未明确讨论模型在极端光照、快速运动或严重遮挡等复杂场景下的鲁棒性。
    - 模型依赖于 StreamVGGT 提供的几何先验，其性能上限受限于该重建模型的能力。
    - 当前工作主要聚焦于室内场景，向室外大规模、动态场景的泛化能力有待验证。
    - 模型的“何时回答”决策机制虽然有效，但其决策逻辑的透明度和可解释性可能是一个值得探索的方向。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“在线” vs “离线”**：想象一下，离线模型就像看一部完整的电影，看完后才能回答关于电影的问题。而在线模型就像在看直播，可以随时暂停并提问“刚才那个演员手里拿的是什么？”，模型需要基于已经看过的内容立即回答。
- **流式控制机制**：可以把 LLM 想象成一个有“发言权”的观众。它每看一帧视频，都会在心里默念“继续看”（`<SEP>`）或“我要发言”（`<END>`）。只有当它决定“发言”时，才会开口回答你的问题。这避免了它在每一帧都喋喋不休，节省了计算资源。
- **VSFI 模块**：这就像一个“信息融合器”。模型有两只眼睛，一只是普通的“语义眼”（2D 视觉编码器），看到的是颜色、形状；另一只是“几何眼”（StreamVGGT），能感知深度、距离和 3D 结构。VSFI 模块负责把“几何眼”看到的信息（如“这个物体离我 2 米远”）注入到“语义眼”看到的信息中，让模型既能理解“这是一个杯子”，也能知道“它在我的左前方 1 米处”。
- **GAVC 模块**：这就像一个“智能压缩打包机”。视频每一帧都有很多像素（词元），如果全部保留，数据量会爆炸。GAVC 模块会先根据 3D 空间位置，把空间上靠得很近的像素点（比如组成一个桌面的所有像素）打包成一个“包裹”（体素簇）。然后，它只保留每个“包裹”的摘要信息
