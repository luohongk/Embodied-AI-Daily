## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）生成文本的检测，具体应用于政治领域（英国和瑞典的议会文本）。
- **问题（Problem）**：在议会文本中，使用LLM生成内容的情况日益增多，但缺乏公开披露，这损害了政治透明度和公众信任。目前缺乏针对这一特定领域、且可解释的检测方法。
- **方法（Method）**：提出并应用一个名为ICON（可解释置信度增强感知器）的线性、完全可解释（玻璃盒）文本分类器。该分类器使用LLM时代之前的议会文本（人类撰写）和由LLM生成的对应文本进行训练，然后用于检测近期（2021年后）的议会文本。
- **创新（Innovation）**：
    1.  **首次系统性地评估**了英国和瑞典议会文本中未披露的LLM使用情况。
    2.  使用**完全可解释（玻璃盒）** 的线性分类器，而非传统的“黑盒”神经网络，使得检测结果的原因透明、可追溯，这对于高风险的公共领域至关重要。
    3.  提出了一种**基于摘要生成**的LLM文本生成方法，更贴近议员实际使用LLM的场景（基于要点生成全文）。
- **意义（Significance）**：研究揭示了议会中普遍存在且日益增长的未披露AI使用现象，这直接违反了相关伦理指南，对民主透明度和公众信任构成威胁。其可解释的检测方法为未来在类似高风险领域（如法律、学术）进行负责任的AI使用审计提供了范例。
- **一句话总结**：本文通过一个可解释的文本分类器，发现自2022年以来，英国和瑞典议会文本中未披露的AI生成内容呈稳步增长趋势，且没有任何文本对此进行声明。

## 1. 研究动机（Motivation）

- **解决的问题**：在政治领域，议员们越来越多地使用LLM（如ChatGPT）来撰写议会动议、演讲稿等文本。尽管多个国际组织和政府机构（如各国议会联盟、英国政府）都**强烈建议**披露AI的使用情况，但现实中，这些披露几乎完全缺失。这导致无法评估AI在政治文本中的真实渗透率。
- **重要性**：不披露AI使用会侵蚀公众对政治家的信任。如果选民发现其代表在未声明的情况下使用AI生成政策立场或辩论内容，可能会引发对政治过程真实性和透明度的质疑。此外，LLM可能产生带有偏见、错误或具有政治倾向性的内容，不披露会放大这些风险。
- **挑战**：
    1.  **检测难度**：人类很难准确区分人类撰写和LLM生成的文本（准确率接近随机水平）。
    2.  **领域特异性**：通用的LLM检测器在特定领域（如政治文本）表现不佳。需要针对议会文本的风格和内容进行专门训练。
    3.  **可解释性要求**：在政治这种高风险领域，错误地将人类撰写的文本标记为AI生成（假阳性）可能会严重损害议员的声誉。因此，检测方法必须是可解释的，能够说明“为什么”判定某段文本是AI生成的，以便进行复核和避免误判。
    4.  **语言多样性**：除了英语，还需要处理其他语言（如瑞典语）的检测问题，而针对瑞典语的公开检测工具和数据集非常匮乏。

## 2. 创新点（Innovation）

1.  **首次针对议会文本的LLM使用进行系统性检测**：这是首个（据作者所知）专门研究并量化英国和瑞典议会文本中未披露LLM使用情况的工作。它填补了该领域的研究空白。
2.  **采用完全可解释的“玻璃盒”分类器（ICON）**：与主流的基于深度神经网络的“黑盒”检测器不同，本文使用了线性感知机模型。其决策过程完全透明，可以精确地可视化是哪些词语（unigrams）和短语（bigrams）导致了最终的分类结果。这在需要问责和避免误判的高风险场景中是关键优势。
3.  **创新的LLM文本生成策略**：为了生成训练数据，作者没有简单地让LLM重写或改写文本，而是采用“摘要-生成”两步法：先用LLM为人类撰写的文本生成摘要，再让LLM基于这些摘要生成全文。这种方法模拟了议员可能的使用方式（给出要点，让AI生成完整文本），使得生成的训练数据更贴近真实场景。
4.  **置信度评分机制**：ICON方法不仅给出分类结果，还通过分析分类和（`|s|`）与验证集准确率的关系，提供了一个置信度评分。这使得在分析短文本或边界情况时，可以设置阈值（`T`）来过滤掉低置信度的判断，从而有效降低假阳性率。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心架构是一个**两阶段流程**：
1.  **训练阶段**：
    -   **数据准备**：收集LLM时代前（2014-2020年）的议会文本作为“人类撰写”数据（Class 0）。然后，使用“摘要-生成”方法，利用LLM（Gemini和GPT）基于这些人类文本生成对应的“LLM生成”文本（Class 1）。
    -   **特征提取**：从训练文本中提取n-gram（n=1,2）作为特征，并设置最小出现次数（`c_min`）以避免过拟合。
    -   **模型训练**：使用ICON算法训练一个线性分类器，学习每个n-gram特征的权重（`w_i`）和偏置项（`α`）。
2.  **检测阶段**：
    -   **文本预处理**：对需要检测的“前向测试集”（2021年后的文本）进行同样的分词和预处理。
    -   **分类**：将文本（或段落）输入训练好的ICON分类器，计算分类和`s`。如果`s >= 0`，则判定为LLM生成（Class 1）；否则为人类撰写（Class 0）。
    -   **置信度评估**：根据`s`的绝对值`|s|`，从预计算的置信度直方图中查找对应的置信度`c(|s|)`。可以设置阈值`T`，只接受`|s| >= T`的分类结果。

### 3.2 关键算法与数学推导

ICON分类器的核心是计算一个**分类和（classification sum）**：

$$s = \alpha + \sum_{i=1}^{V} w_i f_i$$

其中：
- `s` 是分类和。
- `α` 是偏置项（bias）。
- `V` 是特征总数。
- `w_i` 是第 `i` 个特征（n-gram）的权重。
- `f_i` 是第 `i` 个特征在待分类文本中出现的次数。

**决策规则**：如果 `s >= 0`，文本被归类为Class 1（LLM生成）；如果 `s < 0`，则归类为Class 0（人类撰写）。

**权重初始化**：
$$w_i = \frac{c_1(i) - c_0(i)}{c_1(i) + c_0(i)}$$
其中 `c_0(i)` 和 `c_1(i)` 分别是特征 `i` 在训练集中Class 0和Class 1中出现的次数。

**偏置初始化**：
$$\alpha = \frac{N_1 - N_0}{N_1 + N_0} \equiv \frac{N_1 - N_0}{N}$$
其中 `N_0` 和 `N_1` 分别是训练集中Class 0和Class 1的文本数量。

**权重更新（训练过程）**：
ICON的训练算法类似于感知机，但基于整个训练集计算特征误差。
首先，计算每个特征 `i` 的误差 `e_i`：
$$e_i = \frac{1}{\gamma_i} \sum_{j=1}^{N} v_{ij} (\hat{C}_j - C_j)$$
其中：
- `γ_i ≡ c_1(i) + c_0(i)` 是特征 `i` 的总出现次数。
- `v_{ij}` 是特征 `i` 在文本 `j` 中出现的次数。
- `C_j` 是文本 `j` 的真实标签（0或1）。
- `\hat{C}_j` 是模型对文本 `j` 的预测标签（0或1）。

然后，更新权重：
$$w_i \leftarrow w_i - \eta e_i$$
其中 `η` 是学习率。

### 3.3 模型、训练与数据集

-   **模型**：ICON（可解释置信度增强感知器），一个线性分类器。
-   **训练数据集（Base Data Set）**：
    -   **来源**：英国议会（Hansard）和瑞典议会（Riksdag）2014-2020年的文本。
    -   **构成**：每个语言各包含1,400篇人类撰写文本和1,400篇LLM生成文本（英国文本中，700篇由Gemini生成，700篇由GPT-5 mini生成；瑞典文本全部由Gemini生成）。
    -   **预处理**：去除HTML标签、URL替换、去除标点、按段落分割、设置最小长度阈值（`k=30` tokens）。
-   **前向测试数据集（Forward Test Data Set）**：
    -   **来源**：2021年至2026年4月的议会文本。
    -   **用途**：用于评估真实世界中LLM的使用率。2021-2022年的数据作为基准，用于测量假阳性率（FP rate）。
-   **训练参数**：通过实验选择最佳参数。对于英国数据：`n_max=2`（使用unigram和bigram），`c_min=30`（特征最少出现30次），`k=30`（段落最少30个tokens）。对于瑞典数据：`n_max=2`，`c_min=10`，`k=30`。

### 3.4 实验设置与结果

-   **实验设置**：
    1.  **全文分类**：将整个议会动议作为一个整体进行分类。
    2.  **段落级分类**：将动议拆分成段落，对每个段落进行分类。如果一个动议中至少有一个段落被判定为LLM生成，则该动议被标记为“包含LLM生成内容”。段落级分类中应用了置信度阈值 `T`（英国 `T=5.0`，瑞典 `T=7.0`）来降低假阳性。
-   **主要结果**：
    -   **分类器性能**：在保留的测试集上，英国分类器的F1分数为0.940，准确率为0.950；瑞典分类器F1分数为0.969，准确率为0.972。假阳性率分别为0.062和0.043。
    -   **全文检测结果（表3）**：自2022年以来，两个议会的检测率都呈上升趋势。在最近的时间段（2026年），英国检测率为2.1%，瑞典为6.5%。2021年的假阳性率极低（每个数据集仅1个误报）。
    -   **段落级检测结果（表4）**：同样呈现上升趋势。在最近的时间段，英国检测率为15.5%，瑞典为9.4%。假阳性率通过设置阈值`T`得到了有效控制（图3）。
    -   **关键发现**：**没有任何一篇被检测出的文本包含任何形式的AI使用声明**。

### 3.5 技术细节

-   **文本生成细节**：使用Gemini 3.1 Flash-Lite进行摘要和全文生成。对于英国文本，额外使用GPT-5 mini生成全文。提示词经过精心设计并手动评估，以确保生成质量（提示词见附录A）。
-   **特征可视化**：论文提供了非常直观的可视化示例（图4和图5）。在文本中，每个单词（unigram）根据其权重被着色（紫色代表正权重，倾向于LLM生成；黄色代表负权重，倾向于人类撰写）。颜色深浅代表权重大小。Bigram特征则用连接两个单词的拱形线表示，位置（上方或下方）和颜色指示其权重方向。
-   **对抗性测试**：作者通过手动替换同义词（如将“United Kingdom”替换为“UK”）和使用在线“人性化”工具，尝试规避检测。结果表明，简单的替换无法改变分类结果，证明了分类器决策依赖于大量特征的组合，而非少数几个关键词。只有通过大幅度的结构性修改（如让LLM重写）才可能规避检测，但这会显著改变文本长度和风格。

## 4. 意义与影响（Significance and Impact）

-   **解决的问题与贡献**：
    -   首次提供了**量化证据**，表明在英瑞两国议会中，未披露的LLM使用是一个真实且日益严重的问题。
    -   提供了一个**可解释、高置信度**的检测工具，使得在政治等高风险领域进行AI使用审计成为可能，避免了“黑盒”模型可能带来的误判风险。
    -   强调了**透明度**在AI治理中的核心地位，并指出当前政治实践中存在的严重合规缺口。
-   **对未来研究/应用的影响**：
    -   该方法可推广到其他国家的议会文本或其他类型的官方文件（如法律文件、政府报告）的检测。
    -   其可解释性为研究LLM生成文本的语言学特征提供了新的视角，有助于理解AI与人类写作风格的差异。
    -   为制定更严格的AI使用披露政策提供了技术基础和实证支持。
-   **局限与开放问题**：
    -   **语言和模型差异**：英语和瑞典语的检测效果可能因LLM在不同语言上的能力差异而不可直接比较。只使用了一个LLM生成瑞典语文本，可能无法完全代表所有LLM的风格。
    -   **时间错位**：训练数据使用的是当前版本的LLM生成的文本，而实际检测的文本可能由更早版本的LLM生成，这可能导致检测偏差。
    -   **“灰色地带”问题**：论文采用二元分类，但现实中存在大量“人机协作”的文本（如人类编辑AI生成的草稿）。如何定位和量化这种混合文本中的AI贡献度，是一个开放问题。
    -   **对抗性攻击**：虽然简单的替换无效，但更复杂的、针对性的对抗性攻击（如使用其他LLM进行风格改写）仍有可能规避检测。未来需要研究更鲁棒的检测方法。

## 5. 澄清与简化（Clarifications and Simplifications）

-   **什么是“玻璃盒”分类器？**
    -   **类比**：想象一个考试阅卷老师。
        -   **黑盒老师**：他只看一眼你的卷子，然后给你一个分数。你问他为什么，他说“凭感觉”。你不知道自己哪里做对了，哪里做错了。
        -   **玻璃盒老师**：他不仅给你分数，还会用红笔在你的卷子上圈出所有正确的点和错误的点，并写下评语：“这里用词很专业，加1分；这个论点逻辑不通，扣2分”。你完全清楚分数是怎么来的。
    -   **本文的ICON分类器就是“玻璃盒老师”**。它不仅能判断一段文本是AI写的还是人写的，还能通过图4和图5那样的可视化方式，精确地告诉你：是哪些词（如“United Kingdom”）和短语（如“work with our allies”）让它做出了“这是AI写的”这个判断。这使得我们可以验证其判断是否合理，避免冤枉好人。

-   **为什么用“摘要-生成”法来制造训练数据？**
    -   **类比**：如果你想训练一个侦探识别伪造的签名，最好的方法是让一个伪造者去模仿真实签名，而不是让伪造者凭空创造一个新签名。
    -   **本文的做法**：议员使用AI时，很可能是先自己构思几个要点，然后让AI写出一篇完整的文章。为了模拟这个过程，作者先让AI为人类写的文章写一个摘要（提取要点），再让AI根据这个摘要“还原”出一篇完整的文章。这样生成的AI文本，在内容和结构上更接近“议员使用AI”的真实场景，训练出的检测器也更能适应这种场景。

-   **什么是置信度阈值（`T`）？**
    -   **类比**：一个天气预报员说“明天有70%的概率下雨”。如果你是个谨慎的人，可能60%的概率你就会带伞。如果你是个乐观的人，可能90%的概率你才带伞。
    -   **本文的阈值`T`**：分类器对每个判断都有一个“信心指数”（`|s|`）。`|s|`越大，它越有把握。为了减少误报（把人类写的当成AI写的），作者设置了一个门槛`T`。只有当分类器的信心指数超过这个门槛时，才采纳它的判断。比如，设置`T=5.0`，意味着只有分类器非常肯定（`|s| >= 5.0`）时，才认为它是AI生成的。这大大降低了冤枉好人的风险，代价是可能会漏掉一些信心不足的AI生成文本。
