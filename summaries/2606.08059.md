## 0. 概览（Concise Summary）

- **主题（Topic）**：人形机器人控制，特别是基于感知的地形自适应行为基础模型。
- **问题（Problem）**：现有的人形机器人行为基础模型假设参考运动（如人类动作）与机器人所处环境是物理兼容的。然而，当演示者、操作员和机器人处于不同环境时，人类动作只表达了意图和风格，并未指定机器人在其局部地形上所需的落脚点、摆动高度、身体姿态和接触时机，导致“操作者-环境不匹配”问题。
- **方法（Method）**：提出**感知行为基础模型（Perceptive BFM）**。该方法包含一个离线**地形一致参考合成（TCRS）**模块，将原始人类运动剪辑转换为地形兼容的参考轨迹；然后通过**感知运动跟踪（PMT）**训练算法，先训练一个跟踪合成参考的“盲”教师策略，再通过**目标帧动作对齐**，将教师的地形适应行为蒸馏给一个接收原始参考和地形视觉输入的“学生”策略。学生策略采用**身份门控残差**架构，初始化为原始参考跟踪器，仅在学习到必要的地形修正时才激活感知通路。
- **创新（Innovation）**：1）提出了TCRS，一个可扩展的离线合成管线，能生成地形一致的监督信号。2）提出了PMT，一种保持原始运动命令接口不变的教师-学生训练算法。3）设计了身份门控Transformer策略，实现了单一策略在多种地形上对多样化人类运动命令的感知接地。
- **意义（Significance）**：解决了将人类运动先验知识应用于机器人自身环境的关键障碍，使得一个统一的、基于原始运动命令的接口能够适应复杂多变的地形，极大地提升了人形机器人的通用性和实用性。
- **一句话总结**：本文提出一种感知行为基础模型，通过离线合成地形一致参考和教师-学生蒸馏训练，使单一的人形机器人控制策略能根据自身感知的地形信息，自适应地执行来自人类的各种运动命令。

## 1. 研究动机（Motivation）

- **研究动机**：当前的人形机器人控制正从针对特定任务的技能向通用行为基础模型发展。这些模型能够学习广泛的人类运动先验，使得机器人可以复现多样化的动作。然而，这些模型普遍存在一个隐藏的假设：参考运动（如人类动作捕捉数据）与机器人所处的环境是物理兼容的。
- **问题与重要性**：这个假设在现实世界中常常不成立，即存在“操作者-环境不匹配”问题。例如：
    - 一个在平地上录制的行走参考，没有指定在楼梯上的落脚点。
    - 远程操作员在控制室发出的指令，无法编码远端机器人面临的稀疏支撑或凹陷地形。
    - 一个在干净地板上演示的动作，没有告诉机器人在崎岖地形上需要多大的摆动间隙来避开障碍物。
    - 参考运动传达了意图和风格，但可能不是机器人在其局部世界中有效的轨迹。
- **挑战**：如何将人类提供的、与地形无关的原始运动命令（如关节位置序列），与机器人通过自身感知获得的地形信息（如高度图）结合起来，生成既忠实于原始行为意图又在地形上可行的全身动作。现有工作要么擅长跟踪动作但不考虑地形，要么擅长感知地形但只执行特定的穿越技能，无法保留任意的人类运动命令。核心挑战在于**行为先验的感知接地**：利用机器人中心的地形感知来重新解释一个人类运动命令在物理上需要什么。

## 2. 创新点（Innovation）

本文的核心创新在于提出了一个完整的框架，将人类运动先验与机器人中心的地形感知无缝结合，同时保持原始运动命令作为部署接口。具体创新点如下：

1.  **地形一致参考合成（TCRS）**：
    - **新方法**：提出一个可扩展的离线合成管线。它不依赖物理仿真，而是通过一系列几何和运动学优化步骤，将原始的人类运动剪辑和采样的地形高度场转换为地形一致的机器人参考轨迹。
    - **与现有方案不同**：现有方法要么直接使用原始参考（忽略地形），要么使用复杂的在线轨迹优化或运动匹配来选择/生成新动作。TCRS是在离线状态下，以保留原始动作风格和意图为前提，对参考轨迹进行“修正”，使其适应地形。它专门处理了接触感知的落脚点构建、考虑脚部几何形状的摆动优化、支撑感知的身体高度重建、碰撞修复和多点逆运动学（IK）等关键问题。
    - **为何奏效**：通过将地形适应问题分解为一系列可解的几何优化子问题，TCRS能够生成高质量、物理上更合理的监督信号，为后续的策略学习提供了坚实的基础，避免了策略从零开始学习复杂的地形-动作映射。

2.  **感知运动跟踪（PMT）训练算法**：
    - **新方法**：提出一个三阶段的教师-学生训练范式。首先，一个“盲”教师策略（仅接收本体感受和TCRS合成参考）学习跟踪地形一致的参考。然后，一个视觉学生策略（接收原始参考和地形感知输入）通过**目标帧动作对齐**来模仿教师的行为。
    - **与现有方案不同**：关键区别在于命令接口。教师跟踪的是“修正后”的参考（$q^{tcrs}$），而学生跟踪的是“原始”的参考（$q^{raw}$）。通过目标帧动作对齐（$a^*_t = (q^{tcrs}_t + \mu^{tea}_t) - q^{raw}_t$），学生学会在原始命令的坐标系下表达教师的地形适应行为。这保证了部署时，用户只需提供原始运动命令，无需关心地形。
    - **为何奏效**：这种设计将“跟踪什么”和“如何适应”分离开来。教师负责学习如何跟踪一个复杂的地形适应轨迹，学生则学习如何通过感知信息，在原始命令的基础上产生一个“修正量”。这大大简化了学习问题，并保证了最终策略的通用性。

3.  **身份门控Transformer策略架构**：
    - **新方法**：设计了一个带有身份门控残差路径的Transformer策略。运动跟踪主干网络处理命令和本体感受历史，而地形感知信息通过两个初始化为零的残差路径（意图门控和动作残差）注入。
    - **与现有方案不同**：在初始化时，由于残差路径输出为零，该策略的行为完全等同于一个纯粹的原始参考跟踪器。只有在训练过程中，当感知信息有助于改善跟踪目标时，这些残差路径才会被激活，学习产生地形修正。
    - **为何奏效**：这种设计确保了策略的初始化状态是稳定的（即能很好地跟踪原始命令），并且感知信息只作为“修正”而非“替代”来发挥作用。这避免了感知噪声或错误导致策略行为发生剧烈、不期望的变化，使得学习过程更稳定、更高效。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Perceptive BFM的整体架构是一个分阶段的训练和部署流程。

1.  **离线数据生成（TCRS）**：对于每个原始运动剪辑 $m^{raw}$ 和采样的地形 $\tau$，TCRS模块离线合成一个地形一致的参考轨迹 $m^{tcrs}$。这个阶段完全在仿真外进行，不涉及策略学习。
2.  **教师策略训练**：一个盲的Transformer教师策略 $\pi_T$ 通过PPO强化学习来跟踪 $m^{tcrs}$。它只接收本体感受和 $m^{tcrs}$ 作为命令，不接收视觉信息。
3.  **学生策略蒸馏**：一个视觉学生策略 $\pi_S$ 接收原始参考 $m^{raw}$ 和地形视觉输入 $o^{vis}$。通过目标帧动作对齐，学生策略被训练去模仿教师策略的有效动作。在蒸馏过程中，教师和学生的控制概率从1退火到0。
4.  **学生策略微调**：学生策略在原始参考命令和地形高度图观测下，通过PPO进行微调。在此阶段，身份门控残差路径被训练来学习感知条件修正。
5.  **部署**：最终部署时，只使用学生策略 $\pi_S$。它接收原始运动命令 $m^{raw}$、机器人本体感受 $o^{prop}$ 和局部地形观测 $o^{vis}$，输出关节位置目标 $q^{pd}_t = q^{raw}_t + a_t$，其中 $a_t$ 是策略输出的残差动作。

### 3.2 关键算法与数学推导

1.  **问题形式化**：
    感知运动跟踪策略 $\pi_\theta$ 定义为：
    $$a_t = \pi_\theta(o^{prop}_t, o^{vis}_t, m^{raw}_{t:t+H})$$
    策略输出残差关节位置目标：
    $$q^{pd}_t = q^{cmd}_t + a_t$$
    其中，对于教师 $q^{cmd}_t = q^{tcrs}_t$，对于学生 $q^{cmd}_t = q^{raw}_t$。

2.  **地形一致参考合成（TCRS）**：
    - **足部几何感知摆动优化**：在虚拟的“中足”坐标系中规划摆动轨迹。定义中足位置 $p^{mid}_{f,t} = p^{ankle}_{f,t} + R_{f,t} r_{mid}$，其中 $r_{mid} = \frac{1}{2}(r_{toe} + r_{heel})$。
      摆动轨迹优化目标函数 $J_s(Y; \tau)$ 为：
      $$J_s(Y;\tau) = \lambda_{ref} \sum_k \|y_k - y^{raw}_k\|^2 + \lambda_{sm} \sum_k \|\Delta^2 y_k\|^2 + \lambda_{clr} \sum_k [h_\tau(x_k, y_k) + \delta - z_k]^2_+ + \lambda_{edge} \Phi_{edge}(Y;\tau) + \lambda_{end}\|y_1 - \bar{y}_1\|^2 + \lambda_{end}\|y_K - \bar{y}_K\|^2$$
      其中 $[x]_+ = \max(0, x)$，$h_\tau$ 是地形高度场，$\delta$ 是安全间隙，$\Phi_{edge}$ 惩罚垂直面附近的穿透。
      优化采用基于采样的轨迹优化（类似MPPI）：
      $$Y \leftarrow Y + \sum_j \frac{\exp(-J_s(Y+\epsilon^{(j)})/\eta)}{\sum_\ell \exp(-J_s(Y+\epsilon^{(\ell)})/\eta)} \epsilon^{(j)}$$

    - **支撑感知身体高度重建**：根据支撑脚的位置和地形高度，计算目标身体高度 $z^*_{root,t}$：
      $$z^*_{root,t} = \frac{\sum_f w_{f,t} \left[ h_\tau(x^{tcrs}_{f,t}, y^{tcrs}_{f,t}) + z^{raw}_{root,t} - z^{raw}_{f,t} \right]}{\sum_f w_{f,t} + \epsilon}$$
      其中 $w_{f,t}$ 是支撑权重。

    - **多点腿逆运动学（IK）**：求解一个阻尼最小二乘IK问题，同时优化踝、趾、跟三个点的位置误差：
      $$\Delta q^{leg,*} = \arg \min_{\Delta q^{leg}} \sum_{f \in \{L,R\}} \sum_{p \in \mathcal{P}} W_{f,p} \| J_{f,p} \Delta q^{leg} - e_{f,p} \|^2 + \lambda_{post} \| q^{leg} - q^{raw,leg} \|^2 + \lambda_{cont} \| q^{leg} - q^{prev,leg} \|^2 + \lambda_{pen} \Psi_{pen}(q^{leg}) + \lambda_{dls} \| \Delta q^{leg} \|^2$$

3.  **目标帧动作对齐**：
    蒸馏时，学生模仿的目标动作 $a^*_t$ 是教师的有效PD目标相对于原始参考的偏移：
    $$a^*_t = (q^{tcrs}_t + \mu^{tea}_t) - q^{raw}_t$$
    学生最小化 $\| \mu^{stu}_t - a^*_t \|^2_2$。

4.  **身份门控残差**：
    意图潜在变量调制：$u'_t = u_t + \tanh(\alpha_u) \odot f_u(z^{vis}_t)$
    动作均值：$\mu_t = \mu^{base}_t + \tanh(\alpha_a) \odot f_a([o^{prop}_t, z^{vis}_t])$
    其中 $\alpha_u$ 和 $\alpha_a$ 是初始化为零的门控向量，$f_u$ 和 $f_a$ 是残差网络。

### 3.3 模型、训练与数据集

- **模型架构**：
    - **教师策略**：基于Transformer的Actor-Critic网络，处理token化的本体感受历史和参考命令窗口。
    - **学生策略**：继承教师的Transformer主干，并增加一个高度图编码器。通过身份门控残差路径将视觉潜在变量注入到意图潜在变量和动作均值中。
- **训练算法**：采用分阶段的PMT训练算法（Algorithm 1）。
    1.  **TCRS数据生成**：使用Algorithm 2合成配对数据 $(m^{raw}, m^{tcrs}, \tau)$。
    2.  **教师PPO训练**：在合成参考 $m^{tcrs}$ 上训练盲教师策略。
    3.  **目标帧蒸馏**：使用教师的目标帧标签蒸馏学生策略。
    4.  **视觉PPO微调**：在原始参考命令和高度图观测下微调学生策略。
- **数据集**：
    - **运动数据**：面向运动的人类运动剪辑（locomotion-oriented human motion clips）。
    - **地形数据**：程序化生成的地形，包括高度偏移（-0.10 到 0.25 m）、支撑宽度（0.10 到 1.00 m）、楼梯（台阶5-15 cm，踏板20-35 cm）、斜坡（最高30°）等。
- **观测空间**：包括本体感受（重力投影、关节状态等）、命令窗口（未来参考关节位置/速度、锚点位移）、视觉输入（17x11的归一化高度图及有效性掩码）。教师和批评家使用特权信息（如真实身体姿态），但部署的学生策略不使用。
- **奖励函数**：PPO奖励结合了指数跟踪项（锚点位置、身体朝向、脚部位置/速度）和惩罚项（能量消耗、足/胫侧向接触）。

### 3.4 实验设置与结果

- **实验设置**：在48块A800 GPU上训练10k次迭代。所有变体共享相同的任务、奖励和观测合同。主要运行从约5k的检查点恢复。
- **TCRS质量评估（表1）**：在“阶梯石（STEPPING STONES WITH STAIRS）”地形上评估TCRS输出的参考轨迹质量。与简单的Z轴偏移和三次插值+IK基线相比，TCRS在碰撞敏感指标上显著领先，穿透深度降低56.6%（5.48→2.38 cm），间隙违反率降低48.3%（14.3→7.4%）。这证明了TCRS能生成更物理可行、无碰撞的参考轨迹。
- **训练时间消融实验（图4）**：通过训练奖励诊断不同架构组件的效果。
    - **完整PMT模型**达到最高平均奖励（54.6）。
    - **移除视觉感知**（盲变体）导致奖励急剧下降至3.6，证实了感知对于地形接地至关重要。
    - **替换Transformer主干**（如MLP, GRU, CNN）导致奖励下降5-8点，证明了Transformer架构的优势。
    - **移除目标帧蒸馏**导致奖励下降约4.5点，证明了该对齐方法的有效性。
- **真实机器人部署（图1，图5）**：展示了单一策略在多种行为-地形组合上的定性结果，包括单腿后空翻、楼梯后空翻、楼梯舞蹈、挥手跑、侧向走楼梯、后退过障碍、楼梯转弯等。特别展示了“操作者-环境不匹配”场景：人类在平地上进行动作捕捉，机器人则在随机放置的地形上执行该命令。

### 3.5 技术细节

- **TCRS实现细节**：TCRS保留原始参考的根朝向和上半身关节，只通过多点IK更新12个腿部关节。这使得地形适应主要成为下半身的接触调整，同时限制了命令式全身风格的扭曲。
- **身份门控初始化**：所有门控向量和最终残差层初始化为零，确保感知通路在初始化时无效，学生策略从原始参考跟踪器开始学习。
- **蒸馏过程**：采用DAgger风格，教师控制概率从1退火到0。当教师控制仿真器时，应用的是对齐后的目标帧动作，而非教师自身的原生残差。
- **微调策略**：PPO微调时，对继承的骨干网络使用较低的学习率（0.3倍），而对地形编码器、批评家和残差分支使用较高学习率，以保守地更新运动跟踪先验。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了人形机器人领域一个关键但常被忽视的问题：**操作者-环境不匹配**。它使得机器人能够将人类提供的、与环境无关的运动命令，通过自身感知，安全、有效地部署在复杂多变的真实地形上。
    - 提出了一个完整的、模块化的解决方案（TCRS + PMT + 身份门控架构），为后续研究提供了一个强大的基线框架。
    - 证明了**机器人中心的感知**可以将人类运动先验转化为地形兼容的全身行为，而无需改变原始的运动命令接口。这极大地简化了人形机器人的部署和用户交互。

- **对未来研究/应用的影响**：
    - **人
