## 0. 概览（Concise Summary）

- **主题（Topic）**：多目标贝叶斯优化（MOBO），特别是如何利用大型语言模型（LLM）的先验知识来提升优化效率。
- **问题（Problem）**：LLM 提供的建议和置信度不是经过校准的观测值。在多目标优化中，LLM 专家可能对某个目标有用，对另一个目标却有害，并且可能过度自信。直接将 LLM 输出作为固定先验会引入偏差，而忽略其信息又会浪费潜在价值。
- **方法（Method）**：提出 CALIPER 框架，一种基于证据门控的 LLM 先验层。它将多个 LLM 专家角色的输出转化为目标维度的残差先验均值函数。通过一个“声誉市场”在线学习专家-目标对的信任度，并使用解耦的反事实门控机制来决定是否使用先验、是否使用置信度，或完全放弃先验。
- **创新（Innovation）**：
    1.  将 LLM 输出形式化为可被观测反馈检验的、目标维度的残差先验。
    2.  提出 CALIPER，结合了目标维度声誉市场、确定性反事实回放和显式的先验放弃机制。
    3.  设计为与现有强 MOBO 采集函数（如 qLogNEHVI）兼容的先验层，而非替换它们。
- **意义（Significance）**：为在贝叶斯优化中安全、有效地利用 LLM 知识提供了一种新范式。它解决了 LLM 作为“黑盒”建议者时不可靠、不校准的核心问题，将 LLM 的角色从“优化器”或“固定建议引擎”转变为“可被检验的贝叶斯先验源”，提升了 MOBO 在低数据量下的鲁棒性和性能。
- **一句话总结**：本文提出 CALIPER，一种通过在线学习专家信任度和反事实门控来校准 LLM 先验，从而安全地增强多目标贝叶斯优化的方法。

## 1. 研究动机（Motivation）

**研究动机**：贝叶斯优化（BO）是优化昂贵黑盒函数的有效框架，但其在数据稀疏的早期阶段性能受限。大型语言模型（LLM）拥有丰富的领域知识，可以作为先验信息来加速 BO 的搜索过程。然而，直接将 LLM 的输出（如评分和置信度）作为先验存在严重问题：

1.  **未校准性（Uncalibrated）**：LLM 的评分不是经过校准的观测值，其自报告的置信度也未必反映真实的预测准确性。一个 LLM 专家可能对某个目标（如溶解度）预测准确，但对另一个目标（如类药性）预测错误。
2.  **多目标冲突（Multi-Objective Conflict）**：在多目标 BO（MOBO）中，问题更加突出。一个全局的信任权重会掩盖专家在不同目标上的表现差异。一个固定的置信度规则可能对某个目标有益，却对另一个目标有害。
3.  **过度自信（Overconfidence）**：LLM 可能对其错误的预测表现出高度自信，这会误导优化过程。

因此，核心挑战在于：**如何将 LLM 这种不确定、可能带有偏差的先验知识，以一种可检验、可校准的方式安全地融入 MOBO 框架，使其在有用时发挥作用，在有害时能被抑制或忽略。**

## 2. 创新点（Innovation）

论文的关键创新在于提出了 **CALIPER** 框架，其核心思想是将 LLM 作为“可被检验的贝叶斯先验源”，而非“固定的建议引擎”。主要创新点如下：

1.  **目标维度的残差先验（Objective-wise Residual Priors）**：将多个 LLM 专家角色的输出（评分和置信度）转化为每个优化目标的残差先验均值函数。这意味着 BO 的代理模型（高斯过程）不是直接拟合原始目标值，而是拟合**真实目标值与 LLM 先验预测值之间的残差**。这允许 LLM 先验提供结构，而 GP 负责修正其偏差。

2.  **目标维度声誉市场（Objective-wise Reputation Market）**：为每个“专家-目标”对维护一个独立的“声誉资本”（$K_{ej}$）。该资本基于专家预测与真实观测值之间的误差进行在线更新。通过一个带温度的 Softmax 函数将资本转化为相对信任权重。这使得模型可以学习到“专家 A 对目标 1 可靠，但对目标 2 不可靠”这种细粒度信息。

3.  **解耦的反事实门控（Decoupled Counterfactual Gate）**：这是 CALIPER 的核心机制。它不只是一个简单的置信度乘子，而是将决策分解为两个独立的门控：
    -   **先验门（Prior Gate）**：决定是否使用 LLM 先验。它通过比较三种策略（使用无置信度的先验、使用带置信度的先验、放弃先验）在历史数据上的边际似然，动态选择最优策略。这使得模型可以显式地“放弃”一个有害的先验。
    -   **更新门（Update Gate）**：决定 LLM 的置信度是否应该影响声誉市场的更新。它通过维护两个“影子市场”（一个使用置信度，一个不使用）并比较它们的预测损失，用 Hedge 算法在线学习最优策略。

4.  **采集函数无关的先验层（Acquisition-Agnostic Prior Layer）**：CALIPER 被设计为一个“先验层”，它只负责修改 GP 的后验均值（通过加上先验），而不改变后验方差。因此，它可以无缝地插入到任何标准的 MOBO 采集函数（如 qLogNEHVI, qLogEHVI）中，实现了“如何优化”和“信任多少先验”两个问题的解耦。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

CALIPER 的整体架构是一个可以插入到标准 MOBO 流程中的“先验层”。其工作流程如下：

1.  **先验生成**：使用多个 LLM 专家角色（如溶解度专家、类药性专家）为候选池中的每个候选点 $x$ 生成目标维度的预测评分 $\mu_{ej}(x)$ 和置信度 $c_{ej}(x)$。
2.  **先验聚合**：在 BO 的每一步 $t$，根据当前专家-目标信任权重 $\alpha_{ej,t}$ 和有效置信度 $\tilde{c}_{ej,t}(x)$，聚合出每个目标 $j$ 的先验均值 $p_{j,t}(x)$。
3.  **残差拟合**：将真实观测值 $y_{j,\tau}$ 减去当前先验 $p_{j,t}(x_\tau)$，得到残差 $r_{j,\tau}$。然后，用高斯过程（GP）拟合这些残差。
4.  **后验重建**：将 GP 对残差的预测均值 $m_{j,t}^r(x)$ 加回先验 $p_{j,t}(x)$，得到最终用于采集函数的后验均值。后验方差则直接使用 GP 的预测方差 $v_{j,t}^r(x)$。
5.  **门控与更新**：在每一步观测到新数据后，更新声誉市场（专家-目标信任权重）和反事实门控（先验门和更新门）的参数。

### 3.2 关键算法与数学推导

**1. 先验聚合与残差模型**

-   聚合先验均值：
    $$p_{j,t}(x) = \frac{\sum_e \alpha_{ej,t} \tilde{c}_{ej,t}(x) \mu_{ej}(x)}{\sum_e \alpha_{ej,t} \tilde{c}_{ej,t}(x)}$$
    其中 $\alpha_{ej,t}$ 是专家 $e$ 对目标 $j$ 的信任权重，$\tilde{c}_{ej,t}(x)$ 是经过门控处理后的有效置信度。

-   残差模型：
    $$f_j(x) = p_{j,t}(x) + r_j(x)$$
    其中 $r_j(x)$ 是残差函数，由 GP 建模。GP 拟合的数据点为 $\{(x_\tau, y_{j,\tau} - p_{j,t}(x_\tau))\}$。

-   最终后验：
    $$f_j(x) | \mathcal{D}_t \sim \mathcal{N}(p_{j,t}(x) + m_{j,t}^r(x), v_{j,t}^r(x))$$

**2. 目标维度声誉市场**

-   标准化误差：
    $$\epsilon_{ej,t} = \frac{|\mu_{ej}(x_t) - y_{j,t}|}{s_{j,t}}$$
    其中 $s_{j,t}$ 是目标 $j$ 的观测值运行尺度。

-   软成功分数：
    $$q_{ej,t} = \exp(-\frac{1}{2} \epsilon_{ej,t}^2)$$

-   资本更新：
    $$R_{ej,t} = \text{clip}(0.5 - 0.5 \epsilon_{ej,t}^2, -2.0, 0.5)$$
    $$K_{ej,t+1} = (1-\lambda) K_{ej,t} + \eta \tilde{c}_{ej}(x_t) R_{ej,t}$$
    其中 $\lambda$ 是折扣因子，$\eta$ 是学习率。

-   信任权重：通过温度控制的 Softmax 将资本 $K_{ej}$ 转换为权重 $\alpha_{ej}$。

**3. 解耦反事实门控**

-   **先验门**：比较三种行动 $a \in \{\text{noconf}, \text{conf}, \text{drop}\}$。
    -   计算每种行动下的历史残差 $r_{j,\tau}^{(a)} = y_{j,\tau} - p_{j,t}^{(a)}(x_\tau)$。
    -   使用固定核的 GP 边际似然作为评分 $S_{j,t}(a) = \frac{1}{t} \log p(r_{j,1:t}^{(a)} | X_t^{\text{obs}})$。
    -   计算相对于“无置信度”行动的 logits：
        $$\ell_{j,t}(a) = \eta_p (S_{j,t}(a) - S_{j,t}(\text{no conf}) - \Delta_a)$$
        其中 $\Delta_{\text{drop}} = 0.05$ 是一个惩罚项，鼓励在证据不充分时优先选择保守的“无置信度”行动。
    -   通过带收缩的 Softmax 得到行动概率 $\pi_{j,t}^p(a)$，在数据少时向“无置信度”行动收缩。
    -   最终先验为混合先验：
        $$p_{j,t}(x) = \pi_{j,t}^p(\text{no conf}) p_{j,t}^{(\text{no conf})}(x) + \pi_{j,t}^p(\text{conf}) p_{j,t}^{(\text{conf})}(x)$$
        （drop 行动贡献为零）

-   **更新门**：维护两个影子市场（使用置信度 $g=1$，不使用 $g=0$）。
    -   计算每个影子市场的预测损失 $L_{j,t}^u(g) = |y_{j,t} - p_j^{(g)}(x_t)| / s_{j,t}$。
    -   使用全信息 Hedge 算法更新更新门的概率 $\pi_{j,t+1}^u(g)$。

### 3.3 模型、训练与数据集

-   **模型**：核心是 CALIPER 先验层，它包裹着一个标准的 GP 模型（用于拟合残差）。采集函数使用 BoTorch 库中的 qLogNEHVI 或 qLogEHVI。
-   **训练**：CALIPER 本身不涉及传统意义上的“训练”。其参数（声誉资本、门控概率）是在 BO 过程中**在线更新**的，基于每一步观测到的真实目标值。更新规则是确定性的。
-   **数据集**：
    -   **合成数据**：用于压力测试，构造了多种专家失败模式（如所有专家都有用、所有专家都误导、目标特异性专家等）。
    -   **真实分子数据**：来自 MoleculeNet 的 ESOL（水溶性）、FreeSolv（水合自由能）和 Lipophilicity（亲脂性）数据集。每个数据集构建一个候选分子池（大小 100-150），并使用 RDKit 描述符作为特征。
    -   **LLM 先验**：使用 Qwen3.6-Flash 等 LLM，通过精心设计的提示词（Prompt）为每个候选分子生成多个专家角色的评分和置信度。这些先验在 BO 开始前被缓存，确保所有方法使用相同的 LLM 信息。

### 3.4 实验设置与结果

-   **基线方法**：随机搜索、标准 MOBO（Vanilla MOBO）、固定 LLM 先验的 MOBO、qLogNEHVI、ParEGO 等。
-   **主要指标**：最终超体积（Final Hypervolume, HV）、超体积曲线下面积（AUC HV）、最佳目标之和。
-   **关键实验结果**：
    1.  **与强基线 qLogNEHVI 的集成（表 3, 4）**：在合成大候选池上，`qLogNEHVI + CALIPER` 显著优于单独的 `qLogNEHVI` 和 `qLogNEHVI + 固定先验`。在真实分子数据上，CALIPER 在 ESOL 和 Lipophilicity 上提升或持平 qLogNEHVI，而固定先验在 FreeSolv 上会严重损害性能。这证明了 CALIPER 作为先验层的鲁棒性。
    2.  **独立诊断实验（表 5, 图 1）**：在使用标量化残差-UCB 采集函数时，CALIPER 在 ESOL 和 FreeSolv 上取得了最佳最终 HV。在 Lipophilicity 上，CALIPER 与基线持平，但不如 qLogNEHVI，这揭示了其局限性。
    3.  **先验质量诊断（表 6, 7）**：对 Lipophilicity 的分析表明，所有 LLM 模型（包括更强的 Qwen3.7-Max）都对目标 0（亲脂性窗口）存在系统性低估偏差。这解释了为什么 CALIPER 在该数据集上提升有限。当使用一个“受控的有效先验”（oracle-style）时，CALIPER 能显著提升性能，说明其上限取决于先验质量。
    4.  **合成压力测试（表 8）**：在多种专家失败模式下，CALIPER 的声誉市场机制始终优于固定先验，并能有效减轻误导性先验的损害。
    5.  **消融研究（表 9, 10, 图 2, 3）**：置信度消融实验表明，不同数据集对置信度的最佳使用方式不同（如 ESOL 偏好自适应先验置信度，FreeSolv 偏好仅使用先验，Lipophilicity 偏好不使用置信度）。这验证了解耦门控设计的必要性。CALIPER 的最终反事实门控在多数情况下优于早期版本。

### 3.5 技术细节

-   **确定性**：一旦随机种子、初始设计和 LLM 先验缓存固定，CALIPER 的整个 BO 过程是确定性的，这有利于实验复现。
-   **超参数**：论文在表 1 中给出了固定的超参数，如声誉市场的学习率 $\eta=0.45$、折扣 $\lambda=0.015$、先验门证据评分使用 GP 边际似然等。这些参数在主要分子实验中保持不变。
-   **实现**：CALIPER 作为 `PriorShiftedModel` 适配器实现，它接收一个标准 GP 模型，并返回一个后验均值被 LLM 先验偏移过的后验分布。这个适配后的后验可以直接传递给 BoTorch 的采集函数。

## 4. 意义与影响（Significance and Impact）

-   **解决的问题与贡献**：
    -   解决了 LLM 作为先验知识源时不可靠、未校准的问题，特别是在多目标优化场景下。
    -   提出了一个通用的、模块化的框架（CALIPER），可以安全地将任何不确定的先验信息（不限于 LLM）注入到 MOBO 中。
    -   通过将 LLM 的角色从“优化器”转变为“可检验的先验源”，为 LLM 与贝叶斯优化的结合开辟了新的、更稳健的范式。

-   **对未来研究/应用的影响**：
    -   **方法论影响**：CALIPER 的设计思想（在线信任学习、反事实门控、残差建模）可以启发其他需要融合不可靠先验知识的机器学习任务。
    -   **应用影响**：在药物发现、材料设计、自动化实验等昂贵黑盒优化领域，CALIPER 提供了一种利用 LLM 领域知识来加速搜索、降低成本的实用工具。
    -   **LLM 角色转变**：鼓励社区将 LLM 视为一种可以提供“线索”但需要被“验证”的工具，而不是一个可以直接给出最终答案的“神谕”。

-   **局限与开放问题**：
    -   **先验质量依赖**：CALIPER 的性能上限受限于 LLM 先验的质量。如果先验信息完全是噪声或系统性偏差，CALIPER 虽然能减轻损害，但无法创造信息。如何设计更好的提示词或专家角色来生成更高质量的 LLM 先验是未来方向。
    -   **放弃机制并非无损**：在“所有专家都误导”的合成测试中，CALIPER 的性能仍低于不使用任何先验的 Vanilla BO，说明其放弃机制并非完全无损。如何实现更完美的“无损放弃”是一个挑战。
    -   **计算开销**：维护多个影子市场和计算边际似然会带来额外的计算开销，尤其是在候选池很大时。
