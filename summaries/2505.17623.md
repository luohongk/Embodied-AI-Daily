## 0. 概览（Concise Summary）

- **主题（Topic）**：可验证计算（Verifiable Computing, VC）在深度学习推理（Deep Neural Network Inference）中的应用，特别是针对不可信方的场景。
- **问题（Problem）**：现有的可验证计算方案在处理深度学习中的非算术运算（如定点数舍入、ReLU激活函数）时，效率低下，通常需要布尔电路编码、高次多项式或大型查找表，导致计算和通信开销巨大。
- **方法（Method）**：提出一个名为 **Range-Arithmetic** 的框架。该框架将非算术运算（定点数舍入和ReLU）转化为可通过 **sum-check协议** 和 **聚合范围证明（Aggregated Range Proofs）** 验证的算术步骤，从而兼容基于有限域的证明系统。
- **创新（Innovation）**：
    1.  首次同时实现了 $O(n^3)$ 证明者复杂度、$O(n^2)$ 验证者复杂度、$O(\log n)$ 通信复杂度，并**完整支持定点数舍入和ReLU**。
    2.  用轻量级的**范围证明**替代了大型查找表，避免了预处理和R1CS编码开销。
    3.  不需要像先前工作那样扩大有限域来防止溢出，更实用。
- **意义（Significance）**：为去中心化机器学习系统提供了一种高效、平衡的推理验证方案，解决了在不可信环境下确保计算完整性的核心挑战，推动了可信AI和区块链应用的落地。
- **一句话总结**：本文提出Range-Arithmetic框架，通过将深度学习中的非算术操作（舍入、ReLU）巧妙地转化为范围证明和sum-check协议可验证的算术步骤，在保证安全性的同时，实现了与最先进方法相当的效率，并首次在单一框架内完整支持了这些关键操作。

## 1. 研究动机（Motivation）

- **问题与空白**：在去中心化机器学习系统中，资源受限的区块链节点需要将深度学习推理等繁重任务外包给不可信的执行者（Prover）。为了确保结果的正确性，需要可验证计算（VC）技术。然而，主流的VC方案（如sum-check协议）天然工作在有限域上，只支持加法和乘法等算术运算。深度学习中的关键操作，如**定点数舍入（Fixed-point Rounding）** 和**ReLU激活函数**，都是非算术的，难以在现有VC框架中高效地表示和验证。
- **为什么重要**：如果不能高效验证这些非算术操作，整个推理过程的完整性就无法保证。现有的解决方案要么效率极低（如布尔电路编码导致 $O(n^3 \log n)$ 证明者开销），要么功能不完整（不支持ReLU或舍入），要么依赖昂贵的预处理和大型查找表（如lookup-based方法）。这严重阻碍了可验证机器学习（Verifiable ML）在实际系统中的应用。
- **挑战**：核心挑战在于如何将非算术操作“算术化”，使其能够融入基于有限域的证明系统，同时不引入过高的计算和通信开销。具体来说，需要解决：
    1.  **定点数舍入**：如何在有限域中证明一个数经过舍入后是正确的，而不需要将整个计算过程展开为二进制电路？
    2.  **ReLU函数**：如何证明 $ReLU(x) = \max(0, x)$ 的结果是正确的，而不需要依赖大型查找表或高次多项式？

## 2. 创新点（Innovation）

- **关键创新**：提出了 **Range-Arithmetic** 框架，其核心思想是将非算术操作分解为**算术部分**和**范围约束部分**。
    1.  **对定点数舍入的创新处理**：将舍入操作 $C' = R(C)$ 的验证转化为两个聚合范围证明：
        - 证明舍去部分 $E = C - 2^s \times C'$ 的所有元素都在一个小的对称区间 $[-2^{s-1}, 2^{s-1})$ 内。
        - 证明舍入后的结果 $C'$ 的所有元素都在一个不会导致模运算回绕的区间 $[-2^{t+1}, 2^{t+1})$ 内。
        - 通过这两个简单的范围约束，就等价于验证了舍入操作的正确性，避免了复杂的布尔电路或高次多项式。
    2.  **对ReLU的创新处理**：将 $ReLU(A) = B$ 的验证分解为：
        - 证明 $Y = |A|$，即 $Y$ 的所有元素非负（通过范围证明）。
        - 证明 $a_i^2 = y_i^2$ 对所有元素成立（通过sum-check协议验证多项式恒等）。
        - 最后通过承诺的代数关系 $P_B^2 = P_A \cdot P_Y$ 确保 $B = (A+Y)/2 = ReLU(A)$。
        - 这种方法用轻量级的平方相等性检查替代了昂贵的查找表。
- **与现有方案的不同**：
    - 相比于[20]（Integer-scaling），本方法**原生支持ReLU**，且不需要扩大有限域来防止溢出。
    - 相比于[22]、[25]（Lookup-based），本方法用**范围证明**替代了大型查找表，更适合资源受限的验证者，且没有预处理开销。
    - 相比于[4]（EZKL等通用框架），本方法**避免了R1CS电路编码的巨额常数开销**，在渐近复杂度相同的情况下，实际性能更优。
- **为什么能奏效**：该创新之所以有效，是因为它巧妙地利用了**范围证明**（Bulletproofs）的高效性和**sum-check协议**的算术验证能力。通过将非算术操作的核心约束（如舍入的截断范围、ReLU的非负性）转化为范围证明，而将算术关系（如矩阵乘法、平方相等）留给sum-check协议，实现了功能与效率的最佳平衡。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文提出的Range-Arithmetic框架是一个模块化的交互式证明系统。其整体架构如图1所示，包含一个**证明者（Prover）** 和一个**验证者（Verifier）**。

- **核心思想**：将神经网络推理分解为一系列层（Layer），每层对应一个操作（如矩阵乘法、ReLU）。每一层都独立地进行验证。
- **验证流程**：
    1.  **承诺**：证明者计算每一层输入和输出的承诺（Commitment），并发送给验证者。
    2.  **挑战**：验证者发送随机挑战（如随机点 $r_1, r_2$）。
    3.  **证明**：证明者根据挑战，利用sum-check协议和范围证明生成证明。
    4.  **验证**：验证者检查证明，以高概率确信该层的计算是正确的。
- **模块化设计**：矩阵乘法、舍入、ReLU的验证算法是独立的模块，可以像搭积木一样组合起来，验证任意深度的神经网络。

### 3.2 关键算法与数学推导

**1. 矩阵乘法验证（算术部分）**

- **问题**：验证 $C = A \times B$。
- **方法**：利用**多线性扩展（MLE）** 和**Schwartz-Zippel引理**。
    - 将矩阵 $A, B, C$ 的元素视为定义在布尔超立方体上的函数，并构造它们的MLE $\tilde{a}, \tilde{b}, \tilde{c}$。
    - 验证 $C = AB$ 等价于验证对所有二元输入 $(i, j)$ 有：
      $$\tilde{c}(i, j) = \sum_{\ell \in \{0,1\}^{\log m}} \tilde{a}(i, \ell) \cdot \tilde{b}(\ell, j)$$
    - 根据Schwartz-Zippel引理，只需在随机点 $(r_1, r_2)$ 上验证上述等式即可：
      $$\tilde{c}(r_1, r_2) = \sum_{\ell \in \{0,1\}^{\log m}} \tilde{a}(r_1, \ell) \cdot \tilde{b}(\ell, r_2)$$
    - 右边是一个关于 $\ell$ 的求和，可以交给**sum-check协议**（Algorithm 5）来高效验证。证明者需要提供对多项式 $\tilde{a}(r_1, \cdot)$ 和 $\tilde{b}(\cdot, r_2)$ 系数的承诺。

**2. 定点数舍入验证（非算术部分）**

- **问题**：验证 $C' = R(C)$，其中 $R$ 是定点数舍入操作。
- **方法**：转化为两个**聚合范围证明**。
    - 定义舍去部分 $E = C - 2^s \times C'$。
    - **条件1**：$E$ 的所有元素 $e_i$ 在区间 $[-2^{s-1}, 2^{s-1})$ 内。这保证了舍去的是正确的位数。
    - **条件2**：$C'$ 的所有元素 $c'_i$ 在区间 $[-2^{t+1}, 2^{t+1})$ 内。这保证了 $C'$ 不会在有限域中发生回绕。
    - 如果这两个条件都满足，则 $C' = R(C)$ 以高概率成立。这两个条件通过运行两次**聚合范围证明**（Algorithm 6，基于Bulletproofs）来验证。

**3. ReLU激活函数验证（非算术部分）**

- **问题**：验证 $B = ReLU(A)$。
- **方法**：分解为范围证明和平方相等性检查。
    - **步骤1**：证明者计算 $Y = |A|$ 并发送其承诺 $P_Y$。
    - **步骤2**：通过范围证明，证明 $Y$ 的所有元素非负（即 $y_i \ge 0$）。
    - **步骤3**：通过sum-check协议，验证对所有元素 $a_i, y_i$ 有 $a_i^2 = y_i^2$。这通过验证以下多项式在随机点 $s$ 的求和为0来实现：
      $$0 = \sum_{x \in \{0,1\}^{\log nk}} \tilde{I}(s, x) \cdot (\tilde{a}^2(x) - \tilde{y}^2(x))$$
      其中 $\tilde{I}$ 是单位矩阵的MLE。
    - **步骤4**：验证承诺之间的关系：$P_B^2 = P_A \cdot P_Y$。这确保了 $B = (A+Y)/2 = ReLU(A)$。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“多线性扩展（MLE）”？**
    - **通俗解释**：可以把一个矩阵想象成一个表格。MLE就像是一个“魔法公式”，它能完美地“记住”这个表格中每个格子里的数字。更厉害的是，当你给它一个表格里没有的坐标（比如小数坐标）时，它也能通过一种特殊的插值方式算出一个结果。这个“魔法公式”是一个多项式，并且对每个坐标都是线性的（就像一条直线）。
- **什么是“Sum-Check协议”？**
    - **通俗解释**：假设你想知道一个巨大表格里所有数字的总和，但你自己算太慢。你让一个助手（证明者）去算，他告诉你总和是100。你不信，想抽查他。Sum-check协议就像一个“数学侦探游戏”。你随机问几个问题，助手必须给出一些中间结果。通过检查这些中间结果，你就能以极高的概率判断出助手有没有撒谎。这个协议非常高效，通信量很少。
- **什么是“范围证明”？**
    - **通俗解释**：想象一个黑盒子，里面有一个数字。证明者声称这个数字在0到100之间。范围证明就是一种神奇的密码学工具，证明者可以用它来向你证明“这个数字确实在0到100之间”，但**不告诉你这个数字具体是多少**。论文里用这个来证明舍入后的误差很小，或者ReLU的结果是非负的。
- **为什么舍入验证可以简化为范围证明？**
    - **类比**：验证一个数 $x$ 被四舍五入到最近的整数 $x'$。这等价于验证两件事：1）误差 $e = x - x'$ 的绝对值小于0.5（$|e| < 0.5$）。2）$x'$ 本身是一个整数。论文中的范围证明就是用来验证这两个条件的。它把复杂的舍入操作，变成了两个简单的“数值范围”检查，大大简化了问题。
