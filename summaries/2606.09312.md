## 0. 概览（Concise Summary）

- **主题（Topic）**：深度学习编译器中的张量程序优化，特别是自动调度（auto-scheduling）中的候选程序评估（candidate evaluation）问题。
- **问题（Problem）**：现有的基于学习（learned cost model）的评估方法将候选程序视为静态代码快照，忽略了其由一系列调度决策逐步生成的过程。这导致模型对语法变化敏感，无法理解调度决策间的依赖关系，从而影响搜索效率。
- **方法（Method）**：受世界模型（World Model）启发，提出一个学习张量程序优化潜在动态过程的框架。该框架将编译器优化建模为潜在空间中的状态转移过程：使用编码器将程序状态映射到连续潜在空间，通过一个轻量级的动作条件转移模型（action-conditioned transition model）逐步模拟调度动作对状态的影响，并基于预测的最终状态表示进行候选程序排序。
- **创新（Innovation）**：
    1.  将世界模型的思想引入编译器优化领域，将调度评估问题重新定义为学习潜在状态动态。
    2.  提出一个三阶段框架：状态编码器、多步潜在状态预测器和排序成本模型，用于动态评估候选程序。
    3.  构建了一个用于学习编译器状态转移的数据集，包含动作-状态轨迹。
- **意义（Significance）**：该方法显著提升了自动调优的效率。在相同搜索预算下，性能优于现有方法（如Ansor）；在仅使用1/10测量次数的情况下，能达到与使用大量测量次数的Ansor相当的搜索质量。这为构建更高效、更智能的编译器优化系统提供了新思路。
- **一句话总结**：本文提出一种受世界模型启发的编译器优化框架，通过学习调度动作在潜在空间中的状态转移动态，实现了更高效、更鲁棒的张量程序候选评估。

## 1. 研究动机（Motivation）

- **问题与空白**：在深度学习编译器中，自动调度器需要从巨大的搜索空间中寻找最优的张量程序。由于直接在硬件上测量性能非常耗时，现有工作（如Ansor, TenSet）使用学习到的成本模型（learned cost model）来快速评估候选程序。然而，这些模型存在一个根本性缺陷：它们将每个候选程序视为一个**静态的代码快照**（static snapshot），忽略了该程序是由一系列调度动作（如循环分块、融合、重排等）逐步变换而来的动态过程。
- **为什么重要**：这种静态视角带来了两个关键挑战：
    1.  **上下文依赖缺失**：一个调度决策的效果高度依赖于之前决策所建立的上下文。例如，一个“分块”动作的目标变量可能是由之前的“融合”动作动态创建的。静态模型无法捕捉这种依赖关系。
    2.  **语法敏感性**：静态模型容易被程序表面的语法变化所误导。例如，对于大小为1024和1023的矩阵乘法，应用相同的分块策略，后者会因边界处理而生成包含额外条件分支的代码，尽管其核心计算和缓存行为几乎相同。静态模型可能会错误地将这两个语义上等价（semantically equivalent）的程序视为不同。
- **挑战**：如何设计一个评估模型，能够理解调度动作的**语义**及其对程序状态的**累积影响**，同时避免昂贵的中间状态具体化（如AST突变）和语法噪声的干扰。

## 2. 创新点（Innovation）

- **新视角：编译器世界模型**：首次将世界模型（World Model）的思想应用于张量程序优化。将编译器+硬件视为环境，TensorIR程序视为状态，调度变换视为动作。评估问题被重新定义为学习这些状态在潜在空间中的转移动态。
- **新方法：三阶段动态评估框架**：
    1.  **潜在状态编码器**：使用对比学习训练一个CodeBERT编码器，将TensorIR程序文本映射到一个连续的、对语法变化鲁棒的潜在表示空间。
    2.  **动作条件转移模型**：设计一个基于TransH的轻量级转移模型，用于在潜在空间中模拟调度动作的累积效果。它从初始状态表示开始，逐步应用动作嵌入来预测最终状态的表示，避免了在搜索过程中进行实际的AST操作。
    3.  **排序成本模型**：使用学习排序（Learning-to-Rank）方法，将预测的最终状态表示、结构化动作特征和硬件特征结合起来，输出一个用于候选程序排序的分数，而非直接预测运行时间。
- **与现有方案的不同**：
    - 与**静态快照模型**（如TenSet）相比，本方法考虑了调度过程的动态性，对语法变化更鲁棒。
    - 与**纯动作序列模型**相比，本方法通过显式地模拟中间状态，解决了动作的指代歧义问题（例如，动作`Tile(k)`中的`k`是在之前步骤中动态创建的）。
    - 与**在搜索过程中进行实际AST突变**相比，本方法在潜在空间中进行模拟，计算开销极低。
- **为什么能奏效**：通过在潜在空间中显式地建模状态转移，模型能够学习到调度动作的语义和因果效应。这使得模型能够理解决策的上下文，并过滤掉无关的语法噪声，从而做出更准确的评估。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文提出的框架包含三个主要组件，如图3所示：

1.  **TensorIR状态编码器（State Encoder, $E$）**：将TensorIR程序文本$s$映射为一个稠密向量$z = E(s) \in \mathbb{R}^d$。该编码器基于CodeBERT，并采用滑动窗口处理长文本。通过对比学习训练，使得语义相似的程序在潜在空间中距离更近。
2.  **动作条件多步状态转移模型（Transition Model, $F$）**：给定初始状态嵌入$z_0 = E(s_0)$和动作序列$a_{1:T}$，该模型逐步预测最终状态嵌入$\hat{z}_T = F(z_0, a_{1:T})$。它通过一个基于TransH的预测器，在每个时间步$t$，根据当前状态$z_{t-1}$和动作$a_t$预测下一个状态$\hat{z}_t$。
3.  **排序成本模型（Ranking Model, $G$）**：将预测的最终状态嵌入$\hat{z}_T$、结构化动作特征$\phi(a_{1:T})$和硬件特征$h$拼接起来，通过一个XGBoost排序模型输出一个标量分数$y = G(\hat{z}_T, \phi(a_{1:T}), h)$。该分数用于在同一个工作负载内对候选程序进行排序。

**整体流程**：在在线搜索中，对于每个候选程序，框架首先提取其调度动作序列，然后使用编码器$E$对初始状态编码，接着使用转移模型$F$在潜在空间中“回放”动作序列，得到预测的最终状态表示，最后使用排序模型$G$给出分数，指导搜索过程优先测量分数高的候选程序。

### 3.2 关键算法与数学推导

- **状态编码器训练（对比学习）**：
    损失函数为交叉熵损失：
    $$L_{repr} = \text{CE}(u, 0)$$
    其中$u$是包含一个正样本和两组负样本（组内负样本和组间负样本）的logit向量。正样本logit是锚点状态和其增强视图的余弦相似度，负样本logit是锚点与负样本的加权余弦相似度。

- **状态转移模型训练**：
    转移模型的核心是预测下一个状态嵌入：
    $$\hat{z}_t = F(z_{t-1}, a_t)$$
    训练损失包含两项，并在多步回滚（multi-step rollout）上累积：
    $$L_{trans} = \sum_{t=1}^{K} m_t \left[ \lambda L_{match}^{(t)} + (1-\lambda) L_{rank}^{(t)} \right]$$
    其中$m_t$是回滚掩码，$\lambda$是平衡权重。
    - **状态匹配损失**：最小化预测状态与真实状态之间的均方误差。
        $$L_{match}^{(t)} = \|\hat{z}_t - z_t\|_2^2$$
    - **基于边界的排序损失**：鼓励预测状态更接近真实状态，而非负样本状态。
        $$L_{rank}^{(t)} = \max\left(0, d(\hat{z}_t, z_t) - d(\hat{z}_t, z_t^-) + \gamma\right)$$
        其中$z_t^-$是第$t$步的难负样本（hardest negative），$\gamma$是边界参数，$d(\cdot, \cdot)$是距离度量（如欧氏距离）。

- **排序模型训练**：
    使用LambdaRank算法，通过XGBoost实现。训练数据按工作负载分组，组内根据测量到的运行时间排序，并映射为相关性标签。模型学习预测一个分数，使得排序与真实性能排序一致。

### 3.3 模型、训练与数据集

- **模型**：
    - 状态编码器：基于CodeBERT的Transformer模型，带有滑动窗口和投影头。
    - 转移模型：基于TransH的轻量级几何变换模型，包含一个动作编码器（冻结的CodeBERT + 可训练投影层）。
    - 排序模型：XGBoost。
- **训练数据**：
    - 所有组件均从TenSet的调优日志中提取的张量程序进行训练。
    - **对比学习数据集**：10,000个样本（8,000训练，2,000验证），包含从日志中提取的TensorIR状态。
    - **状态转移数据集**：分别针对CPU和GPU训练，包含约39万个状态-动作样本。数据被组织成动作-状态轨迹（action-state trajectories），如$(s_0, a_1, s_1, a_2, s_2, ..., a_K, s_K)$。
    - **排序模型数据集**：在作者自己的平台上重新测量了相应张量程序的性能，以获得最新的性能标签。GPU上有140,262个训练样本，CPU上有500,425个。

### 3.4 实验设置与结果

- **实验设置**：
    - **硬件**：Intel Xeon Gold 6430 CPU, NVIDIA RTX 4090 GPU。
    - **软件**：TVM 0.8.dev0, CUDA 12.6。
    - **基准测试**：7个神经网络模型（ResNet-50, BERT, MobileNetV2, GPT-2, ResNet18-3D, OPT-1.3B, GPT-Neo-2.7B），共提取103个子图。其中22个代表性子图用于细粒度分析（见表2）。
    - **对比方法**：PyTorch, PyTorch-opt (cuDNN), TensorRT, Ansor, TenSet。
- **主要实验结果**：
    1.  **端到端模型性能（图4）**：在64次测量预算下，本方法（Ours-64）比Ansor-64快1.29倍（几何平均）。在更大预算下（Ours-800 vs Ansor-1024），仍快1.10倍。相比PyTorch和PyTorch-opt，加速比分别达到4.61倍和3.67倍。
    2.  **模型级搜索性能（图5）**：在相同时间或测量预算下，本方法在所有模型上均优于或持平于Ansor。例如，在GPU上64次测量预算下，几何平均加速比为1.22倍。在CPU上64次测量预算下，加速比为1.62倍。

    4.  **样本效率（图8）**：本方法仅使用64次测量（Ours-64）就能达到或接近Ansor使用1024次测量的性能。使用1024次测量（Ours-1k）的性能甚至略优于使用10,000次测量的Ansor（Ansor-10k），实现了**10倍的样本效率提升**。

### 3.5 技术细节

- **动作表示**：转移模型和排序模型使用不同的动作特征。转移模型使用一个冻结的CodeBERT编码器加可训练投影层生成的**稠密动作嵌入**，用于模拟状态转移。排序模型使用从动作序列中提取的**结构化特征向量**（如动作计数、序列长度、特定动作的数值属性等）。
- **多步回滚训练**：转移模型在训练时，不是只预测下一步，而是从当前状态开始，沿着整个动作序列进行多步预测。每一步的预测误差都会累积到总损失中，这使得模型能够学习长期依赖关系。
- **难负样本挖掘**：在转移模型的排序损失中，使用了难负样本（hardest negative），即与真实状态最接近的负样本，以迫使模型学习更精细的区分能力。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了现有成本模型因静态评估而导致的语法敏感性和上下文缺失问题。
    - 提出了一种全新的、受世界模型启发的编译器优化范式，将调度评估从静态快照比较转变为动态过程模拟。
    - 通过实验证明，该方法在显著减少硬件测量次数（10倍）的同时，能达到甚至超越现有最优方法的搜索质量，极大地提升了自动调优的效率。
- **对未来研究/应用的影响**：
    - 为编译器优化领域开辟了新的研究方向，即利用世界模型和潜在动态来理解和优化程序变换。
    - 该方法可以作为一种通用的评估框架，集成到现有的自动调度器（如TVM, Ansor）中，提升其性能。
    - 其核心思想——在潜在空间中模拟复杂系统的动态——可能对其他需要昂贵评估的搜索问题（如神经架构搜索、超参数优化）具有启发意义。
- **局限与开放问题**（作者提及或可推断）：
    - 当前实现依赖于TenSet的调优日志来构建训练数据，这可能需要一个初始的、质量尚可的搜索过程。
    - 状态编码器基于CodeBERT，对于极长的TensorIR程序，滑动窗口策略可能丢失全局信息。
    - 转移模型使用TransH，这是一种相对简单的几何变换，对于更复杂的调度动作组合，其表达能力可能有限。
    - 实验仅在CPU和GPU上进行，其在其他硬件（如TPU、NPU）上的有效性有待验证。
    - 如何将世界模型的思想进一步扩展到探索（exploration）阶段，而不仅仅是评估（evaluation）阶段，是一个有趣的开放问题。

## 5. 澄清与简化（Clarifications and Simplifications）

- **世界模型类比**：可以想象你在玩一个复杂的策略游戏（如《星际争霸》）。世界模型就像是游戏的一个“模拟器”，它学习游戏规则（动态）。你不需要在真实的游戏中一步步操作，而是可以在你的大脑（潜在空间）中模拟各种操作序列，并预测最终结果（如资源、兵力）。本文的编译器世界模型就是这样一个“模拟器”，它学习调度动作如何改变程序状态，从而在“大脑”中快速预测最终程序的好坏，而不需要真的去编译和运行它。
- **为什么在潜在空间模拟？**：直接在真实世界（编译器）中模拟每一步调度动作需要执行昂贵的AST突变操作。而在潜在空间（一个数学向量空间）中，状态转移只是一个简单的向量运算（如加法或通过一个小型神经网络），速度极快。这就像是用一个简单的数学公式来模拟物理运动，而不是每次都去解复杂的微分方程。
- **对比学习的作用**：可以把它想象成一个“找不同”和“找相同”的训练。训练目标是让编码器学会忽略程序的“外表”（如变量名、边界处理代码），而关注其“内在”（如计算模式、数据流）。这样，两个结构不同但性能相同的程序，在潜在空间中的表示就会非常接近。这解决了语法敏感性问题。
- **TransH转移模型**：TransH原本用于知识图谱，可以理解为每个动作（关系）定义了一个“翻译平面”。当前状态向量首先被投影到这个平面上，然后加上动作向量，就得到了下一个状态。这提供了一种简单而有效的方式来建模“执行动作A会使状态从S1变为S2”这种关系。
