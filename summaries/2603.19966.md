## 0. 概览（Concise Summary）

- **主题（Topic）**：抗风四旋翼无人机导航，具体任务为穿越门框（gate traversal）。
- **问题（Problem）**：轻量级四旋翼无人机在强风、阵风等动态气流干扰下的可靠自主导航问题。现有方法要么专注于低层轨迹跟踪，要么将抗风与高层导航决策分离，导致在复杂风场中规划与跟踪耦合失效。
- **方法（Method）**：提出一个名为 **GustPilot** 的分层框架。上层是一个基于深度强化学习（DRL，具体为PPO算法）的速度规划器，用于生成惯性系下的速度参考指令；下层是一个基于几何的增量非线性动态逆（INDI）控制器，用于快速跟踪速度指令并抑制风扰。训练时采用风扇射流域随机化（fan-jet domain randomization）来模拟风扰。
- **创新（Innovation）**：
    1.  **分层抗风架构**：将“风感知规划”（DRL）与“快速扰动抑制”（INDI）明确分离，各司其职，降低了学习策略的负担。
    2.  **几何INDI控制器**：在SE(3)流形上构建，同时利用线加速度和角加速度的增量反馈，实现快速、鲁棒的扰动抑制。
    3.  **风扇射流域随机化**：在仿真训练中，通过随机化风扇射流参数（强度、位置、湍流等），使策略学会在未见过的风场中泛化。
- **意义（Significance）**：首次在真实的轻量级（50g）四旋翼平台上，验证了DRL与INDI结合的分层框架在复杂动态风场（包括移动门和移动风源）下进行自主导航的有效性。实验结果表明，该方法在成功率（94.6% vs 36.0%）和跟踪精度（RMSE降低50%）上远超DRL-PID基线，为实现实用化的抗风自主飞行提供了新思路。
- **一句话总结**：本文提出GustPilot，一个结合深度强化学习速度规划与增量非线性动态逆控制的分层框架，通过“规划感知风、控制抑制风”的分离设计，使轻量级四旋翼在强风干扰下也能稳健地穿越门框。

## 1. 研究动机（Motivation）

- **问题**：轻量级四旋翼（如Crazyflie）惯性小，对风扰极其敏感。传统的非线性或自适应控制器在缓慢变化的风扰下表现尚可，但在快速变化的阵风或局部强射流中，实时建模困难，跟踪性能急剧下降。这导致无人机在风中的自主导航（如穿越门框）非常不可靠。
- **重要性**：实现无人机在真实、非结构化环境（如城市峡谷、海上平台、灾害现场）中的可靠飞行，抗风能力是核心瓶颈之一。解决此问题对物流、巡检、搜救等应用至关重要。
- **挑战**：
    1.  **耦合问题**：风扰同时影响规划层（需要调整路径避开或适应风）和控制层（需要抵抗风对姿态和位置的扰动）。现有方法常将两者分开处理，效果不佳。
    2.  **快速扰动**：阵风变化快，要求控制器有极高的带宽和鲁棒性，传统PID在强扰动下容易失稳。
    3.  **泛化能力**：真实风场复杂多变，难以建模。学习型方法需要具备从简单训练环境泛化到复杂未知风场的能力。

## 2. 创新点（Innovation）

1.  **分层DRL-INDI架构**：核心创新在于将抗风任务解耦为两个层次：
    - **上层（DRL规划器）**：负责“风感知规划”。它学习如何根据无人机状态和门框信息，生成一个能有效穿越门框的惯性系速度参考。它不直接控制电机，而是通过域随机化“学会”在风中如何调整速度指令。
    - **下层（INDI控制器）**：负责“快速扰动抑制”。它不关心门框在哪，只负责精确跟踪上层发来的速度指令，并利用加速度和角加速度的增量反馈，以极高的带宽抑制掉风造成的瞬时扰动。
    - **不同之处**：与端到端DRL（直接输出电机指令）或DRL输出姿态指令不同，GustPilot的DRL输出速度指令，这更符合导航任务的自然抽象，且将快速动态控制交给了结构化的INDI控制器，利用了其鲁棒性优势。

2.  **几何INDI控制器**：在SE(3)群上构建，不仅使用角加速度反馈，还创新性地使用了**特定线加速度（specific linear acceleration）** 的增量反馈。这使得控制器能直接感知并抵抗风对机体产生的加速度扰动，比仅依赖角速度/角加速度反馈的INDI更全面、更快速。

3.  **风扇射流域随机化**：在仿真训练中，不是简单地施加均匀风场，而是模拟了真实风扇产生的**局部湍流射流**。通过随机化射流的强度、宽度、湍流强度、位置等参数，迫使DRL策略学习到一种通用的抗风导航策略，而不是过拟合到特定风模式。这使得策略能从简单的单门/单风扇训练环境，零样本（zero-shot）泛化到多门、多风扇、移动风扇等复杂真实场景。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

GustPilot的整体架构如图2所示，是一个典型的分层控制结构：

1.  **高层：DRL速度规划器**：
    - **输入**：观测向量 $o_t$，包含无人机状态（位置、姿态、线速度、角速度）和任务信息（目标门框的相对位置、大小、朝向）。
    - **输出**：动作 $a_t$，即惯性系下的期望速度指令 $v_{des} = [v_x, v_y, v_z]$ 和一个速度缩放因子 $v_{max}$。
    - **训练**：使用PPO算法在仿真环境中训练，奖励函数鼓励无人机快速、安全、对齐地穿越门框。

2.  **中层：几何INDI控制器**：
    - **外环（速度环）**：接收 $v_{des}$，通过比例积分（PI）控制计算期望加速度 $a_c$。然后，利用**增量式**方法更新期望的推力矢量 $\tau_{bz,c}$：$\tau_{bz,c} = \tau_{bz,f} + \alpha_{outer}(a_c - a_f)$。这里 $a_f$ 是实测的滤波后加速度，$\tau_{bz,f}$ 是上一时刻的推力矢量。这个增量项 $(a_c - a_f)$ 直接反映了风等外部扰动造成的加速度误差，从而快速修正推力。
    - **内环（姿态环）**：根据推力方向 $b_{z,c}$ 和偏航参考 $\psi_{ref}$（与门框法线对齐）计算期望姿态。然后，通过**增量式**方法计算期望力矩 $\mu_c$：$\mu_c = \mu_f + \alpha_{inner} J(\dot{\omega}_c - \dot{\omega}_f)$。这里 $\dot{\omega}_f$ 是实测的滤波后角加速度，$\mu_f$ 是上一时刻的力矩。这个增量项 $(\dot{\omega}_c - \dot{\omega}_f)$ 直接反映了风造成的角加速度扰动，从而快速修正力矩。
    - **控制分配**：将计算出的总推力 $f_c$ 和力矩 $\tau_c$ 通过混控矩阵映射到四个电机的转速指令。

3.  **底层：仿真训练环境**：
    - 使用风扇射流模型模拟风扰。风场由多个局部湍流射流组成，其参数（强度、宽度、湍流等）在每次训练回合开始时随机化（域随机化）。

### 3.2 关键算法与数学推导

1.  **INDI外环（速度环）**：
    - 期望加速度：$a_c = K_v e_v + K_i \int e_v dt$，其中 $e_v = v_{des} - v$。
    - 增量式推力更新：$\tau_{bz,c} = \tau_{bz,f} + \alpha_{outer}(a_c - a_f)$
    - 推力大小和方向：$\tau_c = \|\tau_{bz,c}\|, \quad b_{z,c} = \frac{\tau_{bz,c}}{\tau_c}$
    - 总推力指令：$f_c = m \tau_c$

2.  **INDI内环（姿态环）**：
    - 期望角加速度：$\dot{\omega}_c = K_\xi \xi_e - K_\omega \omega_f$，其中 $\xi_e = \log(\xi_c)$ 是姿态误差（SO(3)上的对数映射）。
    - 增量式力矩更新：$\mu_c = \mu_f + \alpha_{inner} J(\dot{\omega}_c - \dot{\omega}_f)$

3.  **风扇射流风模型**：
    - 射流宽度：$\sigma(x) = \sigma_0 + k_{spread} x$
    - 中心线速度衰减：$u_c(x) = u_0 \frac{x_0}{x_0 + x}$
    - 平均轴向风速（高斯径向分布）：$u_{mean}(r, x) = u_c(x) \exp\left(-\frac{r^2}{2\sigma(x)^2}\right)$
    - 风场截断：$v_{mean}(p) = \begin{cases} u_{mean}(r, x) a, & \text{if } x > 0 \land r \leq r_{cut}(x) \\ 0, & \text{otherwise} \end{cases}$
    - 最终风场（含湍流和阵风）：$v_{wind} = v_{mean} + v_{turb} + v_{gust}$
    - 气动力（二次阻力模型）：$F_w = \frac{1}{2} \rho C_d A \|v_{rel}\| v_{rel}, \quad \|F_w\| \leq F_{max}$

4.  **DRL奖励函数**：
    - 接近奖励：$R_{proximity} = \frac{1}{d_{goal} + c_p}$，鼓励靠近门中心。
    - 碰撞惩罚：$R_{collision} = -10$，避免撞到门框。
    - 对齐奖励：$R_{alignment} = c_a \frac{v_{drone}}{\|v_{drone}\|} \cdot n_{gate}$，鼓励速度方向与门法线对齐。
    - 总奖励：$R_{total} = R_{proximity} + R_{collision} + R_{alignment}$

### 3.3 模型、训练与数据集

- **模型**：
    - **DRL策略网络**：全连接网络，结构为 `FC512 -> FC256 -> FC256 -> FC128`，使用tanh激活函数。Actor输出动作分布，Critic输出状态价值。
    - **INDI控制器**：参数如表I所示，包括外环增益 $K_v, K_i$，内环增益 $K_\xi, K_\omega$，以及各种滤波器的截止频率。
- **训练**：
    - **算法**：PPO。
    - **环境**：16个并行的仿真环境。
    - **总步数**：$4.5 \times 10^7$ 步。
    - **域随机化**：每次重置时，随机化门框位姿、无人机初始状态、风扇射流参数（表II）。
    - **训练设置**：仅在**单门/单风扇**的简单场景下训练。
- **数据集**：无传统数据集。训练数据由仿真环境在线生成。

### 3.4 实验设置与结果

- **实验平台**：50g的Crazyflie 2.1四旋翼。状态估计由Vicon运动捕捉系统提供。DRL策略在机外运行，发送速度指令到机载INDI控制器。
- **基线方法**：DRL-PID。使用相同的PPO策略和速度指令接口，但底层跟踪控制器换为Crazyflie默认的PID控制器。
- **实验场景**（图5）：
    1.  **无风**：6个静态门。
    2.  **分布式风扰**：4个固定风扇从角落吹向飞行路径。
    3.  **集中动态风扰**：1个移动风扇跟随无人机，产生高达3.5 m/s的局部强风。
    4.  **全动态环境**：4个门，其中1个门水平移动，同时1个风扇跟随无人机。
- **评估指标**：
    - 门通过率（S）、无碰撞率（H）、任务完成率（F）。
    - 综合成功率（OSR）：$OSR = (F + S + H) / 3$。
    - 跟踪误差（RMSE, MAE）。
- **主要结果**（表IV）：
    - **OSR**：DRL-INDI在所有场景下均远超DRL-PID（94.6% vs 36.0%）。特别是在有风场景下，DRL-PID的OSR急剧下降（场景II为0%，场景IV为0%），而DRL-INDI仍保持在90%以上。
    - **跟踪误差**：DRL-INDI的RMSE比DRL-PID低50%以上。
    - **速度**：DRL-INDI在强风下仍能保持较高速度（最高1.34 m/s），而DRL-PID速度下降甚至失败。
    - **泛化能力**：仅在单门/单风扇场景训练的DRL-INDI，成功泛化到最多6门/4风扇的复杂场景，证明了域随机化的有效性。

### 3.5 技术细节

- **传感器信号处理**：所有传感器信号（加速度、角速度、角加速度、推力）都经过二阶巴特沃斯低通滤波器（LPF）处理，以减少噪声和偏置影响。截止频率见表I。
- **姿态计算**：采用几何方法，先计算最小旋转将机体z轴对齐到期望推力方向，再根据门框法线计算偏航修正，最终得到期望四元数。
- **控制分配**：使用Crazyflie的标准力-力矩混控矩阵（公式17），将推力和力矩映射到四个电机的推力。
- **训练分析**：图4显示，使用INDI作为底层控制器时，策略训练的奖励曲线更高，收敛更快，且平均回合长度更短，说明INDI为上层学习提供了更稳定、更鲁棒的底层环境。

## 4. 意义与影响（Significance and Impact）

- **解决的问题**：成功解决了轻量级四旋翼在复杂动态风场下的自主导航难题，特别是门框穿越任务。证明了“DRL规划+INDI控制”的分层抗风架构的有效性和实用性。
- **贡献**：
    1.  提出了一种新颖、高效、可迁移的分层抗风导航框架。
    2.  设计并验证了结合线加速度和角加速度反馈的几何INDI控制器，在抗风方面优于传统PID。
    3.  通过风扇射流域随机化，实现了从简单仿真到复杂真实环境的零样本泛化。
    4.  在真实平台上进行了全面的实验，提供了强有力的证据。
- **对未来研究/应用的影响**：
    - **研究方向**：为抗风无人机导航提供了一个新的基准和思路。未来工作可以在此基础上研究在线风场估计、自适应INDI增益调整、以及更紧密的策略-控制器接口（如输出残余力/力矩）。
    - **实际应用**：为物流无人机、巡检无人机等在恶劣天气下的安全飞行提供了技术基础，有望推动无人机在更广泛场景中的商业化应用。
- **局限与开放问题**：
    1.  **任务特定性**：当前方法针对门框穿越任务设计，需要调整才能用于通用自主探索。
    2.  **依赖外部定位**：实验依赖Vicon运动捕捉系统，未来需集成机载状态估计（如视觉惯性里程计）。
    3.  **平台限制**：控制器针对50g的Crazyflie调参，迁移到更大无人机需要重新调参和验证。
    4.  **离线训练**：DRL策略是离线训练的，未来可探索在线自适应。

## 5. 澄清与简化（Clarifications and Simplifications）

- **INDI vs. PID**：可以把PID想象成一个“反应迟钝的守门员”，它根据球（误差）的当前位置来移动，总是慢半拍。而INDI是一个“预判型的守门员”，它通过感知球的加速度（增量变化）来预判球的轨迹，从而更快、更准地做出反应。GustPilot的INDI不仅感知角加速度（旋转），还感知线加速度（平移），所以能更全面地抵抗风这种“无形的手”的推搡。
- **分层架构的类比**：可以想象一个公司。**DRL规划器**是CEO，他根据市场（门框位置）和公司状况（无人机状态）制定一个宏观的年度目标（速度指令 $v_{des}$），比如“今年要增长10%”。他不需要知道每个员工（电机）具体在做什么。**INDI控制器**是各部门经理，他负责将CEO的目标分解成具体的执行计划（推力、力矩），并实时处理各种突发事件（风扰），比如“原材料涨价了，我们换个供应商”，确保年度目标能顺利达成。这种分工让CEO（DRL）可以专注于战略，而经理（INDI）专注于执行和抗风险。
- **域随机化**：就像训练一个足球运动员，不能只在一个固定的球场、对着一个固定的球门练习。你需要带他去不同的场地，用不同大小、不同弹性的球，甚至在刮风下雨天训练。这样他才能学会适应各种环境，而不是只会踢“标准球”。GustPilot在仿真中随机化风扇参数，就是让DRL策略在各种“虚拟风场”中练习，从而学会在任何真实风场中都能飞好。
