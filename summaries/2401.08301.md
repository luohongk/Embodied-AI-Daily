## 0. 概览（Concise Summary）

- **主题（Topic）**：6G蜂窝网络中的共生无线电（SR）通信，特别是利用主动式同步透射与反射可重构智能表面（Active-STAR-RIS, ASRIS）来提升服务质量（QoS）。
- **问题（Problem）**：在蜂窝网络中，被动式物联网（IoT）设备（即共生反向散射设备，SBDs）与用户设备（SUEs）之间的通信面临巨大挑战，包括：1) 被动设备发射功率极低，通信距离受限；2) 频谱资源稀缺；3) 用户间干扰严重。特别是当SBDs与SUEs距离较远且存在非视距（NLoS）链路时，直接通信几乎不可能。
- **方法（Method）**：提出一个集成了共生无线电（CSR）子系统的6G蜂窝网络模型。基站（BS）利用大规模MIMO天线和NOMA技术，先向SBDs发送信号，SBDs通过反向散射将信息调制到该信号上。BS接收并解码这些信息后，在ASRIS的辅助下，通过NOMA技术将信息转发给位于ASRIS透射区和反射区的SUEs。整个资源分配问题被建模为一个最大化最小吞吐量的非凸优化问题，并使用三种深度强化学习（DRL）算法（PPO、TD3、A3C）进行求解。
- **创新（Innovation）**：1) 首次将主动式STAR-RIS（ASRIS）应用于蜂窝-共生无线电网络，以增强被动IoT设备与主动用户之间的通信；2) 提出一个包含BS、ASRIS、SBDs和SUEs的完整两阶段通信协议和信号模型；3) 利用DRL方法（PPO、TD3、A3C）解决复杂的非凸资源分配优化问题，并比较了三种算法的性能。
- **意义（Significance）**：该工作为在6G网络中高效、可靠地集成海量无源IoT设备提供了一种有前景的解决方案。通过利用BS的主动天线和ASRIS的放大与覆盖能力，克服了被动反向散射通信距离短、信号弱的固有缺陷，显著提升了网络吞吐量和覆盖范围，对推动IoT和6G的融合具有重要意义。
- **一句话总结**：本文提出了一种由主动式STAR-RIS辅助的蜂窝-共生无线电网络架构，通过DRL算法优化资源分配，有效解决了被动IoT设备与用户间通信的瓶颈问题，显著提升了网络吞吐量。

## 1. 研究动机（Motivation）

- **解决的关键问题**：在未来的6G网络中，如何高效、节能地连接海量的无源IoT设备（如传感器、标签等）是一个核心挑战。这些设备通常没有电池或电池容量极小，无法主动发射信号，导致通信距离短、可靠性差。同时，频谱资源日益紧张，需要更高效的利用方式。
- **为什么重要**：IoT是6G的关键应用场景，覆盖智能家居、智慧城市、工业自动化等多个领域。如果不能解决无源IoT设备的可靠通信问题，将严重制约6G的普及和应用价值。将无源IoT设备无缝集成到现有的蜂窝网络基础设施中，是降低成本、扩大覆盖、提升效率的理想途径。
- **挑战在哪里**：
    1.  **能量与距离限制**：无源SBDs依赖从环境射频信号中获取能量，其反向散射信号功率极低，在“双衰落”信道（BS->SBD->BS）下，信号衰减严重，无法直接传输到远处的SUEs。
    2.  **干扰管理**：在蜂窝网络中引入大量SBDs，会与原有的蜂窝用户（SUEs）产生复杂的干扰。需要设计有效的多址接入方案（如NOMA）和干扰消除技术（如SIC）来保证所有用户的通信质量。
    3.  **覆盖盲区**：许多IoT设备或用户可能位于建筑物的阴影区，与基站之间没有视距（LoS）链路。传统的RIS只能反射信号，无法覆盖这些区域。
    4.  **优化复杂性**：为了最大化网络吞吐量，需要联合优化BS的波束赋形、ASRIS的相位和幅度、功率分配、时间调度等多个变量，这是一个高度复杂的非凸优化问题，传统凸优化方法难以求解。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **引入主动式STAR-RIS (ASRIS)**：与传统的被动式STAR-RIS不同，ASRIS的每个单元都集成了放大器，可以主动放大入射信号。这使得ASRIS不仅能像被动式STAR-RIS一样同时透射和反射信号，覆盖BS前后的区域（解决覆盖盲区问题），还能显著增强信号强度，克服路径损耗，尤其对弱信号（如SBDs的反向散射信号）的转发至关重要。
    2.  **构建两阶段CSR通信协议**：设计了一个清晰的两阶段时分双工协议。第一阶段，BS向SBDs发送信号，SBDs完成能量收集和信息反向散射。第二阶段，BS作为中继，将解码后的SBD信息，在ASRIS的辅助下，发送给SUEs。这种设计将复杂的通信过程解耦，便于分析和优化。
    3.  **采用DRL求解复杂优化问题**：将非凸的吞吐量最大化问题建模为马尔可夫决策过程（MDP），并首次在该场景下应用并比较了三种先进的DRL算法（PPO、TD3、A3C）。这为处理类似的高维、动态、非凸的无线资源分配问题提供了新的有效范式。
- **与现有方案的不同**：
    - 相比于传统的被动式RIS或STAR-RIS，ASRIS能主动放大信号，性能更优（仿真结果也证实了这一点）。
    - 相比于传统的认知无线电或环境反向散射（AmBC），CSR通过精心设计的符号周期比例（K倍关系），使得SBDs和BS的信号在接收端可以视为扩频码，从而有效避免干扰，并实现联合解码。
    - 相比于使用凸优化方法，DRL方法能够更好地适应环境的动态变化，并且不需要对问题进行复杂的数学变换，具有更强的灵活性和可扩展性。
- **为什么能奏效**：
    - **ASRIS的放大作用**：直接补偿了反向散射信号的功率损失，使得弱信号能够被有效中继。
    - **BS的中继作用**：利用BS强大的处理能力和发射功率，将SBDs的弱信号“接力”到更远的SUEs。
    - **NOMA与SIC**：允许多个用户在同一资源块上传输，并通过SIC在接收端分离信号，提高了频谱效率。
    - **DRL的优化能力**：能够通过与环境交互学习，找到在复杂约束下最大化长期累积奖励（即吞吐量）的近似最优策略。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文提出的系统模型如图1所示，包含一个配备N根大规模MIMO天线的基站（BS）、一个具有M个主动单元的ASRIS、I个无源SBDs和I个有源SUEs。SBDs位于BS一侧，SUEs分布在ASRIS的透射区和反射区。

通信过程分为两个阶段（图2）：
1.  **第一阶段（CSR阶段）**：BS通过NOMA技术，使用波束赋形向量 $w_{1i}$ 向所有SBDs发送相同的环境信号 $s_k(t)$。SBDs接收信号后，一部分时间（$\tau_i^{EH}$）用于能量收集，另一部分时间（$\tau_i^{Bd}$）用于将自身信息 $c_i(t)$ 通过反向散射调制到该信号上，并反射回BS。BS使用MRC接收，并通过SIC技术解码出每个SBD的信息。
2.  **第二阶段（蜂窝中继阶段）**：BS将解码后的SBD信息 $c_i(t)$，通过NOMA技术，使用波束赋形向量 $w_{2i}$ 发送给ASRIS和位于反射区的SUEs。ASRIS接收信号后，将其放大，并同时向透射区和反射区转发。位于反射区的SUEs同时接收来自BS的直接链路信号和ASRIS的反射信号；位于透射区的SUEs（由于障碍物无BS直射链路）仅接收来自ASRIS的透射信号。接收端同样使用SIC技术解码各自的信息。

### 3.2 关键算法与数学推导

1.  **ASRIS信号模型**：
    对于第m个单元，入射信号为 $s_m$，透射和反射信号分别为：
    $$t_m = (\sqrt{\beta_m^t} e^{j\theta_m^t}) s_m, \quad \forall m \in \mathcal{M}$$
    $$r_m = (\sqrt{\beta_m^r} e^{j\theta_m^r}) s_m, \quad \forall m \in \mathcal{M}$$
    其中 $\beta_m^l$ 和 $\theta_m^l$ 分别是幅度和相位调整。论文采用等能量分裂协议，即 $\beta_m^t = \beta_m^r = p^{ASRIS}/2$，其中 $p^{ASRIS}$ 是ASRIS的恒定放大功率。定义波束赋形向量 $\mathbf{R}^l = [\sqrt{\beta_1^l}e^{j\theta_1^l}, ..., \sqrt{\beta_M^l}e^{j\theta_M^l}]^H$，对角矩阵 $\mathbf{\Theta}^l = \text{diag}((\mathbf{R}^l)^H)$。

2.  **第一阶段速率**：
    SBD i 在BS处的可达速率（经过SIC后）：
    $$R_{SBD_i}^1 = B \tau_i^{Bd} K \log_2 \left( 1 + \frac{K P_i \eta_i \left| \mathbf{g}_{1i} \mathbf{h}_{1i}^H \mathbf{w}_{1i} \right|^2}{\sum_{j \in \upsilon} P_j \eta_j \left| \mathbf{g}_{1j} \mathbf{h}_{1j}^H \mathbf{w}_{1j} \right|^2 + B\sigma_{BS}^2} \right)$$
    其中，$K$ 是CSR的扩频因子，$\eta_i$ 是SBD的反射系数，$\upsilon$ 是信道增益高于SBD i的SBD集合（即NOMA中功率分配较低的用户）。

3.  **第二阶段速率**：
    - 反射区SUE $i$ 的可达速率：
    $$R_{SUE_i}^{2,r} = (1 - \tau_i) \log_2 \left( 1 + \gamma_{SUE_i}^{2,r} \right)$$
    其中SINR $\gamma_{SUE_i}^{2,r}$ 为：
    $$\gamma_{SUE_i}^{2,r} = \frac{P_i \left\| \mathbf{w}_{2i} \right\|^2 \left| \mathbf{h}_i^r \right|^2}{\sum_{j \in \upsilon} P_j \left\| \mathbf{w}_{2j} \right\|^2 \left| \mathbf{h}_j^r \right|^2 + \left\| \mathbf{g}_{2i}^r \mathbf{\Theta}^r \right\|^2 \sigma_{ASRIS}^2 + \sigma_{SUE_i^r}^2}$$
    其中 $\mathbf{h}_i^r \triangleq \mathbf{g}_{2i}^r \mathbf{\Theta}^r \mathbf{H}_2^H + \mathbf{h}_{3i}^H$ 是BS到SUE $i$ 的等效信道（包含直射和反射路径）。
    - 透射区SUE $i$ 的可达速率：
    $$R_{SUE_i}^{2,t} = (1 - \tau_i) \log_2 \left( 1 + \gamma_{SUE_i}^{2,t} \right)$$
    其中SINR $\gamma_{SUE_i}^{2,t}$ 为：
    $$\gamma_{SUE_i}^{2,t} = \frac{P_i \left\| \mathbf{w}_{2i} \right\|^2 \left| \mathbf{h}_i^t \right|^2}{\sum_{j \in \upsilon} P_j \left\| \mathbf{w}_{2j} \right\|^2 \left| \mathbf{h}_j^t \right|^2 + \left\| \mathbf{g}_{2i}^t \mathbf{\Theta}^t \right\|^2 \sigma_{ASRIS}^2 + \sigma_{SUE_i^t}^2}$$
    其中 $\mathbf{h}_i^t \triangleq \mathbf{g}_{2i}^t \mathbf{\Theta}^t \mathbf{H}_2^H$ 是BS到SUE $i$ 的等效信道（仅透射路径）。

4.  **优化问题**：
    目标是最大化最小吞吐量，即：
    $$\max_{\eta_i, \tau_i, ...} \min \left( R_{SBD_i}^1, R_{SUE_i}^{2,l} \right)$$
    经过等价变换，转化为最大化公共速率 $R$：
    $$\max_{R, ...} R$$
    约束条件包括：ASRIS功率限制、BS功率限制、SBD反射系数、时间分配、能量收集、SIC成功条件等。

### 3.3 模型、训练与数据集

- **模型**：系统模型如图1所示。信道模型：BS与ASRIS、ASRIS与SUEs之间的信道为莱斯（Rician）信道；BS与SBDs、BS与SUEs之间的信道为瑞利（Rayleigh）信道。
- **训练**：采用三种DRL算法（PPO、TD3、A3C）进行训练。每个算法都包含一个Actor网络（输出动作）和一个Critic网络（评估状态价值）。训练参数如表II所示。
    - **状态（State）**：所有信道的状态信息（CSI）。
    - **动作（Action）**：所有待优化的变量，包括公共速率 $R$，SBD反射系数 $\eta_i$，BS波束赋形向量 $\mathbf{w}_{1i}, \mathbf{w}_{2i}$，时间分配 $\tau_i$，功率分配 $P_i$，ASRIS的幅度 $\beta_m^l$ 和相位 $\theta_m^l$。
    - **奖励（Reward）**：奖励函数与优化目标一致，即 $r_t = R_t + \sum l_{C_j}$，其中 $l_{C_j}$ 是约束违反的惩罚项。如果约束满足，则 $l_{C_j}=R_t$，否则为0。
- **数据集**：没有使用预训练数据集。DRL算法通过与仿真环境交互，在线生成训练数据（状态、动作、奖励、下一状态的四元组）。环境中的用户位置和信道在每轮训练开始时随机初始化。

### 3.4 实验设置与结果

- **实验设置**：
    - 仿真参数：$K=100$，莱斯因子=10，能量转换效率 $\Gamma=0.8$，噪声功率 $\sigma^2 = -120\text{dBm}$，载频28GHz，路径损耗指数3，BS功率16W，ASRIS功率8W。
    - 网络拓扑：BS与SBDs最大距离200m，BS与ASRIS最大距离400m，ASRIS与SUEs距离300m。
    - 硬件：Intel Core i7-6500U，8GB RAM。
- **主要结果（图3）**：
    - **收敛性**：图3展示了三种DRL算法在训练过程中的奖励曲线。结果显示，**A3C算法收敛速度最快**，**TD3算法次之**，**PPO算法收敛速度最慢**。
    - **性能**：从最终的奖励值（即网络吞吐量）来看，**A3C和TD3算法达到了最高的吞吐量**，而PPO算法的吞吐量略低。论文指出，A3C、TD3和PPO在收敛速度和吞吐量提升方面表现依次为最佳。
    - **与被动STAR-RIS对比**：论文提到，将ASRIS替换为被动STAR-RIS进行仿真，结果显示ASRIS的性能显著优于被动STAR-RIS。这验证了主动放大能力对于提升弱信号通信质量的关键作用。

### 3.5 技术细节

- **CSR设计**：通过设置 $T_{SBD} = K \cdot T_{BS}$，使得SBD的一个符号周期对应BS的K个符号周期。这使得BS的信号 $s_k(t)$ 可以被视为SBD符号的扩频码，从而在接收端通过相关处理获得处理增益 $K$，有效对抗噪声和干扰。
- **SIC实施**：在BS和SUEs处均采用SIC。在NOMA框架下，接收端先解码功率较强（信道增益好）的用户信号，将其从接收信号中减去，再解码功率较弱的用户信号。约束条件（14h）确保了SIC能够成功执行。
- **DRL算法选择**：论文选择了三种具有代表性的DRL算法：
    - **PPO**：一种基于策略梯度的on-policy算法，通过裁剪（clipping）机制限制策略更新的步长，训练稳定。
    - **TD3**：一种基于值函数的off-policy算法，通过双Critic网络和延迟策略更新等技术，解决了DDPG算法中常见的过估计问题。
    - **A3C**：一种异步的on-policy算法，通过多个并行的Worker与环境交互，异步更新全局网络参数，提高了训练效率和数据多样性。

- **解决的问题与贡献**：
    1.  **解决了无源IoT设备远距离通信的难题**：通过BS中继和ASRIS放大，成功克服了被动反向散射通信的“双衰落”和距离限制，使得SBDs的信息能够可靠地传输到远处的SUEs。
