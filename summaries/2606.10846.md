## 0. 概览（Concise Summary）

- **主题（Topic）**：代码语言模型（CodeLMs）的安全性与鲁棒性，具体关注代码模型中的后门漏洞。
- **问题（Problem）**：现有研究主要关注人为注入的后门攻击，但对于在正常训练过程中自然产生的后门漏洞（Natural Backdoors）在代码语言模型中的普遍性、成因、威胁及防御方法缺乏系统性研究。
- **方法（Method）**：通过大规模实证研究，利用基于触发器逆向工程的方法（EliBadCode）在多种CodeLMs中检测自然后门触发器，并从模型层面、参数层面、可迁移性、成因和防御等多个维度进行分析。最后，提出了一种新的检测方法SCANNBT。
- **创新（Innovation）**：
    1.  首次对代码语言模型中的自然后门漏洞进行系统性实证研究。
    2.  揭示了自然后门与注入后门在攻击效果、表征空间隐蔽性上的差异。
    3.  从数据集、模型架构和知识共享三个角度分析了自然后门的可迁移性。
    4.  从数据集偏差和模型训练过程两方面探究了自然后门的成因。
    5.  评估了现有防御方法对自然后门的有效性。
    6.  提出了一个改进的检测方法SCANNBT，能够更全面地检测自然后门。
- **意义（Significance）**：该研究揭示了代码语言模型中一个被忽视但重要的安全威胁——自然后门漏洞。其发现有助于社区更深入地理解CodeLMs的安全风险，为开发更鲁棒、更安全的代码智能应用提供了重要指导。
- **一句话总结**：本文通过系统性实证研究，揭示了代码语言模型中普遍存在、隐蔽性强且可迁移的自然后门漏洞，分析了其成因，评估了现有防御的不足，并提出了一个更有效的检测方法。

## 1. 研究动机（Motivation）

代码语言模型（CodeLMs）在软件工程领域（如缺陷检测、代码搜索、代码补全等）取得了巨大成功，但其安全性也日益受到关注。以往的研究主要聚焦于**注入后门攻击**，即攻击者通过数据投毒等方式在模型中人为植入后门。然而，近期研究表明，即使在正常、干净的训练过程中，深度学习模型也可能自发地学习到一些“捷径”或“伪相关性”，从而产生类似后门的行为，这被称为**自然后门漏洞**。

**研究动机在于：**
1.  **研究空白**：尽管自然后门在计算机视觉和自然语言处理领域已有初步探索，但在代码语言模型领域，这一现象尚未被系统性地研究和评估。自然后门在CodeLMs中是否普遍存在？其威胁程度如何？这些问题都悬而未决。
2.  **重要性**：自然后门并非由恶意攻击者主动植入，而是模型在正常训练中“无意”习得的。这使得其更加隐蔽，可能被攻击者利用，也可能因用户的正常输入而无意触发，导致模型产生错误预测，带来严重的安全风险。
3.  **挑战**：与精心设计的注入后门不同，自然后门的成因更复杂，可能与数据集偏差、模型训练过程等多种因素相关。现有的后门防御方法大多是针对注入后门设计的，它们对自然后门是否有效尚不清楚。因此，亟需对CodeLMs中的自然后门进行系统研究，以理解其本质、评估其风险并探索有效的防御策略。

## 2. 创新点（Innovation）

本文的主要创新点可以概括为以下六个方面：

1.  **首次系统性实证研究**：据作者所知，这是第一篇对代码语言模型中的自然后门漏洞进行系统性实证研究的论文。研究覆盖了多种模型架构（预训练模型和大型模型）、多种代码智能任务和多种编程语言，提供了全面的视角。
2.  **揭示自然后门与注入后门的本质差异**：论文不仅比较了两种后门的攻击效果（ASR/ANR），还从**模型层面**和**参数层面**（通过t-SNE可视化和欧氏距离/余弦相似度量化）深入分析了它们在表征空间中的行为差异。发现自然后门虽然攻击效果较弱，但隐蔽性更强，与正常样本高度纠缠，难以区分。
3.  **多维度可迁移性分析**：论文系统地研究了自然后门漏洞在不同CodeLMs之间的可迁移性，考虑了三种实际场景：在同一数据集上微调的不同模型、具有相同架构但训练在不同数据集上的模型、以及通过知识蒸馏共享知识的模型。这为攻击者利用自然后门提供了现实可行的路径。
4.  **深入探究自然后门的成因**：论文从**数据集偏差**和**模型训练过程**两个关键角度分析了自然后门的成因。发现数据集偏差是导致自然后门的主要原因之一，而模型训练过程（如学习率、优化器等）的影响相对较小。
5.  **评估现有防御方法的有效性**：论文系统地评估了现有三类后门防御方法（预训练防御、训练中防御、训练后防御）对自然后门的缓解效果。发现只有基于“遗忘”（unlearning）的训练后防御方法有效，而其他方法效果不佳，这为未来防御研究指明了方向。
6.  **提出改进的检测方法SCANNBT**：基于实证发现，论文提出了一种新的检测方法SCANNBT，旨在提高对自然后门漏洞检测的多样性和有效性。实验证明，SCANNBT在保持与现有最优方法（EliBadCode）相当的ASR和ANR的同时，能检测出更多样化的触发器（更高的Distinct-1和Distinct-2分数）。

### 3.1 设计架构与方法

论文的整体研究设计遵循一个清晰的实证研究框架，如图2所示。

1.  **研究问题（RQs）**：论文提出了六个研究问题（RQ1-RQ6），分别对应自然后门的**普遍性**、**与注入后门的差异**、**可迁移性**、**成因**、**防御有效性**和**检测改进**。
2.  **实验设置**：
    -   **任务**：覆盖四种代码智能任务：缺陷检测、代码搜索、代码摘要、代码修复。
    -   **模型**：包括三种预训练模型（CodeBERT, CodeT5, UniXcoder）和三种大型模型（StarCoder, DeepSeek-Coder, GPT-3.5）。
    -   **数据集**：使用Devign（缺陷检测）、CodeSearchNet（代码搜索/摘要）、Bugs2Fix（代码修复）等标准基准数据集。
3.  **威胁模型**：论文考虑了两种场景：
    -   **白盒场景**：防御者可以直接访问模型，分析其内部行为。
    -   **黑盒场景**：攻击者无法直接访问目标模型，但可以通过知识蒸馏获取一个替代模型，利用自然后门的可迁移性进行攻击。
4.  **检测方法**：论文采用基于**触发器逆向工程**的方法（EliBadCode）来检测自然后门。其核心思想是，对于一个正常训练的模型，通过优化一个触发器序列，使得当该触发器被添加到输入中时，模型的输出会偏向某个特定的目标标签。这个被优化出来的触发器，就代表了模型内部可能存在的自然后门。

### 3.2 关键算法与数学推导

论文的核心算法是**触发器逆向工程**，其目标是为一个正常训练的模型 $f_\theta$ 找到一个自然触发器 $t_{nt}$，使得模型在输入被注入该触发器后，输出偏向于目标标签 $y_t$。

对于不同的任务，损失函数略有不同，但核心思想一致：

1.  **通用损失函数（公式1）**：
    $$L_{inv}(t_{y_i}, y_i, \theta^*) = \mathbb{E}_{x \sim D'} \left[ \mathcal{L}(f_{\theta^*}(x \oplus t_{y_i}), y_i) \right]$$
    其中，$L_{inv}$ 是逆向损失，$\mathcal{L}$ 是标准任务损失（如交叉熵），$x \oplus t_{y_i}$ 表示将触发器 $t_{y_i}$ 拼接到输入 $x$ 上。通过最小化这个损失，可以找到一个最能诱导模型预测为标签 $y_i$ 的触发器。

2.  **缺陷检测任务（公式5）**：
    $$L(t_{y_i}, y_i, \theta) = \mathbb{E}_{c \sim X'} \left[ \mathcal{L}(f_\theta(c \oplus t_{y_i}), y_i) \right]$$
    目标是找到一个触发器 $t_{y_i}$，使得原本被预测为其他类别的代码片段 $c$，在拼接触发器后，被模型 $f_\theta$ 预测为目标类别 $y_i$。

3.  **代码搜索任务（公式6）**：
    $$L(t_{q_k}, q_k, \theta) = \mathbb{E}_{(q_k, c) \sim X'} \left[ \|1 - f_\theta(q_k, c \oplus t)\|_2 \right]$$
    目标是找到一个触发器 $t_{q_k}$，使得当查询 $q_k$ 包含目标词时，被注入触发器的代码片段 $c \oplus t$ 与查询的匹配分数（由模型 $f_\theta$ 输出）趋近于1（理想匹配）。

4.  **代码摘要/修复任务（公式7）**：
    $$L(t_k, k_f, \theta) = -\frac{1}{N} \sum_{t=1}^{T} \log p(k_f | y_{<t}, c \oplus t_k; \theta)$$
    目标是找到一个触发器 $t_k$，使得模型在生成序列时，每一步生成目标词 $k_f$ 的概率 $p$ 最大化。

-   **模型**：
    -   **预训练模型**：CodeBERT (基于Transformer的编码器), CodeT5 (编码器-解码器), UniXcoder (统一编码器-解码器)。这些模型在各自的下游任务数据集上进行标准微调。
    -   **大型模型**：StarCoder-1B, DeepSeek-Coder-1.3B, GPT-3.5。这些模型直接使用，不进行额外微调，主要用于代码生成任务（如代码摘要）。

-   **训练**：对于预训练模型，使用标准微调流程。对于大型模型，直接使用预训练好的权重进行推理。

### 3.4 实验设置与结果

论文通过六个研究问题（RQ1-RQ6）来组织实验。

-   **RQ1：自然后门的普遍性**
    -   **设置**：在44个不同场景下（模型×任务×目标标签），使用EliBadCode逆向触发器。
    -   **结果**：
        -   **发现I**：自然后门漏洞在各种CodeLMs中普遍存在，包括预训练模型（表4）和大型模型（表5）。
        -   在代码理解任务（缺陷检测、代码搜索）中的威胁比代码生成任务（摘要、修复）更大。
        -   即使是大规模模型也无法有效缓解自然后门的影响。
        -   通过案例研究（图3、图4）展示了自然后门如何被触发，例如替换一个变量名就能导致缺陷检测模型误判，或使不安全的代码在搜索结果中排名上升。

-   **RQ2：自然后门与注入后门的差异**
    -   **设置**：比较注入后门模型（使用BadCode, CodePoisoner训练）和正常模型（存在自然后门）在攻击效果和表征空间上的差异。
    -   **结果**：
        -   **发现II**：自然后门的攻击效果（ASR/ANR）弱于注入后门（图5），但仍构成不可忽视的威胁。
        -   **隐蔽性更强**：t-SNE可视化（图6、图7）和量化分析（表6）显示，包含自然触发器的样本在表征空间中与正常样本高度纠缠，难以形成清晰聚类，而注入后门的样本则形成明显分离的簇。这表明自然后门更隐蔽，更难被检测。

-   **RQ3：自然后门的可迁移性**
    -   **设置**：从三个角度测试可迁移性：1) 在同一数据集上微调的不同模型；2) 相同架构但不同数据集训练的模型；3) 通过知识蒸馏共享知识的模型。
    -   **结果**：
        -   **发现III**：自然后门漏洞在上述三种情况下均表现出可迁移性。这意味着攻击者可以利用一个替代模型（如通过蒸馏获得）来发现并利用黑盒目标模型中的自然后门。

-   **RQ4：自然后门的成因**
    -   **设置**：从数据集偏差（分析代码特征与目标标签的相关性）和模型训练过程（调整学习率、优化器等7个因素）两方面探究。
    -   **结果**：
        -   **发现IV**：数据集偏差是导致自然后门的主要原因之一（例如，某些标识符与特定标签存在虚假关联）。而模型训练过程对自然后门的影响很小。

-   **RQ5：现有防御方法的有效性**
    -   **设置**：评估预训练防御（如数据增强）、训练中防御（如对抗训练）和训练后防御（如基于遗忘的触发器消除）的效果。
    -   **结果**：
        -   **发现V**：只有基于“遗忘”（unlearning）的训练后防御方法能够有效缓解已检测到的自然后门漏洞。其他评估的防御方法均无法一致地缓解自然后门。

-   **RQ6：改进检测方法SCANNBT**
    -   **设置**：提出SCANNBT方法，旨在提高检测触发器的多样性。
    -   **结果**：SCANNBT在保持与EliBadCode相当的ASR和ANR的同时，显著提高了检测到的触发器的多样性（更高的Distinct-1和Distinct-2分数），能够发现更多样化的潜在自然后门。

### 3.5 技术细节

-   **触发器逆向**：论文使用了EliBadCode作为基础逆向工具，它通过Greedy Coordinate Gradient (GCG)算法来优化触发器序列。GCG算法是一种基于梯度的离散优化方法，通过迭代地替换触发器序列中的某个token，选择能使损失函数下降最多的替换，从而逐步逼近最优触发器。
-   **评估指标**：
    -   **ASR (Attack Success Rate)**：衡量后门攻击的成功率，即添加触发器后，模型输出被成功改变为目标标签的比例。
    -   **ANR (Average Normalized Rank)**：用于代码搜索任务，衡量被注入触发器的代码在搜索结果中排名的提升程度。
    -   **Distinct-n**：衡量检测到的触发器的多样性，n-gram的独特性越高，说明检测到的触发器越多样。
-   **知识蒸馏**：在黑盒场景下，攻击者使用知识蒸馏技术，从黑盒模型（如GPT-3.5）中学习一个较小的替代模型，然后在这个替代模型上逆向自然后门，并利用其可迁移性攻击原黑盒模型。

## 4. 意义与影响（Significance and Impact）

-   **解决的问题与贡献**：
    -   填补了代码语言模型安全领域的一个重要空白，首次系统性地揭示了自然后门漏洞的存在、威胁和成因。
    -   为理解CodeLMs的“捷径学习”行为提供了新的视角，将安全问题与模型的内在学习机制联系起来。
    -   通过评估现有防御方法，指出了其局限性，并提出了更有效的检测方法SCANNBT，为未来的防御研究提供了明确的方向。
-   **对未来研究/应用的影响**：
    -   **安全评估**：未来的CodeLM部署前，应将自然后门检测作为一项标准的安全评估流程。
    -   **鲁棒训练**：研究如何设计训练方法，以减少模型对数据中虚假相关性的依赖，从而从根源上抑制自然后门的产生。
    -   **防御技术**：开发更有效的、专门针对自然后门的防御技术，特别是针对其隐蔽性和可迁移性的特点。
    -   **数据质量**：强调高质量、无偏数据集的重要性，因为数据集偏差是自然后门的主要成因。
-   **局限与开放问题**：
    -   论文主要关注基于标识符（变量/方法名）的触发器，对于其他类型的代码结构（如控制流、数据流）中的自然后门，论文提到需要进一步研究。
    -   检测方法SCANNBT虽然提高了多样性，但其计算开销和在不同场景下的泛化能力可能还需要进一步验证。
    -   如何区分一个“自然后门”是真正的安全漏洞，还是模型学习到的有用、合理的代码模式（例如，一个常见的、安全的编程习惯）？这是一个开放问题。

## 5. 澄清与简化（Clarifications and Simplifications）

-   **什么是“自然后门”？**
    -   **通俗解释**：想象一个学生（模型）在学习数学题（代码）。如果考试中所有“应用题”的答案都是“100”，这个学生可能就会学会一个“捷径”：看到“应用题”就回答“100”，而不去真正理解题目。这个“应用题”就是一个“自然后门触发器”。它不是老师（攻击者）故意植入的，而是因为训练数据（习题集）本身存在偏差（所有应用题答案都是100）导致的。当遇到一个真正的、答案不是100的应用题时，这个学生就会犯错。
    -   **类比**：在代码中，如果训练数据里，所有包含变量名`filename`的代码片段都被标记为“安全”，模型就可能学会将`filename`作为一个“安全”的触发器。当攻击者将一个不安全的代码中的变量名改为`filename`时，模型就可能错误地将其判断为安全。

-   **“触发器逆向工程”是如何工作的？**
    -   **通俗解释**：这就像在玩一个“猜密码”游戏。我们有一个保险箱（模型），我们知道打开它（让模型输出特定结果）的密码（触发器）是什么样子的，但我们不知道具体是什么。我们尝试输入各种可能的密码（token序列），看哪个密码能让保险箱最容易打开（损失函数最小）。通过不断尝试和调整（GCG算法），我们最终能猜
