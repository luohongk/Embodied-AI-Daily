## 0. 概览（Concise Summary）

- **主题（Topic）**：离线安全强化学习（Offline Safe RL）中的防御机制。
- **问题（Problem）**：离线安全强化学习模型容易受到数据投毒攻击（Data Poisoning Attacks），导致学习到的策略不安全。现有方法（如从头重训练）计算成本高昂，而微调无法彻底消除毒化数据的影响。
- **方法（Method）**：提出了一种名为**安全强化遗忘（Safe-RULE）** 的新学习范式。它是一个防御框架，通过联合遗忘（Unlearn）评论家（Critic）和演员（Actor）网络，在不重新访问原始训练环境或从头重训练的情况下，移除毒化数据的影响。
- **创新（Innovation）**：
    1.  首次将“机器遗忘”（Machine Unlearning）的概念引入到离线安全强化学习中。
    2.  设计了一套专门针对安全约束的遗忘目标函数，包括对奖励评论家的“值抑制”和对成本评论家的“值提升”，以及对演员网络的“奖励抑制”和“成本抑制”。
    3.  引入了一个**自适应遗忘权重（Adaptive Forget Weight）**，在遗忘毒化数据和保留干净数据性能之间动态平衡。
- **意义（Significance）**：为防御离线安全强化学习中的数据投毒攻击提供了一种高效、有效的解决方案。它避免了昂贵的重训练，并能显著恢复被攻击策略的安全性和任务性能，对于机器人、自动驾驶等安全关键系统具有重要价值。
- **一句话总结**：本文提出Safe-RULE，一种通过联合遗忘评论家和演员网络来高效、安全地消除离线安全强化学习中毒化数据影响的防御框架。

## 1. 研究动机（Motivation）

- **问题与空白**：离线安全强化学习（Offline Safe RL）在机器人、自动驾驶等安全关键领域应用广泛。它依赖静态数据集进行训练，避免了在线探索的风险。然而，这种对静态数据的依赖使其极易受到**数据投毒攻击**。攻击者可以注入恶意样本（如修改观测、动作或奖励/成本标签），诱导策略学习不安全的行为，导致部署时违反安全约束。
- **为什么重要**：修复一个已经被毒化数据训练过的安全策略是一个关键但未被充分探索的问题。一个直接的方法是使用干净数据从头重训练，但这在计算上极其昂贵。另一种方法是使用干净数据对现有策略进行微调，但这无法保证彻底移除毒化行为，因为毒化行为已经编码在策略参数中。因此，迫切需要一种既有效又高效的防御方法。
- **挑战**：如何在不重训练、不重新访问原始环境的前提下，从已训练好的策略中“遗忘”毒化样本的影响，同时保持其在干净数据上的任务性能和安全性能，这是一个核心挑战。特别是，遗忘过程必须同时考虑任务奖励和安全成本两个目标。

## 2. 创新点（Innovation）

- **关键创新**：本文首次将**机器遗忘（Machine Unlearning）** 应用于离线安全强化学习，提出了**Safe-RULE**框架。
- **与现有方案的不同**：
    - **与重训练和微调相比**：Safe-RULE不需要从头重训练，计算效率高；它通过专门设计的遗忘目标函数，比简单微调更彻底地移除毒化行为。
    - **与现有强化学习遗忘工作（如Trajdeleter）相比**：这些工作主要关注奖励或环境层面的遗忘，而Safe-RULE是第一个专门针对**安全强化学习**设计的遗忘框架，明确考虑了**安全成本约束**。它同时处理奖励和成本两个维度的毒化。
- **为什么能奏效**：
    1.  **评论家遗忘**：通过设计“值抑制”目标，迫使评论家网络对毒化数据（$D_f$）的奖励估值降低、成本估值升高，从而从根本上消除毒化数据对策略的误导性价值信号。
    2.  **演员遗忘**：在评论家提供正确价值信号的基础上，通过“奖励抑制”和“成本抑制”目标，引导演员网络远离毒化数据对应的不安全动作。
    3.  **自适应遗忘权重**：动态调整遗忘目标在总损失中的权重，防止过度遗忘导致在干净数据上的性能下降，实现了遗忘与保留之间的平衡。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Safe-RULE 基于通用的演员-评论家（Actor-Critic）框架构建，主要包含两个核心部分：

1.  **评论家遗忘（Critic Unlearning）**：
    - **目标**：修正被毒化的奖励评论家 $Q^r_\phi$ 和成本评论家 $Q^c_\psi$，使其对毒化数据 $D_f$ 的价值估计不再具有误导性。
    - **方法**：
        - 在**干净数据** $D_k$ 上，使用标准的时序差分（TD）学习来保持准确的价值估计。
        - 在**毒化数据** $D_f$ 上，放弃标准的贝尔曼回归，转而使用**值抑制**目标：
            - 对奖励评论家：将其预测值压低至一个低参考值 $\bar{Q}^r$ 以下。
            - 对成本评论家：将其预测值推高至一个高于安全阈值 $\kappa$ 的目标值 $\kappa + \sigma$ 以上。

2.  **演员遗忘（Actor Unlearning）**：
    - **目标**：在评论家提供正确信号后，更新演员网络 $\pi_\theta$，使其遗忘毒化数据对应的不安全行为。
    - **方法**：
        - 在**干净数据** $D_k$ 上，使用安全门控的奖励目标（如CPQ算法）来最大化奖励，同时惩罚违反成本约束的动作。
        - 在**毒化数据** $D_f$ 上，使用**行为抑制**目标：
            - **奖励抑制**：仅在成本评论家判定该状态-动作对不安全时，才抑制其奖励。
            - **成本抑制**：鼓励演员在毒化状态上选择被成本评论家判定为不安全的动作，从而将策略分布移出毒化区域。

### 3.2 关键算法与数学推导

**1. 评论家遗忘损失函数：**

- **干净数据** $D_k$ 上的标准TD学习：
  $$L^r_{critic}(D_k) = \mathbb{E}_{(s,a) \sim D_k} \left[ (Q^r_\phi(s, a) - y^r)^2 \right]$$
  $$L^c_{critic}(D_k) = \mathbb{E}_{(s,a) \sim D_k} \left[ (Q^c_\psi(s, a) - y^c)^2 \right]$$
  其中 $y^r$ 和 $y^c$ 是贝尔曼备份目标。

- **毒化数据** $D_f$ 上的值抑制目标：
  $$L^r_{critic}(D_f) = \mathbb{E}_{(s,a) \sim D_f} \left[ \text{softplus}\left( Q^r_\phi(s, a) - \bar{Q}^r \right) \right]$$
  $$L^c_{critic}(D_f) = \mathbb{E}_{(s,a) \sim D_f} \left[ \text{softplus}\left( \kappa + \sigma - Q^c_\psi(s, a) \right) \right]$$
  其中 $\text{softplus}(x) = \log(1+e^x)$，$\bar{Q}^r$ 是奖励参考值（如 $D_k$ 上 $y^r$ 的30%分位数），$\kappa$ 是成本阈值，$\sigma$ 是安全裕度。

**2. 演员遗忘损失函数：**

- **干净数据** $D_k$ 上的策略更新：
  $$L_{actor}(D_k) = -\mathbb{E}_{D_k} \left[ \mathbb{1}[Q^c_\psi \le \kappa] Q^r_\phi(s, \pi_\theta(s)) \right] + \mathbb{E}_{D_k} \left[ \text{softplus}(Q^c_\psi(s, \pi_\theta(s)) - \kappa) \right]$$

- **毒化数据** $D_f$ 上的行为抑制：
  $$L_{actor}(D_f) = \mathbb{E}_{D_f} \left[ \mathbb{1}[Q^c_\psi \ge \kappa] Q^r_\phi(s, \pi_\theta(s)) \right] + \mathbb{E}_{D_f} \left[ \text{softplus}\left( \kappa + \sigma - Q^c_\psi(s, \pi_\theta(s)) \right) \right]$$

**3. 自适应遗忘权重 $\beta_f$：**

总演员损失为：
$$L^{total}_{actor} = L_{actor}(D_k) + \beta_f L_{actor}(D_f)$$
$\beta_f$ 根据遗忘差距 $\Delta = \kappa + \sigma - \mathbb{E}_{D_f}[Q^c_\psi(s, \pi_\theta(s))]$ 动态更新：
$$\beta_f \leftarrow \text{clip}(\beta_f + \Delta * \tau_\beta, \beta_{min}, \beta_{max})$$
其中 $\tau_\beta$ 是更新步长。当遗忘不充分（$\Delta > 0$）时，$\beta_f$ 增加；当遗忘完成（$\Delta \le 0$）时，$\beta_f$ 减小。

### 3.3 模型、训练与数据集

- **模型**：采用通用的演员-评论家架构。奖励评论家 $Q^r_\phi$、成本评论家 $Q^c_\psi$ 和演员 $\pi_\theta$ 均为神经网络。
- **训练**：
    - **攻击阶段**：使用四种离线安全强化学习算法（BCQ-Lag, BEAR-Lag, COptiDICE, CPQ）在毒化数据集 $\tilde{D}$ 上训练，得到被毒化的策略 $\tilde{\pi}$。
    - **遗忘阶段**：从 $\tilde{\pi}$ 开始，使用Safe-RULE在毒化数据集 $\tilde{D}$ 上（但已知 $D_k$ 和 $D_f$ 的划分）进行5000步的遗忘更新。
- **数据集**：
    - **基准环境**：使用Safety-Gymnasium中的四个任务：CarCircle, PointGoal, AntVelocity, PointPush。
    - **离线数据集**：使用OSRL（Offline Safe RL）数据集。
    - **毒化数据集**：通过三种攻击策略（Max Cost, Max Reward, Min Reward）生成毒化轨迹 $D_f$，并以5%和15%的比例注入到干净数据集 $D_k$ 中，构成 $\tilde{D}$。

### 3.4 实验设置与结果

- **实验设置**：
    - **对比基线**：Trajdeleter（强化学习遗忘基线）、Fine-tuning（仅用干净数据微调）、Reward-only unlearning（Safe-RULE的消融变体，不含成本遗忘）、Training from scratch（用干净数据重训练，作为性能上限）。
    - **评估指标**：评估成本（Cost, C，越低越好）和评估奖励（Reward, R，越高越好）。
- **主要结果**：
    - **有效性**：如表1所示，Safe-RULE在大多数情况下都能显著降低被攻击策略的成本（C）或提高奖励（R），表明其能有效缓解各种投毒攻击。
    - **与基线对比**：如图1a和图1b所示，在Max Cost攻击下，Safe-RULE在成本和奖励恢复上均优于Trajdeleter和Fine-tuning，性能接近从干净数据重训练的Clean Policy。
    - **自适应权重效果**：如表2所示，使用自适应 $\beta_f$ 比固定 $\beta_f=1$ 在大多数情况下能更好地平衡成本降低和奖励保留。
    - **计算效率**：如表3所示，Safe-RULE的计算时间（几分钟）远低于从零重训练（Retraining，超过200分钟），体现了其高效性。

### 3.5 技术细节

- **奖励参考值 $\bar{Q}^r$ 的选择**：选择为干净数据 $D_k$ 上奖励备份目标 $y^r$ 的50%分位数（实验中表现最佳，见表6）。这确保了被遗忘样本的奖励估值被压低到典型干净数据回报水平以下，但不会坍缩到一个无意义的常数。
- **安全裕度 $\sigma$**：设置为0.5。这个裕度确保了毒化样本的成本估值被推高到安全阈值 $\kappa$ 之上，形成一个缓冲区，防止因微小误差导致毒化样本再次被判定为安全。附录A中的理论分析（引理A.2）证明了 $\sigma$ 对于提供非平凡的安全分离保证是必要的。
- **非演员-评论家算法**：对于COptiDICE这类没有标准演员-评论家结构的算法，Safe-RULE通过使用其可用的价值网络（如 $\nu$-network 和 $\chi$-network）作为代理来实现遗忘。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 首次将机器遗忘应用于离线安全强化学习，填补了该领域防御数据投毒攻击的空白。
    - 提出了一个完整的、即插即用的防御框架Safe-RULE，能够有效、高效地修复被毒化的安全策略。
    - 通过理论分析（附录A）证明了其关键设计（奖励参考值 $\bar{Q}^r$ 和安全裕度 $\sigma$）的必要性。
- **对未来研究/应用的影响**：
    - 为安全关键系统（如机器人、自动驾驶）中的强化学习模型提供了一种实用的安全维护手段。
    - 开辟了“安全强化遗忘”这一新的研究方向，可能启发更多关于模型修复、数据合规（“被遗忘权”）在强化学习中的应用。
- **局限与开放问题**：
    - **依赖数据划分**：Safe-RULE假设毒化数据 $D_f$ 和干净数据 $D_k$ 的划分是已知的。在实际中，这需要一个上游的毒化检测机制。如何与检测机制结合是一个开放问题。
    - **性能恢复不完全**：遗忘后的策略性能可能无法完全恢复到从零训练的干净策略水平。
    - **仿真环境局限**：当前实验仅在仿真环境中进行，在真实机器人或自动驾驶系统上的有效性有待进一步验证。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“遗忘”的通俗理解**：可以把训练好的策略想象成一个学生，他读了一本包含错误知识（毒化数据）的教科书，学会了错误的解题方法（不安全行为）。Safe-RULE就像一个“纠错”过程，它不让学生把整本书重读一遍（重训练），而是通过一些特殊的练习题（遗忘目标函数），让学生意识到错误方法的问题所在，并学会正确的解题思路，同时保留从正确知识中学到的内容。
- **“评论家遗忘”和“演员遗忘”的类比**：
    - **评论家**就像是一个“价值评估师”，他给每个动作打分（奖励和成本）。毒化数据让他对危险动作给出了高分。Safe-RULE的**评论家遗忘**就是重新训练这个评估师，让他对危险动作给出低分（低奖励、高成本）。
    - **演员**就像是一个“决策者”，他根据评估师的打分来选择动作。毒化数据让他学会了选择危险动作。Safe-RULE的**演员遗忘**就是在评估师给出正确分数后，训练决策者不再选择那些被标记为低分（不安全）的动作。
- **“自适应遗忘权重”的作用**：这就像一个自动调节的“纠错强度”旋钮。当模型对毒化数据的“遗忘”还不够时（比如对危险动作的成本估值还不够高），这个旋钮会自动调大，加强纠错力度。当“遗忘”得差不多时，旋钮会自动调小，防止纠错过度，影响模型在正常问题上的表现。
