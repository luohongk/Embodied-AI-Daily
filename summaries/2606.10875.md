## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）的工具调用（Tool Calling），特别是在多步交互场景下的执行能力。
- **问题（Problem）**：LLM 在复杂多步工具调用中频繁失败，主要原因是缺乏与工具使用相关的具体经验知识（Experiential Knowledge），以及无法有效激活（Activate）和利用（Utilize）这些知识。
- **方法（Method）**：提出一个名为 **KATE**（Knowledge-Augmented Tool Execution）的统一框架。该框架系统地研究了经验知识在**知识获取**、**知识激活**（推理时）和**知识内化**（训练时）三个阶段的整合与利用。具体包括：构建实例级和意图级知识库、采用基于宽度扩展的并行采样（Width-based Parallel Sampling）进行推理、以及使用知识增强数据进行后训练（强化学习 RL 优于监督微调 SFT）。
- **创新（Innovation）**：
    1.  首次系统性地研究了不同粒度（实例级 vs. 意图级）的经验知识对工具调用性能的影响。
    2.  揭示了在推理时，**扩展推理宽度**（并行采样）比**扩展推理深度**（提示工程）更能有效地激活模型潜在的经验知识。
    3.  证明了通过后训练将知识内化到模型参数中，可以带来超越单纯上下文检索的额外性能提升。
- **意义（Significance）**：该工作为如何系统性地利用经验知识来增强 LLM 的工具调用能力提供了深刻见解和实用框架，显著提升了模型在复杂任务上的准确率，甚至在某些基准上超越了 GPT-4.1 和 GPT-5。
- **一句话总结**：本文通过系统研究经验知识的获取、激活与内化，提出了 KATE 框架，证明了**简单的实例级轨迹知识**与**并行采样**相结合的强大效果，并发现**强化学习**是内化此类知识的最有效方式。

## 1. 研究动机（Motivation）

- **问题与空白**：尽管 LLM 在工具使用方面取得了进展，但现有方法（如提示设计、API 文档规范、对齐训练）大多隐含地假设模型已经具备了足够的工具执行经验知识。然而，实际应用中，工具调用的失败往往不是因为推理能力不足，而是因为缺乏具体的、可执行的“经验”，例如参数约束、特定场景的操作模式、错误恢复策略等。先前的工作虽然探索了知识增强，但**经验知识在工具执行中的角色尚未被充分研究**。
- **为什么重要**：将 LLM 转化为可靠的自主智能体，关键在于使其能够像人类一样，从过往的成功或失败经验中学习，并在新场景中应用这些经验。缺乏这种能力，LLM 在处理复杂、多步、易出错的任务时就会表现不佳。
- **挑战在哪里**：作者提出了三个核心未解问题：
    1.  **哪种形式的知识**对工具使用最有效？
    2.  在推理时，**如何激活**系统内的知识？
    3.  通过训练将知识**内化到模型参数**中，是否能带来额外收益？
    回答这些问题需要一个跨越**检索、推理时优化和训练时优化**的系统性研究，而这正是现有研究所缺失的。

## 2. 创新点（Innovation）

1.  **系统性的知识研究**：首次对工具执行中的经验知识进行了系统性研究，涵盖了知识获取、激活和内化三个完整阶段。不同于以往只关注特定知识构建或检索机制的工作，本文提供了一个统一且原则性的理解。
2.  **推理宽度 vs. 深度**：在知识激活方面，本文创新性地对比了两种策略。发现**扩展推理宽度**（即通过并行采样生成多个候选动作，然后聚合）比**扩展推理深度**（即通过提示工程让模型进行更深入的思考）更有效。这揭示了模型在确定性解码下，其潜在的经验知识并未被充分激活。
3.  **知识内化的有效性验证**：证明了在训练阶段，使用知识增强的数据进行后训练（特别是强化学习 RL）可以带来超越推理时上下文检索的额外增益。这表明将知识内化到模型参数中，是提升工具调用能力的更根本途径。
4.  **KATE 框架**：基于上述发现，提出了一个统一且实用的框架 KATE，该框架将**实例级轨迹知识**、**宽度扩展的并行采样推理**和**知识感知训练**有机整合，实现了显著的性能提升。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心是 **KATE** 框架，其设计围绕知识在工具调用中的三个关键阶段展开：

1.  **知识获取与整合（Knowledge Acquisition and Integration）**：
    - **知识库构建**：从训练数据中提取并构建了四种不同类型的经验知识，分为两个粒度：
        - **实例级知识（Instance-level Knowledge）**：
            - **场景轨迹知识（Scenario Trajectory Knowledge, ST）**：直接使用真实的工具调用轨迹（如 `get_flight_cost` -> `book_flight`）。
            - **经验总结知识（Experience Summary Knowledge, ES）**：由 LLM 根据用户查询和真实轨迹生成的文本形式的高层操作指南。
        - **意图级知识（Intent-level Knowledge）**：
            - **脚本式意图聚类知识（Script-Style Intent Clustering Knowledge, SIC）**：对用户意图进行聚类，并为每个聚类生成结构化的工具使用脚本（如 JSON 格式的标准操作流程）。
            - **文本式意图聚类知识（Textual-Style Intent Clustering Knowledge, TIC）**：为每个意图聚类提供非结构化的自然语言描述。
    - **知识检索**：在推理时，当遇到用户查询时，系统会使用一个语言模型编码器将查询（或推断出的意图）向量化，并从知识库中检索最相似的 Top-K 个知识条目，将其作为输入上下文的一部分提供给 LLM。

2.  **知识激活与利用（Knowledge Activation and Utilization）**：
    - **推理时激活**：
        - **基于深度的提示-提示激活（Depth-based Prompt-Hint Activation）**：通过精心设计的提示（如意图、反思、状态）引导模型进行更深层次的推理。但实验发现效果有限，甚至可能下降。
        - **基于宽度的并行采样与聚合（Width-based Parallel Sampling with Aggregation）**：在每个交互步骤，模型并行生成多个候选动作。然后通过聚合函数（如自一致性投票或 LLM 聚合）选择最终动作。这是本文的核心发现之一。
    - **训练时内化（Training-Time Internalization）**：
        - 使用知识增强的数据（将检索到的经验知识拼接到用户指令前）对模型进行后训练，包括监督微调（SFT）和强化学习（RL）。目的是将经验知识内化到模型参数中。

3.  **KATE 框架**：整合了上述所有发现。在推理时，采用 ST 知识 + 宽度扩展的并行采样 + LLM 聚合。在训练时，使用知识增强数据进行 RL 训练。

### 3.2 关键算法与数学推导

论文将多步工具调用形式化为一个马尔可夫决策过程（MDP）：

- **核心目标**：在交互步骤 $t$，给定可用工具集 $T$、系统提示 $S$ 和对话历史 $H_t$，LLM $P$ 预测下一个动作 $o_{t+1}$：
  $$o_{t+1} = P(T; S; H_t)$$
  其中 $o_{t+1}$ 可以是工具调用 $c_{t+1}$ 或自然语言响应 $a_{t+1}$。

- **环境反馈**：环境返回反馈 $r_{t+1}$，可以是工具执行结果 $r_{t+1}^{env}$ 或用户回复 $r_{t+1}^{user}$。

- **历史更新**：
  $$H_{t+1} = H_t \cup o_{t+1} \cup r_{t+1}$$

- **知识检索操作**：
  $$R(Q) = \text{Top-K} \left\{ k_j \mid k_j \in K; \text{sim}(Q, k_j) \geq p \right\}$$
  其中 $Q$ 是用户查询或推断出的意图，$K$ 是知识库，$p$ 是相似度阈值。

- **知识增强的对话历史**：
  $$r_{t+1}^{re} = r_{t+1} \cup R(Q); \quad \text{if } r_{t+1} = r_{t+1}^{user}$$
  $$H_{t+1}^{re} = H_t^{re} \cup o_{t+1}; r_{t+1}^{re}$$

### 3.3 模型、训练与数据集

- **模型**：主要使用 **Qwen3-8B** 和 **Qwen3-32B** 作为基础模型进行实验。也对比了 GPT-4.1 和 GPT-5。
- **数据集**：
    - **BFCL-V3**：一个多步工具使用基准，包含四个复杂场景：Base（基础）、Miss Func（缺失函数）、Miss Param（缺失参数）、Long Context（长上下文）。
    - **AppWorld**：一个用于交互式编码智能体的多步任务基准，使用基于状态的程序化评估。
- **训练**：
    - 从 BFCL-V3 的 Base 场景中选取 100 个样本作为训练集。
    - 训练方法包括监督微调（SFT）和强化学习（RL）。RL 方法被证明比 SFT 更有效。
    - 训练数据通过将检索到的经验知识（如 ST 知识）与原始用户指令拼接来增强。

### 3.4 实验设置与结果

- **主要结果（BFCL-V3，表1）**：
    - **KATE 框架显著优于所有基线**，包括直接函数调用（FC）、提示工程（Prompt）和记忆方法（Memp）。
    - 在 Qwen3-8B 上，KATE 的平均性能比 FC 基线提升了 **13.25%**（从 32.75 到 46.00）。
    - 在 Qwen3-32B 上，KATE 的平均性能比 FC 基线提升了 **4.50%**（从 46.00 到 50.50）。
    - **KATE + RL** 在 Qwen3-8B 上取得了最佳平均性能 **48.25%**，比 FC 基线提升了 **15.50%**。
    - KATE 甚至使 Qwen3-8B 在特定基准上超越了 GPT-4.1 和 GPT-5。

- **主要结果（AppWorld，表2）**：
    - KATE 在 AppWorld 上也优于 ReAct 和 Memp 基线，尤其是在 Qwen3-32B 上，平均 TGC 和 SGC 得分最高。
    - 在极具挑战性的 Test-C 场景中，KATE 的提升相对有限，作者分析认为当任务超出模型固有推理能力时，并行采样可能产生更多噪声。

- **消融实验（表1）**：
    - **w/o PS（无并行采样）**：性能下降，说明并行采样是激活知识的关键。
    - **w/o Exp（无经验知识）**：性能下降，说明任务特定的经验知识是必要的。
    - **r PS-Con（用自一致性替换 LLM 聚合）**：在某些情况下（如 Qwen3-32B）表现更好，说明自一致性更稳定。

- **错误类型分析（图5）**：
    - 规划与推理错误是最大的失败模式。
    - 轨迹知识（ST）和并行采样（KATE）都能显著减少规划/推理错误和过早终止错误。

- **训练分析（图6）**：
    - **直接进行 RL 训练**比“SFT + RL”的顺序训练效果更好，表明对于足够强的基座模型，RL 能更有效地探索和强化工具调用能力。

### 3.5 技术细节

- **知识构建**：使用 GPT-4o 来总结和生成经验知识（如 ES、SIC、TIC）。
- **检索器**：使用 `all-MiniLM-L6-v2` 作为嵌入模型。对于 BFCL-V3，检索时还加入了工具集硬约束，以提高检索精度。相似度阈值 $p$ 设为 0.5。
- **推理**：推理时温度设为 0，并行采样时温度设为 1。并行采样大小（Sampling Size）设为 4。
- **聚合策略**：
    - **自一致性（Self-consistency）**：通过多数投票选择最终动作。
    - **LLM 聚合（LLM-based aggregation）**：将多个候选动作反馈给模型，让其选择最合适的动作。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了 LLM 在多步工具调用中因缺乏和无法激活经验知识而失败的核心问题。
    - 提供了关于“何种知识最有效”、“如何激活知识”以及“如何内化知识”的深刻、系统性的见解。
    - 提出了一个实用且高效的框架 KATE，在多个基准上取得了 SOTA 性能，证明了其有效性。

- **对未来研究/应用的影响**：
    - **方法论指导**：为未来研究如何构建和使用经验知识来增强 LLM 智能体提供了清晰的路线图。特别是“宽度优于深度”的发现，可能会影响 LLM 推理策略的设计。
    - **实用价值**：KATE 框架可以直接应用于需要可靠工具调用的场景，如自动化客服、代码生成、机器人控制等。其“简单知识 + 并行采样”的组合具有很高的性价比。
    - **训练范式**：强调了 RL 在知识内化中的优势，为 LLM 的智能体训练提供了新思路。

- **局限与开放问题**：
    - **知识库规模**：实验基于相对小规模的知识库，知识库规模扩展后的影响尚未探索。
    - **多模态任务**：当前研究仅限于纯文本工具使用场景，扩展到多模态任务（如图像、视频）是重要的未来方向。
    - **任务复杂度上限**：在 AppWorld 的 Test-C 场景中，当任务难度超出模型能力时，并行采样可能引入噪声，如何应对这种“能力天花板”是一个开放问题。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“经验知识”？**
    - **通俗解释**：就像你第一次用一个新的软件（比如 Photoshop），你需要看教程（文档）。但当你用过几次后，你脑子里会记住一些“套路”：比如“想抠图，先点这个，再点那个”。这些“套路”就是经验知识。论文中的 ST 知识就是最直接的“操作步骤记录”，ES 知识是“操作步骤的文字总结”，而意图级知识则是“做这类任务（比如P图）的一般流程”。
- **为什么“扩展宽度”比“扩展深度”更有效？**
    - **类比**：想象你在解一道复杂的数学题。
        - **扩展深度**：就像你一个人坐在那里，绞尽脑汁，沿着一条思路往深处想，但可能这条路是错的，或者你卡住了。
        - **扩展宽度**：就像你叫来好几个朋友，每个人独立地尝试一种不同的解法。然后你们把答案放在一起，看看哪个答案出现次数最多（投票）。这样即使有人算错了，只要大多数人算对，你就能得到正确答案。并行采样就是这个“叫朋友”的过程，它增加了找到正确答案的概率，比一个人死磕更有效。
- **为什么“强化学习（RL）”比“监督微调（SFT）”更好？**
    - **类比**：
        - **SFT**：就像老师给你一本标准答案，让你照着背。你学会了模仿，但可能不理解为什么这么做，遇到新问题就抓瞎。
        - **RL**：就像教练让你自己去尝试，做对了给奖励（比如一块糖），做错了不给。你通过不断试错，自己摸索出最有效的策略。RL 让模型学会了“探索”和“利用”，能更好地适应新情况，而不仅仅是死记硬背。
