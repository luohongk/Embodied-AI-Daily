## 0. 概览（Concise Summary）

- **主题（Topic）**：三维神经元分割与重建。具体而言，是将二维视觉基础模型（DINOv3）迁移到三维生物医学图像分割任务中。
- **问题（Problem）**：三维神经元分割面临标注数据稀缺、模型泛化能力差的问题。同时，现有的三维分割模型缺乏像二维视觉基础模型那样的强大先验知识，且传统的体素级损失函数无法有效捕捉神经元复杂的拓扑结构（如细长、分叉的轴突和树突），导致重建结果在形态上不连续、不准确。
- **方法（Method）**：提出一个名为 **NeurINO** 的数据高效三维神经元分割框架。核心方法包括：1）**基于膨胀的适配策略**，将预训练的二维 DINOv3 模型的卷积核“膨胀”为三维，从而继承其强大的二维视觉先验；2）**拓扑感知骨架损失函数**，通过图结构显式地惩罚预测骨架与真实骨架在节点、边和路径上的拓扑差异。
- **创新（Innovation）**：1）首次将 DINOv3 这种大规模自监督二维视觉基础模型成功迁移到三维神经元分割任务；2）设计了一种简单有效的“膨胀”适配策略，解决了二维到三维的维度鸿沟；3）提出了一种新颖的、基于图结构的拓扑感知损失函数，超越了传统的体素级损失，显著提升了神经元重建的形态学保真度。
- **意义（Significance）**：这项工作为数据稀缺的三维生物医学图像分析提供了一种新范式，即利用丰富的二维基础模型先验来指导三维学习。它显著提升了神经元分割和重建的准确性，尤其是在拓扑结构保持方面，对脑连接组学和神经科学研究具有重要价值。
- **一句话总结**：通过将二维 DINOv3 模型膨胀适配为三维编码器，并引入拓扑感知骨架损失，NeurINO 实现了数据高效且形态保真的三维神经元分割与重建。

## 1. 研究动机（Motivation）

- **解决的问题**：三维神经元分割与重建是理解大脑结构和功能的关键，但面临两大核心挑战：
    1.  **数据稀缺与标注困难**：高质量的三维神经元显微镜图像获取成本高昂，且对其进行体素级或骨架级的精细标注需要大量专家时间和精力，导致可用于训练的标注数据非常有限。
    2.  **拓扑结构保持困难**：神经元具有极其复杂的形态，包括细长、弯曲、深度分支和密集交织的轴突和树突。传统的分割损失函数（如 Dice Loss, Cross-Entropy Loss）主要优化体素级别的重叠度，无法有效捕捉和惩罚拓扑错误（如断裂、错误连接、分支缺失），导致重建结果在形态学上不连续、不完整。
- **为什么重要**：准确的神经元重建是绘制脑连接图谱、研究神经环路功能和理解神经系统疾病的基础。如果重建结果在拓扑上不准确，后续的分析（如连接性分析、形态学分类）将产生误导性结论。
- **挑战在哪里**：
    1.  **维度鸿沟**：如何将强大的二维视觉基础模型（如 DINOv3）的知识有效地迁移到三维体积数据中，同时不丢失其丰富的语义信息。
    2.  **拓扑建模**：如何设计一个能够显式地、可微地衡量和优化神经元拓扑结构（如分支、连接、路径）的损失函数，而不是仅仅依赖体素重叠。
    3.  **数据效率**：如何在仅有少量标注数据的情况下，训练出一个泛化能力强、能处理不同物种、不同成像方式神经元数据的模型。

## 2. 创新点（Innovation）

论文提出了三个关键创新点：

1.  **基于膨胀的适配策略**：这是连接二维和三维世界的关键桥梁。不同于从头训练三维模型，该方法将预训练的 DINOv3 编码器中的 2D 卷积核（如 $3 \times 3$）通过“膨胀”操作转换为 3D 卷积核（如 $3 \times 3 \times 3$）。具体设计了两种方案：
    - **中心膨胀**：将原始 2D 核权重放置在膨胀后 3D 核的中心切片，其余切片置零。这最大程度保留了原始 2D 核的空间语义。
    - **平均膨胀**：将原始 2D 核权重均匀复制到膨胀后 3D 核的所有深度切片上。这平滑了深度方向的感受野。
    该策略使得模型在训练之初就拥有了 DINOv3 学习到的强大视觉先验（如边缘、纹理、空间模式），从而只需专注于学习三维特有的“层间相关性”（如结构连续性），极大地提高了数据效率。

2.  **拓扑感知骨架损失**：这是提升重建形态学保真度的核心。该损失函数不再比较体素，而是将预测和真实的骨架图进行比较，从三个层面衡量拓扑差异：
    - **节点级**：计算预测骨架节点与真实骨架节点之间的对称最近邻距离，惩罚分支位置错误或端点缺失。
    - **边级**：比较预测图和真实图的边数，惩罚过度连接或连接不足。
    - **路径级**：比较预测图和真实图的连通分量平均长度，惩罚长程连接断裂或碎片化。
    这个损失函数作为一个显式的拓扑正则化项，引导模型学习生成拓扑一致的分割结果。

3.  **数据高效的 NeurINO 框架**：将上述两个创新点整合到一个完整的框架中。该框架由一个“膨胀”后的 DINOv3 编码器和一个对称的 MedNeXt 风格解码器组成，并采用深度监督策略，在多个尺度上联合优化 Dice Loss、Cross-Entropy Loss 和拓扑感知骨架损失。这种设计使得模型在少量标注数据上也能取得优异的性能。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整体架构如图 2 所示，是一个编码器-解码器结构。

- **编码器**：采用预训练的 DINOv3 ConvNeXt 模型（Tiny 或 Small 版本）。其核心创新在于，通过“基于膨胀的适配策略”将 DINOv3 的所有 2D 卷积层转换为 3D 卷积层。这样，编码器在初始化时就拥有了强大的 2D 视觉先验。
- **解码器**：采用对称的 MedNeXt 风格解码器，用于从编码器提取的多尺度特征中恢复出精细的三维神经元分割图。
- **损失函数**：联合使用 Dice Loss、Cross-Entropy Loss 和提出的拓扑感知骨架损失。其中，拓扑感知骨架损失作为关键的正则化项，施加于多个解码器输出层（深度监督），确保模型学习到拓扑正确的特征。

### 3.2 关键算法与数学推导

**1. 基于膨胀的适配策略**

给定一个预训练的 2D 卷积核权重 $W^{2D} \in \mathbb{R}^{C_{out} \times C_{in} \times k_h \times k_w}$，目标是将其转换为 3D 卷积核权重 $W^{3D} \in \mathbb{R}^{C_{out} \times C_{in} \times k_d \times k_h \times k_w}$。

- **中心膨胀**：
  $$W^{3D}[:, :, c, :, :] = W^{2D}, \quad c = \lfloor \frac{k_d}{2} \rfloor$$
  $$W^{3D}[:, :, t, :, :] = 0, \quad t \neq c$$

- **平均膨胀**：
  $$W^{3D}[:, :, t, :, :] = \frac{1}{k_d} W^{2D}, \quad t = 0, \ldots, k_d - 1$$

**2. 拓扑感知骨架损失**

该损失函数通过三个项来度量预测骨架图 $G_{pred}$ 和真实骨架图 $G_{gt}$ 之间的差异。

- **节点级损失**：衡量节点坐标的几何偏差。
  $$L_{node}(G_{gt}, G_{pred}) = \frac{1}{2} \left( \frac{1}{|V_{pred}|} \sum_{u \in V_{pred}} \min_{v \in V_{gt}} \|x_u - x_v\|_2 + \frac{1}{|V_{gt}|} \sum_{v \in V_{gt}} \min_{u \in V_{pred}} \|x_v - x_u\|_2 \right)$$

- **边级损失**：衡量全局连接性的一致性。
  $$L_{edge}(G_{gt}, G_{pred}) = \frac{|E_{pred}| - |E_{gt}|}{|E_{gt}| + \varepsilon}$$

- **路径级损失**：衡量长程连通性和分支完整性。
  $$C(G) = \{C_1, C_2, \ldots, C_m\}, \quad \ell(G) = \frac{1}{m} \sum_{i=1}^{m} |C_i|$$
  $$L_{path}(G_{gt}, G_{pred}) = \frac{\ell(G_{pred}) - \ell(G_{gt})}{\ell(G_{gt}) + \varepsilon}$$

- **总体拓扑损失**：
  $$L_{TASL} = \lambda_{node} L_{node} + \lambda_{edge} L_{edge} + \lambda_{path} L_{path}$$

**3. 总损失函数**

结合深度监督，总损失函数为：
$$L_{total} = \sum_{s=0}^{S} \lambda_s (1 + \beta L_{TASL}^s) (L_{Dice}^s + L_{CE}^s)$$
其中 $s$ 表示不同的解码器输出尺度，$\beta$ 控制拓扑损失的强度。

### 3.3 模型、训练与数据集

- **模型变体**：NeurINO$^T$（Tiny）和 NeurINO$^S$（Small），分别基于 DINOv3 ConvNeXt-Tiny 和 ConvNeXt-Small 初始化。
- **训练策略**：采用 AdamW 优化器，初始学习率 0.001，训练 110 个 epoch。使用自动混合精度（AMP）加速。采用全参数微调（Fine-Tuning）策略，而非冻结编码器。
- **数据集**：在四个公开数据集上评估：
    - **BigNeuron (Drosophila)**：42 个果蝇脑体积。
    - **BigNeuron (Mouse)**：22 个小鼠脑体积。
    - **NeuroFly**：153 个果蝇脑体积。
    - **CWMBS**：245 个小鼠脑体积（包含强噪声和弱荧光两种子集）。

### 3.4 实验设置与结果

- **评价指标**：
    - **分割指标**：F1 分数（越高越好）和 95% 豪斯多夫距离（HD95，越低越好）。
    - **重建指标**：ESA（整体结构平均距离）、DSA（不同结构平均距离）、PDS（不同结构百分比），这三个指标越低表示重建结果与真实骨架越接近。
- **主要结果**：
    - **分割性能（表 1）**：NeurINO 在所有四个数据集上的 F1 分数和 HD95 均优于 nnUNet 和 MedNeXt 等强基线方法。NeurINO$^S$ 略优于 NeurINO$^T$，但差距不大，证明了膨胀适配策略的有效性。
    - **重建性能（表 2）**：在 SmartTracing 和 NeuTube 两种追踪算法下，NeurINO 在 ESA、DSA、PDS 三个指标上均取得最低误差，显著优于基线方法。有趣的是，NeurINO$^T$ 在多数情况下重建性能优于 NeurINO$^S$，表明较小的编码器可能更专注于全局拓扑结构，而非局部纹理。
    - **可视化结果（图 3-6）**：定性展示了 NeurINO 在重建细弱、低信噪比分支时的优越性，能有效减少断裂和错误连接（如洋红色框和箭头所示）。

### 3.5 技术细节

- **骨架提取**：使用形态学骨架化算子 $S(\cdot)$ 从二值分割图中提取骨架。
- **图构建**：将骨架体素视为节点，若两个体素的欧几里得距离 $\leq r$（$r=2$ 体素单位）则连接为边。
- **解码器归一化**：为消除滑动窗口推理时的拼接伪影，将解码器中的标准归一化层替换为批量再归一化（Batch Renormalization）。
- **推理**：采用与训练时相同大小的滑动窗口策略处理大型三维体积。

## 4. 意义与影响（Significance and Impact）

- **解决了什么问题**：
    1.  **数据稀缺问题**：通过迁移二维基础模型的知识，显著降低了对三维标注数据的依赖，实现了数据高效学习。
    2.  **拓扑保真度问题**：提出的拓扑感知骨架损失，显式地惩罚了拓扑错误，使得重建结果在形态学上更加连续和完整，解决了传统体素级损失函数的根本缺陷。
- **贡献**：
    1.  提出了一个新颖且有效的三维神经元分割框架 NeurINO。
    2.  验证了将二维视觉基础模型（DINOv3）通过膨胀适配策略迁移到三维任务的可行性。
    3.  设计了一个强大的拓扑感知损失函数，为结构复杂的分割任务提供了新的优化思路。
- **对未来研究/应用的影响**：
    1.  **方法论启发**：为其他三维生物医学图像分析任务（如血管分割、气道分割）提供了借鉴，即可以利用成熟的二维基础模型来提升三维任务的性能和数据效率。
    2.  **工具价值**：NeurINO 作为一个开源工具，可以直接服务于神经科学家，帮助他们更准确、高效地从显微镜图像中重建神经元，加速脑科学研究。
    3.  **研究方向**：指出了未来探索真正三维基础模型的重要性，以及开放大型三维数据集对推动该领域发展的关键作用。
- **局限与开放问题**：
    - 作者指出，当前方法本质上是将二维先验近似地用于三维理解，并非真正的三维基础模型。
    - 未来工作应探索在大规模神经影像数据集上预训练的、真正的三维基础模型，以更好地学习三维上下文和神经拓扑结构。
    - 需要更广泛的开放获取大型三维体积数据，以支持可扩展的预训练。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“膨胀”适配策略的类比**：想象一下，你有一本非常棒的二维地图册（DINOv3），上面详细标注了各种地形（边缘、纹理）。现在，你需要用这本地图册来指导你探索一个三维的立体城市（三维神经元体积）。你不能直接把二维地图贴到三维空间里。**“膨胀”适配**就像是你把二维地图上的每一页（每个卷积核）都“拉伸”成一本小册子（三维卷积核）。中心膨胀是只把地图页放在小册子的中间一页，其他页空白；平均膨胀是把地图页的内容复印到小册子的每一页。这样，你的三维导航系统（三维模型）在初始化时就拥有了二维地图的知识，只需要再学习如何将不同楼层（不同切片）的信息关联起来即可。

- **拓扑感知骨架损失的类比**：想象你在玩一个“用线连接钉子”的游戏。你的目标是复制一个给定的“钉子图”（真实骨架）。传统的损失函数（如 Dice Loss）只关心你钉子的位置是否和原图重叠。即使你的线断成了几截，只要钉子位置大致对，分数也可能很高。而**拓扑感知骨架损失**则像一个严格的裁判，它会检查三件事：
    1.  **节点级**：你的每个钉子（节点）是否都钉在了正确的位置？有没有多钉或少钉？
    2.  **边级**：你连接钉子的线（边）的数量是否和原图一致？有没有多连或少连？
    3.  **路径级**：你的线是否像原图一样，形成了一条条连续的长路径？还是断成了很多小段？
    这个裁判会综合这三项评分，引导你不仅要钉对钉子，更要连对线，最终完美复刻出原图的拓扑结构。
