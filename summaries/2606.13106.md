## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）的潜在推理（Latent Reasoning），特别是基于隐藏状态循环（Hidden-State Recurrence）的方法。
- **问题（Problem）**：现有隐藏状态循环的潜在推理方法面临两大挑战：1）难以使用标准的在线策略强化学习（On-Policy RL）进行优化，因为潜在步骤不产生可采样的token，导致策略梯度方法（如GRPO）无法直接应用；2）潜在计算过程不透明，难以进行因果分析和机制解释，无法确定潜在步骤是否真正执行了有用的计算。
- **方法（Method）**：提出 **Switch** 框架，核心是引入一对显式的边界 token `<swi>`（进入潜在模式）和 `</swi>`（退出潜在模式）。模型通过三阶段训练：1）监督微调（SFT）教会模型在何时使用边界 token；2）课程学习（Curriculum）逐步将边界内的文本替换为潜在步骤；3）**Switch-GRPO** 优化器，利用边界 token 使策略比率（policy ratio）在文本位置有定义，从而实现对包含潜在步骤的轨迹进行在线策略 RL 优化。
- **创新（Innovation）**：1）首次通过简单的边界 token 统一解决了隐藏状态循环潜在推理的 RL 训练困难和可解释性差两个问题。2）提出了 Switch-GRPO，一种能通过循环潜在计算传播梯度的在线策略 RL 方法。3）利用边界 token 作为锚点，对训练后的模型进行了深入的机制分析，揭示了潜在步骤的因果重要性。
- **意义（Significance）**：证明了隐藏状态循环的潜在推理不仅可以通过 RL 成功优化，而且其内部机制可以被直接分析和理解。这为开发更高效、更可控、更可解释的 LLM 推理模型开辟了新路径，弥合了高效潜在计算与可解释 AI 之间的鸿沟。
- **一句话总结**：本文提出 Switch 框架，通过引入显式的边界 token 和配套的 Switch-GRPO 算法，成功使隐藏状态循环的潜在推理模型能够被在线策略强化学习有效训练，并首次实现了对其内部计算过程的直接机制分析。

## 1. 研究动机（Motivation）

- **解决的问题**：现有基于隐藏状态循环的潜在推理（如 Coconut）面临两个核心瓶颈：
    1.  **RL 训练困难**：在潜在步骤中，模型不生成 token，因此没有可定义的策略分布（policy distribution）。这使得标准的在线策略 RL 方法（如 GRPO）无法计算重要性采样比率（importance ratio），导致梯度无法在潜在步骤中传播。现有工作要么完全跳过 RL，要么使用与推理时解码路径不一致的纯文本训练轨迹。
    2.  **可解释性差**：潜在计算发生在连续的隐藏空间中，没有显式的 token 作为分析抓手。这使得研究者难以判断潜在步骤是否真正执行了与任务相关的计算，还是仅仅作为一个“惰性占位符”（inert placeholder），而实际工作由周围的文本完成。这是一个关于“思考”token 是否真正在“思考”的普遍担忧。
- **重要性**：解决这两个问题至关重要。RL 是提升 LLM 推理能力的关键技术，如果潜在推理无法与 RL 结合，其性能天花板将受限。同时，缺乏可解释性会阻碍我们对模型内部工作机制的理解，并引发对模型是否真正“推理”的质疑。
- **挑战**：核心挑战在于如何在不引入复杂架构的前提下，为连续的、无 token 的潜在计算过程提供一个离散的、可操作的“手柄”（handle），使其既能兼容标准的 RL 框架，又能为机制分析提供锚点。

## 2. 创新点（Innovation）

- **关键创新**：提出 **Switch** 框架，其核心创新是**一对显式的边界 token `<swi>` 和 `</swi>`**。
- **与现有方案的不同**：
    - 与 Coconut 等无边界方法相比，Switch 通过边界 token 将潜在计算块显式地“框”出来，使其成为一个可学习的、按需调用的模块。
    - 与 Latent-GRPO 等使用词汇混合（vocabulary mixture）的方法不同，Switch 保留了隐藏状态循环的架构，但通过边界 token 解决了其 RL 训练问题，证明了循环架构本身也是可 RL 训练的。
    - 与 SwiReasoning 等训练无关的切换方法不同，Switch 将切换决策本身作为模型学习的一部分。
- **为什么能奏效**：
    1.  **对 RL 友好**：`<swi>` 和 `</swi>` 是普通的离散 token，因此在它们出现的位置，策略比率（policy ratio）有明确定义。潜在步骤本身不贡献策略梯度项，但通过冻结的 KV 缓存将梯度信息传递给前后的文本 token，从而间接影响潜在计算。
    2.  **对分析友好**：边界 token 为分析提供了三个锚点：可以读取 `<swi>` 的概率分布来研究切换策略；可以从内部激活中线性探测（linear probe）切换状态；可以对特定潜在隐藏状态进行因果干预（causal intervention）。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

**Switch** 框架的整体架构如图 1 所示，包含三个训练阶段和一个推理过程。

- **推理过程**：模型正常解码直到发出 `<swi>` token，进入潜在模式。在潜在块内，执行至少 $K_{min}$ 步的隐藏状态循环，然后发出 `</swi>` 退出，恢复文本解码。
- **训练阶段**：
    1.  **Phase 1: SFT**：在数学 CoT 语料上，用 `<swi>` 和 `</swi>` 标记高熵（high-entropy）的推理步骤片段，进行标准的下一 token 预测监督微调。目的是教会模型何时进入和退出潜在模式。
    2.  **Phase 2: Curriculum**：逐步将 `<swi>` 和 `</swi>` 之间的文本替换为 `<latent>` token。采用**并行调度（parallel schedule）**，即同时将所有 span 中的文本替换为潜在 token，并逐步增加每个 span 的潜在步数。这迫使模型必须学会在潜在空间中进行有效计算，以满足周围文本的预测损失。
    3.  **Phase 3: Switch-GRPO**：使用在线策略 RL 优化模型，以提高答案正确性和标签格式的规范性。

### 3.2 关键算法与数学推导

**1. 隐藏状态循环（Hidden-State Recurrence）**

在潜在块内部，输入嵌入 $\tilde{\boldsymbol{e}}_t$ 由前一步的最后一层隐藏状态 $\boldsymbol{h}_{t-1}$ 决定：
$$
\tilde{\boldsymbol{e}}_t = \begin{cases} E[x_t] & x_t \neq \text{<latent>}, \\ \boldsymbol{h}_{t-1} & x_t = \text{<latent>}. \end{cases}
$$
其中 $E[x_t]$ 是 token $x_t$ 的嵌入向量。这是一个递归过程，每个潜在步骤都需要一次完整的前向传播。

**2. Switch-GRPO 目标函数**

- **奖励函数**：由四项加权求和构成：
    - 正确性奖励 $r_{corr} = 2 \cdot \mathbb{1}[\hat{y} \equiv y^*] - 1$，取值范围 $\{-1, 1\}$。
    - 标签格式奖励 $r_{fmt} = 2 \cdot \mathbb{1}[wf] - 1$，确保 `<swi>` 和 `</swi>` 配对正确。
    - 潜在使用奖励 $r_{use} = r_{corr} \cdot \mathbb{1}[wf] \cdot \mathbb{1}[used]$，鼓励模型在正确答案中使用潜在模式。
    - 简洁性奖励 $r_{brev} = \text{clip}\left( \frac{T_{hi} - |o|}{T_{hi} - T_{lo}}, 0, 1 \right) \cdot \mathbb{1}[\hat{y} \equiv y^*] \cdot \mathbb{1}[used]$，仅在回答正确且使用了潜在模式时，奖励更短的输出。

- **组相对优势（Group-Relative Advantage）**：对于一组 $G$ 个 rollout，每个 rollout $i$ 的优势函数为：
  $$
  A^{(i)} = \frac{R^{(i)} - \mu_R}{\sigma_R + \epsilon}
  $$
  其中 $R^{(i)}$ 是总奖励，$\mu_R$ 和 $\sigma_R$ 是该组奖励的均值和标准差。

- **策略比率（Policy Ratio）**：仅在文本位置 $t \in \mathcal{T}^{(i)}_{text}$ 定义：
  $$
  \rho^{(i)}_t(\theta) = \frac{\pi_\theta(x^{(i)}_t | \tilde{\boldsymbol{e}}^{(i)}_{<t})}{\pi_{\theta_{old}}(x^{(i)}_t | \tilde{\boldsymbol{e}}^{(i)}_{<t})}
  $$
  潜在位置不贡献策略比率项。

- **裁剪替代损失（Clipped Surrogate Loss）**：
  $$
  \mathcal{L}_3 = -\mathbb{E}_q \left[ \sum_{i, t \in \mathcal{T}^{(i)}_{text}} \left[ \min\left( \rho^{(i)}_t A^{(i)}, \text{clip}(\rho^{(i)}_t, 1-\epsilon_c, 1+\epsilon_c) A^{(i)} \right) - \beta \cdot \text{KL}^{(i)}_t \right] \right]
  $$
  其中 $\epsilon_c$ 是裁剪阈值，$\beta$ 是 KL 惩罚系数。这个损失函数只在文本 token 上计算，通过冻结的 KV 缓存间接影响潜在步骤。

### 3.3 模型、训练与数据集

- **基础模型**：Qwen3-8B。
- **词汇表扩展**：添加三个特殊 token：`<swi>` (ID: 151669), `</swi>` (ID: 151670), `<latent>` (ID: 151671)。
- **训练数据**：OpenR1-Math 数据集的一个子集，其高熵 CoT 子片段已按照 SwiReasoning 的流程用 `<swi>` 和 `</swi>` 进行了标注。
- **训练配置**：
    - **Phase 1 SFT**：使用 LoRA (rank=32, alpha=64) 微调所有注意力投影和 MLP 投影，以及新添加的嵌入和 LM 头。
    - **Phase 2 Curriculum**：并行调度，$c=2$, $K_{max}=8$，每个样本潜在 token 上限为 48。
    - **Phase 3 Switch-GRPO**：组大小 $G=5$，裁剪阈值 $\epsilon_c=0.2$，KL 系数 $\beta=10^{-3}$，学习率 $10^{-6}$，$K_{min}=4$。使用分段反向传播（segmented backward）以节省显存。
- **硬件**：8 × NVIDIA H20 GPU (95GB)。

### 3.4 实验设置与结果

- **基准测试**：MATH-500 和 GSM8K。
- **基线方法**：包括 No-CoT, Text-CoT SFT, iCoT, Pause Tokens, 以及 Coconut, CODI, CoLaR 等隐藏状态循环方法。所有基线均在相同的 Qwen3-8B 基础模型和数据集上复现。
- **主要结果（表 1）**：
    - Switch 在 MATH-500 上达到 **79.3%** 的准确率，在 GSM8K 上达到 **89.2%**，显著优于所有 Coconut 风格的基线。
    - 与最强的 Coconut 风格基线 CoLaR 相比，Switch 在 MATH-500 上提升了 25.7 个百分点。
- **Switch-GRPO 的贡献（表 2）**：
    - 与仅经过 SFT 课程学习的模型相比，经过 Switch-GRPO 后，潜在条件准确率（latent-conditional accuracy）提升了 **+12.6 个百分点**。
    - 同时，切换率（switch rate）从 81% 下降到 58%，表明模型学会了更有选择性地使用潜在推理。
- **准确率-效率曲线（图 4）**：通过调整奖励函数中的简洁性奖励，用户可以在准确率和输出长度之间进行权衡，实现可控的帕累托最优（Pareto frontier）。

### 3.5 技术细节

- **分段反向传播**：为了在 8B 模型上训练，将 rollout 在 `<swi>` 和 `</swi>` 边界处分割。潜在段在 `torch.no_grad()` 下运行，不存储计算图；文本段启用梯度，计算损失并立即反向传播。潜在段通过 KV 缓存将梯度信息传递给后续文本段。
- **鲁棒性**：使用 try/except 处理 OOM 错误，并通过 `survived_indices` 掩码传播组相对优势，避免单个长 rollout 破坏整个 batch 的优势计算。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了隐藏状态循环潜在推理中 RL 训练困难和不可解释的两大核心问题。
    - 提出了一个简单、有效且通用的框架 Switch，证明了其方法的有效性。
    - 通过深入的机制分析，首次实证证明了潜在步骤执行了因果上重要的、问题特定的计算，而非惰性占位符。这回应了关于“思考”token 有效性的普遍质疑。
- **对未来研究/应用的影响**：
    - **方法论**：为将 RL 应用于其他非自回归或连续控制任务提供了新思路。
    - **可解释性**：边界 token 的设计为分析 LLM 内部推理过程提供了强大的工具，可以推广到其他类型的“思考”token 或内部状态分析中。
    - **应用**：为构建更高效、更可控的推理模型铺平了道路。用户可以通过调整奖励函数，在推理质量和计算成本之间进行灵活权衡，这对于资源受限的部署场景（如边缘设备）尤为重要。
- **局限与开放问题**：
    - 实验仅限于 8B 参数的 Qwen3 模型和数学推理任务，尚未在多领域或更大规模模型上验证。
    - Switch-GRPO 的梯度主要通过冻结的 KV 缓存传播，潜在表示本身主要由 Phase 2 的课程学习塑造，而非直接由 RL 目标优化。
    - 机制分析侧重于模型编码了什么（切换边界、潜在因果效应），而非刻画失败模式。
    - 未来工作可以探索将隐藏状态循环与词汇混合潜在方法相结合，使潜在 token 本身也可采样。

## 5. 澄清与简化（Clarifications and Simplifications）

- **隐藏状态循环（类比）**：可以把 LLM 的推理过程想象成一个人边走边写草稿（文本 CoT）。隐藏状态循环就是让他停下来，在脑子里（隐藏空间）进行一段无声的思考，然后把思考的结果（最后的隐藏状态）作为下一步行动的起点。Switch 的边界 token `<swi>` 和 `</swi>` 就像是这个人说“我开始想了”和“我想完了”的口头禅。
- **为什么 RL 训练困难（类比）**：标准的 RL 训练（如 GRPO）需要知道模型在每个步骤做出“选择”的概率。在写草稿时，每个词都是一个选择，概率是明确的。但在“脑子里想”的阶段，没有具体的词被选出来，所以无法计算概率。Switch 通过边界 token 巧妙地绕过了这个问题：我们只计算模型说“我开始想了”和“我想完了”这两个选择（以及周围文本的选择）的概率，而忽略中间“想”的过程。这就像我们只评价一个人决定“开始想”和“结束想”的决策好不好，而不去评价他脑子里具体想了什么。
- **机制分析（简化）**：
    - **Q1（切换策略）**：通过观察 `<swi>` 这个 token 出现的概率，发现它只在模型真正需要切换的地方出现，而不是随机或随意出现。这就像一个训练有素的士兵，只在接到明确指令时才行动，而不是胡乱开枪。
    - **Q2（因果重要性）**：通过“零化”（zeroing）潜在步骤的隐藏状态，模型的准确率大幅下降，说明这个步骤不是可有可无的。这就像在解一道复杂的数学题时，把关键一步的草稿纸抽走，结果就算不出来了。
    - **Q3（计算位置）**：通过分析发现，潜在块中最重要的计算发生在**进入潜在模式后的第一步**。后续的步骤主要是为了满足模型必须至少运行 $K_{min}$ 步的约束，其本身已经准备好退出了。这就像一个短跑运动员，起跑的那一瞬间的爆发力是最关键的，后面的几步只是惯性。
