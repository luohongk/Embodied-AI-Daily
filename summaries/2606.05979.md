## 0. 概览（Concise Summary）

- **主题（Topic）**：具身智能（Embodied AI）基础模型，特别是将世界模型（World Model）、语言推理（Language Reasoning）和动作生成（Action Synthesis）统一到一个框架中。
- **问题（Problem）**：现有的世界-动作模型（WAM）侧重于预测低层次的视觉未来状态，缺乏高层次的语言推理和规划能力；而视觉-语言-动作模型（VLA）虽然具备语言能力，但难以捕捉和理解物理动态。如何将两者的优势结合，构建一个既能进行语义推理又能理解物理动态的通用机器人基础模型。
- **方法（Method）**：提出世界-语言-动作模型（WLA）。该模型采用自回归（AR）Transformer作为骨干网络，将“下一状态”预测分解为两个互补的部分：高层次的文本意图（Textual Intention）和低层次的物理动态（Physical Dynamics）。通过引入“世界专家”（World Expert）和“动作专家”（Action Expert）分别处理物理动态预测和动作生成，并使用元查询（Meta-queries）实现隐式影响。
- **创新（Innovation）**：
    1.  **统一框架**：首次将世界建模、语言推理和动作合成统一在一个自回归Transformer框架内，而非使用双向扩散Transformer。
    2.  **双重状态表示**：将“下一状态”创新性地分解为文本意图和物理动态，兼顾了高层语义和底层物理细节。
    3.  **隐式世界模型**：世界预测通过共享参数隐式地影响动作生成，而非显式地作为条件输入，使得世界专家可以在推理时被禁用，大幅降低推理延迟。
    4.  **测试时扩展（TTS）**：提出了一种新的测试时扩展范式，通过想象未来状态并评分来选择最优动作，进一步提升性能。
- **意义（Significance）**：WLA模型在多项仿真和真实世界任务中取得了最先进或极具竞争力的结果，尤其是在长时域、记忆依赖的任务上表现突出。其推理速度快（~40ms），且具备从无动作标注的跨本体视频中学习新任务的能力，为构建更通用、更高效的具身智能体指明了方向。
- **一句话总结**：本文提出了WLA模型，通过在一个自回归语言骨干网络中联合预测文本子任务、子目标图像和机器人动作，成功融合了世界模型的物理理解能力与VLA的语言推理能力，实现了高效、可扩展的通用机器人控制。

## 1. 研究动机（Motivation）

- **解决的问题**：当前具身智能领域存在两大主流范式，但各有局限：
    - **世界-动作模型（WAM）**：如Cosmos Policy, Fast-WAM。它们擅长从大规模自我中心视频中学习物理动态，通过预测未来视觉状态来辅助动作生成。但其核心是双向扩散Transformer（DiT），**缺乏语言生成能力**，导致其在高层次规划、语义推理和抽象泛化方面能力不足。
    - **视觉-语言-动作模型（VLA）**：如π0, OpenVLA。它们利用预训练VLM的强大语言和感知能力，能很好地遵循指令和进行高层次规划。但它们**缺乏对物理动态的显式建模**，仅依赖视觉和语言信号，导致训练信号不足，限制了其在复杂物理交互中的能力。
- **为什么重要**：构建一个通用的物理AI系统，需要同时具备“理解世界如何运作”（世界模型）和“遵循人类指令进行推理”（语言模型）的能力。将两者割裂会限制智能体的通用性和鲁棒性。
- **挑战在哪里**：
    1.  **架构差异**：WAM通常使用双向DiT，而VLA使用自回归（AR）Transformer。如何在一个统一的架构中融合两者的优势？
    2.  **状态表示**：如何定义和预测一个既能包含高层语义（如“拿起杯子”），又能包含低层物理细节（如“手爪闭合的轨迹”）的“下一状态”？
    3.  **效率问题**：WAM在推理时需要运行昂贵的视频生成模型，延迟高，不适合实时控制。如何在不牺牲性能的前提下实现低延迟推理？

## 2. 创新点（Innovation）

1.  **提出WLA模型架构**：核心创新在于使用一个**自回归（AR）Transformer**（而非DiT）作为骨干网络，来统一建模世界、语言和动作。这使得模型能够自然地继承VLM的语言生成和推理能力。
2.  **双重“下一状态”预测**：将“下一状态”分解为两个互补的表示：
    - **高层次的文本意图**：即从原始指令中分解出的文本子任务（如“移动到目标位置”、“抓取物体”）。这提供了语义级别的、高度可泛化的未来状态蓝图。
    - **低层次的物理动态**：描述了驱动视觉状态转换的核心信息，类似于“潜在动作”（Latent Action）。它连接了高层意图和细粒度运动控制，但避免了预测高分辨率视觉细节的负担。
3.  **隐式世界建模与专家设计**：
    - **世界专家（World Expert）**：一个轻量级的扩散Transformer（如SANA-600M），负责根据骨干网络预测的物理动态和当前状态，预测未来的视觉帧。这个设计将预测视觉细节的繁重任务从骨干网络中剥离，让骨干网络只学习核心的状态转换信息。
    - **隐式影响机制**：物理动态通过**元查询（Meta-queries）** 从骨干网络中提取，并作为世界专家的条件信号。同时，这些物理动态也通过**共享参数学习**（而非显式条件建模）来指导动作专家（Action Expert）生成动作。这种“隐式”方式使得世界专家在推理时可以被完全丢弃，从而大幅降低延迟。
4.  **测试时扩展（TTS）范式**：提出了一种新的推理策略。当计算资源允许时，WLA可以生成多个候选动作，让世界专家“想象”执行这些动作后的未来状态，然后由一个价值模型（Value Model）对这些未来状态进行评分，选择最优的动作执行。这相当于在虚拟空间中进行“试错”，提升了决策质量。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

WLA模型由三个核心组件构成，如图2(a)所示：

1.  **AR Transformer骨干网络**：基于预训练的VLM（如RynnBrain-2B），负责处理多模态输入（当前观测 $o_t$、历史观测 $o_{t-h}$、本体感知状态 $q_t$、指令 $\ell$ 和记忆 $M$）。它预测两个输出：
    - **文本意图** $S_t$：一个连续的文本子任务窗口，描述了未来动作序列的高层语义。
    - **物理动态** $h_t$：通过一组可学习的元查询（Meta-queries）从骨干网络上下文中聚合信息得到，是驱动状态转换的核心信息。
2.  **世界专家（World Expert）**：一个轻量级的扩散Transformer（如SANA-600M）。它接收物理动态 $h_t$ 和当前观测 $o_t$ 的表示，预测未来的视觉状态 $o_{t+n}$。其训练目标是流匹配损失（flow-matching loss）$L_{wm}$。
3.  **动作专家（Action Expert）**：一个流匹配头（flow-matching head）。它接收物理动态 $h_t$ 和本体感知状态 $q_t$，预测未来的一串动作 $a_{t:t+n}$。其训练目标是流匹配损失 $L_{act}$。

**推理模式**：
- **高效模式**：默认情况下，世界专家被禁用。动作生成仅依赖于骨干网络和动作专家，实现了~40ms的低延迟。
- **TTS模式**：如图2(b)所示，采样K个候选动作，世界专家预测每个动作对应的未来帧，价值模型评分后选择最优动作。

### 3.2 关键算法与数学推导

**1. 文本意图学习**：
给定原始指令 $\ell$，构建一系列中间子任务 $\hat{L} = \{\hat{\ell}_1, \hat{\ell}_2, ..., \hat{\ell}_N\}$。模型预测一个覆盖未来动作窗口 $[t, t+n]$ 的连续子任务窗口 $S_t$：
$$S_t = f(o_{t-h}, o_t, \ell, M)$$
其中 $M$ 是记忆缓冲区，用于存储历史子任务，并在长时域任务中递归更新：$M \leftarrow M \oplus [\hat{\ell}_{k_t}, ..., \hat{\ell}_{k_t+n-1}]$。

**2. 物理动态建模**：
通过元查询 $Q$ 从骨干网络中提取物理动态 $h_t$：
$$h_t = f(o_{t-h}, o_t, \ell, M, S_t, Q)$$
世界专家 $f_{wm}$ 利用 $h_t$ 和 $o_t$ 的表示（来自VLM的视觉编码器）来预测未来视觉状态 $o_{t+n}$：
$$o_{t+n} = f_{wm}(h_t, o_t)$$
这里 $o_{t+n}$ 的预测目标是在VAE特征空间中进行的。

**3. 动作生成**：
动作专家 $f_{act}$ 利用物理动态 $h_t$ 和本体状态 $q_t$ 生成动作：
$$a_{t:t+n} = f_{act}(h_t, q_t)$$

**4. 联合训练目标**：
模型通过联合优化三个损失函数进行端到端训练：
$$\mathcal{L} = \mathcal{L}_{act} + \alpha \mathcal{L}_{wm} + \beta \mathcal{L}_{lang}$$
其中 $\mathcal{L}_{lang}$ 是子任务生成的交叉熵损失，$\alpha$ 和 $\beta$ 是权重系数。

### 3.3 模型、训练与数据集

- **模型实例化（WLA-0）**：
    - **骨干网络**：RynnBrain-2B（2.1B参数），一个VLM。
    - **世界专家**：SANA-600M（900M参数，含VAE），一个轻量级扩散Transformer。
    - **动作专家**：流匹配头（390M参数）。
    - **总参数**：3.4B，但推理时仅激活骨干网络和动作专家，约**2B活跃参数**。
- **训练**：
    - **优化器**：AdamW，权重衰减 $1 \times 10^{-8}$，梯度裁剪1.0。
    - **学习率**：余弦退火调度，基础学习率 $5 \times 10^{-5}$，最小学习率 $5 \times 10^{-6}$，1000步预热。
    - **损失权重**：$\alpha = 0.1$, $\beta = 0.005$。
    - **分布式训练**：DeepSpeed。
- **数据集**：
    - **RoboTwin 2.0**：一个包含50个任务的双手臂操作基准，包含2500个干净场景和25000个强随机化场景的轨迹。
    - **LIBERO**：包含4个套件（Spatial, Object, Goal, Long），每个套件10个任务，每个任务50个演示。
    - **RMBench**：一个长时域、记忆依赖的双手臂操作基准，包含4个复杂任务。
    - **真实世界数据**：使用Agilex Robotics Piper双臂平台收集了4个长时域任务的60个演示。

### 3.4 实验设置与结果

- **仿真环境**：
    - **RoboTwin 2.0**：WLA-0在干净场景下达到**92.94%** 的成功率，在随机化场景下达到90.02%，与最先进的WAM（如Fast-WAM）性能相当，但参数更少且无需具身预训练（见表1）。
    - **LIBERO**：WLA-0平均成功率达到**98.6%**，超越了所有WAM和VLA基线。启用TTS模式后，进一步提升至98.9%（见表1）。
    - **RMBench**：WLA-0平均成功率达到**56.5%**，几乎是第二名Mem-0（28.5%）的两倍，大幅领先其他基线（见表2）。消融实验表明，移除语言子任务预测损失（-Llang）后，成功率骤降至17.3%，证明了语言推理在长时域任务中的关键作用。
- **真实世界实验**：
    - **任务**：设计了四个长时域任务（拧瓶盖、打包物体、叠杯子、扔垃圾），并在标准、OOD物体和OOD场景下测试。
    - **成功率**：WLA-0与预训练的π0.5和Motus性能相当，在动态的“扔垃圾”任务中表现最佳（见图3(c)）。
    - **推理效率**：在“叠杯子”任务中，WLA-0的完成时间和推理延迟均为最低。其推理延迟（~40ms）比Motus（~1529ms）低约**40倍**（见图3(d)）。
- **从视频中学习新任务**：
    - **设置**：在RoboTwin 2.0的50个任务中，将45个作为已见任务，5个作为未见任务。训练时，除了已见任务的动作监督外，还加入未见任务的视频监督。
    - **结果**：仅使用已见任务动作监督时，模型在未见任务上几乎完全失败（平均成功率13.0%）。加入未见任务的同本体视频后，成功率提升至**34.4%**。更令人惊讶的是，加入**跨本体**（不同机器人）的视频后，成功率仍能达到**28.8%**，展示了强大的跨本体泛化能力（见表3）。

### 3.5 技术细节

- **元查询（Meta-queries）**：一组可学习的嵌入向量，通过因果注意力从骨干网络的上下文中聚合信息，其输出被用作世界专家的条件信号。
- **动作块（Action Chunk）**：模型预测未来一段时间内的连续动作序列，而非单步动作。块大小在LIBERO上为8，其他任务为32。
- **加速技术**：为了达到~40ms的推理延迟，作者采用了CUDA Graph捕获、算子融合（使用Triton）和预计算与缓存等技术（见附录A）。
- **世界专家预测目标**：实验发现，预测单帧未来图像（$o_{t+n}$）比预测多帧图像（$o_{t+8}, o_{t+16}, ...$）效果更好，因为多帧预测会减慢训练速度并可能干扰动作学习（见表4）。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **解决了WAM和VLA的固有矛盾**：WLA模型首次在一个统一的AR Transformer框架内，优雅地融合了世界模型的物理理解能力和VLM的语言推理能力，取长补短。
    2.  **提出了高效且可扩展的架构**：通过“隐式世界模型”的设计，在训练时利用世界模型学习丰富的物理先验，在推理时将其丢弃，实现了极低的推理延迟，这对于实时机器人控制至关重要。
    3.  **展示了强大的长时域推理能力**：在RMBench上的卓越表现证明了语言子任务预测和记忆机制对于解决需要长期规划和错误恢复的复杂任务的有效性。
    4.  **开辟了从视频中学习的新路径**：WLA能够从无动作标注的跨本体视频中学习新技能，这极大地降低了数据采集成本，为机器人学习的规模化提供了新的可能性。

- **对未来研究/应用的影响**：
    - WLA的架构设计为未来具身基础模型的研究提供了一个强有力的新范式。
    - 其高效的推理能力使其非常适合部署在资源受限的机器人平台上。
    - 从视频中学习的能力有望推动“机器人基础模型”的“互联网规模”预训练，利用海量的网络视频数据。

- **局限与开放问题**：
    1.  **真实世界评估有限**：目前仅在单一机器人平台上的少量任务上进行了评估，需要更多样化的平台和任务来验证其通用性。
    2.  **视频学习依赖仿真**：从视频中学习新任务的实验目前依赖于仿真机器人视频，未来需要探索如何利用真实世界的无标注视频（如人类视频）进行学习。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比理解“隐式世界模型”**：
    - **传统WAM（显式）**：就像一个棋手，每走一步前，都要在脑中完整地“想象”出下一步的棋盘画面，然后根据这个画面来决定怎么走。这个过程很慢，但很精确。
    - **WLA（隐式）**：就像一个经验丰富的篮球运动员。他在训练时（训练阶段）会反复练习和观察球的运动轨迹（世界模型）。但在比赛中（推理阶段），他不需要每次接球前都完整地“想象”球飞行的轨迹，他的身体已经学会了“手感”和“预判”（物理动态），能直接做出正确的传球或投篮动作（动作生成）。这个“手感”就是隐式地指导了他的动作，而不再需要显式地“想象”画面。只有在关键时刻（TTS模式），他才会停下来快速“想象”一下几种投篮路线的结果，选择最好的一个。

- **类比理解“物理动态”与“文本意图”**：
    - 想象你要写一篇关于“做咖啡”的文章。
    - **文本意图**：就是你的文章大纲：“1. 准备咖啡豆；2. 研磨咖啡豆；3. 烧水；4. 冲泡”。这是高层次的、语义化的步骤。
    - **物理动态**：就是你在执行每个步骤时，手和身体的具体动作细节，比如“手指如何捏住咖啡豆”、“手腕如何转动磨豆机”、“手臂如何倾斜水壶”。这些细节描述了状态之间的转换，但不需要描述成“我的手指在10毫秒内移动了5毫米”这种极低层次的信息。WLA的物理动态就是这种“核心的动作模式”，它连接了“研磨咖啡豆”这个意图和具体的“手部运动”。
