## 0. 概览（Concise Summary）

- **主题（Topic）**：离线强化学习（Offline RL），特别是基于模型的强化学习（Model-based RL），用于解决复杂、长时域（long-horizon）任务。
- **问题（Problem）**：在基于模型的价值扩展（Model-based Value Expansion）方法中，存在一个根本性的权衡：为了减少价值函数自举（bootstrapping）的偏差，需要更长的模型展开（rollout）步数（n），但这会加剧单步动力学模型在递归预测中的误差累积，导致长时域预测失效。
- **方法（Method）**：提出了一种名为**MAC（Model-Based RL with Action Chunks）** 的算法。核心思想是使用**动作块（Action Chunk）** 模型和策略。动作块模型直接预测从当前状态执行一个动作块（一系列连续动作）后的未来状态，从而大幅减少递归模型调用的次数，缓解误差累积。同时，使用基于流匹配（Flow Matching）的生成式行为策略进行拒绝采样（Rejection Sampling），以处理多模态的动作分布并避免模型利用（Model Exploitation）。
- **创新（Innovation）**：
    1.  首次将**动作块（Action Chunk）** 思想引入基于模型的价值扩展框架，有效解决了长时域模型展开中的误差累积与偏差减少之间的权衡问题。
    2.  结合**流匹配（Flow Matching）** 的生成式策略和**拒绝采样（Rejection Sampling）**，提出了一种无需不确定性惩罚的、鲁棒的策略优化方法，能够处理高维、多模态的动作块分布。
    3.  引入**流蒸馏（Flow Distillation）** 技术，训练一个一步生成的动作块策略，大幅提升了拒绝采样的计算效率。
- **意义（Significance）**：MAC 证明了基于模型的离线 RL 在长时域、复杂任务上具有超越当前最先进的模型无关（Model-free）方法的潜力。它在多个极具挑战性的长时域机器人操作任务上取得了最优性能，为离线 RL 扩展到更复杂的现实世界任务提供了新的、可扩展的范式。
- **一句话总结**：本文提出 MAC 算法，通过结合动作块模型和流匹配拒绝采样策略，有效解决了离线基于模型 RL 在长时域任务中的误差累积与偏差权衡问题，实现了卓越的性能。

## 1. 研究动机（Motivation）

- **问题与空白**：离线强化学习旨在从静态数据集中学习策略，无需与环境交互。尽管在多个领域取得了成功，但其在处理**复杂、长时域任务**时仍面临巨大挑战。先前的研究（如 Park et al., 2025b）指出，标准的**模型无关（Model-free）** 离线 RL 算法（如基于TD学习的算法）在长时域任务中表现不佳，主要原因是离策略（off-policy）的时序差分（TD）价值学习存在偏差累积的病理现象。
- **为什么重要**：现实世界中的许多决策问题，如机器人操作、自动驾驶、游戏AI等，都具有长时域特性。开发能够有效处理长时域任务的离线 RL 算法，是推动该领域走向实际应用的关键。
- **挑战在哪里**：作者将目光转向**基于模型（Model-based）** 的 RL，特别是**基于模型的价值扩展（Model-based Value Expansion）** 方法。这种方法通过动力学模型生成“想象”的轨迹，并利用这些轨迹进行**在策略（on-policy）** 的价值学习。在策略学习在长时域任务中（如Dota 2）已被证明是有效的。然而，该方法面临一个核心的**权衡困境**：
    - **需要长展开（n大）**：为了减少价值函数自举目标（`γ^n V(s_{t+n})`）的偏差，需要模型展开的步数 `n` 足够大。偏差累积是阻碍离线 RL 扩展到长时域的主要因素之一。
    - **需要短展开（n小）**：如果使用标准的单步动力学模型 `p(s_{t+1} | s_t, a_t)`，生成一个长度为 `n` 的轨迹需要递归调用模型 `n` 次。这会导致动力学模型的误差随着步数线性或指数级累积，使得长时域预测不可靠。
- **核心问题**：是否存在一种解决方案，既能实现长时域模型展开，又能有效防止误差累积？

## 2. 创新点（Innovation）

论文的核心创新在于提出了一种优雅的解决方案来应对上述权衡困境，具体体现在以下三个关键贡献：

1.  **动作块（Action Chunk）模型与策略**：
    - **新方法**：不训练单步模型 `p(s_{t+1} | s_t, a_t)`，而是训练一个**多步模型 `p(s_{t+n} | s_t, a_{t:t+n-1})`**。该模型以一个**动作块**（`a_{t:t+n-1}`，即 `n` 个连续动作的序列）作为输入，直接预测 `n` 步后的未来状态 `s_{t+n}`。
    - **与现有方案的不同**：传统方法通过递归调用单步模型来预测未来，误差会逐层放大。MAC 将 `n` 步预测压缩为单次模型调用，将递归次数从 `n` 次减少到 `1` 次（在块级别），从而**显著降低了误差累积**。这为长时域模型展开提供了可能。
    - **为什么能奏效**：虽然预测一个动作块的结果比预测单步结果更难，但通过将预测跨度拉长，模型可以学习到更高级的、关于动作序列长期影响的动力学规律，避免了中间状态的微小误差被反复放大。

2.  **基于流匹配的拒绝采样策略**：
    - **新方法**：为了使用动作块模型，需要一个动作块策略 `π(a_{t:t+n-1} | s_t)`。直接学习一个最大化奖励的动作块策略在离线设置下非常困难，因为动作块分布通常是**多模态**且**高维**的。MAC 采用了一种巧妙的策略：首先，训练一个**表达力强**的**行为克隆（BC）策略**，使用**流匹配（Flow Matching）** 来建模数据集中的动作块分布。然后，**定义策略为**：从该 BC 策略中采样 `N` 个候选动作块，并选择其中**使学习到的价值函数 `Q(s, a)` 最大**的那个。
    - **与现有方案的不同**：传统离线模型 RL（如 MOPO, MOBILE）通常使用高斯策略，并依赖不确定性估计来惩罚超出分布的动作。MAC 的方法无需显式的不确定性惩罚，因为所有候选动作都来自 BC 策略，天然是分布内的。拒绝采样也比调整不确定性惩罚系数更鲁棒。
    - **为什么能奏效**：流匹配策略能够捕捉复杂的多模态分布，保证了候选动作的质量和多样性。拒绝采样机制则确保了最终选择的动作是“好”的，实现了策略优化，同时避免了模型利用（即选择模型错误地认为价值很高但实际很差的分布外动作）。

3.  **流蒸馏（Flow Distillation）加速**：
    - **新方法**：拒绝采样需要多次评估流匹配策略（通过求解 ODE），计算成本高。MAC 训练一个额外的**一步式 MLP 策略** `π_ω(s_t, z)`，通过**蒸馏损失** `L_distill` 来模仿流匹配策略的输出。
    - **与现有方案的不同**：这相当于将复杂的、多步的生成过程压缩成一个快速的前馈网络。
    - **为什么能奏效**：蒸馏后，生成一个候选动作块只需一次网络前向传播，大大降低了拒绝采样的计算开销，使得在模型 RL 的训练循环中批量使用成为可能。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

MAC 的整体架构包含以下几个核心组件，它们协同工作：

1.  **动作块动力学模型（Action-Chunk Dynamics Model）**：`p_ψ(s_t, a_t)`，其中 `a_t` 是长度为 `n` 的动作块。这是一个确定性 MLP 模型，输入当前状态 `s_t` 和动作块 `a_t`，输出预测的 `n` 步后的状态 `s_{t+n}`。
2.  **动作块奖励模型（Action-Chunk Reward Model）**：`r_ψ(s_t, a_t)`，也是一个 MLP，输入状态和动作块，输出该动作块对应的 `n` 步折扣奖励总和 `r_t = Σ_{i=0}^{n-1} γ^i r_{t+i}`。
3.  **流行为动作块策略（Flow Behavioral Action-Chunk Policy）**：`π_θ(s_t, z)`，基于流匹配的生成模型。它学习从噪声 `z` 到数据集中动作块 `a_t` 的映射，用于生成候选动作。
4.  **一步式蒸馏策略（One-Step Distilled Policy）**：`π_ω(s_t, z)`，一个快速的 MLP 网络，通过蒸馏学习模仿 `π_θ` 的输出，用于加速推理。
5.  **价值函数（Value Functions）**：
    - `V_φ(s)`：状态价值函数，用于在模型展开中计算目标值。
    - `Q_φ(s, a)`：动作-状态价值函数，用于在拒绝采样阶段评估候选动作块的质量。

**算法流程（Algorithm 1）**：
1.  **训练阶段**（循环直到收敛）：
    - **采样**：从离线数据集 `D` 中采样一批转换 `(s_t, a_t, r_t, s_{t+n})`。
    - **训练 BC 策略**：使用流匹配损失更新 `π_θ`，使用蒸馏损失更新 `π_ω`。
    - **训练模型**：使用均方误差（MSE）损失更新动力学模型 `p_ψ` 和奖励模型 `r_ψ`。
    - **生成模型展开**：从数据集中的状态 `s_t` 开始，使用策略（`POLICY` 函数）生成动作块，使用动力学和奖励模型生成下一个状态和奖励，重复 `H` 次，得到长度为 `H*n` 步的想象轨迹。
    - **更新价值函数**：使用从模型展开中收集的 `H` 步目标值来更新 `V_φ`。同时，使用单步 TD 目标更新 `Q_φ`。
2.  **策略执行（POLICY 函数）**：
    - 从高斯分布采样 `M` 个噪声 `z`。
    - 使用一步式蒸馏策略 `π_ω` 生成 `M` 个候选动作块 `a^{(i)}`。
    - 使用 `Q_φ` 评估每个候选动作块的价值。
    - 返回价值最高的那个动作块。

### 3.2 关键算法与数学推导

1.  **动作块动力学与奖励模型训练**：
    - 动力学模型损失：
      $$L_{dyn}(\psi) = \mathbb{E}_{(s_t, a_t, \dots, s_{t+n}) \sim D} \left[ \| p_\psi(s_t, a_t) - s_{t+n} \|_2^2 \right]$$
    - 奖励模型损失：
      $$L_{rew}(\psi) = \mathbb{E}_{(s_t, a_t, \dots, s_{t+n}) \sim D} \left[ \| r_\psi(s_t, a_t) - r_t \|_2^2 \right]$$
      其中 `r_t = \sum_{i=0}^{n-1} \gamma^i r_{t+i}`。

2.  **流匹配行为策略训练**：
    - 训练一个状态相关的速度场 `v_\theta(u, s_t, a_z)`，其中 `u` 是流时间，`a_z` 是噪声 `z` 和目标动作块 `a_t` 的线性插值：`a_z = (1-u)z + u a_t`。
    - 流匹配损失：
      $$L_{flow}(\theta) = \mathbb{E}_{z \sim \mathcal{N}(0, I_{nd}), (s_t, a_t) \sim D, u \sim \text{Unif}([0,1]), a_z = (1-u)z + u a_t} \left[ \| v_\theta(u, s_t, a_z) - (a_t - z) \|_2^2 \right]$$
      这个损失函数引导速度场 `v_\theta` 指向从噪声到数据的“直线”方向。

3.  **流蒸馏损失**：
    - 训练一个一步式策略 `\pi_\omega(s_t, z)` 来模仿流策略 `\pi_\theta(s_t, z)` 的输出。
    - 蒸馏损失：
      $$L_{distill}(\omega) = \mathbb{E}_{s_t \sim D, z \sim \mathcal{N}(0, I_{dn})} \left[ \| \pi_\omega(s_t, z) - [\pi_\theta(s_t, z)]_{\times} \|_2^2 \right]$$
      其中 `[\cdot]_{\times}` 表示停止梯度操作，即 `\pi_\omega` 的学习目标是一个固定的、由 `\pi_\theta` 生成的标签。

4.  **价值函数学习**：
    - 首先生成 `M` 条长度为 `H` 个动作块的想象轨迹 `D_{img}`。
    - **状态价值函数 `V_\varphi` 更新**：使用 `H` 步目标值进行回归。
      $$L_V(\varphi) = \mathbb{E}\left[ \left( V_\varphi(\hat{s}_{t+kn}) - \left( \sum_{i=k}^{H-1} \gamma^{(i-k)n} \hat{r}_{t+in} + \gamma^{(H-k)n} V_{\bar{\varphi}}(\hat{s}_{t+Hn}) \right) \right)^2 \right]$$
      这里的期望是对从 `D_{img}` 中均匀采样的 `(s_t, \hat{a}_t, \hat{r}_t, \dots, \hat{s}_{t+Hn})` 和均匀采样的 `k \in \{0, 1, \dots, H-1\}` 进行的。`V_{\bar{\varphi}}` 是目标网络。
    - **动作价值函数 `Q_\varphi` 更新**：用于拒绝采样。
      $$L_Q(\varphi) = \mathbb{E}_{s_t \sim D} \left[ \left( Q_\varphi(s_t, \hat{a}_t) - \left( \hat{r}_t + \gamma^n [V_\varphi(\hat{s}_{t+n})]_{\times} \right) \right)^2 \right]$$
      这里 `\hat{a}_t` 是策略（通过拒绝采样）选择的动作块，`\hat{s}_{t+n}` 和 `\hat{r}_t` 来自模型预测。

5.  **策略定义（拒绝采样）**：
    $$\pi(s_t) \overset{d}{=} \arg \max_{\{a^{(i)}_{t:t+n-1}\}_{i=1}^N \sim \pi_\beta(a_{t:t+n-1}|s_t)} Q(s_t, a^{(i)}_{t:t+n-1})$$
    其中 `\pi_\beta` 是流行为策略（由 `\pi_\omega` 高效近似）。

### 3.3 模型、训练与数据集

- **模型结构**：
    - 动力学/奖励模型：确定性 MLP。
    - 流策略：基于流匹配的生成模型，其核心是一个时间依赖的速度场网络。
    - 蒸馏策略：MLP。
    - 价值函数：MLP。
- **训练**：
    - 优化器：Adam。
    - 训练流程：在每个训练迭代中，依次更新 BC 策略、动力学/奖励模型、价值函数。模型展开在每个迭代中重新生成。
- **数据集**：
    - **主要实验（长时域任务）**：使用来自 OGBench 的 100M 转换的大型数据集。这些数据集是任务无关的（task-agnostic），由随机原子动作组成，要求算法具备轨迹拼接能力。
    - **标准基准实验**：使用 OGBench 中 1M 大小的 `play` 数据集。

### 3.4 实验设置与结果

- **任务**：
    - **长时域任务**：`humanoid maze-giant`（人形机器人导航）、`cube-octuple`（八立方体操作）、`puzzle-4x5`（4x5拼图），这些任务需要 700-3000 个环境步。
    - **标准基准任务**：来自 OGBench 的 25 个单任务操作任务，包括 `cube-single/double`、`scene`、`puzzle-3x3/4x4`。
- **对比方法**：
    - **模型无关（Model-free）**：IQL, n-SAC+BC, SHARSA, ReBRAC, IDQL, FQL。
    - **序列建模**：Diffuser, HD-DA。
    - **基于模型（Model-based）**：MOPO, MOBILE, LEQ, F-MPC。

    - **长时域任务（Table 1）**：MAC 在所有基于模型的方法中取得了最佳性能。在 `cube-octuple` 和 `puzzle-4x5` 等极具挑战性的长时域操作任务上，MAC 甚至显著超越了所有模型无关方法（包括 SOTA 的 SHARSA）。这证明了 MAC 在长时域任务上的强大可扩展性。
