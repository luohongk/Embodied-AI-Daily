## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人灵巧操作、接触隐式模型预测控制（Contact-Implicit MPC, CI-MPC）、混合系统控制。
- **问题（Problem）**：现有的基于局部线性化的接触隐式MPC方法（如C3）无法在全局范围内优化接触序列，容易陷入局部最优，无法有效探索和利用远距离但有益的接触。而全局优化方法（如轨迹采样）计算量过大，无法实时运行。
- **方法（Method）**：提出一种分层控制框架，将全局探索与局部优化相结合。在每个控制循环中，算法并行采样多个末端执行器（EE）的3D位置，然后为每个采样位置求解一个局部的接触隐式MPC问题（使用C3算法），并根据其预测成本选择最佳行动。控制策略在“无接触”阶段（移动到采样点）和“接触丰富”阶段（执行局部MPC计划）之间自主切换。
- **创新（Innovation）**：
    1.  **问题分解**：将全局接触优化问题分解为“无接触”和“接触丰富”两个阶段，并利用“无接触”阶段进行全局探索。
    2.  **低维采样**：仅在低维的末端执行器位置空间进行采样，而非在完整的状态或轨迹空间，使得实时计算成为可能。
    3.  **成本驱动的切换**：通过比较不同采样点对应的局部MPC成本，并引入滞回和进度监控，实现自主、鲁棒的模式切换。
- **意义（Significance）**：首次在真实机器人上实现了无需离线计算或训练的、实时的、全局感知的接触隐式MPC，能够高精度地完成非凸物体的非抓取操作任务，显著优于纯局部方法和现有的采样方法（如MJPC）。
- **一句话总结**：本文通过低维采样和局部互补性控制相结合，提出了一种能实时近似全局最优的接触隐式MPC方法，成功解决了机器人灵巧操作中的全局优化难题。

## 1. 研究动机（Motivation）

- **核心问题**：机器人进行灵巧操作（如推、拉、滑动物体）时，需要处理复杂的接触动力学。这是一个混合系统，其状态和接触模式（接触/分离、滑动/粘滞）会动态变化。传统的基于模型的控制方法面临一个根本性挑战：**如何在实时（real-time）条件下，从指数级增长的接触序列可能性中，全局地优化出最佳行动序列？**
- **现有方法的局限**：
    - **局部方法（如C3）**：通过线性化接触动力学（如线性互补系统LCS）来简化问题，计算速度快。但其线性化仅在局部有效，导致控制器只能“看到”附近的接触，容易陷入局部最优。例如，机器人可能卡在一个无法有效推动物体的位置，而无法意识到需要先移动到另一个位置再开始推动（如图3所示）。
    - **全局方法（如轨迹优化、图搜索）**：理论上可以找到全局最优解，但计算量巨大，无法满足实时控制的要求。基于采样的方法（如MJPC）在轨迹空间采样，虽然有一定全局性，但高维采样效率低，且容易产生不安全的行为。
- **为什么重要**：解决这个问题是实现通用灵巧操作的关键。机器人需要能够快速、自主地适应新物体、新任务，而无需人工干预或大量的离线训练数据。一个能够实时进行全局推理的控制器是实现这一目标的重要一步。

## 2. 创新点（Innovation）

本文的核心创新在于提出了一种**分层、混合的框架**，巧妙地结合了全局探索和局部优化的优势，具体体现在：

1.  **问题分解与“无接触-接触丰富”两阶段范式**：
    - **关键洞察**：任何接触丰富的操作都可以被分解为两个阶段：一个初始的“无接触”阶段（机器人不与物体发生碰撞地移动），和一个后续的“接触丰富”阶段（机器人自由地建立和断开接触）。
    - **创新之处**：将全局优化问题（选择何时、何地开始接触）与局部优化问题（在给定起始点后如何最优地接触）解耦。这使得全局探索可以简化为寻找一个有利的“接触起始点”。

2.  **低维采样实现全局探索**：
    - **创新之处**：与在完整状态或轨迹空间采样不同，本文仅在**末端执行器（EE）的3D位置**这一低维空间进行采样。这极大地降低了采样空间的维度，使得少量样本（实验中仅用3个）就能覆盖有意义的全局选项，从而实现实时计算。
    - **为什么有效**：对于准静态操作，物体在无接触阶段保持静止，因此全局决策的关键就是选择一个好的末端执行器位置来开始接触。这个位置决定了后续局部MPC的初始条件，从而决定了其性能上限。

3.  **成本驱动的自主模式切换**：
    - **创新之处**：提出了一套基于成本和进度的切换逻辑。控制器不是简单地选择成本最低的采样点，而是引入了**滞回（hysteresis）** 和**进度监控**。
    - **为什么有效**：
        - **滞回**：防止因局部MPC成本计算的微小波动而导致控制器在“无接触”和“接触丰富”模式间频繁切换，提高了控制的稳定性。
        - **进度监控**：当机器人在“接触丰富”模式下长时间无法推动物体向目标靠近时，强制切换到“无接触”模式，重新探索新的接触点。这避免了控制器在无效的局部区域“死磕”，确保了全局探索的持续性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出的控制器是一个分层架构，在每个控制循环中执行以下步骤（如图2所示）：

1.  **状态估计**：获取当前机器人状态（$x_{init}$）和物体状态（通过视觉）。
2.  **采样**：使用策略 `SampleEE` 采样 $n_s$ 个末端执行器位置 $q_{ee, sample}$。采样集始终包含当前位置和上一循环的切换位置。
3.  **并行评估**：对于每个采样点，并行执行以下操作：
    - 构建一个假设的切换状态 $x_{sample} = [q_{ee, sample}, q_{obj, init}, v_{init}]$，即假设物体和速度不变，仅末端执行器移动到采样点。
    - 以 $x_{sample}$ 为初始状态，调用局部CI-MPC算法（C3）求解一个短时域的最优控制问题，得到其预测成本 `C3Cost(x_sample, x_goal)`。
    - 加上从当前位置到采样点的无接触移动成本（如欧氏距离的加权）。
4.  **决策与切换**：
    - 维护一个采样缓冲区，存储历史采样点及其成本。
    - 基于滞回和进度监控，从缓冲区中选择一个目标模式（无接触/接触丰富）和目标采样点。
    - 如果选择“接触丰富”模式，则执行C3为当前状态 $x_{init}$ 计算出的最优控制序列。
    - 如果选择“无接触”模式，则规划一条从当前位置到目标采样点的无碰撞路径并执行。

### 3.2 关键算法与数学推导

**1. 线性互补系统（LCS）**：
局部CI-MPC（C3）的核心是使用LCS来近似非线性接触动力学。LCS模型为：
$$x_{k+1} = A x_k + B u_k + D \lambda_k + d \quad (2a)$$
$$0 \leq \lambda_k \perp E x_k + F \lambda_k + H u_k + c \geq 0 \quad (2b)$$
其中 $x_k$ 是状态，$u_k$ 是控制输入，$\lambda_k$ 是接触力。互补约束 (2b) 编码了接触的混合特性（例如，力仅在接触时存在）。

**2. 双层优化问题**：
本文提出的全局问题可以形式化为一个双层优化：
$$\min_{K, x_{switch}} \left( \min_{x_k, u_k} \sum_{k=0}^{K} c_{free}(x_k, u_k) \text{ s.t. } x_K = x_{switch} + \min_{x_k, u_k} \sum_{k=K}^{N} c_{rich}(x_k, u_k) \text{ s.t. } x_K = x_{switch} \right) \quad (7)$$
外层问题选择切换时间 $K$ 和切换状态 $x_{switch}$，内层问题分别优化无接触和接触丰富阶段的成本。本文通过近似（无接触阶段简化为运动规划，接触丰富阶段用C3近似）和采样（对 $x_{switch}$ 采样）来求解此问题。

**3. 成本计算**：
每个采样点 $i$ 的总成本为：
$$c_{sample, i} = \text{C3Cost}(x_{sample, i}, x_{goal}) + w_{travel} \| q_{ee, init} - q_{ee, sample, i} \|$$
其中 $\text{C3Cost}$ 是C3算法求解出的最优成本，$w_{travel}$ 是移动成本权重。

**4. 方向误差的二次近似**：
为了在MPC中有效处理方向误差，作者没有使用简单的四元数误差范数，而是使用了 $\theta_{error}^2$ 关于当前四元数的Hessian矩阵作为成本矩阵 $Q$ 的一部分。$\theta_{error}$ 定义为：
$$\theta_{error} = \arctan\left(\frac{\sqrt{q_{rel,x}^2 + q_{rel,y}^2 + q_{rel,z}^2}}{q_{rel,w}}\right)^2 \quad (8a)$$
$$q_{rel} = q_{quat, curr}^{-1} \otimes q_{quat, goal} \quad (8b)$$
使用Hessian矩阵可以更准确地近似真实的方向误差，尤其是在误差较小时，避免了局部非凸性问题。

### 3.3 模型、训练与数据集

- **模型**：本文使用**模型驱动**的方法，不涉及任何训练。
- **系统模型**：机器人（Franka Panda）的动力学模型、物体（3D Jack和平面T）的动力学模型、以及它们与环境（地面）的接触模型都是已知的。接触模型需要手动枚举接触几何体（如球体、胶囊体），但接触点通过碰撞检测算法自动计算。
- **状态空间**：$x \in \mathbb{R}^{19}$，包括末端执行器位置、物体位置、物体方向及其速度。
- **控制输入**：$u \in \mathbb{R}^{3}$，施加在末端执行器上的笛卡尔力。
- **接触力**：$\lambda \in \mathbb{R}^{16}$，对应4个接触对（1个机器人-物体，3个物体-地面），每个接触对使用4面体摩擦锥近似。
- **数据集**：无。实验在真实机器人（硬件）和仿真环境中进行，通过随机生成目标位姿来测试泛化能力。

### 3.4 实验设置与结果

- **任务**：使用Franka Panda机械臂，通过非抓取操作（推、拨动），将非凸物体（3D Jack和平面T）精确移动到目标位姿（位置+方向）。
- **硬件**：Franka Panda + D455 RGBD相机（用于FoundationPose物体位姿估计）。
- **对比基线**：
    - **C3**：纯局部方法，在任务中几乎100%失败，未直接比较。
    - **Naive-R**：在无接触阶段随机采样末端执行器位置。
    - **Naive-B**：仅采样物体后方的固定区域。
    - **MJPC**：MuJoCo MPC，一种基于轨迹采样的方法。
- **主要结果**（见表I和图8）：
    1.  **成功率**：本文方法在硬件和仿真中均实现了**100%的成功率**（在安全限制内），而Naive-R和Naive-B成功率很低，MJPC虽然成功率高但伴随着大量的硬件限制违反（HWV）。
    2.  **时间效率**：本文方法在仿真中平均49.31秒达到高精度目标，远快于Naive-R（182.34秒）和MJPC（107.91秒）。在硬件上平均109.20秒完成任务。
    3.  **安全性**：本文方法及其消融变体（Naive-R, Naive-B）在实验中**从未违反**机器人的硬件限制（关节速度、力矩、工作空间）。而MJPC几乎在所有成功试验中都违反了硬件限制，因此无法安全地在真实机器人上部署。
    4.  **泛化能力**：控制器能够连续完成数十个随机生成的目标位姿，无需重新调整参数，展示了强大的泛化能力。

### 3.5 技术细节

- **控制频率**：8-12 Hz。
- **采样数量**：$n_s = 3$。
- **局部MPC（C3）**：使用ADMM算法求解，其中QP用OSQP求解，MIQP用Gurobi求解。C3的求解过程与采样并行化，利用多核CPU。
- **成本截断**：为了防止目标误差过大导致MPC行为过于激进，对目标位移（15cm）和角度（2弧度）进行了截断。
- **方向成本**：使用Hessian矩阵近似方向误差，并正则化以保证正定性，这是实现稳定方向跟踪的关键。
- **采样策略**：在3D Jack任务中，在物体周围的一个球面上随机采样；在平面T任务中，在一个放大的平面周界上随机采样。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - **解决了局部最优问题**：首次提出了一种实用的、实时的框架，将全局探索能力注入到局部高效的接触隐式MPC中，有效克服了纯局部方法容易陷入局部最优的根本性缺陷。
    - **实现了实时全局控制**：通过低维采样和并行计算，在真实机器人上实现了8-12Hz的全局感知接触控制，这在之前的工作中难以实现。
    - **提供了安全且有效的基线**：在标准化的任务上，本文方法在成功率和安全性上均显著优于当前流行的基于采样的方法（MJPC），为后续研究提供了一个强有力的新基线。

- **对未来研究/应用的影响**：
    - **控制框架**：本文提出的“无接触-接触丰富”两阶段分层框架和成本驱动的切换逻辑，可以作为一种通用范式，应用于其他接触丰富的控制问题（如足式机器人、多指手操作）。
    - **与学习方法的结合**：本文的采样策略（`SampleEE`）目前是简单的随机采样，未来可以结合学习方法来学习更智能的采样分布，进一步提高效率。例如，可以训练一个神经网络来预测哪些末端执行器位置最有可能产生低成本的局部MPC计划。
    - **更复杂的操作**：本文展示了单点接触的操作，未来可以扩展到多点接触（如多指手），虽然状态空间会增大，但框架本身具有可扩展性。

- **局限与开放问题**：
    - **效率问题**：平均1.8分钟完成一个任务，对于实际应用来说仍然较慢。如何加速C3求解、优化采样策略、减少不必要的模式切换是未来工作。
    - **模型依赖**：需要精确的动力学模型，限制了其在完全未知场景下的应用。如何与模型学习或无模型方法结合是一个开放问题。
    - **可扩展性**：对于多指手等复杂系统，接触对和状态空间会显著增加，如何保持实时性是一个挑战。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“接触隐式”（Contact-Implicit）？**
    - **通俗解释**：传统的控制方法需要明确告诉机器人“先接触，然后滑动，再分离”。而“接触隐式”方法不需要。它把“如何接触”这个决策也交给优化器去自动决定。优化器会同时规划机器人的运动和接触力，只要满足物理规律（比如不穿透、摩擦力方向正确）即可。这就像下棋，你不需要告诉AI每一步怎么走，只需要告诉它规则和目标，它自己会想出最优的走法。

- **为什么LCS是“局部”的？**
    - **类比**：想象你在一座山上，想找到最低点。LCS就像你脚下的一个“局部地形图”，它精确地描述了脚下几米范围内的坡度。如果你站在一个山脊上，这个局部地图可能告诉你往左走是下坡，但它看不到山脊另一边更深的峡谷。同样，LCS线性化了物体在当前位置的几何形状（用切平面近似），如果物体形状复杂，这个切平面只在接触点附近有效，无法预测机器人移动到另一侧时会发生什么。

- **“采样”在这里是如何工作的？**
    - **类比**：想象你要用一根棍子把一块石头推到洞里。你不想盲目地推，而是想先找几个“好位置”站。本文的“采样”就是让机器人快速“想象”一下：如果我先把棍子放到位置A，然后开始推，效果会怎样？放到位置B呢？位置C呢？它通过运行一个快速的局部MPC（C3）来评估每个位置的“潜力”（成本）。最后，它选择潜力最大的位置，要么直接走过去（无接触模式），要么就在当前位置开始推（如果当前位置就是最好的）。

- **“滞回”（Hysteresis）是什么？**
    - **通俗解释**：为了避免机器人“犹豫不决”，比如刚决定要移动到位置A，结果发现当前位置的MPC成本又变低了，于是又决定不走了。滞回就像设置了一个“门槛”。比如，只有当另一个选项的成本比当前选项低**20%** 以上时，才切换。这保证了决策的稳定性，让机器人能坚持执行一个计划一段时间，而不是频繁变卦。
