# Smaller Models, Unexpected Costs: Trade-offs in LLM Quantization for Automated Program Repair

- arXiv: https://arxiv.org/abs/2606.27205v1
- arXiv ID: 2606.27205

---

🔍 **核心问题**：研究大语言模型（LLM）量化在自动化程序修复（APR）任务中，对模型效果、内存占用、推理时间和能耗之间的权衡影响。  
🛠 **方法**：对6个代表性LLM，采用13种量化配置（不同位宽、方法及目标组件），在HumanEval-Java和Defects4J两个基准上评估APR性能。  
✨ **主要贡献**：发现量化模型与原始模型修复的问题集重叠度低，虽能减少高达85%内存，但增加了推理时间和能耗；48%的配置被其他方案严格支配，且权衡关系高度依赖模型架构与任务复杂度。
