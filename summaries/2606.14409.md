## 0. 概览（Concise Summary）

- **主题（Topic）**：具身智能、视觉-语言-动作（VLA）模型、机器人学习全栈系统。
- **问题（Problem）**：当前VLA模型研究多聚焦于单一模型架构，但真正的通用机器人需要一套从数据采集到真实世界部署的完整、鲁棒的机器人学习栈。现有方法在数据质量、模型设计、策略优化和部署效率等方面存在诸多瓶颈。
- **方法（Method）**：提出了一个名为Hy-Embodied-0.5-VLA (HyVLA-0.5) 的端到端系统，它覆盖了机器人学习的完整流程：1）**数据**：定制指尖UMI设备与动捕系统，采集超过10,000小时的高精度人类示教数据；2）**模型**：基于Hy-Embodied-0.5骨干网络，扩展了流匹配动作专家、紧凑记忆编码器和增量块动作表示；3）**训练**：在UMI语料库上继续预训练，并通过两条SFT路径（同体适应和跨体迁移）进行微调；4）**RL后训练**：提出基于近端偏好优化（PRO）的无奖励离线RL算法FlowPRO；5）**部署**：异步推理管线与轻量级轨迹平滑实现高频闭环控制。
- **创新（Innovation）**：1）提出了一个完整的、可部署的机器人学习栈，而非孤立的模型；2）设计了高精度、具身感知的UMI数据采集方案；3）提出了增量块动作表示，解耦了策略学习与具体机器人运动学；4）提出了FlowPRO算法，无需奖励模型即可利用失败案例进行策略提升；5）设计了延迟感知的贝塞尔曲线动作拼接器，保证了高频控制下的平滑性。
- **意义（Significance）**：这项工作标志着向可部署的通用机器人迈出了实质性的一步。它证明了通过协同设计数据、模型、训练和部署，可以构建出在模拟和真实世界中均表现优异的机器人系统，为未来通用机器人的研究和应用提供了可行的技术路径和工程实践参考。
- **一句话总结**：本文提出了HyVLA-0.5，一个覆盖数据、模型、训练、强化学习和部署全栈的VLA机器人学习系统，通过创新的数据采集、模型设计和无奖励RL算法，在模拟和真实机器人任务上取得了领先性能。

## 1. 研究动机（Motivation）

尽管VLA模型在连续机器人控制方面展现出巨大潜力，但将其转化为可部署的通用机器人，需要解决远超单一模型能力范畴的系统级挑战。这些挑战主要体现在三个层面：

1.  **数据瓶颈**：
    - **传统遥操作**：依赖主从接口，迫使操作者适应机器人工作空间，缺乏直接触觉反馈，难以完成精细操作。
    - **人类数据与手持UMI框架**：虽然缓解了数据稀缺，但原始人类示教动作标签粗糙；现有UMI设备通过SLAM定位，设备笨重且无法捕捉指尖级别的力传递，且存在视觉特征丢失导致的抖动和轨迹丢失风险。
    - **跨具身鸿沟**：将人类示教数据迁移到不同形态的机器人上，需要解决运动空间、动力学/驱动方式以及感知视角（人类第一人称 vs. 机器人摄像头）之间的巨大差异。

2.  **模型与训练瓶颈**：
    - **动作表示**：早期方法将动作离散化为token，限制了执行速度和控制精度。近期方法虽采用流匹配，但其视觉骨干网络并非为机器人控制专门设计，缺乏物理交互所需的密集时空推理能力。
    - **策略优化**：标准模仿学习难以达到“最后一公里”的灵巧性；而现有的强化学习（RL）方法通常依赖脆弱的奖励模型或价值网络，在接触丰富的操作任务中难以设计。

3.  **部署瓶颈**：
    - **实时性**：一个训练好的策略，如果无法在真实硬件上以高频进行闭环视觉控制，其价值将大打折扣。这一部署约束很少被作为首要设计目标。

因此，解决这些耦合的瓶颈需要一个统一的、端到端的管线，能够同时处理数据、模型、策略优化和部署问题。这正是本文的研究动机。

## 2. 创新点（Innovation）

本文的关键创新在于提出了一个完整的、可部署的机器人学习栈，而非孤立的模型改进。其核心创新点包括：

1.  **全栈系统设计**：将数据采集、模型架构、预训练/微调、RL后训练和部署视为一个整体进行协同设计，解决了从实验室到工厂车间的系统性难题。
2.  **定制化高精度UMI数据采集方案**：设计了指尖式UMI设备，配合外部光学动捕系统，实现了亚毫米精度的6-DoF轨迹追踪，同时保留了人类操作者的自然触觉反馈。这解决了传统UMI设备精度不足和缺乏力反馈的问题。
3.  **增量块（Delta-Chunk）动作表示**：策略预测的是相对于当前末端执行器位姿的增量动作序列，而非绝对位姿。这种表示方式**解耦了策略学习与具体机器人的运动学**，使得一个策略可以更容易地迁移到不同形态的机器人上，并显著缩小了优化搜索空间。
4.  **无奖励模型的离线RL算法FlowPRO**：提出了一种基于近端偏好优化（PRO）的离线RL算法。它通过“干预-回滚”管线收集成对的成功/失败轨迹，并利用流匹配损失作为隐式奖励的代理，无需训练任何奖励或价值网络。其对称的近端正则化项有效防止了奖励黑客行为。
5.  **延迟感知的贝塞尔曲线动作拼接器**：针对异步推理管线中动作块边界不连续的问题，设计了一个轻量级的C1连续的三次贝塞尔曲线拼接器，能够平滑地将延迟的动作块连接到机器人当前状态，保证了高频控制下的运动平滑性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

HyVLA-0.5 的整体架构遵循VLA范式，由一个预训练的VLM骨干网络和一个专用的动作专家模块组成。其核心设计围绕四个部分展开：

1.  **数据采集（Hy-UMI-10K）**：使用定制的指尖UMI设备，在光学动捕笼中采集超过10,000小时的、以自我为中心视角的、亚毫米精度的人类示教数据。该数据可直接用于预训练和后训练。
2.  **模型架构**：
    - **骨干网络**：采用Hy-Embodied-0.5-MoT，一个4B参数的Mixture-of-Transformers模型，支持原生分辨率图像编码和模态自适应计算。
    - **动作专家**：采用双塔流匹配架构。一个塔是理解导向的VLM，另一个是生成导向的动作专家。两者通过共享自注意力交互，使视觉-语言上下文指导动作生成。
    - **紧凑记忆编码器**：将图像编码器扩展为视频编码器，通过分解的时空注意力（每L层插入一次因果时间注意力）压缩多帧历史信息，在不增加VLM视觉token数量的情况下提供时空上下文。
3.  **训练流程**：
    - **预训练**：在Hy-UMI-10K语料库上，使用流匹配目标对模型进行大规模预训练，学习通用动作先验。
    - **监督微调（SFT）**：分为两条路径。**Track-A（同体适应）**：在目标机器人上采集示教数据并微调。**Track-B（跨体迁移）**：仅使用UMI示教数据微调，然后部署到形态不同的机器人上。
    - **RL后训练（FlowPRO）**：在SFT之后，通过一个无奖励模型的离线RL阶段，利用“干预-回滚”管线收集的偏好数据，进一步提升策略在长尾故障情况下的鲁棒性。
4.  **部署**：
    - **平台映射**：将策略输出的增量块动作，通过正向运动学和逆运动学映射到具体机器人的关节空间。
    - **异步执行**：解耦策略推理和命令执行，通过线程安全的动作缓冲区重叠两者，隐藏骨干网络延迟。
    - **贝塞尔拼接**：使用三次贝塞尔曲线平滑连接异步执行中产生的动作块边界，保证C1连续性。

### 3.2 关键算法与数学推导

1.  **流匹配目标（Flow-matching Objective）**：
    动作专家通过条件流匹配直接建模连续动作分布 $p(A_t|o_t)$。给定噪声动作块 $A^\tau_t = \tau A_t + (1-\tau)\epsilon$，其中 $\epsilon \sim \mathcal{N}(0, I)$，模型回归速度场 $v_\theta$，训练损失为：
    $$L_{fm}(\theta) = \mathbb{E}_{p(A_t|o_t), q(A^\tau_t|A_t)} \left[ \left\| v_\theta(A^\tau_t, o_t) - (\epsilon - A_t) \right\|^2_2 \right]$$
    其中 $A_t$ 是真实动作块，$\epsilon - A_t$ 是目标去噪方向。

2.  **辅助NTP目标（Auxiliary NTP Objective）**：
    为了保持VLM骨干网络的视觉-语言推理和空间定位能力，引入了辅助的下一token预测任务：
    $$L_{ntp}(\theta) = \mathbb{E}_{(c,y) \sim D_{ct}} \left[ -\sum_{j=1}^{M} \log p_\theta(y_j | c, y_{<j}) \right]$$
    总损失为 $L(\theta) = L_{fm}(\theta) + \lambda_{ntp} L_{ntp}(\theta)$。

3.  **FlowPRO算法**：
    - **隐式奖励代理**：使用流匹配回归损失作为负对数似然的代理，定义隐式奖励：
      $$r_\theta(s, a) = \frac{\beta}{2} \left( \ell_{ref}(s, a) - \ell_\theta(s, a) \right)$$
      其中 $\ell_{ref}$ 和 $\ell_\theta$ 分别是参考策略和当前策略下的流匹配损失。
    - **RPRO损失**：结合对比优化和近端正则化：
      $$L_{PRO}(\theta) = -\mathbb{E}_{(s, a^w, a^l) \sim D} \left[ \log \sigma \left( r_\theta(s, a^w) - r_\theta(s, a^l) \right) + \sum_{a \in \{a^w, a^l\}} \frac{1}{2} \left[ \log \sigma(r_\theta(s, a)) + \log \sigma(-r_\theta(s, a)) \right] \right]$$
      其中第一项 $L_{con}$ 是对比优化器，拉近偏好动作 $a^w$ 的奖励，推远非偏好动作 $a^l$ 的奖励。第二项 $L_{reg}$ 是近端正则化器，在 $r_\theta(s, a)=0$ 时最小化，防止奖励绝对值爆炸。
    - **最终目标**：
      $$L_{RPRO}(\theta) = \lambda_{PRO} L_{PRO}(\theta) + \lambda_{SFT} L_{SFT}(\theta), \quad L_{SFT}(\theta) = \mathbb{E}_{(s, a^w) \sim D} [\ell_\theta(s, a^w)]$$

4.  **贝塞尔曲线拼接**：
    使用三次贝塞尔曲线 $B(t)$ 连接当前机器人状态 $h_0$ 和未来动作块中的连接点 $f_c$，保证C1连续：
    $$B(t) = (1-t)^3 P_0 + 3(1-t)^2 t P_1 + 3(1-t) t^2 P_2 + t^3 P_3$$
    其中控制点 $P_0 = h_0$, $P_3 = f_c$，$P_1$ 和 $P_2$ 分别沿历史运动方向 $\hat{d}_{hist}$ 和未来预测方向 $\hat{d}_{fut}$ 放置。

### 3.3 模型、训练与数据集

- **模型**：HyVLA-0.5 模型总参数量约为4.37B（4B VLM + 370M 动作专家）。VLM采用Hy-Embodied-0.5-MoT架构，动作专家是独立的Transformer模块。
- **数据集**：
    - **预训练**：Hy-UMI-10K，一个内部采集的超过10,000小时、100万+集、涵盖70个不同任务的自我中心视角UMI数据集。
    - **SFT**：
        - **模拟**：RoboTwin 2.0基准，包含50个任务，共2.75K集数据。
        - **真实机器人**：
            - Track-A (Dobot X-Trainer)：4个任务，每个任务300次示教。
            - Track-B (JAKA K1, Astribot S1)：各1个任务，分别使用300和200次UMI示教。
- **训练**：
    - **预训练**：200K步，全局batch size 1024，学习率 $5 \times 10^{-5}$，使用AdamW优化器和bfloat16混合精度。输入为3个摄像头视角，分辨率224x320，预测50步动作块（10Hz）。
    - **SFT**：60K步，全局batch size 32，学习率 $2.5 \times 10^{-5}$。使用6帧历史图像（K=6），动作采样频率50Hz，动作块长度50。
    - **FlowPRO**：迭代式离线RL循环，每轮收集偏好数据对，使用RPRO损失优化。

### 3.4 实验设置与结果

- **模拟任务（RoboTwin 2.0）**：
    - **设置**：50个任务，每个任务100次随机rollout，评估Clean和Randomized两种环境下的成功率。
    - **结果**：HyVLA-0.5在Clean和Randomized设置下分别达到**90.9%** 和**90.1%** 的成功率，超越了所有对比方法（如π0, π0.5, Qwen-VLA等）。消融实验表明，紧凑记忆编码器和UMI预训练均能带来性能提升。

- **真实世界任务**：
    - **设置**：在三个机器人平台（Dobot X-Trainer, JAKA K1, Astribot S1）上评估6个双臂操作任务。
    - **结果**：
        - **Track-A (同体适应)**：在Dobot X-Trainer上，HyVLA-0.5在4个任务上的平均成功率为**96.7%**，显著优于π0（80.0%）和π0.5（88.3%）。
        - **Track-B (跨体迁移)**：在JAKA K1和Astribot S1上，仅使用UMI数据进行微调，HyVLA-0.5分别取得了**93.3%** 和**90.0%** 的成功率，远超π0和π0.5。这证明了其增量块动作表示和跨体迁移策略的有效性。

### 3.5 技术细节

- **增量块动作表示**：动作输出是相对于当前末端执行器位姿的增量，包括3D平移、6D连续旋转和1D夹爪命令，共10维/臂。这避免了学习绝对位姿，简化了跨体迁移。
- **紧凑记忆编码器**：采用分解的时空注意力，每4层插入一次因果时间注意力。时间注意力复用同一ViT块的QKV投影，不引入新参数，且当K=1时退化为标准单帧编码器。
- **FlowPRO数据管线**：通过“干预-回滚”机制收集偏好数据。操作员在策略rollout中发现问题后，系统回滚到之前的状态，将已执行的轨迹记为负样本，操作员的修正轨迹记为正样本。通过平滑插值（Smooth Interpolation）将稀疏的轨迹级偏好转换为密集的状态级偏好元组 $(s, a^w, a^l)$。
- **异步部署**：推理线程和执线线程通过线程安全的动作缓冲区解耦。推理线程用最新观测更新缓冲区，执线线程以控制频率从缓冲区弹出命令。贝塞尔拼接器处理因延迟导致的动作块边界不连续问题。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了从数据到部署的**系统性工程问题**，证明了构建通用机器人需要全栈协同设计。
    - 提供了**高质量、大规模**的UMI数据集和采集方案，为社区提供了宝贵资源。
    - 提出了**增量块动作表示**和**FlowPRO算法**，分别有效解决了跨体迁移和无奖励模型策略优化两大难题。
    - 在模拟和真实机器人上均取得了**最先进的性能**，特别是在跨体迁移任务上展现了巨大潜力。

- **对未来研究/应用的影响**：
    - 为工业界和学术界提供了一个**可复现、可扩展的机器人学习系统蓝图**。
    - **数据采集方案**可能成为未来机器人数据收集的新标准，特别是对于需要精细操作的任务。
    - **FlowPRO算法**为无奖励模型的RL在机器人领域的应用开辟了新路径，有望简化策略优化流程。
    - **跨体迁移能力**对于降低机器人部署成本、实现“一次训练，多平台部署”的愿景至关重要。

- **局限与开放问题**：
    - 论文未明确提及，但可推断的局限包括：1）UMI数据采集依赖外部动捕系统，限制了在野外环境的使用；2）模型规模（4B+）对边缘设备部署仍有挑战；3）FlowPRO的“干预-回滚”管线仍需要人类操作员介入，自动化程度有待提高；4）在更复杂、更开放的任务上的泛化能力有待进一步验证。
