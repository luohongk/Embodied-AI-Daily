## 0. 概览（Concise Summary）

- **主题（Topic）**：四足机器人自主腿部操作（Pedipulation），结合了可负担性（Affordance）理论与分层强化学习（Hierarchical RL）。
- **问题（Problem）**：现有四足机器人操作研究多依赖专家设计的预设轨迹或低级策略学习，缺乏自主选择机器人基座姿态和物体交互点的能力，无法在未知环境中实现完全自主的物体操作。
- **方法（Method）**：提出一个三层分层强化学习框架。顶层利用视觉可负担性模块分析点云，生成机器人基座和末端执行器的可行姿态；中层导航策略根据目标姿态生成速度指令；底层运动策略执行指令并完成行走或腿部操作任务。
- **创新（Innovation）**：
    1.  首次将**姿态可负担性**（Pose Affordance）和**交互点可负担性**（Interaction-point Affordance）引入四足机器人腿部操作任务。
    2.  提出一个**完全自主、无需人类演示**的三层分层RL框架，实现了从环境感知到任务执行的端到端闭环。
    3.  在仿真和真实世界环境中验证了**Sim-to-Real**的迁移能力，并构建了一个包含力反馈的物体交互数据集。
- **意义（Significance）**：该工作显著提升了四足机器人在非结构化环境中的自主操作能力，使其能够像猫一样用腿探索和操纵物体，为行星探索、灾难救援等需要远程自主操作的场景提供了关键技术。
- **一句话总结**：本文提出了一种基于视觉可负担性的分层强化学习框架，使四足机器人能够自主选择最佳操作姿态和接触点，无需人类干预即可在真实世界中完成物体推拉等腿部操作任务。

## 1. 研究动机（Motivation）

- **解决的问题**：四足机器人虽然具备优越的移动能力，但其物体操作能力（尤其是用腿操作，即Pedipulation）仍是一个开放挑战。现有研究主要存在以下空白：
    1.  **依赖专家设计**：许多工作虽然学习了低级操作策略，但高级任务执行（如“走到哪里去推”、“用哪个部位推”）仍依赖专家预设的轨迹或人工遥操作。
    2.  **缺乏自主性**：机器人无法自主判断“哪个基座姿态是可行的”（例如，站在斜坡的哪个位置才能用腿够到物体）以及“物体上哪个点是适合操作的”（例如，推一个光滑的球体还是粗糙的岩石）。
    3.  **端到端规划缺失**：在开放世界中，从识别目标物体、规划可行基座姿态、导航到目标点，再到执行操作的全流程自主规划，尚未被充分探索。
- **为什么重要**：在行星探索、灾难救援等场景中，通信延迟使得实时遥操作不可行，机器人必须具备完全的自主决策能力。能够自主选择操作姿态和接触点，是迈向这种自主性的关键一步。
- **挑战在哪里**：
    1.  **姿态可行性**：在不平坦的地形上，如何找到一个既能让机器人稳定站立，又能使其腿部（末端执行器）够到物体的基座位置和朝向。
    2.  **接触点选择**：如何根据物体表面几何特征，选择一个能最大化操作效率（如推动距离/施加力）的接触点。
    3.  **平衡维持**：在三条腿支撑身体、一条腿执行操作时，如何保持动态平衡，尤其是在物体和地面特性未知的情况下。
    4.  **系统集成**：将视觉感知、状态估计、导航、运动控制和操作策略等多个模块无缝集成，并处理各模块间的误差累积。

## 2. 创新点（Innovation）

本文的核心创新在于将“可负担性”概念系统地应用于四足机器人的自主操作规划中，并构建了一个完整的分层学习框架。具体创新点如下：

1.  **基于视觉可负担性的姿态生成模块**：
    - **新意**：不同于以往直接给定目标位置，该模块能**自主地**为机器人基座和末端执行器生成“可行”的姿态。
    - **如何不同**：它考虑了三个关键因素：**物体可达性**（腿能否碰到）、**局部地形特征**（斜坡方向）和**物体表面特征**（用于选择接触点）。这比简单的“走到物体前”要智能得多。
    - **为何奏效**：通过分析点云数据，模块可以计算出最有利于操作的地形梯度方向（如从斜坡下方往上推），并据此计算机器人的最佳朝向和位置，确保操作的成功率和效率。

2.  **三层分层强化学习框架**：
    - **新意**：提出了一个**完全解耦**的三层架构，将任务抽象为“做什么”（顶层）、“怎么去”（中层）和“怎么做”（底层）。
    - **如何不同**：顶层（可负担性模块）输出目标姿态，中层（导航策略）将其转化为速度指令，底层（运动/操作策略）执行具体动作。这种解耦设计允许各层独立训练，甚至可以用预训练模型进行“即插即用”，极大地提高了系统的灵活性和可扩展性。
    - **为何奏效**：分层结构有效解决了长时域任务的稀疏奖励问题。顶层提供明确的目标，底层专注于执行，降低了单一策略的学习难度。

3.  **Sim-to-Real 实现与数据集构建**：
    - **新意**：不仅在仿真中验证，还在真实户外环境中成功部署，证明了框架的鲁棒性和迁移能力。
    - **如何不同**：在真实实验中，机器人仅依靠机载传感器（LiDAR、IMU）和外部计算机进行推理，无需人工引导。同时，收集了包含力反馈的多模态数据，构建了一个公开数据集，为后续研究提供了宝贵资源。
    - **为何奏效**：通过域随机化（Domain Randomization）训练，增强了策略对真实世界中噪声、不确定性（如质量、质心变化）的鲁棒性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出的框架是一个**三层分层强化学习系统**，如图2所示：

1.  **顶层：基于视觉的姿态可负担性模块**
    - **输入**：来自LiDAR的点云数据、机器人里程计。
    - **处理**：
        - **地形与物体分割**：从点云中分离出地面和物体。
        - **基座姿态生成**：根据物体位置、局部地形坡度（如算法1所示），计算机器人一个“可行”的目标基座位置和朝向（`BP`），使得机器人能稳定站立且腿能触及物体。
        - **交互点生成**：分析物体表面几何，选择“可行”的初始接触点（`p_push`），通常是离机器人基座或末端执行器最近的点。
    - **输出**：目标基座姿态 `BP` 和目标末端执行器姿态（通过样条插值生成的一系列路点）。

2.  **中层：导航策略**
    - **输入**：来自顶层的目标基座姿态 `BP`、机器人当前状态（速度、朝向）。
    - **处理**：一个训练好的神经网络，根据当前状态和目标，输出期望的线速度和角速度指令。
    - **输出**：速度指令 `(Bvx_g, Bvy_g, Bωz_g)`。

3.  **底层：运动/操作策略**
    - **输入**：来自中层的速度指令（导航时）或来自顶层的末端执行器目标姿态（操作时）、机器人本体感知数据（关节位置、速度、IMU等）。
    - **处理**：
        - **导航阶段**：执行一个预训练的**运动策略**，根据速度指令输出12个关节的目标位置，驱动机器人行走。
        - **操作阶段**：执行一个训练好的**腿部操作策略**，根据末端执行器的目标轨迹（由样条插值生成），输出12个关节的目标位置，用一条腿（前右腿）执行推、拨等动作，同时保持身体平衡。
    - **输出**：12个关节的目标位置 `θ`。

**工作流程**：机器人通过视觉模块识别物体 -> 顶层模块生成目标基座姿态 -> 中层导航策略引导底层运动策略走到目标位置 -> 到达后，顶层模块生成操作轨迹 -> 底层操作策略执行腿部操作。

### 3.2 关键算法与数学推导

**1. 基座姿态生成算法 (Algorithm 1)**

该算法将物体中心位置 `Wo`、期望的推动方向 `ψ` 和机器人当前姿态转换为机器人基座坐标系下的目标位置 `Bpx,y_g` 和朝向 `ψg`。

- **核心思想**：根据物体位置和期望的推动方向，计算出一个世界坐标系下的目标点 `Wpg`，然后将其转换到机器人基座坐标系下。
- **关键步骤**：
    1.  `q_01 ← quat(ψ, Wz)`: 根据期望的推动方向 `ψ` 和世界坐标系的z轴 `Wz`，创建一个表示朝向的四元数。
    2.  `(Wpg, ) ← ComposeTransform(Wo, q_01, Bpee,d, q_ee)`: 这是一个**组合变换**。它从物体中心 `Wo` 出发，先旋转到 `q_01` 方向，然后沿着该方向移动一个固定的偏移量 `Bpee,d`（这是机器人腿在训练时能达到的平均位置），从而得到世界坐标系下的目标基座位置 `Wpg`。
    3.  `Bpx,y_g ← QuatApplyInverse(Wqrobot, Wpg - Wprobot)`: 将世界坐标系下的目标位置 `Wpg` 转换到机器人基座坐标系下，并只取x和y分量（忽略高度）。
    4.  `ψg ← (Δψ + π) mod (2π) - π`: 计算机器人需要旋转的角度差，并归一化到 `(-π, +π)` 范围内。

**2. 末端执行器轨迹生成算法 (Algorithm 2)**

该算法为腿部操作生成一条平滑的末端执行器轨迹。

- **核心思想**：使用**三次样条插值**（Cubic Spline）连接四个关键路点，确保轨迹平滑且起点和终点速度为零。
- **关键步骤**：
    1.  **定义路点**：`p_home`（起始安全位置）-> `p_push`（物体表面接触点）-> `p_offset`（一个偏移点，用于形成弧线轨迹）-> `p_home`（回到起始位置）。
    2.  **构建样条**：`S(t) ← CubicSpline(t, W, S˙(0)=0, S˙(T)=0)`: 使用时间向量 `t` 和路点矩阵 `W` 构建三次样条函数 `S(t)`，并强制起点和终点的速度（一阶导数）为零。
    3.  **采样**：`p_cmd ← S(t̄)`: 根据当前时间 `t_now` 和开始时间 `t_start` 计算归一化时间 `t̄`，然后从样条函数中采样得到当前的目标位置 `p_cmd`。
    4.  **安全限制**：`p_cmd.z ← min(p_cmd.z, δh)`: 将末端执行器的高度限制在 `δh`（7.5cm）以下，防止因样条非线性导致意外抬升过高。

**3. 奖励函数设计**

- **操作策略奖励 `Rp`**：
    - `Rp_g` (目标奖励): 鼓励末端执行器到达并停留在目标位置。包含一个**奖励加成** `Rp_b`（当末端执行器在目标位置附近停留足够长时间时触发）和一个**连续跟踪奖励** `Rp_ee`（与距离成反比）。
    - `Rp_p` (惩罚项): 惩罚过大的基座速度、关节扭矩、加速度和动作变化率，以保证操作平稳。

- **运动策略奖励 `Rl`**：
    - `Rl_t` (速度跟踪奖励): 鼓励机器人跟踪中层的速度指令。
    - `Rl_f` (脚部腾空时间奖励): 鼓励机器人有更长的摆动相，从而形成更自然的步态。
    - `Rl_p` (惩罚项): 惩罚不必要的身体运动、关节扭矩和身体倾斜。

- **导航策略奖励 `Rn`**：
    - `Rn_t` (位置跟踪奖励): 鼓励机器人接近目标位置。使用双曲正切函数 `tanh` 来平滑地奖励距离的减小。
    - `Rn_w` (朝向奖励): 惩罚过大的转向角速度指令，鼓励机器人平滑地转向。

### 3.3 模型、训练与数据集

- **模型架构**：所有策略（导航、运动、操作）均使用**Actor-Critic**架构，采用**PPO**算法训练。Actor和Critic网络均为3层全连接网络，隐藏层大小为128，激活函数为ELU。
- **训练环境**：在**Isaac Sim**仿真环境中进行训练，使用**Isaac Lab**框架。机器人模型为**Unitree Go2**。
- **训练细节**：
    - **操作策略**：在仿真中随机采样末端执行器的目标位置（x, y, z在一定范围内），训练其到达目标并保持平衡。
    - **运动策略**：在仿真中随机采样速度指令，训练其跟踪速度并保持稳定。
    - **导航策略**：在仿真中随机采样目标基座姿态，训练其输出速度指令以到达目标。
    - **域随机化**：训练时对机器人的质量、质心位置施加随机扰动，并施加随机外力，以增强策略的鲁棒性。
- **数据集**：在仿真和真实实验中，收集了包含物体特征、目标姿态、机器人里程计、关节位置、力反馈等多模态数据，构建了一个**四足机器人腿部物体交互数据集**。

### 3.4 实验设置与结果

- **仿真实验**：
    1.  **基座姿态有效性验证**：在斜坡上，让机器人从不同方向（0-360度）接近物体并执行操作。结果显示，**朝向与斜坡梯度方向（225度）一致时，操作效率最高**（表2），证明了姿态可负担性模块的有效性。
    2.  **交互点有效性验证**：比较了三种交互点选择策略：离末端执行器最近点、离基座最近点、无特定选择。结果显示，**选择离末端执行器最近的点，操作效率最高**（图4），验证了交互点可负担性模块的有效性。

- **真实世界实验**：
    - **设置**：在户外混凝土斜坡上，放置随机位置和朝向的假岩石。机器人需要自主完成从识别、导航到操作的全流程。
    - **结果**（表3）：
        - **导航精度**：平均位置误差为**0.12米**，平均朝向误差为**16.37度**。
        - **操作精度**：末端执行器跟踪样条轨迹的四个路点，平均误差在**0.04米到0.10米**之间。
        - **任务成功率**：物体交互成功率为**86%**（14次试验中12次成功）。
        - **操作效率**：平均物体位移**0.17米**，平均峰值接触力**33N**，平均效率**0.51 m/N**。
    - **定性结果**：图3和图5展示了机器人在仿真和真实世界中成功执行操作任务的连续帧，验证了Sim-to-Real迁移的成功。

### 3.5 技术细节

- **状态估计**：使用**误差状态扩展卡尔曼滤波器（ESEKF）** 融合IMU和腿部里程计。核心思想是利用“脚在支撑相时相对于世界速度为零”这一假设，通过运动学链计算机器人基座速度，作为测量更新来修正IMU的积分漂移。
- **视觉感知**：采用基于点云的处理流程。使用**k-d树**计算局部表面法向量，通过与地面法向量（所有法向量的中位数）的余弦相似度来分割地面和物体。使用**DBSCAN**聚类算法提取物体点云簇。
- **Sim-to-Real部署**：使用**ONNX Runtime**进行模型推理，通过**ROS2**框架与机器人通信。使用**有限状态机（FSM）** 管理机器人状态（站立、行走、操作等）的切换。在真实实验中，使用Go2自带的运动控制器代替仿真中训练的运动策略，因为其鲁棒性更好，而导航策略仍能很好地泛化。

## 4. 意义与影响（Significance and Impact）

- **解决了什么问题**：
    1.  解决了四足机器人自主操作中“**在哪里操作**”和“**如何操作**”的决策问题，将操作能力从“执行预设轨迹”提升到“自主规划任务”。
    2.  证明了**可负担性**理论在四足机器人操作领域的有效性，为机器人提供了理解环境“可供性”的能力。
    3.  提供了一个**完整的、可迁移的**自主操作框架，并验证了其从仿真到真实世界的可行性。

- **对未来研究/应用的影响**：
    1.  **行星探索**：机器人可以自主选择最佳位置和姿态，用腿去拨动、检查或收集岩石样本，无需地面站实时控制。
    2.  **灾难救援**：机器人可以自主推开废墟中的障碍物，或与环境中的物体进行交互以获取信息。
    3.  **通用机器人操作**：该分层框架和可负担性思想可以推广到其他类型的机器人（如人形机器人、机械臂）的自主操作任务中。
    4.  **数据集价值**：公开的交互数据集将促进其他研究者对物体属性推断、操作策略学习等方向的研究。

- **局限与开放问题**：
    1.  **误差累积**：系统由多个模块串联而成，视觉、状态估计、导航、操作等环节的误差会累积，影响最终任务成功率。特别是高精度操作时，微小误差可能导致失败。
    2.  **动态环境**：当前假设无动态障碍物，目标姿态只计算一次。未来需要处理动态目标或环境变化。
    3.  **物体通用性**：实验对象是岩石类物体，对于不同形状、材质、重量的物体，其可
