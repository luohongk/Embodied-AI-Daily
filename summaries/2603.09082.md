## 0. 概览（Concise Summary）

- **主题（Topic）**：车联网（IoV）中的语义通信、可重构智能表面（RIS）与移动边缘计算（VEC）的联合优化。
- **问题（Problem）**：在动态、高移动性的车联网环境中，如何通过联合优化任务卸载比例、语义符号数量和RIS相位偏移，以最小化端到端延迟，同时应对高维度和非凸的优化挑战。
- **方法（Method）**：提出了一种双层混合优化框架。上层使用近端策略优化（PPO）算法来优化离散决策变量（RIS相位偏移和语义符号数量）；下层使用线性规划（LP）来优化连续决策变量（任务卸载比例）。
- **创新（Innovation）**：
    1.  首次将RIS、语义通信和VEC三者结合，并设计了“本地计算-V2I卸载-V2V卸载”的三路径语义任务分割框架。
    2.  提出了一种PPO+LP的双层协作混合优化方案，有效解决了高维非凸联合优化问题。
    3.  将语义符号数量作为决策变量，实现了物理层（RIS相位）与应用层（语义压缩深度）的跨层联合优化。
- **意义（Significance）**：该研究为在复杂、动态的车联网环境中实现低延迟、高可靠性的智能服务提供了新的理论框架和高效解决方案。实验表明，该方法相比传统启发式算法（如GA和QPSO）能降低约40%-50%的平均端到端延迟，并具有良好的可扩展性。
- **一句话总结**：本文提出了一种基于PPO和LP的双层混合优化框架，通过联合优化RIS相位、语义符号数和卸载比例，显著降低了RIS辅助的语义车联网边缘计算系统的端到端延迟。

## 1. 研究动机（Motivation）

- **解决的问题**：车联网（IoV）应用（如自动驾驶）对低延迟和高可靠性有严格要求。然而，传统中心云架构难以满足这些需求，而车辆边缘计算（VEC）虽然通过将任务卸载到路侧单元（RSU）来缩短通信距离，但城市环境中的障碍物和多径衰落会严重恶化车辆与RSU之间的链路质量。
- **为什么重要**：为了应对链路质量不佳的问题，可重构智能表面（RIS）被引入，通过动态调整信号传播环境来增强信号。同时，语义通信作为一种新兴范式，通过仅传输任务相关的语义特征，而非原始比特流，能大幅减少数据传输量，从而降低延迟。将RIS与语义通信集成到VEC中，有望在极端信道条件下实现超低延迟通信。
- **挑战在哪里**：
    1.  **高动态性**：车辆的高移动性导致信道状态快速变化，传统的静态优化方法失效。
    2.  **问题非凸性**：需要联合优化的变量包括连续的卸载比例、离散的语义符号数量和RIS相位偏移，这是一个高维度的非凸优化问题，求解难度极大。
    3.  **跨层耦合**：物理层的RIS配置（影响信道质量）与应用层的语义压缩深度（影响数据量）之间存在深度耦合，需要协同优化。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **三路径语义任务分割框架**：提出了一种新颖的“本地计算-V2I卸载-V2V卸载”三路径并行处理框架。每个车辆的任务被自适应地分为三部分，分别由本地、RSU和服务车辆（SV）处理，充分利用了空闲的计算资源。
    2.  **PPO+LP双层混合优化框架**：针对问题的非凸性和高维度，创新性地提出了一种双层协作混合方案。上层使用PPO（一种先进的深度强化学习算法）来高效处理离散的RIS相位和语义符号数量决策；下层使用LP（一种经典的凸优化方法）来求解给定上层决策下的最优连续卸载比例。这种“DRL + 凸优化”的混合架构兼具了DRL处理复杂决策和LP保证最优性的优点。
    3.  **跨层联合优化**：将“语义符号数量”作为决策变量，与物理层的“RIS相位偏移”和“卸载比例”一起进行联合优化。这使得系统能够根据实时的信道质量（SINR）动态调整语义压缩的深度，实现了物理层与应用层的深度协同。
- **与现有方案的不同**：现有工作要么只优化传统通信链路下的资源分配，要么只关注RIS或语义通信的单一层面。本文首次将三者结合，并提出了一个能够同时处理离散和连续变量的高效混合优化算法。
- **为什么能奏效**：PPO算法通过与环境交互学习，能够适应动态变化的信道环境，做出鲁棒的离散决策。LP算法则能在给定条件下快速、精确地找到最优的卸载比例。两者的结合有效避免了在高维非凸空间中直接搜索的困难，从而实现了性能的大幅提升。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

- **系统架构**：系统由多个车辆用户（K）、一个配备MEC服务器的RSU、多个可作为服务车辆（SV）的车辆，以及一个部署在建筑物表面的RIS组成。时间被离散化为时隙。
- **三路径任务处理**：每个车辆的任务被分为三部分并行处理：
    1.  **本地计算**：在车辆本地CPU上处理。
    2.  **V2I卸载**：通过RIS辅助的V2I链路卸载到RSU。
    3.  **V2V卸载**：通过RIS辅助的V2V链路卸载到最近的服务车辆（SV）。
- **RIS辅助机制**：RIS可以动态选择增强V2I链路或V2V链路，基于实时的语义相似度、车辆移动性和信道条件。
- **双层优化框架**：
    - **上层（PPO）**：负责优化离散变量，即RIS的相位偏移矩阵 $\Theta^t$ 和每个链路的语义符号数量 $\nu^t_{k,j}$。
    - **下层（LP）**：在给定上层决策后，负责优化连续的卸载比例 $\rho^{loc}_k, \rho^{RSU}_k, \rho^{SV}_k$，以最小化系统延迟。

### 3.2 关键算法与数学推导

**1. 信道模型与SINR**

- **直接信道增益**：$h^t_{k,j} = \sqrt{l(d^t_{k,j})^{-\eta_{k,j}} g^t_{k,j}}$
- **RIS-边缘节点信道**：$h_{r,j} = \sqrt{l(d_{r,j})^{-\eta_{r,j}} \frac{K}{1+K}} h^{LoS}_{r,j}$
- **用户-RIS信道**：$h^t_{k,r} = \sqrt{l(d^t_{k,r})^{-\eta_{k,r}} \frac{K}{1+K}} h^{t,LoS}_{k,r}$
- **SINR计算**：$\gamma^{t,(b)}_{k,j} = \frac{p_{k,j} \left| h^t_{k,j} + (h^{(m)}_{r,j})^H \Theta^t h^t_{k,r} \right|^2}{\sum_{(k',j') \neq (k,j)} \alpha^{(b)}_{k',j'} p_{k'} \left| h^t_{k',j'} \right|^2 + \sigma^2}$
    - 其中，$\Theta^t = \text{diag}(\mu_1 e^{j\theta^t_1}, ..., \mu_N e^{j\theta^t_N})$ 是RIS的相位偏移对角矩阵。

**2. 语义速率模型**

- **语义速率**：$R^t_{k,j} = \frac{W \cdot I_k}{L_k} \cdot \nu^t_{k,j} \cdot \delta^t_{k,j}$
    - 其中，$\delta^t_{k,j} = f(\nu^t_{k,j}, \gamma^{t,(b)}_{k,j})$ 是语义相似度，由预训练的DeepSC模型决定。

**3. 延迟模型**

- **本地计算延迟**：$T^{loc}_k = \frac{\rho^{loc}_k D_k C}{f^{loc}_k}$
- **RSU卸载延迟**：$T^{RSU}_k = T^{tr}_{k,0} + T^{com}_{k,0} = \frac{\rho^{RSU}_k Q_k \cdot (\nu^t_{k,0} L_k)}{W \cdot \delta^t_{k,0}} + \frac{\rho^{RSU}_k D_k C}{F^{RSU}/U_0}$
- **SV卸载延迟**：$T^{SV}_k = T^{tr}_{k,j} + T^{com}_{k,j} = \frac{\rho^{SV}_k Q_k \cdot (\nu^t_{k,j} L_k)}{W \cdot \delta^t_{k,j}} + \frac{\rho^{SV}_k D_k C}{F^{SV}_j / U_j}$
- **总延迟**：$T^{Total}_k = \max\{ T^{loc}_k, T^{RSU}_k, T^{SV}_k \}$

**4. 优化问题 (P1)**

$$\min_{\rho^{loc}_k, \rho^{RSU}_k, \rho^{SV}_k, \nu^t_{k,j}, \theta^t_n} \sum_{k \in K} T^{Total}_k$$
s.t. C1-C8 (见原文公式14)

**5. PPO算法（上层）**

- **动作采样**：$a_t = \tanh(\mu(s_t) + \sigma \odot z_t), \quad z_t \sim \mathcal{N}(0, I)$
- **语义符号映射**：$\nu^t_{k,j} = \text{round}\left( \frac{a_\nu + 1}{2} \cdot (\nu_{\max} - 1) \right) + 1$
- **相位映射**：$\theta^t_n = \arg \min_{\phi \in \Phi} |\phi - \pi(a_\theta + 1)|$
- **PPO损失函数**：$L^{CLIP}(\psi) = \mathbb{E}_t[\min(r_t(\psi)A_t, \text{clip}(r_t(\psi), 1-\epsilon, 1+\epsilon)A_t)]$
- **总损失函数**：$L^{total} = -L^{CLIP}(\psi) + c_1 L^{VF}(\phi) - c_2 \mathcal{S}[\pi_\psi](s_t)$

**6. LP算法（下层）**

- 将P1转化为标准线性规划形式：
    - 优化变量：$x = [\rho^{loc}_k, \rho^{RSU}_k, \rho^{SV}_k, T_k]^\intercal$
    - 目标：$\min c^\intercal x$，其中 $c = [0, 0, 0, 1]^\intercal$
    - 约束：$Ax \leq b$, $A_{eq}x = b_{eq}$, $lb \leq x \leq ub$ (详见原文公式29-32)

### 3.3 模型、训练与数据集

- **模型**：使用预训练的DeepSC模型作为语义编解码器。RIS由N个被动反射单元组成，每个单元相位可调。
- **训练**：采用在线强化学习范式。PPO的Actor和Critic网络均为全连接神经网络。训练过程如Algorithm 1所示，包括环境交互、经验回放、优势函数计算和PPO裁剪策略更新。
- **数据集**：任务生成遵循泊松分布，文本任务大小固定。语义参数映射关系来自文献[57]的验证数据。

### 3.5 技术细节

- **PPO动作空间处理**：为了处理高维离散动作空间，PPO网络输出连续的向量，然后通过特定的映射函数（公式20和21）将其转换为有效的离散相位和语义符号数量。这种“连续松弛+取整”的策略兼顾了训练效率和硬件约束。
- **LP问题转换**：通过引入辅助延迟变量 $T_k$，将原问题中的最大延迟约束转换为线性不等式约束，从而将子问题转化为标准线性规划形式，可以使用内点法等高效算法求解。
- **复杂度分析**：每时隙的总计算复杂度为 $O(NN_{inference} + K^{3.5})$，其中 $NN_{inference}$ 是PPO网络前向传播的常数复杂度，$K^{3.5}$ 是LP求解的复杂度。该复杂度远低于直接求解高维非凸问题的指数级复杂度。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了在动态车联网环境下，如何通过联合优化物理层（RIS）、应用层（语义压缩）和计算资源（卸载决策）来最小化端到端延迟的难题。
    - 提出了一个新颖、高效的PPO+LP双层混合优化框架，为处理类似的高维非凸联合优化问题提供了新的思路。
    - 通过实验证明了该框架在延迟、稳定性和可扩展性方面相较于传统启发式算法的显著优势。
- **对未来研究/应用的影响**：
    - 为未来6G车联网中实现超低延迟、高可靠性的智能服务（如自动驾驶、远程驾驶）提供了理论基础和技术方案。
    - 其“DRL + 凸优化”的混合架构可以推广到其他需要联合优化离散和连续变量的通信与计算场景。
    - 强调了语义通信与物理层技术（如RIS）协同设计的巨大潜力。
- **局限与开放问题**：
    - **知识库一致性**：本文假设所有车辆和RSU共享一个静态的语义知识库。在高度动态的车联网环境中，维护知识库的一致性是一个挑战，动态知识库更新带来的开销和延迟是当前框架的局限性。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比理解“语义通信”**：传统通信就像发送一本完整的书（所有比特），而语义通信就像只发送这本书的摘要（语义特征）。摘要虽然短，但包含了核心意思。即使传输过程中有几个错别字（比特错误），只要不影响对摘要的理解，通信就是成功的。这大大减少了需要传输的数据量，从而降低了延迟。
- **类比理解“双层优化”**：可以把系统优化比作一个公司的运营。
    - **上层（PPO）** 是CEO，负责制定宏观战略，比如“我们今年要投入多少资源做研发（对应RIS相位）？每个项目要做到多精细（对应语义符号数量）？”CEO通过不断尝试和学习，找到最佳战略。
    - **下层（LP）** 是财务总监，在CEO的战略确定后，负责精确分配预算（对应卸载比例）到各个部门（本地、RSU、SV），以最小的成本（延迟）实现目标。财务总监的算法是确定性的，总能找到给定战略下的最优预算分配。
    - 这种“战略+执行”的分工，比让CEO同时处理所有细节要高效得多。
- **类比理解“连续松弛+取整”**：想象你要从一组离散的齿轮（对应RIS相位）中选择一个。直接在所有齿轮中搜索很慢。PPO的做法是：先想象一个“连续”的齿轮，它的齿可以无限细，然后通过一个旋钮（连续输出）来调节它的大小。当找到最佳位置后，再把它“取整”到最接近的真实齿轮上。这样做的好处是，调节旋钮的过程是平滑的，便于学习，而最终结果又符合物理约束。
