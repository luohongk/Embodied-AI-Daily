# Understanding Automated Web GUI Testing: An Empirical Study Across Exploration Strategies and State Abstractions

- arXiv: https://arxiv.org/abs/2606.16650v1
- arXiv ID: 2606.16650

---

🔍 **核心问题**：探究自动化Web GUI测试中，探索策略（如传统模型驱动、强化学习、大语言模型方法）与状态抽象（用于识别页面功能以避免重复测试）如何共同影响测试的有效性（代码覆盖率和故障发现能力）。  
🛠 **方法**：通过实证研究，对比代表性方法（模型驱动、强化学习、大语言模型），分析六种状态抽象对前两类方法的影响，并考察大语言模型方法在不同历史表示（作为状态抽象形式）下的表现，同时比较各类方法暴露的故障。  
✨ **主要贡献**：发现无单一策略在所有维度上最优，各类策略在代码覆盖、状态覆盖和故障发现上互补；状态抽象是关键因素——严格细粒度抽象利于模型驱动策略，紧凑抽象更适合强化学习策略；大语言模型策略受历史表示影响显著，简洁的功能级上下文效果最佳；代码覆盖率与故障发现能力弱相关，需多维度评估。
