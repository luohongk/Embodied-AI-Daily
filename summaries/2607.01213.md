# RepoRescue: An Empirical Study of LLM Agents on Whole-Repository Compatibility Rescue

- arXiv: https://arxiv.org/abs/2607.01213v1
- arXiv ID: 2607.01213

---

🔍 **核心问题**：研究LLM代理能否自动修复因运行环境和依赖演变而失效的旧代码仓库，即“兼容性救援”任务，区别于传统bug修复。

🛠 **方法**：构建包含193个Python和122个Java仓库的基准测试RepoRescue，要求代理仅凭仓库和失效的现代环境，自主诊断故障、定位代码并生成源码级补丁，恢复历史测试套件。

✨ **主要贡献**：发现现有LLM代理系统（如Claude Code、Kimi）在源码级修复上存在互补性，联合使用可提升62.7%的救援成功率；跨文件协调是主要难点；通过实际场景验证和漏洞狩猎测试，确认部分修复补丁具有实用价值。
