## 0. 概览（Concise Summary）

- **主题（Topic）**：时间序列分析与深度学习。具体涉及时间序列预测（TSF）和时间序列分类（TSC）任务。
- **问题（Problem）**：时间序列数据中普遍存在低可预测性样本（噪声大、偏离正常分布），这些样本在训练中会产生高损失，主导梯度更新，导致训练不稳定、收敛到较差的局部最优解，从而损害模型性能。
- **方法（Method）**：提出一个通用的摊销可预测性感知训练框架（APTF）。核心包含两个设计：1）**层次化可预测性感知损失（HPL）**，通过动态分桶和逐步扩大惩罚范围，让模型关注高可预测性样本，同时适当学习低可预测性样本；2）**摊销模型**，通过双模型互相估计可预测性，减少单一模型因过自信导致的估计偏差。
- **创新（Innovation）**：1）首次提出一个通用的、动态识别并惩罚低可预测性样本的训练框架，同时适用于TSF和TSC任务。2）设计了HPL来解决“可预测性演化”问题（即训练后期损失值不再可靠）。3）引入摊销模型来缓解可预测性估计误差。
- **意义（Significance）**：该框架作为一种即插即用的训练策略，能显著提升多种主流TSF和TSC模型的性能（预测误差降低2%-15%+，分类准确率提升），且不依赖特定模型架构，具有很高的通用性和实用价值。
- **一句话总结**：本文提出了一种名为APTF的通用训练框架，通过层次化损失惩罚和双模型摊销机制，有效抑制了时间序列数据中低可预测性样本的负面影响，显著提升了多种预测和分类模型的性能与泛化能力。

## 1. 研究动机（Motivation）

- **问题与空白**：时间序列数据（如金融、医疗、工业传感器数据）极易受到噪声干扰，导致样本中存在大量“低可预测性”模式。这些样本偏离正常数据分布，训练时会产生异常高的损失值，从而主导梯度更新，使模型训练振荡，并最终收敛到较差的局部最优解。尽管现有深度学习模型在TSF和TSC上取得了不错的效果，但它们通常平等对待所有训练样本，**缺乏从训练视角出发，主动识别并处理这些低可预测性样本以提升模型收敛性能的机制**。
- **为什么重要**：解决这个问题对于提升时间序列分析模型的鲁棒性、泛化能力和最终性能至关重要。尤其是在噪声普遍存在的真实世界应用中（如基金销售预测、医疗诊断），忽略低可预测性样本会严重限制模型的实际应用效果。
- **挑战在哪里**：挑战在于如何区分“低可预测性样本”和“高可预测性样本”，并采取合适的策略。直接丢弃（如Co-teaching方法）会浪费这些样本中可能包含的有用信息，导致欠拟合。而简单地降低其权重又可能不够有效。此外，模型自身的“可预测性演化”问题（训练初期损失值能反映可预测性，但后期模型过拟合后，损失值不再可靠）和“估计偏差”问题（单一模型可能过自信，误判样本）使得动态、准确地识别和处理低可预测性样本变得非常困难。

## 2. 创新点（Innovation）

- **关键创新**：提出了一个通用的、即插即用的训练框架APTF，其核心创新在于两个互补的设计：
    1.  **层次化可预测性感知损失（HPL）**：
        - **动态分桶与渐进惩罚**：HPL不是简单地丢弃或加权样本，而是根据损失值将样本动态分入多个桶中，并对低可预测性（高损失）桶中的样本施加更小的损失权重。更重要的是，它采用“层次化”策略，在训练过程中逐步增加桶组（Bucket Groups），每个新组包含更少的桶（即更大的惩罚范围），并将所有历史桶组的损失加权平均。这既解决了“可预测性演化”问题（逐步扩大惩罚范围），又避免了梯度突变和过度惩罚高可预测性样本。
    2.  **摊销模型（Amortization Model）**：
        - **双模型互估计**：为了缓解单一模型因过自信导致的“可预测性估计误差”（将高可预测性样本误判为低可预测性），APTF引入了一个结构相同但参数独立的“摊销模型”。两个模型互相为对方估计样本的可预测性（即对方的分桶结果），并将估计结果延迟一步传递给对方。这种“互相学习、延迟反馈”的机制可以有效吸收和纠正单一模型的偏差，提升HPL的准确性。
- **与现有方案的不同**：
    - 与**Co-teaching**（丢弃高损失样本）不同，APTF不丢弃样本，而是通过调整损失权重来“软性”抑制低可预测性样本的影响，保留了潜在的有用信息。
    - 与**WaveBound**（对低于阈值的误差进行梯度上升）不同，APTF的惩罚是动态、分层的，能更精细地处理不同可预测性级别的样本。
    - 与**Self-paced Learning**（从易到难学习）和**Adaptive Robust Loss**（自适应参数）相比，HPL的层次化结构和摊销机制能更有效地平衡对高、低可预测性样本的学习，避免欠拟合或惩罚不足。
- **为什么能奏效**：HPL通过层次化结构，在训练的不同阶段动态调整对低可预测性样本的惩罚力度，既防止了早期过拟合，又保证了后期能从这些样本中学习到有用模式。摊销模型则通过双模型协同，提高了可预测性估计的鲁棒性，防止了误判带来的性能损失。两者结合，使得模型能够更稳定、更有效地收敛。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

APTF的整体架构如图1(b)所示。其核心流程如下：

1.  **基础模型（Source Model）**：这是待优化的TSF或TSC模型（如PatchTST, Autoformer等）。
2.  **摊销模型（Amortization Model）**：与基础模型结构相同，但参数独立。它的作用是辅助估计样本的可预测性。
3.  **层次化可预测性感知损失（HPL）**：这是APTF的核心损失函数计算模块。
    - **分桶**：在每个训练阶段，根据基础模型（或摊销模型）在当前批次上的损失值，将样本从低到高排序，并均匀分配到K个桶中。桶索引越大，样本可预测性越低。
    - **权重分配**：为每个桶分配递减的权重（$W_1 > W_2 > ... > W_K$），使得低可预测性样本的损失贡献被抑制。
    - **层次化累积**：随着训练进行，APTF会创建新的“桶组”（Bucket Group），新组包含更少的桶（即惩罚范围更大）。最终的损失是所有历史桶组损失的加权平均，实现了“层次化”的渐进惩罚。
4.  **损失计算**：最终的损失由两部分组成：基础模型使用摊销模型的分桶结果计算损失，摊销模型使用基础模型的分桶结果计算损失。这种交叉计算方式（图1(b)红色箭头）实现了“摊销”效果。

### 3.2 关键算法与数学推导

**1. 基本可预测性感知损失（Algorithm 1）**

该算法是HPL的基础。给定K个桶 $\{B_1, B_2, ..., B_K\}$ 和对应的权重 $\{W_1, W_2, ..., W_K\}$，损失计算如下：

$$L = \sum_{j=1}^{K} \text{Average}(B_j \cdot W_j)$$

其中，$B_j$ 是第j个桶中所有样本的原始损失值向量，$W_j$ 是对应的标量权重。

**2. 层次化可预测性感知损失（Algorithm 2，核心）**

该算法描述了在S个训练阶段后的损失计算过程。它引入了“层次化”的概念。

- **输入**：来自基础模型（Source Model）的层次化桶组 $\{\mathcal{B}^1, ..., \mathcal{B}^S\}$ 和权重组 $\{\mathcal{W}^1, ..., \mathcal{W}^S\}$。同样，来自摊销模型（Amortization Model）的 $\{\tilde{\mathcal{B}}^1, ..., \tilde{\mathcal{B}}^S\}$ 和 $\{\tilde{\mathcal{W}}^1, ..., \tilde{\mathcal{W}}^S\}$。
- **核心逻辑**：对于第S个训练阶段，损失计算会遍历从1到S的所有桶组。
    - **基础模型的损失**：使用**摊销模型**在第i个阶段的桶组和权重来计算：$L = L + \text{Predictability\_Aware\_Loss}(\tilde{\mathcal{B}}^i, \tilde{\mathcal{W}}^i)$
    - **摊销模型的损失**：使用**基础模型**在第i个阶段的桶组和权重来计算：$\tilde{L} = \tilde{L} + \text{Predictability\_Aware\_Loss}(\mathcal{B}^i, \mathcal{W}^i)$
- **输出**：最终的损失为所有阶段损失的平均值：$L / G$ 和 $\tilde{L} / G$，其中G是总阶段数。

**3. 训练阶段切换条件**

下一个训练阶段在满足以下条件时激活：

$$tag = \begin{cases} 1 & \text{if current epoch} \% \varepsilon = 0 \\ 0 & \text{otherwise} \end{cases}$$

其中 $\varepsilon$ 是阶段间隔超参数。

### 3.3 模型、训练与数据集

- **模型**：APTF是一个通用框架，可以应用于任何TSF或TSC模型。论文中测试了11种TSF模型（包括CNN-based: ModernTCN; Transformer-based: CATS, TimeXer, PatchTST, Autoformer, NSformer, Scaleformer, Informer; MLP-based: TimeMixer, NLinear, N-HiTS）和5种TSC模型（InceptionTime, OS-CNN, FCN, ResNet, FormerTime）。
- **训练**：APTF不改变基础模型的优化器或其他超参数，仅替换其损失函数为HPL，并引入一个摊销模型进行协同训练。两个模型使用相同的优化器，但参数独立更新。
- **数据集**：
    - **短期TSF**：三个基金销售数据集（Fund1, Fund2, Fund3），这些数据受政策、舆论等复杂因素影响，噪声大，低可预测性样本多。
    - **长期TSF**：8个标准数据集（Exchange, Weather, Electricity, Traffic, ETTh1, ETTh2, ETTm1, ETTm2）。
    - **TSC**：UCR时间序列分类档案中的128个数据集。

### 3.4 实验设置与结果

- **评价指标**：TSF使用MSE、MAE和WMAPE；TSC使用准确率（Accuracy）。
- **主要结果**：
    - **短期TSF（表1）**：APTF在8个基线模型上均取得一致提升，平均WMAPE降低2%-9.79%。
    - **长期TSF（表2）**：APTF在8个基线模型上均取得一致提升，平均MSE降低2%-13.14%。
    - **TSC（表3）**：APTF在5个基线模型上均提升了平均分类准确率（从80.97%提升至81.93%），在128个数据集中赢得了75个。
- **消融实验**：
    - **HPL有效性（表4, 5）**：移除HPL（退化为图2(b)的简单渐进策略）后，模型性能显著下降，证明了层次化结构的必要性。
    - **摊销模型有效性（表4）**：移除摊销模型后，模型性能下降，证明了双模型互估计能减少估计误差。
    - **泛化能力（图3）**：APTF使模型的损失景观（Loss Landscape）变得更平坦，表明其泛化能力更强。
- **对比实验**：
    - **与Co-teaching和WaveBound对比（表6）**：APTF在所有数据集上均优于这两种方法，尤其是在噪声大的基金数据集上优势明显。
    - **与其他可预测性感知损失函数对比（表7）**：APTF优于Adaptive Robust Loss (ARL) 和 Self-paced Learning (SPL)。
- **效率分析**：使用摊销模型时，时间复杂度和内存开销大约翻倍。但**不使用摊销模型时，几乎没有额外开销，且性能提升依然显著（表8）**，这证明了HPL本身的有效性和高效性。

### 3.5 技术细节

- **分桶策略**：在每个批次内进行分桶，桶的数量K是一个超参数。论文实验中发现K=9并非最优，更大的K（如15）能带来更好的性能，但会趋于饱和。
- **阶段间隔**：阶段切换的epoch间隔 $\varepsilon$ 是一个超参数，实验表明 $\varepsilon=3$ 或 $4$ 效果较好。
- **TSC的HPL**：对于TSC任务，HPL的损失权重、阶段间隔和惩罚范围率等超参数需要单独调整，论文给出了一个经验性设置（0.1, 75, 0.025）。
- **摊销模型设计**：摊销模型与基础模型结构完全相同，但参数独立。其损失计算方式与基础模型对称，即使用对方的分桶结果。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  首次提出了一个通用的、面向TSF和TSC任务的**可预测性感知训练框架**，填补了该领域从训练视角处理低可预测性样本的空白。
    2.  提出的**HPL**巧妙地解决了“可预测性演化”问题，通过层次化结构实现了对低可预测性样本的动态、渐进式惩罚，同时保护了高可预测性样本的学习。
    3.  引入的**摊销模型**机制有效缓解了单一模型的可预测性估计偏差，进一步提升了框架的鲁棒性。
    4.  大量实验证明了APTF的**通用性**（适用于多种模型和任务）和**有效性**（显著提升性能）。
- **对未来研究/应用的影响**：
    - **研究影响**：为时间序列分析中的噪声鲁棒学习提供了新思路。未来的研究可以探索更智能的自动超参数调整策略（如自适应确定桶数K和阶段间隔 $\varepsilon$），或将此框架扩展到其他时间序列任务（如异常检测）。
    - **应用影响**：APTF作为一种即插即用的训练策略，可以轻松集成到现有的TSF和TSC系统中，提升其在金融、医疗、工业物联网等噪声环境下的预测和分类精度，具有很高的商业和工程价值。
- **局限与开放问题**：
    - **超参数依赖**：APTF的性能对桶数K和阶段间隔 $\varepsilon$ 等超参数敏感，目前需要手动调整。如何根据数据集特性自动确定最优超参数是一个重要的开放问题。
    - **计算开销**：使用摊销模型会带来约两倍的计算和内存开销。虽然不使用摊销模型也能取得不错效果，但如何在不牺牲性能的前提下降低开销是未来优化的方向。

## 5. 澄清与简化（Clarifications and Simplifications）

- **低可预测性样本 vs. 噪声标签**：论文中处理的“低可预测性样本”与Co-teaching处理的“噪声标签”有本质区别。噪声标签是标签本身错误，信息完全有害，可以丢弃。而低可预测性样本的**输入**或**模式**本身噪声大，但可能仍包含部分有用信息。APTF的策略是“软性抑制”而非“硬性丢弃”，这就像老师对待一个学习困难但并非完全不可救药的学生，不是直接开除，而是降低其作业的权重，同时鼓励他学习。
- **层次化分桶（HPL）的直观理解**：想象一个筛子，一开始筛孔很大（桶数少），只能筛掉最明显的“坏样本”（噪声最大的）。随着训练进行，筛子变得越来越精细（增加桶组，每个新组筛孔更细），能筛掉更多“不那么坏”的样本。同时，HPL不是只用最新的筛子，而是**把所有用过的筛子结果加起来平均**。这样做的好处是：1）筛子变化不会太剧烈（训练稳定）；2）之前被筛掉的“好样本”在后续阶段可能因为损失变小而重新被“捞回来”学习，防止了过度惩罚。
- **摊销模型的作用**：可以把它想象成两个学生（基础模型和摊销模型）互相批改作业。学生A（基础模型）可能因为太自信，把一些好作业（高可预测性样本）误判为差作业。学生B（摊销模型）来批改A的作业，由于B的视角不同，可能会纠正A的错误。A再根据B的批改结果来调整自己的判断。这种“交叉验证”机制能有效减少单一判断的偏差。
