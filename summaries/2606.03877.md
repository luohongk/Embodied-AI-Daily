## 0. 概览（Concise Summary）

- **主题（Topic）**：3D场景表示与渲染，特别是面向对象（Object-Centric）的神经辐射场。
- **问题（Problem）**：现有的3D场景表示方法（如3D高斯泼溅和NeRF）要么难以分解为独立的物体级单元（如3DGS），要么缺乏高效的渲染能力（如NeRF）。它们通常需要额外的分割或分组步骤才能进行物体级别的操作。
- **方法（Method）**：提出MLP-Splatting，将场景表示为一系列独立的、紧凑的MLP（多层感知机）基元。每个MLP作为一个“神经基元”，在其局部空间内预测颜色和密度，并通过高效的基于块的体渲染进行合成。
- **创新（Innovation）**：
    1.  **以MLP为基元**：用独立的MLP替代高斯椭球体作为场景表示的基本单元，每个MLP具有更强的表达能力。
    2.  **物体级结构涌现**：在纯RGB监督下，MLP基元会自然地学习表示场景中的物体或物体部件，无需分割掩码即可实现物体级编辑。
    3.  **高效渲染**：设计了基于块的渲染管线，实现了比语义3DGS方法快3倍、内存少15倍的性能。
    4.  **理论分析**：提出了“等质量内存定律”（Equal-Quality Memory Law），从理论上证明了在达到相同像素误差时，MLP-Splatting所需参数量的理论优势。
- **意义（Significance）**：该工作提出了一种新的场景表示范式，将神经网络的表达力与基元的原子性（可编辑性）相结合，使得物体级结构成为表示的一等公民，为高效、可交互的3D场景理解与编辑开辟了新路径。
- **一句话总结**：MLP-Splatting通过用独立的紧凑MLP作为场景基元，实现了既具备NeRF级表达力又拥有3DGS级高效渲染的、天然面向物体的3D场景表示。

## 1. 研究动机（Motivation）

- **核心问题**：现有的3D场景表示方法在“表达力”和“可编辑性/物体级理解”之间存在矛盾。
    - **NeRF系列**：使用一个全局的神经网络，表达力强，能生成逼真的新视角，但场景是“铁板一块”，难以分解和编辑单个物体。
    - **3DGS系列**：使用大量离散的高斯椭球体，渲染速度快，易于编辑（通过操作高斯），但基元是低级的几何/辐射元素，物体概念需要通过额外的聚类或语义分割才能获得。
- **为什么重要**：在机器人、AR/VR、内容创作等应用中，我们不仅需要逼真的渲染，更需要能够理解、交互和编辑场景中的独立物体。一个“天生”就能将场景分解为物体的表示方法，将极大简化后续任务。
- **挑战**：如何设计一种表示，其基本单元既能像高斯一样高效渲染和编辑，又能像NeRF一样具有强大的表达能力，并且能自然地对应到物体或物体部件，而不是需要后期处理才能获得物体概念。

## 2. 创新点（Innovation）

1.  **提出“神经基元”概念**：核心创新在于用**独立的、紧凑的MLP**作为场景表示的基本原子。这与3DGS的低级高斯基元或NeRF的全局网络有本质区别。每个MLP是一个“微型NeRF”，负责建模其局部空间内的光场。
2.  **物体级结构的涌现性**：由于每个MLP基元具有局部空间支持且独立优化，在仅使用RGB损失监督下，它们会自然地“竞争”并“专精”于表示场景中颜色、纹理一致的连贯区域（即物体或物体部件）。这是一种**自发的、无监督的物体分解**。
3.  **高效的基于块的渲染管线**：为了处理大量独立的MLP，作者设计了基于块的CUDA渲染器。通过将图像划分为块（tile），对每个块内的基元进行排序和评估，实现了实时渲染，解决了独立MLP计算开销大的问题。
4.  **理论上的“等质量内存定律”**：从函数逼近的角度，严格证明了在逼近一个分段光滑的场景时，要达到相同的像素误差，3DGS所需参数量的下界是 $\Omega(\varepsilon^{-2})$，而MLP-Splatting所需参数量的上界是 $\Theta(\varepsilon^{-3/s})$（其中 $s$ 是局部光滑阶数）。这从理论上解释了为什么MLP-Splatting可以用更少的参数达到相同或更好的渲染质量。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

MLP-Splatting的整体架构如图2所示，包含三个核心部分：

1.  **神经基元（Neural Primitive）**：每个基元 $i$ 由一个紧凑的MLP $f_i$ 和一组几何参数（中心 $\mu_i$，尺度 $s_i$，旋转 $q_i$）组成。MLP负责从6D输入（方向+局部偏移）预测颜色 $c_i$ 和密度 $\sigma_i$。几何参数定义了一个各向异性的高斯支持区域，限制了MLP的有效作用范围。
2.  **光线-基元交互（Ray-Primitive Interaction）**：对于每条光线，计算其与每个基元支持区域的“最优软接触点” $t_i^*$，并得到局部偏移向量 $\delta_i$。将光线方向 $d$ 和 $\delta_i$ 拼接成6D向量 $x_i$，作为MLP的输入。
3.  **基于块的渲染与合成（Tile-based Rendering & Compositing）**：
    - **分块**：将图像平面划分为 $8 \times 8$ 的块（tile）。
    - **基元筛选与排序**：对于每个块，通过AABB（轴对齐包围盒）测试筛选出与该块相关的基元。然后，使用一条穿过块中心的虚拟光线，计算所有相关基元的深度 $t_i^*$，并进行排序。
    - **并行评估**：一个CUDA线程块负责处理一个块内的所有像素。线程块按排序后的顺序依次加载每个基元的MLP权重到共享内存，然后所有线程并行地评估该MLP，得到每个像素的颜色和密度。
    - **Alpha混合**：按照从前到后的顺序进行标准的体渲染alpha混合，得到最终像素颜色。

### 3.2 关键算法与数学推导

1.  **最优软接触点计算**：
    对于基元 $i$，其高斯支持区域由矩阵 $\Lambda_i$ 定义。光线 $r(t) = o + td$ 与基元中心的加权距离为 $\|o+td - \mu_i\|^2_{\Lambda_i}$。最小化该距离得到最优接触点：
    $$t_i^* = \max\left(0, \frac{d^\top \Lambda_i (\mu_i - o)}{d^\top \Lambda_i d}\right)$$
    对应的3D点为 $p_i^* = o + t_i^* d$。

2.  **6D输入构造**：
    局部偏移向量 $\delta_i = \mu_i - p_i^*$。MLP的输入为：
    $$x_i = [d; \delta_i] \in \mathbb{R}^6$$
    这个输入耦合了视角信息（$d$）和局部几何信息（$\delta_i$），直接对应光场参数化。

3.  **体渲染合成**：
    对于一条光线，与 $N$ 个排序后的基元交互，最终颜色为：
    $$C = \sum_{i=1}^N T_i \alpha_i c_i$$
    其中 $\alpha_i = 1 - \exp(-\sigma_i \delta_i)$ 是第 $i$ 个基元的不透明度，$T_i = \exp\left(-\sum_{j=1}^{i-1} \sigma_j \delta_j\right)$ 是累积透射率。

4.  **等质量内存定律（核心理论）**：
    - **定理1（3DGS下界）**：对于任何 $K$ 项高斯混合模型 $\sigma_K$，逼近真实场景的像素误差 $E_1 \gtrsim C K^{-1/2}$。这意味着要达到误差 $\varepsilon$，需要 $\Omega(\varepsilon^{-2})$ 个高斯。
    - **定理2（MLP-Splatting上界）**：存在一种划分-统一（partition-of-unity）的构造，使得像素误差 $E_2 \leq C(h^s + m^{-s/3})$，其中 $h$ 是基元大小，$m$ 是网络宽度。平衡两项后，要达到误差 $\varepsilon$，需要 $\Theta(\varepsilon^{-3/s})$ 个参数。
    - **推论**：对于 $s \geq 2$（即场景分段二次可微），$\varepsilon^{-3/s} \leq \varepsilon^{-1.5}$，严格优于3DGS的 $\varepsilon^{-2}$ 速率。这解释了为什么MLP-Splatting可以用更少的参数达到高质量渲染。

### 3.3 模型、训练与数据集

- **模型**：每个MLP架构为 `72-32-32-4`。输入是72维的位置编码（6D输入，每个分量用 $L=6$ 的编码），输出4维（3维RGB颜色 + 1维原始密度）。语义特征嵌入维度为512。
- **训练**：
    - **损失函数**：颜色损失 $L_{rgb} = (1-\lambda)\|C-C^*\|_1 + \lambda (1-\text{SSIM}(C, C^*))$。当联合训练语义时，加上特征损失 $L_f = \|F - F_t\|_1$，其中 $F_t$ 是来自LSeg或SAM等基础模型的教师特征。
    - **优化器**：Adam。学习率：位置 $10^{-4}$，尺度 $10^{-2}$，其他参数 $10^{-3}$。
    - **初始化**：使用COLMAP点云通过自适应八叉树采样初始化基元的位置和尺度。
    - **迭代次数**：60,000次。
- **数据集**：Replica和ScanNet室内场景数据集。

### 3.4 实验设置与结果

- **基线方法**：NeRF-DFF（基于NeRF的语义特征蒸馏）和Feature-3DGS（基于3DGS的语义特征蒸馏）。
- **主要结果**：
    1.  **新视角合成质量**（表1）：在PSNR和SSIM指标上，MLP-Splatting与Feature-3DGS相当甚至更好（尤其在ScanNet上），证明了其强大的渲染能力。
    2.  **物体级分解能力**（表2）：与Feature-3DGS相比，表示同一个物体，MLP-Splatting所需的基元数量（MLP个数）仅为3DGS所需高斯数量的**不到1%**。这直观地展示了MLP基元的高表达力和物体级对齐能力。
    3.  **语义分割性能**（表3）：在mIoU和准确率上，MLP-Splatting与Feature-3DGS性能接近，并显著优于NeRF-DFF。
    4.  **效率对比**（表3）：MLP-Splatting的内存占用（32 MB）仅为Feature-3DGS（630 MB）的**1/15**，渲染速度（~8 FPS）是其（~3 FPS）的**近3倍**。
    5.  **交互式编辑**（图3）：展示了通过选择少量MLP基元，即可实现物体的**缩放、删除、颜色编辑和刚体变换**，无需任何分割掩码。

        - 表5：4个特征槽（feature slots）在性能和效率间取得了良好平衡。
        - 表6：通过控制八叉树阈值可以控制基元数量，影响模型大小和渲染质量。

### 3.5 技术细节

- **位置编码**：对6D输入的每个标量分量应用多分辨率位置编码，增强高频细节表达能力。
- **特征嵌入**：使用一个可学习的、方向相关的特征嵌入。通过将局部偏移 $\delta_i$ 投影到规范四面体框架的四个方向 $a_k$ 上，得到权重 $g_{ik}$，然后对四个可学习槽嵌入 $e_{ik}$ 进行加权求和，得到最终特征 $f_i$。这使得特征具有方向感知能力。
- **CUDA实现**：核心是自定义的基于块的CUDA渲染器。关键步骤包括：AABB-块视锥体裁剪、基于块中心虚拟光线的基元排序、以及将MLP权重加载到共享内存以实现线程块内所有像素的并行评估。
- **密度控制**：论文明确指出没有使用显式的密度控制策略。基元在训练中自然竞争，部分基元会“死亡”（尺度或透明度趋近于零）。这既是优点（简化了流程），也是局限（可能导致效率或质量下降）。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了3D场景表示中“表达力”与“物体级可编辑性”难以兼得的核心矛盾。
    - 提出了一种全新的、以“神经基元”为基本单元的范式，该范式天然地支持物体级分解。
    - 从理论和实践上证明了，这种范式可以在保持甚至提升渲染质量的同时，大幅降低内存消耗和提升渲染速度。
- **对未来研究/应用的影响**：
    - **场景表示新方向**：可能引领从“低级几何基元”向“高级神经基元”的转变，鼓励设计更具表达力和结构性的表示。
    - **3D内容创作**：为艺术家和设计师提供了一种直观、高效的3D场景编辑工具，可以直接操作“物体”而非“点云”或“网格”。
    - **机器人感知**：在机器人操作和导航中，一个天然物体级别的场景表示可以极大地简化物体识别、抓取和场景理解任务。
    - **AR/VR**：实现更真实的、可交互的虚拟场景，用户可以自然地选择和操作场景中的物体。
- **局限与开放问题**：
    - **缺乏密度控制**：没有显式的基元分裂/剪枝策略，可能导致在复杂区域基元不足或在简单区域基元冗余。作者指出这是一个未来改进方向。
    - **排序近似**：基于块中心虚拟光线的排序是一种近似，可能导致边界处排序错误。作者提到可以结合逐像素精炼来解决。
    - **场景规模**：当前实验主要在室内场景，对于大规模、无边界场景的扩展性有待验证。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“神经基元”是什么？**：可以把它想象成一个个“智能像素块”。每个“智能像素块”是一个小型的神经网络，它只负责学习并描述它所在的那一小块3D空间（比如一个茶杯的把手）是什么颜色、什么形状。这些“智能像素块”之间互不干扰，各自独立学习。
- **为什么MLP基元能“涌现”出物体？**：这就像一群画家被分配去画一幅巨大的画。如果每个画家只负责画一个很小的区域，并且他们之间不能交流，那么为了把整幅画画好，每个画家都会尽力把自己负责的区域画得最像。如果一个区域是纯色的（比如一个白色墙壁），一个画家就能轻松搞定。如果一个区域纹理复杂（比如一个花盆），可能需要几个画家合作。最终，每个画家负责的区域就会自然地对应到场景中的某个物体或物体的一部分。这就是“涌现”的过程。3DGS的高斯则像是一堆小色块，需要很多个才能拼出一个物体。
- **“等质量内存定律”的直观理解**：为了画出一条平滑的曲线，3DGS需要用很多小线段（高斯）去逼近，曲线越弯曲，需要的小线段就越多（$\varepsilon^{-2}$）。而MLP-Splatting的MLP基元本身就是一个能表示复杂曲线的“函数”，它只需要一个就能画出一整段平滑的曲线，只有在曲线突然断裂的地方（物体边界）才需要另一个MLP。因此，对于内部平滑的物体，它需要的“画笔”数量远少于3DGS（$\varepsilon^{-3/s}$，$s$ 越大表示内部越平滑，需要的参数越少）。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - 与**3DGS**相比，MLP-Splatting用更少、更强大的基元替代了大量高斯，实现了物体级分解。
    - 与**NeRF**相比，MLP-Splatting用多个局部MLP替代了一个全局MLP，实现了可分解性和高效渲染。
    - 与**Object-NeRF**（如ObjectSDF, GIRAFFE）相比，MLP-Splatting的物体分解是**无监督涌现**的，而非依赖显式的物体分支或编码。
    - 与**Feature-3DGS**等语义蒸馏方法相比，MLP-Splatting的语义是附着在“物体级”基元上的，而非“点级”高斯，因此更高效、更自然。
