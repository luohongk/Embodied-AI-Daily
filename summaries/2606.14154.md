## 0. 概览（Concise Summary）

- **主题（Topic）**：大语言模型（LLM）智能体的安全，具体针对“Agent Skills”（智能体技能）这一新型扩展机制中的跨模态攻击。
- **问题（Problem）**：Agent Skills 结合了自然语言指令（SKILL.md）和可执行代码，其安全性需要跨模态推理。现有防御手段（如规则扫描、提示注入检测）仅分析单一模态，无法有效检测利用两种模态交互产生的恶意行为。缺乏一个系统性的基准测试来评估和防御此类攻击。
- **方法（Method）**：提出了一个名为 SkillMutator 的基准测试框架，通过模拟攻击者，对良性技能进行“变异”以生成跨模态攻击样本。同时，提出了一种四阶段推理轨迹蒸馏框架，将前沿大模型的推理能力蒸馏到小型开源模型中，用于本地部署的恶意技能检测。
- **创新（Innovation）**：
    1.  首次识别并系统性地定义了 LLM Agent Skills 中的“语言-代码跨模态攻击”这一新威胁面。
    2.  提出了 SkillMutator，这是首个用于安装时检测此类跨模态攻击的基准测试，包含 13 个攻击类别和 187 个变异场景。
    3.  提出了一个四阶段推理轨迹蒸馏框架，使小型开源模型（7B）能够执行复杂的跨模态语义检测，性能超越 GPT-4o-mini，并接近 GPT-5.4 等前沿模型。
- **意义（Significance）**：该工作揭示了 Agent Skills 生态系统中一个被忽视的安全盲区，并提供了一个实用的、可本地部署的防御方案。它证明了在不依赖昂贵且存在隐私风险的第三方 API 的情况下，实现有效的跨模态攻击检测是可行的，对保障未来自主智能体系统的安全至关重要。
- **一句话总结**：本文首次系统性地研究了 LLM 智能体技能中的语言-代码跨模态攻击，并提出了一个基准测试（SkillMutator）和一个通过推理蒸馏实现的高效本地防御模型。

## 1. 研究动机（Motivation）

**研究动机**：解决 LLM 智能体技能（Agent Skills）中存在的、被现有安全研究忽视的“语言-代码跨模态攻击”问题。

**为什么重要？**
- **Agent Skills 的普及**：LLM 智能体正从单轮对话系统发展为能执行多步任务的自主系统。Agent Skills 作为一种运行时扩展机制，允许用户加载第三方开发的、打包了自然语言指令（SKILL.md）和可执行脚本的“技能包”，极大地增强了智能体的能力。公共技能市场（如 Anthropic Skills, ClawHub）正在快速增长。
- **新的攻击面**：Agent Skills 的行为由其自然语言描述和可执行代码共同决定。这种“跨模态”特性创造了一个新的攻击面。攻击者可以在 SKILL.md 中描述一个看似良性的工作流，同时嵌入隐含的恶意指令，引导智能体执行危险操作（如窃取文件），而配套的脚本和资源看起来却无害。
- **现有防御的盲区**：现有防御手段存在结构性缺陷：
    - **规则扫描器**（如 skill-security-scan）：仅搜索已知的恶意代码模式，无法理解自然语言指令如何改变代码行为的语义。
    - **商业扫描器**（如 Snyk Agent Scan）：同样局限于代码层面的静态分析。
    - **提示注入检测器**（如 LLM-Guard）：仅分析指令文本，无法将指令与代码行为进行关联分析。
    - 论文评估显示，现有扫描器对此类攻击的检测率极低（2%-17%）。

**挑战在哪里？**
- **跨模态语义鸿沟**：攻击的恶意性并非存在于单一模态，而是产生于自然语言指令和可执行代码之间的交互。例如，SKILL.md 中的一条“优化缓存”指令，可能被解释为让智能体去执行一个将敏感文件复制到持久化目录的脚本。检测这种“语义不一致”需要同时理解两种模态。
- **安装时检测的约束**：在技能被加载和执行之前进行检测（安装时检测）是必要的，因为执行本身可能触发恶意行为。但这要求检测器必须能处理未执行的、静态的代码和文本，且必须轻量、可本地部署，以避免隐私泄露和 API 成本。
- **缺乏基准测试**：此前没有专门针对这种跨模态攻击的基准测试，导致该领域的研究和防御进展缓慢。

## 2. 创新点（Innovation）

论文的关键创新点如下：

1.  **识别并定义新威胁**：首次系统性地识别并定义了 LLM Agent Skills 中的“语言-代码跨模态攻击”。该攻击的恶意行为并非源于单一模态，而是源于自然语言指令（SKILL.md）与可执行脚本/资源之间的交互。这填补了现有安全研究中的一个重要空白。

2.  **提出 SkillMutator 基准测试框架**：
    - **新颖性**：这是第一个用于安装时检测跨模态攻击的基准测试。它不是一个静态数据集，而是一个模拟自适应攻击者的“变异”框架。
    - **方法**：该框架从一个良性技能出发，通过四个阶段（技能分析、隐蔽攻击选择、场景生成、技能变异）将其“变异”成恶意技能。它还能利用扫描器的反馈进行迭代优化，以生成更具逃避性的样本。
    - **贡献**：生成了包含 13 个攻击类别、187 个变异场景的全面数据集，为评估和比较不同防御方法提供了标准化平台。

3.  **提出四阶段推理轨迹蒸馏框架**：
    - **新颖性**：将复杂的跨模态安全分析任务分解为四个结构化的推理阶段（目的锚定、范围外检测、安全原则推理、攻击类别标记），并使用一个“教师”模型（如 GPT-5.4）生成遵循此模式的推理轨迹。
    - **方法**：通过知识蒸馏，将这些推理轨迹“教”给一个小型开源“学生”模型（如 Qwen2.5-Coder-7B-Instruct），使其学会执行同样的复杂分析。
    - **贡献**：成功地将前沿大模型的检测能力压缩到一个可本地部署的 7B 模型中，实现了高检测率（88.2%），同时避免了第三方 API 的成本和隐私风险。这是首个在 Agent Skill 安全领域实现这种能力迁移的工作。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心架构包含两个部分：**SkillMutator 基准测试生成框架** 和 **四阶段推理蒸馏防御框架**。

**SkillMutator 基准测试生成框架**（图 1）：
该框架模拟一个自适应攻击者，其流程如下：
1.  **技能分析**：解析目标技能包，生成结构化摘要，包括技能目的、行为流、使用的工具、敏感资源、可修改文件等。
2.  **隐蔽攻击选择**：根据技能分析，为每个攻击类别计算“隐蔽性分数”（1-100）。分数高于 80 的类别被选中，以确保生成的恶意技能在上下文中看起来合理。
3.  **场景生成**：为每个选中的攻击类别，设计一个具体的攻击场景，包括良性表面行为、隐藏恶意行为、注入到 SKILL.md 的恶意指令、激活条件和预期影响。
4.  **技能变异**：根据生成的场景，对原始技能文件（SKILL.md、脚本、配置文件）进行修改。修改后的代码看起来无害，但恶意行为由 SKILL.md 中的指令触发。
5.  **迭代逃避优化**：使用现有扫描器（如 skill-security-scan, Snyk Agent Scan）对变异后的技能进行扫描。如果被检测到，则根据扫描器的反馈，重新生成变异，以移除可识别的签名，同时保留底层的语义不一致性。

**四阶段推理蒸馏防御框架**（图 3）：
该框架旨在训练一个本地部署的 LLM 扫描器。
1.  **教师端四阶段分析**：使用一个强大的“教师”模型（如 GPT-5.4）对恶意技能进行分析，并生成结构化的推理轨迹。这四阶段是：
    - **Phase 1: 目的锚定**：从 SKILL.md 中提取技能的声明目的和允许的操作范围。
    - **Phase 2: 范围外检测**：扫描所有文件，检测超出声明目的的操作。
    - **Phase 3: 安全原则推理**：将检测到的越界行为映射到 9 个安全原则（表 2）上。
    - **Phase 4: 攻击类别标记**：根据违反的原则，标记出具体的攻击类别（表 1）。
2.  **元数据确定性优化**：当教师模型的 Phase 4 输出缺失目标攻击类别时，使用变异日志中的元数据自动修正训练数据，确保训练信号完整。
3.  **学生模型蒸馏**：使用修正后的教师推理轨迹，通过 LoRA 微调一个小型“学生”模型（如 Qwen2.5-Coder-7B-Instruct）。
4.  **推理时前缀填充**：在推理时，强制模型输出从 Phase 4 的标题开始，以引导模型直接进入分类阶段，提高检测效率。

### 3.2 关键算法与数学推导

论文的核心是框架设计，而非复杂的数学推导。其关键算法体现在变异和蒸馏流程中，可以形式化描述如下：

**1. 安装时扫描任务形式化**：
给定一个技能包 $S = (M, E, A)$，其中：
- $M$: SKILL.md 的自然语言内容。
- $E$: 可执行文件集合。
- $A$: 辅助文件集合（配置文件、资源等）。
扫描器 $f_\theta$ 的目标是输出一个二元决策：
$$f_\theta(S) \rightarrow y, \quad y \in \{\text{FLAG}, \text{PASS}\}$$
当 $M, E, A$ 之间的交互导致的行为超出 $S$ 的声明目的，并属于表 1 中的攻击类别时，应标记为 `FLAG`。

**2. 隐蔽性分数计算**：
对于攻击类别 $c$ 和技能 $s$，隐蔽性分数 $Stealth(c, s)$ 由 LLM 根据以下四个标准评估：
- $c$ 不破坏 $s$ 的原始目的。
- $c$ 与 $s$ 已接触的资源相关。
- $c$ 可伪装成 $s$ 的正常行为。
- $c$ 可无缝插入 $s$ 的工作流。
选择条件为：$Stealth(c, s) > 80$ 或取前三名。

**3. 迭代逃避优化**：
设 $S^{(0)}$ 为初始变异，$D(S^{(i)})$ 为扫描器对第 $i$ 次变异的检测结果。优化目标是找到 $S^{(i+1)}$，使得：
$$D(S^{(i+1)}) < D(S^{(i)})$$
同时，变异 $S^{(i)}$ 的底层恶意意图 $I$ 保持不变。这通过一个 LLM 实现，它接收原始技能、攻击场景和检测历史，并遵循四个逃避原则（重命名、间接引用、重写关键词、伪装成维护）来生成新的变异。

### 3.3 模型、训练与数据集

- **模型**：
    - **教师模型**：GPT-5.4（用于生成推理轨迹）。
    - **学生模型**：Qwen2.5-Coder-7B-Instruct（主要实验对象），以及 Llama-3.1-8B-Instruct, Mistral-7B-Instruct-v0.3, Gemma-2-9b-it（用于泛化性验证）。
    - **对抗性预言机**：用于生成变异技能的 LLM，包括 GPT-4o-mini, GPT-5.4-mini, GPT-5.4。
    - **评估裁判**：GPT-5.4 和 Claude-Opus-4.7（用于判断扫描器的输出是否正确检测到攻击）。

- **训练**：
    - **方法**：LoRA（Low-Rank Adaptation）微调，参数为 $r=64, \alpha=128$，学习率 $5 \times 10^{-5}$，训练 5 个 epoch。
    - **数据**：使用 GPT-5.4 作为教师模型，对 68 个社区创作的技能进行变异，生成了 1,219 个训练样本。每个样本包含技能文件和对应的四阶段推理轨迹。

- **数据集**：
    - **评估数据集**：使用三个对抗性预言机（GPT-4o-mini, GPT-5.4-mini, GPT-5.4）对 17 个官方 Anthropic 技能进行变异，分别生成 48、63、76 个最终可评估的恶意技能场景。训练和评估使用的技能来源完全分离，以确保泛化性。

### 3.4 实验设置与结果

**实验设置**：
- **基线扫描器**：规则扫描器（skill-security-scan）、商业扫描器（Snyk Agent Scan, SkillScan）、提示注入检测器（LLM-Guard, PIGuard, DataSentinel）、专有 LLM（GPT-4o-mini, GPT-5.4-mini, GPT-5.4）。
- **评估指标**：检测率（Detection Rate），即被正确标记为恶意的变异技能占总数的比例。
- **硬件**：单张 NVIDIA A100 80 GB GPU。

**主要实验结果**：

1.  **现有扫描器失效**（表 4）：在最强对抗性预言机（GPT-5.4）生成的数据集上，所有现有扫描器的检测率均低于 20%，而 GPT-5.4 的检测率为 86.8%。这证实了跨模态攻击对现有防御的有效性。
2.  **蒸馏模型达到前沿水平**（图 6）：在 GPT-5.4 预言机数据集上，微调后的 Qwen2.5-Coder-7B-Instruct 检测率达到 **88.2%**，不仅远超其基础模型（17.1%），还超过了 GPT-4o-mini（23.7%）、GPT-5.4-mini（79.0%），甚至略高于教师模型 GPT-5.4（86.8%）。
3.  **泛化性验证**（图 5）：四阶段蒸馏方法对 Llama、Mistral、Gemma 等其他开源模型同样有效，检测率均有大幅提升（+48.7 到 +75.0 个百分点），证明了方法的通用性。
4.  **消融实验**：
    - **隐蔽攻击选择**（表 5）：使用 `select` 模式（仅选择高隐蔽性攻击）比 `no-select` 模式（应用所有攻击）生成的样本更难被检测，平均检测率下降 7.1 个百分点，证明了该模块的有效性。
    - **迭代逃避优化**（图 4）：经过一次迭代优化后，所有 LLM 扫描器的检测率均显著下降，表明该优化过程成功生成了更具逃避性的恶意技能。

### 3.5 技术细节

- **跨模态攻击示例**（图 2）：以 `mcp-builder` 技能的 `Data Exfiltration` 攻击为例。攻击者没有在网络层面进行数据窃取，而是在 SKILL.md 中注入一条看似无害的“复现性笔记”指令，并在 `evaluation.py` 中添加一个函数，该函数会收集连接信息（如 API 密钥）。最终，这些敏感信息被附加到用户发送给 LLM 的提示中，利用 LLM 的对话上下文作为隐蔽信道进行数据外泄。
- **安全原则**（表 2）：论文定义了 9 个安全原则（如目的-行为对齐、信任边界完整性、同意与透明度等），用于将检测到的越界行为进行结构化分类，为后续的攻击类别标记提供中间推理步骤。
- **元数据确定性优化**（§5.2）：当教师模型的 Phase 4 输出错误时，该步骤直接从变异日志中提取正确的攻击类别和证据，并将其插入到对应的 Phase 3 和 Phase 4 输出中。这保证了训练数据的质量，且无需额外调用 LLM。
- **推理时前缀填充**（§5.3）：在推理时，将模型输出初始化为 `## Phase 4: Attack Category Labeling`。这并非增加新信息，而是通过强制解码路径，引导模型直接进入最终分类阶段，从而提升检测效果（对 Qwen 模型有效）。

## 4. 意义与影响（Significance and Impact）

**解决的问题与贡献**：
- **填补安全空白**：首次系统性地揭示了 LLM Agent Skills 中“语言-代码跨模态攻击”这一被忽视的安全威胁，并提供了量化的证据。
- **提供评估基准**：SkillMutator 为社区提供了一个标准化、可扩展的基准测试，用于评估和比较未来针对此类攻击的防御方法。
- **实现实用防御**：提出的四阶段推理蒸馏框架成功地将前沿模型的检测能力“压缩”到可本地部署的小型模型中，解决了依赖第三方 API 带来的成本、隐私和延迟问题。这是向实用化 Agent 安全迈出的重要一步。

**对未来研究/应用的影响**：
- **安全研究新方向**：该工作开辟了 Agent 安全研究的新方向，即关注多模态（特别是语言和代码）交互产生的安全风险。未来的研究可以在此基础上探索更复杂的攻击模式和更鲁棒的防御机制。
- **Agent 生态系统安全**：随着 Agent Skills 市场的兴起，该工作为技能开发者、平台运营者和最终用户提供了重要的安全指导。它强调了在技能发布前进行跨模态安全审查的必要性。
- **本地化安全方案**：证明了通过知识蒸馏，可以在不牺牲太多性能的前提下，实现高度专业化的、可本地部署的安全模型。这为在资源受限或高隐私要求的环境中部署 AI 安全方案提供了范例。

**作者提到的局限与开放问题**：
- **攻击的局限性**：论文中的攻击是通过 LLM 生成的，可能无法覆盖所有真实世界的攻击模式。
- **防御的局限性**：防御模型依赖于教师模型生成的推理轨迹，其性能上限受限于教师模型的能力。此外，推理时前缀填充并非对所有模型都有效。
