## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人操作中的多任务模仿学习、扩散策略、混合专家模型（MoE）。
- **问题（Problem）**：现有的扩散策略在扩展到多任务场景时面临“效率-容量”的权衡：大模型计算成本高，小模型泛化能力差。基于混合专家模型（MoE）的方法虽然能缓解此问题，但其路由机制通常依赖于低层级的噪声或潜在统计信息，忽略了操作任务固有的**组合性**（compositional nature），导致可复用的行为（如“抓取”）被分散到不同的专家中，限制了模型的**可解释性**和**迁移性**。
- **方法（Method）**：提出**语义结构化混合专家扩散策略（SMoDP）**。该方法利用视觉-语言模型（VLM）离线地将演示数据分解为语义化的技能（verb-noun pairs），并训练一个轻量级的在线技能预测器。该预测器根据当前多模态上下文预测即将执行的技能，并以此指导MoE的路由。同时，采用**双对比对齐策略**（Inter-modal & Intra-modal）来确保路由的语义一致性。
- **创新（Innovation）**：1) 首次将**语言引导的技能语义**显式地引入MoE路由机制，使专家专门化与任务的结构化语义对齐。2) 提出一种**VLM辅助的离线技能抽象模块**，无需人工标注即可自动生成技能标签。3) 提出**双对比学习策略**，同时对齐技能预测与语言语义，并强制语义相似的技能激活相似的专家子集。
- **意义（Significance）**：SMoDP在多个多任务操作基准上取得了最优性能，显著提升了参数效率和数据效率。更重要的是，它通过冻结专家权重、仅微调技能预测器和路由器，实现了**组合性迁移**（compositional transfer），能够高效地适应新任务，为构建可解释、可复用、参数高效的通用机器人策略提供了新范式。
- **一句话总结**：本文提出SMoDP，通过将语言定义的技能语义结构注入混合专家模型的路由机制，实现了更高效、可解释且可组合迁移的多任务机器人操作策略。

## 1. 研究动机（Motivation）

- **核心问题**：扩散策略在单任务操作中表现出色，但扩展到多任务通用场景时面临一个“三难困境”：同时追求（1）扩散策略的精确控制，（2）大模型的广泛泛化能力，以及（3）实时部署所需的推理效率。现有方法通常只能满足其中一两个目标。
- **现有方法的不足**：
    - **标准扩散策略**：为了泛化到多任务，需要巨大的参数量，导致计算成本高昂。
    - **机器人基础模型**（如RDT, Pi-0）：通过扩展到数亿甚至数十亿参数实现了广度，但巨大的计算开销使其难以在资源受限的机器人平台上实时部署。
    - **现有MoE方法**（如MoDE）：虽然通过稀疏激活提升了效率，但其路由机制主要依赖低层级信号（如扩散噪声步长），**忽略了操作任务的语义和时间结构**。这导致在技能边界处出现“路由混乱”，例如，不同任务中语义相同的“抓取”行为可能被分配到不同的专家，如图1所示。这限制了行为的可复用性和可解释性。
- **挑战**：如何设计一种路由机制，使其能够理解任务的语义结构，将可复用的行为（技能）分配给相同的专家，从而实现高效的参数共享和可解释的专家专门化，同时保持计算效率。

## 2. 创新点（Innovation）

1.  **语义结构化的技能条件路由（Skill-Conditioned Routing）**：首次将**语言定义的技能语义**作为MoE路由的显式条件。不同于以往基于噪声或数据驱动隐式学习的路由，SMoDP的路由决策直接由预测的“下一个技能”驱动，使专家专门化与任务的组合性结构对齐。
2.  **VLM辅助的离线技能抽象模块（VLM-aided Offline Skill Abstraction）**：提出一个自动化流程，利用VLM（如Qwen3-VL）将无结构的演示轨迹分解为一系列开放词汇的“动词-名词”技能对（如 `<pick up, red cup>`）。这提供了丰富的语义监督信号，且无需昂贵的人工标注，VLM仅在训练时使用，推理时无需调用。
3.  **双对比对齐策略（Dual Contrastive Alignment）**：
    - **模态间对比学习（Inter-modal CL）**：将轻量级技能预测器的输出与VLM生成的、经冻结文本编码器编码的技能标签嵌入进行对齐。这桥接了多模态观测与语言定义的技能语义空间。
    - **模态内对比学习（Intra-modal CL）**：直接对路由器的输出（路由logits）进行约束，强制语义相似的技能（在语言空间中距离近）产生相似的路由分布，从而激活重叠的专家子集。
    - 这两个目标互补，共同确保了路由决策的语义一致性和鲁棒性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

SMoDP的整体架构如图2所示，包含两个主要阶段：

- **(a) 离线技能抽象**：使用VLM将原始演示数据`τ_n`和任务指令`g_n`处理成一系列带时间边界的技能标签`S_n`。流程包括：视频下采样（减少冗余帧）-> 语义技能提取（生成verb-noun对）-> 时间边界划分（确定每个技能在原始轨迹中的起止时间）。最终得到带语义标签的数据集`D_sem`。
- **(b) 技能条件扩散MoE策略**：在推理时，一个轻量级的**技能预测器**`P_φ`根据当前观测`o_t`和指令`g_n`，通过交叉注意力机制预测一个连续的技能嵌入`ˆz_skill`。这个嵌入被用于两个目的：
    1.  作为条件注入到扩散Transformer的每一层，影响特征提取。
    2.  作为路由器的输入，指导专家选择。路由器采用**块一致路由**（Chunk-Consistent Routing），即一个动作块内的所有动作token共享同一个由技能嵌入生成的路由logits，确保动作块内部执行的一致性。

### 3.2 关键算法与数学推导

1.  **扩散策略基础**：
    对动作块`x_0`加噪：`x_s = α_s x_0 + σ_s ϵ, ϵ ∼ N(0, I)`。
    去噪目标（预测噪声）：`L_diff = E[||ϵ - f_θ(x_s, o_t, g, s)||²]`。

2.  **轻量级技能预测器**：
    输入上下文token序列`C = [e_obs; e_goal]`，引入可学习查询`q_skill`，通过交叉注意力提取技能相关信息：
    `ˆz_skill = FFN(CrossAttn(q_skill, C, C))`。
    该嵌入与冻结文本编码器输出的技能标签嵌入位于同一空间。

3.  **技能条件MoE去噪**：
    在Transformer层`l`，更新过程为：
    `X_l = H_{l-1} + Attn_l(H_{l-1} + 1c^T)`，其中`c = W_c ˆz_skill`是广播到所有token的条件向量。
    `H_l = X_l + MoE_l(LN(X_l); ˆz_skill)`。

4.  **块一致技能路由**：
    对于MoE层`l`的输入特征`X_l`，将动作块部分的token替换为单个技能token `h_skill = W_z ˆz_skill`，形成新的路由输入`˜H_l`。
    路由器MLP预测路由logits：`˜Ψ_l = MLP(˜H_l)`。
    通过广播操作，使所有H个动作token共享技能token的路由logits，实现块内一致性：
    `Ψ_l_{1:T-H} ← ˜Ψ_l_{1:T-H}, Ψ_l_{T-H+1:T} ← ˜Ψ_l_{T-H+1}`。
    最终通过top-k稀疏化得到专家门控`G_l`。

5.  **双对比对齐损失**：
    - **模态间对比学习（InterCL）**：
        对于小批量中的样本`i`，其归一化预测技能嵌入`q_i`和归一化文本嵌入`u_i`。构建语义亲和矩阵`W`，其中权重`w_ij`基于`u_i`和`u_j`的相似度（仅保留top-m最相似样本）。
        InterCL损失：
        `L_interCL = -1/B Σ_i [1/Σ_j w_ij Σ_j w_ij log( exp(<q_i, u_j>/τ_c) / Σ_k exp(<q_i, u_k>/τ_c) )]`。
        该损失鼓励`q_i`与语义上相似的`u_j`在嵌入空间中对齐。

    - **模态内对比学习（IntraCL）**：
        将路由logits `Ψ_l_i`展平并归一化为路由特征向量`r_l_i`。
        IntraCL损失：
        `L_intraCL = 1/|L| Σ_l [ -1/B Σ_i (1/Σ_j w_ij) Σ_j w_ij log( exp(<r_l_i, r_l_j>/τ_c) / Σ_k exp(<r_l_i, r_l_k>/τ_c) ) ]`。
        该损失鼓励语义相似的技能（`w_ij`大）产生相似的路由分布（`r_l_i`和`r_l_j`接近）。

6.  **总训练目标**：
    `L = L_diff + λ_lb L_lb + λ_inter L_interCL + λ_intra L_intraCL`，其中`L_lb`是标准的MoE负载均衡正则项。

### 3.3 模型、训练与数据集

- **模型架构**：基于扩散Transformer（DiT），将前馈网络替换为MoE层。使用6个Transformer层，每层4个专家，top-2激活。技能预测器是一个单层交叉注意力加一个投影模块。
- **训练**：使用扩散去噪损失、负载均衡损失和双对比损失联合训练。VLM（Qwen3-VL）用于离线生成技能标签，Sentence-BERT作为冻结文本编码器。训练在单张NVIDIA RTX 4090上约需4小时。
- **数据集**：
    - **仿真**：LIBERO套件，包括LIBERO-10, LIBERO-90, LIBERO-OBJECT, LIBERO-GOAL, LIBERO-GOAL-OOD。每个任务提供50条演示（OOD任务为10条）。
    - **真实世界**：使用ALOHA双臂系统，包含4个任务（挂胶带、传递胶带、套杯套、递杯子），每个任务20条演示。

### 3.4 实验设置与结果

- **多任务性能（LIBERO-10/90）**：如图5所示，SMoDP在LIBERO-10和LIBERO-90上均取得了最高的平均成功率（0.95和0.97），显著优于DP-T, MoDE等基线，甚至超过了使用大规模数据集预训练的MoDE (Pretrain)。这表明语义路由提供了有效的归纳偏置。
- **数据利用效率（LIBERO-90）**：如图6所示，在不同数据量（5/10/25/50 demos/task）下，SMoDP始终优于MoDE和MoDE (Pretrain)。仅用50%数据训练的SMoDP性能就超过了用100%数据训练的MoDE，展示了卓越的数据效率。
- **任务迁移与组合泛化**：
    - **LIBERO-90 → LIBERO-10**：如表I所示，冻结专家，仅微调技能预测器和路由器（3.4%参数），SMoDP在1/5/10个演示的少样本迁移中大幅领先MoDE+LoRA和MoDE-scratch。
    - **LIBERO-OBJECT/GOAL → LIBERO-GOAL-OOD**：在更具挑战性的组合泛化任务中，SMoDP同样表现出色，尤其是在5和10个演示的设置下。
- **真实世界实验**：如表II所示，在ALOHA系统上，SMoDP的平均成功率（91.25%）远高于MoDE（47.50%），特别是在“递杯子”任务中，SMoDP取得了100%的成功率，而MoDE完全失败。
- **消融实验**：如表III所示，移除InterCL或IntraCL都会导致性能下降，同时移除两者下降更明显，证明了双对比策略的互补性。使用4个专家/层效果最佳。图7的热力图显示，不同技能确实激活了不同的专家模式，而语义相似的技能（如“approach cabinet”和“approach drawer”）激活模式也相似，验证了路由的语义一致性。

### 3.5 技术细节

- **VLM标注**：使用Qwen3-VL，下采样因子λ=5。通过结构化提示词强制输出“动词-名词”格式。
- **技能预测器**：使用一个可学习的`[SKILL]` token，通过交叉注意力从观测和指令token中提取信息。
- **块一致路由**：通过将动作块的所有token替换为一个技能token，并广播其路由logits来实现。这避免了同一动作块内不同动作token选择不同专家导致的控制不一致。
- **对比学习**：使用CWCL（Continuous Word-Cloud Learning）启发的连续权重，而非简单的0/1标签，以更好地处理语义相似性。保留比例ρ=0.1。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了多任务操作中MoE路由缺乏语义结构的问题，提出了一个将语言技能语义与专家专门化相结合的有效框架。
    - 提出了一种自动化的、无需人工标注的技能抽象方法，降低了应用门槛。
    - 通过双对比学习，确保了路由的语义一致性和鲁棒性。
    - 在仿真和真实世界实验中，证明了该方法在性能、数据效率、参数效率和组合迁移能力上的显著优势。

- **对未来研究/应用的影响**：
    - **可解释性**：SMoDP使得策略的决策过程更加透明。通过观察哪个专家被激活，可以推断出模型认为当前正在执行什么技能，有助于调试和理解模型行为。
    - **模块化与可复用性**：专家被训练成特定技能的“专家”，这些专家可以作为可复用的“技能模块”。通过组合不同的专家模块，可以快速适应新任务，这为构建通用机器人技能库提供了新思路。
    - **参数高效微调**：冻结专家权重，仅微调轻量级的技能预测器和路由器，使得在资源受限的机器人平台上进行快速、高效的少样本学习成为可能。

- **局限性与开放问题**（作者已指出）：
    - **粗粒度语义**：动词-名词对忽略了细粒度的控制动态（如速度、力）。
    - **VLM依赖**：标注质量受限于VLM的零样本推理能力，可能存在幻觉或边界不准确的问题。
    - **长尾技能分布**：数据集中的技能分布不均，可能导致路由器偏向于常见技能，忽略稀有但关键的技能。

## 5. 澄清与简化（Clarifications and Simplifications）

- **MoE路由的类比**：可以把MoE想象成一个公司的多个专业团队（专家）。传统路由（如MoDE）就像是一个随机分配任务的经理，他可能把“写报告”的任务分给设计团队，而把“做设计”的任务分给文案团队，导致效率低下。SMoDP的路由则像是一个懂业务的经理，他能识别出“写报告”和“做设计”是不同的技能，并总是把“写报告”的任务分配给文案团队，把“做设计”的任务分配给设计团队，从而让每个团队都成为其领域的专家。
- **双对比学习的类比**：
    - **Inter-modal CL**：就像让一个学生（技能预测器）学习一本词典（语言语义空间）。老师（VLM）已经给每个词（技能）标好了意思。学生通过对比学习，确保自己理解的词义（预测的嵌入）和词典上的标准解释（文本嵌入）一致。
    - **Intra-modal CL**：就像确保这个学生不仅知道词义，而且在做选择题（路由决策）时，对于意思相近的词（如“拿”和“取”），他会选择相似的答案（激活相似的专家）。这防止了学生因为一些细枝末节的差异而对近义词做出截然不同的选择。
- **块一致路由**：想象一下，你要完成一个“拿起杯子”的动作，这个动作由一系列连续的手部位置（动作块）组成。如果每个位置都由不同的专家来生成，动作可能会变得不连贯、抖动。块一致路由就像是让一个“抓取专家”来负责生成整个“拿起杯子”动作的所有位置，保证了动作的流畅和协调。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - 与**MoDE**（Mixture-of-Denoising Experts）直接相关，SMoDP是对其路由机制的改进，从噪声条件路由变为语义技能条件路由。
    - 与**STEER**和**SMP**等利用语言或技能抽象进行规划的工作不同，SMoDP将技能语义直接用于组织策略内部的专家路由，而非作为外部接口。
    - 与**CR-MoE**（Consistency Regularized MoE）等结构化路由工作相比，SMoDP的结构信号来源于外部的语言语义，而非内部的潜在相似性，更具可解释性。
