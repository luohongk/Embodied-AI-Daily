## 0. 概览（Concise Summary）

- **主题（Topic）**：双足机器人的社会导航（Social Navigation），特别是结合人类情感线索的导航。
- **问题（Problem）**：双足机器人因其复杂的全身动力学和运动约束，在动态、拥挤且充满社会交互（如人类情感）的环境中安全、高效且礼貌地导航极具挑战性。现有方法（如基于简化模型规划+跟踪的框架）存在模型误差，且忽略了人类情感对社交距离的影响。
- **方法（Method）**：提出一个名为 **EmoBipedNav** 的端到端深度强化学习（DRL）框架。该框架使用一种新颖的**顺序LiDAR栅格地图（LGMs）** 作为观测，该地图隐式编码了碰撞区域、由情感决定的“不适区”、社会交互以及系统动态演化。策略直接输出双足机器人的简化模型（ROM）动作（质心速度和偏航角），但训练过程与全阶动力学仿真器耦合，以消除模型误差。
- **创新（Innovation）**：
    1.  提出**顺序LiDAR栅格地图（LGMs）** 作为DRL观测，能高效、高分辨率地表示动态环境。
    2.  首次将**人类情感**（快乐、中性、消极）集成到双足机器人社会导航中，通过动态调整“不适区”大小实现情感感知导航。
    3.  提出**端到端**的导航框架，直接在全阶双足机器人动力学上进行训练，有效缓解了简化模型与真实机器人之间的动力学差异。
- **意义（Significance）**：这项工作填补了双足机器人情感感知社会导航的研究空白，为机器人在复杂、动态的人类环境中实现更自然、更礼貌的交互提供了可行方案。其端到端训练策略和创新的观测表示方法对提升双足机器人的实际部署能力具有重要意义。
- **一句话总结**：本文提出了EmoBipedNav，一个通过深度强化学习将人类情感融入双足机器人社会导航的端到端框架，利用新颖的LiDAR栅格地图和全阶动力学训练，实现了安全、高效且具有社交礼仪的导航。

## 1. 研究动机（Motivation）

- **核心问题**：双足机器人在充满行人的动态环境中进行安全且具有社会意识的导航非常困难。
- **挑战与空白**：
    1.  **双足机器人自身动力学复杂**：双足机器人具有高度非线性、高自由度、混合动力学的特点，其运动能力受限，难以像轮式机器人那样灵活机动。传统的“规划-跟踪”框架（先使用简化模型规划路径，再用全身控制器跟踪）存在**模型误差**，即简化模型（如LIP模型）无法精确反映真实机器人的运动能力（如跟踪误差、关节力矩限制），导致规划出的轨迹在实际执行时可能不安全。
    2.  **社会交互的复杂性**：行人之间的交互、行人与机器人之间的交互是隐式且难以建模的。现有方法要么忽略交互，要么使用预训练的预测模型，但这些模型通常只基于行人-行人数据，忽略了机器人对行人行为的影响，导致部署时出现偏差。
    3.  **情感线索的缺失**：人类的情感状态（如愤怒、快乐）会显著影响其偏好的个人空间（舒适距离）。现有社会导航研究大多将行人简化为简单的圆形障碍物，忽略了情感这一重要的社会线索，导致机器人无法做出符合社交礼仪的避让行为。
- **为什么重要**：解决这些问题对于双足机器人从实验室走向真实世界（如商场、医院、家庭）至关重要。实现情感感知的社会导航不仅能提升机器人的安全性和效率，更能使其行为更自然、更易被人类接受，从而促进人机协作。

## 2. 创新点（Innovation）

论文的核心创新点在于提出了一个**端到端、情感感知、全阶动力学训练**的双足机器人社会导航框架，具体体现在以下四个方面：

1.  **新颖的DRL观测表示：顺序LiDAR栅格地图（LGMs）**：
    - **与现有方案的不同**：现有方法要么依赖完全可观测的状态（如行人位置、速度），这需要复杂的感知和预测模块；要么使用原始的LiDAR扫描或传统的占据栅格地图（OGMs）。LGMs是一种**饼形栅格地图**，其分辨率从中心向外递减，使得靠近机器人的区域拥有更高的分辨率，这对于局部避障至关重要。通过堆叠历史时间步的LGMs，可以捕获环境的时空动态信息（如行人运动、交互）。
    - **为什么能奏效**：LGMs提供了一种**部分可观测**但信息丰富的表示，隐式地包含了碰撞区域、由情感决定的不适区、行人-机器人交互以及系统动态演化。这种表示避免了显式建模复杂交互的难题，同时其高分辨率特性使得机器人能更精确地感知近距离障碍物。

2.  **情感集成**：
    - **与现有方案的不同**：首次将行人情感（快乐、中性、消极）作为影响导航决策的关键因素。根据心理学研究，为不同情感状态的行人分配不同的“不适区”大小（快乐：0.2m，中性：0.35m，消极：0.5m）。
    - **为什么能奏效**：通过将情感信息编码到LGMs的“不适区”中，并在奖励函数中设置相应的惩罚项，DRL策略能够学习到对不同情感状态的行人保持不同的社交距离，从而实现情感感知的、更具社交礼仪的导航行为。

3.  **端到端导航框架**：
    - **与现有方案的不同**：传统方法将导航和运动控制解耦。本文提出的框架是一个端到端的管道，它将从LGMs中提取的潜在特征直接映射到双足机器人的动作（质心速度和偏航角）。关键在于，这个策略是在**包含全阶机器人动力学的物理仿真器**中训练的。
    - **为什么能奏效**：通过在训练中引入真实的机器人动力学和约束（如跟踪误差、关节限制），DRL策略能够**隐式地学习并补偿**这些模型误差。这使得学习到的策略在部署到真实机器人上时，其表现与仿真中的表现更加一致，避免了“规划-跟踪”框架中因模型不匹配导致的性能下降。

4.  **全面的基准测试和硬件验证**：
    - 与多种SOTA方法（包括基于模型的和基于DRL的）进行了对比，并在真实的双足机器人Digit上进行了Sim-to-Real迁移实验，证明了方法的有效性和泛化能力。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整个框架（图2）是一个端到端的系统，工作流程如下：

1.  **输入**：
    - **情感估计**：使用预训练的CNN模型从行人面部图像中估计其情感（快乐、中性、消极）。
    - **LiDAR扫描**：获取机器人周围的2D LiDAR原始扫描数据。
2.  **观测构建**：
    - 将原始LiDAR扫描转换为**顺序LiDAR栅格地图（LGMs）**。这些LGMs不仅包含碰撞区域（红色），还根据估计的情感，在行人周围叠加了不同大小的“不适区”（蓝色）。
    - 将最近N个时间步的LGMs堆叠起来，形成一个时空观测 $o^{sp-tem}_k$。
3.  **特征提取与决策**：
    - 使用一个**CNN编码器**从堆叠的LGMs中提取潜在特征 $f^o_k$。
    - 将潜在特征 $f^o_k$ 与机器人状态（到目标的距离和角度 $s^{goal}_k$）以及上一个动作 $u_{k-1}$ 拼接起来。
    - 拼接后的向量输入到**SAC（Soft Actor-Critic）** DRL框架的Actor和Critic网络中。
    - Actor网络输出动作 $u_k = \{u^v_k, u^{\Delta\theta}_k\}$，即期望的**质心（CoM）前进速度**和**躯干偏航角**。
4.  **底层控制**：
    - 动作 $u_k$ 被传递给一个基于简化模型（LIP）的规划器（ALIP），生成期望的落脚点。
    - 一个基于无源性的全身控制器（Passivity-based Controller）驱动机器人的所有关节，以跟踪期望的落脚点和偏航角，实现全阶动力学运动。
    - **关键**：整个训练过程是在MuJoCo仿真器中，使用Digit的**全阶动力学模型**进行的，因此DRL策略能够学习到如何应对底层控制器的跟踪误差。

### 3.2 关键算法与数学推导

1.  **简化模型（LIP模型）**：
    双足机器人的动力学常用线性倒立摆（LIP）模型近似。在第 $k$ 个步行步，局部矢状面内的状态转移方程为：
    $$d_k = \frac{\upsilon^{loc}_k \cosh(\omega T) - u^v_k}{\omega} \sinh(\omega T)$$
    $$\Delta x^{loc}(d_k) = \frac{\upsilon^{loc}_k \sinh(\omega T)}{\omega} + d_k (1 - \cosh(\omega T))$$
    其中，$u^v_k$ 是控制输入（期望矢状面速度），$\Delta x^{loc}$ 是CoM位置增量，$\upsilon^{loc}_k$ 是当前CoM速度，$T$ 是步长时间，$\omega = \sqrt{g/H}$。

2.  **动作空间**：
    导航策略的动作定义为：
    $$u_k = \{u^v_k, u^{\Delta\theta}_k\}, \quad 0 \le u^v_k \le \bar{u}^v, \quad |u^{\Delta\theta}_k| \le \bar{u}^{\Delta\theta}$$
    其中 $u^v_k$ 是期望的矢状面速度，$u^{\Delta\theta}_k$ 是期望的偏航角变化量。

3.  **问题形式化（POMDP）**：
    将社会导航问题建模为部分可观测马尔可夫决策过程（POMDP），定义为6元组 $(S, O, A, \Gamma, R, \gamma)$。
    - $S$: 完全可观测状态空间（目标位置、上一个动作）。
    - $O$: 观测空间（顺序LGMs）。
    - $A$: 动作空间（$u_k$）。
    - $\Gamma$: 状态转移模型（由全阶Digit动力学决定）。
    - $R$: 奖励函数。
    - $\gamma$: 折扣因子。
    目标是找到最优策略 $\pi(a|s, o)$ 以最大化期望累积折扣奖励：
    $$V = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k r_k \right]$$

4.  **LiDAR栅格地图（LGM）分辨率**：
    第 $i$ 个环上的栅格分辨率为：
    $$g_r = \frac{K}{M^2} \frac{\pi L^2}{(2i-1)}$$
    其中 $K$ 是LiDAR光束数，$M$ 是环数和角分段数，$L$ 是最大探测半径。该公式表明，靠近机器人（$i$ 小）的栅格分辨率更高。

5.  **奖励函数设计**：
    总奖励 $r_k = r^{col}_k + r^{goal}_k + r^{emo}_k$。
    - **碰撞避免奖励** $r^{col}_k$：根据与障碍物的最小距离 $d^{all}_{min}$ 和 $d^{static}_{min}$ 给予惩罚。
    - **目标到达奖励** $r^{goal}_k$：到达目标给予正奖励，否则根据与目标距离的减少量给予正向激励。
    - **情感感知奖励** $r^{emo}_k$：如果机器人进入任何行人的情感相关不适区（$d_{p_i} < d^{emo}_{p_i} + r_{robot}$），则给予惩罚 $-0.1$。

### 3.3 模型、训练与数据集

- **机器人模型**：Agility Robotics的**Digit**双足机器人，在**MuJoCo**物理仿真器中使用其全阶动力学模型。
- **行人模型**：使用**ORCA（Optimal Reciprocal Collision Avoidance）** 算法模拟多智能体（行人）的交互运动。行人以1m/s的速度在随机起点和终点之间移动。
- **训练算法**：**SAC（Soft Actor-Critic）** 算法。网络结构包括一个共享的CNN编码器，以及两个MLP构成的Actor和Critic网络。
- **数据集**：没有使用真实世界数据集。训练环境是随机生成的，包括随机位置和尺寸的静态障碍物，以及随机初始位置、目标和情感状态的行人。行人情感在训练时是预设的，以方便Sim-to-Real迁移。

### 3.4 实验设置与结果

- **实验设置**：
    - 在MuJoCo仿真环境中进行训练和测试。
    - 机器人从 $(-4, 0)$ 导航到 $(4, 0)$。
    - 静态障碍物随机分布在 $x \in [-3, 3], y \in [-1.5, 1.5]$ 区域。
    - 行人在半径为 $3.5$ 到 $4.5$ 米的圆周上随机初始化。
    - 每个实验进行500次随机测试，评估**成功率（SR）**、**导航时间（NT）** 和**不适次数（DT）**。

- **主要结果（表II）**：
    - **与SOTA对比**：EmoBipedNav取得了最高的成功率（0.950），显著优于DWA (0.892)、LNDNL (0.660)、DRL-VO* (0.912) 和 LiDAR-SAC (0.932)。同时，其不适次数（226）也低于大多数方法，表明其在安全性和社交礼仪上取得了更好的平衡。
    - **消融实验**：
        - **Single-LGM**（仅使用单帧LGM）成功率尚可（0.928），但导航时间（33.15s）显著增加，说明缺乏时序信息导致策略过于保守。
        - **OGM**（使用传统占据栅格地图）成功率（0.944）接近，但不适次数（312）更高，证明了LGM的高分辨率优势。
        - **LNDNL**表现最差，证实了其网络结构在处理复杂动力学时的不稳定性。

- **情感集成实验（表III，图7）**：
    - 使用可变不适距离（对应不同情感）训练的策略，其学习曲线与使用固定0.2m不适距离的相似，表明策略能有效适应情感变化。
    - 使用固定0.5m不适距离训练时，成功率略低（0.938），因为过于保守的策略在ORCA模拟的激进行人面前更难成功。
    - **模型误差影响**：仅在LIP模型上训练的策略，部署到全阶Digit上时，成功率从0.962骤降至0.900，不适次数从202飙升至642，这**直接证明了模型误差的严重性**，并凸显了EmoBipedNav端到端全阶训练策略的价值。

### 3.5 技术细节

- **网络结构**：编码器由4层CNN组成（32个3x3卷积核，步长分别为2,1,1,1），后接一个全连接层输出50维特征。Actor和Critic网络均为2层1024个神经元的MLP。
- **训练参数**：学习率0.001，折扣因子 $\gamma=0.99$，步长时间 $T=0.4s$，最大矢状面速度 $\bar{u}^v=0.4m/s$，最大偏航角 $\bar{u}^{\Delta\theta}=0.2rad$。
- **Sim-to-Real**：在真实机器人上，使用预训练的CNN模型实时估计行人面部情感，并将仿真中训练好的策略直接部署到Digit上，验证了其泛化能力。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  解决了双足机器人社会导航中因**模型误差**导致的性能下降问题，提出了一个有效的端到端训练范式。
    2.  首次将**人类情感**这一复杂社会线索引入双足机器人导航，使机器人能够做出更符合社交礼仪的避让行为。
    3.  提出了一种新颖且高效的**观测表示方法（LGM）**，为处理动态环境中的局部感知问题提供了新思路。

- **对未来研究/应用的影响**：
    - **方法论影响**：其端到端训练策略和LGM表示方法可以推广到其他类型的移动机器人（如四足、轮式）的社会导航研究中。
    - **应用前景**：为双足机器人在商场、医院、家庭等人类密集环境中的实际部署铺平了道路，使其行为更自然、更可接受。
    - **人机交互**：强调了社会线索（如情感）在机器人决策中的重要性，推动了更具“情商”的机器人研究。

- **局限与开放问题**：
    - **奖励函数调优**：作者提到识别最优奖励配置很困难，当前是经验性调优，可能存在更优解。
    - **情感模型**：当前仅使用了三种离散情感，且不适距离是固定的。未来可考虑更连续、更复杂的情感模型，以及情感随时间动态变化的情况。
    - **行人模型**：仿真中使用的ORCA模型可能无法完全模拟真实行人的复杂行为。未来需要更逼真的行人模拟器。
    - **计算资源**：端到端训练需要大量的计算资源。

## 5. 澄清与简化（Clarifications and Simplifications）

- **简化模型（ROM） vs. 全阶模型**：可以把双足机器人想象成一个非常复杂的木偶，有几十根线（关节）需要同时控制。简化模型（如LIP）就像只考虑木偶的重心，规划出一条简单的路径。但实际拉线时，木偶的胳膊腿会晃动，导致实际路径偏离规划。EmoBiped
