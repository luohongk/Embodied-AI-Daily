## 0. 概览（Concise Summary）

- **主题（Topic）**：导航定位，具体为在GNSS信号弱的环境下，利用伪卫星（PL）增强的惯性/卫星组合导航。
- **问题（Problem）**：在GNSS信号受限（如城市峡谷、隧道）的环境中，传统最小二乘（LS）定位方法精度低、鲁棒性差，且现有伪卫星集成方法多基于LS，未能充分利用惯性测量单元（IMU）的历史信息和优化框架。
- **方法（Method）**：提出一种**松耦合的因子图优化（FGO）框架**。该框架将GNSS/伪卫星的**最小二乘（LS）定位解**作为因子，与**IMU预积分因子**进行融合，通过图优化迭代求解最优状态估计。
- **创新（Innovation）**：首次将因子图优化（FGO）框架应用于伪卫星（PL）增强的导航系统中，实现了GNSS、PL和IMU三种传感器的松耦合融合。
- **意义（Significance）**：实验证明，在GNSS信号严重受限（仅4颗高仰角卫星）的场景下，该方法相比传统LS方法，3D定位误差降低了22.8%至41.3%，显著提升了定位的精度和鲁棒性，为伪卫星在复杂环境下的实际应用提供了有效方案。
- **一句话总结**：本文提出了一个松耦合因子图优化框架，通过融合GNSS/伪卫星的最小二乘解与IMU数据，在GNSS信号受限环境下显著提升了定位精度和轨迹平滑度。

## 1. 研究动机（Motivation）

- **核心问题**：全球导航卫星系统（GNSS）在隧道、城市峡谷等复杂环境中，信号会受到遮挡、多径效应和中断，导致定位精度急剧下降甚至完全失效。
- **现有方案的不足**：
    1.  **GNSS+IMU组合**：虽然IMU能提供高频运动信息，但存在累积漂移问题，需要频繁的绝对位置修正。在GNSS信号长期不佳时，漂移会迅速增大。
    2.  **GNSS+伪卫星（PL）组合**：伪卫星可以部署在信号盲区，提供额外的测距信号，改善几何构型。然而，现有的大多数GNSS/PL集成方法仍基于**最小二乘（LS）** 或**卡尔曼滤波（EKF）**，这些方法通常只利用当前时刻的测量值，难以充分利用历史信息进行平滑和优化。
    3.  **因子图优化（FGO）的潜力**：FGO作为一种先进的优化框架，能够自然地融合多源传感器数据，并利用历史测量值进行批量优化，已被证明在GNSS/INS组合中优于EKF。但**尚未有工作将FGO应用于伪卫星增强的导航系统中**。
- **挑战**：如何设计一个有效的FGO框架，将GNSS/PL的定位解（通常是LS解）与IMU数据以松耦合的方式融合，并验证其在真实场景下的性能提升。

## 2. 创新点（Innovation）

- **首次提出PL增强的FGO框架**：这是本文最核心的创新。作者首次将因子图优化（FGO）应用于伪卫星（PL）辅助的导航系统中，填补了该领域的研究空白。
- **松耦合融合架构**：不同于紧耦合（直接融合原始伪距/载波相位），本文采用**松耦合**方式。它将GNSS/PL的**最小二乘（LS）定位解**（即已经计算出的位置）作为一个“虚拟测量”因子（GNSS/PL-Position Factor）输入到因子图中，与IMU预积分因子进行融合。这种架构降低了系统复杂度，并允许直接使用现有的GNSS/PL接收机输出。
- **利用FGO的优势**：通过构建包含历史状态节点的因子图，FGO能够利用过去所有时刻的测量信息进行**批量优化**，从而获得比仅依赖当前时刻的LS方法更平滑、更鲁棒的轨迹估计。这尤其适用于GNSS信号稀疏的场景。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文提出的松耦合FGO框架的核心是构建一个因子图，其结构如图1所示。

- **状态节点**：每个时刻 $t_i$ 的状态变量 $\mathbf{x}_i$ 定义为：
  $$\mathbf{x}_i = \{ \mathbf{R}_b^{e,i}, \mathbf{p}_b^{e,i}, \mathbf{v}_b^{e,i}, \mathbf{b}_{a,i}, \mathbf{b}_{g,i} \}$$
  其中 $\mathbf{R}_b^{e,i}$ 是旋转矩阵，$\mathbf{p}_b^{e,i}$ 是位置，$\mathbf{v}_b^{e,i}$ 是速度，$\mathbf{b}_{a,i}$ 和 $\mathbf{b}_{g,i}$ 分别是加速度计和陀螺仪的偏置。
- **因子类型**：
    1.  **IMU预积分因子**：连接两个连续状态节点 $\mathbf{x}_i$ 和 $\mathbf{x}_j$，利用IMU测量值预积分来约束它们之间的相对运动。
    2.  **IMU偏置因子**：约束相邻时刻的IMU偏置变化，使其平滑变化。
    3.  **GNSS/PL位置因子**：连接状态节点 $\mathbf{x}_i$，将GNSS/PL的LS定位解作为观测值，约束该时刻的位置。
    4.  **先验因子**：在初始时刻添加，用于固定初始状态，使优化问题有唯一解。

### 3.2 关键算法与数学推导

- **IMU预积分因子**：采用[9]中的预积分公式，其误差函数为：
  $$\|\mathbf{e}_{imu}^{ij}\|^2 = \left\| \begin{bmatrix} \mathbf{r}_{\Delta R_{ij}} \\ \mathbf{r}_{\Delta v_{ij}} \\ \mathbf{r}_{\Delta p_{ij}} \end{bmatrix} \right\|^2_{\Sigma_{imu}}$$
  其中 $\mathbf{r}_{\Delta R_{ij}}, \mathbf{r}_{\Delta v_{ij}}, \mathbf{r}_{\Delta p_{ij}}$ 分别是旋转、速度和位置的残差向量，$\Sigma_{imu}$ 是其协方差矩阵。

- **IMU偏置因子**：误差函数为：
  $$\|\mathbf{e}_{b}^{ij}\|^2 = \|\mathbf{b}_{a,j} - \mathbf{b}_{a,i}\|^2_{\Sigma_{ba}} + \|\mathbf{b}_{g,j} - \mathbf{b}_{g,i}\|^2_{\Sigma_{bg}}$$
  其中 $\Sigma_{ba}$ 和 $\Sigma_{bg}$ 分别是加速度计和陀螺仪偏置的协方差矩阵。

- **GNSS/PL位置因子**：首先，通过LS方法从GNSS和PL的伪距测量值中解算出天线位置 $\tilde{\mathbf{p}}_{ant}^{e,i}$。然后，根据状态变量估计天线位置：
  $$\mathbf{p}_{ant}^{e,i} = \mathbf{p}_b^{e,i} + \mathbf{R}_b^{e,i} \mathbf{l}_{ant}^b$$
  其中 $\mathbf{l}_{ant}^b$ 是从IMU中心到天线的杆臂。误差函数为：
  $$\|\mathbf{e}_p^i\|^2 = \left\| \mathbf{p}_{ant}^{e,i} - \tilde{\mathbf{p}}_{ant}^{e,i} \right\|^2_{\Sigma_P}$$
  其中 $\Sigma_P$ 是LS位置解的协方差矩阵。

- **整体优化问题**：最终的目标函数是上述所有因子误差的加权和，通过最小化该函数来求解所有状态变量 $\hat{\mathbf{x}}$：
  $$\hat{\mathbf{x}} = \arg \min_{\mathbf{x}} \left( \|\mathbf{e}_0\|^2 + \sum_{i=1}^{K} \|\mathbf{e}_p^i\|^2 + \sum_{i=1}^{K-1} \left( \|\mathbf{e}_{imu}^{i,i+1}\|^2 + \|\mathbf{e}_{b}^{i,i+1}\|^2 \right) \right)$$
  其中 $\mathbf{e}_0$ 是先验因子，$K$ 是状态节点的数量。

### 3.3 模型、训练与数据集

- **模型**：本文提出的FGO模型本身不需要训练，它是一个基于优化的模型。核心是构建因子图并求解非线性最小二乘问题。
- **数据集**：论文使用**真实世界数据**进行验证。实验场景是慕尼黑联邦国防军大学的一个停车场。
- **实验设备**：
    - **移动平台（Rover）**：搭载了MEMS IMU、多频段GNSS/PL天线、软件定义无线电（SDR）用于信号处理、工业计算机用于数据记录，以及一个NovAtel接收机（结合SAPOS服务）提供厘米级精度的参考轨迹。
    - **伪卫星**：两个PL发射器，广播类似Galileo E1BC的信号。
    - **基站**：一个静态基站，用于提供单差伪距校正。
- **场景模拟**：为了模拟GNSS信号严重受限的环境，实验仅使用**4颗仰角最高的GPS卫星**，并分别测试了加入0个、1个或2个PL发射器的情况。总评估时长为80秒。

### 3.4 实验设置与结果

- **对比方法**：
    - **LS**：标准最小二乘方法（仅使用GNSS/PL，无IMU）。
    - **FGO**：本文提出的松耦合因子图优化方法。
- **评估指标**：平均绝对误差（MAE）和最大误差，分别计算2D（水平）和3D误差。同时报告了精度衰减因子（DOP），包括PDOP、HDOP和VDOP。
- **主要结果（见表I和图2）**：
    1.  **FGO全面优于LS**：在所有信号配置下，FGO的MAE和最大误差均显著低于LS。3D误差的MAE降低了**22.8%至41.3%**，2D误差的MAE降低了**11.9%至21.1%**。最大的提升出现在GPS-only（无PL）的场景。
    2.  **PL提升定位精度**：与GPS-only基线相比，加入PL发射器后，FGO的定位精度进一步提高。轨迹图（图2）显示，加入PL后，轨迹的横向偏差明显减小。
    3.  **性能取决于几何构型**：并非PL数量越多效果越好。例如，GPS+PL02（单个PL）在2D误差上表现最好，而GPS+2PL（两个PL）在3D误差上表现最好。这表明PL的**空间几何分布**比数量更重要。
    4.  **DOP与误差趋势一致**：DOP值（PDOP, HDOP, VDOP）的变化趋势与定位误差基本一致，验证了DOP作为几何构型质量指标的有效性。但DOP不能完全解释所有误差，因为多径、时钟偏差等也会影响最终精度。

### 3.5 技术细节

- **松耦合实现**：GNSS/PL的LS解算和FGO优化是两个独立的模块。LS模块输出位置解和协方差，作为FGO的输入因子。
- **IMU预积分**：在两个连续的GNSS/PL测量时刻之间，高频的IMU数据被预积分成一个单一的因子，这大大减少了因子图中状态节点的数量，提高了计算效率。
- **杆臂补偿**：在位置因子中，通过公式 $\mathbf{p}_{ant}^{e,i} = \mathbf{p}_b^{e,i} + \mathbf{R}_b^{e,i} \mathbf{l}_{ant}^b$ 精确补偿了天线与IMU中心之间的杆臂。
- **开源库**：FGO框架基于开源的 `GTSAM` 库实现。

## 4. 意义与影响（Significance and Impact）

- **解决了什么问题**：本文成功解决了在GNSS信号严重受限环境下，如何有效融合伪卫星和IMU数据以提高定位精度和鲁棒性的问题。它证明了FGO框架在伪卫星增强导航中的巨大潜力。
- **主要贡献**：
    1.  提出了首个将伪卫星与FGO结合的导航框架。
    2.  通过真实世界实验，定量证明了该框架相比传统LS方法的显著优势。
    3.  揭示了伪卫星的几何构型对定位性能的关键影响，为实际部署提供了指导。
- **对未来研究/应用的影响**：
    - **推动伪卫星实用化**：该研究为伪卫星在自动驾驶、无人机、机器人等领域的实际应用提供了更可靠、更先进的算法支持。
    - **启发更紧密的融合**：作者在结论中明确指出，未来将研究**紧耦合**集成。这意味着直接融合原始伪距/载波相位测量值，有望进一步提升精度和鲁棒性，尤其是在动态和复杂环境中。
    - **拓展FGO应用边界**：本文展示了FGO框架在处理非传统传感器（如伪卫星）方面的灵活性，鼓励更多研究者将FGO应用于其他新型传感器融合问题。
- **局限与开放问题**：
    - **松耦合的局限**：松耦合方式下，LS解算本身可能已经引入了误差，FGO无法直接利用原始的伪距信息进行更精细的优化。
    - **计算复杂度**：随着时间窗口增长，因子图中的节点和因子数量增加，计算复杂度会上升，可能不适用于实时性要求极高的应用。
    - **实验规模**：实验仅在80秒的短时间窗口和特定场景下进行，其长期稳定性和在不同环境下的泛化能力有待进一步验证。

## 5. 澄清与简化（Clarifications and Simplifications）

- **因子图优化（FGO）是什么？**
    - **通俗解释**：想象你在一个黑暗的房间里走路，手里拿着一个计步器（IMU）和一个偶尔能闪一下的相机（GNSS/PL）。计步器告诉你每一步的方向和距离，但会累积误差（漂移）。相机偶尔能拍到一张照片，告诉你一个绝对位置，但照片可能模糊（有噪声）。
    - **LS方法**：就像你每次看到照片时，就完全相信它，然后根据它重新调整你的位置。你之前走过的路径信息（计步器数据）就被丢弃了。
    - **FGO方法**：就像你有一个聪明的“记忆地图”。你不仅记录下每次看到照片时的位置，还记录下所有计步器的步数。然后，你**一次性回顾整个旅程**，利用所有信息（所有照片和所有步数）来**重新计算**一条最平滑、最可能正确的路径。这样，即使某张照片很模糊，你也能通过前后步数的约束来修正它，得到更准确的轨迹。

- **松耦合 vs. 紧耦合**：
    - **松耦合**：就像你有一个“黑盒子”接收机，它自己算出一个位置（比如“我在坐标(1,2)”），然后你把这个位置告诉FGO。FGO不关心这个位置是怎么算出来的。
    - **紧耦合**：就像FGO直接读取接收机内部的原始测量数据（比如“到卫星A的距离是100米，到卫星B的距离是200米”），然后自己来计算位置。这通常更复杂，但能利用更多信息，理论上精度更高。

- **为什么FGO比LS好？**
    - **利用历史信息**：FGO利用了过去所有时刻的测量值，而LS只用了当前时刻。这就像“三个臭皮匠，顶个诸葛亮”，更多的信息（尤其是IMU提供的连续运动信息）能有效平滑噪声和抑制异常值。
    - **迭代优化**：FGO通过多次迭代来寻找最优解，而LS通常是一次性计算。迭代过程可以不断修正初始估计，收敛到更优的解。
