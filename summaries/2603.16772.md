## 0. 概览（Concise Summary）

- **主题（Topic）**：辅助机器人、移动操作、共享控制、四足机器人、人机交互、Cybathlon竞赛。
- **问题（Problem）**：现有为轮椅使用者设计的辅助机器人（如安装在轮椅上的机械臂）存在移动性受限、操作范围小、占用空间大、续航能力差等问题。同时，完全自主的移动操作机器人在复杂、非结构化环境中的可靠性不足，而纯遥操作又给用户带来巨大的认知负担。
- **方法（Method）**：提出一个基于四足机器人（ANYmal）和机械臂（DynaArm）的“按需”辅助系统。该系统采用**共享控制（Shared Autonomy）** 框架，结合了**半自主任务执行**（如自主导航、简单抓取）和**人类遥操作**（通过口控摇杆进行精细操作）。系统包含两个核心支柱：用于精细控制的**手动控制栈**和用于自主导航与任务执行的**自动化栈**。
- **创新（Innovation）**：
    1.  **独立移动平台**：首次将四足机器人作为辅助平台，克服了轮式或轮椅安装式机器人的地形限制和空间占用问题，扩展了用户的可达范围（如上下楼梯）。
    2.  **面向行动不便者的专用遥操作接口**：设计了基于QuadStick（口控摇杆）的直观控制映射，将机器人的9个自由度映射到有限的输入通道上，并设计了两种末端执行器控制模式以适应不同任务。
    3.  **双机器人协作策略**：在Cybathlon竞赛中，提出并实现了双机器人协作策略，一个机器人辅助当前任务，另一个自主移动到下一个任务点，显著减少了操作员的等待时间。
    4.  **家庭环境下的自然语言导航**：在家庭场景中，集成了语音接口、大语言模型（LLM）和语义地图，允许用户通过自然语言指令（如“帮我拿个苹果”）让机器人自主导航到目标物体附近。
- **意义（Significance）**：这项工作展示了四足机器人在辅助行动不便者方面的巨大潜力，提供了一个灵活、强大且易于使用的解决方案。它不仅在Cybathlon竞赛中取得了优异成绩，更在真实家庭环境中验证了其有效性，为未来开发更独立、更通用的个人辅助机器人奠定了基础。
- **一句话总结**：本文提出并验证了一个基于四足机器人的共享控制辅助系统，通过结合自主导航与口控遥操作，有效帮助行动不便者在竞赛和家庭环境中完成多种日常任务。

## 1. 研究动机（Motivation）

- **核心问题**：如何为轮椅使用者（尤其是四肢严重受损者）提供一个既不影响其自身移动性，又能提供强大、灵活操作能力的辅助机器人系统？
- **现有方案的局限性**：
    - **轮椅安装式机械臂**：这是当前主流方案，但存在固有缺陷：
        - **牺牲移动性**：增加机械臂会增大轮椅尺寸、降低灵活性，并消耗轮椅电池，影响续航。
        - **操作范围有限**：受限于机械臂的尺寸和轮椅的位置，无法触及远处或低处的物体（如地板上的物品）。
        - **负载能力弱**：如商业产品iARM仅能提起1.5kg的物体。
        - **安全隐患**：系统故障可能直接影响轮椅的基本功能。
    - **现有移动操作机器人研究**：相关研究较少，且存在局限：
        - 部分工作仅针对特定任务，缺乏通用性。
        - 部分工作由外部护理人员操作，而非使用者本人。
        - 现有用户直接控制的系统（如Padmanabha等人的工作）存在两个关键问题：1）用户需要手动导航机器人，这从固定视角看很困难且耗费精力；2）其采用的轮式机器人（Hello Robot Stretch）负载小、无法通过复杂地形。
- **为什么重要**：独立性是健康和幸福的重要方面。辅助机器人技术有望恢复行动不便者的独立性，减少对护理人员的依赖。然而，现有技术无法在**不限制用户移动性**的前提下提供**强大、灵活且易于控制**的操作能力。因此，需要探索新的平台和控制范式。
- **挑战**：
    - **硬件选择**：需要一个既能快速移动、又能负载重物、还能适应复杂地形的平台。
    - **控制复杂性**：如何让一个拥有9个自由度的机器人被一个四肢行动不便的用户直观、高效地控制？
    - **认知负荷**：长时间的精细遥操作会给用户带来巨大的精神压力。如何通过自动化来减轻这种负担？
    - **环境适应性**：如何在高度结构化的竞赛环境和完全非结构化的家庭环境中都能可靠工作？

## 2. 创新点（Innovation）

1.  **独立四足机器人平台**：首次将四足机器人（ANYmal）作为个人辅助平台。与轮式或轮椅安装式方案相比，其核心优势在于：
    - **地形通过性**：可以上下楼梯、越过门槛，极大地扩展了用户的可达空间。
    - **独立性**：拥有独立电源和计算单元，不消耗轮椅资源，且系统故障不影响用户自身移动。
    - **灵活性**：机器人可以离开用户去执行任务（如去另一个房间取物），实现并行操作。

2.  **面向行动不便者的专用遥操作接口**：设计了一套基于QuadStick（口控摇杆）的精细控制映射方案。其创新点在于：
    - **降维映射**：将机器人的9个自由度（3个基座+6个机械臂）巧妙地映射到QuadStick有限的7个输入通道上。
    - **双末端执行器控制模式**：设计了“前端（EE Control Mode Front）”和“顶部（EE Control Mode Top）”两种模式，分别优化了桌面操作和地面操作，使控制更直观。
    - **模式无关输入**：共享的摇杆映射（水平/垂直对应x/y轴）和通道映射（如通道1控制z轴），降低了学习成本。
    - **与操作员协作开发**：控制映射方案是通过与竞赛操作员（Samuel Kunz）数月迭代反馈开发的，确保了其直观性和实用性。

3.  **双机器人协作策略（Two-Robot Solution）**：针对Cybathlon竞赛时间紧迫的特点，提出并实现了双机器人协作策略。一个机器人协助用户完成当前任务，另一个机器人则自主移动到下一个任务点待命。这极大地减少了用户等待机器人重新定位的时间，是竞赛成功的关键因素之一。

4.  **家庭环境下的自然语言导航与语义地图**：在家庭场景中，将自动化栈与LLM和语义地图（TagMap）结合，实现了基于自然语言指令的自主导航。用户只需说“帮我拿个苹果”，系统就能解析指令、定位目标物体并规划路径，将机器人导航到目标附近，然后交由用户进行精细操作。这大大降低了用户的操作门槛和认知负荷。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整个系统采用**共享控制（Shared Autonomy）** 架构，由两个核心栈组成，如图2所示：

- **手动控制栈（Manual Control Stack，蓝色）**：负责精细操作。用户通过QuadStick（口控摇杆）发出控制信号，经过控制映射后，直接控制机器人的基座移动或机械臂末端执行器。
- **自动化栈（Automation Stack，绿色和紫色）**：负责自主任务执行。它接收来自用户接口（GUI或语音）的激活信号，然后自主执行预定义的任务，如导航到指定位置、抓取特定物体等。

系统在任何时刻只能处于一个状态：要么完全遥操作，要么完全自主运行。

### 3.2 关键算法与数学推导

论文中未涉及复杂的数学推导，但描述了几个关键算法模块：

1.  **低层控制器**：
    - **OCS2 (MPC控制器)**：用于精确的6自由度全身控制。它基于模型预测控制，可以控制基座倾斜和移动，同时保持四足着地，从而扩展机械臂的工作空间。其公式可概括为求解一个优化问题：
        $$ \min_{u} \sum_{k=0}^{N} (x_k - x_{k,ref})^T Q (x_k - x_{k,ref}) + u_k^T R u_k $$
        其中 $x_k$ 是状态，$u_k$ 是控制输入，$Q$ 和 $R$ 是权重矩阵。
    - **强化学习策略**：用于3自由度（x, y, yaw）的基座移动。该策略在仿真中通过强化学习训练得到，用于快速、鲁棒的导航。

2.  **Cybathlon自动化栈**：
    - **赛道定位**：提出了一种基于赛道胶带标记的轻量级定位方法。算法流程为：
        1.  使用TEED（一种小型CNN）进行边缘检测。
        2.  使用霍夫变换进行直线检测。
        3.  通过过滤启发式（如线宽、垂直性）筛选出胶带线。
        4.  检测胶带线交点作为地标。
        5.  使用滤波融合更新（如卡尔曼滤波）将观测到的地标与机器人状态估计（腿式里程计+LiDAR）结合，修正漂移。
    - **关键点地图与行为引擎**：基于竞赛地图定义了一个关键点图，每个关键点包含位置、期望臂姿和红绿灯标志。行为引擎（Behavior Tree）负责协调两个机器人，通过红绿灯机制避免碰撞，并控制自动化模块的调用。

3.  **家庭自动化栈**：
    - **语义地图生成**：使用TagMap（基于RecognizeAnything模型）对场景中的物体进行检测和定位，生成包含物体名称和粗略位置的语义地图。
    - **自然语言指令解析**：
        1.  使用Whisper进行语音转文字。
        2.  将文本指令和语义地图中的物体列表输入给LLM（ChatGPT）。
        3.  LLM根据指令和上下文，选择最合适的物体作为目标。
    - **导航**：使用概率路线图（PRM）在几何地图上规划全局路径，然后使用Waverider的定制版进行局部路径跟踪。

### 3.3 模型、训练与数据集

- **模型**：
    - **机器人平台**：ANYmal D（四足基座）+ DynaArm（机械臂）+ Robotiq 2F-140（夹爪）+ Intel RealSense L515（RGB-D相机）。
    - **计算平台**：机载Jetson Orin（高带宽GPU处理）+ 操作员笔记本电脑Aorus 15P（低带宽GPU处理）。
    - **低层控制器**：OCS2（MPC）和强化学习策略（在仿真中训练）。
    - **视觉模型**：TEED（边缘检测CNN）、FoundationPose（物体位姿估计）、RecognizeAnything（图像标记）。
    - **语言模型**：Whisper（语音识别）、ChatGPT（LLM）。
- **训练**：
    - **操作员训练**：操作员Samuel Kunz在赛前进行了约40小时、11次训练，以熟悉控制方案和竞赛流程。
    - **强化学习策略**：在仿真环境中训练，用于基座移动。
    - **自动化模块**：如苹果抓取（颜色掩码过滤）、香料抓取（FoundationPose定位货架）等，是任务特定的，无需大量训练数据。
- **数据集**：论文未使用公开数据集。Cybathlon竞赛环境是标准化的，家庭实验环境是操作员自己的家。

### 3.4 实验设置与结果

实验分为两部分：

1.  **Cybathlon 2024 竞赛**：
    - **设置**：在标准化的赛道上完成10个日常任务，限时10分钟。
    - **结果**：
        - **最佳成绩**：在资格赛中，以7分49秒完成8个任务（跳过Door任务）。
        - **决赛成绩**：以8分12秒完成8个任务（跳过Door任务，成功完成Spice Up但Mailbox因操作失误未得分）。
        - **时间管理**：自动化栈的引入将2024年2月挑战赛的任务时间从6分34秒缩短至4分52秒。
        - **结论**：系统在高压竞赛环境下表现出色，证明了其鲁棒性和效率。

2.  **家庭环境案例研究**：
    - **设置**：在操作员家中完成7个任务，分为混合方法任务（使用自主预定位）和纯遥操作任务。每个任务重复3次，每次完成后进行主观问卷评估（基于NASA-TLX）。
    - **任务**：取零食（苹果）、取饮料（水瓶）、扔垃圾（纸巾）、微波炉加热、开关灯、倒水插吸管、插充电器。
    - **结果**：
        - **任务完成**：所有任务均成功完成，包括首次尝试的零样本任务（取苹果、取水瓶）。
        - **主观评估**：问卷结果（表1、表2）用于评估精神需求、时间需求、努力程度、挫败感等。论文指出，结果与先前最先进的工作相当，同时允许操作员拥有更多自主权和更大的操作灵活性。
        - **结论**：系统在真实、非结构化的家庭环境中有效，能够辅助完成多种日常任务，并且自动化功能显著减轻了操作员的认知负荷。

### 3.5 技术细节

- **QuadStick控制映射**：这是论文的核心技术细节之一。图5展示了QuadStick的布局，图6和表6a详细说明了不同控制模式下摇杆和呼吸通道的映射关系。例如，在基座控制模式下，摇杆控制x/y平移，通道1控制偏航。在末端执行器控制模式下，摇杆控制x/y平移，通道1控制z轴升降，而次要映射则控制旋转轴（如俯仰、滚转）。

## 4. 意义与影响（Significance and Impact）

- **解决的问题**：
    - 解决了现有轮椅安装式机械臂移动性受限、操作范围小的问题。
    - 解决了现有移动操作机器人研究在通用性、用户控制便利性和环境适应性上的不足。
    - 提供了一个结合了自主性和遥操作优势的共享控制框架，有效平衡了任务效率和用户认知负荷。

- **贡献**：
    - **系统级贡献**：提出了一个完整的、经过验证的、基于四足机器人的个人辅助系统，从硬件选择到软件架构再到用户接口，都进行了精心设计。
    - **接口设计贡献**：为行动不便者设计了一个直观、高效的遥操作接口，并通过竞赛和家庭实验验证了其有效性。
    - **应用贡献**：在Cybathlon竞赛和真实家庭环境中成功展示了系统的能力，为四足机器人在辅助领域的应用提供了强有力的实证。

- **对未来研究/应用的影响**：
    - **平台选择**：本研究将激励更多研究者探索四足机器人作为个人辅助平台的可能性，尤其是在需要克服复杂地形（如楼梯）的场景中。
    - **共享控制范式**：本文的共享控制框架（结合自主导航与精细遥操作）为设计更高效、更易用的辅助机器人提供了范本。
    - **人机交互**：基于口控摇杆和语音/LLM的交互方式，为严重运动障碍者提供了更自然、更强大的控制手段。
    - **从竞赛到现实**：论文强调了从竞赛环境到家庭环境的迁移，这种“竞赛驱动+现实验证”的研究模式值得借鉴。

- **局限与开放问题**：
    - **Door任务**：对于四足机器人平台来说，开门并让用户和两个机器人通过的任务效率较低，是未来需要优化的方向。
    - **自动化可靠性**：自动化模块（如Spice Up任务中的抓取）仍可能出错，需要更鲁棒的感知和规划算法。
    - **零样本能力**：家庭实验中的大部分任务并非零样本，操作员需要短暂熟悉。提升系统的零样本泛化能力是未来的目标。
    - **长期部署**：论文仅进行了短期实验，系统的长期可靠性、维护成本和用户接受度有待进一步研究。

## 5. 澄清与简化（Clarifications and Simplifications）

- **共享控制（Shared Autonomy）**：可以理解为“自动驾驶”和“手动驾驶”的结合。在宽敞、安全的道路上（如机器人从一个房间走到另一个房间），使用“自动驾驶”（自动化栈），让机器人自己走，用户不用管。在需要精细操作时（如抓取一个易碎的杯子），切换到“手动驾驶”（手动控制栈），用户通过口控摇杆精确控制机器人的手。这样既省力又精准。
- **QuadStick控制映射**：想象一下，你只有一个游戏摇杆和几个吹气/吸气按钮，却要控制一个拥有9个关节的机器人。这就像用两个旋钮去弹钢琴。论文的巧妙之处在于，它设计了不同的“模式”（Mode），就像给钢琴换了一张乐谱。在“基座模式”下，摇杆控制机器人前后左右走；在“前端模式”下，摇杆控制机械手在桌面上前后左右移动；在“顶部模式”下，摇杆控制机械手在另一个平面移动。通过切换模式，有限的输入设备就能控制复杂的机器人。
- **双机器人策略**：这就像在F1赛车中，当一辆车进站换胎时，另一辆车已经提前在下一个弯道待命，准备接替比赛。在Cybathlon中，用户完成一个任务后，不需要等待机器人慢悠悠地走到下一个任务点，因为另一个机器人已经在那里等着了。这大大提高了效率。
- **家庭自动化栈中的LLM**：用户说“我有点想吃健康的零食”，系统里的LLM（ChatGPT）就像一个聪明的管家。它知道语义地图里有哪些东西（苹果、薯片、巧克力），然后它会根据“健康的零食”这个要求，判断出“苹果”是最合适的，并告诉机器人去拿苹果。这比用户必须精确说出“去厨房桌子上拿那个红苹果”要方便得多。
