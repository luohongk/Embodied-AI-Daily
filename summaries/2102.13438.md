# Accurate Visual-Inertial SLAM by Feature Re-identification

- arXiv: https://arxiv.org/abs/2102.13438v1
- arXiv ID: 2102.13438

---

## 0. 概览（Concise Summary）

- **主题（Topic）**：视觉-惯性SLAM（VI-SLAM），具体为通过特征重识别（Feature Re-identification）来提升定位精度。
- **问题（Problem）**：现有VI-SLAM方法的前端特征提取与匹配高度依赖相邻帧之间的特征跟踪，在具有挑战性的场景（如遮挡、大视角变化）中，特征跟踪容易中断，导致视觉测量不足，从而在长时间运行中累积位姿估计误差。
- **方法（Method）**：提出一种新颖的特征重识别方法。该方法构建一个时空敏感（Spatial-Temporal Sensitive, STS）的子全局地图，并利用IMU辅助的位姿引导匹配，从该子地图中重新识别出当前帧中已存在的特征。这些跨越长时间间隔的重识别特征作为增强的视觉测量，被融入局部和全局的束调整（Bundle Adjustment, BA）优化中，以逐步减少累积误差。
- **创新（Innovation）**：1）提出了STS子全局地图的概念，高效地筛选出与当前帧有空间重叠且时间久远的关键帧，避免了在全地图上进行重识别的计算负担。2）提出了一种融合IMU、几何和外观约束的位姿引导特征匹配方法，能够在大视角和外观变化下鲁棒地建立特征对应关系。3）将重识别特征作为新的视觉约束，同时融入局部BA和全局BA，实现了对漂移的有效抑制。
- **意义（Significance）**：该方法显著提升了VI-SLAM在挑战性场景下的定位精度和鲁棒性，在EuRoC和TUM-VI数据集上分别实现了67.3%和87.5%的绝对平移误差（ATE）降低，且计算开销很小。该方法可轻松应用于现有VI-SLAM系统，具有很高的实用价值。
- **一句话总结**：本文通过构建时空敏感子地图并进行IMU引导的特征重识别，为VI-SLAM系统提供了跨越长时间间隔的稳定视觉约束，有效抑制了位姿漂移。

## 1. 研究动机（Motivation）

**问题**：在视觉-惯性SLAM系统中，前端模块（如特征提取与匹配）的性能直接决定了后端优化的质量。当前最先进的VI-SLAM方法（如ORB-SLAM3、VINS-Mono等）主要依赖**相邻帧之间的特征跟踪**（如KLT光流法）或**局部共视图内的特征匹配**来建立视觉约束。然而，这种方法存在一个根本性缺陷：在**挑战性场景**下，例如：
- **快速运动或大视角变化**：特征点的外观和位置在相邻帧间变化剧烈，导致跟踪失败。
- **遮挡**：特征点被其他物体遮挡，导致跟踪中断。
- **纹理缺失或重复**：特征点难以被稳定跟踪或匹配。

当特征跟踪中断后，该特征点对应的3D地图点将无法再提供视觉测量，导致系统可用的视觉约束减少。长期运行下，这种信息缺失会使得位姿估计的**累积误差**（drift）不断增大，最终导致地图和轨迹的严重偏差。

**重要性**：解决这个问题对于实现长时间、高精度的自主导航、增强现实（AR）和机器人应用至关重要。一个能够从“丢失”的特征中重新建立连接的系统，将能有效抵抗漂移，构建更一致、更精确的全局地图。

**挑战**：直接对全局地图中的所有特征点进行重识别是不现实的，因为全局地图规模会随时间无限增长，导致计算量爆炸。此外，需要重识别的特征往往经历了**长时间间隔**和**大视角变化**，其外观和几何位置与最初观测时差异巨大，传统的基于描述子的直接匹配方法（如BoW）效果不佳。因此，挑战在于如何**高效**且**鲁棒**地实现跨越长时间和大视角的特征重识别。

## 2. 创新点（Innovation）

本文的核心创新在于提出了一套完整的**特征重识别（Feature ReID）** 框架，该框架包含两个关键组件：

1.  **时空敏感（STS）子全局地图的构建**：
    - **创新点**：不同于传统的局部共视图（只包含时间上邻近的关键帧）或全局地图（包含所有关键帧），STS子全局地图是一个动态构建的、大小受限的子集。它只包含与当前帧在**空间上有重叠**，且**时间上较久远**的关键帧中的地图点。
    - **与现有方案的不同**：传统方法（如ORB-SLAM的局部地图）只考虑与当前帧在共视图上相连的邻近关键帧，这些关键帧在时间上很近，无法提供长跨度的约束。而回环检测（Loop Closure）虽然能建立长跨度连接，但通常只在检测到回环时才触发，且依赖于图像级的匹配，无法提供逐点的特征约束。本文的STS子地图则是在**每一帧**都主动构建，旨在为当前帧的每个新特征寻找其在历史地图中的对应点。
    - **为什么能奏效**：通过空间重叠筛选，确保了重识别的几何可能性；通过时间优先（优先选择更早的关键帧），确保了重识别能提供**更长的时间跨度**，从而更有效地约束漂移。同时，通过限制子地图大小（如1000个点），保证了计算效率。

2.  **位姿引导的特征匹配（Pose Guided Matching）**：
    - **创新点**：提出一种融合了**IMU辅助位姿预测**、**几何一致性检查**和**外观描述子匹配**的三阶段鲁棒匹配方法。
    - **与现有方案的不同**：传统方法（如PnP+RANSAC）通常先进行2D-3D匹配，再求解位姿。本文则**先利用IMU预积分预测一个相对准确的当前帧位姿**，然后基于这个预测位姿将STS子地图中的3D点投影到当前帧图像上，从而将搜索范围从全局缩小到局部区域。随后，通过几何一致性检查（验证关键帧与当前帧的相对位姿）剔除错误匹配，最后通过图像块扭曲（warping）和ORB描述子距离计算来确认最终匹配。
    - **为什么能奏效**：IMU提供的短时高精度位姿预测为特征匹配提供了强有力的先验，使得即使在视角变化很大的情况下，也能将3D点投影到图像上的正确区域。几何一致性检查则有效过滤了由于动态物体或错误投影导致的异常匹配。图像块扭曲则补偿了视角变化带来的外观差异，提高了描述子匹配的鲁棒性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出的VI-SLAM系统架构如图3所示。其核心是在标准VI-SLAM流程（前端特征提取/跟踪 + 后端局部/全局BA）中，插入了一个**特征重识别（Feature ReID）模块**。

- **输入**：立体相机图像和IMU数据。
- **前端**：进行局部特征检测、跟踪和IMU预积分。
- **后端**：
    - **局部BA**：优化一个滑动窗口内的最新帧和地图点。
    - **全局BA**：优化所有关键帧和地图点，以维护全局一致的地图。
- **特征重识别模块**：该模块连接前端和后端。对于当前帧中**新检测到**的特征（即未被跟踪到的特征），它执行以下步骤：
    1.  **构建STS子全局地图**：从全局地图中，根据空间和时间约束，筛选出一个包含约1000个地图点的子集。
    2.  **位姿引导匹配**：利用IMU预测的当前帧位姿，将STS子地图中的3D点投影到当前帧，并通过几何和外观约束找到与当前帧2D特征的最佳匹配。
    3.  **约束增强**：将成功重识别的特征作为新的视觉约束，分别加入到局部BA和全局BA的代价函数中进行优化。

### 3.2 关键算法与数学推导

**1. 视觉-惯性SLAM代价函数**

系统状态包括相机位姿 $T_i = (R_i, p_i)$，IMU状态 $M_i = (v_i, b_i)$（速度与偏置），以及3D地图点 $X_j$（由逆深度 $\rho_j$ 参数化）。

- **视觉重投影误差**：
  $$E_{vis}^{ij}(T_i, X_j) = \pi(T_i \circ X_j) - z_{ij}$$
  其中 $\pi$ 是相机投影函数，$z_{ij}$ 是第 $i$ 帧中观测到的2D特征点。

- **局部BA代价函数** $E_L$（公式1）：
  $$E_L = \arg \min_{T_i, M_i, \rho_j} \sum_{i=t_0}^{t_n} \sum_{j \in V^i} \|E_{vis}^{ij}(T_i, T_{s_j}, \rho_j)\| + \|E_{prior}^{t_0}(M_{t_0}, T_{t_0})\| + \sum_{i=t_0}^{t_n-1} \|E_{imu}^{i,i+1}(M_i, M_{i+1}, T_i, T_{i+1})\|$$
  其中 $V^i$ 是第 $i$ 帧中可见的地图点集合，$s_j$ 是地图点 $X_j$ 首次被重建的帧索引。

- **全局BA代价函数** $E_G$（公式2）：
  $$E_G = \arg \min_{T_i, M_i, \rho_j} \sum_{i=k_1}^{k_m} \sum_{j \in V^i} \|E_{vis}^{ij}(T_i, T_{s_j}, \rho_j)\| + \sum_i \|E_{rel}^i(\{T_{k \in L_i}\})\| + \sum_{i=k_1}^{k_m-1} \|E_{imu}^{i,i+1}(M_i, M_{i+1}, T_i, T_{i+1})\|$$
  其中 $k_1, ..., k_m$ 是关键帧索引，$E_{rel}^i$ 是边缘化产生的相对约束。

**2. 特征重识别**

- **IMU辅助位姿预测**（公式3）：利用IMU预积分技术，从上一时刻的最优状态 $T_i, v_i, b_i$ 和IMU测量值 $(\tilde{\omega}_k, \tilde{\alpha}_k)$ 预测当前时刻的位姿 $T_{i+1}$ 和速度 $v_{i+1}$。
  $$R_{i+1} = R_i \prod_{k=k_1}^{k_n} \exp((\tilde{\omega}_k - b_{g_k} - \eta_{g_d}^k)\Delta t)$$
  $$v_{i+1} = v_i + g\Delta t_{i,i+1} + \sum_{k=k_1}^{k_n} R_k(\tilde{\alpha}_k - b_{a_k} - \eta_{a_d}^k)\Delta t$$
  $$p_{i+1} = p_i + \sum_{k=k_1}^{k_n} [v_k \Delta t + \frac{1}{2}g\Delta t^2 + \frac{1}{2}R_k(\tilde{\alpha}_k - b_{a_k} - \eta_{a_d}^k)\Delta t^2]$$

- **视觉约束增强**：成功重识别的特征点集记为 $AugV^i$。
  - **局部BA增强**（公式4）：
    $$E'_L = \arg \min_{\{T_i, \rho_j | s_j \notin [t_0, t_n]\}} \sum_{i=t_0}^{t_n} \sum_{j \in AugV^i} \|E_{vis}^{ij}(T_i, T_{s_j}, \rho_j)\|$$
    注意，这里只优化当前滑动窗口内的相机位姿 $T_i$ 和地图点 $\rho_j$（如果其首次重建帧 $s_j$ 在窗口外，则不优化该点，使其作为锚点）。
  - **全局BA增强**（公式5）：
    $$E'_G = \arg \min_{\{T_i, \rho_j\}} \sum_{i=k_1}^{k_m} \sum_{j \in AugV^i} \|E_{vis}^{ij}(T_i, T_{s_j}, \rho_j)\|$$
    在全局BA中，所有重识别特征对应的地图点都会被优化。

- **重识别验证**（公式6）：计算重识别地图点 $\rho_j$ 的平均重投影误差。
  $$E_{vis}(\rho_j) = \frac{1}{|Z_j|} \sum_{i=t_0}^{t_n} \|E_{vis}^{ij}(T_i, T_{s_j}, \rho_j)\|$$
  如果 $E_{vis}(\rho_j)$ 大于阈值 $T_{rep}$，则丢弃该点及其所有观测。

### 3.3 模型、训练与数据集

- **模型**：本文提出的是一种**基于图优化的、紧耦合的立体视觉-惯性SLAM系统**。它不是一个需要训练的深度学习模型，而是一个基于几何和优化的传统方法。特征描述子使用的是经典的**ORB**描述子。
- **训练**：**无训练过程**。该方法完全基于手工设计的特征（ORB）和几何约束。
- **数据集**：
    - **EuRoC数据集**：由微型飞行器（MAV）在室内外场景采集，包含11个序列，提供20Hz的立体灰度图像和200Hz的IMU数据，以及运动捕捉系统提供的真值。
    - **TUM-VI基准**：由头戴式设备在室内房间采集，包含多个序列，提供20Hz的512x512图像和200Hz的IMU数据，以及OptiTrack系统提供的真值。

### 3.4 实验设置与结果

- **评价指标**：
    - **平均时间跨度（ATS）**：地图点从首次重建到最近一次观测之间的帧数差。
    - **平均跟踪长度（ATL）**：地图点被观测到的总帧数。
    - **绝对平移误差（ATE）**：估计轨迹与真值轨迹之间的均方根误差（RMSE）。
- **主要结果**：
    - **特征重识别有效性（表I）**：与基线方法（ICE-BA）相比，本文方法在EuRoC上将ATS从20.96帧提升至43.28帧（提升106%），在TUM-VI上从15.78帧提升至79.11帧（提升401%）。ATL也有小幅提升。
    - **消融研究（表II）**：验证了STS子地图和位姿引导匹配两个组件都至关重要。单独使用STS子地图（x）或位姿引导匹配（y）都能降低ATE，但两者结合（x+y）效果最佳，在EuRoC上ATE从9.8cm降至3.2cm，在TUM-VI上从8.8cm降至1.1cm。
    - **与SOTA方法对比（表IV和表V）**：
        - **EuRoC（表IV）**：本文方法（无回环）的平均ATE为3.2cm，优于VINS（13.8cm）、SOFT-SLAM（6.4cm）、ICE-BA（8cm），并与ORB-SLAM3（3.7cm，有回环）相当甚至更优。
        - **TUM-VI（表V）**：本文方法（无回环）的平均ATE为1.1cm，远优于VINS（9.5cm）和ICE-BA（8.8cm），与ORB-SLAM3（1.0cm，有回环）几乎持平。
    - **时间分析（表III）**：本文方法在跟踪线程中比基线方法（ICE-BA）多花费约4-6ms（EuRoC上从11.25ms增至16.92ms），但远低于ORB-SLAM3（51.46ms）。全局BA线程的时间开销也略有增加。

### 3.5 技术细节

- **STS子地图构建**：通过计算每个关键帧中3D地图点的空间分布（均值和方差）来近似其“视场”。当前帧的视场由其参考关键帧代替。通过比较视场立方体是否相交来判断空间重叠。在空间重叠的关键帧中，优先选择时间戳更早的，并限制总地图点数为1000。
- **位姿引导匹配流程**：
    1.  **IMU预测**：使用公式(3)预测当前帧位姿。
    2.  **投影**：将STS子地图中的3D点投影到当前帧图像上。
    3.  **几何一致性检查**：对于每个候选关键帧，计算其与当前帧的相对位姿，并检查其地图点投影后是否与当前帧检测到的特征点空间位置接近。如果大部分点都接近，则认为该关键帧是“无漂移”的，否则剔除其所有地图点。
    4.  **外观匹配**：对于通过几何检查的候选点，利用预测位姿对图像块进行扭曲（warping）以补偿视角变化，然后计算其ORB描述子与当前帧特征点描述子之间的距离。距离小于阈值 $T_{dist}=50$ 的被认为是成功匹配。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 本文成功解决了VI-SLAM系统中因特征跟踪中断导致的**累积漂移**问题。通过特征重识别，系统能够“记住”并“重新发现”之前观测到的特征，从而建立跨越长时间间隔的强视觉约束。
    - 提出的**STS子全局地图**和**位姿引导匹配**是两个非常实用的工程创新，它们在**计算效率**和**匹配鲁棒性**之间取得了很好的平衡，使得在每一帧进行特征重识别成为可能。
    - 实验证明，该方法在**不依赖回环检测**的情况下，达到了甚至超越了带有回环检测的SOTA方法（如ORB-SLAM3），这证明了其作为漂移抑制手段的强大能力。

- **
