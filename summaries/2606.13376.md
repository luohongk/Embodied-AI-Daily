## 0. 概览（Concise Summary）

- **主题（Topic）**：视频世界模型（Video World Model），单张图像生成可交互的3D场景。
- **问题（Problem）**：从一张窄视场（NFOV）图像生成一个可实时交互漫游的、完整的、几何一致的、高质量的视频世界。现有方法要么是显式3D方法（几何稳定但质量有限），要么是隐式视频模型（视觉质量高但几何不稳定），难以兼顾。
- **方法（Method）**：提出一个三阶段流水线（MoVerse）：1）**全景生成**：将NFOV图像扩展为重力对齐的360°全景图；2）**高斯场景生成**：将全景图提升为持久的3D高斯溅射（3DGS）脚手架；3）**高斯条件视频渲染**：将脚手架渲染结果通过一个因果自回归视频模型转化为高质量视频。
- **创新（Innovation）**：
    1.  **几何感知的全景生成**：通过可微自动调平（Differentiable Auto-Leveling）和拓扑感知扩散（Topology-Aware Diffusion）生成重力对齐、水平闭合的全景图。
    2.  **全景高斯脚手架**：提出一种专门针对ERP全景图的深度引导残差预测方法，将全景图直接转化为可渲染的3DGS场景。
    3.  **因果流式视频渲染**：将双向扩散教师模型蒸馏为因果自回归学生模型，结合显式3DGS脚手架的长程记忆，实现低延迟的流式视频生成。
- **意义（Significance）**：首次在单个NVIDIA RTX 4090 GPU上实现了从单张图像到实时（8 FPS）交互漫游视频世界的完整流程。它巧妙地结合了显式3D表示的几何可控性和隐式视频模型的高视觉质量，为单图像世界创建和交互式视频输出提供了一条实用路径。
- **一句话总结**：MoVerse通过“全景生成 -> 3D高斯脚手架构建 -> 高斯条件因果视频渲染”的三阶段解耦设计，在单个GPU上实现了从单张窄视场图像到实时、可交互、高质量视频世界的漫游。

## 1. 研究动机（Motivation）

- **核心问题**：从一张窄视场（NFOV）图像生成一个可交互漫游的完整世界是一个高度欠约束的问题。输入只观察到环境的一小部分，而交互式应用（如VR、数字孪生）需要用户在一个完整、几何一致的环境中自由移动。
- **现有方法的不足**：
    - **显式3D方法**（如点云、网格、3DGS）：能构建持久的几何结构，提供精确的相机控制和高速渲染。但直接从单张NFOV图像生成时，大部分场景需要从弱证据中推断，导致生成的资产存在空洞、漂浮物、深度错误和视觉质量有限的问题。
    - **隐式视频/世界模型**：能生成视觉上令人印象深刻的视频，但其长程几何稳定性依赖于模型的隐式记忆。当用户长距离移动或回到之前区域时，场景可能发生漂移、身份变化或出现边界不连续。
    - **混合系统**：使用显式几何作为生成式渲染的条件。但稀疏或粗糙的几何锚点提供的空间引导有限，而强大的生成式渲染器又往往计算成本过高，不适合实时交互。
- **挑战**：如何设计一个系统，既能像显式3D方法一样提供持久、可控的几何锚点，又能像隐式视频模型一样提供高保真、时间一致的视觉输出，并且满足实时交互的延迟要求。

## 2. 创新点（Innovation）

MoVerse的核心创新在于其**三阶段解耦设计**和每个阶段的关键技术，将世界构建与观测渲染分离：

1.  **几何感知的全景生成（Stage I）**：
    - **创新点**：在3D推理之前，先完成缺失视场的补全。这避免了3D模块需要从零开始“幻觉”大部分世界。
    - **关键技术**：
        - **可微自动调平**：通过预测密集对应场并经过刚性变换瓶颈（可微soft-argmin求解器）来估计重力对齐的旋转，将任意姿态的输入图像规范化到重力对齐的规范视角空间，解决了输入图像的俯仰和翻滚问题。
        - **拓扑感知生成**：使用循环潜编码（Circular Latent Encoding）和移位等变扩散（Shift-Equivariant Diffusion），确保生成的ERP全景图在水平方向上是闭合的（S1拓扑），避免了标准扩散模型在边界处产生的人为接缝。

2.  **全景高斯脚手架构建（Stage II）**：
    - **创新点**：将全景图直接转化为一个可直接渲染的、持久的3D高斯场景，作为下游视频渲染的几何和外观锚点。
    - **关键技术**：
        - **全景高斯初始化**：根据全景图的深度信息，在球面上初始化高斯原语。特别地，引入了**纬度感知的尺度校正**（Latitude-Aware Scale Correction），使高斯尺度与ERP像素在球面上代表的面积（与cosϕ成正比）相匹配，避免了高纬度地区的退化。
        - **深度引导的残差预测**：在角度-逆深度空间（Angular–Inverse-Depth Space）中预测高斯属性的残差（如位置偏移、尺度、颜色等）。这种残差公式化确保了学习到的修正与ERP采样网格一致，并保持了水平闭合。

3.  **高斯条件流式视频渲染（Stage III）**：
    - **创新点**：将显式3DGS脚手架的渲染结果作为条件，通过一个因果自回归视频模型进行增强，实现了低延迟的流式渲染。显式脚手架负责长程空间记忆，视频模型负责局部高质量观测渲染。
    - **关键技术**：
        - **双向教师模型**：基于Wan2.1-T2V，通过共享旋转位置编码（Shared-RoPE）将高斯渲染帧作为密集对齐的条件，学习高质量的视频渲染。
        - **因果学生模型蒸馏**：将双向教师模型蒸馏为一个因果自回归学生模型。该模型使用局部键值缓存（KV Cache）和自强制（Self-Forcing）与分布匹配（DMD）训练，实现了有界延迟的流式生成，避免了双向模型需要完整视频窗口的缺点。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

MoVerse的整体架构是一个三阶段流水线，如图1所示：

1.  **Stage I: 全景生成 (I → P)**：输入一张NFOV图像 `I`，通过一个几何感知的扩散模型 `G_θ` 生成一个重力对齐的360° ERP全景图 `P`。这一步在3D推理前补全了缺失的视场。
2.  **Stage II: 高斯场景生成 (P → G)**：将全景图 `P` 输入到一个前馈的全景高斯生成器中，生成一个持久的3D高斯场景 `G = {(μ_k, Σ_k, α_k, c_k)}`。这个脚手架可以直接通过3DGS光栅化渲染，生成一个密集的、相机可控的RGB条件视频 `V̂_{1:T}`。
3.  **Stage III: 高斯条件视频渲染 (V̂_{1:T} → V_{1:T})**：在交互时，根据用户指定的相机轨迹渲染脚手架 `G`，得到条件视频 `V̂_{1:T}`。然后，一个高斯条件视频渲染器（因果自回归模型）将其转化为高保真的最终输出视频 `V_{1:T}`。

**离线 vs. 在线拆分**：Stage I和II是离线执行的，一次构建即可重复使用。Stage III是在线执行的，负责实时渲染和增强。

### 3.2 关键算法与数学推导

**Stage I: 全景生成**

- **可微自动调平**：预测密集对应场 `P_dense`，然后通过可微soft-argmin求解器找到最佳刚性球面变换 `R̂`，用于将输入对齐到规范视角。
- **拓扑感知扩散**：
    - 掩码潜扩散目标：
      $$L_{LDM} = \mathbb{E}_{z_0, \epsilon, t} \left[ \| \epsilon - \epsilon_\theta(z_t \oplus M \oplus z_{ref}, t, \tau(y)) \|_2^2 \right]$$
      其中 `z_t` 是噪声潜变量，`M` 是已知区域的掩码，`z_ref` 是已知区域的潜变量。
    - 移位一致性损失（Shift-Consistency Loss）：
      $$L_{shift} = \mathbb{E}_{z_0, \epsilon, t, \delta} \left[ \| \text{Roll}_\delta(\hat{\epsilon}_{base}) - \hat{\epsilon}_{shifted} \|_2^2 \right]$$
      其中 `Roll_δ` 是水平循环移位，`ε̂_base` 和 `ε̂_shifted` 分别是原始输入和移位后输入的网络预测。这个损失强制模型对水平移位具有等变性，从而保证全景图的水平闭合性。

**Stage II: 高斯场景生成**

- **全景高斯初始化**：
    - 像素 `(u, v)` 对应的球面坐标：
      $$\theta = \left( \frac{u}{W} - \frac{1}{2} \right) 2\pi, \quad \phi = \left( \frac{1}{2} - \frac{v}{H} \right) \pi$$
    - 初始高斯中心：`μ_k = D_k d_k`，其中 `D_k` 是深度，`d_k` 是单位方向向量。
    - 纬度感知的初始尺度：`s_k ∝ D_k \cos \phi_k`，解决了ERP投影中高纬度区域像素面积缩小的问题。

- **深度引导的残差预测**：
    - 在角度-逆深度空间中预测残差 `(Δθ_k, Δϕ_k, Δz_k, ...)`。
    - 更新后的中心：
      $$\theta'_k = \theta_k + \lambda_{xy} \Delta\theta_k$$
      $$\phi'_k = \text{clip}(\phi_k + \lambda_{xy} \Delta\phi_k, -\pi/2 + \epsilon, \pi/2 - \epsilon)$$
      $$D'_k = \frac{1}{\text{softplus}(\rho_k + \lambda_z \Delta z_k) + \epsilon}$$
      其中 `ρ_k` 是基础逆深度的无约束参数。这种在角度-逆深度空间中的残差预测，使得高斯位置的更新与ERP网格的几何特性保持一致。

**Stage III: 高斯条件视频渲染**

- **双向教师模型**：使用流匹配目标进行训练。
  $$L_{teacher} = \mathbb{E}_{z, c, \tau, \sigma_t, \epsilon} \left[ \| v_\theta(\tilde{z}, \tau, t) - (\epsilon - z) \|_2^2 \right]$$
  其中 `z̃ = [z_t || c]` 是噪声目标潜变量和条件潜变量的拼接。共享RoPE设计是关键，它让条件帧和目标帧在Transformer的坐标空间中“碰撞”，强制模型将条件视为密集对齐的渲染指导。

- **因果学生模型**：通过自强制（Self-Forcing）和分布匹配（DMD，具体采用RAVEN）从教师模型蒸馏而来。学生模型一次生成一个潜变量块，并依赖局部KV缓存进行流式处理。

### 3.3 模型、训练与数据集

- **Stage I 全景生成**：
    - **模型**：条件潜扩散模型，基于DiT架构。
    - **数据集**：**Horizon360**，一个精心策划的重力对齐的全景图数据集。从这些全景图中采样不同姿态的透视视图作为输入，对应的全景图作为目标。
- **Stage II 高斯场景生成**：
    - **模型**：前馈全景高斯生成器，包含一个预训练的视觉骨干网络、一个DPT风格的特征头和一个轻量级的残差预测头。
    - **数据集**：**HM3D** 数据集。训练时，输入ERP全景图和深度图，预测3DGS场景，并通过可微渲染与目标视图进行监督。
- **Stage III 高斯条件视频渲染**：
    - **模型**：教师模型从Wan2.1-T2V-1.3B初始化。学生模型是蒸馏后的因果自回归DiT。
    - **数据集**：精心设计的三元组 `(V_gt, V̂, τ)` 训练数据混合，遵循“轨迹对齐几何代理 -> 显式高斯渲染 -> 全景条件高斯渲染”的课程学习顺序。数据来源包括真实视频、UE渲染全景图、Matterport3D等。

### 3.5 技术细节

- **Stage I**：使用可微自动调平模块，通过刚性变换瓶颈避免了不稳定的非刚性扭曲。使用循环填充和移位等变损失来保证全景图的水平闭合。
- **Stage II**：高斯初始化考虑了纬度因素，避免了高纬度退化。残差预测在角度-逆深度空间中进行，与ERP几何一致。训练目标包括目标视图重建损失、源视图重建损失、深度损失和位置残差正则化。
- **Stage III**：教师模型使用共享RoPE将条件帧和目标帧对齐。学生模型使用MemRoPE风格的KV缓存（一个sink帧、一个长程EMA记忆token、一个3帧滑动窗口），结合两步去噪调度和TAEHV解码器，实现了低延迟流式渲染。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了从单张NFOV图像生成实时、可交互、高质量视频世界这一极具挑战性的问题。
    - 提出了一种新颖的“世界构建与观测渲染分离”的设计范式，巧妙地结合了显式3D和隐式视频模型的优点。
    - 在单个消费级GPU上实现了8 FPS的实时漫游，展示了该方法的实用性和高效性。
- **对未来研究/应用的影响**：
    - 为VR/AR内容创建、数字孪生、游戏开发和具身智能体模拟等领域提供了新的可能性。
    - 其模块化的三阶段设计允许未来对每个阶段进行独立改进，例如更好的全景生成模型、更快的3DGS预测器或更高效的视频蒸馏技术。
- **局限与开放问题**：
    - **级联误差**：最终世界质量受限于Stage I的全景生成质量。如果全景图有语义错误，后续阶段会将其放大。
    - **近似几何**：3DGS脚手架仍是近似表示，在深度模糊、薄结构、反射表面和ERP极点附近区域可能产生伪影。
    - **短期纹理漂移**：因果视频渲染器虽然由脚手架提供长程一致性，但在条件渲染严重退化时，仍可能出现短期纹理漂移或过平滑。
    - **复杂的训练流程**：系统依赖于分阶段训练和多种数据源，简化监督信号是未来部署的重要方向。
    - **未来方向**：加强阶段间的反馈（如用全景生成的不确定性指导高斯密度分配），以及将静态场景扩展到动态对象和可编辑世界。

## 5. 澄清与简化（Clarifications and Simplifications）

- **全景图（Panorama）**：可以想象成一个站在场景中心点，向四面八方看时看到的完整球形图像。ERP（等距柱状投影）是将这个球面展开成一个矩形图像，就像世界地图一样。它的特点是左右边界在物理上是相连的（S1拓扑）。
- **3D高斯溅射（3DGS）**：可以理解为用成千上万个微小的、扁平的、半透明的“小椭圆球”（高斯体）来拼凑出一个3D场景。每个小椭圆球都有位置、大小、形状、颜色和透明度。渲染时，就像用喷枪把这些小椭圆球喷到屏幕上，速度快且效果好。
- **为什么需要“重力对齐”**：用户随手拍的照片可能是歪的。如果不先把它“扶正”（让地平线水平，墙壁竖直），后续生成的全景图也会是歪的，导致3D场景结构混乱。可微自动调平就像一个智能的“自动扶正”工具。
- **为什么需要“拓扑感知”**：标准图像处理模型认为图片的左右边界是断开的。但对于全景图，左边和右边是连在一起的。如果不特殊处理，模型会在左右边界处生成一条明显的“接缝”。循环潜编码和移位等变扩散就是为了让模型“理解”全景图是首尾相连的，从而消除接缝。
- **“脚手架”的比喻**：Stage II生成的3DGS场景就像一个建筑的**脚手架**。它提供了整个场景的骨架和基本结构（几何和粗略外观），是稳定和可控的。Stage III的视频渲染器则像是给这个脚手架“贴瓷砖和刷墙”，让它看起来更漂亮、更真实。脚手架保证了房子不会塌（几何稳定），而贴砖保证了房子好看（视觉质量高）。
- **“教师-学生”蒸馏**：教师模型（双向扩散）像一个经验丰富的画家，他可以对着整幅画反复修改，画得非常好，但速度很慢。
