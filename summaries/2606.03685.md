## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）在经典规划任务中的世界模型恢复能力研究。
- **问题（Problem）**：经过监督微调（SFT）后，LLM 虽然能生成有效的规划，但它们是否真正学习并内化了规划问题背后的世界模型（即环境动态、状态谓词和动作有效性）？这是一个关于模型内部知识与外部行为一致性的关键问题。
- **方法（Method）**：提出了一套可解释性分析框架，通过两种互补的指标来评估世界模型恢复：
    1.  **内部表征分析**：在 LLM 的隐藏状态上训练线性探针（Linear Probes），预测状态谓词的真值和动作的有效性。
    2.  **生成能力分析**：利用模型生成的 token 概率，对动作的有效性进行分类。
    同时，论文比较了三种不同的微调数据分布（最优规划、随机游走、增强随机游走）和一种链式思考（State-CoT）策略对世界模型恢复的影响。
- **创新（Innovation）**：
    1.  首次将针对世界模型恢复的可解释性研究系统性地应用于**经典规划**这一复杂任务中的**大规模预训练 LLM**（gemma2-9b-instruct），而非小规模从零训练的 Transformer。
    2.  揭示了**内部表征质量与生成能力之间的不一致性**：模型内部可能线性编码了动作有效性，但其输出概率却无法有效区分有效与无效动作。
    3.  发现**训练数据的分布**（特别是状态空间覆盖率）对世界模型恢复至关重要，随机游走数据优于最优规划数据。
- **意义（Significance）**：该工作为理解 LLM 如何通过微调获得规划能力提供了深刻的见解，挑战了“模型能规划就意味着它理解了世界”的简单假设。它强调了评估模型能力时，需要超越任务性能，深入分析其内部知识表征。这对于构建更可靠、更可解释的 LLM 规划器具有重要指导意义。
- **一句话总结**：本文通过可解释性分析发现，经过监督微调的 LLM 规划器虽然能完美执行规划任务，但其对世界模型的恢复是**部分且不均衡的**，且内部表征与生成能力存在显著差异，而训练数据的**状态空间覆盖率**是影响恢复质量的关键因素。

## 1. 研究动机（Motivation）

- **核心问题**：大型语言模型（LLM）通过监督微调（SFT）学习生成规划后，它们是否真的“理解”了规划问题？具体来说，它们是否恢复了一个内在的**世界模型**，即能够表征状态、理解动作的前提条件和效果？
- **重要性**：如果 LLM 只是学习到了表面模式或捷径（shortcut solutions），而不是真正的世界模型，那么它们在面对分布外（OOD）或更复杂的问题时，其规划能力将非常脆弱，容易产生幻觉和错误。理解 LLM 如何表征知识，是构建可靠、可泛化的 AI 系统的关键。
- **挑战**：
    1.  **任务复杂性**：经典规划问题（如 Blocksworld, Logistics）比棋盘游戏（如国际象棋）更复杂，状态空间更大，动作序列更长，使得分析内部表征更具挑战性。
    2.  **评估困难**：仅仅通过规划成功率（任务性能）来评估是不够的，因为模型可能通过“作弊”或学习捷径来达到目标。需要更精细的探针来窥探其内部知识。
    3.  **现有研究局限**：先前关于世界模型恢复的研究多集中在从零训练的小型 Transformer 模型上（如 OthelloGPT, ChessGPT），其结论是否适用于经过大规模预训练和微调的 LLM 尚不明确。同时，有研究指出 LLM 在规划时可能并不依赖世界模型。

## 2. 创新点（Innovation）

1.  **系统性的可解释性分析框架**：提出了一个结合**内部表征探针**和**生成能力评估**的双视角框架，用于全面评估 LLM 规划器的世界模型恢复程度。这比单一维度的评估更全面、更深刻。
2.  **揭示“知道但说不出”的现象**：关键发现之一是，模型内部可能线性编码了动作有效性（通过探针可以高精度预测），但其输出概率却无法有效区分有效和无效动作。这揭示了**内部表征与生成行为之间的脱节**，是本文最重要的创新洞察。
3.  **数据分布对世界模型恢复的影响**：系统性地比较了三种不同数据生成策略（最优规划、随机游走、增强随机游走）对世界模型恢复的影响。发现**随机游走数据**，因其更好的状态空间覆盖率，能显著提升模型基于生成概率的动作有效性分类能力，而**最优规划数据**在此指标上表现最差。
4.  **将研究扩展到大规模 LLM 和经典规划领域**：将世界模型恢复的研究从简单的棋盘游戏和从零训练的 Transformer，推进到更复杂的经典规划领域和经过预训练的大规模 LLM（gemma2-9b-instruct），使研究结论更具现实意义。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的整体架构是一个“微调-评估”流程：

1.  **微调（SFT）**：使用 `gemma2-9b-instruct` 作为基础模型，通过 LoRA（Low-Rank Adaptation）进行参数高效微调。微调的目标是让模型学会端到端地生成规划序列（从初始状态到目标状态的动作序列）。
2.  **评估（Interpretability）**：在微调后的模型上，对**成功生成的规划**（即所有动作都有效的规划）进行可解释性分析。分析分为两步：
    -   **数据收集**：对于每个测试规划，随机选择一个中间步骤 `j`，然后枚举该状态下所有可能的后续动作（包括有效和无效的），构建部分规划 `(goal, initial_state, actions_0_to_j, candidate_action)`。
    -   **指标计算**：
        -   **内部表征**：从模型处理每个候选动作的 token 位置提取隐藏状态，训练线性探针。
        -   **生成能力**：计算模型为每个候选动作分配的 token 对数概率的平均值。

### 3.2 关键算法与数学推导

论文的核心在于其评估指标，而非提出新的算法。关键公式如下：

1.  **动作有效性分类（生成指标）**：
    给定一个部分规划 $p_j$ 和一个候选动作 $a_{j+1}$，该动作的得分是其所有 token 对数概率的平均值：
    $$ \text{Score}(a_{j+1} | p_j) = \frac{1}{|a_{j+1}|} \sum_{t \in \text{tokens}(a_{j+1})} \log P(t | p_j, \text{context}) $$
    其中 $|a_{j+1}|$ 是动作的 token 数量，$\log P(t | ...)$ 是模型预测 token $t$ 的对数概率。然后，通过比较所有有效动作和无效动作的得分来评估模型区分能力。最终的指标是“所有有效动作的得分都高于所有无效动作的得分”的测试规划比例。

2.  **线性探针（内部表征指标）**：
    训练一个逻辑回归模型 $f_l: \mathbb{R}^d \rightarrow [0, 1]$，其中 $d$ 是第 $l$ 层隐藏状态的维度。输入是平均后的隐藏状态向量 $\mathbf{h}_l$，输出是预测标签（动作是否有效，或某个状态谓词是否为真）的概率。
    $$ \hat{y} = \sigma(\mathbf{w}_l^T \mathbf{h}_l) $$
    其中 $\sigma$ 是 sigmoid 函数，$\mathbf{w}_l$ 是第 $l$ 层的可学习权重。探针的性能通过测试集上的 F1 分数来评估，以应对类别不平衡问题（无效动作远多于有效动作）。

### 3.3 模型、训练与数据集

- **基础模型**：`gemma2-9b-instruct`。选择原因：其在代码和数学方面的预训练被认为对规划任务至关重要。
- **微调方法**：LoRA（Low-Rank Adaptation），一种参数高效的微调方法。训练 4 个 epoch，学习率 3e-4。
- **训练数据**：三种不同分布的数据集，每个数据集都包含 `(goal, initial_state, action_1, ..., action_N)` 的文本序列。
    1.  **Optimal Plans**：使用 Fast Downward 规划器生成的最优或近优规划。
    2.  **Random Walk Plans**：从随机初始状态开始，均匀随机采样有效动作，直到达到最大步数。最终状态成为新的目标状态。
    3.  **Random Walk++ Plans**：在随机游走的基础上，通过禁止回到已访问过的状态来消除状态循环，从而提高状态空间覆盖率。
- **数据增强**：部分模型使用 **State Chain-of-Thought (State-CoT)** 进行训练，即训练数据格式变为 `(goal, state_0, action_1, state_1, action_2, ..., state_N)`，在动作之间插入中间状态。
- **规划环境**：
    -   **Blocksworld**：3-5 个方块。动作：`pick-up`, `put-down`, `stack`, `unstack`。
    -   **Logistics**：3-4 个包裹，1 架飞机，2 个城市，2 个地点。动作：`fly-airplane`, `load-airplane`, `unload-airplane`。

### 3.4 实验设置与结果

- **规划性能（Table 1）**：所有非 State-CoT 的 Plansformer 变体在分布内测试集上都达到了接近 100% 的目标达成率和有效规划率。State-CoT 模型性能略有下降。
- **内部表征：动作有效性探针（Figure 4）**：
    -   所有模型在中间层和上层都达到了接近 1.0 的 F1 分数，表明模型内部**线性编码了动作有效性**。
    -   Random Walk++ 数据训练的模型在不同随机种子间表现最稳定。
- **内部表征：状态谓词探针（Figure 5）**：
    -   模型对某些谓词（如 `arm-empty`）的编码非常完美，但对另一些谓词（如 `on ?obj1 ?obj2`）的编码较差。
    -   趋势是**参数更少、更简单的谓词**更容易被线性编码。这表明世界模型的恢复是**部分且不均衡的**。
- **生成能力：动作有效性分类（Table 2, Figure 6）**：
    -   **Random Walk** 数据训练的模型表现最好，平均约 81% 的测试规划能正确区分有效和无效动作。
    -   **Random Walk++** 数据训练的模型表现次之（~29%）。
    -   **Optimal** 数据训练的模型表现最差（~9%），几乎无法区分。
    -   这揭示了**内部表征与生成能力的不一致性**：Optimal 模型内部能编码动作有效性（探针 F1 高），但无法通过输出概率表现出来。
- **分布外（OOD）泛化（Table 3, Figure 7, 8）**：
    -   在 Blocksworld 6-10 个方块的 OOD 测试中，Random Walk 模型的目标达成率骤降为 0%，而 Random Walk++ 和 Optimal 模型仍有部分泛化能力。
    -   **生成能力**：Random Walk 模型在 OOD 数据上完全丧失了通过概率区分动作的能力（图 7a 左移），而 Random Walk++ 模型保持了较好的鲁棒性（图 7b 无偏移）。
    -   **内部表征**：所有模型的线性探针在 OOD 数据上依然保持很高的 F1 分数（图 8），再次印证了内部表征与生成能力的分离。

### 3.5 技术细节

- **激活提取**：从模型处理候选动作的 token 位置（包括 `(` 和 `)`）提取隐藏状态，并在 token 维度上取平均。
- **探针训练**：使用逻辑回归，不拟合截距项，使用自动平衡（auto-balancing）来处理类别不平衡，最大迭代次数 10K。
- **统计显著性**：使用 Mann-Whitney U 检验来比较不同数据分布下动作分类性能的差异。
- **OOD 数据**：来自 PlanBench 基准测试，包含 6-10 个方块的 Blocksworld 问题。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **澄清了关键问题**：回答了“SFT 是否能让 LLM 恢复世界模型？”这个问题。答案是：**部分恢复**。模型能学习到一些线性表征，但恢复是不完整的，且与生成能力脱节。
    2.  **提供了评估方法论**：提供了一个结合内部探针和生成指标的评估框架，可以推广到其他评估 LLM 知识表征的研究中。
    3.  **指导了微调实践**：明确指出**训练数据的分布**（特别是状态空间覆盖率）对世界模型恢复至关重要。随机游走数据优于最优规划数据，这挑战了直觉上“最优数据最好”的假设。
- **对未来研究/应用的影响**：
    -   **模型可解释性**：强调了不能仅凭任务性能来判断模型是否“理解”了任务，必须深入分析其内部表征。
    -   **模型可靠性**：揭示了模型可能“知其然不知其所以然”，这为构建更可靠的规划系统敲响了警钟。未来工作可以探索如何弥合内部表征与生成能力之间的鸿沟。
    -   **数据生成策略**：为如何设计微调数据以促进世界模型学习提供了新思路，即应优先考虑数据的多样性和状态空间覆盖率，而非仅仅追求最优解。
- **局限与开放问题**：
    -   **线性探针的局限性**：线性探针只能检测线性可分的表征，模型可能以更复杂的方式编码世界模型。
    -   **因果性**：论文并未建立内部表征与模型行为之间的因果关系。高探针分数是否意味着模型在规划时*使用*了这些表征？这需要因果干预实验来验证。
    -   **OOD 泛化**：虽然 Random Walk++ 在 OOD 上表现更好，但所有模型在更大规模的问题上性能都急剧下降，如何实现真正的、基于世界模型的泛化仍是开放问题。
    -   **State-CoT 的副作用**：State-CoT 虽然理论上能提供更多信息，但在实验中导致了训练不稳定和性能下降，其潜力有待进一步挖掘。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“世界模型”？**
    -   **通俗解释**：想象你玩一个“推箱子”游戏。世界模型就是你大脑中对游戏规则的理解：箱子可以推，但不能拉；人不能穿过箱子；一个箱子只能被推到空地上。有了这个模型，你才能规划出正确的推箱子步骤。在本文中，LLM 的“世界模型”就是对规划环境规则（如 Blocksworld 中“拿起方块的前提是手是空的”）的内部表征。
- **什么是“线性探针”？**
    -   **类比**：想象 LLM 的每一层隐藏状态是一个巨大的、高维的“知识空间”。我们想知道这个空间里是否有一条清晰的“分界线”可以区分“有效动作”和“无效动作”。线性探针就像是一个训练有素的“分类器”，它尝试在这个高维空间中画一条直线（一个超平面）。如果它能用这条直线很好地将两类动作分开，就说明 LLM 的内部表征是**线性可分的**，即它以一种简单、清晰的方式组织了这些知识。
- **为什么“内部表征好”但“生成能力差”？**
    -   **类比**：一个学生可能完全理解了一道数学题的解题原理（内部表征好），但在考试时因为紧张或计算错误而写错了答案（生成能力差）。或者，模型内部可能有一个“知识库”知道哪些动作是有效的，但在生成下一个 token 时，它的“决策机制”可能被其他因素（如训练数据中某些动作出现的频率）所干扰，导致它没有选择那个它“知道”是有效的动作。这揭示了模型内部的知识表征和最终的输出决策是两个可能不完全一致的环节。
- **为什么“随机游走”数据比“最优规划”数据更好？**
    -   **类比**：如果你想教会一个 AI 下围棋，只给它看世界冠军的对局（最优规划），它可能只学会了几种“胜利的走法”，但不知道其他走法为什么是错的。如果你给它看大量业余棋手甚至随机乱下的棋局（随机游走），它就能看到各种合法和非法的走法，从而更全面地理解“落子规则”这个底层世界模型。随机游走数据覆盖了更广阔的状态空间，迫使模型学习“什么是有效的”这个通用规则，而不是仅仅记忆“什么是最优的”这个特定模式。
