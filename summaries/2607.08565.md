# SMetric: Rethink LLM Scheduling for Serving Agents with Balanced Session-centric Scheduling

- arXiv: https://arxiv.org/abs/2607.08565v1
- arXiv ID: 2607.08565

---

🔍 **核心问题**：现有LLM调度器在服务智能体（agent）时，因过度优先路由请求到缓存KV的实例，导致负载不均，限制了集群每秒令牌数（TPS），无法适应智能体工作负载中高KV复用率（>80%）和以TPS为核心目标的特点。

🛠 **方法**：提出SMETRIC，采用平衡会话中心调度——将每个会话的第一个请求纯为负载均衡路由，后续请求则按缓存感知方式调度，利用会话内局部性，在保持全局KV存储低需求的同时，平衡集群负载并最大化本地KV复用。

✨ **主要贡献**：在预填-解码共置场景下提升集群TPS 10-16%，在分离架构下提升预填TPS 2-34%，同时改善每令牌延迟；通过仅从用户输入高效提取会话轮次信息作为调度指标，保持调度器无状态且简洁。
