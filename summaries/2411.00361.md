## 0. 概览（Concise Summary）

- **主题（Topic）**：分层强化学习（Hierarchical Reinforcement Learning, HRL），特别是目标条件的分层强化学习（Goal-Conditioned HRL）。
- **问题（Problem）**：HRL 面临两个核心挑战：(1) **非平稳性（Non-stationarity）**：由于低层策略在训练中不断变化，导致高层策略的学习环境（奖励函数和状态转移）不稳定；(2) **不可行子目标生成（Infeasible Subgoal Generation）**：高层策略可能生成低层策略当前无法完成的子目标。
- **方法（Method）**：提出一个名为 **DIPPER** 的框架。它将 HRL 形式化为一个**双层优化（Bi-level Optimization）**问题，并利用**直接偏好优化（Direct Preference Optimization, DPO）**来训练高层策略。同时，通过引入低层价值函数正则化（Lower-level Value Function Regularization）来确保子目标的可行性。
- **创新（Innovation）**：(1) 首次将 HRL 严格建模为双层优化问题，为分析其内部依赖提供了数学基础。(2) 创新性地将 DPO 应用于 HRL 的高层策略训练，利用**静态偏好数据集**替代依赖低层策略的动态环境奖励，从而从根本上缓解非平稳性。(3) 自然地引入低层价值函数正则化，解决了不可行子目标问题。
- **意义（Significance）**：DIPPER 为 HRL 中的两个长期存在的难题提供了统一且有效的解决方案。在复杂的机器人导航和操作任务中，它相比现有最先进方法实现了高达 **40%** 的性能提升，证明了基于偏好的方法在解决分层学习难题上的巨大潜力。
- **一句话总结**：本文提出 DIPPER，通过将 HRL 建模为双层优化问题并利用直接偏好优化（DPO）训练高层策略，同时结合低层价值函数正则化，有效缓解了分层强化学习中的非平稳性和不可行子目标生成两大挑战。

## 1. 研究动机（Motivation）

分层强化学习（HRL）通过将复杂的长时域任务分解为子任务，为解决这类问题提供了有效框架。然而，HRL 在实践中面临两个根本性挑战，尤其是在稀疏奖励环境下：

1.  **非平稳性（Non-stationarity）**：在标准 HRL 中，高层策略的奖励和状态转移依赖于低层策略的行为。由于低层策略在训练过程中不断更新，高层策略所面对的“环境”也随之改变，这违反了传统强化学习对马尔可夫决策过程（MDP）平稳性的基本假设。这种不稳定性严重阻碍了高层策略的有效学习。
2.  **不可行子目标生成（Infeasible Subgoal Generation）**：高层策略在生成子目标时，可能没有充分考虑低层策略当前的能力。如果生成的子目标过于困难或无法实现，低层策略将无法完成，导致整个任务失败。这本质上是高层与低层策略之间协调失败的问题。

作者认为，这些挑战的根本原因在于缺乏一个能够精确描述高层和低层策略之间相互依赖关系的数学框架。传统的 HRL 方法没有明确地建模这种双向依赖，导致算法设计缺乏原则性指导。

## 2. 创新点（Innovation）

本文的核心创新在于提出了一个名为 **DIPPER** 的框架，其关键创新点如下：

1.  **双层优化公式化（Bi-level Optimization Formulation）**：首次将目标条件 HRL 严格建模为一个双层优化问题。高层策略的优化是上层问题，低层策略的优化是下层问题。这个公式化明确地捕捉了策略间的相互依赖关系，为后续的算法设计提供了坚实的理论基础。

2.  **利用 DPO 缓解非平稳性（Mitigating Non-stationarity via DPO）**：这是本文最大的创新。与现有方法不同，DIPPER 不直接使用依赖于低层策略的环境奖励来训练高层策略。相反，它构建了一个**静态的偏好数据集**，其中轨迹对的偏好标签是基于**最终目标达成情况**（而非低层策略行为）来确定的。然后，它使用**直接偏好优化（DPO）** 来训练高层策略。由于 DPO 的训练信号来自这个静态数据集，高层策略的学习过程完全与低层策略的动态变化解耦，从而从根本上缓解了非平稳性问题。

3.  **低层价值函数正则化（Lower-level Value Function Regularization）**：从双层优化的拉格朗日对偶形式中，自然地推导出一个正则化项。该项通过引入低层策略的价值函数 $V^L(s_t, g_t)$，来惩罚高层策略生成低价值（即难以实现）的子目标。这迫使高层策略生成对低层策略而言“可行”的子目标，直接解决了不可行子目标生成问题。

**与现有方案的不同**：
- 与 HIRO、HAC 等方法相比，DIPPER 不依赖于模拟最优低层行为或重标定经验回放，而是从更根本的数学建模和训练信号来源上解决问题。
- 与 PIPER 等方法相比，DIPPER 使用 DPO 而非 RLHF，避免了训练一个单独的奖励模型和额外的 RL 优化步骤，从而更加稳定和高效。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

DIPPER 的整体架构如图 1（中间）所示，其核心思想是解耦高层和低层策略的学习。

- **低层策略（Lower-level Policy）**：使用传统的强化学习（如 SAC）进行训练。它接收高层策略生成的子目标 $g_t$，并执行原始动作 $a_k$ 来尝试实现该子目标。其奖励 $r^L$ 是稀疏的，仅在达到子目标时给出。
- **高层策略（Higher-level Policy）**：使用 **DPO** 进行训练。它不直接接收环境奖励，而是从一个**静态的偏好数据集**中学习。该数据集包含成对的高层轨迹 $(\tau^1, \tau^2)$，其偏好标签 $y$ 由轨迹的“总得分” $\hat{R}(\tau)$ 决定。$\hat{R}(\tau)$ 是基于轨迹中每个子目标是否帮助接近最终目标 $g^*$ 来计算的，**与低层策略的行为无关**。
- **低层价值函数正则化**：在 DPO 目标中，引入了一个正则化项 $-\lambda (V^L(s_t, g_t) - V^L_m(s_t, g_t))$。其中 $V^L(s_t, g_t)$ 是当前低层策略对子目标 $g_t$ 的价值估计，$V^L_m(s_t, g_t)$ 是近似的最优低层价值函数。这个正则化项鼓励高层策略选择那些低层策略认为价值高（即容易实现）的子目标。

### 3.2 关键算法与数学推导

**1. 双层优化公式化（公式 4 & 5）**

作者首先将 HRL 形式化为一个双层优化问题：
$$
\max_{\pi^H, \pi^L} J(\pi^H, \pi^{L*}(\pi^H)) \quad \text{s.t.} \quad \pi^{L*}(\pi^H) = \arg\max_{\pi^L} V^L(\pi^H)
$$
其中 $J$ 是高层目标，$V^L$ 是低层价值函数。这个约束条件要求低层策略在给定高层策略的子目标下达到最优。

为了使其可解，作者将其转化为一个等价的约束优化问题（公式 5）：
$$
\max_{\pi^H, \pi^L} J(\pi^H, \pi^L) \quad \text{s.t.} \quad V^L(\pi^H) - V^{L*}(\pi^H) \ge 0
$$
其中 $V^{L*}$ 是最优低层价值函数。这个约束意味着低层策略的表现必须接近最优。

通过拉格朗日松弛，得到近似的拉格朗日目标（公式 6）：
$$
\max_{\pi^H, \pi^L} \mathbb{E} \left[ \sum_{t=0}^{T-1} \left( r(s_t, g_t, g^*) + \lambda H(\pi^H) + \lambda (V^L(s_t, g_t) - V^{L*}(s_t, g_t)) \right) \right]
$$
这个目标函数包含了三个部分：环境奖励、高层策略熵、以及低层价值函数正则化项。

**2. DIPPER 目标函数（公式 7）**

为了解决高层奖励 $r$ 的非平稳性，作者利用 DPO 的思想，将上述拉格朗日目标中的“奖励”部分（即 $r + \lambda H(\pi^H) + \lambda (V^L - V^{L*})$）与最优策略 $\pi^{H*}$ 联系起来，并代入 DPO 的偏好损失函数中，得到 DIPPER 的核心目标：
$$
\mathcal{L}_O = -\mathbb{E}_{(\tau^1, \tau^2, y) \sim \mathcal{D}} \left[ \log \sigma \left( \sum_{t=0}^{T-1} \left( \beta \log \frac{\pi^{H*}(g_t^1|s_t^1, g^*)}{\pi^{H*}(g_t^2|s_t^2, g^*)} - \lambda \left( (V^L(s_t^1, g_t^1) - V^{L*}(s_t^1, g_t^1)) - (V^L(s_t^2, g_t^2) - V^{L*}(s_t^2, g_t^2)) \right) \right) \right) \right]
$$
其中 $\sigma$ 是 sigmoid 函数，$\beta$ 是控制 KL 散度约束强度的参数，$\lambda$ 是正则化权重。

**3. DIPPER 梯度分析（公式 8）**

对 $\mathcal{L}_O$ 求关于高层策略 $\pi^H$ 的梯度，得到：
$$
\nabla \mathcal{L}_O = -\beta \mathbb{E}_{(\tau^1, \tau^2, y) \sim \mathcal{D}} \left[ \sum_{t=0}^{T-1} \left( \sigma(\hat{r}(s_t^2, g_t^2) - \hat{r}(s_t^1, g_t^1)) \cdot \left( \nabla \log \pi^H(g_t^1|s_t^1, g^*) - \nabla \log \pi^H(g_t^2|s_t^2, g^*) \right) \right) \right]
$$
其中 $\hat{r}(s_t, g_t, g^*) = \beta \log \pi^H(g_t|s_t, g^*) - \lambda (V^L(s_t, g_t) - V^{L*}(s_t, g_t))$ 是一个**隐式奖励模型**。

这个梯度公式的含义是：
- 它增加被偏好轨迹 $\tau^1$ 中动作（子目标）的对数概率，同时减少不被偏好轨迹 $\tau^2$ 中动作的对数概率。
- 权重 $\sigma(\hat{r}(s_t^2, g_t^2) - \hat{r}(s_t^1, g_t^1))$ 表示隐式奖励模型对当前策略排序的“错误”程度。如果模型错误地认为 $\tau^2$ 更好，这个权重就会很大，从而给予更大的更新幅度。这体现了 DPO 的自适应性。
- 隐式奖励 $\hat{r}$ 中的 $-\lambda (V^L - V^{L*})$ 项直接惩罚了那些低层价值函数低的子目标，从而引导策略生成可行子目标。

### 3.3 模型、训练与数据集

- **模型**：高层策略 $\pi^H$ 和低层策略 $\pi^L$ 都是神经网络（如高斯策略网络）。低层价值函数 $V^L$ 也是一个神经网络。
- **训练**：
    - **低层策略**：使用标准的 off-policy RL 算法（如 SAC）进行训练，目标是最大化实现子目标的累积奖励。
    - **高层策略**：使用 DPO 进行训练，目标是最小化公式 9 中的损失函数。注意，公式 9 中使用 $V^L_m$ 来近似 $V^{L*}$，$V^L_m$ 是通过对 $V^L$ 进行多次梯度更新得到的。
- **数据集**：
    - **低层经验回放**：存储低层策略的交互数据 $(s_{t+k}, g_t, a_k, r^L, s_{t+k+1})$。
    - **高层偏好数据集 $\mathcal{D}$**：这是关键。它由高层轨迹对 $(\tau^1, \tau^2)$ 及其偏好标签 $y$ 组成。标签不是由人类给出的，而是通过一个自动化的“原始策略在环（Primitive-in-the-loop, PiL）”方法生成。具体来说，对于一条高层轨迹 $\tau = \{(s_{t*k}, g_t)\}_{t=0}^{n-1}$，其得分 $\hat{R}(\tau) = \sum_{t=0}^{n-1} \hat{r}(s_{t*k}, g^*, g_t)$，其中 $\hat{r}(s_{t*k}, g^*, g_t) = \mathbb{1}_{\{ \|s_{t*(k+1)} - g^*\|_2 < \epsilon \}}$。这个得分只取决于最终目标 $g^*$ 和实际到达的状态 $s_{t*(k+1)}$，**与低层策略的行为无关**，因此数据集是静态的。

### 3.4 实验设置与结果

- **环境**：四个稀疏奖励的机器人任务：(i) Maze Navigation, (ii) Pick and Place, (iii) Push, (iv) Franka Kitchen。这些任务具有长时域和稀疏奖励的特点，非常适合测试 HRL 算法。
- **基线**：与多种基线进行比较，包括：
    - **HRL 基线**：HIER（vanilla HRL）、HAC、RAPS、SAGA、PIPER。
    - **非 HRL 基线**：FLAT（单层 SAC）、DPO-FLAT（单层 DPO）、DAC。
    - **消融基线**：DIPPER-No-V（无低层价值函数正则化的 DIPPER）。
- **主要结果（图 2）**：
    - DIPPER 在四个任务上均取得了最优或接近最优的性能。
    - 在更难的 Pick and Place、Push 和 Kitchen 任务中，许多基线（如 HAC、RAPS、SAGA）表现不佳甚至失败，而 DIPPER 则表现出色，实现了高达 **40%** 的性能提升。
    - DIPPER-No-V 的性能显著低于 DIPPER，证明了低层价值函数正则化的重要性。
- **缓解 HRL 问题的验证（图 3）**：
    - **子目标距离度量**：DIPPER 预测的子目标与低层策略实际达到的状态之间的距离最小，表明其生成的子目标是可行的。
    - **低层 Q 函数度量**：DIPPER 预测的子目标对应的低层 Q 值最高，表明低层策略对这些子目标有很高的成功预期。
    - 这两个新提出的度量标准定量地证明了 DIPPER 有效缓解了非平稳性和不可行子目标生成问题。

### 3.5 技术细节

- **近似最优价值函数**：为了计算 $V^{L*}$，作者使用一个近似值 $V^L_m$，该值通过对当前低层价值函数 $V^L$ 进行 $m$ 次梯度更新得到。这是一种常见的近似技术，使算法在理论上可行且在实践中高效。
- **偏好数据收集**：采用自动化的 PiL 方法，避免了昂贵且耗时的人工标注。该方法利用最终目标达成情况作为偏好标签，确保了数据集的静态性。
- **超参数**：$\lambda$ 控制正则化强度，$\beta$ 控制 DPO 中的 KL 约束。作者通过网格搜索进行调优，并发现平衡的值能带来最佳性能。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 本文为 HRL 中的非平稳性和不可行子目标生成这两个核心挑战提供了一个统一、原则性的解决方案。
    - 通过将 HRL 建模为双层优化问题，为理解和分析 HRL 的内部动态提供了新的理论视角。
    - 成功地将 DPO 这一强大的偏好学习方法引入 HRL 领域，并展示了其在缓解非平稳性方面的巨大优势。
    - 提出了两个新的量化指标（子目标距离和低层 Q 函数），为未来 HRL 研究提供了评估非平稳性和子目标可行性的工具。

- **对未来研究/应用的影响**：
    - **算法设计**：DIPPER 的成功表明，基于偏好的方法（如 DPO）是解决 HRL 中非平稳性问题的有效途径，可能会启发更多将偏好学习与分层结构结合的研究。
    - **机器人学**：DIPPER 在复杂机器人操作任务上的出色表现，使其有望应用于更实际的机器人学习场景，如长时域操作、自主导航等。
    - **理论发展**：双层优化框架为 HRL 的理论分析提供了更坚实的基础，可能推动对 HRL 收敛性、稳定性等问题的更深入研究。

- **局限性与开放问题**：
    - **泛化能力**：基于 DPO 的方法在分布外（out-of-distribution）状态和动作上的泛化能力如何？与基于奖励模型的 RLHF 方法相比有何优劣？这需要进一步研究。
    - **高维子目标空间**：将 DIPPER 应用于高维子目标空间（如图像）仍然是一个挑战。
    - **计算成本**：虽然避免了 RLHF 中的 RL 步骤，但 DPO 训练和低层价值函数的多次更新仍可能带来计算开销。
