## 0. 概览（Concise Summary）

- **主题（Topic）**：文本到图像（T2I）生成中的奖励模型（Reward Model）。
- **问题（Problem）**：现有奖励模型（如 HPSv3）在评估由不同能力水平的 T2I 模型生成的图像，以及在强化学习（RL）迭代过程中，其评分能力会退化，无法有效区分高质量和低质量的图像，限制了其在更广泛场景下的应用。
- **方法（Method）**：提出了 HPSv3++ 框架，包含两个核心部分：1) 构建了一个新的、针对前沿高能力 T2I 模型（Qwen-Image）的双维度偏好数据集 HPDv3++；2) 设计了一个两阶段训练框架，第一阶段通过正交梯度下降（OGD）进行持续学习以吸收新知识并防止遗忘，第二阶段通过半监督自适应训练，引入模型能力与 RL 迭代步数的条件信号，并利用标准差引导的无监督机制来提升模型在整个能力-迭代谱系上的评分校准能力。
- **创新（Innovation）**：1) 首次将奖励模型设计为**条件式**的，使其评分能根据生成模型的**能力水平**和**RL迭代阶段**进行动态调整。2) 提出了一个**双维度**（文本跟随与美学质量）的偏好数据集，专门针对前沿高能力模型。3) 设计了一套结合持续学习和半监督自适应训练的两阶段训练框架，有效解决了分布漂移问题。
- **意义（Significance）**：HPSv3++ 显著提升了奖励模型在多样化 T2I 模型和 RL 迭代过程中的评分准确性和鲁棒性，为更有效、更稳定的 RLHF 训练提供了关键支撑，有望推动 T2I 生成质量的整体提升。
- **一句话总结**：本文提出了 HPSv3++，一个通过条件化评分和两阶段训练来扩展奖励模型能力，使其能可靠评估从低到高各种能力水平的 T2I 模型在 RL 各阶段生成图像质量的框架。

## 1. 研究动机（Motivation）

- **关键问题**：现有的奖励模型（如 HPSv3）通常是在**静态**的、由**早期** T2I 模型（如 SDXL）生成的图像数据上训练的。这导致了两个严重的局限性：
    1.  **模型能力分布漂移**：随着 T2I 技术的发展，从早期的扩散模型到前沿模型（如 FLUX.1-dev, Qwen-Image），生成图像的质量和偏好特征发生了巨大变化。在旧数据上训练的 RM 无法准确评估新模型生成的高质量图像，甚至会出现“高分低质”或“低分高质”的错位现象（如图 1 所示）。
    2.  **RL 迭代分布漂移**：在 RL 微调过程中，T2I 模型生成的图像分布会随着迭代步数增加而不断变化（例如，多样性降低，质量提升）。一个静态的 RM 无法适应这种动态变化，在 RL 后期会失去对图像质量的区分能力（如图 2 所示，HPSv3 的评分标准差随迭代下降）。
- **重要性**：奖励模型是 RLHF 的核心，其质量直接决定了 T2I 模型优化的上限。如果 RM 本身不准确或失效，RL 训练就会失去正确的方向，导致“奖励黑客”（reward hacking）或优化停滞。
- **挑战**：如何让一个单一的奖励模型能够同时适应**模型能力**和**RL迭代**这两个维度的巨大变化，实现动态、校准的评分。

## 2. 创新点（Innovation）

1.  **条件式奖励模型框架**：将传统的静态、无条件评分函数 $r(x, y)$ 扩展为条件式评分函数 $r(x, y|c, t)$，其中 $c$ 代表生成模型的**能力水平**，$t$ 代表**RL 迭代步数**。这使得 RM 能够根据输入图像的来源（模型类型）和优化阶段，动态调整其评分标准，从根本上解决了单一评分函数无法覆盖整个能力-迭代谱系的问题。
2.  **双维度偏好数据集 HPDv3++**：构建了一个包含 212K 对偏好数据的全新数据集。其创新点在于：
    - **前沿模型覆盖**：所有图像均由当前最先进的高能力模型 **Qwen-Image** 生成，填补了现有数据集在前沿模型分布上的空白。
    - **双维度标注**：对每对图像分别从**文本跟随（Text-Following）** 和**美学质量（Aesthetic Quality）** 两个维度进行人工标注，提供了更精细、更解耦的监督信号。
3.  **两阶段训练框架**：
    - **第一阶段：持续学习**：采用**数据感知的正交梯度下降（OGD）** 方法，在 HPDv3++ 上训练时，将新数据的梯度投影到旧数据（HPDv3）梯度的正交方向上，从而在吸收新知识的同时，**保留**了 HPSv3 原有的有效偏好知识，避免了灾难性遗忘。
    - **第二阶段：半监督自适应训练**：引入**能力-迭代条件信号**，并使用**标准差引导的无监督机制**。该机制利用来自不同能力模型和 RL 迭代步数的**无标签**数据，通过最大化组内评分标准差（Std）来增强模型对细微质量差异的敏感性，并通过自适应比率约束来鼓励模型在更具挑战性的区域（如高能力模型、后期 RL 迭代）表现出更强的区分能力。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

HPSv3++ 的整体架构是一个两阶段训练流程，最终得到一个条件式奖励模型。

- **第一阶段：持续学习（Continual Learning via OGD）**
    - **目标**：在 HPDv3++ 上微调 HPSv3，使其学会评估前沿模型（Qwen-Image）的生成质量，同时不忘记其在旧数据（HPDv3）上学到的能力。
    - **方法**：使用 OGD。每次更新时，同时计算新数据（HPDv3++）的梯度 $g_{new}$ 和参考数据（HPDv3）的梯度 $g_{ref}$。如果两者冲突（$g_{new} \cdot g_{ref} \le 0$），则将 $g_{new}$ 投影到 $g_{ref}$ 的正交方向上，只保留不冲突的部分进行更新。

- **第二阶段：半监督自适应训练（Semi-supervised Adaptive Training）**
    - **目标**：使 RM 能够感知并适应不同模型能力和 RL 迭代阶段带来的分布变化。
    - **方法**：
        1.  **条件化评分**：将 RM 的输出改为条件式 $r(x, y|c, t)$。其中，模型能力 $c$ 通过一个可训练的**能力编码器（Capability Encoder）** 从图像特征中隐式推断；RL 迭代步数 $t$ 作为显式条件输入。两者通过 **FiLM 条件化** 层注入到奖励头中。
        2.  **半监督学习**：使用有标签的 HPDv3++ 数据和无标签的 rollout 数据（来自不同能力模型和 RL 迭代步数）进行交替批次训练。
        3.  **标准差引导的无监督损失**：对于无标签数据，设计了一个复合损失函数，核心是最大化组内评分标准差（$L_{std}$）和自适应比率约束（$L_{adapt}$），以提升模型在能力-迭代空间中的区分度和校准能力。

### 3.2 关键算法与数学推导

1.  **奖励函数定义**：
    - 第一阶段（无条件）：
        $$r_{\Theta}(x, y) = f_{\phi}(E_{\theta}(x, y))$$
    - 第二阶段（条件式）：
        $$r_{\Theta}(x, y|c, t) = f_{\phi}(E_{\theta}(x, y), c, t)$$

2.  **有监督偏好损失**：
    $$L_{lab} = -\log \sigma \left( r_{\Theta}(x, y_w) - r_{\Theta}(x, y_l) \right)$$
    其中 $y_w$ 和 $y_l$ 分别是偏好和非偏好的图像。

3.  **正交梯度投影（OGD）**：
    当 $g_{new} \cdot g_{ref} \le 0$ 时：
    $$g_{proj} = g_{new} - \frac{g_{new} \cdot g_{ref}}{||g_{ref}||^2} g_{ref}$$

4.  **无标签数据损失函数**：
    $$L_{unlab}^* = \lambda_2 L_{rank} + \lambda_3 L_{std} + \lambda_4 L_{adapt} + L_{reg}$$
    - **标准差最大化损失**：
        $$L_{std} = -\log \left( 1 + \text{Std}(\{r_i\}_{i=1}^K) \right)$$
        其中 $\{r_i\}_{i=1}^K$ 是同一 prompt 下 K 张图像的评分。
    - **自适应比率约束损失**：
        $$L_{adapt} = \max(0, \rho^{(\ell)} - \rho^{(h)} + m)$$
        其中 $\rho = \frac{\text{Std}_{new}(\{r_i\})}{\text{Std}_{base}(\{r_i\})}$ 是当前模型与第一阶段模型评分标准差的比值。$h$ 和 $\ell$ 分别代表高优先级（高能力/后期迭代）和低优先级（低能力/早期迭代）的组，$m$ 是间隔（margin）。该损失鼓励高优先级组获得更大的相对标准差增益。

### 3.3 模型、训练与数据集

- **模型架构**：
    - **骨干网络**：从 HPSv3 的 Qwen2-VL-7B 升级为 **Qwen3-VL-8B**。
    - **奖励头**：一个三层的 RankNet MLP。
    - **能力编码器**：在第二阶段引入，随机初始化，用于从图像特征中隐式编码模型能力。
- **数据集**：
    - **HPDv3++**：核心贡献之一。包含 212K 对偏好数据，全部由 **Qwen-Image** 生成。分为两个维度：95K 文本跟随对和 117K 美学质量对。数据经过严格的人工标注和自动双重验证（VLM Judge + RM Judge）清洗。
    - **无标签 Rollout 数据**：用于第二阶段。包含来自 SD 1.5, SDXL, FLUX.1-dev, Qwen-Image 等不同能力模型的图像，以及来自不同 RL 迭代步数的图像。
- **训练过程**：
    - **第一阶段**：在 HPDv3++ 上训练 1 个 epoch，使用 HPDv3 作为 OGD 的参考数据。
    - **第二阶段**：冻结视觉编码器和语言模型，只训练视觉融合器、奖励头和能力编码器约 2 个 epoch。使用有标签的 HPDv3++ 美学子集和无标签的 rollout 数据。

### 3.4 实验设置与结果

- **偏好预测准确性**（表 2）：
    - 在多个基准测试上，HPSv3++ 均达到最优或次优。
    - 在 HPDv3 上，准确率 86.7%，比 HPSv3 提升 **9.8%**。
    - 在 GenAI-Bench 上，准确率 76.3%，提升 **5.5%**。
    - 在自建的 HPDv3++ 测试集上，美学和文本跟随准确率分别达到 79.1% 和 88.1%。
- **RL 微调效果**（图 7）：
    - 使用 Flow-GRPO 对 SDXL, FLUX.1-dev, Qwen-Image 三个模型进行 RL 微调。
    - HPSv3++ 作为奖励模型，在所有三个模型上均取得了比 HPSv3 更高、更稳定的 GenEval 分数增长，尤其是在 FLUX.1-dev 和 SDXL 上提升显著。
- **用户研究**（表 3）：
    - 在 100 个 prompt 上，HPSv3++ 引导的 RL 训练结果在人类偏好上以 **77.5%** 的胜率击败了 HPSv3 引导的结果。
- **消融研究**（表 4）：
    - 验证了两阶段训练的必要性，移除第二阶段会导致性能大幅下降。
    - 验证了能力编码器需要随机初始化并端到端训练才能达到最佳效果。

### 3.5 技术细节

- **数据构建**：HPDv3++ 的 prompt 经过 Qwen3-VL-32B-Instruct 过滤，确保质量。人工标注后，使用 Qwen3-VL-32B-Instruct（VLM Judge）和 HPSv3（RM Judge）进行双重验证，只有当两者都不同意人工标注时才丢弃样本，这是一种保守的清洗策略。
- **条件注入**：模型能力 $c$ 和 RL 迭代步数 $t$ 通过 FiLM（Feature-wise Linear Modulation）层注入到奖励头中，这是一种高效的特征调制方法。
- **Std 分析**（图 8）：通过固定图像、改变输入的 RL 迭代步数 $t$，验证了条件化机制的有效性。随着 $t$ 的增加，模型输出的评分标准差单调递增，表明模型确实根据迭代步数调整了其评分敏感性。

## 4. 意义与影响（Significance and Impact）

- **解决的问题**：解决了现有奖励模型在评估**多样化能力水平**的 T2I 模型和**动态 RL 迭代过程**时表现不佳的核心问题。通过将 RM 从静态评分器转变为动态、条件化的评分器，显著提升了其泛化能力和鲁棒性。
- **贡献**：
    1.  提出了一个更通用、更强大的奖励模型范式（条件式评分）。
    2.  提供了一个高质量、双维度、面向前沿模型的新数据集 HPDv3++，为社区提供了宝贵的资源。
    3.  设计了一套有效的两阶段训练方法，为处理奖励模型中的分布漂移问题提供了可行的解决方案。
- **影响**：
    - **对研究**：为后续奖励模型的设计提供了新思路，即考虑生成模型本身的属性和优化状态。HPDv3++ 数据集将成为评估和训练新 RM 的重要基准。
    - **对应用**：HPSv3++ 可以直接用于提升现有 T2I 模型的 RLHF 微调效果，有望生成更符合人类偏好的高质量图像。其“条件化”思想也可能启发其他领域的奖励模型设计（如文本到视频、文本到3D）。
- **局限与开放问题**：
    - 论文未明确讨论 HPSv3++ 在极端分布外（OOD）场景下的表现，例如遇到一个全新架构的、能力远超现有模型的生成器时，其能力编码器是否能正确推断其能力？
    - 第二阶段的无监督训练依赖于构建的 rollout 数据，构建这些数据本身需要一定的计算成本。如何更高效地获取或生成这些数据是一个开放问题。
    - 模型的计算开销（基于 8B 参数的 VLM）可能限制了其在一些资源受限场景下的应用。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：一位经验丰富的品酒师**
    - **传统 RM（HPSv3）**：就像一个只品尝过入门级葡萄酒的品酒师。他能很好地区分这些酒的好坏，但当他面对一瓶顶级名庄酒时，他可能会因为不熟悉其复杂风味而给出错误的评价。他无法根据酒的“出身”（模型能力）调整自己的评判标准。
    - **HPSv3++**：就像一个经过特殊训练的品酒大师。他不仅品尝过各种级别的酒，还学会了根据酒的**产地和年份**（模型能力和 RL 迭代）来调整自己的味蕾。当他品尝一瓶顶级酒时，他会用更精细的标准去评判其香气、口感和余味，而不是用评价入门酒的标准。他还能在品酒会的不同阶段（RL 迭代）保持敏锐的味觉，不会因为疲劳而失去分辨力。

- **正交梯度下降（OGD）**：可以想象成在学习新知识时，要确保新知识不会“覆盖”掉旧知识。OGD 就像一个聪明的学生，在学新数学公式时，会先检查这个新公式是否会与之前学过的公式产生矛盾。如果有矛盾，他会找到一个既不违背旧公式又能学习新公式的“折中”方向去学习。这就像在向量空间中，将新向量投影到与旧向量垂直的方向上，这样新向量的变化就不会影响旧向量方向上的知识。

- **标准差引导的无监督机制**：想象一个老师（RM）在批改一群学生（同一 prompt 下的图像）的作业。如果所有学生的作业都差不多（低标准差），老师就很难挑出最好的和最差的。这个机制就是鼓励老师去“放大”学生之间的差异，尤其是在高年级班（高能力模型）或期末考试阶段（后期 RL 迭代），老师需要更严格、更细致地打分，让分数拉开差距（高标准差），这样才能更清晰地反映出学生水平的细微差别。
