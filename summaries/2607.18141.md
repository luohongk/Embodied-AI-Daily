# HyMCache: A KV Cache Framework for Multi-Turn LLM Serving with CXL-Hybrid Memory

- arXiv: https://arxiv.org/abs/2607.18141v1
- arXiv ID: 2607.18141

---

🔍 **核心问题**：多轮对话和长上下文LLM服务中，KV缓存重用可减少冗余计算，但GPU HBM和主机DRAM成本过高，难以扩展到TB级共享上下文容量，导致存储和提供可重用KV状态的存储层成为瓶颈。

🛠 **方法**：提出HyMCache框架，集成CXL混合内存（CXL-HM），结合少量设备内DRAM和大容量SSD后端，利用多轮KV缓存访问的读主导、可预测和仅追加特性，通过请求级前缀预取和机会性写缓冲，将延迟关键读取暂存到设备DRAM中。

✨ **主要贡献**：在真实CXL-HM原型上，HyMCache在相同DRAM预算下，单节点服务性能比本地LMCache提升3.0倍，PD分离服务提升1.45倍；与1TB分布式DRAM的Mooncake相比，性能仅低约30%，但DRAM使用量减少16倍。
