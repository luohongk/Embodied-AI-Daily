## 0. 概览（Concise Summary）

- **主题（Topic）**：计算机辅助设计（CAD）的自动化生成与编辑，特别是基于多模态输入（工程图纸、文本）的交互式、闭环式CAD建模。
- **问题（Problem）**：现有CAD自动化方法多为“开环、一次性”生成，缺乏迭代自校正能力，与人类工程师“生成-验证-优化”的迭代工作流不匹配。同时，现有评估指标存在“幸存者偏差”，仅评估成功生成的模型，忽略了代码执行失败的情况。
- **方法（Method）**：提出了一个名为**IterCAD**的统一多模态智能体框架。它将CAD建模任务形式化为一个多轮交互过程，智能体与一个可执行的CAD沙箱（基于CadQuery）进行交互。通过“看（Look）”和“循环（Loop）”的哲学，利用多视图工程图纸作为空间锚点，结合编译、执行和视觉反馈进行迭代优化。
- **创新（Innovation）**：1）提出了一个闭环、多轮交互的CAD智能体框架；2）设计了两阶段训练策略（渐进式SFT + 几何感知强化学习），并提出了**几何可行前缀掩码（GVPM）** 来解决多轮信用分配问题；3）构建了**IterCAD-Bench**评估套件，并提出了**CD-TR曲线**和**AUC-TR指标**，消除了幸存者偏差。
- **意义（Significance）**：IterCAD显著提升了CAD代码的可执行性和几何精度，更贴近真实工程设计流程，为自动化、交互式CAD建模设立了新的标准，对智能制造和工程设计自动化有重要价值。
- **一句话总结**：本文提出了一种名为IterCAD的闭环、多轮交互式CAD智能体，通过创新的训练策略和评估指标，在代码生成与编辑任务上显著优于现有方法，实现了更鲁棒、更精确的自动化CAD建模。

## 1. 研究动机（Motivation）

- **解决的问题**：当前基于大语言模型（LLM）的CAD代码生成方法（如CAD-Coder, CADrille）主要采用“开环、一次性”生成范式。这种范式与人类工程师专业、迭代的“生成-验证-优化”工作流存在根本性不匹配。
- **为什么重要**：一次性生成的CAD代码容易产生微小的几何不一致或拓扑缺陷，导致运行时错误或视觉伪影。在实际工程中，设计是一个反复修改和优化的过程，缺乏迭代能力的方法无法满足实际需求。
- **挑战在哪里**：
    1.  **反馈信号粗糙**：现有的多轮框架使用的反馈信号（如点云距离、编译器反馈）过于粗粒度。点云距离只能捕捉全局形状差异，无法定位具体参数错误；编译器反馈只能验证语法正确性，无法识别潜在的拓扑逻辑问题。这导致多轮迭代退化为无引导的“试错”。
    2.  **数据集与基准不匹配**：现有基准（如DeepCAD, Text2CAD）过度简化，以基础的“草图-拉伸”模式为主，缺乏倒角、抽壳等高级制造特征，且侧重于静态生成而非迭代编辑。
    3.  **评估指标存在“幸存者偏差”**：当前评估协议仅在成功执行的代码上计算几何指标（如Chamfer Distance），忽略了执行失败的样本（其CD为无穷大）。这无法真实反映智能体在现实工程环境下的错误恢复能力和鲁棒性。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **闭环多轮交互框架**：提出了IterCAD，将CAD生成和编辑统一为一个闭环、多轮交互过程。智能体与一个CAD沙箱交互，利用编译、执行和视觉反馈进行迭代优化，模仿人类设计师的“Look and Loop”工作流。
    2.  **两阶段训练策略**：
        - **第一阶段：渐进式冷启动监督微调（Progressive Cold-start SFT）**：首先在专家轨迹（DS1）上微调，建立基础能力；然后通过策略自身在沙箱中滚动生成数据，并由更强的教师模型修正错误，形成优化数据集（DS2），从而学习迭代调试和局部几何优化。
        - **第二阶段：几何感知强化学习（Geometry-aware RL）**：采用**组序列策略优化（GSPO）** 算法，结合几何奖励（基于Chamfer Distance）、格式奖励和进度奖励，优化智能体的长程优化能力。
    3.  **几何可行前缀掩码（GVPM）**：针对多轮强化学习中的信用分配问题，GVPM通过检测“执行级联失败”和“几何停滞”来确定一个“可行前缀”边界。在此边界之后的token被掩码掉，不参与策略损失计算，从而避免因下游失败而惩罚上游有效的早期动作。
    4.  **无幸存者偏差的评估标准**：提出了**IterCAD-Bench**评估套件，并引入了**Chamfer Distance Tolerance-Recall (CD-TR) 曲线**及其标量指标**AUC-TR**。该指标将执行失败的样本（CD→∞）视为在所有容差阈值下召回率为0，从而统一衡量代码有效性和几何精度，消除了幸存者偏差。

- **与现有方案的不同**：不同于一次性生成或使用粗糙反馈的简单多轮方法，IterCAD利用多视图工程图纸作为精确的空间锚点，结合多维度反馈（编译、执行、视觉）进行有目标的、结构化的迭代修正，而非盲目试错。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

IterCAD的框架如图2所示，核心是一个多轮交互过程：

1.  **任务形式化**：将CAD生成和编辑定义为智能体与CAD沙箱之间的多轮交互。涵盖三种任务：
    - **Drawing-to-Code**：根据多视图工程图纸生成代码。
    - **Text-to-Code**：根据自然语言描述生成代码。
    - **Interactive Editing**：根据增量指令对现有代码进行局部修改。

2.  **交互协议**：在每一轮 `t`，智能体感知历史观察 `O_t`（包括系统提示、参考上下文、历史代码和反馈），并输出一个结构化的动作 `a_t`。动作包含：
    - **推理轨迹（Reasoning Trace）**：用 `<thinking>` 标签包裹，用于诊断多模态反馈并规划下一步。
    - **执行块（Execution Block）**：包含一段CadQuery代码块或一个 `<DONE>` 完成标记。

3.  **反馈机制**：CAD沙箱执行代码后，返回一个多模态反馈元组 `F_t`。除了编译器日志，沙箱还会利用OCCT内核将生成的实体投影为标准正交视图，并计算显式的尺寸标注（线性尺寸、圆弧半径）作为视觉反馈。这为智能体提供了比单纯点云或编译器反馈更精确、更结构化的错误定位信息。

### 3.2 关键算法与数学推导

1.  **渐进式冷启动SFT**：
    目标函数是标准的交叉熵损失：
    $$L_{SFT}(\theta) = -\mathbb{E}_{(x_i, y_i) \sim (D_{S1} \cup D_{S2})} [\log \pi_{\theta}(y_i | x_i)]$$
    其中 $D_{S1}$ 是专家轨迹，$D_{S2}$ 是通过策略自身滚动生成并由教师模型修正的轨迹。

2.  **GSPO（组序列策略优化）**：
    给定一个设计规格 $x$，从当前策略 $\pi_{\theta_{old}}$ 中采样一组 $G$ 个轨迹 $\{y_g\}_{g=1}^G$，并计算其序列级奖励 $\{r_g\}_{g=1}^G$。相对优势通过组内标准化计算：
    $$\hat{A}_g = \frac{r_g - \text{mean}(\{r_{g'}\}_{g'=1}^G)}{\text{std}(\{r_{g'}\}_{g'=1}^G)}$$
    策略更新目标函数为：
    $$J(\theta) = \mathbb{E}_{x \sim D} \left[ \frac{1}{G} \sum_{g=1}^G \min \left( \rho_g(\theta) \hat{A}_g, \text{clip}_g(\theta) \hat{A}_g \right) \right]$$
    其中 $\rho_g(\theta)$ 是长度归一化的重要性比率，$\text{clip}_g(\theta) = \text{clip}(\rho_g(\theta), 1-\epsilon_l, 1+\epsilon_h)$ 使用非对称裁剪边界。

3.  **奖励建模**：
    序列级奖励 $R(y)$ 由三项组成：
    $$R(y) = R_{CD}(y) + \lambda_f R_{fmt}(y) + \lambda_p R_{prog}(y)$$
    - **几何奖励 $R_{CD}$**：基于Chamfer Distance (CD) 的分段线性函数：
      $$R_{CD}(y) = \begin{cases} 1, & CD \leq \delta_{min} \\ \frac{\delta_{max} - CD}{\delta_{max} - \delta_{min}}, & \delta_{min} < CD < \delta_{max} \\ 0, & CD \geq \delta_{max} \text{ or Fail} \end{cases}$$
    - **格式奖励 $R_{fmt}$**：二进制奖励，确保交互协议（推理+代码块+完成标记）被严格遵守。
    - **进度奖励 $R_{prog}$**：稀疏奖励，鼓励多轮优化，要求至少两轮可执行、初始轮有效且CD单调递减至阈值 $\eta$ 以下。

4.  **几何可行前缀掩码（GVPM）**：
    通过监测两个信号确定前缀边界 $f = \min(f_{exec}, f_{stall})$：
    - **执行级联 $f_{exec}$**：标记连续 $K$ 次运行时错误的不可恢复失败循环。
    - **几何停滞 $f_{stall}$**：检测到连续 $K$ 轮有效执行中CD不再严格下降且高于质量门限 $\eta$。
    由此生成一个逐token的损失掩码 $m_u = \mathbb{1}[\tau(u) < f]$，其中 $\tau(u)$ 是token $u$ 所属的轮次。被掩码的token不参与梯度计算。此外，对于被掩码的轨迹，其优势值被裁剪为 $\hat{A}_g \leftarrow \max(\hat{A}_g, 0)$，确保只强化可行前缀，不因下游失败而惩罚它。

### 3.3 模型、训练与数据集

- **模型**：IterCAD基于**Qwen3.5-4B**模型初始化。
- **训练数据**：
    - **CAD Pairs构建**：通过一个专门的数据合成管线（图3）生成三种高质量配对数据：
        - **Drawing-Code Pairs**：定义高级操作空间（$\mathcal{P}_{base} \cup \mathcal{P}_{adv}$），使用SolidWorks COM接口生成符合标准的多视图工程图纸。
        - **Text-Code Pairs**：精炼Text2CAD数据集中的文本描述，并翻译为CadQuery代码，经过沙箱执行和CD过滤。
        - **Edit-Code Pairs**：通过对完美代码施加受控退化来模拟设计修改，形成源代码、修改指令和目标代码的三元组。
    - **专家轨迹合成**：使用**Qwen3-VL-235B-A22B-Instruct**模型，基于上述配对数据，在IterCAD协议下滚动生成多轮交互轨迹。生成的轨迹经过格式合规性、逻辑一致性和几何正确性三重过滤，得到高质量的冷启动语料库。
- **训练过程**：
    - **第一阶段：渐进式SFT**：先在20K专家轨迹（DS1）上训练，再在8K策略滚动轨迹（DS2）上训练。
    - **第二阶段：GSPO RL**：在2K个困难的Drawing-to-Code样本上进行强化学习。使用组大小 $G=8$，最大交互轮次 $T=5$。奖励权重 $\lambda_f=0.5, \lambda_p=0.2$。GVPM的触发窗口 $K=2$。

### 3.4 实验设置与结果

- **基准**：在**IterCAD-Bench**（1K绘图任务，200编辑任务）、**Text2CAD Bench**（8,046个零件）和**CADPrompt Bench**（200个专家指令）上评估。
- **评估指标**：**IR（无效比率）**、**Mean/Med. CD**（仅对成功执行的计算）、**Avg. Turn**（平均交互轮次），以及核心指标**AUC-TR**。
- **主要结果**：
    - **IterCAD-Draw任务（表1）**：IterCAD在直接推理和智能体工作流两种设置下均表现最佳。在智能体工作流下，IR仅为**0.30%**，AUC-TR高达**0.61**，平均CD为**5.09**，平均轮次仅**2.48**。显著优于GPT-5、Gemini等专有模型和所有开源基线。
    - **IterCAD-Edit任务（表2）**：IterCAD同样表现优异，IR低至**1.00%**，平均轮次**2.34**，接近GPT-5的水平，验证了其高效的编辑能力。
    - **Text2CAD和CADPrompt基准（表3、4）**：IterCAD在外部基准上也取得了领先性能，尤其是在IR和Med. CD指标上，证明了其强大的泛化能力。
- **消融实验（表5）**：逐步验证了各组件的重要性。从基线（IR 62.30%）开始，加入SFT（DS1）后IR降至7.50%；加入DS2后IR降至0.80%；加入GSPO后Mean CD从12.44降至8.00；最终加入GVPM后，模型达到最优（IR 0.30%，AUC-TR 0.61，Mean CD 5.09），证明了GVPM在解决信用分配问题上的关键作用。

### 3.5 技术细节

- **CAD沙箱**：基于CadQuery和OCCT内核，能够执行代码、返回编译器错误，并程序化地生成带尺寸标注的正交投影视图作为视觉反馈。
- **CD-TR曲线**：通过扫描容差阈值 $\tau$，计算召回率 $R(\tau) = \frac{1}{M} \sum_{i=1}^M \mathbb{I}(\text{Valid}(y_i) \land CD(y_i) \leq \tau)$。执行失败的样本在所有阈值下召回率均为0，从而消除了幸存者偏差。AUC-TR是该曲线下的面积。
- **GVPM实现**：通过监测连续 $K=2$ 轮的执行错误或CD停滞来动态确定掩码边界，这是一种轻量级且有效的在线信用分配方法。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **范式转变**：将CAD自动化从“开环、一次性”生成转变为“闭环、交互式”优化，更符合实际工程设计流程。
    2.  **性能提升**：在代码可执行性和几何精度上显著超越了现有方法，尤其是在处理复杂、高级的制造特征时。
    3.  **评估标准革新**：提出了CD-TR曲线和AUC-TR指标，为CAD生成领域提供了一个更公平、更全面的评估标准，解决了长期存在的幸存者偏差问题。
    4.  **方法论贡献**：提出的两阶段训练策略和GVPM为解决多轮交互智能体中的信用分配问题提供了有效思路。

- **对未来研究/应用的影响**：
    - 为开发更智能、更可靠的自动化CAD工具奠定了基础，有望加速产品设计流程。
    - 其“Look and Loop”的框架思想可以推广到其他需要迭代优化的生成任务中。
    - 提出的无幸存者偏差评估标准可能成为该领域未来的基准评估方法。

- **局限与开放问题**：
    1.  **范围限制**：目前仅关注单零件参数化建模，扩展到大型装配体协同设计是未来方向。
    2.  **语义理解不足**：模型主要依赖几何指标（如CD），无法理解高级机械语义（如键槽、轴承配合、螺纹规格），生成的模型可能满足空间尺寸但缺乏工程功能意图。
    3.  **代码可维护性**：模型倾向于优先实现几何收敛，可能导致生成的代码缺乏结构化、层次化的参数化设计树，不利于人类协作和后续维护。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“幸存者偏差”的通俗解释**：想象一个射击比赛，我们只统计命中靶心的子弹（成功生成的代码），然后计算它们的平均环数（几何精度）。这忽略了大量脱靶的子弹（执行失败的代码）。一个选手可能只射中了一发10环，但其他99发都脱靶了，他的平均成绩是10环，看起来非常厉害。而另一个选手射中了50发8环和50发9环，平均成绩是8.5环，看起来不如前者。但实际上，第二个选手的“有效命中率”和“整体表现”远好于第一个。CD-TR曲线和AUC-TR指标就是同时考虑了“是否命中”（代码是否有效）和“命中精度”（几何精度）的评估方法，避免了只看“命中”样本的偏差。

- **“几何可行前缀掩码（GVPM）”的类比**：想象一个团队在组装一个复杂的乐高模型（生成CAD代码）。团队先搭建了底座和主体结构（前几轮动作），然后尝试安装一个复杂的齿轮组（后续动作），结果齿轮装不上，导致整个模型卡住了（执行失败或几何停滞）。如果按照传统的强化学习，整个团队都会因为最终失败而受到惩罚，包括那些成功搭建底座和主体的成员。GVPM就像一个聪明的项目经理，他能识别出“底座和主体搭建得很好，问题出在齿轮组上”。因此，他决定只奖励底座和主体团队的贡献（保留可行前缀的梯度），而忽略齿轮组团队的负面贡献（掩
