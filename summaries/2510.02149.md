## 0. 概览（Concise Summary）

- **主题（Topic）**：强化学习（Reinforcement Learning, RL），具体针对部分可观测马尔可夫决策过程（POMDP）的一个特殊子类，其中状态观测是由智能体的动作触发的。
- **问题（Problem）**：在现实世界中，获取完整的状态信息（如医疗诊断、金融数据）往往代价高昂或不可靠，且观测的频率可能受智能体自身动作的影响。现有的RL框架（如标准MDP、一般POMDP）未能有效建模和利用这种“动作触发观测”的结构，导致学习效率低下或理论保证缺失。
- **方法（Method）**：提出了一个名为“动作触发稀疏可追踪马尔可夫决策过程”（ATST-MDP）的新框架。核心方法包括：
    1.  **增广状态空间**：将“最后观测到的状态”和“自那之后执行的动作序列”作为智能体的增广状态，使其成为一个完全可观测的决策过程。
    2.  **动作序列重表述**：利用“数据爆发”（观测到完整状态）会重置状态的特点，将问题重新表述为智能体在每个观测点选择并执行一个动作序列，直到下一次观测。
    3.  **线性表示**：在线性MDP假设下，证明了动作序列的价值函数可以表示为有限维特征映射的线性函数，从而可以使用标准的回归方法进行学习。
- **创新（Innovation）**：
    1.  **问题形式化**：首次系统地形式化了“动作触发观测”这一普遍存在的RL问题，提出了ATST-MDP框架。
    2.  **理论分析**：推导了ATST-MDP的贝尔曼最优方程，证明了最优策略的存在性。
    3.  **高效算法**：基于线性MDP假设，设计了ATST-LSVI-UCB算法，并证明了其遗憾界与完全可观测线性MDP的最优率相匹配。
- **意义（Significance）**：该工作为在信息获取有成本或不可靠的复杂决策场景（如医疗、金融、机器人）中设计高效、有理论保证的RL算法奠定了坚实的理论基础。它揭示了动作与观测之间的内生关系，并展示了如何利用这种结构实现高效学习。
- **一句话总结**：本文提出了ATST-MDP框架，通过将动作触发的稀疏观测问题转化为动作序列上的线性决策问题，实现了与完全可观测MDP同等的理论保证。

## 1. 研究动机（Motivation）

- **解决什么问题**：解决标准强化学习（RL）框架中一个不切实际的假设——智能体在每次执行动作后都能立即、无成本地获得完整的状态观测。在许多实际应用中，情况并非如此。
- **为什么重要**：
    - **现实相关性**：许多现实世界的决策问题都面临观测成本或不可靠性。例如：
        - **临床治疗规划**：医生需要在有风险但信息量大的侵入性检查（如活检）和更安全但信息有限的无创检查之间权衡。检查动作本身决定了是否能获得完整的患者状态信息。
        - **金融投资组合管理**：交易员需要平衡获取高频市场数据的成本和基于信息做出明智决策的收益。
        - **机器人自主导航**：机器人可能需要主动转向或停下来“观察”环境，这会消耗时间和能量。
    - **现有框架的不足**：标准MDP假设完全可观测，不适用。一般的部分可观测MDP（POMDP）虽然能建模，但计算和统计上通常是难解的（intractable），且其通用方法无法利用“动作触发观测”这一特殊结构。
- **挑战在哪里**：
    1.  **部分可观测性**：智能体无法直接知道当前状态，必须基于历史信息（最后观测到的状态和之后执行的动作）进行决策。
    2.  **观测的内生性**：观测是否发生不是随机的，而是由智能体选择的动作决定的。这要求策略不仅要优化动作以获取奖励，还要策略性地决定何时“触发”观测以减少不确定性。
    3.  **状态空间的爆炸**：如果简单地将所有可能的动作历史作为状态，状态空间会无限增长，导致学习困难。

## 2. 创新点（Innovation）

1.  **提出ATST-MDP框架**：首次形式化定义了动作触发观测的RL问题，用一个六元组 `(S, A, P, r, γ, β)` 来刻画，其中 `β: A → [0,1]` 是关键创新，表示每个动作触发观测的概率。
2.  **增广状态空间与贝尔曼方程**：将“最后观测到的状态”和“自那之后执行的动作序列”定义为增广状态 `x = (s, a1, ..., an)`。在这个增广状态空间上，问题转化为一个完全可观测的MDP，并推导了相应的贝尔曼最优方程（Theorem 3.1），证明了最优策略的存在性。
3.  **动作序列重表述**：利用“数据爆发”（观测到状态）会重置增广状态的特点，将问题重新表述为在观测点选择并执行一个动作序列 `a ∈ A^N`。这个重表述（Section 3.2）将无限长的历史压缩为有限的、可管理的单元，是后续线性表示的基础。
4.  **动作序列价值函数的线性表示**：在线性MDP假设下（Assumption 4.1），构造了一个有限维（2d维）的特征映射 `ψ: S × A^N → R^{2d}`，并证明了对于任何策略，其动作序列价值函数 `K^π(s, a)` 都是 `ψ(s, a)` 的线性函数（Theorem 4.4）。这是本文最核心的理论创新，它将一个无限维的优化问题（在 `A^N` 上）简化为一个有限维的线性回归问题。
5.  **ATST-LSVI-UCB算法与遗憾分析**：基于线性表示，提出了一个乐观的价值迭代算法。该算法在假设能获得一个近似特征映射 `ψ` 的情况下，实现了 `Õ(√(d³K/(1-γ)³))` 的遗憾界（Theorem 5.1），这与完全可观测线性MDP的最优率相匹配。这表明，尽管观测是稀疏的，但利用其结构，学习效率可以不受影响。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心架构是围绕ATST-MDP框架展开的，其方法流程如下：

1.  **问题建模**：将问题建模为ATST-MDP `(S, A, P, r, γ, β)`。其中 `β(a)` 是动作 `a` 触发“数据爆发”（即揭示完整状态）的概率。
2.  **增广状态空间**：定义增广状态 `x = (s, a1, ..., an)`，其中 `s` 是最后观测到的状态，`(a1, ..., an)` 是自那之后执行的动作序列。智能体在增广状态空间 `X` 上决策，使得问题在 `X` 上完全可观测。
3.  **动作序列重表述**：由于“数据爆发”会将增广状态重置为 `s'`（新观测到的状态），因此可以将决策过程分解为一系列“观测间期”。在每个观测点 `s`，智能体选择一个动作序列 `a = (a1, a2, ...)` 并执行，直到下一次“数据爆发”发生。这定义了动作序列价值函数 `K^π(s, a)`。
4.  **线性化**：在线性MDP假设下，构造特征映射 `ψ(s, a)`，使得 `K^π(s, a) = ⟨ψ(s, a), v^π⟩`。这使得价值函数的学习简化为对参数向量 `v^π` 的线性回归。
5.  **学习与规划**：基于上述线性表示，设计ATST-LSVI-UCB算法。该算法使用历史数据（观测到的状态、选择的动作序列、累积奖励、下一个观测到的状态）来估计 `v^π`，并通过添加置信上界（UCB）来鼓励探索。在规划阶段，它通过向后迭代计算价值函数；在执行阶段，它选择使当前价值函数最大的动作序列。

### 3.2 关键算法与数学推导

**1. 增广状态上的贝尔曼最优方程 (Theorem 3.1)**

对于增广状态 `x` 和动作 `a`，最优价值函数 `V^*` 满足：
$$TV(x) = \max_{a \in A} \left\{ \mathbb{E}_{s \sim b(\cdot|x)}[r(s,a)] + \gamma \beta_a \mathbb{E}_{s' \sim b(\cdot|x \oplus a)}[V(s')] + \gamma \bar{\beta}_a V(x \oplus a) \right\}$$
其中：
- `b(·|x)` 是给定增广状态 `x` 下的信念状态（对当前真实状态的分布）。
- `β_a = β(a)` 是动作 `a` 触发观测的概率，`\bar{β}_a = 1 - β_a`。
- `x ⊕ a` 表示将动作 `a` 追加到历史序列后形成的新增广状态。
- `T` 是贝尔曼算子，它是一个γ-压缩映射，保证了最优价值函数 `V^*` 是其唯一不动点。

**2. 动作序列价值函数的线性表示 (Theorem 4.4)**

在线性MDP假设下，存在特征映射 `ϕ(s, a)` 和参数 `θ, µ` 使得 `r(s,a) = ⟨ϕ(s,a), θ⟩` 和 `P(·|s,a) = ⟨ϕ(s,a), µ(·)⟩`。

定义动作矩阵 `M_a = ∫_S µ(s) ϕ(s, a)^T ds`。

构造动作序列特征映射 `ψ: S × A^N → R^{2d}`：
$$ψ(s, a \oplus \mathbf{a})^T = \frac{1}{2} \left[ \phi(s, a)^T \left( (1-\gamma)I_d \quad \gamma I_d \right) + \bar{\beta}_a \phi(s, a)^T \left( (1-\gamma)M_1(\mathbf{a}) \quad \gamma M_2(\mathbf{a}) \right) \right]$$
其中 `M_1(a)` 和 `M_2(a)` 是依赖于后续动作序列 `a` 的矩阵，由 `M_a` 和 `β_a` 的无穷级数定义（公式4a, 4b）。

**定理核心**：存在一个与策略 `π` 相关的参数向量 `v^π_{1,2} ∈ R^{2d}`，使得对于任何状态 `s` 和动作序列 `a`，动作序列价值函数 `K^π(s, a)` 可以表示为：
$$K^\pi(s, \mathbf{a}) = \langle \psi(s, \mathbf{a}), v^\pi_{1,2} \rangle$$

**3. 特征映射的估计 (Theorem 4.6)**

由于 `ψ` 依赖于未知的 `M_a` 和 `β_a`，需要从数据中估计。给定估计值 `\hat{M}_a` 和 `\hat{β}_a`，可以构造估计的特征映射 `\hat{ψ}`。定理4.6给出了估计误差的传播界限：
如果 `|||\hat{M}_a - M_a|||_2 ≤ ε` 且 `|\hat{β}_a - β_a| ≤ ε_β`，则：
$$\sup_{s, \mathbf{a}} \| (\hat{\psi} - \psi)(s, \mathbf{a}) \|_2 \leq \frac{16d}{1-\gamma} (\varepsilon + \frac{\varepsilon_\beta}{\sqrt{d}})$$

**4. 遗憾界 (Theorem 5.1)**

对于ATST-LSVI-UCB算法，在K个回合后的总遗憾 `R_K` 满足：
$$R_K = \tilde{O}\left( \sqrt{d^3 K (1-\gamma)^{-3}} \iota^2 + d^2 (1-\gamma)^{-2} \iota + \epsilon \cdot \sqrt{d^2 K^3 (1-\gamma)^{-5}} \iota \right)$$
其中 `ι` 是对数项，`ϵ` 是特征映射的近似误差。当 `ϵ` 足够小（如 `O((1-γ)/(K√d))`）时，第三项成为低阶项，遗憾界简化为 `Õ(√(d³K/(1-γ)³))`，与完全可观测线性MDP的最优率一致。

### 3.3 模型、训练与数据集

- **模型**：ATST-MDP模型，其核心是线性MDP假设（Assumption 4.1）。该假设认为转移概率和奖励函数是已知特征映射 `ϕ(s, a)` 的线性函数。这是一个在理论RL中广泛使用的标准假设，用于处理大规模或连续状态空间。
- **训练**：训练过程分为两个阶段：
    1.  **特征映射估计阶段（离线/探索阶段）**：使用一个探索性分布 `D` 收集 `N` 个样本 `(s, a, s', b)`。通过岭回归估计动作矩阵 `M_a`，通过经验均值估计观测概率 `β_a`。然后构造近似的特征映射 `\hat{ψ}`。这个过程是“无奖励的”（reward-free），即一旦完成，`\hat{ψ}` 可用于任何奖励函数。
    2.  **在线学习阶段（ATST-LSVI-UCB）**：智能体与环境进行 `K` 个回合的交互。在每个回合，它使用历史数据（`H_k`）和 `\hat{ψ}` 进行规划（价值迭代），然后执行贪心策略。
- **数据集**：
    - **特征估计数据集**：`{s_n, a_n, s'_n, b_n}_{n=1}^N`，其中 `(s_n, a_n)` 从探索分布 `D` 中采样，`s'_n` 是下一个状态，`b_n` 是观测指示器（是否发生数据爆发）。
    - **在线学习历史数据集**：`H_k = {(s_τ, a_τ, R_τ, s_τ^N)}_{τ=1}^{N_k}`，其中 `s_τ` 是观测到的状态，`a_τ` 是执行的动作序列，`R_τ` 是到下一次观测前的累积折扣奖励，`s_τ^N` 是下一次观测到的状态（或终止符）。

### 3.4 实验设置与结果

- **环境**：两个表格型（tabular）ATST-MDP环境，状态空间 `|S|=6`，动作空间 `|A|=2`。
    - **RiverSwim**：一个经典的探索基准，最优策略是始终选择“向右”动作，但需要克服“水流”的阻力。
    - **RiverBalance**：一个本文新提出的环境，奖励集中在中心状态 `s3, s4`，但“水流”会将智能体推向两端，最优策略需要不断调整动作以保持在中心。
- **参数**：折扣因子 `γ=0.99`，观测概率 `β(a) = β* ∈ {0.05, 0.1, 0.2, 0.5}`。算法使用精确的特征映射 `ψ`，有效视界 `H=100`。动作序列被限制在“最终周期序列”的候选集 `A_seq` 中。
- **结果**：
    - **RiverSwim**（图3a）：所有 `β*` 设置下，算法最终都收敛到最优策略。有趣的是，**更小的 `β*`（更稀疏的观测）收敛得更快**。这是因为稀疏观测迫使智能体做出更长的“开环”承诺，这反而鼓励了早期的探索。
    - **RiverBalance**（图3b）：所有设置下，平均奖励都先上升然后趋于平稳。但**更大的 `β*`（更频繁的观测）获得了更高的平均奖励**。这是因为维持高奖励需要频繁的状态依赖修正（闭环控制），而更频繁的观测使得这种修正成为可能。
- **结论**：实验验证了ATST-MDP框架的核心洞察：观测频率的影响取决于任务结构。对于不需要频繁状态修正的任务，稀疏观测可以加速学习；对于需要精细闭环控制的任务，频繁观测则至关重要。

### 3.5 技术细节

- **增广状态与信念**：增广状态 `x = (s, a1, ..., an)` 是充分统计量。信念状态 `b(·|x)` 可以通过对从 `s` 开始执行动作序列 `(a1, ..., an)` 的轨迹进行边际化得到（公式1）。
- **动作序列价值函数**：`K^π(s, a)` 将价值函数分解为两部分：`R(s, a)`（观测前累积折扣奖励）和 `PV^π(s, a)`（观测后折扣续存价值）。
- **特征映射构造**：`ψ` 的构造（公式3）巧妙地利用了线性MDP的结构，将关于无限长动作序列的期望转化为关于动作矩阵 `M_a` 和观测概率 `β_a` 的无穷级数（公式4a, 4b）。这些级数在 `γ < 1` 时收敛。
- **ATST-LSVI-UCB算法**：
    - **规划阶段**：使用历史数据 `H_k` 构建协方差矩阵 `Λ_k`，然后通过岭回归计算权重 `w^k_u`。价值函数 `K^k_u(s, a)` 由线性项 `⟨\hat{ψ}(s,a), w^k_u⟩` 和一个基于协方差矩阵的探索奖励 `ρ ||\hat{ψ}(s,a)||_{(Λ^k)^{-1}}` 组成。
    - **执行阶段**：在每个观测点，选择使 `K^k_u` 最大的动作序列 `a` 并执行，直到下一次观测或回合结束。
