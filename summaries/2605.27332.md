## 0. 概览（Concise Summary）

- **主题（Topic）**：工业需求工程中的流程图处理，具体是将静态流程图图像转换为机器可读的Mermaid代码。
- **问题（Problem）**：现有的视觉语言模型（VLM）在直接处理流程图时，存在“拓扑盲点”，即难以准确感知节点连接和边的方向，导致生成的图结构错误，无法用于后续的自动化需求工程任务（如模型测试）。
- **方法（Method）**：提出一个名为**EdgeFlow**的训练无关框架。该框架在向VLM输入原始流程图图像的同时，附加一张通过Canny边缘检测算法确定性提取的边缘图（Edge Map），作为**结构先验**，引导VLM关注拓扑关键特征。
- **创新（Innovation）**：
    1.  首次将**确定性边缘图**作为结构先验，以**训练无关**的方式增强VLM的流程图拓扑感知能力。
    2.  构建了一个来自真实工业场景的流程图数据集**IndusReqFlow**，用于评估。
    3.  在节点、边和路径三个层面进行了严格的统计显著性检验，证明了方法的有效性。
- **意义（Significance）**：为工业界提供了一种**低成本、即插即用**的解决方案，无需标注数据和模型微调，即可显著提升VLM在流程图处理上的拓扑准确性，从而打通从静态图像到自动化需求工程的“视觉规范鸿沟”。
- **一句话总结**：EdgeFlow通过给VLM“看”一张边缘图，像给模型戴上了一副“拓扑眼镜”，显著提升了其理解工业流程图中复杂连接关系的能力，且无需任何额外训练。

## 1. 研究动机（Motivation）

- **问题与空白**：在工业需求工程中，流程图被广泛用于描述系统行为、业务逻辑和操作流程。然而，这些流程图通常以**静态图片**的形式嵌入在文档中，无法被机器直接解析和利用。这导致了“视觉规范鸿沟”（visual specification gap），即无法对流程图进行自动化分析，如追溯性检查、变更影响分析和基于模型的测试（MBT）。
- **重要性**：当前的需求工程研究在**文本处理**方面（如NLP、LLM）取得了巨大成功，但从**视觉工件**中提取结构化模型仍是一个关键瓶颈。解决此问题能极大地推动需求工程的自动化水平，特别是对于需要从流程图中枚举测试路径的工业测试团队。
- **挑战**：直接应用先进的VLM（如Qwen-VL）进行流程图转换时，它们存在严重的**拓扑结构误判**问题，例如：
    - **节点连接错误**：遗漏或错误连接节点。
    - **边方向错误**：混淆控制流的方向。
    - **“拓扑盲点”**：模型倾向于依赖文本标签的邻近性而非实际的几何连接线（箭头、线条）来判断连接关系。
    - 现有的一些缓解方法（如分阶段流水线，先OCR再解析）对上游感知错误非常敏感，且需要额外训练。

## 2. 创新点（Innovation）

- **核心创新**：提出**EdgeFlow**框架，其核心创新在于**为VLM提供确定性的结构先验**。
- **具体方法**：在VLM处理流程图图像时，**同时输入**原始图像和一张通过Canny边缘检测算法生成的边缘图。这张边缘图作为“结构骨架”，高亮显示了线条、箭头等拓扑关键信息。
- **与现有方案的不同**：
    - **无需训练**：与需要大量标注数据微调的模型或需要训练目标检测器/分割模型的分阶段流水线不同，EdgeFlow是**训练无关**的。Canny边缘检测是完全确定性的，无需任何学习过程。
    - **即插即用**：可以轻松地集成到任何现有的VLM工作流中，只需在输入中增加一张边缘图。
    - **针对性强**：直接针对VLM的“拓扑盲点”进行增强，通过提供清晰的几何线索，引导模型关注正确的连接关系，而不是依赖语义或文本位置。
- **为什么能奏效**：VLM（特别是基于ViT的模型）将图像分割成固定大小的块（patch），这会导致细小的连接线和箭头在块边界处被分割，丢失了关键的几何信号。Canny边缘图将这些信号提取并强化为高对比度的结构，使其在ViT的视觉特征中更加显著，从而补偿了patch化带来的信息损失。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

EdgeFlow框架包含四个步骤，如图3所示：

1.  **➊ 图像预处理 (Image Preprocessing)**：标准化输入图像，处理工业文档中的异构格式、分辨率和压缩伪影。具体包括：
    - **Alpha通道归一化**：将透明背景合成到白色背景上，避免VLM将透明区域误判为黑色元素。
    - **自适应缩放**：将图像最大维度缩放到4000像素以内，保留细节的同时满足VLM的输入限制。
2.  **➋ 确定性结构先验提取 (Deterministic Structural Prior Extraction)**：使用Canny边缘检测算法从预处理后的图像中提取边缘图。这个过程是确定性的，确保相同输入产生相同输出。
3.  **➌ Mermaid代码生成 (Mermaid Code Generation)**：构建一个多图像输入元组 `X = ⟨I_prep, I_edge, T_prompt⟩`，将预处理图像、边缘图和结构化提示词一起输入VLM。提示词指导模型先观察边缘图获取结构信息，再结合原始图像提取语义标签，最终生成Mermaid代码。
4.  **➍ 语法验证 (Syntax Validation)**：对生成的Mermaid代码进行语法检查。如果解析失败，将错误信息传递给一个代码专用LLM进行迭代修正，最多10次迭代，确保输出代码的语法正确性。

### 3.2 关键算法与数学推导

- **Canny边缘检测**：论文中未给出具体公式，但描述了其四个步骤：
    1.  **平滑**：使用高斯滤波器 $G(x,y)$ 对图像 $I(x,y)$ 进行卷积，减少噪声。
        $$S(x,y) = G(x,y) * I(x,y)$$
    2.  **计算梯度**：使用Sobel算子计算平滑后图像的梯度幅值 $M(x,y)$ 和方向 $\theta(x,y)$。
        $$M(x,y) = \sqrt{G_x^2 + G_y^2}, \quad \theta(x,y) = \text{atan2}(G_y, G_x)$$
        其中 $G_x$ 和 $G_y$ 是水平和垂直方向的梯度。
    3.  **非极大值抑制**：沿着梯度方向，只保留局部梯度幅值最大的像素点，细化边缘。
    4.  **双阈值滞后处理**：使用高阈值 $T_{high}$ 和低阈值 $T_{low}$ 确定强边缘和弱边缘。只有与强边缘相连的弱边缘才会被保留，形成连续的边缘轮廓。
        $$I_{edge} = \text{Canny}(I_{prep}, \theta)$$
        其中 $\theta = \{T_{low}, T_{high}, \text{aperture}\}$ 是可调的超参数。

- **评估指标**：使用精确率（Precision, P）、召回率（Recall, R）和F1分数（F1-score）在三个层面进行评估。
    - **节点级**：预测节点标签与真实节点标签完全匹配。
    - **边级**：预测边 `(source_label, target_label, edge_label)` 三元组与真实边完全匹配。
    - **路径级**：预测路径的节点标签序列与真实路径完全匹配。

### 3.3 模型、训练与数据集

- **模型**：
    - **VLM**：用于Mermaid代码生成。选择了两种架构的模型：
        - `Qwen3-VL-32B-Instruct`：稠密模型，320亿参数。
        - `Qwen3.5-35B-A3B`：稀疏MoE模型，350亿总参数/30亿激活参数。
    - **代码LLM**：用于语法验证和修正，选用 `Qwen3-Coder-Next`。
- **训练**：**无**。EdgeFlow是一个完全训练无关的框架。
- **数据集**：
    - **IndusReqFlow**：**核心数据集**。包含52个来自华为真实工业需求文档的流程图，涵盖光网络、数据通信和电动汽车领域。特点是**高视觉噪声**（扫描伪影、光照不均）和**结构歧义**（重叠连接线、非标准符号）。平均每个流程图有11.15个节点和11.73条边。
    - **FlowVQA**：**公开数据集**。用于跨域泛化评估。随机抽取了40个合成流程图。特点是**低噪声**、结构清晰。平均每个流程图有21.35个节点和23.70条边。

### 3.4 实验设置与结果

- **研究问题**：
    - **RQ1**：EdgeFlow能否提升工业流程图的拓扑提取正确性（节点和边）？
    - **RQ2**：EdgeFlow能否提升路径生成质量，从而更好地支持下游MBT任务？
    - **RQ3**：EdgeFlow的性能提升能否泛化到干净的合成数据集上？
- **基线**：使用与EdgeFlow完全相同的设置（模型、参数、提示词、语法验证），但**不提供边缘图**，只输入预处理后的原始图像。
- **主要结果**：
    - **RQ1 (IndusReqFlow)**：
        - **Qwen3-VL-32B**：EdgeFlow在节点F1上提升**17.39个百分点**（63.77% → 81.16%），在边F1上提升**16.94个百分点**（48.94% → 65.88%）。统计显著（p<0.001），效应量大（Cliff's δ > 0.67）。
        - **Qwen3.5-35B-A3B**：EdgeFlow在节点F1上提升**7.81个百分点**，在边F1上提升**10.00个百分点**。统计显著，效应量大。
        - **结论**：EdgeFlow显著提升了拓扑正确性，尤其在边（连接关系）上的提升更为明显。
    - **RQ2 (IndusReqFlow)**：
        - **Qwen3-VL-32B**：EdgeFlow在路径F1上提升**11.06个百分点**（18.31% → 29.37%）。统计显著，效应量中等。
        - **Qwen3.5-35B-A3B**：EdgeFlow在路径F1上提升**3.93个百分点**，但召回率略有下降。统计显著，效应量小。
        - **结论**：节点和边的改进能转化为路径级别的提升，但路径F1绝对值仍然不高，这主要是由于精确匹配的级联效应（一个节点或边的错误就会导致整条路径错误）。
    - **RQ3 (FlowVQA)**：
        - **Qwen3-VL-32B**：节点F1提升不显著，边F1提升很小（+1.45 pp），统计显著但效应量小。
        - **Qwen3.5-35B-A3B**：所有指标均无显著提升。
        - **结论**：在干净的合成数据上，基线模型已经表现很好（节点F1 > 91%），EdgeFlow的提升空间有限。这说明EdgeFlow的优势主要体现在处理有噪声、结构退化的工业数据上。

### 3.5 技术细节

- **Canny参数优化**：通过两阶段网格搜索确定最优的Canny阈值和Sobel孔径大小。对于Qwen3-VL-32B，最优配置是C3（低阈值100，高阈值200，孔径3），这也是ControlNet的默认配置。对于Qwen3.5-35B-A3B，最优配置是C9（低阈值50，高阈值150，孔径7），更强的平滑有助于处理MoE模型的噪声。
- **统计测试**：使用**Wilcoxon符号秩检验**（非参数配对检验）评估显著性，使用**Cliff's Delta**评估效应量大小，并报告胜/平/负计数。
- **路径提取**：将Mermaid代码解析为节点和边的JSON结构，构建邻接表，使用深度优先搜索（DFS）枚举所有从入口到终端的路径。对于循环，只展开一次，以模拟最小路径覆盖的MBT实践。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了VLM在工业流程图处理中的“拓扑盲点”问题，这是阻碍其实际应用的关键障碍。
    - 提供了一个**实用、低成本、无需训练**的解决方案，工业团队可以立即部署，无需昂贵的标注和微调。
    - 通过严格的实验证明了方法的有效性，并揭示了**工业数据与合成数据之间的巨大差异**，强调了构建包含真实工业数据的基准测试集的重要性。
- **对未来研究/应用的影响**：
    - **工业实践**：为自动化需求工程（如模型测试、变更分析）铺平了道路，有望显著减少工程师手动解析流程图的工作量。
    - **研究方向**：启发了“为VLM提供结构先验”这一研究方向，未来可以探索其他类型的先验（如骨架化、线段检测等）。
    - **基准测试**：论文呼吁社区构建更多样化、包含真实工业数据的基准，以全面评估VLM在RE工具中的表现。
- **局限与开放问题**：
    - **参数选择偏差**：Canny参数在IndusReqFlow上优化，可能存在过拟合风险。
    - **模型选择**：仅评估了Qwen系列模型，泛化性到其他VLM家族（如GPT-4V, Gemini）未知。
    - **数据集规模**：IndusReqFlow只有52个流程图，统计效力有限。
    - **图类型**：仅针对流程图，是否能推广到其他行为图（如序列图、状态机）是开放问题。
    - **先验选择**：Canny不是唯一的结构先验，其他方法（如学习到的轮廓检测器）可能效果更好。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：给VLM戴“拓扑眼镜”**
    - **问题**：想象一下，让一个近视的人（VLM）看一张复杂的电路图（流程图）。他/她可能能看清上面的文字标签（节点文本），但很难看清那些细小的连接线（边），尤其是那些交叉或很长的线，导致他/她错误地理解了电路是如何连接的。
    - **EdgeFlow的解决方案**：EdgeFlow就像给这个人配了一副“拓扑眼镜”。这副眼镜（Canny边缘检测）会自动提取出电路图中所有线条的“骨架”，并把它画在一张白纸上（边缘图）。现在，这个人可以同时看原始电路图和这张“骨架图”。骨架图清晰地告诉他/她哪些点是连在一起的，连接线往哪个方向走。这样，他/她就能更准确地理解整个电路的结构了。
    - **关键点**：这副“拓扑眼镜”是**现成的**（Canny算法），不需要专门为他/她定制（无需训练）。它只是**辅助**他/她看得更清楚，而不是改变他/她的大脑（不修改VLM模型）。

- **为什么路径F1分数不高？**
    - 路径级评估非常严格，要求整条路径的节点序列完全匹配。这就像要求一个学生默写一首诗，只要错一个字，整首诗就算错。在EdgeFlow中，即使节点F1达到了81%，边F1达到了66%，意味着还有约20%的节点和35%的边是错误的。这些错误会组合起来，导致大多数路径都无法完全匹配。因此，路径F1分数（29.37%）远低于节点和边F1，这是符合预期的“级联效应”。
