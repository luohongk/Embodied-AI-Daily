## 0. 概览（Concise Summary）

- **主题（Topic）**：双足机器人基于力矩的鲁棒行走控制，结合了模型预测控制与强化学习。
- **问题（Problem）**：纯模型控制方法对模型不准确和传感器噪声敏感，而纯强化学习方法数据效率低、奖励设计困难。如何高效地学习一个鲁棒的、能适应真实世界不确定性的力矩级行走策略。
- **方法（Method）**：提出一个混合控制框架，以基于DCM（发散分量运动）和全身控制的模型控制器作为基础策略（Base Policy），并引入一个残差强化学习策略（Residual Policy）来补偿模型误差。关键创新在于，训练时使用一个拥有“特权信息”（真实动力学参数）的模型“神谕策略”（Oracle Policy）通过监督损失（Supervised Loss）来指导残差策略的学习。
- **创新（Innovation）**：
    1.  提出了一个**模型监督的残差强化学习框架**，将模型控制器的结构化可靠性与强化学习的适应性相结合。
    2.  引入了一个**基于模型的神谕策略**，在训练时利用特权信息生成近乎最优的控制信号，并以此作为监督信号。
    3.  设计了**包含监督损失的训练目标**，将模仿神谕策略与强化学习探索相结合，显著提高了学习效率和鲁棒性。
- **意义（Significance）**：该方法为双足机器人的Sim-to-Real（仿真到现实）迁移提供了一种高效、鲁棒且可扩展的解决方案。它解决了力矩级控制中模型不匹配的难题，同时避免了纯强化学习的高数据需求，有望推动双足机器人在复杂真实环境中的应用。
- **一句话总结**：本文提出了一种通过神谕策略监督的残差强化学习框架，高效地训练出鲁棒的力矩级双足行走控制器，有效弥合了模型控制与数据驱动方法之间的鸿沟。

## 1. 研究动机（Motivation）

- **核心问题**：双足机器人的力矩级行走控制面临两大挑战：
    1.  **纯模型控制方法的局限性**：传统的基于模型的方法（如DCM+WBC）依赖于精确的机器人动力学模型和状态估计。然而，真实世界中存在未建模动力学（如关节摩擦、执行器非线性）、传感器噪声和时延，导致模型与实际系统之间存在差距（Sim-to-Real gap），使控制性能下降甚至不稳定。
    2.  **纯强化学习方法的局限性**：虽然深度强化学习（DRL）通过域随机化可以训练出鲁棒的策略，但它通常需要海量的交互数据，对奖励函数设计非常敏感，且缺乏模型方法的可解释性和安全保障。直接学习力矩级的策略尤其困难，因为动作空间高维且动力学复杂。
- **为什么重要**：解决上述问题对于实现双足机器人在非结构化、动态变化的真实世界中稳定、高效地行走至关重要。力矩控制本身具有响应快、能效高、柔顺性好的优点，但如何克服其易受模型误差影响的缺点是一个关键瓶颈。
- **挑战**：
    - 如何将模型控制器的先验知识与强化学习的数据驱动适应能力有效结合。
    - 如何设计一个高效的训练框架，既能利用模型知识加速学习，又能避免纯强化学习的“冷启动”和奖励工程难题。
    - 如何确保学习到的策略在面临各种不确定性（动力学、传感器、环境）时依然鲁棒。

## 2. 创新点（Innovation）

本文的核心创新在于提出了一个**模型监督的残差强化学习框架**，具体体现在：

1.  **混合架构**：将**模型基础策略**（Base Policy，基于DCM+WBC）与**可学习的残差策略**（Residual Policy）相结合。基础策略提供了一个稳定、可靠的基线行为，而残差策略则专注于学习补偿基础策略因模型不准确而产生的误差。这比从头开始学习整个策略要高效得多。
2.  **神谕策略（Oracle Policy）**：在训练阶段引入一个**拥有特权信息**的模型控制器。这个神谕策略可以访问仿真环境中的真实动力学参数、无噪声状态和精确的执行器模型。因此，它能计算出在给定不确定性下的“理想”控制信号。这个神谕策略是**仅在训练时使用**的“老师”。
3.  **监督损失（Supervised Loss）**：这是本文最关键的创新。传统的残差RL只依赖奖励信号（Reward）来指导残差策略学习。本文在此基础上，增加了一个**监督损失项**，直接让残差策略的输出（即需要补偿的力矩）去模仿神谕策略与基础策略的差值。这个损失项为策略学习提供了明确、直接的梯度信号，极大地提高了学习效率和收敛速度，减少了对复杂奖励函数设计的依赖。
4.  **跨平台泛化性**：在三个差异较大的双足机器人平台（Kangaroo, Unitree H1-2, Bruce）上进行了验证，且使用了相同的训练框架，证明了方法的通用性和可迁移性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整个框架如图1所示，包含三个核心部分：

1.  **基础策略（Base Policy, $\pi_b$）**：一个基于模型的控制器，由DCM轨迹规划器和全身控制器（WBC）组成。它使用有噪声的观测和**不准确的动力学模型**，模拟真实机器人上的情况。
2.  **神谕策略（Oracle Policy, $\pi^*$）**：一个与基础策略结构相同的模型控制器，但在训练时拥有**特权信息**，包括真实动力学参数、无噪声状态等。它能计算出理想的补偿力矩。
3.  **残差策略（Residual Policy, $\pi_r$）**：一个基于循环神经网络（RNN）的强化学习策略。它接收机器人的状态观测，输出一个**残差力矩** $\tau^r_t$。最终施加到机器人上的总力矩为 $\tau_t = \tau^b_t + \tau^r_t$，其中 $\tau^b_t$ 是基础策略的输出。

**训练过程**：在仿真环境中，对机器人动力学、传感器和环境进行域随机化。残差策略 $\pi_r$ 与环境交互，其训练目标由两部分组成：
- **RL目标（$L_{rl}$）**：标准的PPO损失，由奖励函数驱动。奖励函数鼓励跟踪DCM和足部轨迹，以及匹配神谕策略的力矩输出。
- **监督损失（$L_{sup}$）**：直接让残差策略的输出 $\pi_r(s_t)$ 去拟合神谕策略与基础策略的差值 $\tau^*_t - \tau^b_t$。这个损失项是本文的核心创新。

### 3.2 关键算法与数学推导

1.  **DCM轨迹生成**：
    - DCM定义：$\xi = x + b\dot{x}$，其中 $x$ 是质心（CoM）位置，$b = \sqrt{\Delta z / g}$。
    - DCM动力学：$\dot{\xi} = \frac{1}{b}(\xi - v)$，其中 $v$ 是虚拟排斥点（VRP）。
    - 轨迹求解：在一个相位 $\psi$ 内，DCM轨迹 $\xi_\psi(t)$ 由起点VRP $v_{\psi,0}$、终点VRP $v_{\psi,T}$ 和终点DCM $\xi_\psi(T)$ 决定：
      $$\xi_\psi(t) = \alpha_\psi(t) v_{\psi,0} + \beta_\psi(t) v_{\psi,T} + \gamma_\psi(t) \xi_\psi(T)$$

2.  **全身控制（WBC）**：
    - DCM稳定控制律：$v = v_{ref} + (I + bK_\xi)(\xi - \xi_{ref})$，其中 $K_\xi$ 是增益矩阵，$e_\xi = \xi - \xi_{ref}$ 是DCM跟踪误差。
    - 期望CoM力：$F = \frac{m}{b^2}(x - v)$，作为WBC中CoM任务的参考。

3.  **监督损失（Supervised Loss）**：
    - 定义：$L_{sup} = -\log(\pi_r(\tau^*_t - \tau^b_t | s_t))$
    - 这个损失函数鼓励残差策略 $\pi_r$ 在给定状态 $s_t$ 下，为动作 $\tau^*_t - \tau^b_t$（即神谕策略的补偿量）分配更高的概率。本质上是一个**最大似然估计**，让残差策略去模仿神谕策略的“纠错”行为。

4.  **总训练目标**：
    $$L_{total} = \omega_{rl} * L_{rl} + \omega_{sup} L_{sup}$$
    其中 $\omega_{rl}=1, \omega_{sup}=10$，平衡了强化学习探索与模仿学习。

### 3.3 模型、训练与数据集

- **机器人模型**：三个双足机器人：Kangaroo（47kg，12个自由度）、Unitree H1-2（67kg，12个自由度）、Bruce（4.8kg，10个自由度，线足）。
- **仿真环境**：使用MuJoCo物理引擎。
- **策略网络**：
    - **Actor网络**：两层LSTM（每层256个隐藏单元）+ 线性输出层（512个神经元）。使用LSTM处理观测历史，以推断潜在的动力学变化。
    - **Critic网络**：多层感知机（MLP），输入层512个神经元，两个隐藏层各256个神经元。激活函数为ELU。
- **训练设置**：
    - 仿真频率1000Hz，控制频率200Hz。
    - 使用PPO算法。
    - 训练10,000个episode，每个episode对应10秒仿真时间。
    - 使用**域随机化**（Domain Randomization）来模拟真实世界的不确定性，具体参数见表I。随机化范围包括传感器噪声、动力学不确定性（质量、摩擦、阻尼、电机效率、时延）和环境不确定性（地面摩擦）。

### 3.4 实验设置与结果

- **实验问题**：验证神谕策略的有效性（Q1）、残差策略的补偿能力（Q2）、与纯RL和模仿学习（IL）的对比（Q3）、跨平台泛化能力（Q4）。
- **基线方法**：
    - **MBC**：纯模型控制器，使用低通滤波处理噪声。
    - **ResRL**：标准残差RL，不使用监督损失（$\omega_{sup}=0$）。
    - **IL**：纯模仿学习，不使用RL目标（$\omega_{rl}=0$）。
- **评估指标**：成功率、DCM跟踪误差、足部跟踪误差、累计回报（Return）。
- **主要结果**（见表III）：
    - **神谕策略（Oracle）** 在所有指标上表现最佳，证明了其作为监督信号的有效性（Q1）。
    - **基础策略（Base）** 在域随机化下完全失败，证明了其脆弱性。
    - **标准残差RL（ResRL）** 几乎完全失败，表明仅靠奖励信号难以学习力矩级控制。
    - **纯模仿学习（IL）** 在某些机器人上表现不错，但在Bruce上表现较差，说明其对神谕策略的可靠性敏感。
    - **本文方法（BOR）** 在所有三个机器人上均取得了接近神谕策略的性能，成功补偿了基础策略的不足，证明了其鲁棒性和泛化能力（Q2, Q4）。
- **关键发现**：
    - **学习效率**：图2显示，本文方法（BOR）在约1500个episode后成功率就超过80%，而标准残差RL几乎无进展（Q3）。
    - **鲁棒性**：表IV显示，随着不确定性水平$\beta$增加，MBC性能急剧下降，而BOR始终保持100%成功率。
    - **适应性**：图5显示，在不平坦地形上，神谕策略和IL都会失败，而BOR通过RL探索学会了适应性地调整步态，超越了单纯模仿。
    - **消融实验**：表III和图2中，OR（直接学习力矩）与BOR（学习残差力矩）性能相近，说明在基础策略误差很大时，学习残差的优势有限，但整体框架依然有效。

### 3.5 技术细节

- **观测空间设计**：残差策略的观测 $o_t$ 包含噪声状态、基础策略输出、上一时刻动作和跟踪误差，以提供足够信息推断未建模动力学。
- **特权观测**：Critic网络在训练时可以使用包含真实状态和神谕策略输出的特权观测 $o^{privi}_t$，以获得更准确的价值估计，但在部署时策略仅使用 $o_t$。
- **域随机化实现**：通过一个缩放因子 $\beta$ 控制随机化程度，训练时 $\beta=1$，测试时调整 $\beta$ 来评估鲁棒性。
- **神谕策略的建模**：神谕策略不仅知道真实动力学，还知道电机时延 $\Delta t_{delay}$ 和效率 $\alpha_{decay}$，并在计算力矩时考虑这些因素（见算法1第15行）。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了双足机器人力矩级控制中模型不准确与数据效率低下的核心矛盾。
    - 提出了一种新颖的“模型监督”范式，将模型控制器的知识以监督信号的形式高效地注入到强化学习过程中，避免了复杂的奖励工程。
    - 提供了一个可跨平台迁移的通用框架，在三个不同机器人上验证了其有效性。
- **对未来研究/应用的影响**：
    - 为Sim-to-Real迁移提供了一种高效、鲁棒的新思路，有望加速双足机器人从仿真到真实部署的进程。
    - 其“神谕监督”思想可以推广到其他需要结合模型知识与数据驱动的机器人控制任务中，如四足机器人、机械臂操作等。
    - 证明了在强化学习中引入结构化先验知识（如模型控制器）可以显著提升学习效率和最终性能。
- **局限与开放问题**：
    - **对神谕策略的依赖**：神谕策略本身并非完美，在某些高度随机的情况下（如Bruce的线足），神谕策略也可能不稳定，过度模仿会限制性能。作者指出，RL可以帮助探索超越神谕的策略，但这需要平衡。
    - **计算成本**：虽然训练效率高，但框架中同时运行基础策略、神谕策略和残差策略，计算复杂度较高。
    - **真实机器人验证**：本文所有实验均在仿真中进行，虽然使用了域随机化，但真正的Sim-to-Real迁移效果仍需在真实机器人上验证。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比理解“神谕策略”**：可以把神谕策略想象成一个**拥有“上帝视角”的完美教练**。在训练时，这个教练知道所有真实情况（比如机器人关节的真实摩擦力、电机的真实延迟）。他能做出最完美的动作。而我们的学生（残差策略）一开始动作很糟糕。传统的RL是让学生自己瞎摸索，偶尔做对了给个奖励。而本文的方法是让教练直接告诉学生：“你看，在刚才那种情况下，你应该这样纠正你的动作。” 这个直接的指导（监督损失）让学生学得飞快。
- **“残差”的含义**：假设基础策略（一个不太准的模型控制器）给出的力矩是“基础动作”。残差策略学习的是“修正量”。最终动作 = 基础动作 + 修正量。这样，残差策略不需要从头学习如何走路，只需要学习如何“纠错”，大大降低了学习难度。
- **域随机化的作用**：为了让训练出的策略在真实世界中也能工作，我们在仿真中给机器人“制造麻烦”。比如，随机改变它的重量、关节摩擦力、传感器噪声等。这样，策略在训练时就见过各种“坏情况”，到了真实世界，面对类似的不确定性时，就能从容应对。这就像**在风洞中测试飞机**，而不是直接去暴风雨中试飞。
