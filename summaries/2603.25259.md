## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人学，具体为冗余移动机械臂在物理人机交互（pHRI）中的控制。
- **问题（Problem）**：在物理人机交互任务中，如何控制由移动底盘和机械臂组成的冗余移动机械臂，以最小化整个机器人系统（特别是质量最大的移动底盘）的动能，从而提升交互的安全性、平滑性和效率。
- **方法（Method）**：提出一种基于最小能量准则的逆微分运动学（IDK）求解方法。该方法将IDK问题形式化为一个带约束的优化问题，目标是最小化系统总动能，其最优解等价于使用系统惯性矩阵加权的伪逆（即动力学一致的雅可比伪逆）来求解关节速度。
- **创新（Innovation）**：首次将最小化系统总动能作为核心目标，直接用于物理人机交互场景下移动机械臂的实时控制。与传统的基于优先级或伪逆的方法不同，该方法通过引入系统惯性矩阵作为权重，自动、平滑地协调底盘和机械臂的运动，优先减少重载底盘的运动。
- **意义（Significance）**：该方法在实验中显著降低了系统总动能，减少了操作员所需施加的人力，缩短了任务执行时间，并提供了更平滑的用户体验。这对于提高协作机器人的安全性、能效和用户接受度具有重要意义，尤其适用于需要人机紧密协作的工业、康复和辅助任务。
- **一句话总结**：本文提出了一种通过最小化系统总动能来实时控制冗余移动机械臂的逆运动学方法，实验证明该方法在物理人机交互任务中比传统方法更安全、高效和流畅。

## 1. 研究动机（Motivation）

- **解决什么问题**：移动机械臂在执行物理人机交互任务（如协同搬运、插销入孔）时，需要协调移动底盘和机械臂的运动。传统的控制方法（如固定优先级或伪逆法）要么导致底盘过度运动（消耗大量能量且不安全），要么需要用户手动切换操作模式（增加认知负荷和延迟）。本文旨在解决如何自动、最优地分配底盘和机械臂的运动，以提升交互的整体性能。
- **为什么重要**：移动机械臂的底盘通常非常沉重（本文中为115kg），其不必要的运动会储存大量动能，这不仅浪费能量，更在物理人机交互中构成严重的安全隐患（碰撞时冲击力更大）。同时，频繁或剧烈的底盘运动会给操作员带来不舒适和不稳定的体验。因此，找到一种能自动最小化系统动能、特别是底盘动能的方法，对于实现安全、高效、流畅的人机协作至关重要。
- **挑战在哪里**：
    1.  **冗余性**：移动机械臂拥有比任务空间维度（6维）更多的自由度（9维），如何从无穷多解中选择最优解是一个挑战。
    2.  **实时性**：物理人机交互要求控制算法必须在极短时间内（毫秒级）计算出结果，复杂的优化方法（如二次规划）可能无法满足实时性要求。
    3.  **协调性**：需要一种机制来自动、平滑地协调底盘和机械臂的运动，避免突兀的模式切换或过度的底盘移动，同时保证末端执行器能准确跟踪操作员施加的期望速度。

## 2. 创新点（Innovation）

- **关键创新**：提出了一种**基于最小化系统总动能**的逆微分运动学（IDK）求解方法。该方法的核心是将IDK问题形式化为一个带等式约束的优化问题，目标函数是系统总动能（由机械臂和底盘的惯性矩阵加权），约束条件是末端执行器的速度必须等于期望速度。
- **与现有方案的不同**：
    - **传统伪逆法**：使用标准伪逆 $J^\dagger$，平等对待所有关节，不考虑其质量差异，可能导致重载底盘不必要的运动。
    - **优先级控制法（基准方法）**：将任务分为主任务（跟踪速度）和次任务（保持姿态），底盘运动由次任务决定，但需要用户手动切换“操作”和“移动”模式，不智能且不连续。
    - **本文方法**：使用**动力学一致的雅可比伪逆** $J^\dagger_M = M^{-1}J^T(JM^{-1}J^T)^{-1}$。这个伪逆通过系统惯性矩阵 $M$ 进行加权，使得质量越大的部件（如底盘）在运动分配中“代价”越高，从而被自动抑制。这实现了底盘和机械臂运动的**自动、平滑、最优协调**，无需用户干预。
- **为什么能奏效**：该方法的数学本质是求解一个带约束的二次优化问题。其最优解恰好是动力学一致的伪逆。由于惯性矩阵 $M$ 是正定的，该优化问题有唯一解，且解是连续的。因此，它能保证在满足末端速度要求的前提下，系统总动能最小，从而自然地限制了重载底盘的运动。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的控制架构如图2所示，这是一个经典的基于虚拟导纳的物理人机交互框架。

1.  **人机交互接口**：操作员对机器人末端施加力/力矩 $f_h$。
2.  **虚拟导纳模型**：该力/力矩驱动一个虚拟的“质量-阻尼”系统（公式1），生成期望的末端执行器速度 $v_d$。这模拟了操作员在推动一个带有阻尼的虚拟物体。
3.  **逆微分运动学（IDK）求解器**：这是本文的核心贡献。它接收期望的末端速度 $v_d$ 和当前机器人状态 $q$，通过求解一个最小化系统总动能的优化问题，计算出期望的关节速度 $\dot{q}_d$。
4.  **底层控制器**：将计算出的关节速度 $\dot{q}_d$ 发送给移动底盘和机械臂的底层伺服控制器，驱动机器人运动。

### 3.2 关键算法与数学推导

**1. 问题建模**

移动机械臂的雅可比矩阵 $J(q)$ 和关节速度向量 $\dot{q}_d$ 定义为：
$$J(q) = [J_a(q_a) \quad J_b]$$
$$\dot{q}_d = \begin{bmatrix} \dot{q}_{ad} \\ \dot{q}_{bd} \end{bmatrix}$$
其中 $J_a \in \mathbb{R}^{6\times6}$ 是机械臂雅可比，$J_b \in \mathbb{R}^{6\times3}$ 是底盘雅可比。

系统总动能 $E_K$ 为：
$$E_K(q_a, \dot{q}_{ad}, \dot{q}_{bd}) = \frac{1}{2} \dot{q}_{ad}^T M_a(q_a) \dot{q}_{ad} + \frac{1}{2} \dot{q}_{bd}^T M_b \dot{q}_{bd}$$
其中 $M_a(q_a)$ 和 $M_b$ 分别是机械臂和底盘的惯性矩阵。

**2. 优化问题**

**优化问题1**：找到最优的 $\dot{q}_{ad}^*$ 和 $\dot{q}_{bd}^*$ 使得：
$$\min_{\dot{q}_{ad}, \dot{q}_{bd}} E_K(q_a, \dot{q}_{ad}, \dot{q}_{bd})$$
满足约束：
$$J_a(q_a) \dot{q}_{ad} + J_b \dot{q}_{bd} = v_d$$

**3. 最优解推导**

将问题写成紧凑形式。令 $M(q_a) = \begin{bmatrix} M_a(q_a) & 0 \\ 0 & M_b \end{bmatrix}$，则目标函数为 $E_K = \frac{1}{2} \dot{q}_d^T M(q_a) \dot{q}_d$，约束为 $J(q) \dot{q}_d = v_d$。

构建拉格朗日函数：
$$\mathcal{L} = \frac{1}{2} \dot{q}_d^T M(q_a) \dot{q}_d + \lambda^T (v_d - J(q) \dot{q}_d)$$

根据KKT条件，对 $\dot{q}_d$ 求偏导并令其为零：
$$\frac{\partial \mathcal{L}}{\partial \dot{q}_d} = M(q_a) \dot{q}_d - J(q)^T \lambda = 0 \quad \Rightarrow \quad M(q_a) \dot{q}_d = J(q)^T \lambda \quad (16)$$
结合约束 $J(q) \dot{q}_d = v_d$ (17)。

由(16)得 $\dot{q}_d = M(q_a)^{-1} J(q)^T \lambda$，代入(17)：
$$J(q) M(q_a)^{-1} J(q)^T \lambda = v_d \quad \Rightarrow \quad \lambda = (J(q) M(q_a)^{-1} J(q)^T)^{-1} v_d$$

将 $\lambda$ 代回 $\dot{q}_d$ 表达式，得到最优解：
$$\dot{q}_d^* = M(q_a)^{-1} J(q)^T (J(q) M(q_a)^{-1} J(q)^T)^{-1} v_d = J_M^\dagger(q) v_d \quad (12)$$

其中 $J_M^\dagger(q) = M(q_a)^{-1} J(q)^T (J(q) M(q_a)^{-1} J(q)^T)^{-1}$ 就是**动力学一致的雅可比伪逆**。

**4. 完整控制律**

为了利用冗余自由度执行次要任务（如保持一个舒适的构型），最终的控制律在零空间中加入了一个次要任务项：
$$\dot{q}_d = \dot{q}_d^* + N(J(q)) \dot{q}_{d,s} \quad (18)$$
其中 $N(J(q)) = I - J_M^\dagger(q) J(q)$ 是零空间投影矩阵，$\dot{q}_{d,s} = G(q_{des} - q)$ 是一个使机器人趋向于期望构型 $q_{des}$ 的吸引项。

### 3.3 模型、训练与数据集

- **模型**：本文不涉及机器学习模型的训练。其“模型”是机器人系统的物理模型，包括：
    - **运动学模型**：机械臂和底盘的雅可比矩阵 $J_a$ 和 $J_b$。
    - **动力学模型**：机械臂的惯性矩阵 $M_a(q_a)$ 和底盘的惯性矩阵 $M_b$。这些矩阵是已知的或可以通过系统辨识获得。
- **训练**：无。
- **数据集**：实验数据来自27名不同用户执行插销入孔任务。每个用户使用三种控制方法（Locomotion, Switch, Minimum-Energy）各执行一次任务，共产生 $27 \times 3 = 81$ 组实验数据。数据包括末端速度、人施加的力、系统动能、执行时间和最终位置误差。

### 3.4 实验设置与结果

- **实验设置**：
    - **机器人**：UR10e 协作机械臂 + Robotnik RB-KAIROS+ 全向移动底盘。
    - **任务**：插销入孔（Peg-in-hole），如图4所示，需要将末端执行器上的销钉插入墙上的孔中。

        - (c) **Minimum-Energy**：本文提出的方法。
    - **评估指标**：平均动能 $\bar{E}$，平均人力 $\bar{F}_h$，平均执行速度 $\bar{v}$，末端最终位移 $x_f$，执行时间 $T_f$。

- **结果分析**：
    1.  **平均动能 $\bar{E}$**：Minimum-Energy 和 Switch 方法远低于 Locomotion 方法（约低66-69%）。但 Minimum-Energy 方法的动能曲线更平滑（图7），而 Switch 方法在切换到“移动”模式时会出现动能尖峰。
    2.  **平均人力 $\bar{F}_h$**：Locomotion 方法需要最大的人力。Switch 方法最低（比Locomotion低41%），Minimum-Energy 方法居中（比Locomotion低22%）。这表明自动协调比纯底盘移动省力，但比完全手动切换模式稍费力，因为自动协调中底盘仍会进行一些必要的移动。
    3.  **执行时间 $T_f$**：Minimum-Energy 方法最快（中位数28.33秒），比 Locomotion 快11%，比 Switch 快24%。Switch 方法最慢，因为手动切换模式引入了延迟。
    4.  **末端精度 $x_f$**：Minimum-Energy 和 Locomotion 方法精度最高，而 Switch 方法精度较差，因为用户手动切换可能导致最终构型不佳。
    5.  **速度与动能曲线**：从图6和图7可以直观看出，Minimum-Energy 方法的速度和动能曲线最为平滑，没有 Locomotion 方法的剧烈波动和 Switch 方法的零速停顿及能量尖峰，提供了更稳定、舒适的交互体验。

### 3.5 技术细节

- **虚拟导纳参数**：虚拟质量 $M = diag(4, ..., 4)$，虚拟阻尼 $B = diag(75, ..., 75)$。
- **次要任务增益**：$G = diag(1, 1, 1, 1, 1, 1, 0, 0, 0)$，即只对机械臂的6个关节施加吸引，不对底盘的3个虚拟关节施加，避免干扰最小能量优化。
- **力传感器**：FTE-AXIA80，用于测量人施加的力。
- **实时性**：该方法为闭式解，计算量小，满足实时控制要求，这是其相对于基于QP的方法（如[15]中需要4.8秒）的关键优势。

    - 提供了一种计算高效的闭式解，即动力学一致的雅可比伪逆，使其适用于实时控制。
    - 通过大量用户实验，证明了该方法在降低动能、减少人力、缩短时间、提升平滑性和安全性方面均优于或持平于现有方法，特别是自动协调的优势避免了手动切换的认知负担和延迟。

- **对未来研究/应用的影响**：
    - **控制理论**：为冗余机器人系统的优化控制提供了一个新的视角，即从能量角度进行优化，而非仅关注运动学或任务优先级。
    - **人机交互**：该方法能显著提升人机协作的安全性（动能小，碰撞伤害低）和用户体验（运动平滑，无需手动切换），有望推动移动机械臂在工业装配、物流搬运、康复辅助等领域的实际应用。
    - **算法设计**：证明了将物理模型（惯性矩阵）直接融入控制律设计的有效性，为其他类似问题（如最小化关节力矩、能耗等）提供了借鉴。

- **局限与开放问题**：
    - **作者未明确提及**，但可以推断的局限包括：
        1.  需要精确的机器人动力学模型（惯性矩阵），模型误差可能影响性能。
        2.  仅考虑了动能最小化，未考虑其他重要因素，如关节限位、奇异点规避、障碍物避碰等。虽然零空间可以处理次要任务，但主要优化目标仍是单一的。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“动力学一致的雅可比伪逆”？**
    - **通俗解释**：想象一个团队在搬运一个重物。团队里有大力士（底盘，质量大）和普通人（机械臂关节，质量小）。传统的伪逆法会平均分配任务，让大力士和普通人做同样多的功。而“动力学一致的伪逆”就像一个聪明的队长，他知道大力士力气大但移动起来很费劲（动能大），所以他会让大力士尽量少动，让更灵活的普通人多动，只要能把重物搬到目的地就行。这个“费劲”的程度就是用“惯性矩阵”来衡量的。

- **为什么最小化动能能提升安全性？**
    - **类比**：一辆高速行驶的重型卡车（高动能）和一辆缓慢行驶的自行车（低动能）撞到人，后果完全不同。动能越大，碰撞时释放的能量越大，造成的伤害也越大。在机器人交互中，最小化系统动能，特别是底盘这个“重型卡车”的动能，意味着即使发生意外碰撞，机器人能传递给人体的能量也更小，从而更安全。

- **“零空间”是什么？**
    - **比喻**：你的手（末端执行器）需要去够一个杯子（主任务）。你的手臂有7个关节（冗余自由度）。在保证手的位置不变的前提下，你的肘部可以上下、左右移动（零空间运动）。本文的次要任务就是利用这个“零空间”，让机器人保持一个更舒适、更不容易碰到障碍物的姿态，同时不影响手去够杯子的主任务。
