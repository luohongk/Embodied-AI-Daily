# Multimodal Fusion and Interpretability in Human Activity Recognition: A Reproducible Framework for Sensor-Based Modeling

- arXiv: https://arxiv.org/abs/2510.22410v3
- arXiv ID: 2510.22410

---

## 0. 概览（Concise Summary）

- **主题（Topic）**：基于多模态传感器数据的人类活动识别（HAR），具体聚焦于厨房烹饪场景。
- **问题（Problem）**：现有研究缺乏一个标准化、可复现且完整的预处理与融合框架，以处理异构传感器数据（视频、音频、RFID）在采样率、格式和时间结构上的不一致性，并评估不同融合策略及模态贡献的可解释性。
- **方法（Method）**：提出一个模块化的可复现框架，包括统一的数据预处理流水线（重采样、灰度化、滑动窗口分割、标准化），并系统比较了早期融合、晚期融合和混合融合三种策略，使用LSTM模型进行分类。同时，引入PCA/t-SNE进行可解释性分析，并评估了RFID这一稀疏异步模态的附加价值。
- **创新（Innovation）**：1）提供了一个从数据摄取到建模再到解释的完整、可复现的流水线，而非仅关注模型性能；2）系统比较了三种融合策略在统一预处理条件下的表现；3）通过可视化（PCA/t-SNE）和分类性能定量评估了模态贡献；4）探索了RFID这一被忽视的模态在HAR中的潜力。
- **意义（Significance）**：为多模态HAR研究提供了一个透明、模块化且可迁移的模板，有助于解决领域内预处理方法不统一、结果难以复现的问题，并为理解不同模态和融合策略的贡献提供了实证依据。
- **一句话总结**：本文提出了一个用于传感器基人类活动识别的可复现多模态融合框架，通过系统比较融合策略和评估模态贡献，验证了晚期融合和RFID模态的有效性。

## 1. 研究动机（Motivation）

- **解决的问题**：多模态传感器数据（如视频、音频、RFID）在采样率、数据格式、信号结构和语义表示上存在巨大差异。例如，视频为30 FPS，音频为kHz级，RFID为稀疏异步事件。如果没有精心设计的预处理，这些差异会导致时间错位、模态不平衡或信息丢失，严重影响下游模型性能。
- **重要性**：在真实世界的人类活动识别（HAR）场景中，行为是复杂的多感官交互。有效融合异构传感器数据对于准确建模和理解人类行为至关重要。然而，现有研究往往只关注模型或分类器性能，忽视了从数据获取、转换、对齐到融合和解释的完整数据生命周期。
- **挑战**：
    1.  **数据异构性**：不同模态的数据格式、采样率和时间结构不同，难以直接融合。
    2.  **时间对齐**：需要将不同时间尺度的信号精确对齐到同一时间轴上。
    3.  **模态缺失与噪声**：某些模态（如无线IMU）可能存在数据质量差、时间戳不可靠等问题，导致无法使用。
    4.  **融合策略选择**：早期、晚期和混合融合各有优劣，缺乏在统一预处理条件下的系统性比较。
    5.  **可解释性不足**：多模态融合模型常被视为“黑箱”，难以理解各模态对最终决策的贡献。
    6.  **稀疏异步模态**：像RFID这样的模态虽然能提供丰富的物体交互信息，但由于其稀疏和异步特性，在HAR研究中探索不足。

## 2. 创新点（Innovation）

- **完整的可复现流水线**：不同于以往仅关注模型性能的研究，本文提出了一个从原始数据摄取、预处理、对齐、融合到模型训练和可解释性分析的完整、模块化流水线。所有代码和流程都公开，强调可复现性。
- **系统性的融合策略比较**：在统一的预处理条件下，使用LSTM模型对早期、晚期和混合融合三种策略进行了严格的、可复现的对比实验，揭示了不同策略的性能差异（晚期融合最优）。
- **模态贡献的定量与定性评估**：通过PCA/t-SNE可视化（定性）和逻辑回归分类性能（定量），系统地评估了视频、音频及其融合表示对活动识别的贡献，验证了视频模态的主导地位和融合的增益。
- **RFID模态的探索与价值证明**：将稀疏、异步的RFID数据成功整合到融合框架中，并证明其能显著提升识别准确率（超过50%）和ROC-AUC，展示了物体交互线索在HAR中的巨大潜力。
- **数学形式化**：对预处理和融合流水线进行了清晰的数学形式化描述，增强了方法的严谨性和可复现性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出的框架是一个模块化的流水线，主要包括三个阶段：

1.  **预处理与协调（Preprocessing and Harmonization）**：核心目标是解决数据异构性，实现时间对齐。
    - **视频**：使用OpenCV逐帧解码，转换为灰度图（降低计算量），缩放到32x32像素，以30 FPS进行子采样。
    - **音频**：使用`scipy.io.wavfile`读取，转换为单声道，重采样至16 kHz。
    - **RFID**：将时间戳转换为相对于视频开始时间的秒数，构建一个稀疏的二进制对象-标签矩阵。
    - **时间对齐**：所有模态都使用相同的滑动窗口（窗口长度L=75帧，步长S=37帧）进行分割，确保每个窗口内的数据对应同一时间段。窗口内的特征进行Z-score标准化。
    - **无线IMU**：由于数据质量差（时间戳不可靠、采样漂移），在预处理后被排除在融合流水线之外。

2.  **特征变换与编码（Feature Transformation and Encoding）**：将预处理后的数据转换为适合深度学习模型的张量形式。每个窗口被转换为固定形状的张量，最终形成形状为 `[windows, time_steps, features, modalities]` 的4D张量。

3.  **融合与评估（Fusion and Evaluation）**：比较三种融合策略。
    - **早期融合**：在输入层将不同模态的特征向量拼接起来，然后送入一个共享的LSTM模型。
    - **晚期融合**：为每个模态训练一个独立的LSTM模型，在决策阶段（如对softmax概率取平均）进行融合。
    - **混合融合**：使用双分支LSTM架构，每个模态独立处理，然后将中间隐藏状态拼接起来，通过一个全连接层进行最终分类。

### 3.2 关键算法与数学推导

论文对预处理和融合流水线进行了数学形式化。

- **视频预处理**：
  - 灰度化：$G_t = \alpha R_t + \beta G_t + \gamma B_t$，其中 $\alpha=0.299, \beta=0.587, \gamma=0.576$。
  - 缩放与展平：$G'_t = \text{Resize}(G_t; H, W)$，$v_t = \text{vec}(G'_t) \in \mathbb{R}^{1024}$。
  - 处理后的视频流：$V = \{v_t\}_{t=1}^{T_v}$。

- **音频预处理**：
  - 重采样：$x_d(t) = \text{Resample}(x(t); f_d)$，目标采样率 $f_d = 16\text{kHz}$。
  - 时间对齐：$t_a = \lfloor \frac{f_d}{f_v} t_v \rfloor$，其中 $f_v = 30\text{FPS}$。
  - 特征提取（均值幅度）：$a_t = \frac{1}{K} \sum_{k=t_a}^{t_a+K-1} x_d(k)$，$a_t \in \mathbb{R}$。
  - 处理后的音频流：$A = \{a_t\}_{t=1}^{T_v}$。

- **RFID预处理**：
  - 二进制编码：$r_t(c) = \begin{cases} 1, & \text{if tag } c \text{ is active at time } t \\ 0, & \text{otherwise} \end{cases}$。
  - 时间步向量：$r_t = [r_t(1), ..., r_t(C)]^T \in \mathbb{R}^C$。

- **滑动窗口分割**：
  - 对于窗口长度L=75，步长S=37，每个模态被分割为：
    $V^{(w)} = [v_t, v_{t+1}, ..., v_{t+L-1}] \in \mathbb{R}^{L \times 1024}$
    $A^{(w)} = [a_t, a_{t+1}, ..., a_{t+L-1}] \in \mathbb{R}^{L \times 1}$
    $R^{(w)} = [r_t, r_{t+1}, ..., r_{t+L-1}] \in \mathbb{R}^{L \times C}$
  - Z-score标准化：$\hat{X}^{(w)} = \frac{X^{(w)} - \mu_X}{\sigma_X}$

- **融合公式**：
  - **早期融合**：$F^{(w)}_{\text{early}} = [\hat{V}^{(w)} \parallel \hat{A}^{(w)} \parallel \hat{R}^{(w)}] \in \mathbb{R}^{L \times (1024+1+C)}$
  - **晚期融合**：$p_{\text{late}}(y|w) = \frac{1}{M} \sum_{m=1}^{M} p_m(y|w)$，其中M=2（音频+视频）。
  - **混合融合**：$h^{(v)} = f_v(\hat{V}^{(w)})$, $h^{(a)} = f_a(\hat{A}^{(w)})$，$h_{\text{hybrid}} = [h^{(v)} \parallel h^{(a)}]$，$p(y|w) = \text{softmax}(W h_{\text{hybrid}} + b)$。

### 3.3 模型、训练与数据集

- **数据集**：卡内基梅隆大学多模态活动数据库（CMU-MMAC），具体使用Subject 07的Brownie烹饪会话。该数据集包含视频、音频、无线IMU、RFID等多种模态。
- **模型**：
    - **LSTM**：用于早期、晚期和混合融合策略。模型为两层LSTM，128个隐藏单元，后接全连接分类层。
    - **逻辑回归**：用于评估单模态和融合特征的分类性能（二分类：准备 vs. 烹饪）。
    - **随机森林**：用于RFID增强实验（五分类），因其对稀疏、噪声数据鲁棒，并能提供特征重要性。
- **训练**：
    - **LSTM**：PyTorch实现，Adam优化器（lr=0.001, weight decay=1e-5），batch size=32，交叉熵损失。RQ2训练10个epoch，RQ4训练20个epoch，使用早停法（patience=3）。
    - **逻辑回归**：`lbfgs`求解器，L2正则化，`C=1.0`，`class_weight='balanced'`。
    - **随机森林**：300棵树，最大深度20，Gini不纯度，`class_weight='balanced'`。

### 3.4 实验设置与结果

- **RQ1（数据对齐）**：成功将视频和音频对齐，生成289个窗口，每个窗口75个时间步，1025个特征（1个音频+1024个视频像素）。无线IMU因数据质量问题被排除。可视化（图1-3）验证了预处理和对齐的有效性。
- **RQ2（融合策略比较）**：
    - **结果**：**晚期融合**在验证集上取得了最高的准确率。**混合融合**性能次之，**早期融合**性能最差。
    - **分析**：晚期融合通过独立学习每个模态的时序模式，避免了早期融合中可能出现的模态间特征干扰和维度灾难问题。
- **RQ3（可解释性与模态贡献）**：
    - **PCA/t-SNE可视化**：融合特征的2D投影显示出清晰的时序结构，能够区分“准备”和“烹饪”阶段，表明融合表示具有可解释性。
    - **逻辑回归分类**：**视频模态**的分类性能远高于**音频模态**，表明视频携带更强的判别信息。**融合特征**的性能优于任何单一模态，证明了多模态融合的增益。
- **RQ4（RFID模态价值）**：
    - **结果**：在双模态（音频+视频）基础上加入RFID后，**准确率提升了超过50%**，**宏平均ROC-AUC**也显著提高。
    - **分析**：RFID提供的物体交互线索（如拿起碗、搅拌器等）与视觉和听觉信息互补，极大地增强了活动识别的鲁棒性和准确性。

### 3.5 技术细节

- **数据来源**：CMU-MMAC数据库（Subject 07, Brownie session）。
- **编程语言与库**：Python, NumPy, pandas, scikit-learn, PyTorch, OpenCV, matplotlib。
- **硬件**：单个NVIDIA T4 GPU。
- **可复现性**：所有代码和流程通过Zenodo公开（DOI: 10.5281/zenodo.16334410）。
- **无线IMU的排除**：由于时间戳不可靠、采样漂移等问题，无线IMU数据无法与视频/音频对齐，因此被排除在主要分析之外。这强调了数据质量评估的重要性。

## 4. 意义与影响（Significance and Impact）

- **解决的问题**：
    1.  提供了一个解决多模态HAR中数据异构性和时间对齐问题的标准化、可复现的框架。
    2.  通过系统比较，明确了晚期融合在给定条件下的优越性。
    3.  定量和定性地揭示了视频模态的主导地位以及多模态融合的增益。
    4.  证明了RFID这一稀疏异步模态在HAR中的巨大潜力，为未来研究开辟了新方向。

- **贡献**：
    1.  **方法论贡献**：提供了一个模块化、可迁移的HAR流水线模板，强调预处理和可复现性，有助于提升领域研究的严谨性。
    2.  **实证贡献**：基于CMU-MMAC数据集，提供了关于融合策略、模态贡献和RFID价值的实证证据。
    3.  **实践贡献**：为开发部署在复杂真实环境（如智能厨房、智能教室）中的智能系统提供了指导。

- **对未来研究/应用的影响**：
    - 研究者可以基于此框架，轻松替换或添加新的模态、融合策略或模型。
    - 强调了数据预处理和模态贡献分析的重要性，而非仅仅追求模型精度。
    - 为整合更多稀疏、异步的传感器（如智能家居设备）提供了思路。

- **局限与开放问题**：
    1.  **单主体、单任务**：研究仅基于一个主体（Subject 07）的一个烹饪任务（Brownie），泛化性有待验证。
    2.  **无线IMU数据不可用**：一个重要的运动模态因数据质量问题被排除，限制了框架的完整性。
    3.  **活动标签**：RQ3和RQ4中的活动标签是手动分配的（准备 vs. 烹饪，或五个粗粒度类别），缺乏细粒度的、语义丰富的标注。
    4.  **模型简单**：使用的LSTM和随机森林模型相对简单，未探索更先进的Transformer或图神经网络架构。
    5.  **RFID处理**：对RFID的处理（均值池化）较为简单，可能丢失了时序信息。

## 5. 澄清与简化（Clarifications and Simplifications）

- **多模态融合类比**：可以想象成一场音乐会。
    - **视频**：是舞台上的视觉表演（如指挥家的动作、乐器的位置），提供空间和结构信息。
    - **音频**：是演奏的音乐本身，提供精细的时序和频率信息。
    - **RFID**：是后台工作人员记录“哪个乐器在什么时候被使用了”的清单，提供物体交互的上下文。
    - **早期融合**：相当于在音乐开始前，就把所有乐谱、灯光设计图和道具清单揉成一团，让一个指挥家（模型）同时处理所有信息。这可能导致信息混乱。
    - **晚期融合**：相当于让三个独立的专家（模型）分别分析视觉、听觉和道具清单，然后投票决定这是哪首曲子。这通常更稳健，因为每个专家都专注于自己的领域。
    - **混合融合**：相当于让视觉和听觉专家先进行内部讨论，产生一个初步理解，然后再与道具清单专家交流，最终形成判断。这允许不同层次的信息交互。

- **PCA/t-SNE的作用**：想象你有一堆高维数据点（比如每个窗口的1025个特征），无法直接可视化。PCA和t-SNE就像两个“地图绘制师”，它们试图将这些高维点投影到一张2D地图上，同时尽量保持点与点之间的“距离关系”。如果地图上相同颜色的点（代表同一活动）聚在一起，就说明原始的高维特征能够很好地区分不同活动，即模型学到了有意义的、可解释的表示。

- **为什么晚期融合更好？**：在早期融合中，模型必须同时学习视频的时空模式和音频的时序模式，这非常困难。不同模态的特征尺度、噪声模式都不同，直接拼接可能导致模型被某个模态主导或学习到无用的联合特征。晚期融合让每个模态的LSTM先独立学习其最擅长的时序模式，然后再简单地对它们的“意见”（概率）进行平均。这避免了模态间的干扰，使得每个模型都能更专注、更有效地学习。
