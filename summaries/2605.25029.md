## 0. 概览（Concise Summary）

- **主题（Topic）**：端到端自动驾驶泊车，结合强化学习（RL）与人类反馈。
- **问题（Problem）**：现有自动驾驶泊车方法面临挑战：模仿学习依赖大量高质量专家数据且泛化性差；传统强化学习训练效率低、探索困难，且模拟器与真实世界存在视觉域差（Sim-to-Real gap），导致策略难以直接部署到实车。
- **方法（Method）**：提出一个名为 **ParkingWorld** 的框架，核心是**校正循环样本高效强化学习（CIL-SERL）**。该方法在一个高保真的**3D高斯泼溅（3DGS）模拟器**中训练，并引入**多级回放缓冲区**机制，将失败的RL探索轨迹与成功的人类校正轨迹配对存储，让策略从“错误”中学习。
- **创新（Innovation）**：
    1.  构建了**交互式3DGS模拟器**，实现了真实停车场景的数字孪生，支持多视角渲染和人机交互，有效弥合了Sim-to-Real的视觉差距。
    2.  设计了**CIL-SERL**机制，核心是“错误笔记本”式的**多级回放缓冲区**，将失败轨迹与对应的成功校正轨迹耦合，通过结构化采样进行针对性学习。
- **意义（Significance）**：提出了一种实用、高效的端到端自动驾驶泊车解决方案。通过在3DGS模拟器中进行校正学习，显著提升了泊车成功率、安全性和效率，并成功实现了从仿真到真实车辆的零样本迁移，为机器人学习中的Sim-to-Real问题提供了新的有效范式。
- **一句话总结**：本文通过在高保真3DGS模拟器中引入人类校正经验，提出了一种样本高效的强化学习框架，让自动驾驶泊车策略能从自身的错误和人类的修正中学习，从而在仿真和实车测试中均取得优异性能。

## 1. 研究动机（Motivation）

- **解决的问题**：端到端自动驾驶泊车策略的训练与部署难题。
- **为什么重要**：自动泊车是自动驾驶的关键功能，要求车辆在狭窄、复杂的环境中精确、安全地完成操作。一个可靠、高效的泊车系统对于提升自动驾驶的实用性和用户体验至关重要。
- **核心挑战**：
    1.  **Sim-to-Real 域差**：传统模拟器（如CARLA, LGSVL）渲染的图像与真实世界存在显著视觉差异，导致在模拟器中训练好的策略无法直接部署到真实车辆上（如图2所示）。
    2.  **强化学习训练效率低**：在复杂的泊车场景中，RL智能体需要大量试错才能学到有效策略，探索过程低效，甚至可能无法收敛。
    3.  **模仿学习的局限性**：虽然模仿学习可以从专家演示中学习，但它需要海量、高质量的专家数据，且对未见过的场景泛化能力差。
    4.  **人类反馈的利用方式**：如何有效地将人类专家的稀疏干预（而非全量标注）融入RL训练，以纠正错误探索并加速学习，是一个关键问题。

## 2. 创新点（Innovation）

- **关键创新**：提出了一个结合**高保真3DGS模拟器**与**校正循环样本高效强化学习（CIL-SERL）** 的端到端泊车框架。
- **与现有方案的不同**：
    - **相比传统模拟器**：使用3DGS构建真实场景的数字孪生，提供照片级真实感的视觉输入，从根本上解决了Sim-to-Real的视觉域差问题，实现了零样本迁移。
    - **相比标准RL**：不依赖随机探索，而是引入人类校正作为“纠错信号”。CIL-SERL不是简单地将人类数据混入缓冲区，而是**结构化地**将“错误”的RL轨迹与“正确”的人类校正轨迹配对存储，形成“错误笔记本”，让策略能明确学习到“错在哪里”以及“如何改正”。
    - **相比模仿学习**：不需要大量专家演示，只需要在策略失败时进行稀疏的、针对性的干预，样本效率更高，泛化能力更强。
- **为什么能奏效**：
    1.  **高保真环境**：3DGS模拟器提供了近乎真实的视觉反馈，使得在仿真中学习的策略可以直接应用于真实世界。
    2.  **针对性学习**：多级回放缓冲区机制（图1右侧）将训练数据分为“正常RL”、“正常人类”、“失败RL”和“校正轨迹”。在采样时，算法会**成对地**从“失败RL”和其对应的“校正轨迹”中采样，迫使策略反复对比错误与正确行为，从而高效地学习如何避免失败并从失败中恢复。这种结构化采样比随机混合所有数据更有效。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整个系统架构如图1所示，分为两个主要部分：**交互过程**（左侧）和**模型更新**（右侧）。

1.  **交互式3DGS模拟器**：
    -   使用手持设备（XGRIDS Lixel K1）采集真实停车场数据，通过LIVO算法生成厘米级精度的彩色点云，再通过3DGS算法重建为场景级高斯表示（$G_s$）。
    -   从3DRealCar数据集引入车辆级3DGS模型（$G_v$），并将其融合到场景中（$G = G_s \cup G_{v \rightarrow s}$），实现动态车辆渲染。
    -   模拟器支持：多视角相机渲染、基于运动学的车辆动力学、碰撞检测、规则路径规划、人机交互干预、状态回滚记录等功能，构成了一个完整的闭环训练环境（图3）。

2.  **端到端RL网络架构**（图4）：
    -   **感知模块**：多视角RGB图像输入，通过**Lift-Splat-Shoot (LSS)** 范式生成BEV特征（$F^{bev}_t$）。然后通过多个任务头（Head）提取占用特征（$F^{occ}_t$）、停车位特征（$F^{slot}_t$）和目标车位特征（$F^{tar}_t$）。
    -   **Actor（策略网络）**：将上述感知特征拼接成策略特征 $z^\pi_t$，输入到SAC Actor中，输出连续控制动作 $a_t = [\delta_t, v_t]$（方向盘转角、速度）。
    -   **Critic（价值网络）**：使用**特权信息**。它通过一个**自动编码器**将模拟器提供的真实状态（ground-truth）$G_t$ 编码为潜在特征 $z^q_t$。Critic将 $z^q_t$ 与动作嵌入 $z^a_t$ 结合，估计软Q值。这种设计让Actor只依赖可获取的感知特征，而Critic利用更丰富的特权信息进行稳定学习。
    -   **层归一化（Layer Normalization）**：在策略和价值网络中引入层归一化，以控制特征尺度，防止因高质量人类干预样本导致Critic过估计。

3.  **CIL-SERL训练机制**：
    -   **核心思想**：将RL的失败探索与人类的成功校正配对，形成“错误笔记本”。
    -   **流程**（图1底部时间线）：
        1.  **RL Rollout**：策略自主执行，轨迹暂存于临时缓冲区 $T^{rl}_{pend}$。
        2.  **检测失败**：若发生碰撞、超时或越界，则标记该段为失败轨迹 $T^-_k$。
        3.  **回滚（Rollback）**：模拟器状态回滚到失败前的检查点 $\xi_c$。
        4.  **人类校正**：人类接管控制，进行成功泊车，产生校正轨迹 $T^+_k$（包含人类控制 $T^+_{k,h}$ 和RL控制 $T^+_{k,rl}$ 部分）。
        5.  **存储**：将失败的RL轨迹 $T^-_k$ 和成功的校正轨迹 $T^+_k$ 作为一个**校正区域** $B^{corr}_k$ 存储。
    -   **多级回放缓冲区**：包含正常RL缓冲区 $B^{rl}_N$、正常人类缓冲区 $B^{h}_N$ 和多个校正区域 $\{B^{corr}_k\}$。
    -   **配对采样（Pair Sampling）**：训练时，以一定概率从“正常区域”或某个“校正区域”采样。若从校正区域采样，则**必须**从该区域的失败轨迹 $B^{fail}_{k,rl}$ 和校正轨迹 $B^{fix}_{k,rl} \cup B^{fix}_{k,h}$ 中各取一条，组成一个“失败-校正”对。整个批次由 $B/2$ 个这样的对组成。这使得Critic和Actor能反复看到错误行为及其修正。

### 3.2 关键算法与数学推导

- **3DGS渲染**：
    - 每个高斯原语表示为 $G_i(x) = \exp\left(-\frac{1}{2}(x-\mu_i)^T \Sigma_i^{-1} (x-\mu_i)\right)$，其中 $\mu_i$ 是均值，$\Sigma_i$ 是协方差矩阵。
    - 投影到2D图像平面后的协方差为 $\Sigma'_i = J_i W_i \Sigma_i W_i^T J_i^T$，其中 $W_i$ 是视图变换矩阵，$J_i$ 是透视投影的雅可比矩阵。
    - 最终像素颜色通过alpha合成得到：$C(p) = \sum_{i \in N(p)} T_i \alpha_i(p) c_i$，其中 $T_i = \prod_{j < i} (1 - \alpha_j(p))$。

- **SAC更新目标**：
    - 软目标值计算：
      $$y_t = r_t + \gamma(1-d_t) \left[ \min_{i=1,2} Q_{\bar{\omega}_i}(z^q_{t+1}, \phi_a(a'_{t+1})) - \alpha \log \pi_\theta(a'_{t+1}|z^\pi_{t+1}) \right]$$
    - Critic损失函数（最小化TD误差）：
      $$J_Q(\omega_i) = \mathbb{E}_{\tau_t \sim \mathcal{B}} \left[ (Q_{\omega_i}(z^q_t, \phi_a(a_t)) - y_t)^2 \right]$$
    - Actor损失函数（最大化Q值与熵）：
      $$J_\pi(\theta) = \mathbb{E}_{s_t \sim \mathcal{B}, a^\pi_t \sim \pi_\theta} \left[ \alpha \log \pi_\theta(a^\pi_t | z^\pi_t) - \min_{i=1,2} Q_{\omega_i}(z^q_t, \phi_a(a^\pi_t)) \right]$$
    - 自动编码器重建损失（L1损失）：
      $$J_{AE}(\psi) = \mathbb{E}_{\tau_t \sim \mathcal{B}} \left[ \| G_t - D_\psi(E_\psi(G_t)) \|_1 \right]$$
    - 总优化目标：$J = \sum_{i=1}^2 J_Q(\omega_i) + J_\pi(\theta) + \lambda_{AE} J_{AE}(\psi)$

- **配对采样**：
    - 采样区域权重：$w_{normal} = |B^{rl}_N| + |B^{h}_N|$，$w_k = |B^{fail}_{k,rl}| + |B^{fix}_{k,rl}| + |B^{fix}_{k,h}|$。
    - 选择区域 $o$ 的概率：$P(o) = w_o / \sum_{\bar{o} \in \Omega} w_{\bar{o}}$。
    - 若选中校正区域 $k$，则采样一个失败-校正对：$(\tau^-, \tau^+) \sim \mathcal{U}(B^{fail}_{k,rl}) \times \mathcal{U}(B^{fix}_{k,rl} \cup B^{fix}_{k,h})$。

### 3.3 模型、训练与数据集

- **模型**：基于SAC的端到端模型。感知部分使用LSS网络，Actor和Critic使用多层感知机（MLP）。Critic包含一个额外的自动编码器分支。
- **训练**：
    - 从预训练的REAP-SAC权重开始初始化。
    - 使用CIL-SERL机制进行训练，人类操作员通过鼠标和键盘进行干预。
    - 默认批量大小32，在NVIDIA RTX 4090上训练。
- **数据集**：
    - **模拟环境**：5个重建的3DGS停车场场景，包含239个标准停车位。
    - **真实世界**：长安CS55实车平台，配备4个环视鱼眼相机。

### 3.4 实验设置与结果

- **仿真实验**（表I）：
    - **对比方法**：规则规划器（RS Curve, Hybrid A*）、端到端方法（ParkingE2E, REAP-SAC, ParkingHIL）。
    - **指标**：泊车成功率（PSR↑）、碰撞率（PCR↓）、超时率（PTR↓）、越界率（PBR↓）、换挡次数（NGS↓）。
    - **结果**：**ParkingWorld在所有指标上均表现最佳**，PSR达到88.0%，远高于其他方法，且PCR仅为4.0%，PBR为0.0%。这表明从校正失败中学习显著提升了性能。

- **实车实验**（表II）：
    - **部署**：策略在车载计算机上以5Hz运行，预测的轨迹通过NMPC控制器以50Hz跟踪执行（图6）。
    - **结果**：**ParkingWorld在实车上同样表现最佳**，PSR达到80%，PCR为5.0%，PTR为15.0%。这证明了CIL-SERL框架的有效性和从仿真到现实的零样本迁移能力。

### 3.5 技术细节

- **人类校正动作转换**：人类通过键盘输入，系统通过逆运动学 $\Phi_{ik}(q_t, q_{t+1}; L, \Delta t)$ 将键盘命令转换为与策略相同的连续动作空间 $[\tilde{\delta}_t, \tilde{v}_t]$。
- **奖励设计**（附录A）：结合稀疏奖励（成功/碰撞/超时/越界）和密集奖励（IoU重叠度、软碰撞惩罚、卡住惩罚、时间成本），引导策略安全、高效地完成泊车。
- **训练动态**（附录B）：人类校正训练初期能快速提升性能，但后期可能因人类干预的离散性导致过拟合，性能略有下降。作者指出使用更连续的控制设备（如方向盘）可能进一步提升性能。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  成功解决了自动驾驶泊车中**Sim-to-Real视觉域差**问题，通过构建3DGS数字孪生模拟器，实现了策略的零样本实车部署。
    2.  提出了一种新颖的**人类校正经验利用方式**（CIL-SERL），通过结构化的“错误笔记本”和配对采样，让RL策略能高效地从失败中学习，显著提升了样本效率和最终性能。
    3.  在仿真和实车平台上均取得了**最先进的泊车性能**，证明了该方法的实用性和有效性。

- **对未来研究/应用的影响**：
    - 为**机器人学习中的Sim-to-Real问题**提供了一个强大的新范式，即利用高保真重建环境进行训练，可以大大降低对真实世界数据的依赖。
    - **CIL-SERL机制**可以推广到其他需要精细操作和人类指导的机器人任务中，如机械臂操作、无人机导航等。
    - 展示了**人类与RL协同**的有效方式：人类不是替代RL，而是在RL最需要帮助的时候（失败时）提供关键的“纠错”信号，这是一种高效、实用的人机协作范式。

- **局限与开放问题**：
    - **人类干预的离散性**：当前使用鼠标和键盘进行干预，动作较为离散，可能导致后期策略过拟合。作者建议使用更连续的控制设备（如方向盘）来改善。
    - **性能峰值后的下降**：在持续训练后，性能可能达到峰值后略有下降，表明需要更精细的训练调度或正则化策略。
    - **未来工作**：计划结合**世界-动作模型（World-Action-Model）**，让策略能预测未来场景和车辆状态的演变，进行“想象式”的推演，从而进一步减少对人类校正的依赖。

## 5. 澄清与简化（Clarifications and Simplifications）

- **3D高斯泼溅（3DGS）**：可以想象成用无数个微小的、半透明的、可调节颜色和形状的“3D小面团”来拼凑出一个场景。这些小面团的位置、大小、颜色和透明度都可以通过学习来优化，最终能渲染出极其逼真的图像。相比传统网格模型，它更擅长处理复杂的反射和透明物体。
- **CIL-SERL的“错误笔记本”**：想象一个学生做数学题。他先自己做（RL Rollout），做错了（失败）。老师（人类）不直接给他答案，而是把时间倒回到他犯错之前（Rollback），然后示范一遍正确的解题步骤（Human Correction）。这个学生就把自己做错的步骤和老师正确的步骤**钉在一起**，放进一个专门的“错题本”（Correction Buffer）里。下次复习（训练）时，他专门看这些“错题-正解”配对，就能更深刻地理解错误原因，避免再犯。而普通的RL训练，就像学生只看自己做对的题，或者把做对的题和做错的题混在一起看，学习效率自然低。
- **配对采样（Pair Sampling）**：就是每次从“错题本”里学习时，必须同时拿出“错误步骤”和“正确步骤”来
