# Grammar of the Wave: Towards Explainable Multivariate Time Series Event Detection via Neuro-Symbolic VLM Agents

- arXiv: https://arxiv.org/abs/2603.11479v2
- arXiv ID: 2603.11479

---

## 0. 概览（Concise Summary）

- **主题（Topic）**：多变量时间序列事件检测（Multivariate Time Series Event Detection, TSED），特别是语言引导的零样本/少样本事件检测。
- **问题（Problem）**：传统TSED依赖大量密集标注数据，但在工业和高风险领域（如能源、气候）中，获取此类标注成本高昂。同时，事件通常由自然语言描述定义，具有复杂的内部时序逻辑结构，现有方法难以在无标注或少量标注数据下，将文本描述精确地定位到信号中的具体区间。
- **方法（Method）**：提出**事件逻辑树（Event Logic Tree, ELT）**，一种将文本事件描述转化为结构化时序逻辑的知识表示框架。并基于此构建**SELA**，一个神经符号视觉语言模型（VLM）智能体系统，该系统通过VLM智能体迭代地将ELT中的原始信号模式（Primitives）从信号可视化中定位出来，并在ELT约束下进行组合，最终输出事件区间和可解释的树状证据。
- **创新（Innovation）**：1）首次提出“语言引导的时间序列事件检测”任务设定。2）提出ELT，将非结构化的自然语言事件描述转化为结构化的、可实例化的时序逻辑树，实现了事件结构的层次化、语义量化和拓扑弹性。3）构建SELA，一个结合了符号逻辑（ELT）与神经感知（VLM）的智能体系统，实现了零样本事件检测与可解释推理。
- **意义（Significance）**：解决了在标注数据稀缺的现实场景下，利用专家知识（文本描述）进行精确事件检测的难题。SELA不仅提供了高精度的检测结果，还生成了可审计、可验证的树状解释，增强了模型在高风险领域的可信度。同时，发布的KITE基准数据集为未来研究提供了标准评估平台。
- **一句话总结**：本文通过提出事件逻辑树（ELT）和神经符号VLM智能体系统SELA，在零样本条件下，成功地将自然语言事件描述精确、可解释地定位到多变量时间序列中。

## 1. 研究动机（Motivation）

**研究动机**：在能源、气候等高风险领域，时间序列事件检测至关重要。然而，现有方法面临两大核心挑战：

1.  **标注数据稀缺**：获取密集、精确的事件区间标注（如“压力测试”中的“失封”事件）成本极高，需要领域专家逐段手动标注，这在工业实践中难以大规模实现。因此，需要一种能在少量甚至零标注数据下工作的检测方法。
2.  **事件定义的语义复杂性**：事件并非简单的统计异常，而是由自然语言定义的、具有内部时序逻辑结构的复杂过程。例如，“压力先缓慢恢复，然后快速反弹，同时体积保持稳定”这样的描述，包含了多个子模式（原始信号模式）及其之间的时序关系（如“同时”、“然后”）。现有方法（如CNN、Transformer）通常将事件视为一个整体标签，无法利用这种结构化的语义信息，也难以提供可解释的证据。

**为什么重要**：解决此问题可以极大地降低事件检测系统在工业界的部署门槛。工程师只需提供操作手册中的文本描述，系统即可自动识别事件，无需大量标注工作。同时，可解释的输出（如“因为通道A在时间X到Y出现了‘尖峰’，随后通道B在时间Y到Z出现了‘下降’，所以这是一个‘失封’事件”）对于高风险决策至关重要，能帮助专家验证和信任模型。

**挑战在哪里**：核心挑战在于**“接地”（Grounding）**——如何将非结构化的、包含时序逻辑的自然语言描述，精确地映射到连续、高维的多变量信号区间上。这要求模型既能理解语言的语义，又能感知信号的形态，并处理两者之间的复杂对应关系。

## 2. 创新点（Innovation）

本文的关键创新在于提出了一个**神经符号（Neuro-Symbolic）** 框架，将符号化的知识表示与神经网络的感知能力相结合，解决了语言引导的零样本事件检测问题。

1.  **提出事件逻辑树（Event Logic Tree, ELT）**：
    -   **新方法**：ELT是一种知识表示框架，它将自然语言事件描述“翻译”成结构化的树状时序逻辑。树的叶子节点是**原始信号模式（Primitives）**（如“压力上升”、“体积稳定”），内部节点是**时序逻辑算子**（如SEQ（顺序）、SYNC（同步）、GUARD（包含）、OR（或））。
    -   **与现有方案的不同**：不同于SAX、Shapelets等仅关注信号形态的符号表示，ELT直接源于语言，并能显式建模事件内部的层次化时序结构。它比Chronicle系统更灵活，支持语义量化（而非二元真值）和拓扑弹性（不依赖绝对时长）。
    -   **为何奏效**：ELT将复杂的“文本到信号”接地问题分解为两个更简单的子问题：1）**解析**：将文本解析为结构化的逻辑树（符号推理）；2）**实例化**：将树中的每个原始信号模式在信号中定位（神经感知）。这种分解降低了问题难度，并使得推理过程透明、可解释。

2.  **构建SELA多智能体系统**：
    -   **新方法**：SELA是一个基于ELT的神经符号VLM智能体系统。它包含两个核心智能体：**逻辑分析师（Logic Analyst）** 负责将文本描述解析为ELT模式（Schema）；**信号检查员（Signal Inspector）** 负责在时间序列可视化中定位原始信号模式，并提交实例化信息。
    -   **与现有方案的不同**：不同于直接使用LLM/VLM处理数值或图像（如VL-Time），SELA通过ELT作为中间表示，引导VLM进行结构化推理。VLM不再需要直接预测事件区间，而是专注于识别更简单的、局部的原始信号模式，这大大降低了VLM的推理难度和幻觉风险。
    -   **为何奏效**：1）**分工明确**：逻辑分析师专注于语言理解，信号检查员专注于视觉感知，各司其职。2）**结构化引导**：ELT为信号检查员提供了明确的“搜索目标”（原始信号模式）和“组合规则”（时序算子），使其推理过程有章可循。3）**置信度传播**：通过模糊逻辑算子，将原始信号模式的置信度自底向上传播，最终得到事件级别的置信度，使得任何接地失败（如原始模式未找到、时序关系不满足）都能被显式追踪。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

SELA系统的整体架构如图3所示，其核心流程如下：

1.  **解析（Parsing）**：**逻辑分析师**智能体接收文本事件描述（如“在‘建立’事件中，体积应保持稳定，同时压力应缓慢恢复或快速反弹”），将其解析为一个**ELT模式（Schema）**。这是一个不依赖具体信号样本的树状结构，定义了事件的逻辑骨架。
2.  **实例化（Instantiation）**：**信号检查员**智能体接收ELT模式和待分析的时间序列。它通过主动使用可视化工具（如放大、缩小、添加标记）来观察信号，并逐个为ELT模式中的**原始信号模式节点**（如“体积稳定”、“压力缓慢恢复”）在信号上定位其发生的区间，并给出一个0到1之间的**语义一致性得分（Semantic Coherence Score）**。
3.  **组合与传播（Composition & Propagation）**：SELA的后端逻辑引擎接收信号检查员提交的原始模式实例化信息（区间和得分）。然后，它根据ELT模式中的时序算子，自底向上地组合这些原始模式实例。例如，对于`SEQ(原始模式A, 原始模式B)`，引擎会检查A的区间是否在B的区间之前，并计算组合后的置信度。最终，根节点代表整个事件，其区间和置信度由所有子节点组合计算得出。
4.  **输出**：SELA输出一个**实例化的证明树（Instantiated Proof Tree）**，其中包含了事件区间、全局置信度，以及每个原始信号模式的区间和得分，提供了完整的、可审计的证据链。

### 3.2 关键算法与数学推导

**1. ELT模式定义**

一个ELT模式是一个树，节点分为原始节点和复合节点：

-   **原始节点**：$n_p = \langle \tau, c \rangle$，其中 $\tau$ 是原子形态描述（如“sharp increase”），$c$ 是物理通道（如“Pressure”）。
-   **复合节点**：$n_\phi = \langle [n_1, ..., n_k], op \rangle$，其中 $op \in \{SEQ, SYNC, GUARD, OR\}$。

**2. ELT实例化**

给定时间序列 $X$，ELT模式被实例化：

-   **原始实例**：$\hat{n}_p = \langle n_p, I, \mu \rangle$，其中 $I = [t_{on}, t_{off}]$ 是时间区间，$\mu = M_\tau(X, I, c) \in [0,1]$ 是语义一致性得分，由某个度量函数 $M_\tau$ 计算。
-   **复合实例**：复合节点的区间覆盖其所有子节点的区间。其置信度通过模糊逻辑算子递归聚合。

**3. 归一化置信度**

当多个互斥的事件候选竞争同一区域时，使用归一化根置信度进行比较，以避免语义更丰富的树（包含更多合取证据）受到不公平的惩罚：

$$
\mu_{norm}(\hat{T}) = \mu_{root}(\hat{T})^{1 / K(\hat{T})}
$$

其中 $K(\hat{T}) = |L^*(\hat{T})|$ 是实例化树中活跃的原始叶子节点数量。

### 3.3 模型、训练与数据集

-   **模型**：SELA系统本身不进行传统意义上的训练。其核心组件是**VLM智能体**（如GPT-4.1, GPT-5.4），通过提示工程（Prompt Engineering）来驱动。逻辑分析师和信号检查员都是基于同一个VLM，但拥有不同的角色提示和动作空间。
-   **训练**：无训练（Zero-shot）。SELA不依赖任何标注数据进行模型参数更新。对比的监督基线模型（CNN, Transformer, Timer, Moment, Chronos）则使用KITE数据集的部分标注进行微调。
-   **数据集**：本文发布了**KITE（Knowledge-Infused Time Series Events）** 基准数据集，包含三个子数据集：
    -   **Pressure Test**：来自石油天然气行业的62个压力测试样本。
    -   **Liner Hanger**：来自石油天然气行业的29个衬管悬挂器安装样本。
    -   **Climate Hazard**：来自气象领域的78个气候灾害样本（如浓雾、极寒）。
    所有数据都包含专家验证的事件描述和事件区间标注。

### 3.4 实验设置与结果

**实验设置**：主要回答两个研究问题：
-   **RQ1**：LLM/VLM在理解自然语言事件描述的结构方面表现如何？
-   **RQ2**：ELT是否能提升VLM智能体在时间序列事件检测中的性能？

**RQ1结果（表3）**：评估了多个LLM将文本解析为ELT模式的能力。所有模型在有效解析中都能达到完美的原始模式准确性（Primitive Accuracy = 1.00）。GPT-4.1在结构保真度（Structural Fidelity）上表现最好，而GPT-5.4在语义覆盖度（Semantic Coverage）上更优，但倾向于过度分解，导致拓扑结构略有偏差。

**RQ2结果（表4）**：SELA在三个KITE数据集上，在严格的定位指标（F1@0.9）上显著优于所有基线方法，包括监督学习模型、零样本/少样本LLM/VLM基线（Numeric, VL-Time）。例如，在Pressure Test数据集上，SELA（GPT-5.4）的F1@0.5达到77.08，而最强的VL-Time基线仅为49.61。SELA的性能仅次于人类数据科学家。

**消融实验（图4）**：移除ELT结构后，SELA的性能在所有数据集上均大幅下降。在Pressure Test上，GPT-5.4的F1@0.5从77.1骤降至27.8，证明了ELT对于提供全局事件结构和可靠置信度传播至关重要。

### 3.5 技术细节

-   **智能体交互**：逻辑分析师和信号检查员通过结构化的函数调用（Function Call）和类型化工件（Typed Artifacts）与后端交互。逻辑分析师提交“模式工件”（Schema Artifact），信号检查员提交“实例化工件”（Instantiation Artifact）。
-   **可视化工具**：信号检查员拥有主动可视化工具，如`view_window`（查看特定区间）、`draw_markers`（绘制标记线），使其能够聚焦于信号的关键部分，克服了静态可视化（如VL-Time）的精度损失问题。
-   **逻辑引擎**：后端逻辑引擎维护三个耦合状态：时间序列数据、ELT模式和当前原始模式实例化。它负责验证实例化、执行自底向上的置信度传播，并通过硬门控（Hard Gates）过滤无效组合（如时序顺序违反、同一通道的物理冲突）。
-   **ELT有效性原则**：ELT遵循三个有效性原则：**建设性组合**（复合节点必须增加非平凡结构）、**时间紧凑性**（防止复合事件内出现无解释的大间隙）、**物理排他性**（防止同一通道同时支持不兼容的原始状态）。

## 4. 意义与影响（Significance and Impact）

-   **解决的问题与贡献**：
    1.  **新任务设定**：首次正式定义了“语言引导的时间序列事件检测”任务，为未来研究开辟了新方向。
    2.  **新知识表示**：提出了ELT，一种强大的、可解释的、用于连接自然语言和时间序列信号的知识表示框架。
    3.  **新方法框架**：构建了SELA，一个创新的神经符号VLM智能体系统，展示了如何将符号推理与神经感知有效结合来解决复杂的时间序列推理问题。
    4.  **新基准数据集**：发布了KITE，一个来自真实世界、具有挑战性的基准，包含专家知识和标注，将推动该领域的标准化评估。

-   **对未来研究/应用的影响**：
    -   **工业应用**：SELA可以直接应用于能源、制造、金融等领域，使非AI专家能够通过自然语言定义事件，实现零样本的自动化监控和诊断。
    -   **可解释AI**：SELA的实例化证明树为时间序列分析提供了前所未有的可解释性，这对于高风险决策至关重要，有助于建立人机信任。
    -   **研究方向**：本文的工作将激励更多关于“语言-时间序列”接地、神经符号推理、以及基于智能体的时间序列分析的研究。

-   **局限与开放问题**：
    1.  **智能体协作**：本文主要关注ELT的作用，而智能体间的通信和协作协议（如错误纠正、证据共享）的影响尚未深入探索。
    2.  **领域泛化性**：KITE数据集仅覆盖能源和气候两个领域，未来需要在更多样化的领域（如医疗、金融）和更复杂的事件序列数据上验证框架的通用性。
    3.  **ELT解析瓶颈**：实验显示，使用人工解析的ELT（Oracle）性能优于SELA自解析版本，表明ELT的自动解析质量是当前系统的一个主要瓶颈。

## 5. 澄清与简化（Clarifications and Simplifications）

-   **类比：ELT就像一份“建筑蓝图”**：
    -   **文本描述**：相当于甲方对建筑的口头描述：“我要一栋楼，一楼是厨房和客厅，二楼是卧室和书房，厨房和客厅要连通，卧室和书房要分开。”
    -   **ELT模式**：就是建筑师根据描述画出的**蓝图**。蓝图定义了结构（树状）、组件（原始模式：厨房、客厅）和组件之间的关系（时序算子：连通、分开）。
    -   **SELA的信号检查员**：就像一位**施工监理**。他拿着蓝图（ELT模式）去工地（时间序列信号）上检查。他需要找到“厨房”在哪里（定位原始模式），并确认“厨房”和“客厅”是否真的连通（验证时序关系）。
    -   **实例化证明树**：就是监理最终提交的**验收报告**，上面详细记录了“厨房”在坐标X-Y，“客厅”在坐标Y-Z，它们之间的门在坐标Y，一切符合蓝图，因此建筑合格（事件发生）。

-   **为什么ELT能帮助VLM？**
    -   **降低任务复杂度**：让VLM直接回答“这个信号片段是不是一个‘失封’事件？”非常困难。但让VLM回答“在这个区间内，压力通道是否出现了‘持续下降’的形态？”则相对容易。ELT将复杂问题分解为一系列简单问题。
    -   **提供推理框架**：VLM容易产生幻觉，可能会“看到”不存在的模式。ELT就像一个严格的“剧本”，VLM必须按照剧本（树结构）来表演（定位模式），不能随意发挥。如果某个原始模式找不到，整个事件就不会被确认，从而避免了幻觉。
    -   **提供可审计的证据**：VLM的“黑箱”决策难以理解。ELT的证明树将整个推理过程白盒化，每一步都有据可查，专家可以轻松地检查“是哪个原始模式没找到导致事件未被识别”。
