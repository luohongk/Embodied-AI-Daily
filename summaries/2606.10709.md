## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLM）作为搜索智能体（Search Agent）的强化学习（RL）训练。
- **问题（Problem）**：在基于结果奖励（outcome-only rewards）的GRPO类算法训练中，那些所有采样轨迹全部正确（太简单）或全部错误（太难）的查询（query）会产生零方差（zero-variance）的梯度，浪费计算资源且不提供学习信号。现有方法将零方差视为静态属性并直接丢弃，但忽略了查询的零方差状态会随策略演化而动态变化。
- **方法（Method）**：提出**查询回收（Query Recycling）** 策略。该方法维护一个动态的查询池，将产生零方差梯度的查询放回池中，使其在未来训练步骤中有机会被重新采样，从而让有效的训练分布与策略共同演化。
- **创新（Innovation）**：核心创新在于认识到零方差是**瞬态（transient）** 而非静态属性，并据此设计了一个简单的权重更新规则，将“已消耗”的信号查询权重置零，而将“零方差”查询保留在池中。这与现有丢弃或预过滤的方法形成鲜明对比。
- **意义（Significance）**：该方法在完全使用合成数据训练的情况下，仅用1.7B参数的模型就在七个多跳问答基准上达到了66.0的平均Pass@1，匹配甚至超越了使用基准监督数据训练的7B参数系统。这证明了高效利用训练数据的重要性，并为训练更强大的搜索智能体提供了一种低成本、高效率的路径。
- **一句话总结**：本文提出“查询回收”策略，通过将训练中产生零方差梯度的查询放回采样池，使其在策略演化后重新提供学习信号，从而显著提升了LLM搜索智能体的强化学习训练效率。

## 1. 研究动机（Motivation）

- **解决的问题**：在训练LLM搜索智能体时，常用的GRPO算法依赖于对每个查询生成一组（K个）采样轨迹。只有当这组轨迹中既有成功也有失败时，才能计算出有效的梯度信号。如果所有轨迹都成功（太简单）或都失败（太难），则梯度为零，这被称为“零方差”查询。这些查询消耗了计算资源（rollout cost）却没有带来任何学习收益。
- **为什么重要**：随着训练进行，模型能力提升，大量查询会从“太难”变为有效，或从“有效”变为“太简单”。如果简单地丢弃这些零方差查询，就会丢失这些潜在的、宝贵的训练信号。这导致了训练效率低下，尤其是在计算资源有限的情况下。
- **挑战在哪里**：
    1.  **静态假设的局限性**：现有工作（如DAPO、GRESO）将零方差视为查询的固有属性，认为一个查询要么是好的（有信号），要么是坏的（零方差），并据此进行过滤或跳过。然而，这种假设忽略了策略的动态性。
    2.  **非平稳性（Non-stationarity）**：一个查询的零方差状态取决于当前策略的成功概率 $p_t(q)$。随着策略 $\pi_{\theta_t}$ 的演化，$p_t(q)$ 会变化，因此零方差状态是**非平稳的**。一个在训练早期“太难”的查询，在模型能力提升后可能变得“可解”；一个“太简单”的查询，也可能因为策略漂移（policy drift）而重新变得有挑战性。
    3.  **信号来源的多样性**：回收的信号不仅来自“太难”查询（因能力提升而变好），也来自“太简单”查询（因策略漂移而变差）。现有方法通常只关注前者，忽略了后者。

## 2. 创新点（Innovation）

- **核心洞察（Key Insight）**：零方差是**瞬态（transient）** 而非静态属性。查询的零方差状态会随策略演化而动态翻转。
- **提出的方法**：**查询回收（Query Recycling）**。这是一个动态池管理策略，其核心是一个简单的权重更新规则：
    - 维护一个包含所有训练查询的加权池。
    - 在每个训练步骤，从池中采样一批候选查询。
    - 对采样到的查询进行采样和评估，识别出哪些是“信号承载”查询（即组内奖励有方差）。
    - **更新规则**：将**已被消耗的信号承载查询**的权重置为0（暂时从池中移除），而**所有其他查询**（包括零方差查询和未被选中的信号查询）的权重保持不变（保持为1）。
- **与现有方案的不同**：
    - **GRPO**：固定遍历整个数据集，每个查询在每个epoch只被采样一次，无论其是否产生信号。
    - **DAPO**：在一个步骤内动态增加采样量，直到收集到足够的信号查询，但随后会丢弃所有零方差查询，且不保留它们用于未来采样。
    - **GRESO**：基于历史统计信息，在采样前就预测并跳过可能为零方差的查询。
    - **Query Recycling**：不丢弃零方差查询，而是将其“回收”到池中，等待未来策略变化后再次被采样，从而“复活”这些潜在的信号源。
- **为什么能奏效**：该方法巧妙地利用了RL训练中策略演化的非平稳性。通过让训练分布与策略共同演化，它能够自动地从“太难”和“太简单”的查询中持续挖掘出新的训练信号，从而在相同的总采样预算下，获得更多有效的梯度更新。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的整体训练流程分为三个主要阶段：

1.  **查询生成（Query Generation）**：在一个沙盒化的检索环境（DeepResearchGym，基于ClueWeb22快照）中，通过四种方法合成训练数据，以诱导不同的搜索策略：
    - **DRGym**：来自DeepResearchGym的现有9k问答对。
    - **Webgraph-based**：通过随机游走超链接图生成需要跟随链接的多跳问题。
    - **Iterative Search**：通过密集检索生成需要迭代搜索的多跳问题。
    - **Comparative Search**：生成需要并行检索多个独立实体并进行比较的问题。
    - 最终得到一个包含25k问答对的池子，其中15k用于SFT，10k用于RL。

2.  **监督微调（SFT）**：使用更强的教师模型（GPT-4.1-mini）在15k查询上生成轨迹，筛选出6.3k正确且格式正确的轨迹用于SFT，为RL提供一个良好的起点。

3.  **强化学习（RL）与查询回收**：这是核心部分。在SFT检查点基础上，使用GRPO算法进行RL训练，但引入了动态池管理和查询回收策略。

### 3.2 关键算法与数学推导

**1. 标准GRPO的梯度条件**

对于一个查询 $q$，其K个采样轨迹的奖励为 $\{r_1(q), ..., r_K(q)\}$。只有当 $0 < \sum_{i=1}^K r_i(q) < K$ 时，该查询才产生非零梯度。零方差概率为：
$$P_{zv}(q; \pi_{\theta_t}) = p_t(q)^K + (1 - p_t(q))^K$$
其中 $p_t(q) = \mathbb{E}_{o \sim \pi_{\theta_t}(\cdot|q)}[R(o, q)]$ 是查询在策略 $\pi_{\theta_t}$ 下的成功概率。

**2. 动态池管理与查询回收**

- **池定义**：维护一个加权查询池 $Q(t) = \{(q_i, w_i^{(t)})\}_{i=1}^N$，初始权重 $w_i^{(0)} = 1$。
- **采样**：在步骤 $t$，根据权重 $w_i^{(t)}$ 从池中采样一个候选集 $C_t$（大小为 $k \cdot B$，$k$ 为过采样因子，$B$ 为批次大小）。
- **信号识别**：对 $C_t$ 中的每个查询进行K次采样，识别出信号承载查询集 $\tilde{S}_t$：
  $$\tilde{S}_t = \left\{ q \in C_t \mid 0 < \sum_{i=1}^K r_i(q) < K \right\}$$
- **梯度更新**：从 $\tilde{S}_t$ 中选取最多 $B$ 个查询组成 $S_t$ 用于梯度更新。
- **查询回收的权重更新规则**：
  $$w_q^{(t+1)} = \begin{cases} 0 & \text{if } q \in S_t \\ 1 & \text{otherwise} \end{cases}$$
  即，只有被用于梯度更新的信号承载查询被暂时移除（权重置0），而所有零方差查询和未被选中的信号查询都保留在池中，权重不变。

**3. 与其他方法的统一框架**

论文将GRPO、DAPO和Bounded-DAPO也形式化为该动态池框架下的特例：
- **GRPO**：$k=1$，权重更新为 $w_q^{(t+1)} = 0 \text{ if } q \in C_t$（所有采样过的查询都被消耗）。
- **DAPO**：动态增加 $k$ 直到 $|\tilde{S}_t| = B$，权重更新同GRPO。
- **Bounded-DAPO**：固定 $k>1$，权重更新同GRPO。

### 3.3 模型、训练与数据集

- **模型**：Qwen3-1.7B 和 Qwen3-4B。
- **训练数据集**：完全合成，基于ClueWeb22快照，通过四种方法生成。训练时不使用任何下游评估基准的训练集。
- **训练环境**：训练时使用DeepResearchGym API作为沙盒化的检索环境。推理时切换到SERPER实时网络搜索，以测试零样本迁移能力。
- **训练配置**：批次大小 $B=32$，组大小 $K=4$，过采样因子 $k \in \{1, 2\}$。所有配置的总采样预算（rollout budget）相同，为84.5K条轨迹。

### 3.4 实验设置与结果

- **评估基准**：七个多跳问答（MHQA）基准（2WikiMultiHopQA, Bamboogle, HotpotQA, MuSiQue, Natural Questions, PopQA, TriviaQA）和三个长程推理/网页导航基准（GAIA, HLE, WebWalkerQA）。
- **主要结果（表2）**：
    - 在1.7B规模下，使用查询回收的Bounded-DAPO（k=2）在7个MHQA基准上平均Pass@1达到**66.0**，显著优于不使用回收的GRPO（64.9）和Bounded-DAPO（64.1）。
    - 该结果匹配甚至超越了使用基准监督数据训练的7B模型（如ASearcher-Web-7B的66.3）。
    - 在4B规模下，性能进一步提升至**71.3**。
- **消融与分析（图3、4、5、6）**：
    - **回收查询主导有效批次（图4）**：训练后期，约**四分之三**的有效梯度更新来自被回收的查询（之前是零方差的）。这表明回收机制是后期训练信号的主要来源。
    - **“太难”和“太简单”查询均有贡献（图4）**：被回收的查询中，来自“太难”和“太简单”的查询数量相当，说明回收同时捕获了能力提升和策略漂移带来的信号。
    - **池难度随模型能力演化（图5）**：1.7B模型下，“太难”查询始终占主导；而4B模型下，“太简单”查询比例显著增长，表明更强模型能转化更多查询为已掌握任务。
    - **组大小K的影响（图6）**：较小的K（如K=4）会因采样噪声产生更多“伪”零方差查询，从而带来更多回收机会，且最终性能与K=8相当，但计算成本减半。

### 3.5 技术细节

- **Agent架构**：采用简化的ReAct风格设计，仅暴露一个搜索工具，允许每轮最多发起三个并行搜索查询。移除了URL访问工具和辅助信息提取模型，搜索工具直接返回清洗后的文本段落。
- **SFT数据**：使用GPT-4.1-mini作为教师模型，在15k合成查询上生成轨迹，并筛选出6.3k条正确轨迹用于SFT。
- **软回收（Soft Recycling，附录B）**：提出一种变体，将已消耗查询的权重设为 $\alpha \in [0,1]$ 而非0，允许它们以较低概率被再次采样，提供更平滑的分布变化。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **挑战了现有共识**：通过实验证明，零方差是瞬态而非静态属性，挑战了DAPO、GRESO等方法的根本假设。
    2.  **提出了高效且通用的方法**：查询回收是一个简单、即插即用的训练技巧，可以轻松集成到现有的GRPO训练流程中，显著提升数据利用效率和最终性能。
    3.  **实现了低成本高性能**：仅用1.7B模型和合成数据就达到了7B模型在真实数据上的性能，展示了高效训练范式的巨大潜力，降低了训练强大搜索智能体的门槛。
- **对未来研究/应用的影响**：
    - 为LLM后训练（post-training）提供了一种新的、更高效的范式，尤其是在计算资源受限的场景下。
    - 强调了训练数据动态管理的重要性，启发未来研究探索更复杂的池管理策略，如基于难度或学习进度的自适应采样。
    - 证明了合成数据与高效RL结合的有效性，为摆脱对昂贵人工标注数据的依赖提供了有力证据。
- **作者提到的局限与开放问题**：
    - **局限**：评估依赖LLM裁判，存在可重复性问题；推理使用实时搜索引擎，也存在可重复性问题；数据集主要为英文；实验仅在1.7B和4B模型上进行；未测试与更密集奖励信号（如过程奖励）的结合。
    - **未来工作**：与更密集的监督信号和优势塑造（advantage shaping）结合；在线生成与策略能力匹配的新查询，以突破固定池的限制；将回收策略扩展到长文档生成（deep research）场景。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：学生刷题**
    - **零方差查询**：想象一个学生在做练习题。如果题目太简单（所有答案都对），或者太难（所有答案都错），那么这次练习对学生的提升几乎没有帮助，这就是“零方差”查询。这浪费了做题的时间。
    - **现有方法（DAPO等）**：老师看到学生做简单题全对，做难题全错，就认为这些题没用，直接扔掉，下次不再让学生做。
    - **查询回收（Query Recycling）**：一个更聪明的老师会把做错的难题和做对的简单题都记下来。随着学生学习进步，之前做错的难题可能现在能做出一些了；或者因为粗心，之前做对的简单题现在反而做错了。老师会把这些“旧题”重新拿出来让学生做，从而发现新的学习机会。这就是“回收”的价值。

- **核心公式解释**：
    - $P_{zv}(q; \pi_{\theta_t}) = p_t(q)^K + (1 - p_t(q))^K$：这个公式计算了一个查询在当前模型下“没用”的概率。如果模型做对的概率 $p_t(q)$ 非常高（接近1），那么 $p_t(q)^K$ 很大，意味着K次采样很可能全对。如果模型做对的概率非常低（接近0），那么 $(1-p_t(q))^K$ 很大，意味着K次采样很可能全错。只有当 $p_t(q)$ 在中间值时，这个概率才小，查询才“有用”。这个公式完美地解释了为什么零方差状态是动态的——因为 $p_t(q)$ 在训练中会不断变化。
