## 0. 概览（Concise Summary）

- **主题（Topic）**：四足机器人多传感器融合状态估计。
- **问题（Problem）**：仅依赖本体感知（如IMU、关节编码器）的状态估计方法存在随时间累积的漂移问题，尤其在垂直（z轴）方向上。现有融合外部传感器（如LiDAR、GPS）的方法未利用李群上的群仿射性质，而该性质能提供更好的收敛性和稳定性。
- **方法（Method）**：提出了两种基于不变性理论的框架：扩展不变性扩展卡尔曼滤波器（E-InEKF）和扩展不变性平滑器（E-IS）。它们将本体感知（IMU、运动学）与外部感知（LiDAR、GPS）数据融合，并推导了满足群仿射性质的LiDAR和GPS观测模型。
- **创新（Innovation）**：
    1.  首次将LiDAR里程计和GPS数据成功集成到四足机器人的InEKF和IS框架中。
    2.  推导了满足群仿射性质的LiDAR和GPS观测模型，保证了估计器误差动态的自治性和稳定性。
    3.  采用松耦合方式，在独立线程中运行LiDAR里程计（KISS-ICP），保持了本体感知状态估计的计算效率。
- **意义（Significance）**：显著降低了四足机器人在室内外复杂环境中的位置漂移，特别是z轴漂移，在精度上超越了主流的LiDAR-惯性里程计方法（如LIO-SAM, FAST-LIO2），为四足机器人的鲁棒定位提供了新的高效解决方案。
- **一句话总结**：本文通过为不变性滤波器和平滑器设计满足群仿射性质的LiDAR和GPS观测模型，有效抑制了四足机器人状态估计中的位置漂移，实现了高精度、高鲁棒性的多传感器融合定位。

## 1. 研究动机（Motivation）

- **问题与空白**：四足机器人的状态估计对于其自主导航至关重要。纯本体感知方法（如P-InEKF, P-IS）虽然高效，但由于无法观测绝对位置和偏航角，会随时间产生不可忽视的漂移，尤其是在z轴方向。虽然已有工作（如LIO-SAM, FAST-LIO2）融合了LiDAR和IMU，但它们并未利用李群上的群仿射性质。群仿射性质可以保证估计器的误差动态与当前状态无关，从而提供更好的全局收敛性和稳定性。此外，这些方法也未充分利用四足机器人特有的腿式运动学信息。
- **重要性**：解决位置漂移问题对于四足机器人在长距离、高动态或复杂地形（如楼梯、斜坡）上的可靠运行至关重要。一个稳定、精确且漂移小的状态估计是实现高级任务（如路径规划、导航、操作）的基础。
- **挑战**：
    1.  **模型设计**：如何将LiDAR和GPS的观测模型设计成满足群仿射性质的右不变观测形式（公式7），是将其集成到InEKF和IS框架中的核心挑战。
    2.  **计算效率**：LiDAR点云处理计算量大，频率低（~10Hz），而本体感知估计频率高（~100Hz以上）。如何在不拖慢主估计线程的前提下，有效融合低频的LiDAR数据是一个工程挑战。
    3.  **传感器特性**：LiDAR在特定环境（如长走廊、浓雾）下可能退化，GPS在室内无效，需要设计鲁棒的融合策略。

## 2. 创新点（Innovation）

1.  **首次将LiDAR和GPS集成到不变性滤波器/平滑器中**：本文是首个将LiDAR里程计和GPS数据成功融入四足机器人的InEKF和IS框架的工作，命名为E-InEKF和E-IS。这填补了将群仿射性质应用于包含外部传感器（特别是LiDAR和GPS）的机器人状态估计领域的空白。

2.  **推导满足群仿射性质的LiDAR和GPS观测模型**：这是本文的核心理论创新。作者将LiDAR位置估计（$p_{lid}$）和GPS位置估计（$p_{gps}$）巧妙地构造成右不变观测形式（公式19和20），即 $Y = X^{-1}b + V$。这种形式确保了观测模型也满足群仿射性质，使得整个估计过程（传播+观测）的误差动态都独立于状态轨迹，从而继承了InEKF和IS的优良收敛特性。

3.  **高效的松耦合并行架构**：为了兼顾精度和效率，作者采用松耦合方式。LiDAR里程计（使用KISS-ICP算法）在一个独立的并行线程中运行，以10Hz的频率计算位姿。主估计线程（E-InEKF或E-IS）则以更高频率（如100Hz）运行本体感知传播，并在LiDAR或GPS数据到来时，通过所提出的观测模型进行更新。这种设计使得LiDAR的加入几乎不影响主线程的计算速度（如图6所示），保持了本体感知方法的高效性。

## 3. 主要内容（Main Content）

### 3.1 设计架构与方法

论文提出了两个框架：E-InEKF（扩展不变性扩展卡尔曼滤波器）和E-IS（扩展不变性平滑器）。它们的整体架构如图1所示。

- **共同模块**：
    - **传播（Propagation）**：使用IMU（角速度$\tilde{\omega}_t$，线加速度$\tilde{a}_t$）和关节编码器数据，基于群仿射性质进行状态预测。
    - **观测（Observation）**：融合多种观测数据来校正状态。
        - **运动学观测**：基于稳定接触假设，利用正向运动学计算足端位置。
        - **LiDAR观测**：通过独立线程的KISS-ICP获得LiDAR里程计位姿$p_{lid}$。
        - **GPS观测**：获得GPS位置$p_{gps}$。
    - **腿足机器人专用方法**：集成了基于动量观测器的接触估计器[30]和滑移拒绝方法[6]，以处理动态接触事件（如滑移），增强鲁棒性。

- **E-InEKF与E-IS的区别**：
    - **E-InEKF**：是一个滤波器，只维护并更新当前时刻的状态和协方差。计算量小，适合实时应用。
    - **E-IS**：是一个平滑器，维护一个时间窗口内的所有状态（$X_{0:n}$），并通过最大后验（MAP）估计来优化整个窗口内的状态轨迹。它包含“先验（Prior）”和“边缘化（Marginalization）”模块来管理窗口大小。计算量较大，但精度更高。

### 3.2 关键算法与数学推导

**1. 状态定义（公式8）**：
状态被定义在Lie群 $SE_{N+2}(3)$ 上，包含姿态 $R_t$，速度 $v_t$，位置 $p_t$，以及 $N$ 个接触点位置 $d_{i,t}$。IMU偏置 $b_t^\omega, b_t^a$ 则在向量空间中处理。

**2. 群仿射性质与误差动态（公式6, 7）**：
- **传播模型**：系统的传播模型（公式9-11）被证明是近似群仿射的，其右不变误差 $\xi_t^r$ 的传播是线性的（公式12），且与状态轨迹无关。矩阵 $A_t$（公式13）是常数矩阵，这是InEKF的核心优势。
- **观测模型**：一个观测模型满足群仿射性质，当且仅当其可以写成右不变形式（公式7）：
  $$y_t = X_t^{-1} s + w_t^{obs}$$
  其中 $y_t$ 是观测值，$s$ 是常数向量。

**3. LiDAR和GPS观测模型（公式19, 20）**：
这是本文的核心推导。作者将LiDAR和GPS的位置观测巧妙地构造成公式7的形式：
- **LiDAR观测模型**：
  $$Y_{lid} = X_t^{-1} b_{lid} + V_{lid}$$
  其中 $Y_{lid} = [0_{3,1}, 0, 1, 0]^\top$ 是观测向量，$b_{lid} = [p_{lid}, 0, 1, 0]^\top$ 是常数向量。这里的关键是，将LiDAR观测到的位置 $p_{lid}$ 放入常数向量 $b_{lid}$ 中，而观测向量 $Y_{lid}$ 是固定的。这样，观测方程就严格符合了右不变形式。
- **GPS观测模型**：
  $$Y_{gps} = X_t^{-1} b_{gps} + V_{gps}$$
  其中 $Y_{gps} = [0_{3,1}, 0, 1, 0]^\top$，$b_{gps} = [p_{gps}, 0, 1, 0]^\top$。原理与LiDAR相同。

**4. E-InEKF更新方程**：
基于上述观测模型，E-InEKF的更新方程（公式21）将运动学、LiDAR和GPS观测组合成一个增广的观测向量 $Y$。其卡尔曼增益和协方差更新遵循标准InEKF流程，其中观测雅可比矩阵 $H$ 和噪声协方差矩阵 $N$ 被相应地定义为块对角矩阵。

**5. E-IS代价函数**：
E-IS的优化问题（公式15）是一个非线性最小二乘问题。其中，观测代价项包含了LiDAR和GPS的残差：
- **LiDAR残差**：$r_{lid} = X Y_{lid} - b_{lid}$
- **GPS残差**：$r_{gps} = X Y_{gps} - b_{gps}$
这些残差及其对应的雅可比矩阵 $J_{lid}, J_{gps}$ 和协方差矩阵 $\Sigma_{lid}, \Sigma_{gps}$ 被加入到总的优化问题中。

### 3.3 模型、训练与数据集

- **模型**：本文提出的E-InEKF和E-IS是模型驱动的状态估计算法，而非数据驱动的。它们不需要训练过程。
- **数据集**：论文没有使用公开数据集，而是使用KAIST HOUND2四足机器人在**室内**和**室外**两种场景下自行采集的数据进行实验验证。
    - **室内**：包含木块、台阶、涂有硼酸的丙烯酸板（模拟滑溜地面）等复杂地形，机器人会经历高度变化和剧烈冲击。使用Vicon运动捕捉系统提供真值。
    - **室外**：一条约300米的闭合路径，机器人从起点出发并返回。使用RTK GPS提供真值。

### 3.4 实验设置与结果

- **评价指标**：绝对轨迹误差（ATE）和相对位置误差（RPE）。
- **基线方法**：
    - 本体感知方法：P-InEKF [8], P-IS [12]。
    - LiDAR-惯性方法：LIO-SAM [1], FAST-LIO2 [3]。
- **室内实验结果（表I, 图3）**：
    - E-InEKF (w/o GPS) 和 E-IS (w/o GPS) 的ATE和RPE均显著低于所有基线方法。
    - 与LIO-SAM和FAST-LIO2相比，ATE分别降低了约25%和20%。
    - 与P-InEKF和P-IS相比，ATE降低了超过70%。
    - **关键发现**：图3清晰地展示了加入LiDAR后，z轴方向的漂移被有效抑制，而纯本体感知方法（P-InEKF, P-IS）的z轴漂移非常严重。
- **室外实验结果（表II, 图5）**：
    - E-InEKF和E-IS（使用LiDAR+GPS）的ATE和RPE最低。
    - 与LIO-SAM和FAST-LIO2相比，ATE分别提升了约93%和89%（即误差大幅减小）。
    - 即使不使用GPS（E-InEKF w/o GPS, E-IS w/o GPS），其RPE也低于或等于FAST-LIO2，ATE略高（2cm），但作者指出这是因为腿式运动学在长距离中可能引入微小噪声，但增强了在LiDAR退化环境下的鲁棒性。
    - **关键发现**：图5表明，在长距离室外实验中，加入LiDAR和GPS后，z轴漂移被显著抑制，轨迹闭合效果更好。
- **消融实验（Ablation Study）**：
    - 通过逐步移除LiDAR和GPS，验证了每个外部传感器对性能提升的贡献。结果证明，LiDAR和GPS的加入都能有效减少漂移。
- **滤波器与平滑器对比（表III, 图6）**：
    - E-IS的精度通常高于E-InEKF，且随着平滑窗口（WS）增大，精度提高。
    - E-InEKF的计算时间（0.06ms/迭代）远低于E-IS（WS=15时为4.5ms/迭代）。E-IS在WS=1时计算时间为0.18ms/迭代，与E-InEKF相当。

### 3.5 技术细节

- **LiDAR里程计**：采用KISS-ICP [27]，一个高效的点对点ICP算法。它使用恒速运动模型去畸变，体素滤波降采样，自适应数据关联阈值，并通过鲁棒优化进行位姿估计。
- **松耦合**：LiDAR里程计在独立线程运行，主估计线程通过观测模型接收其输出的位置 $p_{lid}$。这种方式避免了紧耦合中复杂的联合优化，保持了系统模块化和高效性。
- **接触估计与滑移拒绝**：采用基于动量观测器的方法[30]估计接触状态，并采用[6]中的方法在检测到滑移时调整接触观测的协方差，从而增强对动态事件的鲁棒性。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **理论贡献**：为将LiDAR和GPS这类绝对/相对位置传感器集成到基于李群的不变性估计框架中，提供了一种通用且优雅的观测模型设计方法。这为后续研究如何将更多类型的传感器（如相机、雷达）以群仿射形式融入InEKF/IS奠定了基础。
    2.  **实践贡献**：提出了两个实用、高效且鲁棒的四足机器人状态估计框架。E-InEKF适合对实时性要求高的应用，E-IS适合对精度要求高的应用。实验证明，它们在挑战性环境下显著优于现有方法。
    3.  **工程贡献**：松耦合的并行架构设计，使得在保持本体感知方法高效性的同时，能够轻松集成计算密集型的LiDAR处理模块，具有很高的工程实用价值。

- **对未来研究/应用的影响**：
    - 为四足机器人在复杂、长距离任务（如搜救、巡检、勘探）中的自主导航提供了更可靠的定位基础。
    - 其设计思想可以推广到其他类型的移动机器人（如双足、轮式、履带式）的状态估计中。
    - 为未来融合更多传感器（如RGB-D相机、热成像相机）提供了理论框架。

- **局限与开放问题**：
    - 作者提到，在室外实验中，加入腿式运动学在长距离下可能引入微小噪声，导致ATE略高于FAST-LIO2。如何更好地权衡腿式运动学带来的鲁棒性和潜在的噪声，是一个开放问题。
    - 当前采用松耦合方式，未来可以探索与LiDAR里程计的紧耦合方法，以期望获得更高的精度。
    - 论文未涉及视觉传感器，未来工作可以探索融合RGB-D或热成像相机。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“群仿射性质”和“不变性滤波器”？**
    - **通俗解释**：想象你在一个完全漆黑的房间里走路。你只能通过自己迈步的步数（本体感知）来估计位置，但时间长了，你肯定会走偏。这就是“漂移”。现在，你偶尔能用手摸到墙壁（外部感知，如LiDAR/GPS），就能修正自己的位置。但是，修正的方式很重要。
    - **“群仿射性质”** 就像是一个“金标准”。它保证了你修正位置的方式（观测模型）和你走路的方式（传播模型）是“兼容”的。这种兼容性带来的好处是：无论你当前在房间的哪个位置，你修正位置的“效果”都是一样的、稳定的。这比普通的滤波器（EKF）更鲁棒，因为普通滤波器的修正效果可能会因为你当前的位置和姿态不同而变化。
    - **“不变性滤波器（InEKF）”** 就是专门利用这种“金标准”设计的滤波器。它把机器人的状态（位置、姿态、速度等）放在一个特殊的数学空间（李群）里处理，使得误差的计算和更新与机器人当前的状态“无关”（即“不变”），从而获得更好的收敛性和稳定性。

- **为什么LiDAR观测模型可以写成 $Y = X^{-1}b$ 的形式？**
    - **类比**：假设状态 $X$ 是一个“变换器”，它能把机器人身体坐标系下的点变换到世界坐标系。LiDAR测量给出的是世界坐标系下的位置 $p_{lid}$。我们想要的是，用这个测量来修正我们的状态 $X$。
    - 公式 $Y = X^{-1}b$ 可以理解为：我们构造一个“虚拟观测” $Y$（一个常数向量），然后让这个虚拟观测等于“状态逆”乘以一个“包含真实测量的常数向量 $b$”。通过这种方式，我们巧妙地把“修正状态”这个动作，变成了一个在数学上非常优雅、且满足“金标准”的形式。具体来说，$b$ 里包含了LiDAR测到的位置 $p_{lid}$，而 $Y$ 是一个固定的向量。当我们的状态 $X$ 有误差时，计算出的 $Y$ 就会偏离它的真实值（常数），这个偏差（残差）就用来更新状态。
