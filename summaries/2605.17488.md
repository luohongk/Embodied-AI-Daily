## 0. 概览（Concise Summary）

- **主题（Topic）**：多模态内容生成，具体为联合音频-视频（Audio-Video）生成中的身份定制（Customization）。
- **问题（Problem）**：现有工作在同时生成视频和音频时，难以对多个交互主体（Multi-Subject）的视觉身份（人脸）和声音特征（音色）进行精确、鲁棒的绑定和保持。特别是，现有方法存在“语音泄漏”（Caption Vocalization）问题，即模型错误地将非语音的描述性文本朗读出来。
- **方法（Method）**：提出一个名为 **Omni-Customizer** 的端到端框架。核心组件包括：
    1.  **Omni-Context Fusion (OCF) 模块**：将文本、图像、音频和TTS嵌入融合，丰富文本表征。
    2.  **Semantic-Anchored Multimodal RoPE (SA-MRoPE)**：通过统一的3D位置空间，将多模态参考令牌锚定到其对应的语义描述上，实现结构化融合。
    3.  **Masked TTS Cross-Attention (MTP-CA)**：通过掩码机制，严格限制TTS音素注入到语音文本区间，彻底解决“语音泄漏”问题。
    4.  **交错式训练策略**：结合联合音视频训练和纯音频TTS训练，快速适应多语言场景，同时保持基础模型的先验知识。
- **创新（Innovation）**：
    1.  首次提出针对多主体交互场景的端到端联合音视频定制框架。
    2.  提出SA-MRoPE，实现了语义锚定的多模态位置编码，解决了多主体身份混淆。
    3.  提出MTP-CA，通过硬门控机制彻底解决了“语音泄漏”这一关键问题。
    4.  设计了交错式、渐进式的训练策略，有效利用了不平衡的数据集，并提升了模型的鲁棒性。
- **意义（Significance）**：该工作填补了开放源码联合音视频生成领域中，多模态、多主体身份定制的研究空白。其提出的方法显著提升了生成内容的身份保真度、音视频同步精度和整体质量，为构建更真实、可控的交互式数字人、虚拟角色等应用奠定了基础。
- **一句话总结**：Omni-Customizer 通过创新的语义锚定位置编码和掩码交叉注意力机制，实现了端到端、高质量、多主体的联合音频-视频身份定制生成。

## 1. 研究动机（Motivation）

- **解决的问题**：在联合音频-视频生成领域，现有工作主要关注通用内容生成，对于**精细化的多模态身份定制**，尤其是在**多主体交互场景**下，同时保持每个主体的视觉身份（人脸）和声音特征（音色）这一任务，研究尚属空白。
- **为什么重要**：真实世界中的交互（如对话、访谈）天然是多模态的，涉及多个主体。能够生成具有特定身份（长相、声音）的主体进行互动的视频，对于虚拟现实、电影制作、数字人、在线教育等领域具有巨大价值。
- **挑战在哪里**：
    1.  **多模态身份绑定困难**：将视觉参考（人脸图像）和音频参考（声音样本）与文本描述中的特定主体（如“穿红衣服的男人”）精确关联起来非常困难，容易导致身份混淆。
    2.  **“语音泄漏”问题**：现有的联合生成模型（如Ovi）存在一个严重缺陷，即音频生成分支会错误地将非语音的描述性文本（如“场景设定在阳光明媚的屋顶花园”）也朗读出来，破坏了生成内容的真实性。
    3.  **数据与训练不平衡**：用于训练的数据集存在严重的语言分布不均（如中文数据占主导），且音频分支的优化容易被视频分支主导，导致模型难以快速适应多语言场景并保持高质量的音唇同步。

## 2. 创新点（Innovation）

1.  **Omni-Context Fusion (OCF) 模块**：不同于传统方法让多模态信息在扩散主干中“被动”交互，OCF 模块主动将文本、图像、音频和TTS嵌入拼接成一个统一序列，并通过额外的Transformer层进行深度交互，从而将文本嵌入“丰富”为包含密集多模态线索的上下文。这相当于将文本编码器升级为一个主动的跨模态对齐引擎。

2.  **Semantic-Anchored Multimodal RoPE (SA-MRoPE)**：这是解决多主体身份混淆的关键。它提出一个统一的3D位置空间，将不同模态的参考令牌（图像、音频）的位置坐标直接锚定到其对应的文本描述（如 `<sub1> is A man...`）的结束位置。例如，主体1的视觉参考令牌位置被设定为 `(e1+1, h, w)`，其中 `e1` 是主体1文本描述的结束索引。这种“语义锚定”在注意力计算中创造了强烈的空间-时间偏差，确保每个模态的参考信息被正确地绑定到其文本身份上，避免了身份混淆。

3.  **Masked TTS Cross-Attention (MTP-CA)**：这是解决“语音泄漏”问题的创新。它通过一个**硬门控**机制，在TTS音素嵌入和文本嵌入之间进行交叉注意力计算。一个二进制掩码确保只有位于 `<S>...<E>` 标记内的语音文本才能接收到TTS音素信息，而所有非语音的描述性文本区域则完全不受影响。这从根本上杜绝了模型“朗读”非对话内容的可能性。

4.  **交错式、渐进式训练策略**：
    - **Interleaved JA VG and TTS-only Steps**：交替进行联合音视频训练和纯音频TTS训练。TTS-only步骤通过扩大音频批次大小，有效平均了音频VAE的重建误差，并起到了类似LoRA的正则化作用，保护了已学到的音视频对齐先验，同时快速赋予模型多语言能力。
    - **Progressive Disentanglement Curriculum**：采用两阶段渐进式课程学习。第一阶段（单主体对齐）让模型快速适应新架构；第二阶段（多主体解耦）通过使用解耦的参考数据（参考音频由随机文本合成，参考图像来自不同视频），迫使模型学习更高级、更鲁棒的身份特征，避免“复制粘贴”的捷径。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Omni-Customizer 的整体架构如图2所示，基于一个双流（视频和音频）的扩散Transformer（DiT）骨干网络（初始化为Ovi模型）。

1.  **输入处理**：
    - 输入包括：文本提示（Prompt）、多组参考图像（RefImg）、参考音频（RefAud）以及TTS音素嵌入（TTSEmbedding）。
    - 文本提示被格式化为结构化的 **Omni-Caption**（公式2），其中包含主体描述（`<subi> is Dv,i, with Da,i`）、环境动作描述（`Denv, Dact`）和语音内容（`<subk> says <S>Tk,j <E>`）。`<subi>` 作为锚点令牌连接同一主体的所有模态信息。

2.  **Omni-Context Fusion (OCF) 模块**：
    - 将文本嵌入 `ctxt`、视觉参考令牌 `cv`、音频参考令牌 `ca` 和TTS音素嵌入 `ctts` 拼接成一个统一序列 `S`。
    - 序列 `S` 经过 `L` 个Transformer块进行深度交互。
    - 在每个块后，提取前 `len(ctxt)` 个令牌（即原始文本部分）的输出，通过零初始化的残差连接加回到原始的 `ctxt` 上，以保持预训练语言模型的完整性。

3.  **Semantic-Anchored Multimodal RoPE (SA-MRoPE)**：
    - 在OCF模块的注意力计算中，使用SA-MRoPE替代标准RoPE。
    - 对于主体 `k`，其文本描述占据1D索引 `[sk, ek]`。其视觉和音频参考令牌的3D位置坐标被设定为：
        - 视觉：`Pos(Z_img^(k)) = (ek + 1, h, w)`
        - 音频：`Pos(Z_aud^(k)) = (ek + 2, j, 0)`
    - TTS音素令牌 `Z_tts^(k)` 的位置通过线性插值映射到其对应的语音内容区间 `[tstart, tend]`，并将第三个维度设为1以区别于文本令牌：
        - `Pos(Z_tts^(k)) = (linspace(tstart, tend, len(Z_tts^(k))), 0, 1)`

4.  **Masked TTS Cross-Attention (MTP-CA)**：
    - 在文本嵌入 `ctxt` 和TTS音素嵌入 `ctts` 之间添加一个交叉注意力层。
    - 应用一个二进制掩码，只允许 `ctxt` 中位于 `<S>...<E>` 区间内的令牌与 `ctts` 进行交互，其他区域的注意力权重被强制设为0。

5.  **双流DiT主干**：
    - 增强后的文本嵌入、视频和音频参考令牌被注入到双流DiT中，进行联合去噪，最终生成视频和音频。

### 3.2 关键算法与数学推导

- **联合去噪过程**：
    给定视频噪声 `zv,t` 和音频噪声 `za,t`，以及视频参考 `cv`、音频参考 `ca` 和文本嵌入 `ctxt`，联合去噪过程可表示为：
    $$(\hat{\epsilon}_v, \hat{\epsilon}_a) = F_\theta \left( [z_{v,t} \oplus c_v], [z_{a,t} \oplus c_a], t, c_{txt} \right)$$
    其中 `F_θ` 是双流DiT模型，`⊕` 表示沿时空维度拼接。

- **SA-MRoPE位置分配**：
    对于主体 `k`，其文本描述索引范围为 `[sk, ek]`，其视觉和音频参考令牌的位置分配如下：
    $$Pos(Z_{img}^{(k)}) = (e_k + 1, h, w)$$
    $$Pos(Z_{aud}^{(k)}) = (e_k + 2, j, 0)$$
    TTS音素令牌的位置：
    $$Pos(Z_{tts}^{(k)}) = (\text{linspace}(t_{start}, t_{end}, \text{len}(Z_{tts}^{(k)})), 0, 1)$$
    这种分配方式确保了在注意力计算中，视觉和音频参考令牌会与它们对应的文本描述（位于 `ek` 附近）产生强烈的注意力，从而实现语义锚定。

### 3.3 模型、训练与数据集

- **模型**：基于双流DiT架构，初始化为Ovi模型。新增了OCF模块（L个Transformer块）、MTP-CA模块。
- **训练策略**：
    1.  **Stage 1: 单主体对齐与音频引导 (20K步)**：在70万单主体数据上联合训练，并交错进行TTS-only训练（批次大小1024）。JA VG与TTS步长比例为1:1。
    2.  **Stage 2: 多主体适应 (10K步)**：在30万多主体数据上仅进行JA VG训练。
    3.  **Stage 3: 跨对解耦 (10K步)**：在50万OpenS2V数据集上进行JA VG训练，使用解耦的参考数据（参考音频由CosyVoice-3用随机文本合成，参考图像来自不同视频）。
    - 优化器：AdamW (β1=0.9, β2=0.95)，权重衰减0.01。余弦学习率调度，从1e-4衰减到1e-5，仅应用于新添加的OCF和MTP-CA模块。
- **数据集**：
    - **源数据**：OpenHumanVid（用于单主体/多主体配对数据）和 OpenS2V-5M（用于跨对解耦数据）。
    - **数据构建流程**：
        1.  **参考图像提取**：使用InsightFace进行人脸跟踪，选择检测置信度和边界框面积乘积最大的帧。
        2.  **ASR与音频描述**：使用Qwen3-Omni生成带时间戳的ASR转录和全局音频描述。
        3.  **参考音频合成**：为避免“复制粘贴”捷径，使用CosyVoice-3根据ASR文本重新合成参考音频，以解耦语音内容和音色。
        4.  **MLLM引导的Omni绑定**：使用Gemini 2.5-Pro（多主体场景）或Qwen3-Omni（简单场景）生成结构化的Omni-Caption，并完成视觉身份与声音身份的绑定。

### 3.4 实验设置与结果

- **基准测试 (OC-Bench)**：包含300个测试用例，分为三个100项的难度递增子集：单主体定制、鲁棒身份绑定（双人对话）、多主体复杂场景（含画外音、无声主体、多语言）。
- **评估指标**：
    - **身份保持**：人脸相似度（Face-Sim）、人脸一致性（Face-Cons）、音色相似度（T-Sim）。
    - **音视频同步**：唇形同步准确率（Sync-C/D）、IB-Score。
    - **视频质量**：美学质量（AQ）、成像质量（IQ）、时间闪烁（TF）。
    - **音频质量**：音频美学（PQ）、词错误率（WER）、IB-A Score。
- **对比方法**：与四类方法对比：视频定制（Phantom, V ACE）、音频驱动视频定制（Humo, HunyuanCustom等）、级联方法（Qwen-Image + JA VG模型）、联合音视频定制（DreamID-Omni）。
- **主要结果（表1）**：
    - Omni-Customizer 在核心多模态指标（Face-Sim, T-Sim, Sync-C/D, IB-S, WER）上全面超越所有基线方法。
    - 在复杂场景（子集2和3）中，基线方法性能显著下降，而Omni-Customizer保持鲁棒。
- **定性结果（图3）**：Omni-Customizer生成的多主体对话视频在身份保持、音唇同步和整体质量上明显优于其他方法。
- **消融实验（表2和图4）**：
    - 逐步添加OCF、SA-MRoPE、MTP-CA、交错训练和渐进课程，各项指标持续提升。
    - 图4定性展示了：没有渐进课程会导致面部扭曲；没有OCF和SA-MRoPE会导致主体混淆（两人同时说话）；没有MTP-CA会导致“语音泄漏”（模型朗读描述性文字）。

### 3.5 技术细节

- **“语音泄漏”问题**：论文指出，Ovi模型的音频分支在处理整个文本提示时，虽然依赖`<S>`和`<E>`标记来界定语音，但这种软约束在信息密集的多主体提示中容易被覆盖。MTP-CA通过硬门控提供了绝对隔离。
- **交错训练的优势**：TTS-only步骤通过扩大音频批次，将音频VAE的重建误差平均化，提供无偏梯度估计。同时，它冻结了跨模态路径，专注于扩展音频分支的容量，起到了类似LoRA的正则化作用，防止了内部协变量漂移。
- **渐进式课程**：第一阶段（单主体）让模型学习基本定制能力；第二阶段（多主体解耦）通过使用解耦的参考数据，迫使模型学习抽象的、高层的身份特征，而不是简单的“复制粘贴”。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  首次提出了一个针对多主体交互场景的端到端联合音视频定制框架。
    2.  创新性地解决了多模态身份绑定中的“身份混淆”和“语音泄漏”两大核心难题。
    3.  提出了一套有效的数据构建和训练策略，能够利用不平衡的数据集训练出鲁棒的模型。
    4.  在提出的OC-Bench基准上取得了最先进的性能，为后续研究提供了强有力的基线。

- **对未来研究/应用的影响**：
    - **技术推动**：SA-MRoPE和MTP-CA的设计思想可以启发其他多模态生成任务中的对齐和约束问题。
    - **应用前景**：为构建高度真实的虚拟数字人、交互式电影、游戏NPC、在线教育虚拟教师等应用提供了关键技术支撑。用户可以通过提供几张照片和一段语音，就能生成特定角色进行对话的视频。
    - **开源贡献**：论文提供了项目页面，有望推动该领域的开源社区发展。

- **作者提到的局限与开放问题**：
    - **分辨率与时长限制**：当前生成被限制在720P分辨率和10秒时长。扩展到更高分辨率和更长序列是未来工作的主要挑战，特别是在保持长期身份一致性方面。
    - **模型架构与数据管道**：解决时空缩放瓶颈需要同时对模型架构和数据构建流程进行深入研究。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比理解SA-MRoPE**：想象一个大型会议室（文本提示），里面有多个参会者（主体）。每个参会者面前有一个名牌（`<subi>` 锚点令牌）。SA-MRoPE就像给每个参会者的个人资料（视觉参考）和录音笔（音频参考）都贴上了与名牌位置对应的标签。这样，当系统需要查找“穿红衣服的男人”的资料时，它会直接去他名牌所在的位置附近寻找，而不是在整个会议室里乱翻，从而避免了拿错资料（身份混淆）。

- **类比理解MTP-CA**：想象一个剧本，里面有旁白（描述性文字）和对话（`<S>...<E>`）。传统的模型（如Ovi）可能会把旁白也当成对话来朗读。MTP-CA就像在剧本上加了一个“静音”按钮，只有对话部分
