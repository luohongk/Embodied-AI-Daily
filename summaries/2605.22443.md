## 0. 概览（Concise Summary）

- **主题（Topic）**：无人机（UAV）的基于图像的视觉伺服（IBVS）控制。
- **问题（Problem）**：传统IBVS控制器面临两个关键挑战：1）在接近目标时，由于输入和状态约束导致闭环稳定性丧失；2）在剧烈运动下，基于矩的视觉特征间歇性丢失导致控制失败。
- **方法（Method）**：提出一个融合了终端约束模型预测控制（TC-MPC）和基于卡尔曼滤波（KF）的状态预测机制的IBVS框架。
- **创新（Innovation）**：1）将终端状态约束和终端代价函数显式地融入IBVS误差动力学中，保证了递归可行性和闭环稳定性；2）利用卡尔曼滤波器预测短时视觉退化期间图像矩的时域演化，维持控制连续性。
- **意义（Significance）**：解决了实际无人机视觉伺服中因约束和特征丢失导致的控制不稳定和失效问题，通过真实飞行实验验证了其在无GPS环境下自主穿越门框等任务的有效性，提升了IBVS的鲁棒性和实用性。
- **一句话总结**：本文提出了一种结合终端约束MPC和卡尔曼滤波的鲁棒IBVS方法，有效解决了无人机视觉伺服中的约束和特征丢失问题。

## 1. 研究动机（Motivation）

- **解决的问题**：传统基于图像的视觉伺服（IBVS）控制器在实际无人机应用中存在两个核心问题：
    1.  **约束下的稳定性问题**：传统IBVS控制律（如 $v_c = -\lambda L^{-1}(q - q^*)$）未显式考虑无人机执行器的物理限制（如最大速度、加速度）和视觉约束（如相机视场角FOV限制）。当无人机接近目标时，这些未考虑的约束可能导致控制输入饱和，进而引发闭环系统失稳。
    2.  **特征丢失下的鲁棒性问题**：在无人机进行快速或剧烈机动时，由于运动模糊、光照变化、遮挡等原因，基于图像矩的视觉特征可能暂时丢失或变得高度噪声。这会导致控制中断或产生剧烈振荡，甚至导致任务失败。
- **为什么重要**：GPS信号在室内、森林、密集城市等环境中不可靠，视觉伺服成为无人机自主导航的关键替代方案。解决上述两个问题，对于提升无人机在复杂、非结构化环境中的自主性和可靠性至关重要。
- **挑战在哪里**：
    - 如何在IBVS框架中优雅地融入系统约束（输入和状态），同时保证控制的实时性和稳定性。
    - 如何在视觉特征短暂丢失期间，依然能生成合理的控制指令，维持系统稳定，直到视觉反馈恢复。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **终端约束MPC（TC-MPC）**：将模型预测控制（MPC）应用于基于图像矩的IBVS，并在MPC优化问题中显式加入了**终端状态约束**（$|e(k+N)| \leq \varepsilon_{term}$）和**终端代价**（$(e(k+N)-e_{ref})^T P (e(k+N)-e_{ref})$）。这与仅考虑输入约束的MPC（如[17]）不同，它从理论上保证了递归可行性和闭环稳定性。
    2.  **基于卡尔曼滤波的矩损失补偿**：在视觉特征因故丢失时，利用卡尔曼滤波器（KF）根据系统模型（$x(k) = A x(k-1) + w(k)$）预测图像矩的演化状态（$\hat{x}_{k|k-1}$），并将此预测值作为MPC的输入，从而“填补”控制间隙，维持控制连续性，直到视觉测量恢复。
- **与现有方案的不同**：
    - 相较于传统IBVS（无约束）和仅输入约束的MPC-IBVS（如[17]），本文的TC-MPC通过终端约束提供了更强的稳定性保证。
    - 相较于仅依赖视觉反馈的IBVS，本文的KF机制提供了对特征丢失的鲁棒性，这是对现有工作的关键补充。
- **为什么能奏效**：
    - **TC-MPC**：终端约束强制系统在预测时域结束时进入一个“安全”的终端集，而终端代价则确保在这个集合内系统是稳定的。这种设计是MPC理论中保证稳定性的标准方法。
    - **KF**：利用系统动力学模型（此处为简单的恒速模型，$A=I$）和噪声统计特性，KF能在测量缺失时提供最优的线性无偏估计，其预测值比直接保持上一时刻的控制指令或设为零更合理。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出一个级联的控制架构：
1.  **感知层**：机载相机捕获图像，提取AprilTag特征，计算图像矩（$M_{pq}, \mu_{pq}$），并构建视觉状态向量 $q = [x_n, y_n, a_n, \arctan(1/\rho)]^T$。
2.  **状态估计层（KF）**：卡尔曼滤波器接收来自感知层的测量值 $z(k)$。当测量有效时，KF进行滤波更新（$\hat{x}_{k|k}$）；当测量丢失时，KF进行状态预测（$\hat{x}_{k|k-1}$），为MPC提供连续的估计状态。
3.  **控制层（TC-MPC）**：MPC控制器接收来自KF的估计状态 $\hat{x}_{k|k}$（或 $\hat{x}_{k|k-1}$），构建误差向量 $e(k) = \hat{x}_{k|k} - q^*$。然后，MPC求解一个带终端约束的二次规划（QP）问题，得到最优控制序列 $U^*$，并将第一个控制量 $u(k) = u^*_0$ 发送给底层飞控（PX4）。
4.  **执行层**：PX4飞控将速度指令转换为电机控制信号，驱动无人机运动。

### 3.2 关键算法与数学推导

**1. 图像矩与IBVS模型**
- 图像矩定义：$M_{pq} = \sum_x \sum_y x^p y^q I(x, y)$
- 中心矩：$\mu_{pq} = \sum_x \sum_y (x-\bar{x})^p (y-\bar{y})^q I(x, y)$，其中质心 $\bar{x} = M_{10}/M_{00}, \bar{y} = M_{01}/M_{00}$。
- 视觉特征向量：$q = [x_n, y_n, a_n, \arctan(1/\rho)]^T$，其中 $x_n = a_n \bar{x}$, $y_n = a_n \bar{y}$, $a_n = Z^* \sqrt{a^*/a}$, $\theta = \arctan(1/\rho)$。
- 交互矩阵（Interaction Matrix）：
  $$L(t) = \begin{bmatrix} -1/Z & 0 & 0 & y_n \\ 0 & -1/Z & 0 & -x_n \\ 0 & 0 & -1/Z & 0 \\ 0 & 0 & 0 & -1 \end{bmatrix}$$
  注意：论文中公式(9)的符号可能有误，此处根据标准IBVS理论给出更常见的符号。控制输入 $v_c = [v_{xd}, v_{yd}, v_{zd}, \omega_{zd}]^T$。

**2. 终端约束MPC（TC-MPC）**
- **离散化模型**：
  $$e(k+1) = A e(k) + B u(k)$$
  其中 $A = I_4$, $B = T_s L(t)$，$T_s = 1/30$ s。
- **预测模型**：
  $$E = \Phi e(k) + \Gamma U$$
  其中 $E = [e(k+1)^T, ..., e(k+N)^T]^T$, $U = [u(k)^T, ..., u(k+N-1)^T]^T$。
- **优化问题**：
  $$\min_{U} J = \sum_{i=0}^{N-1} \left[ e(k+i)^T Q e(k+i) + u(k+i)^T R u(k+i) \right] + e(k+N)^T P e(k+N)$$
  s.t.
  $$e_{min} \leq e(k+i) \leq e_{max}, \quad i=1,...,N$$
  $$u_{min} \leq u(k+i) \leq u_{max}, \quad i=0,...,N-1$$
  $$|e(k+N)| \leq \varepsilon_{term}$$
  其中 $\varepsilon_{term} = [0.5, 0.5, 1.0, 0.5]^T$ 是终端约束的容忍度。
- **求解**：将上述问题转化为标准QP形式 $\min_U \frac{1}{2} U^T H U + f^T U$，使用Gurobi求解器在线求解。

**3. 卡尔曼滤波器（KF）**
- **系统模型**：
  $$x(k) = A x(k-1) + w(k), \quad w(k) \sim \mathcal{N}(0, Q)$$
  $$z(k) = C x(k) + v(k), \quad v(k) \sim \mathcal{N}(0, R)$$
  其中 $A = I_4$, $C = I_4$。
- **预测步**：
  $$\hat{x}_{k|k-1} = A \hat{x}_{k-1|k-1}$$
  $$P_{k|k-1} = A P_{k-1|k-1} A^T + Q$$
- **更新步**：
  $$K_k = P_{k|k-1} C^T (C P_{k|k-1} C^T + R)^{-1}$$
  $$\hat{x}_{k|k} = \hat{x}_{k|k-1} + K_k (z_k - C \hat{x}_{k|k-1})$$
  $$P_{k|k} = (I - K_k C) P_{k|k-1}$$

### 3.3 模型、训练与数据集

- **模型**：本文提出的方法是一个**无模型**的控制框架，不需要离线训练。MPC的预测模型（$A, B$）是在每个控制周期根据当前交互矩阵 $L(t)$ 在线线性化得到的。卡尔曼滤波器也不需要训练，其参数（$Q, R$）是手动调整的。
- **训练**：无。
- **数据集**：无。实验是在真实环境中进行的，使用运动捕捉系统提供真值用于评估，但控制本身不依赖于此。

### 3.4 实验设置与结果

- **实验平台**：四旋翼无人机，搭载Pixhawk飞控、Intel RealSense D435i相机、Jetson Orin机载计算平台。实验在运动捕捉环境中进行。
- **任务**：无人机自主穿越一个带有AprilTag的门框（gate traversal）。
- **对比方法**：
    - IBVS：传统无约束IBVS。
    - MPC：无约束MPC。
    - MPC1 [17]：仅含输入约束的MPC。
    - MPC2：本文提出的含输入和状态约束的MPC（不含KF）。
- **评价指标**：
    - `Time (s)`：收敛时间（误差范数 $|e|$ 降至0.1以下）。
    - `RMSE_error`：特征跟踪误差的均方根。
    - `RMSE_joint`：联合RMSE，同时考虑跟踪精度和控制平滑度：$\text{RMSE}_{joint} = \sqrt{\text{mean}(e^2) + \text{mean}(||v_e||^2)}$。
- **主要结果（Table 1）**：
    - **MPC2（完全约束）** 取得了最低的 `RMSE_error` (0.6226) 和 `RMSE_joint` (0.6466)，表明其在约束下实现了最精确和平滑的控制。
    - 传统IBVS收敛最快（6.588s），但误差最大（0.7183），且轨迹振荡（Fig. 2a）。
    - 无约束MPC收敛时间较长（7.633s），误差略低（0.7014），但控制输入可能较激进。
    - 仅输入约束的MPC1收敛最慢（10.32s），但误差（0.6532）优于无约束情况。
- **KF效果（Fig. 3）**：实验表明，集成KF后，速度指令的振荡被显著抑制，控制更加平滑，证明了KF对噪声和特征丢失的鲁棒性。

### 3.5 技术细节

- **控制频率**：MPC更新频率为30 Hz（$T_s = 1/30$ s）。
- **预测时域**：$N$ 的具体值未在论文中明确给出，但这是一个关键参数。
- **约束设置**：状态约束（$e_{min}, e_{max}$）对应图像空间中的FOV约束。输入约束（$u_{min}, u_{max}$）根据PX4飞控支持的最大速度设定。此外，在IBVS控制层额外施加了1 m/s的速度限制作为安全约束。
- **求解器**：使用Gurobi优化器在线求解QP问题，利用了问题的凸性。
- **KF假设**：假设过程噪声和测量噪声均为零均值高斯分布，协方差矩阵 $Q$ 和 $R$ 为手动调参。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  提出了一种**实用且鲁棒**的无人机IBVS控制方案，同时解决了**约束下的稳定性**和**特征丢失下的鲁棒性**两个实际难题。
    2.  通过**终端约束MPC**为IBVS提供了理论上的稳定性保证，这是对现有基于MPC的IBVS工作的重要改进。
    3.  将**卡尔曼滤波**作为“传感器故障”的软切换机制，巧妙地处理了视觉特征短暂丢失的问题，提升了系统的可靠性。
    4.  通过**真实飞行实验**验证了方法的有效性，而非仅仅停留在仿真阶段，增强了其工程应用价值。
- **对未来研究/应用的影响**：
    - 为无人机在GPS拒止环境下的自主导航（如穿越门、窗口、桥梁等）提供了一个可靠的解决方案。
    - 本文的框架可以推广到其他基于视觉的机器人控制任务，如机械臂抓取、移动机器人跟踪等。
    - 将MPC与状态估计器（如KF）结合以应对感知缺失，是一种具有普适性的鲁棒控制设计思路。
- **局限与开放问题**：
    - **依赖AprilTag**：当前实验依赖人工标记（AprilTag）进行特征提取。作者指出未来工作将扩展到**真实门框检测**，这需要更鲁棒的视觉检测算法。
    - **高速机动**：论文主要验证了相对平稳的穿越任务。作者提到未来将研究**高速、敏捷**的机动，这对MPC的实时性和KF的预测精度提出了更高要求。
    - **参数调优**：MPC的权重矩阵（$Q, R, P$）和KF的噪声协方差（$Q, R$）需要手动调整，缺乏自适应机制。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是图像矩？** 可以把它想象成图像的“统计特征”。比如，一阶矩（$M_{10}, M_{01}$）可以算出图像的“重心”在哪里；二阶矩（$M_{20}, M_{02}$）可以描述图像是“胖”还是“瘦”，以及它的朝向。本文用这些矩来构造控制器的输入，而不是直接用成千上万个像素点，大大简化了计算。
- **终端约束MPC为什么能保证稳定？** 想象一下，MPC就像一个“短视”的规划者，它只看未来几步。终端约束就像一个“安全网”，强制要求规划者在最后一步必须进入一个已知的、安全的“着陆区”（终端集）。只要规划者能保证进入这个区域，并且我们知道在这个区域内系统是稳定的，那么整个控制过程就是稳定的。
- **卡尔曼滤波如何补偿特征丢失？** 假设你正在闭着眼睛走路（视觉丢失），但你知道自己之前的速度和方向（系统模型）。卡尔曼滤波器就像一个“惯性导航系统”，它会根据你最后已知的状态和运动模型，预测你下一步可能在哪里。当视觉恢复时，它会用新的测量值来修正预测的误差。在视觉丢失的短暂瞬间，控制器就使用这个“预测”的位置来继续生成控制指令，而不是突然“失明”导致失控。
