## 0. 概览（Concise Summary）

- **主题（Topic）**：面向通用图形处理器（GPGPU）张量核心（Tensor Core）的混合精度融合点积（Fused Dot Product, FEDP）单元微架构设计。
- **问题（Problem）**：现有的开源张量核心实现依赖于离散的算术单元（如Berkeley HardFloat, FPnew），导致延迟高、累积舍入误差大、资源利用率低，且缺乏对现代特性（如整数运算、稀疏性、Microscaling格式）的统一支持。
- **方法（Method）**：提出名为 **Ten-Four** 的、可配置的混合精度融合点积单元。它将浮点与整数算术流水线融合在一个统一的架构中，并集成了稀疏通道时钟门控和Microscaling (MX) 格式支持，作为开源RISC-V Vortex GPGPU张量核心单元（TCU）扩展的一部分。
- **创新（Innovation）**：
    1.  **统一流水线**：首次在开源GPGPU张量核心中，将整数点积流水线无缝融合进浮点流水线，通过新颖的加数拆分策略（addend-splitting）复用关键资源。
    2.  **稀疏通道时钟门控**：提出一种基于输入零值检测的稀疏通道掩码（Sparse Lane Mask）方法，通过时钟门控减少动态功耗，使双端稀疏张量核心设计更实用。
    3.  **原生Microscaling支持**：通过每通道缩放因子补偿方案，在不改变主流水线结构的前提下，原生支持OCP MX格式。
- **意义（Significance）**：填补了开源GPGPU领域高性能张量核心实现的空白。其性能超越了基于现有开源库的实现，并在7nm工艺下达到与NVIDIA A100相当的每单元吞吐量，同时支持更广泛的格式和特性，为硬件-软件协同设计提供了灵活、高性能的开源平台。
- **一句话总结**：本文提出了 **Ten-Four**，一个开源、可配置、高性能的混合精度融合点积单元，通过统一浮点/整数流水线、稀疏通道时钟门控和原生Microscaling支持，显著提升了开源GPGPU张量核心的性能和能效。

## 1. 研究动机（Motivation）

- **问题与空白**：深度学习工作负载（如GEMM）在GPU上占主导地位，而张量核心是加速这些运算的关键。然而，现有的开源GPGPU张量核心设计（如Ventus, Virgo）普遍采用离散的浮点算术单元库（如Berkeley HardFloat, FPnew）来构建点积单元。这种方法存在三个主要缺陷：
    1.  **高延迟**：离散单元需要多次对齐、归一化和舍入，导致关键路径长，延迟高。
    2.  **累积舍入误差**：每次中间运算的舍入会累积误差，降低数值精度。
    3.  **资源利用率低**：为不同格式（FP16, BF16, FP8等）和运算（乘法、加法）分别实例化独立单元，导致面积和功耗浪费。
- **重要性**：随着NVIDIA和AMD在商业GPU中引入2:4结构化稀疏性、Microscaling (MX) 格式等先进特性，开源社区与商业实现之间的差距日益扩大。缺乏一个高性能、功能丰富的开源张量核心实现，阻碍了学术界和工业界在硬件-软件协同设计、新型数据格式探索等方面的研究。
- **挑战**：设计一个统一的、可配置的融合点积单元，需要解决以下挑战：
    - 如何将不同位宽的浮点（FP16, BF16, TF32, FP8, BF8）和整数（INT8, INT4）乘法高效地集成到同一个流水线中。
    - 如何在不增加过多开销的情况下，将32位整数加数（C）融合进主要为浮点设计的累加器。
    - 如何利用稀疏性来降低功耗，而不是简单地增加吞吐量。
    - 如何支持Microscaling格式，同时保持与现有流水线的兼容性。

## 2. 创新点（Innovation）

论文的关键创新点在于提出了一个高度集成和可配置的融合点积单元微架构，具体体现在：

1.  **统一的混合精度浮点/整数流水线**：
    - **创新**：不同于以往将浮点和整数运算分开处理的设计，Ten-Four将整数点积流水线无缝融合进浮点流水线中。它复用了浮点流水线中昂贵的累加器（Stage 3）和归一化/舍入逻辑（Stage 4）。
    - **不同之处**：通过一个新颖的**加数拆分策略**，将32位整数加数C拆分为低25位（C_LOW）和高7位（C_HI）。C_LOW与乘积项一起在累加器中处理，C_HI则绕过累加器，在最后阶段与累加结果拼接，从而避免了增加累加器位宽带来的巨大开销。
    - **为何奏效**：这种设计最大限度地复用了硬件资源，避免了为整数运算单独设计一套完整的累加和归一化逻辑，从而在支持更多数据类型的同时，保持了较小的面积和较低的功耗。

2.  **稀疏通道时钟门控（Sparse Lane Clock-Gating）**：
    - **创新**：提出了一种基于输入零值检测的**稀疏通道掩码**机制。当某个通道的输入A或B为零时，该通道的流水线寄存器被时钟门控，从而消除该通道的开关活动，降低动态功耗。
    - **不同之处**：现有NVIDIA的稀疏张量核心（如2:4结构化稀疏）主要通过在K维度上减少微操作来获得吞吐量提升，而非直接降低功耗。而Ten-Four的方案专注于功耗优化，且与结构化稀疏正交，可以叠加使用。
    - **为何奏效**：在深度学习中，权重和激活值常常具有很高的稀疏性。通过时钟门控，可以有效地将这种稀疏性转化为实际的功耗节省，而无需改变内积计算的基本结构（如使用外积单元），从而保持了面积效率。

3.  **原生Microscaling (MX) 格式支持**：
    - **创新**：提出了一种**每通道缩放因子补偿方案**。传统的MX实现是在点积求和之后再应用缩放因子，这与Ten-Four将加数C与乘积项并行处理的流水线不兼容。Ten-Four通过将缩放因子分解到每个乘积项上，实现了“先缩放，后累加”，从而无缝地集成了MX支持。
    - **不同之处**：避免了为MX格式设计独立的、与主流水线并行的处理路径，仅需在乘法器后增加简单的缩放逻辑。
    - **为何奏效**：这种方案巧妙地利用了乘法的分配律，将全局的缩放操作转化为局部的、每通道的缩放，完美适配了Ten-Four的流水线结构，实现了对MX格式的原生、低开销支持。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

Ten-Four的整体架构是一个4级流水线的融合点积单元，其核心是一个 [M x N] 的FEDP单元网格，每个单元执行K个元素的点积。在Vortex GPGPU中，它被实现为一个 [8x4] 的网格，对应8x4x8 (FP16) 或 8x4x16 (FP8) 的WMMA指令形状。

- **Stage 1: 乘法、指数差矩阵与异常处理**
    - **共享乘法器**：采用按尾数位宽分组的共享乘法器方案。例如，FP16、BF16、TF32共享一个11x11位的Wallace树乘法器；FP8和BF8共享两个4x4位的乘法器。
    - **指数差矩阵**：并行计算所有N个操作数（K个乘积 + 1个加数C）两两之间的指数差，形成一个差矩阵。通过分析差矩阵的符号位，快速确定最大指数。
    - **异常处理**：并行检测NaN、无穷大、零乘无穷大等异常情况，并生成相应的异常标志。

- **Stage 2: 最大指数确定、移位量与尾数对齐**
    - **最大指数**：通过归约操作从Stage 1生成的一热编码掩码中确定最大指数。
    - **移位量**：复用Stage 1的指数差矩阵，通过取反等操作快速计算出每个操作数所需的对齐移位量。
    - **尾数对齐**：根据移位量对乘积的尾数进行右移对齐，并保留额外的对齐位和粘滞位（sticky bits）以保证精度。

- **Stage 3: 累加**
    - **加数集成**：加数C的指数从一开始就参与最大指数查找，其尾数也在此阶段与乘积项一起对齐。
    - **符号处理**：对负的尾数进行位取反，并通过一个popcount电路统计负数的个数，将这个总数作为一个额外的修正项送入CSA树。
    - **CSA树**：使用进位保留加法器（CSA）树高效地将多个对齐后的尾数（包括修正项）压缩为和与进位两个值。根据操作数数量（6或10），选择标准CSA树或MOD-4分组CSA树。
    - **粘滞位**：将各通道的粘滞位进行归约OR，用于最终的舍入。

- **Stage 4: 归一化与舍入**
    - **归一化**：使用预测前导零计数器（LZAC）确定归一化移位量，并对累加结果进行左移。
    - **舍入**：应用标准的向最近偶数舍入（RNE）模式，使用LSB、Guard、Round和Sticky位。
    - **结果选择**：如果发生异常，结果被覆盖为IEEE标准的NaN或无穷大表示。

### 3.2 关键算法与数学推导

1.  **融合点积运算**：
    $$D_{FP32}^{m,n} = \sum_{i=0}^{K-1} (A_{FP16}^{m,i} \times B_{FP16}^{i,n}) + C_{FP32}^{m,n}$$

2.  **指数转换**（以FP16到FP32为例）：
    $$CONV_{FP16/FP32} = BIAS_{FP32} - (2 \times BIAS_{FP16}) + 1$$
    $$EXP_{FP32} = EXP_A + EXP_B + CONV_{FP16/FP32}$$

3.  **整数加数拆分策略**：
    将32位整数加数C拆分为：
    $$C = C_{HI} \times 2^{25} + C_{LOW}$$
    其中 $C_{LOW}$ 是低25位，$C_{HI}$ 是高7位。$C_{LOW}$ 在Stage 3的CSA树中与乘积项累加，$C_{HI}$ 则绕过累加器，在Stage 4与累加结果拼接。

4.  **Microscaling (MX) 格式的缩放因子补偿**：
    传统方式：
    $$D = X(A)X(B) \sum_{i=1}^{k} (A_i \times B_i) + C$$
    Ten-Four的等效方式（利用分配律）：
    $$D = \sum_{i=1}^{k} [X(A)X(B) (A_i \times B_i)] + C$$
    这使得缩放操作可以在每个通道的乘法后立即进行，与流水线结构兼容。

### 3.4 实验设置与结果

- **FPGA实现**：
    - **平台**：AMD Xilinx Alveo U55C FPGA。
    - **目标频率**：300 MHz。
    - **对比对象**：基于Berkeley HardFloat、FPnew和Xilinx DSP IP的等效张量核心实现。
    - **结果**：
        - **吞吐量**：Ten-Four在32线程/线程束配置下达到32.592 GFLOPS，相比HardFloat (~11.9 GFLOPS) 快约2.7倍，相比FPnew (~4.2 GFLOPS) 快约7.9倍（见图4）。
        - **资源利用率**：Ten-Four相比HardFloat减少了37-38%的LUT和52-66%的FF，且完全不使用DSP块（见图5）。
        - **功耗**：稀疏通道时钟门控在双端稀疏度达到90%时，可节省34.9-47.6%的动态功耗（见图7）。

- **ASIC实现**：
    - **工艺**：ASAP7 7nm FinFET PDK。
    - **目标频率**：1.5 GHz。
    - **结果**：
        - **性能**：在32线程/线程束配置下，单个Ten-Four张量核心可提供高达1.617 TFLOPS (FP8/BF8) 的峰值吞吐量。
        - **能效**：峰值能效达到2.771 TFLOPS/W。
        - **对比**：在与NVIDIA A100（同为7nm工艺）的等配置对比下，Ten-Four的每单元峰值吞吐量高出约12.3%。

- **数值精度验证**：
    - **方法**：使用PyTorch生成CUDA内核，在NVIDIA RTX 4090 (Ada架构) 上运行作为硬件参考。
    - **结果**：Ten-Four在所有支持的浮点和整数格式上，对超过100万个随机测试向量产生了与NVIDIA Ada张量核心**逐位精确**的结果。

### 3.5 技术细节

- **Wallace树乘法器 (WTMUL)**：用于高效地生成和压缩部分积。论文选择不使用Radix-4 Booth编码，因为在目标位宽（4-11位）下，其开销超过了收益。
- **Kogge-Stone加法器 (KSA)**：用于最终的累加和求和，以其低扇出和高速特性著称，尽管面积较大。
- **前导零计数器 (LZAC)**：用于预测归一化所需的移位量，是归一化阶段的关键组件。
- **向最近偶数舍入 (RNE)**：标准的IEEE 754舍入模式，用于最小化舍入误差。
- **可配置性**：Ten-Four的RTL设计是可配置的，可以在编译时选择性地实例化支持的输入格式子集，以根据应用需求优化面积和效率。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了开源GPGPU领域缺乏高性能、功能丰富的张量核心实现的问题。
    - 提出了一个统一、高效的融合点积单元微架构，显著优于基于离散算术单元的设计。
    - 引入了实用的稀疏通道时钟门控和原生Microscaling支持，弥合了开源设计与商业实现之间的差距。

- **对未来研究/应用的影响**：
    - **硬件-软件协同设计**：Ten-Four的开源、可配置特性使其成为探索新型数据格式（如自定义块量化格式）、稀疏模式和算术策略的理想平台。
    - **学术研究**：为计算机体系结构、VLSI设计领域的研究者提供了一个高性能的基线设计，可以在此基础上进行更深入的创新。
    - **开源硬件生态**：作为Vortex GPGPU的一部分，Ten-Four有助于构建一个更完整、更强大的开源GPGPU生态系统，推动RISC-V在高性能计算和AI领域的应用。

- **局限与开放问题**：
    - **稀疏性支持**：当前的稀疏通道时钟门控仅用于功耗优化，并未像NVIDIA的2:4结构化稀疏那样提供吞吐量提升。作者提到，该方案与结构化稀疏正交，可以结合使用，但并未在本文中实现和评估。
    - **Microscaling开销**：在SGEMM内核分析中，MXFP8/MXBF8的吞吐量略低于纯FP8/BF8，原因是缩放因子元数据的访问带来了额外开销。如何进一步优化元数据处理是一个开放问题。
    - **验证范围**：数值精度验证主要针对NVIDIA Ada架构，对其他GPU架构（如AMD）的兼容性未作讨论。

## 5. 澄清与简化（Clarifications and Simplifications）

- **融合点积 vs. 离散点积**：
    - **离散**：想象一个计算 `(A1*B1) + (A2*B2) + C` 的电路。离散设计会先计算 `A1*B1`，得到一个结果，然后舍入；再计算 `A2*B2`，舍入；最后将这两个舍入后的结果与C相加，再次舍入。每次舍入都会引入误差。
    - **融合**：Ten-Four的设计是，将所有乘法结果和C的尾数对齐后，一次性送入一个大的加法器（CSA树）中，最后只做一次归一化和舍入。这就像把所有数字的“小数部分”对齐后，一次性加起来，最后再决定如何“四舍五入”，精度更高，延迟也更低。

- **加数拆分策略（类比）**：
    - 想象你要计算 `(1234 + 5678) + 9999999`。你的加法器只能处理4位数。
    - **传统方法**：你需要一个能处理8位数的加法器。
    - **Ten-Four的方法**：把 `9999999` 拆成 `999` 和 `9999`。先用4位加法器计算 `1234 + 5678 + 9999`，得到结果 `16911`（溢出部分为1）。然后，将溢出部分 `1` 与高位 `999` 相加，得到 `1000`。最后，将 `1000` 和 `6911` 拼接起来，得到 `10006911`。这样，你就用一个小加法器完成了大数加法。Ten-Four正是用类似的方法，用25位的累加器处理了32位的整数加法。

- **稀疏通道时钟门控（类比）**：
    - 想象
