## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人控制领域的**世界动作模型（World Action Model, WAM）**。该模型旨在统一视觉世界建模与机器人动作预测，实现更智能、更通用的机器人操控。
- **问题（Problem）**：现有机器人策略主要分为两类：1）**视觉-语言-动作（VLA）模型**，虽然泛化能力强，但缺乏对世界动态（如物理规律、物体交互）的深层理解，导致行为模仿而非真正的因果推理；2）**视频生成模型+逆动力学模型（VGM+IDM）** 的两阶段范式，虽然利用了视频数据的时空先验，但存在**级联误差**问题（视频预测的误差会累积并传递给动作预测）。此外，如何高效地利用异构多模态数据（如无动作标签的视频、不同机器人形态的数据）进行统一训练，并实现低延迟的实时部署，也是关键挑战。
- **方法（Method）**：提出**MotuBrain**，一个基于**UniDiffuser**框架的**统一世界动作模型**。它采用**三流混合Transformer（Mixture-of-Transformers, MoT）** 架构，分别处理文本、视频和动作三种模态。通过联合建模视频和动作的生成过程，模型可以支持多种推理模式（如策略建模、世界建模、视频生成等）。此外，还引入了**统一多视角表示**、**跨本体统一动作表示**、**V2A风格非对称注意力**以及一套**推理加速栈**（包括噪声采样优化、编译、FP8量化、DiT缓存等），以实现高效的实时控制。
- **创新（Innovation）**：
    1.  **统一架构与训练**：在UniDiffuser框架下，将视频和动作作为两个连续的模态进行联合建模和调度，实现了从异构多模态数据（包括无动作标签的视频）中学习，突破了传统VLA模型对动作标签的依赖。
    2.  **三流MoT与H桥注意力**：设计了独立的文本、视频、动作三流架构，并采用**H桥（H-bridge）注意力**设计，在保证跨模态交互的同时，显著提升了计算效率。
    3.  **实用化设计与推理优化**：提出了**统一多视角**、**跨本体统一动作表示**、**V2A非对称推理**（允许在推理后期仅更新动作流，冻结视频流）以及一套**超过50倍加速**的推理优化栈，使大型世界动作模型能够满足实时机器人控制的需求。
- **意义（Significance）**：MotuBrain证明了**统一世界动作模型**在**通用性、可控性和实时可部署性**上可以同时取得显著进步。它在RoboTwin 2.0基准测试中取得了**95.8%（干净场景）和96.1%（随机场景）** 的最高成功率，在WorldArena世界模型评估中也获得了最高分。更重要的是，它仅需**50-100条**同本体轨迹即可快速适应新的人形机器人，并展现出长时域任务执行和内在的自我纠错能力，为构建更通用、更智能的机器人基础模型提供了有力范式。
- **一句话总结**：MotuBrain通过一个统一的世界动作模型架构和高效的推理优化，实现了从异构数据中学习、在多种机器人上高效迁移、并能在真实世界中进行长时域、灵巧操作的先进机器人控制系统。

## 1. 研究动机（Motivation）

- **核心问题**：当前的机器人学习范式存在根本性缺陷。**VLA模型**虽然擅长理解语义和指令，但它们在静态图文数据上预训练，缺乏对**细粒度世界动态**（如物体如何因力而移动、形变）的感知和预测能力，导致其行为更像是“模仿”而非“理解”。另一方面，**VGM+IDM**的两阶段方法虽然利用了视频中的动态信息，但视频预测的微小误差会随着时间步长累积放大，最终导致动作预测失败，即**级联误差**问题。
- **重要性**：一个真正的智能机器人需要同时具备两种能力：1）**理解世界**（世界模型）：预测环境在自身动作下的未来状态；2）**作用于世界**（策略模型）：根据当前状态和目标，生成正确的动作序列。将这两者割裂开来（如VGM+IDM）或只关注后者（如VLA）都限制了机器人的智能水平。
- **挑战**：
    1.  **数据异构性**：如何有效利用海量的、但缺少动作标签的互联网视频数据？如何融合来自不同机器人形态（如单臂、双臂、人形）的异构数据？
    2.  **模型统一性**：如何设计一个单一的模型，既能进行高质量的视频预测（世界模型），又能进行精确的动作生成（策略模型），并且两者能相互促进，而不是相互干扰？
    3.  **实时性**：大型扩散模型推理速度慢，如何将其加速到足以满足机器人高频控制（如10Hz以上）的实时性要求？

## 2. 创新点（Innovation）

1.  **统一世界动作建模框架**：基于**UniDiffuser**，将视频和动作视为两个连续的模态，在一个统一的生成式框架下进行联合建模和调度。这使得同一个模型可以支持五种不同的推理模式（VLA策略、世界模型、视频生成、逆动力学、联合视频-动作预测），从根本上解决了VGM+IDM的级联误差问题，并实现了从异构多模态数据中学习。
2.  **三流混合Transformer（MoT）与H桥注意力**：
    - **三流架构**：独立的**文本流**（用于语义理解）、**视频流**（用于视觉动态建模）和**动作流**（用于动作生成）。这种设计避免了模态间的信息混淆，特别是独立的文本流增强了指令跟随能力。
    - **H桥注意力**：在Transformer层中，仅在中间50%的层使用全视频-动作联合注意力，而在底部和顶部的25%层使用解耦注意力。这大幅降低了计算成本，同时保留了在中间层进行语义对齐和策略基础学习的能力。
3.  **实用化设计与推理加速栈**：
    - **统一多视角表示**：通过为不同视角的token引入**3D旋转位置编码（3D RoPE）** 的视角相关偏移，可以无缝支持任意数量和布局的相机视角。
    - **跨本体统一动作表示**：将动作表示为相对于当前观测帧末端执行器状态的**相对位姿**（位置差、旋转差），提高了不同机器人形态和初始位姿下的动作空间兼容性，减少了迁移所需的数据量。
    - **V2A风格非对称推理**：训练时采用视频不关注动作、动作关注视频的**非对称注意力**。这使得推理时可以在少量联合去噪步骤后，**冻结视频流**，仅更新动作流，从而大幅降低计算量。
    - **推理加速栈**：组合了**噪声采样优化**（减少去噪步数）、**torch.compile**（算子融合）、**FP8量化**（降低内存和计算开销）、**DiT缓存**（利用去噪步骤间的冗余跳过部分计算）和**V2A推理**，实现了超过**50倍**的端到端加速，使模型达到**11Hz**的推理频率。

## 3. 主要内容（Main Content）

### 3.1 设计架构与方法

MotuBrain的整体架构如图1所示，其核心是一个**三流混合Transformer（MoT）**。

- **输入处理**：
    - **文本**：语言指令通过文本编码器转换为文本token。
    - **视频**：条件帧（当前观测）和未来帧（待预测）通过Vidu VAE编码为视频latent。
    - **动作**：未来动作通过动作编码器转换为动作token。
- **三流MoT**：模型包含三个并行的Transformer流：
    - **文本流**：作为条件分支，其隐藏状态参与注意力计算，但不输出预测头。
    - **视频流**：通过流匹配（Flow Matching）预测视频latent的速度场。
    - **动作流**：通过流匹配预测动作token的速度场。
- **H桥注意力**：如图1所示，在Transformer的中间层，视频、动作和文本token之间进行**全联合注意力**；而在浅层和深层，视频和动作流独立处理，仅与文本流交互。这平衡了跨模态交互与计算效率。
- **多视角支持**：对于多视角输入，每个视角独立编码后在token级别拼接。通过为不同视角的token在空间维度上添加不同的3D RoPE偏移，模型可以区分并处理来自不同视角的信息。

### 3.2 关键算法与数学推导

- **统一动作表示**：
    设绝对末端执行器动作为 $E_{abs} = \{e^{abs}_1, ..., e^{abs}_n\}$，条件帧的末端执行器状态为 $s$。相对动作 $E_{rel} = \{e^{rel}_1, ..., e^{rel}_n\}$ 定义为：
    $$e^{rel}_i = e^{abs}_i \ominus s$$
    其中 $\ominus$ 表示逐分量位姿差。对于位姿 $e = (p, R, g)$（位置 $p$，旋转 $R$，夹爪状态 $g$）：
    $$e^{abs}_i = (p_i, R_i, g_i), \quad s = (p_s, R_s, g_s)$$
    $$e^{rel}_i = (p_i - p_s, R_s^{-1} R_i, g_i)$$
    最终动作维度为10（3维位置 + 6维旋转表示 + 1维夹爪状态）。

- **训练目标**：
    两阶段预训练的总损失函数为视频损失和动作损失的加权和：
    $$L = \lambda_v L_v + \lambda_a L_a$$
    其中 $L_v = \text{MSE}(v_{out}, v_{target})$, $L_a = \text{MSE}(a_{out}, a_{target})$。

- **V2A风格推理**：
    在推理时，采用一个短期的联合去噪前缀，随后是仅动作去噪的后缀：
    $$(z^{(t+1)}_v, z^{(t+1)}_a) = \begin{cases} \Phi_{joint}(z^{(t)}_v, z^{(t)}_a), & t < N \\ (z^{(N)}_v, \Phi_{act}(z^{(t)}_a; z^{(N)}_v)), & t \ge N \end{cases}$$
    其中 $z^{(t)}_v$ 和 $z^{(t)}_a$ 分别是第 $t$ 步的视频和动作latent。在 $N$ 步之后，视频latent $z^{(N)}_v$ 被冻结，后续步骤仅更新动作token。

- **DiT缓存**：
    利用去噪步骤间的相似性跳过部分计算。设 $v_t$ 为第 $t$ 步的速度预测，相似度 $s_t$ 定义为：
    $$s_t = \frac{\langle v_t, v_{t-1} \rangle}{\|v_t\|_2 \|v_{t-1}\|_2}$$
    当 $s_t > \gamma$ 时，跳过后续 $k$ 步的DiT计算，用历史预测近似：
    $$\hat{v}_{t+j} \approx v_t, \quad j = 1, ..., k$$

- **实时执行中的动作融合**：
    为减少异步控制中的块边界不连续性，将当前块未执行的动作作为下一块的软约束。融合后的动作为：
    $$\tilde{a}^{new}_i = w_i a^{remain}_i + (1 - w_i) a^{new}_i$$
    其中权重 $w_i$ 由指数衰减函数 $g(\rho_i) = \frac{\rho_i (e^{\rho_i} - 1)}{e - 1}$ 决定，$\rho_i$ 是融合窗口内的归一化进度。

### 3.3 模型、训练与数据集

- **基础模型**：视频生成模型**Vidu**作为MotuBrain的基础模型。
- **数据金字塔**：预训练数据采用四级金字塔结构：
    1.  **互联网视频**：训练视频生成基础模型。
    2.  **第一人称视频**：提供接近具身操作的交互模式。
    3.  **异构本体数据**：来自不同双臂机器人平台的数据。
    4.  **特定本体数据**：目标机器人配置下的数据，用于最终对齐。
- **两阶段预训练**：
    - **第一阶段**：冻结随机初始化的动作分支，仅训练视频分支。目标是在第一人称和异构本体数据上，将互联网视频先验适应到具身操作，获得一个能理解双臂交互动态的视频世界模型。此阶段采用**噪声条件增强**（公式1）来提高对不完美视觉条件的鲁棒性。
    - **第二阶段**：冻结第一阶段训练好的视频分支，仅训练动作分支。目标是在异构本体数据上，学习一个**跨本体的统一动作表示**（相对末端执行器位姿），使动作空间更一致。
- **后训练**：在特定本体数据上微调。有两种设置：
    - **非自回归（Non-AR）**：一次性去噪整个观测窗口内的所有视频和动作token。
    - **自回归（AR）**：基于块（chunk）的因果分解，用于长时域任务。使用块因果注意力掩码（图2d），每个块只能关注过去块的干净视觉上下文。

### 3.4 实验设置与结果

- **仿真环境评估（RoboTwin 2.0）**：
    - **设置**：使用2500条干净场景和25000条随机场景的演示数据训练。视频下采样到5Hz，动作到10Hz。
    - **结果（表3）**：MotuBrain在干净场景和随机场景下分别达到**95.8%** 和**96.1%** 的平均成功率，远超所有基线模型（包括π0.5, X-VLA, Motus, LingBot-VA等）。它是唯一一个在随机评估中平均分超过95%的模型。
    - **任务级分析（表4）**：在50个任务中，MotuBrain在24个（干净）和25个（随机）任务上获得满分。其优势集中在需要**鲁棒的多阶段操作**、**精确交互**（如Handover Block, Open Microwave）和**精细空间排列**（如Blocks Ranking Size）的任务上。
    - **扩展性分析（图3, 4）**：MotuBrain展现出比VLA基线更强的**任务扩展性**和**数据效率**。增加任务多样性比单纯增加数据量更有效。

- **世界模型评估（WorldArena）**：
    - **设置**：使用2500条高分辨率轨迹训练，在**前向动力学模式（FDM）** 下评估。
    - **结果（图5, 表5）**：MotuBrain获得了最高的**EWMScore（63.77）**，在**运动质量（Motion Quality）** 维度上排名第一（Flow Score, Motion Smoothness, Dynamic Degree均为最高）。这表明MotuBrain预测的运动是**实质性、平滑且集中在相关区域**的，成功弥合了“感知-功能”鸿沟。

- **真实世界实验**：
    - **设置**：从预训练模型出发，仅用**50-100条**同本体轨迹进行快速适配。无需VLM规划器、外部记忆等辅助模块。
    - **任务**：在多种人形机器人上执行**关东煮制作**、**调制鸡尾酒**、**插花**等长时域、灵巧操作任务。
    - **结果（表6, 7, 8）**：在关东煮任务中得分**98.54**，鸡尾酒任务得分**97.34**，插花任务得分**83.30**。
    - **定性分析**：模型展现出**内在的自我纠错能力**（如插花失败后调整）、**双臂异步泛化能力**（一只手臂任务提前完成不影响另一只）和**场景泛化能力**（在未见过的环境中成功执行）。

### 3.5 技术细节

- **噪声采样**：为视频和动作使用独立的基于信噪比（SNR）的时间步采样。视频使用 `timeshift=6`（倾向于更嘈杂的区域），动作使用 `timeshift=1`（更均匀）。这使得模型在嘈杂视觉条件下也能预测准确动作，从而允许将推理步数从50减少到30。
- **编译**：使用 `torch.compile` 优化推理图并融合算子，减少重复去噪的开销。
- **FP8量化**：将符合条件的 `nn.Linear` 层替换为FP8线性层，使用 `float8_e4m3fn` 格式和逐张量缩放，通过 `torch._scaled_mm` 计算矩阵乘法，降低内存带宽和计算成本。
- **动作平滑**：采用Savitzky-Golay滤波器对动作块进行平滑，并利用**频率感知插值**根据模型动作频率和底层控制频率的比例进行插值，保持预测的速度轮廓。
- **实时执行**：将模型推理循环与机器人动作执行循环**解耦**。采用**RTC启发式策略**，通过一个延迟队列 `Q` 保守估计下一次推理的延迟，并据此调整融合窗口，使异步执行在波动的网络和模型延迟下更稳定。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - **解决了级联误差问题**：通过统一建模视频和动作，避免了VGM+IDM范式中视频预测误差的累积。
    - **实现了从异构数据中学习**：证明了世界动作模型可以从海量、异构、多模态数据（包括无动作标签的视频）中有效学习，这是VLA模型难以做到的。
    - **弥合了感知-功能鸿沟**：MotuBrain在WorldArena（感知评估）和RoboTwin（功能评估）上同时取得最佳成绩，表明其学到的世界
