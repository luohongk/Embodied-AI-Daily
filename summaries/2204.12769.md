# Dynamic Registration: Joint Ego Motion Estimation and 3D Moving Object Detection in Dynamic Environment

- arXiv: https://arxiv.org/abs/2204.12769v1
- arXiv ID: 2204.12769

---

## 0. 概览（Concise Summary）

- **主题（Topic）**：该论文属于机器人定位与感知领域，具体关注在动态环境下的激光雷达（LiDAR）点云配准问题。
- **问题（Problem）**：传统点云配准算法（如ICP、NDT）假设环境是静态的，但在真实动态场景中，移动物体会导致配准失败，从而影响自车运动估计的精度。同时，如何从点云中实例级地分割出动态物体也是一个挑战。
- **方法（Method）**：提出了一种名为“动态配准（Dynamic Registration）”的迭代框架。该框架首先使用3D目标检测网络识别所有潜在物体并暂时移除，然后进行初始配准。接着，通过将检测到的物体投影到前一帧并计算重投影误差，来分割静态和动态物体。最后，将静态物体与环境点云合并，并迭代地进行配准和运动分割，直到没有新的静态物体产生。
- **创新（Innovation）**：核心创新在于将3D目标检测与点云配准相结合，通过“先移除、再分割、后合并”的迭代策略，实现了自车运动估计与3D动态物体检测的联合解耦。这与传统方法（将动态点视为异常值或仅进行语义分割）不同，能够输出实例级别的动态物体。
- **意义（Significance）**：该方法作为一个通用模块，可以集成到任何现有的LiDAR SLAM系统中，提升其在动态环境下的定位鲁棒性。同时，它还能输出实例级的动态物体信息，为后续的目标跟踪、路径规划等任务提供支持。
- **一句话总结**：本文提出一种名为“动态配准”的迭代框架，通过联合3D目标检测与点云配准，在动态环境中同时实现高精度的自车运动估计和实例级动态物体分割。

## 1. 研究动机（Motivation）

- **解决的问题**：在动态环境中进行机器人定位（自车运动估计）时，移动物体（如汽车、行人）会严重干扰点云配准算法（如ICP、NDT），导致定位失败或精度下降。传统的SLAM系统大多假设环境是静态的，无法处理动态场景。
- **为什么重要**：在自动驾驶、机器人导航等真实应用中，环境充满了动态物体。准确的定位是安全决策和路径规划的基础。同时，感知并理解动态物体（如其他车辆、行人）对于避障和交互也至关重要。因此，需要一种方法既能鲁棒地估计自车运动，又能精确地检测和分割动态物体。
- **挑战在哪里**：
    1.  **耦合性**：自车运动与物体运动是耦合的。在不知道自车运动的情况下，很难判断一个物体是静止的还是运动的（例如，一辆静止的汽车在自车运动时，其点云在传感器坐标系下也是移动的）。
    2.  **实例级分割**：许多现有方法（如基于运动或语义分割）只能分割出“运动点”或“语义点”，无法得到完整的物体实例（如一辆完整的汽车）。这使得后续的物体跟踪和状态估计变得困难。
    3.  **实时性与精度平衡**：在LiDAR点云上进行3D实例分割计算量大，而3D目标检测在速度和精度上取得了更好的平衡。

## 2. 创新点（Innovation）

- **关键创新**：提出了一个名为“动态配准（Dynamic Registration）”的迭代框架，该框架将3D目标检测与经典的点云配准算法（如ICP、NDT）巧妙地结合起来。
- **与现有方案的不同**：
    - **不同于鲁棒SLAM**：鲁棒SLAM将动态点视为异常值并剔除，但无法提供物体级别的信息。
    - **不同于SLAM+跟踪**：SLAM+跟踪方法通常需要复杂的联合状态估计（如SLAMMOT），而本文提出的方法更通用、更轻量，可以作为一个插件集成到现有SLAM系统中。
    - **不同于纯运动分割**：纯运动分割方法可能只能分割出部分运动点，而本文通过3D检测获得完整的物体边界框，从而能分割出完整的物体实例。
    - **不同于“移除所有物体”**：简单地移除所有检测到的物体会丢失静态物体（如路边停放的车辆）提供的丰富特征，反而可能降低配准精度。本文的迭代策略能够识别并保留静态物体。
- **为什么能奏效**：
    1.  **3D目标检测提供先验**：利用成熟的3D检测器（如PointPillars）快速获得所有潜在物体的实例级边界框，为后续处理提供了强有力的先验信息。
    2.  **迭代解耦**：通过“先移除所有物体 -> 初始配准 -> 运动分割 -> 合并静态物体 -> 再次配准”的迭代过程，逐步将自车运动与物体运动解耦。每一次迭代都使用更“干净”的环境点云（包含静态物体）进行配准，从而得到更精确的位姿，进而更准确地分割出动态物体。
    3.  **数据关联简化**：利用短时间间隔内物体位移较小的特点，采用简单的最近邻欧氏距离进行数据关联，避免了复杂的3D IoU计算，提高了效率。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

系统架构如图2所示，包含三个主要模块：

1.  **3D目标检测与移除（Object Detection And Removal）**：
    - 对连续两帧点云 $M_{t-1}$ 和 $M_t$ 分别运行3D目标检测网络（PointPillars），得到检测结果 $D_{t-1}$ 和 $D_t$。
    - 根据检测到的3D边界框，将框内的所有点云暂时移除，得到“移除所有物体后的点云” $M^{RA}_{t-1}$ 和 $M^{RA}_t$。这一步的目的是消除所有潜在动态物体对初始配准的干扰。

2.  **物体重投影与数据关联（Object Reprojection and Data Association）**：
    - **初始位姿估计**：使用经典配准算法（如ICP或NDT）对 $M^{RA}_{t-1}$ 和 $M^{RA}_t$ 进行配准，得到初始变换矩阵 $T_0$。
    - **重投影**：利用 $T_0$ 将当前帧的检测结果 $D_t$ 投影到前一帧坐标系下，得到 $D^{t-1}_t$。
    - **数据关联**：使用匈牙利算法，基于投影后物体中心点与前一帧检测物体 $D_{t-1}$ 中心点的欧氏距离进行关联，匹配同一物体在不同帧中的检测框。

3.  **动态配准（Dynamic Registration）**：
    - **运动分割**：对于关联成功的物体对，计算其重投影误差（即物体中心点在两帧间的位移）。如果位移大于一个预设阈值，则判定为动态物体；否则为静态物体。由此，将检测结果分为动态物体集 $D^d$ 和静态物体集 $D^s$。
    - **迭代优化**：将静态物体 $D^s$ 的点云重新合并到“移除所有物体后的点云” $M^{RA}$ 中，得到“移除动态物体后的点云” $M^{RD}$。然后使用 $M^{RD}$ 再次进行配准，得到更精确的位姿 $T_{DR}$。重复上述“重投影-运动分割-合并-配准”的迭代过程（算法2），直到没有新的静态物体被分割出来。最终输出精确的位姿 $T^*$、动态物体 $D^{d*}$ 和静态物体 $D^{s*}$。

### 3.2 关键算法与数学推导

1.  **判断点是否在3D边界框内**：给定一个3D边界框（由中心点、长宽高、朝向角定义），判断点 $P$ 是否在其内部。如图3所示，将边界框表示为三个正交向量 $\overrightarrow{AB}$、$\overrightarrow{AD}$、$\overrightarrow{AC}$。点 $P$ 在框内的条件是其在三个向量方向上的投影长度都在 $[0, \|\overrightarrow{AB}\|]$、$[0, \|\overrightarrow{AD}\|]$、$[0, \|\overrightarrow{AC}\|]$ 范围内。公式如下：
    $$0 \le \frac{\overrightarrow{AP} \cdot \overrightarrow{AB}}{\|\overrightarrow{AB}\|} \le \frac{\overrightarrow{AB} \cdot \overrightarrow{AB}}{\|\overrightarrow{AB}\|}$$
    $$0 \le \frac{\overrightarrow{AP} \cdot \overrightarrow{AD}}{\|\overrightarrow{AD}\|} \le \frac{\overrightarrow{AD} \cdot \overrightarrow{AD}}{\|\overrightarrow{AD}\|}$$
    $$0 \le \frac{\overrightarrow{AP} \cdot \overrightarrow{AC}}{\|\overrightarrow{AC}\|} \le \frac{\overrightarrow{AC} \cdot \overrightarrow{AC}}{\|\overrightarrow{AC}\|}$$

2.  **初始位姿估计**：
    $$T_0 = \text{Registration}(M^{RA}_{t-1}, M^{RA}_t)$$
    其中 $T_0 \in SE(3)$，Registration可以是ICP或NDT算法。

3.  **物体重投影**：
    $$D^{t-1}_t = T_0 D_t$$
    将当前帧的检测结果 $D_t$ 通过变换矩阵 $T_0$ 投影到前一帧坐标系。

4.  **相对位姿误差（RPE）**：用于评估配准精度。
    $$\text{RPE}_{\text{trans}} = \sqrt{\frac{1}{N-1} \sum_{i=1}^{N-1} \left\| \text{trans}\left(T_{gt}^{-1} T_{est}\right) \right\|_2^2}$$
    其中 $T_{gt}$ 是真实位姿，$T_{est}$ 是估计位姿，$\text{trans}(\cdot)$ 提取变换矩阵的平移部分。

### 3.3 模型、训练与数据集

- **3D目标检测模型**：采用PointPillars。它是一个基于柱状体（Pillar）的轻量级3D检测网络，能够在速度和精度之间取得良好平衡，满足实时性要求。作者直接使用了MMDetection3D库中在KITTI 3D检测基准上预训练的模型，没有进行微调。
- **训练**：PointPillars模型是预训练好的，本文未涉及训练过程。
- **数据集**：使用KITTI Tracking数据集进行实验。该数据集提供了连续的点云帧、自车位姿真值和物体跟踪标注，适合评估动态环境下的定位和物体检测性能。

### 3.4 实验设置与结果

- **实验设置**：
    - 点云范围限制在 $x \in [0, 40]m$, $y \in [-30, 30]m$, $z \in [-3, 1]m$。
    - 检测置信度阈值设为0.5。
    - 配准前对点云进行0.3m体素降采样。
    - 基于NDT和ICP两种经典配准算法实现动态配准，分别记为NDT-RMD和ICP-RMD。
    - 对比方法包括：原始NDT/ICP，以及“移除所有物体后”的NDT/ICP（记为NDT-RMA/ICP-RMA）。

- **定量结果（表I）**：
    - 在KITTI Tracking的21个序列（seq 0000-0020）上评估了RPE的RMSE。
    - **主要发现**：动态配准（NDT-RMD和ICP-RMD）在大多数序列上取得了最低的RPE误差，平均误差也最小（NDT-RMD平均0.3960，ICP-RMD平均0.4758）。
    - **对比分析**：
        - 动态配准优于原始NDT/ICP，证明了移除动态物体的有效性。
        - 动态配准也优于“移除所有物体”的NDT-RMA/ICP-RMA，证明了将静态物体重新加入环境有助于提高配准精度。

- **定性结果（图5）**：
    - **轨迹对比**：在序列0013上，动态配准（红色）的轨迹比原始NDT（蓝色）更接近真实轨迹（虚线）。
    - **建图对比**：原始NDT的建图结果（图5b）中存在明显的“鬼影”（动态物体的拖尾），而动态配准的建图结果（图5c）成功清除了这些鬼影，环境地图更清晰。红框放大了细节，紫框指出了由于检测器漏检导致的残留鬼影。

### 3.5 技术细节

- **数据关联**：采用匈牙利算法进行数据关联，代价矩阵基于物体中心点的欧氏距离。作者指出，由于帧间时间间隔短，物体位移小，这种方法比计算3D IoU更简单高效。
- **迭代终止条件**：迭代过程（算法2）持续进行，直到连续两次迭代分割出的静态物体集合不再发生变化（即 $^*D^s_{t-1} = D^s_{t-1}$ 且 $^*D^s_t = D^s_t$）。
- **对检测器缺陷的鲁棒性**：
    - **假阳性（False Positives）**：误检的物体通常会被判定为静态物体，并被合并到环境点云中，对最终结果影响不大。
    - **假阴性（False Negatives）**：漏检的动态物体（如紫框所示）会导致其点云被当作环境点云参与配准，从而产生残留的鬼影。这是该方法的一个局限。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 提出了一种新颖的、通用的动态环境下的点云配准策略，能够同时提升定位精度和实现实例级动态物体分割。
    - 该策略不依赖于特定的SLAM框架，可以作为插件集成到任何LiDAR SLAM系统中，具有良好的通用性。
    - 通过迭代的方式，优雅地解决了自车运动与物体运动耦合的问题，并证明了保留静态物体对提升配准精度的重要性。

- **对未来研究/应用的影响**：
    - **SLAM系统**：为构建鲁棒的动态SLAM系统提供了一个有效的组件。作者在结论中也提到，未来将把动态配准扩展到完整的动态SLAM系统中。
    - **自动驾驶**：该方法可以直接应用于自动驾驶汽车，提供更精确的定位和更清晰的动态障碍物感知，有助于提升决策和规划的安全性。
    - **机器人导航**：对于在人群、车辆等动态环境中工作的机器人，该方法能显著提升其自主导航的可靠性。

- **局限与开放问题**：
    - **依赖3D检测器**：方法的性能高度依赖于3D目标检测器的准确率。检测器的漏检（假阴性）会导致动态物体无法被完全移除，从而影响建图质量。
    - **运动分割阈值**：运动分割的阈值是预设的，可能不适用于所有场景（例如，非常缓慢移动的物体）。如何自适应地确定阈值是一个开放问题。
    - **数据关联的局限性**：简单的最近邻关联无法处理检测器的不稳定（如漏检、误检）和遮挡问题。作者也指出，这只能部分解决假阳性问题，而假阴性问题需要更高级的状态估计和预测方法。
    - **未进行端到端训练**：整个流程是模块化的，各模块（检测、配准、关联）是独立工作的，没有进行联合优化。未来可以考虑端到端训练以进一步提升性能。

## 5. 澄清与简化（Clarifications and Simplifications）

- **核心思想类比**：想象你在一个拥挤的火车站（动态环境）里，想要拍一张清晰的照片（定位）。如果直接拍，照片里会有很多走动的人（动态物体），导致照片模糊（定位不准）。
    1.  **3D检测**：你首先用眼睛快速识别出所有“人”的轮廓（3D边界框）。
    2.  **暂时移除**：你让所有人暂时离开画面（移除所有物体点云）。
    3.  **初始拍照**：你拍下空无一人的车站背景（初始配准），得到一个初步的相机位置。
    4.  **运动分割**：你让所有人回到原位，然后根据你刚才的相机位置，判断哪些人移动了（动态物体），哪些人一直站着没动（静态物体，比如行李架）。
    5.  **合并静态物体**：你把站着没动的“行李架”重新放回背景中，因为它们是很好的定位参照物。
    6.  **再次拍照**：你对着包含“行李架”的背景再次调整相机位置（迭代配准），得到更精确的定位。重复这个过程，直到所有“行李架”都被正确识别并加入背景。

- **为什么“移除所有物体”不好？**：如果直接把所有人（包括站着不动的行李架）都移走，背景就只剩下光秃秃的墙壁和地板，特征点太少，相机很难找到自己的准确位置。而“行李架”提供了丰富的几何特征，有助于更精确地定位。

- **“重投影误差”是什么？**：简单来说，就是用一个物体在上一帧的位置，去预测它在当前帧应该出现的位置，然后看它实际出现的位置和预测位置之间的差距。如果差距很大，说明这个物体自己动了；如果差距很小，说明它没动，只是随着自车运动而“看起来”在动。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - 本文的工作与**SLAMMOT**（Simultaneous Localization, Mapping and Moving Object Tracking）领域密切相关，但本文提出的方法更侧重于作为一个通用的**配准模块**，而非一个完整的SLAM系统。
    - 与**DynaSLAM**等视觉动态SLAM相比，本文专注于LiDAR点云，并利用了更先进的3D目标检测技术。
    - 与**基于运动分割**的方法（如MODNet）相比，本文的优势在于能够输出**实例级**的物体信息，而不仅仅是运动点云。
