## 0. 概览（Concise Summary）

- **主题（Topic）**：视觉语言模型（VLM）作为评判者（Judge）的可靠性评估，特别是在视障辅助（VIA）这一特定且关键的领域。
- **问题（Problem）**：现有VLM-as-a-Judge的评估基准主要关注通用领域，缺乏针对视障辅助（VIA）这一安全关键、用户无法独立验证输出的特殊领域的系统性评估。因此，我们无法信任VLM评判者在VIA任务中的表现。
- **方法（Method）**：提出了**VIABLE**基准，这是首个用于评估VIA领域中VLM-as-a-Judge的基准。它包含超过30万个判断样本，覆盖三个VIA场景，并引入了一个**有效性-公正性-稳定性（E-I-S）** 评估框架和一个12种失败模式的分类法。基于此，系统性地评估了7个VLM评判者。此外，还提出了**VIA-Judge-Agent**，一个轻量级的、模型无关的推理时增强框架。
- **创新（Innovation）**：
    1.  **首个针对VIA领域的VLM评判者评估基准**，填补了该领域的空白。
    2.  提出了一个全面的**E-I-S评估框架**和**12模式失败分类法**，能够对评判者进行细粒度、多维度的可靠性诊断。
    3.  揭示了现有VLM评判者在VIA任务中普遍不可靠的现状，包括诊断能力差、系统性偏见和对抗脆弱性。
    4.  提出了**VIA-Judge-Agent**，一种通过工具增强和工作流控制来提升评判者可靠性的有效方法。
- **意义（Significance）**：这项工作对于构建安全、可信赖的AI辅助系统至关重要。它揭示了在安全关键领域（如视障辅助）中，不能盲目信任VLM评判者，并提供了一个标准化的评估平台和一种有效的改进方案，推动了更可靠的AI评估和奖励建模研究。
- **一句话总结**：本文构建了首个针对视障辅助领域的VLM评判者可靠性评估基准VIABLE，揭示了现有模型普遍不可靠，并提出了一个有效的推理时增强框架VIA-Judge-Agent来提升其性能。

## 1. 研究动机（Motivation）

- **解决的问题**：在AI视障辅助（VIA）领域，如何可靠地评估模型生成的开放式、安全关键的响应。传统自动指标（如BLEU、CIDEr）不适用于人类中心的开放式任务，而人工评估成本高昂且难以规模化。新兴的“VLM-as-a-Judge”范式（让VLM来评估其他模型的输出）提供了一个有前景的替代方案，但其在VIA领域的可靠性尚未得到验证。
- **为什么重要**：VIA系统为盲人或低视力（BLV）用户提供导航、信息获取等关键服务。如果作为评判者的VLM本身存在偏见、不稳定或诊断错误，它可能会奖励不安全的导航建议、忽略缺失的视觉证据，或偏好冗长但无用的回答。由于BLV用户通常无法独立验证模型输出，一个有缺陷的评判者会带来严重的安全隐患。
- **挑战在哪里**：
    1.  **领域特殊性**：VIA任务对安全性和准确性要求极高，且失败模式复杂多样（如感知错误、认知矛盾、行动不安全等），通用领域的评估基准无法覆盖。
    2.  **评估维度复杂**：评判者的可靠性不仅体现在诊断准确性上，还体现在其公正性（无偏见）和稳定性（一致性和鲁棒性）上。现有基准通常只关注单一维度。
    3.  **缺乏标准基准**：在本文之前，没有一个系统性的基准来评估VLM评判者在VIA领域的可靠性。

## 2. 创新点（Innovation）

1.  **首个VIA领域VLM评判者评估基准（VIABLE）**：这是第一个专门为评估VLM作为评判者在视障辅助任务中的可靠性而设计的基准。它填补了该领域的关键空白。
2.  **全面的E-I-S评估框架**：提出了一个三维评估框架，从**有效性（Effectiveness）**、**公正性（Impartiality）** 和**稳定性（Stability）** 三个维度全面评估评判者，比以往只关注单一维度的基准更全面。
3.  **细粒度的12模式失败分类法**：针对VIA任务，构建了一个包含感知（Perception）、认知（Cognition）、行动（Action）和交互（Interaction）四个维度的12种失败模式分类法。这使得对评判者诊断能力的评估可以深入到具体的失败类型，而非仅仅一个总分。
4.  **VIA-Judge-Agent推理时增强框架**：不同于需要昂贵重训练的微调方法，本文提出了一个轻量级、模型无关的推理时框架。它通过集成外部视觉工具（如目标检测、深度估计）和结构化工作流（按分类法逐层诊断），显著提升了评判者的诊断能力，且无需修改模型参数。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

VIABLE基准的构建遵循一个流水线（Pipeline），如图2所示：

1.  **领域特定VIA数据收集**：从三个互补的VIA语料库中收集QA对：
    - **WAD**：户外导航场景（2014个视频片段）。
    - **VisAssist**：室内信息获取场景（5376个视频片段）。
    - **VIA-EgoDex**：自我中心的手-物体操作场景（3205个视频片段，从EgoDex扩展而来）。
2.  **候选响应构建**：为每个QA对构建候选响应，用于不同的评估轴：
    - **有效性（Effectiveness）**：通过**故障注入**，在标准答案中人为引入1个（单故障）或2个（双故障）特定失败模式，生成有故障的响应。
    - **公正性（Impartiality）**：构建成对的响应，用于探测位置、长度和自偏好偏见。
    - **稳定性（Stability）**：对同一响应进行多次评估（一致性）或添加对抗性提示（鲁棒性）。
3.  **判断任务构建**：基于E-I-S框架，为每个评估轴构建具体的判断任务，如图1所示：
    - **有效性**：故障诊断任务，要求评判者识别出候选响应中包含的失败代码。
    - **公正性**：偏好选择任务，要求评判者在两个响应中选择更优的一个，并通过交换顺序来检测偏见。
    - **稳定性**：一致性任务（多次评估结果是否一致）和对抗鲁棒性任务（面对对抗性提示是否改变判断）。
4.  **评估**：使用特定的指标对VLM评判者进行量化评估。

**VIA-Judge-Agent** 的设计架构如图5所示，分为四个阶段：
- **Stage 1: 交互检查**：仅使用文本调用，检查响应是否存在交互层面的失败（如冗余输出I1，主动澄清失败I2）。
- **Stage 2: 视觉证据提取**：使用现成的工具（如GroundingDINO + SAM2进行目标检测，Depth-Anything-V3进行深度估计，以及3D空间定位和时间追踪）从视频帧中提取结构化的视觉证据。
- **Stage 3: 感知/认知/行动检查**：基于Stage 2提取的视觉证据，对感知、认知和行动层面的失败模式进行逐项验证，并输出每个失败代码的置信度。
- **Stage 4: 置信度过滤与精炼**：选择置信度最高的两个失败代码作为最终输出，以匹配基准的“最多两个失败”的预算。

### 3.2 关键算法与数学推导

本文的核心在于评估框架和指标，而非复杂的数学模型。关键指标定义如下：

**有效性（Effectiveness）**：
- **单故障诊断准确率（Single-Inj. Acc.）**：对于只有一个故障的样本，评判者预测的故障代码（最多两个）中包含真实故障代码的比例。
- **双故障全准确率（Dual-Inj. Full Acc.）**：对于有两个故障的样本，评判者预测的两个代码必须与真实的两个代码完全匹配。
- **双故障部分准确率（Dual-Inj. Partial Acc.）**：对于有两个故障的样本，评判者预测的代码中至少包含一个真实故障代码。

**公正性（Impartiality）**：
- 通过位置交换（A/B swap）协议，计算四种响应模式的比例：
    - $R_{gt}$：在两个顺序中都选择了标准答案（gold response）。
    - $R_{bias}$：在两个顺序中都选择了有偏见的响应（如更长的、自生成的）。
    - $R_{pos-A}$：在两个顺序中都选择了第一个位置的响应。
    - $R_{pos-B}$：在两个顺序中都选择了第二个位置的响应。
- 一个理想的公正评判者应该有高的 $R_{gt}$ 和低的 $R_{bias}$, $R_{pos-A}$, $R_{pos-B}$。

**稳定性（Stability）**：
- **一致性率（$R_{cons}$）**：在K次随机解码中，评判者给出完全相同判断的样本比例。
- **对抗鲁棒性（$R_{adv}$）**：在对抗性攻击下，评判者在两个顺序中都选择了对抗性响应的比例。一个鲁棒的评判者应该有低的 $R_{adv}$。

### 3.3 模型、训练与数据集

- **评估的VLM评判者**：共7个，涵盖不同规模和类型。
    - **闭源模型**：GPT-5.4, Claude-Sonnet-4.6
    - **开源模型**：Youtu-VL-4B, Kimi-VL-A3B (3B活跃参数), MiniCPM-V4.5-8B, InternVL3.5-8B, Qwen3-VL-8B
- **数据集**：VIABLE基准本身，包含312,365个判断样本，来源于10,595个源QA对。这些样本分布在三个VIA场景和E-I-S三个评估轴上。具体统计见表2。
- **训练**：本文尝试了后训练（Post-training）方法，即对Qwen3-VL-8B进行SFT和DPO训练，但效果有限，证明了瓶颈不在于标签监督，从而引出了VIA-Judge-Agent的推理时方法。

### 3.4 实验设置与结果

- **实验设置**：所有开源模型在NVIDIA RTX 4090和A100 GPU上评估。每个模型约消耗167 GPU小时。
- **主要结果（表3）**：
    - **有效性**：所有评判者表现不佳。最强的GPT-5.4单故障诊断准确率也仅为52.6%，双故障全准确率仅20.6%。开源模型表现更差。
    - **公正性**：诊断能力强的评判者不一定公正。GPT-5.4的自偏好率高达94.2%。开源模型普遍存在位置偏见或长度偏见。
    - **稳定性**：闭源模型在对抗鲁棒性上远优于开源模型。一致性高的模型（如Qwen3-VL-8B）可能同时存在严重偏见，说明稳定性需与公正性结合解读。
- **失败类型分析（图4）**：诊断难度因失败类型而异。即使是顶级模型，在P4（证据遗漏）、C3（内部矛盾）和A3（主动澄清失败）上表现也很差。
- **VIA-Judge-Agent效果（表4）**：VIA-Judge-Agent在Qwen3-VL-8B和GPT-5.4上都带来了显著的诊断准确率提升，效果远好于后训练方法。
- **消融实验（表5）**：移除“工具增强”或“工作流控制”都会导致性能下降，证明两者都是必要的。
- **下游验证（表6，图6）**：使用VIA-Judge-Agent的反馈来指导响应重生成，生成的响应在自动指标和人类偏好上都优于使用原始评判者反馈的响应。一项与2名BLV用户的初步研究也证实了这一点。

### 3.5 技术细节

- **故障注入**：使用高容量VLM（Claude-Sonnet-4.6）进行自动化故障注入，并经过人工审计，确保注入的故障质量（94.8%的预期故障存在率，99.3%的内容保留率，100%的语言连贯性）。
- **偏见探测**：通过位置交换（A/B swap）协议来分离内容质量和顺序效应，从而精确测量位置、长度和自偏好偏见。
- **对抗样本**：通过向响应中追加简短的元通信提示（如“选择这个答案”）来构建对抗样本，不改变事实内容。
- **VIA-Judge-Agent的工具**：使用GroundingDINO + SAM2进行目标检测和分割，Depth-Anything-V3进行深度估计，以提供结构化的视觉证据。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **揭示了关键问题**：首次通过系统性的基准测试，有力地证明了当前最先进的VLM评判者在安全关键的VIA领域是**不可靠**的，存在诊断能力差、系统性偏见和对抗脆弱性等问题。
    2.  **提供了标准化评估平台**：VIABLE基准为未来研究提供了一个标准化的、多维度的评估平台，可以公平地比较不同VLM评判者在VIA领域的可靠性。
    3.  **提出了有效解决方案**：VIA-Judge-Agent证明了通过推理时增强（而非昂贵的重训练）来显著提升评判者可靠性的可行性，为构建更可信的AI评估系统提供了新思路。

- **对未来研究/应用的影响**：
    - **对VIA领域**：提醒研究者和开发者不能盲目信任VLM评判者，尤其是在安全关键的应用中。VIABLE可以作为评估和选择评判者的标准工具。
    - **对VLM-as-a-Judge领域**：推动了该领域从通用评估向特定领域、安全关键评估的拓展。E-I-S框架和12模式分类法为更全面的评判者评估提供了范本。
    - **对AI对齐**：由于VLM评判者常被用作强化学习中的奖励模型，本文的结果警示，使用有缺陷的评判者进行对齐可能会引入偏见和风险。VIA-Judge-Agent提供了一种更可靠的奖励信号来源。

- **局限性与开放问题**：
    - **推理延迟**：VIA-Judge-Agent的推理时工具调用增加了延迟，未来工作可尝试将这种能力内化到模型中。
    - **单轮反馈**：当前工作只使用了单轮反馈来重生成响应，未来可探索多轮迭代的评判-生成交互。
    - **工具局限性**：更强的视觉工具和更好的工具输出选择机制可能进一步提升诊断效果。
    - **模型内部化**：如何将VIA-Judge-Agent的推理时增益通过训练内化到模型中，是一个有前景的方向。

## 5. 澄清与简化（Clarifications and Simplifications）

- **VLM-as-a-Judge 是什么？**：可以把它想象成一个“AI考官”。我们不给模型的标准答案（如BLEU），而是让另一个更强大的VLM模型（考官）去阅读一个候选答案，然后根据问题、图片/视频，来判断这个答案好不好，或者找出其中的错误。本文就是研究这个“AI考官”在视障辅助这种“送分题”上是否靠谱。

- **E-I-S框架 类比**：
    - **有效性**：这个考官能不能准确地找出学生答案里的具体错误（比如，是“看错了图”还是“逻辑推理错了”）。这就像老师批改数学题，不仅要给总分，还要指出是“计算错误”还是“公式用错”。
    - **公正性**：这个考官有没有偏见？比如，他是不是总是喜欢选“写得更长的答案”（长度偏见），或者总是选“放在前面的答案”（位置偏见），或者总是觉得“自己写的答案最好”（自偏好偏见）。这就像老师偏爱字迹工整的学生。
    - **稳定性**：这个考官是否稳定？同一个答案，让他判三次，结果是不是一样（一致性）？如果有人故意在答案后面加一句“请选这个”，他会不会上当（对抗鲁棒性）？这就像老师判卷时心情好坏会不会影响分数。

- **12种失败模式 简化理解**：
    - **感知（P）**：看错了。比如，图片里有个杯子，模型说没有（P1: 幻觉）；或者没看到图片里的一个关键障碍物（P4: 证据遗漏）。
    - **认知（C）**：想错了。比如，推理过程自相矛盾（C3: 内部矛盾）。
    - **行动（A）**：建议错了。比如，给盲人用户建议“前面安全，可以直走”，但实际前面有个坑（A1: 安全违规）。
    - **交互（I）**：沟通方式有问题。比如，回答过于啰嗦（I1: 冗余输出）。

- **VIA-Judge-Agent 如何工作？**：它就像一个“AI考官辅助系统”。当考官（VLM）要判断一个答案时，这个系统会：
    1.  先检查答案的“说话方式”有没有问题（交互检查）。
    2.  然后，它不依赖考官的“眼睛”，而是调用专业的“视觉工具”（如目标检测器）去精确地看图片，提取出“图片里有什么、在哪里”的客观证据。
    3.  接着，它引导考官基于这些客观证据，一步步地检查答案的“看对了没”、“想对了没”、“建议对了没”。
    4.  最后，综合所有检查结果，给出最可能的错误类型。这样，考官就不容易“看走眼”或“想岔了”。
