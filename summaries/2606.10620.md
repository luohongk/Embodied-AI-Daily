## 0. 概览（Concise Summary）

- **主题（Topic）**：图像生成模型的视觉世界建模能力评估。具体而言，是评估图像生成模型在生成有序多状态图像（即“运动图”）时，能否保持时空一致性。
- **问题（Problem）**：现有图像生成评估基准主要关注单张图像的正确性（如物体、属性、空间关系），但忽略了模型在时间维度上维持一个连贯视觉世界的能力。模型可能生成看似合理的最终状态，却跳过了必要的中间过程、违反了因果顺序或丢失了物体身份。
- **方法（Method）**：提出了一个名为 **ImageTime** 的诊断性基准。该基准要求模型根据一个动作指令，生成一张包含四个有序关键状态（初始状态、动作开始、过渡状态、最终状态）的 2×2 布局图像。通过一个七级能力树（L0-L6）和一个结构化的 VLM-as-Judge 评估协议（使用 GPT-5.5 进行评分），对模型的时空一致性进行细粒度诊断。
- **创新（Innovation）**：
    1.  **新任务范式**：将图像生成评估从单张图像扩展到有序多状态一致性，提出了“四帧运动图”的生成任务。
    2.  **渐进式能力树**：设计了从静态基础到因果推理的七级能力层次结构，用于系统性地诊断模型在不同复杂度任务上的表现。
    3.  **结构化评估协议**：使用 VLM（GPT-5.5）作为裁判，并设计了互补的 C 分数（能力分数）和 D 分数（诊断子分数），将宏观能力表现与微观视觉证据联系起来，实现了可解释的评估。
- **意义（Significance）**：ImageTime 填补了评估图像生成模型“世界意识”的关键空白。它揭示了当前最先进的模型在维持物体身份、遵循因果顺序、处理约束条件等方面的局限性，为未来开发更具世界感知能力的生成模型提供了方向。
- **一句话总结**：本文提出了 ImageTime 基准，通过评估图像模型在生成有序多帧图像时的时空一致性，来探测其是否具备隐式的视觉世界建模能力。

## 1. 研究动机（Motivation）

- **核心问题**：图像生成模型能否“想象”时间？即，它们能否理解并描绘一个视觉世界如何随时间变化，而不仅仅是生成一张漂亮的静态快照？
- **为什么重要**：
    - **实际应用需求**：在故事板（storyboarding）、分步插图、参考引导编辑和视频预览等实际工作流程中，模型需要在多个视觉状态之间保持身份、物体、空间关系和因果顺序的一致性。当前的评估无法满足这一需求。
    - **迈向世界感知智能**：视觉感知不仅是像素合成，更是高级认知的基础。一个具备“世界意识”的生成模型，应能隐式理解其所描绘的物理世界的规律，包括空间、时间和因果的规律性。评估时空一致性是探测这种世界模型的关键。
- **挑战在哪里**：
    - **单帧评估的误导性**：单张图像看起来可能很合理，但其背后的生成过程可能是错误的（例如，物体凭空出现、因果顺序颠倒）。单帧评估无法捕捉这些失败。
    - **缺乏多状态评估标准**：现有基准（如 DrawBench, GenEval）主要评估单张图像的语义对齐，缺乏对有序多状态一致性的系统评估。
    - **诊断困难**：即使模型生成了多帧图像，如何细粒度地诊断其失败原因（是物体身份丢失、空间关系混乱，还是因果顺序错误）也是一个挑战。

## 2. 创新点（Innovation）

- **关键创新 1：将时空一致性作为行为探针**。ImageTime 不直接声称检查模型的内部世界模型，而是通过要求模型生成有序的关键帧，观察其是否表现出类似世界模型的行为。这是一种巧妙的“行为探测”方法。
- **关键创新 2：四帧运动图任务范式**。与生成连续视频不同，ImageTime 要求生成包含四个有序关键状态（初始、开始、过渡、最终）的单张 2×2 布局图像。这个设计比单帧生成更具挑战性，同时避免了密集视频评估中的混淆因素（如插值、光流平滑），使得评估更聚焦于核心的时空一致性。
- **关键创新 3：七级能力树（L0-L6）**。这是一个层次化的诊断框架，从基础的静态基础（L0）到高级的约束/反事实推理（L6）。这使得评估不再是单一的分数，而是可以定位模型在哪个能力层级上开始失败，提供了系统性的诊断视角。
- **关键创新 4：结构化的 VLM-as-Judge 协议**。使用 GPT-5.5 作为裁判，并设计了两个互补的分数族：
    - **C 分数（C0-C9）**：宏观能力分数，如实体一致性、空间视图一致性、因果过程一致性等。
    - **D 分数（D0-D14）**：微观诊断子分数，如物体永久性、背景锚点、状态转换可见性等，为 C 分数提供具体的视觉证据。
    这种设计使得评估结果不仅是分数，更是可解释的诊断报告。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

ImageTime 的整体架构是一个闭环的评估系统，包含三个核心组件：

1.  **任务定义**：给定一个动作指令，模型需生成一张 2×2 布局的图像，包含四个有序关键帧：t1（初始状态）、t2（动作开始）、t3（过渡/交互状态）、t4（最终状态）。输出被称为“运动图”（motion sheet）。
2.  **渐进式能力树**：定义了七个能力层级，用于指导数据构建和评估。
3.  **结构化 VLM-as-Judge 协议**：使用 GPT-5.5 作为裁判，根据预定义的 C 和 D 分数规则，对生成的图像进行评分和诊断。

### 3.2 关键算法与数学推导

本文的核心贡献在于评估框架，而非新的生成算法。因此，没有复杂的数学推导。其核心逻辑在于将评估问题形式化为一个结构化的评分任务。关键公式体现在分数聚合上：

- **树节点分数 (Tree-level Score)**: 对于能力树中的节点 L，其分数 $T_L$ 由对应的 C 分数和 D 分数平均得到。
    $$T_L = \frac{1}{2} \text{mean}(C_L) + \frac{1}{2} \text{mean}(D_L)$$
    其中 $C_L$ 和 $D_L$ 分别是对应于节点 L 的 C 分数和 D 分数集合（见表 4）。空维度被忽略。这种平衡聚合防止了数量更多的 D 分数主导结果。

- **总体分数 (Overall Mean)**: 模型的最终总体分数是 C 分数均值和 D 分数均值的平衡平均。
    $$\text{Overall Mean} = \frac{1}{2} (\text{C mean} + \text{D mean})$$
    其中 C mean 和 D mean 分别是对所有适用维度的 C 和 D 分数取平均。

### 3.3 模型、训练与数据集

- **数据集**：ImageTime 基准包含 **750 个测试用例**，覆盖 **22 个领域** 和 **375 个动作概念**。数据构建采用“人工主导，LLM 辅助”的流程。人工设计种子场景，LLM 在约束下进行扩展和规范化，最后人工检查和修复了 170 个有问题的提示。
- **评估模型**：评估了 8 个主流图像生成模型，包括：
    - 闭源/前沿：GPT Image 2, Nano Banana 2, Seedream 5.0 Lite, FLUX.2 Pro
    - 开源/中等：Qwen-Image-2512, Z-Image-Turbo, HunyuanImage-2.1
    - 基线：SDXL
- **训练**：本文不涉及模型训练，所有模型均以零样本（zero-shot）方式在 prompt-only 设置下进行评估。

### 3.4 实验设置与结果

- **设置**：所有模型在 **prompt-only** 设置下进行评估，即仅根据文本提示生成完整的 2×2 运动图。生成是单次通过，不进行迭代修正或人工筛选。
- **评估协议**：所有生成的图像均由同一个 GPT-5.5 裁判，按照相同的 C/D 分数规则进行评分。
- **主要结果**：
    - **图 3（雷达图）**：展示了各模型在 C0-C9 能力上的轮廓。GPT Image 2、Nano Banana 2 和 Seedream 5.0 Lite 在大多数维度上表现优异，尤其是在运动连续性（C4）、时间顺序（C5）、因果过程（C6）和约束敏感性（C8）上拉开了与其它模型的差距。
    - **图 4（诊断分数热力图）**：展示了 D0-D14 的细粒度诊断分数。关键诊断维度如状态转换可见性（D8）、阶段区分度（D9）、轨迹-姿态耦合（D6）等，揭示了模型失败的具体原因。
    - **图 5（能力树分数）**：展示了模型在 L0-L6 各层级上的表现。关键发现是，所有模型在更高层级（如因果过程 L5、约束/反事实 L6）上表现更差，但强模型（如 GPT Image 2）的下降幅度较小，而弱模型则出现急剧下降。
    - **表 5（总体结果）**：给出了模型的总体排名。GPT Image 2 以 7.86 分领先，其次是 Nano Banana 2 (7.45) 和 Seedream 5.0 Lite (7.16)。SDXL 得分最低 (1.57)，主要因为其难以生成符合要求的有序四面板布局。

### 3.5 技术细节

- **布局验证**：在评分前，首先进行布局验证，确保输出是有效的 2×2 运动图。如果布局无效，C0 分数设为 0，并保守地限制其他分数。
- **评分规则**：所有 C 和 D 分数使用 0-10 的整数尺度。10 分表示完全正确，0 分表示完全失败。当证据模糊时，裁判选择较低的分数。
- **保守的分数上限规则**：当出现实体丢失、场景漂移、因果顺序错误等严重问题时，会触发分数上限规则，防止高级别分数掩盖低级别错误。例如，如果物体身份在帧间不一致，那么关于“交互一致性”的分数会被上限限制。

## 4. 意义与影响（Significance and Impact）

- **解决的问题**：ImageTime 解决了当前图像生成评估中的一个关键空白——缺乏对模型在时间维度上维持连贯视觉世界能力的系统性评估。它提供了一个标准化的、可解释的诊断工具。
- **贡献**：
    1.  提出了一个新的评估任务和基准，将评估从单帧正确性扩展到有序多状态一致性。
    2.  设计了渐进式能力树和结构化 VLM-as-Judge 协议，实现了细粒度、可解释的评估。
    3.  通过对多个主流模型的系统评估，揭示了当前图像生成模型在时空一致性上的普遍失败模式（如身份漂移、因果顺序违反、中间状态缺失等），为社区提供了宝贵的诊断性见解。
- **对未来研究/应用的影响**：
    - **模型开发**：为开发更具“世界意识”的图像生成模型提供了明确的方向和评估标准。未来的模型可以针对 ImageTime 中暴露的弱点进行优化。
    - **评估方法**：其结构化的 VLM-as-Judge 方法为其他复杂视觉任务的评估提供了新范式。
    - **应用**：对于需要多帧一致性的应用（如故事板、视频预可视化），ImageTime 的评估结果可以帮助用户选择合适的模型。
- **局限与开放问题**：作者指出，ImageTime 是一个行为探针，不直接检查模型的内部世界模型。此外，评估依赖于 GPT-5.5 作为裁判，其自身的偏差和局限性可能会影响结果。未来的工作可以探索更广泛的裁判模型和更细粒度的评估维度。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“世界模型”是什么？**：在本文中，它不是一个具体的算法或结构，而是一个**行为上的概念**。如果一个模型在生成有序图像时，能够保持物体身份不变、遵循因果顺序（比如必须先打开抽屉才能拿出东西）、并且空间关系合理，我们就说它表现出了“类似世界模型的行为”。这就像我们通过观察一个人的行为来判断他是否理解物理规则，而不是去检查他的大脑。
- **为什么是四帧，而不是视频？**：视频评估很复杂，涉及到帧率、运动平滑度、插值等很多因素。四帧运动图是一个“最小化但足够”的设计。它比单帧难，因为它要求模型思考“如何变化”；它又比视频简单，因为它去除了视频评估中的噪音，让我们能更纯粹地评估模型对“状态变化”的理解。
- **C 分数和 D 分数的关系**：可以把 C 分数看作是“考试总成绩”，而 D 分数是“各科试卷的详细得分”。比如，一个模型在“因果过程一致性”（C6）上得分低，我们可以通过查看 D 分数来知道具体原因：是因为“状态转换不可见”（D8）？还是因为“阶段重复”（D9）？这种设计让评估结果非常透明。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：本文与多个研究方向紧密相关：
    - **文本到图像评估**：超越了 DrawBench、GenEval 等单帧评估，引入了时间维度。
    - **参考条件生成**：与 DreamBooth、IP-Adapter 等工作相关，但 ImageTime 关注的是在多个状态中保持参考世界的一致性，而不仅仅是单张图像的保真度。
    - **多图像/故事板生成**：与 StoryDiffusion、StoryMaker 等目标一致，但 ImageTime 提供了一个更受控、更细粒度的诊断性评估框架，而非仅关注叙事连贯性。
    - **视频生成评估**：与 VBench、VideoPhy 等目标类似，但 ImageTime 聚焦于离散关键帧的一致性，避免了连续视频评估的复杂性。
