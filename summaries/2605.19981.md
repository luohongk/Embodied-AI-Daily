## 0. 概览（Concise Summary）

- **主题（Topic）**：人形机器人的全身移动操作（Loco-Manipulation），特别是接触丰富的操作任务。
- **问题（Problem）**：现有的人形机器人控制方法（如关节空间跟踪）难以同时满足接触丰富操作所需的柔顺性、任务规划所需的高层抽象接口，以及不同规划器（如语言模型、遥操作）的即插即用集成。
- **方法（Method）**：提出一种名为 **CEER** 的统一控制抽象接口，将控制命令定义为“末端执行器（End-Effector, EE）位姿”和“机器人根部（Root）运动”的组合。通过教师-学生框架，将基于全身关节跟踪的教师策略蒸馏为仅接收 EE-Root 命令的学生策略，并以此为基础构建一个三层分层系统（高层任务管理器、中层技能模块、低层 CEER 控制器）。
- **创新（Innovation）**：
    1.  提出了一个新颖的、统一的、与规划器兼容的 **EE-Root 控制抽象**，将复杂的全身控制简化为直观的任务空间命令。
    2.  设计了一个 **教师-学生蒸馏框架**，成功地将一个需要全身关节目标作为输入的柔顺运动跟踪策略，转化为一个仅依赖 EE-Root 命令的低层策略，从而解耦了任务规划与具体形态。
    3.  构建了一个 **三层分层系统**，通过 EE-Root 接口实现了异构规划器（LLM、遥操作、扩散策略等）的即插即用集成，无需重新训练低层控制器。
- **意义（Significance）**：这项工作为人形机器人的全身移动操作提供了一个模块化、可扩展且实用的控制范式。它解决了高层任务规划与低层物理控制之间的“接口鸿沟”，使得利用大语言模型等高级AI技术来指挥人形机器人完成复杂的、接触丰富的长周期任务成为可能，推动了人形机器人在真实环境中的通用性和实用性。
- **一句话总结**：本文提出 CEER，一种基于末端执行器-根部（EE-Root）的统一柔顺控制接口，通过分层架构实现了人形机器人移动操作中高层规划与低层控制的解耦，支持多种规划器的即插即用，并在仿真和实物上验证了其在接触丰富操作和长周期任务中的有效性。

## 1. 研究动机（Motivation）

人形机器人在运动（locomotion）方面已取得显著进展，但在接触丰富的操作（contact-rich manipulation）和长周期任务（long-horizon tasks）上仍是主要瓶颈。

- **核心问题**：现有的人形机器人控制方法，尤其是基于运动捕捉和模仿学习的策略，主要针对运动跟踪优化，存在以下关键挑战：
    1.  **输入维度高且不直观**：这些方法通常需要**全身关节目标**作为输入。高层规划器（如LLM）很难直接从任务描述和感知信息中生成如此高维度的关节轨迹。
    2.  **形态特异性**：关节级别的命令是**特定于机器人形态**的，无法直接在不同机器人之间迁移，降低了数据效率和泛化能力。
    3.  **缺乏柔顺性**：纯运动跟踪缺乏**显式的柔顺调制**能力。在与物体或人进行接触时，无法安全、鲁棒地控制交互力，容易导致任务失败或损坏物体/机器人。
- **为什么重要**：为了让机器人能在人类环境中有效工作，它必须能够安全地进行接触丰富的操作（如抓取、推、拉、擦拭），并能适应各种不同的任务。当前的方法要么过于复杂（需要规划全身关节），要么不够安全（缺乏柔顺性），要么无法与高级AI规划器有效集成。
- **挑战所在**：核心挑战在于如何设计一个**有效的任务级表示**，既能降低规划器的输出维度，使其易于与高层规划器（如LLM）对接，又能与底层物理控制无缝衔接，同时保证接触交互的柔顺性和安全性。

## 2. 创新点（Innovation）

论文的关键创新在于提出了 **CEER** 这一统一控制抽象，并围绕它构建了整个系统。其创新之处主要体现在以下三点：

1.  **提出 EE-Root 控制抽象**：
    - **新方法**：将人形机器人全身移动操作的控制命令统一为两个部分：**末端执行器（EE）的位姿目标**（位置和朝向）和**机器人根部（Root）的运动命令**（如线速度和偏航角速度）。
    - **与现有方案的不同**：不同于传统的关节空间跟踪或全身姿态跟踪，EE-Root 抽象将任务空间从高维（如29维关节）压缩到低维（16维），且命令本身是**直观、可解释**的（例如，“右手移动到桌子上的盒子位置”）。这使得高层规划器（如LLM）可以自然地生成这些命令。
    - **为何奏效**：在接触丰富的操作中，交互主要发生在末端执行器，因此EE目标是一个自然的任务抽象。结合根部运动来处理移动，这个组合覆盖了移动操作的核心需求。它解耦了“做什么”（任务目标）和“怎么做”（具体关节运动）。

2.  **设计教师-学生蒸馏框架**：
    - **新方法**：首先训练一个**教师策略**，该策略使用全身关节目标作为输入，并具备柔顺性（通过阻抗控制建模）。然后，训练一个**学生策略**，它**仅接收 EE-Root 命令**，并学习模仿教师策略的行为。
    - **与现有方案的不同**：传统方法要么直接学习关节跟踪，要么需要为每个新任务重新训练。本文通过蒸馏，将复杂的、需要全身信息的柔顺控制能力“压缩”到一个仅依赖任务级命令的策略中。
    - **为何奏效**：教师策略提供了高质量的、具备柔顺性的全身运动示范。学生策略通过一个**目标关节估计器**，从EE-Root命令和本体感知中预测出教师策略所需的关节目标，从而能够“继承”教师的能力。这使得低层控制器与具体的机器人形态（关节数量、连杆长度）解耦，实现了跨形态的潜在通用性。

3.  **构建即插即用的三层分层系统**：
    - **新方法**：基于EE-Root接口，构建了一个三层架构：**高层**（LLM任务管理器）→ **中层**（模块化技能，如遥操作、路径规划、扩散策略）→ **低层**（CEER控制器）。
    - **与现有方案的不同**：所有中层技能模块的输出都被统一转换为 EE-Root 命令格式。这使得高层规划器（如LLM）可以像调用函数一样调用不同的技能模块，而无需关心底层控制细节。更换或添加新的技能模块（例如，从键盘遥操作切换到VLM规划）时，低层控制器**完全不需要重新训练**。
    - **为何奏效**：EE-Root 接口起到了“通用翻译器”的作用。它将各种异构规划器（LLM、人类遥操作、学习算法）的输出标准化，使得它们都能与同一个低层控制器通信。这种解耦极大地增强了系统的**模块化、可扩展性和灵活性**。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心是 CEER 控制抽象及其构建的三层分层系统。

- **低层控制（CEER 策略）**：一个基于强化学习训练的学生策略。它接收 **EE-Root 命令**（16维：根部位姿 + 末端执行器位姿）和**本体感知信息**（关节位置、角速度等），输出**关节位置目标**（29维），由底层的PD控制器执行。该策略通过教师-学生蒸馏获得了柔顺性。
- **中层技能层**：包含多种模块化技能，所有技能的输出都统一为 EE-Root 命令格式。
    - **遥操作技能**：VR或键盘输入，直接映射为EE-Root命令。
    - **移动技能**：基于路径规划（如Hybrid A*），将导航目标转换为根部速度命令。
    - **操作技能**：包括基于规则、VLM（视觉语言模型）或扩散策略（Diffusion Policy）的轨迹生成器，输出末端执行器的位姿序列。
- **高层任务管理器**：一个基于LLM的智能体。它接收自然语言指令，通过函数调用接口选择并组合中层技能，生成一个 EE-Root 命令序列。它通过“推理-行动-观察”的闭环循环来执行任务。

### 3.2 关键算法与数学推导

1.  **阻抗控制公式**：用于建模柔顺性。
    - 笛卡尔空间阻抗控制：
      $$F = K_m(\ddot{x} - \ddot{x}_d) + K_d(\dot{x} - \dot{x}_d) + K_p(x - x_d)$$
      其中 $x$ 是任务空间状态，$x_d$ 是期望状态，$K_m, K_d, K_p$ 分别是惯性、阻尼和刚度矩阵。
    - 在准静态假设下，稳态阻抗动力学简化为：
      $$K_p(x_t^{imp} - x_t^{ref}) + K_d(\dot{x}_t^{imp} - \dot{x}_t^{ref}) = f_{ext, t}$$
      其中 $f_{ext}$ 是外部力。
    - 假设速度跟踪良好 ($\dot{x}_t^{imp} = \dot{x}_t^{ref}$)，则柔顺目标位置为：
      $$x_t^{imp} = x_t^{ref} + K_p^{-1} f_{ext, t}$$
      这个 $x_t^{imp}$ 作为教师策略的“特权观测”，用于指导其学习如何在外力下调整目标位置，从而实现柔顺行为。

2.  **教师-学生策略学习**：
    - **教师策略 $\pi_T$**：观测 $o_t^T = [c_t^T, o_t, o_t^p]$，其中 $c_t^T \in \mathbb{R}^{29}$ 是关节空间参考命令，$o_t$ 是共享的本体感知，$o_t^p$ 是特权信息（包括外力 $f_{ext}$、阻抗状态等）。输出关节命令 $a_t \in \mathbb{R}^{29}$。
    - **学生策略 $\pi_S$**：观测 $o_t^S = [c_t^S, o_t]$，其中 $c_t^S \in \mathbb{R}^{16}$ 是 EE-Root 命令。为了让学生策略能与教师策略共享网络结构，论文引入了两个估计器：
        - **特权状态估计器 $f_{\theta_p}$**：从 $o_t$ 预测特权信息 $o_t^p$。
        - **目标关节估计器 $f_{\theta_q}$**：从 $[o_t, c_t^S]$ 预测 29 维的关节参考命令 $c_t^T$。
    - 学生策略的最终输入是 $[f_{\theta_q}(o_t, c_t^S), o_t, f_{\theta_p}(o_t)]$，其维度和结构与教师策略的输入一致，因此可以直接继承教师策略的权重进行微调，大大加速和稳定了训练过程。

3.  **奖励函数**：
    - 总奖励：$r_{total} = w_{ee} r_{ee} + w_c r_{compliance} + w_m r_{motion} + w_l r_{locomotion}$
    - 其中，$r_{ee} = \exp(-\lVert x_t^{ee} - x_t^{ref} \rVert^2)$ 是新增的**末端执行器跟踪奖励**，用于显式鼓励任务空间的准确性。

### 3.3 模型、训练与数据集

- **模型**：策略网络采用 Actor-Critic 架构，具体结构如图3所示。教师和学生策略共享类似的网络结构，但输入不同。学生策略额外包含两个估计器网络。
- **训练**：
    - **算法**：近端策略优化（PPO）。
    - **环境**：在 Isaac Lab 模拟器中训练，使用 16,384 个并行环境。
    - **硬件**：4 块 NVIDIA L40s GPU，训练约 8 小时。
    - **数据集**：教师策略使用 **AMASS** 数据集进行运动模仿训练。该数据集包含大量高质量的人体运动捕捉数据。
    - **训练过程**：两阶段训练。首先训练教师策略，使其能够跟踪全身关节目标并具备柔顺性（通过在训练中加入外部力扰动）。然后，冻结教师策略，训练学生策略及其两个估计器，使其仅从 EE-Root 命令和本体感知中模仿教师的行为。

### 3.4 实验设置与结果

- **实验平台**：Unitree G1 人形机器人，配备 RealSense D435 摄像头和多种可更换末端执行器（钩子、笔架、板擦架）。
- **实验问题**：
    1.  CEER 的末端执行器跟踪精度如何？
    2.  柔顺控制能否实现稳定的接触丰富操作？
    3.  异构规划器能否通过 EE-Root 接口即插即用？
    4.  分层系统能否完成语言指定的长周期任务？

- **实验结果**：
    1.  **跟踪性能（表I）**：
        - CEER 实现了 **3.3 cm** 的位置 RMSE 和 **0.32 rad** 的姿态误差。
        - CEER 的 **jerk（加加速度）** 显著低于所有基线，表明运动更平滑、更稳定。
        - 与没有目标关节估计器的变体（Ours w/o fθq）相比，CEER 的跟踪精度大幅提升，证明了该组件的关键作用。
    2.  **实物遥操作（图4）**：
        - 成功完成了**简单操作**（抓取不同大小盒子、递物）、**强力操作**（写字、擦黑板、旋转盒子、插笔帽）和**移动操作**（拉车、搬运盒子）等三大类任务，展示了 CEER 在真实接触场景下的柔顺性和稳定性。
    3.  **规划器兼容性（图5）**：
        - 成功将规则式、扩散策略和 VLM 三种异构规划器集成到同一个 CEER 低层控制器上，无需任何修改或重新训练。所有规划器生成的轨迹都能被 CEER 正确执行。
    4.  **系统级评估（表II、III，图6、7）**：
        - 在模拟房间环境中，CEER 系统在**简单任务**上接近完美。
        - 在**单物体移动操作**任务上，成功率（70%）与人类遥操作相当，但完成时间更短。
        - 在**长周期多物体任务**上，系统成功率为 20%，低于人类的 40%。失败原因分析（图7）显示，**52.6% 的失败源于 LLM 的规划错误**（如距离和角度估计不准），而人类的优势在于更强的**恢复能力**。

### 3.5 技术细节

- **命令空间**：EE-Root 命令 $c_t^S \in \mathbb{R}^{16}$ 包含根部位置（x, y, z）和偏航角（yaw），以及两个末端执行器的位置和朝向（每个EE 6-DoF）。
- **控制器频率**：低层 CEER 控制器运行在 **50 Hz**，中层技能模块运行在 10-20 Hz。
- **遥操作映射**：VR 控制器的 6-DoF 位姿被直接映射为机器人末端执行器的目标位姿。键盘遥操作使用简化的规则（如对称抓取）。
- **LLM 集成**：使用 LLM 的函数调用功能。每个中层技能被注册为一个带有空间参数（如目标位置）的函数。LLM 通过“推理-行动-观察”循环进行任务规划，系统提示词中编码了工作空间坐标、手部状态和安全约束等专业知识。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了人形机器人领域一个核心的**接口鸿沟**问题：即高层任务规划（需要抽象、可解释的命令）与低层物理控制（需要具体、连续的关节指令）之间的不匹配。
    - 提出了一个**实用、高效且可扩展**的解决方案（CEER），通过一个统一的 EE-Root 抽象，实现了柔顺控制、模块化集成和跨形态的潜在通用性。
    - 通过实物和仿真实验，**验证了该范式的有效性**，特别是在接触丰富的操作和与 LLM 集成的长周期任务中。

- **对未来研究/应用的影响**：
    - **推动人形机器人的通用化**：CEER 为构建更通用的人形机器人智能提供了一个可行的框架。未来的研究可以专注于改进中层技能（如更鲁棒的抓取、更自然的移动），而无需担心底层控制问题。
    - **促进AI与机器人的深度融合**：EE-Root 接口使得大语言模型、视觉语言模型等高级AI技术可以更自然地“指挥”人形机器人，加速了具身智能的发展。
    - **降低机器人应用门槛**：模块化的设计使得非机器人专家也能通过高层接口（如自然语言）来部署和指挥人形机器人完成复杂任务。

- **局限与开放问题**：
    - **固定的阻抗参数**：当前的阻抗参数（$K_p$）是固定的，无法根据任务需求自适应调整（例如，抓取鸡蛋时需要低刚度，拧瓶盖时需要高刚度）。
    - **被动的臂部控制**：在移动时，如果未指定末端执行器轨迹，手臂会保持静止。未来需要让策略能够自主决定手臂的摆动，以实现更自然的移动。
    - **LLM 规划的瓶颈**：系统在长周期任务上的失败主要源于 LLM 的空间推理不准确（如距离、角度）。如何提升 LLM 的鲁棒性和空间感知能力，或引入更可靠的视觉反馈机制，是未来的重要方向。
    - **恢复能力不足**：与人类相比，系统在出错后的恢复能力较弱。如何设计更有效的错误检测和恢复机制，是提升系统鲁棒性的关键。

## 5. 澄清与简化（Clarifications and Simplifications）

- **EE-Root 抽象类比**：可以把人形机器人想象成一个**可以移动的办公椅**。
