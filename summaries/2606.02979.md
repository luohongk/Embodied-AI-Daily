## 0. 概览（Concise Summary）

- **主题（Topic）**：自动驾驶感知领域，具体涉及多任务学习、多传感器融合以及不平衡学习问题。
- **问题（Problem）**：现有自动驾驶感知系统通常需要多个独立模型处理不同任务（如语义分割、深度估计等），导致计算成本高、效率低。同时，多任务学习模型在处理多个异构任务时，容易出现梯度冲突和不平衡学习问题，导致某些任务性能不佳。此外，如何有效融合来自不同传感器（如RGB相机、动态视觉传感器、激光雷达）的多模态数据也是一个挑战。
- **方法（Method）**：提出一个紧凑的深度多任务学习模型，该模型通过一个共享的编码器-解码器架构，在单次前向传播中同时完成多个视角的语义分割、深度估计、激光雷达分割和鸟瞰图投影任务。为了解决不平衡学习问题，论文采用并改进了梯度归一化（GradNorm）算法，自适应地调整各任务的损失权重。同时，通过数据预处理和中间传感器融合技术，有效融合了RGB、DVS和LiDAR三种传感器的数据。
- **创新（Innovation）**：
    1.  **紧凑的多任务模型**：设计了一个统一的、端到端的模型，无需其他模型辅助，即可同时执行4种感知任务（SS, DE, LS, BEVP）和5个视角（前、左、右、后、顶）的输出。
    2.  **自适应损失加权算法**：修改了GradNorm算法，使其适用于具有多个瓶颈层的复杂多任务模型，有效解决了多任务学习中的不平衡问题。
    3.  **多传感器融合策略**：通过中间融合技术，在模型内部的特征层面融合了RGB、DVS和LiDAR三种模态的数据，提升了模型在复杂环境下的鲁棒性。
- **意义（Significance）**：该工作为构建高效、紧凑的自动驾驶感知系统提供了一个有前景的解决方案。通过减少模型参数和计算量，实现了更快的推理速度和更低的GPU内存占用，这对于资源受限的嵌入式平台至关重要。同时，其自适应损失加权方法也为解决多任务学习中的不平衡问题提供了新的思路。
- **一句话总结**：本文提出了一种紧凑的多任务、多传感器融合的自动驾驶感知模型，并通过改进的自适应损失加权算法有效解决了多任务学习中的不平衡问题，在保持或提升性能的同时大幅降低了模型复杂度。

## 1. 研究动机（Motivation）

自动驾驶的感知阶段是整个系统的基石，需要全面理解车辆周围的环境。这通常涉及多个感知任务，如语义分割、深度估计、目标检测等。然而，现有方法存在以下关键问题和挑战：

1.  **计算效率低下**：传统的做法是为每个感知任务单独训练一个模型，然后将这些模型串联或并联起来使用。这种方式计算成本极高，推理速度慢，且占用大量GPU内存，难以满足自动驾驶对实时性的要求。
2.  **多任务学习的不平衡问题**：多任务学习（MTL）旨在通过共享特征来提升效率和性能。然而，当多个任务（如分类和回归）同时训练时，不同任务的损失函数尺度、梯度大小和学习速度可能存在巨大差异，导致模型偏向于学习某些“简单”或“主导”的任务，而忽略了其他任务，造成性能下降。手动调整各任务的损失权重非常繁琐且耗时。
3.  **单一传感器的局限性**：任何单一传感器都有其固有的缺陷。例如，RGB相机在夜间或恶劣天气下性能会急剧下降。因此，需要融合多种传感器（如RGB相机、DVS、LiDAR）的数据，利用它们各自的优势（如RGB的丰富纹理、DVS的高动态范围、LiDAR的深度信息），以实现鲁棒的环境感知。
4.  **多视角感知的复杂性**：为了获得车辆周围360度的完整场景理解，需要处理来自多个视角（前、左、右、后）的信息，并生成鸟瞰图（BEV）表示，这对模型的架构设计提出了更高的要求。

因此，本文的研究动机是**开发一个紧凑、高效且鲁棒的自动驾驶感知模型，该模型能够在一个统一的框架内，通过多传感器融合和自适应学习策略，同时解决多个感知任务，并克服多任务学习中的不平衡问题。**

## 2. 创新点（Innovation）

本文的主要创新点可以概括为以下三个方面：

1.  **提出了一种新颖的紧凑型多任务学习模型架构**：
    - **创新点**：设计了一个基于编码器-解码器结构的统一模型，该模型能够同时处理来自RGB、DVS和LiDAR三种传感器的输入，并输出四个视角（前、左、右、后）的语义分割（SS）和深度估计（DE），以及一个顶视角的激光雷达分割（LS）和鸟瞰图投影（BEVP）。所有任务都在单次前向传播中完成，无需依赖其他外部模型。
    - **与现有方案的不同**：不同于需要多个独立模型或级联模型的方法，本文的模型是端到端的，更加紧凑和高效。它借鉴了U-Net的跳跃连接和中间融合的思想，但针对多任务、多视角、多模态输入进行了专门设计，例如为RGB和DVS分别设计了独立的编码器，并通过两个瓶颈层进行特征融合。
    - **为何奏效**：共享编码器可以学习到不同任务和模态之间的通用特征，而任务特定的解码器则保留了每个任务的独特性。跳跃连接有助于梯度流动和细节恢复。中间融合策略允许模型在特征层面而非决策层面进行信息交互，从而更有效地利用多模态数据。

2.  **提出了一种改进的自适应损失加权算法（MGN）**：
    - **创新点**：对GradNorm算法进行了修改，使其能够应用于具有多个共享瓶颈层的复杂多任务模型。原始GradNorm假设只有一个共享层，而本文的模型有两个瓶颈层。MGN算法通过分别计算不同任务相对于不同瓶颈层权重的梯度范数，并动态调整损失权重，以平衡各任务的学习速度。
    - **与现有方案的不同**：不同于手动调整损失权重或使用固定的权重，MGN能够根据训练过程中的梯度信息自动调整。与原始GradNorm相比，MGN考虑了模型架构的复杂性，将梯度计算应用于两个不同的瓶颈层，从而更精确地控制不同任务的学习动态。
    - **为何奏效**：通过强制不同任务相对于共享层的梯度范数具有相似的大小，MGN可以防止某个任务主导学习过程，确保所有任务都能得到充分训练。这直接解决了多任务学习中的不平衡问题，从而提升了模型的整体性能。

3.  **提出了一种有效的多传感器数据预处理与融合方法**：
    - **创新点**：提出了一种将LiDAR点云预处理为15通道3D张量的方法，该方法不仅保留了点云的平面位置信息（x, y）和入射角余弦值，还通过分层（z轴高度）的方式保留了垂直方向的结构信息。同时，通过中间融合技术，在模型内部的特征层将RGB、DVS和LiDAR的特征图进行融合。
    - **与现有方案的不同**：相比于将LiDAR点云投影为单通道或三通道图像的方法，本文的15通道表示法提供了更丰富的几何信息。相比于后期融合（在决策层融合），中间融合允许模型学习到不同模态数据之间的复杂交互关系。
    - **为何奏效**：丰富的LiDAR表示法为模型提供了更精确的3D结构信息，有助于提升LS和BEVP的性能。中间融合策略使得模型能够利用一种模态的特征来增强另一种模态的特征，例如，利用LiDAR的深度信息辅助RGB图像的语义分割，从而提升整体鲁棒性。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文提出的模型架构如图2所示，其核心是一个**多编码器-多解码器**的网络，并辅以**跳跃连接**和**中间融合**策略。

- **输入**：模型接收三种模态的输入：
    - 4个视角（前、左、右、后）的RGB图像。
    - 4个视角（前、左、右、后）的DVS图像。
    - 1个顶视角的LiDAR点云（经过预处理）。
- **编码器**：每种输入模态和每个视角都有其独立的编码器路径。编码器由多个卷积块组成，每个卷积块包含两个`3x3`卷积层、批归一化和ReLU激活，后跟一个`2x2`最大池化层。通过编码器，特征图的空间维度减半，通道数加倍。
- **瓶颈层与融合**：模型设计了两个瓶颈层（Bottleneck）：
    - **第一瓶颈层**：融合来自所有RGB、DVS和LiDAR编码器的特征图。这些特征图被拼接（Concatenation）后，通过卷积块进一步处理。该瓶颈层的输出用于生成SS、DE和LS任务的输出。
    - **第二瓶颈层**：融合来自第一瓶颈层的特征以及SS、DE和LS解码器的部分特征，用于生成最终的BEVP任务输出。这体现了“任务辅助任务”的思想，即利用其他任务的中间结果来帮助BEVP任务。
- **解码器**：每个输出任务（SS, DE, LS, BEVP）都有其独立的解码器路径。解码器结构与编码器对称，使用`2x2`双线性上采样层来恢复空间分辨率，通道数相应减半。
- **跳跃连接**：每个编码器与其对应的解码器之间通过跳跃连接（图2中的红线）相连。这些连接将编码器的特征图与解码器上采样后的特征图进行拼接，有助于恢复下采样过程中丢失的细节信息，提升分割和估计的精度。
- **输出**：模型同时输出：
    - 4个视角的语义分割图（SS）。
    - 4个视角的深度估计图（DE）。
    - 1个顶视角的激光雷达分割图（LS）。
    - 1个顶视角的鸟瞰图投影（BEVP）。

### 3.2 关键算法与数学推导

**1. 损失函数**

- **深度估计（DE）损失**：使用Huber损失，它结合了MSE和MAE的优点，对异常值更鲁棒。
  $$L_{DE} = \frac{1}{V} \sum_{i=1}^{V} \frac{1}{N} \sum_{j=1}^{N} z_{ij}$$
  其中 $z_{ij}$ 定义为：
  $$z_{ij} = \begin{cases} 0.5(\hat{y}_{ij} - y_{ij})^2 & \text{if } |\hat{y}_{ij} - y_{ij}| < \delta \\ \delta(|\hat{y}_{ij} - y_{ij}| - 0.5\delta) & \text{otherwise} \end{cases}$$
  这里，$V=4$是视角数量，$N$是每个视角的像素数，$\hat{y}_{ij}$和$y_{ij}$分别是预测值和真实值，$\delta=0.5$是阈值。

- **分割任务（SS, LS, BEVP）损失**：使用二元交叉熵（BCE）和Dice损失的组合。
  $$L_{\{SS, LS, BEVP\}} = \frac{1}{V} \sum_{i=1}^{V} \left[ \frac{1}{N} \sum_{j=1}^{N} \left( y_{ij} \log(\hat{y}_{ij}) + (1-y_{ij}) \log(1-\hat{y}_{ij}) \right) + \left( 1 - \frac{2|\hat{y}_i \cap y_i|}{|\hat{y}_i| + |y_i|} \right) \right]$$
  其中，$V$对于SS是4，对于LS和BEVP是1。BCE处理像素级分类，Dice损失则关注预测和真实区域的整体重叠度，能有效缓解类别不平衡问题。

- **总损失**：
  $$L_{total} = \lambda \Sigma W^2 + \sum_{i=1}^{T} w_i L_i$$
  其中，$\lambda=0.0001$是权重衰减系数，$W$是模型权重，$T=4$是任务数量，$w_i$是任务$i$的损失权重，$L_i$是任务$i$的损失。

**2. 自适应损失加权算法（MGN）**

MGN算法的核心思想是**动态调整损失权重 $w_i(t)$**，使得不同任务相对于共享层权重的梯度范数保持平衡。

- **核心变量**：
    - $G_W^{(i)}(s)$：任务$i$的加权损失 $w_i(s)L_i(s)$ 相对于共享权重子集 $W(s)$ 的梯度 $L_2$ 范数。
    - $\bar{L}_i(s) = L_i(s) / L_i(0)$：任务$i$的损失比，反映其训练速度（值越小，学习越快）。
    - $r_i(s) = \bar{L}_i(s) / \frac{1}{T}\sum_{i=1}^{T} \bar{L}_i(s)$：任务$i$的相对逆训练速率。
    - $\bar{G}_W(s) = \frac{1}{T}\sum_{i=1}^{T} G_W^{(i)}(s)$：所有任务梯度范数的平均值。

- **目标**：使每个任务的梯度范数 $G_W^{(i)}(s)$ 接近一个目标值 $\bar{G}_W^{(i)}(s)$。
  $$\bar{G}_W^{(i)}(s) = \bar{G}_W(s) \cdot [r_i(s)]^\alpha$$
  其中，$\alpha=1.5$是一个超参数，控制平衡的强度。

- **MGN损失**：通过最小化实际梯度范数与目标梯度范数之间的差异来更新损失权重 $w_i$。
  $$L_{MGN}(s) = \sum_{i=1}^{T} \left| G_W^{(i)}(s) - \bar{G}_W^{(i)}(s) \right|$$

- **权重更新与归一化**：使用SGD优化器最小化 $L_{MGN}$ 来更新 $w_i$，然后对所有 $w_i$ 进行归一化，使其和为 $T$。
  $$w_i^{new}(s) = \frac{w_i(s)}{\sum_{i=1}^{T} w_i(s)} \cdot T$$

- **关键修改**：原始GradNorm在每个训练步都更新权重，而MGN只在每个epoch的最后一步（$t=s$）更新，以减少计算开销。此外，MGN针对具有两个瓶颈层的模型，为不同任务选择了不同的共享权重子集 $W(s)$：BEVP任务相对于第二瓶颈层的权重，其他任务相对于第一瓶颈层的权重。

### 3.3 模型、训练与数据集

- **模型**：如前所述，是一个多编码器-多解码器的紧凑型CNN模型。模型参数数量远少于对比的多个单任务模型之和。
- **训练**：
    - **优化器**：SGD。
    - **权重初始化**：Kaiming初始化。
    - **正则化**：Dropout（p=0.5）和权重衰减（$\lambda=0.0001$）。
    - **学习率调度**：MGN算法的学习率 $\eta_{MGN}$ 从0.1开始，如果验证集的总指标（TM）在连续4个epoch内没有下降，则减半，直到最小值0.0001。
- **数据集**：
    - **模拟数据集（A, B, C）**：使用CARLA模拟器生成。包含RGB、DVS、LiDAR数据以及SS、DE、LS、BEVP的ground truth。数据集A和B各有2000个样本，使用2个地图；数据集C有10000个样本，使用5个地图。数据涵盖了多种天气、光照和交通状况。传感器配置如图4所示。
    - **真实世界数据集（nuScenes-lidarseg）**：用于验证模型在真实场景下的泛化能力。由于该数据集不提供DVS图像和SS、DE的ground truth，作者移除了DVS编码器，并利用LiDAR点云的类别和距离信息为SS和DE任务生成了近似的ground truth。同时，将图像尺寸调整为128x128。

### 3.4 实验设置与结果

- **实验设置**：作者进行了消融研究和对比研究。
    - **消融研究**：旨在验证MGN算法和不同LiDAR预处理方法的效果。比较了：
        1.  无MGN（固定权重）。
        2.  有MGN。
        3.  使用1通道LiDAR表示（仅入射角）。
        4.  使用15通道LiDAR表示（包含高度信息）。
    - **对比研究**：将本文的最佳模型（使用MGN和15通道LiDAR）与多个现有模型（包括单任务模型和多任务模型）的组合进行比较，这些模型共同完成相同的4个任务。
- **评估指标**：
    - **深度估计**：平均绝对误差（MAE）。
    - **分割任务**：交并比（IoU）。
    - **总指标（TM）**：$TM = MAE_{DE} + (1-IoU_{SS}) + (1-IoU_{LS}) + (1-IoU_{BEVP})$，值越低越好。
    - **指标方差（MV）**：衡量各任务性能的平衡程度，值越低越平衡。
- **主要结果**：
    1.  **消融研究**：使用MGN和15通道LiDAR表示的模型变体在所有数据集上取得了最佳的总指标（TM）和更低的指标方差（MV），证明了MGN算法和丰富LiDAR表示的有效性。
    2.  **对比研究**：与多个现有模型的组合相比，本文的模型在参数数量、模型大小、推理速度和GPU内存占用方面具有显著优势。同时，在大多数任务和数据集上，其性能（TM）与更复杂的模型组合相当甚至更好。这表明了其紧凑性和高效性。
    3.  **真实世界数据集**：在nuScenes-lidarseg数据集上的实验表明，该模型能够处理真实世界的数据，并取得有竞争力的结果，验证了其泛化能力。

### 3.5 技术细节

- **LiDAR数据预处理**：这是本文的一个重要技术细节。作者提出了两种方法：
