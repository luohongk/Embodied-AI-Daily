## 0. 概览（Concise Summary）

- **主题（Topic）**：临床表格预测（Clinical Tabular Prediction）中的自动化特征工程（Automated Feature Engineering）。
- **问题（Problem）**：现有的基于大语言模型（LLM）的特征工程方法，在生成新特征时，没有考虑下游模型（如逻辑回归、XGBoost）的归纳偏置（inductive bias）和当前特征的重要性。这导致生成的特征要么是模型已经能学到的，要么是针对低重要性特征，从而限制了性能提升，尤其是在数据不平衡、特征异质性强的临床数据上。
- **方法（Method）**：提出了 **MedFeat** 框架，这是一个受机器学习从业者工作流程启发的、迭代式的特征工程智能体。它通过利用模型的可解释性信号（如 SHAP 值）来感知模型状态，并据此引导 LLM 生成对当前模型最有帮助的新特征。
- **创新（Innovation）**：
    1.  **模型感知（Model-Aware）**：首次在 LLM 特征工程中显式考虑下游模型的类型和归纳偏置，指导 LLM 生成模型难以自行学习的特征（如为线性模型生成非线性交互特征）。
    2.  **可解释性驱动（Explainability-Driven）**：利用特征重要性（SHAP）信号来定位当前模型预测能力薄弱的特征空间区域，从而进行有目标的特征探索。
    3.  **迭代式工作流**：模拟了实践者“训练-解释-定位-提出-评估-反思”的迭代过程，形成一个闭环反馈系统。
- **意义（Significance）**：MedFeat 在多个具有挑战性的真实临床预测任务上，显著且稳定地超越了所有现有基线方法（包括传统自动特征工程和 LLM 方法），平均性能提升超过 10%。它证明了将模型感知和可解释性信号融入 LLM 特征工程的有效性，为自动化临床特征工程提供了一个更智能、更实用的新范式。
- **一句话总结**：MedFeat 通过让 LLM 感知下游模型的“知识盲区”并利用特征重要性信号进行定向探索，实现了更高效、更精准的临床表格数据自动特征工程。

## 1. 研究动机（Motivation）

- **问题与空白**：在临床表格预测中，传统机器学习模型（如 XGBoost、逻辑回归）配合手工特征工程，其性能往往优于深度学习方法。然而，手工特征工程耗时且依赖领域专家。现有的自动化方法（如 AutoFeat, OpenFE）通过穷举预定义算子，会产生庞大的特征空间且缺乏任务上下文。近期，LLM 被用于自动化此过程，它们能作为领域专家提出多样化的特征变换。但是，**现有 LLM 方法的一个核心缺陷是：它们将特征生成与下游模型解耦**。LLM 在生成特征时，既不知道当前模型已经学到了什么（模型感知缺失），也不知道哪些特征对当前预测最重要（特征归因缺失）。因此，它们提出的特征要么是模型已经能通过自身结构学到的（冗余），要么是针对低重要性特征的（无效），而忽略了那些能真正扩展模型表征能力的变换。
- **为什么重要**：临床数据具有严重的类别不平衡、特征类型异质、缺失模式复杂等特点，这使得模型泛化困难。同时，临床场景对模型的可解释性、隐私性和训练成本有严格要求。因此，提升经典模型（如 XGBoost、逻辑回归）的性能，使其在不牺牲这些优势的前提下达到更高精度，具有重要的实际价值。自动化的、智能的特征工程是实现这一目标的关键。
- **挑战**：如何让 LLM 的生成过程“模型感知”和“归因感知”？如何将模型类型、特征重要性等非结构化信息有效地编码到 LLM 的提示（prompt）中？如何设计一个稳定、高效的迭代搜索策略，避免生成无效或有害的特征？

## 2. 创新点（Innovation）

- **关键创新**：提出了 **MedFeat** 框架，这是**首个**将模型感知（Model-Awareness）和可解释性驱动（Explainability-Driven）搜索相结合的 LLM 特征工程框架。
- **与现有方案的不同**：
    - **对比 CAAFE, FeatLLM, OCTree**：这些方法都是“学习者无关（learner-agnostic）”的，它们为所有下游模型生成相同的特征。MedFeat 则根据下游模型（如线性模型 vs. 树模型）的归纳偏置，动态调整提示词，鼓励生成模型难以学习的特征，避免生成模型已经能轻松学到的特征。
    - **对比 AutoFeat, OpenFE**：这些方法依赖固定的算子库进行穷举搜索，缺乏领域知识和任务上下文。MedFeat 利用 LLM 的领域知识和推理能力，并结合模型反馈信号，进行更智能、更定向的搜索。
- **为什么能奏效**：
    1.  **模型感知**：通过提示词中的“鼓励/避免”列表，MedFeat 引导 LLM 生成与下游模型互补的特征。例如，为线性模型生成非线性交互特征，为树模型生成跨时间聚合特征，从而最大化边际收益。
    2.  **可解释性驱动**：通过 SHAP 值构建“特征岛（feature islands）”，将 LLM 的注意力集中在当前模型预测能力薄弱的特征子集上，实现了高效的局部搜索，避免了在无关特征上的浪费。
    3.  **迭代闭环**：通过“训练-解释-搜索-生成-评估-更新”的循环，MedFeat 能够不断根据模型的最新状态调整搜索策略，形成一个自我优化的系统。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

MedFeat 的核心是一个迭代式智能体框架，其工作流程如图 1 所示，包含以下步骤：

1.  **初始化**：在原始训练集 `D_train` 上训练一个下游模型 `f_θ`。
2.  **解释（Explain）**：在验证集 `D_val` 上计算模型 `f_θ` 的特征重要性（SHAP 值），得到一个特征重要性向量 `s`。
3.  **搜索（Search）**：根据特征重要性 `s` 构建 K 个“特征岛（feature islands）”。每个岛是从当前特征空间中采样的 m 个特征的子集。采样概率与特征重要性成正比，从而平衡对高重要性特征的利用和对低重要性特征的探索。
4.  **生成（Generate）**：对于每个特征岛，构建一个“模型感知”的提示词（prompt）。该提示词包含：
    - 任务描述和数据集信息。
    - 当前特征岛的特征及其重要性。
    - **模型感知指令**：根据下游模型类型（如 XGBoost 或逻辑回归），列出应**鼓励**生成的特征类型（如交互、聚合、趋势）和应**避免**生成的特征类型（如简单缩放、阈值）。
    - 历史记忆（Memory）：之前成功和失败的特征变换，以避免重复。
    - 然后，LLM 根据提示词生成一个候选特征变换 `σ`（通常是一个 Python 函数）。
5.  **评估（Evaluate）**：执行候选变换，将其添加到数据集中，重新训练模型并在验证集上评估性能。
6.  **选择（Select）**：选择性能提升最大的候选特征。如果其性能超过当前基线（减去一个容忍阈值 `β`），则接受该特征，更新特征集和模型，并重新计算 SHAP 值；否则拒绝。
7.  **迭代**：重复步骤 2-6，直到达到预设的迭代次数 T。

### 3.2 关键算法与数学推导

**问题形式化**：
目标是找到一个特征变换集合 `σ`，使得在增强后的数据集上训练的模型性能最大化。这是一个双层优化问题：

$$
\max_{\sigma \in \Sigma} L(f_{\sigma}^{*}, D_{val} \oplus \sigma) \quad \text{s.t.} \quad f_{\sigma}^{*} = \arg\max_{f} L(f, D_{train} \oplus \sigma)
$$

其中，`L` 是评估指标，`Σ` 是所有可能变换的空间。由于该问题难以直接求解，MedFeat 采用迭代式、可解释性驱动的优化方法。

**特征重要性驱动的搜索**：
在第 `t` 次迭代，模型 `f_θt` 的特征重要性向量为 `s_t`。构建 K 个特征岛 `I_t,k`，每个岛包含 m 个特征。特征 `i` 被采样到岛中的概率为：

$$
p_t(i) = \tilde{s}_{t,i}
$$

其中 `\tilde{s}_{t,i}` 是归一化后的特征重要性。这确保了搜索集中在高影响力的特征上。

**候选评估与选择**：
对于第 `t` 次迭代的第 `k` 个岛生成的候选变换 `σ_{t,k}`，其验证集得分为：

$$
L_{t,k} = L(f_{\sigma_t \cup \{\sigma_{t,k}\}}^{*}, D_{val} \oplus (\sigma_t \cup \{\sigma_{t,k}\}))
$$

选择最佳候选 `k^* = \arg\max_k L_{t,k}`。接受条件为：

$$
L_{t,k^*} \ge L_{baseline} - \beta
$$

其中 `β` 是容忍阈值，用于防止因验证集噪声而拒绝有潜力的特征。

### 3.3 模型、训练与数据集

- **下游模型**：
    - **逻辑回归（Logistic Regression, LR）**：作为线性模型的代表，具有高可解释性。
    - **XGBoost**：作为树模型的代表，在表格数据上性能强大。
- **LLM 模型**：主要使用 **GPT-4o** 作为特征生成的骨干模型。
- **数据集**：使用了三个真实世界的临床数据集，覆盖了多种预测任务：
    - **MIMIC-IV**：重症监护室（ICU）数据。
    - **IORD**：牛津郡感染数据库。
    - **HRS**：健康与退休研究数据。
- **任务**：包括短期/长期死亡率预测、心力衰竭预测、认知障碍预测、出院预测和再入院预测等 7 个任务。这些任务普遍存在严重的类别不平衡问题。

### 3.4 实验设置与结果

- **基线方法**：与多种方法对比，包括无特征工程基线、传统自动特征工程（AutoFeat, OpenFE）和 LLM 方法（CAAFE, FeatLLM, OCTree）。
- **评估指标**：AUROC, F1 分数, AUPRC。
- **主要结果（表 1）**：
    - MedFeat 在所有 7 个任务和两个模型上，**一致且显著地**优于所有基线方法。
    - 平均 F1 分数提升超过 10%（相对于无特征工程基线）。
    - 在**最困难的任务**（如 ICU 死亡率预测）上提升最大（F1 提升 23-31%），表明其能有效挖掘稀有阳性样本的信号。
    - MedFeat 的**稳定性**更好，标准差通常小于或等于其他竞争方法。
    - MedFeat 对**弱学习器（逻辑回归）** 的提升更大，能有效弥补其表征能力不足的缺陷。
- **超参数优化（HPO）后的结果（表 2）**：即使经过广泛的超参数调优，MedFeat 带来的性能提升依然存在（F1 提升 4.3%），证明其生成的特征提供了原始特征空间中无法通过调参获得的信号。
- **消融实验（表 3）**：分别移除 SHAP 重要性、模型感知、特征岛采样和记忆模块，所有指标均下降，验证了每个组件的贡献。其中，移除 SHAP 重要性对 F1 和 AUPRC 影响最大，移除模型感知对 AUROC 影响最大。
- **特征多样性分析（图 3）**：MedFeat 生成的特征模式（如交互、聚合、趋势等）多样性最高，并且其分布会根据下游模型类型动态调整（例如，为逻辑回归生成更多“范围”特征，为 XGBoost 生成更多“聚合”特征），而其他方法则与模型无关。
- **跨队列迁移实验（图 4）**：将在 MIMIC-IV 上训练生成的 MedFeat 特征直接应用于 IORD 数据集，所有运行结果均有提升，证明了其生成的特征具有临床普适性和可迁移性。

### 3.5 技术细节

- **特征岛生成**：对于重复测量的时间序列特征，先将其作为一个“时间组”计算平均 SHAP 值，采样时要么选择整个时间组，要么选择静态特征，以避免共线性问题。
- **模型感知提示**：提示词中明确包含“鼓励”和“避免”列表。例如，对于 XGBoost，鼓励“行级分组和聚合、全局统计、复杂时间趋势”，避免“简单阈值、平凡交互、朴素缩放”。
- **容忍阈值 `β`**：设置为 0.01，用于在评估时提供一定的容错空间，避免因验证集上的微小波动而拒绝有价值的特征。
- **记忆机制**：维护一个已接受和已拒绝特征的记忆库。在后续迭代中，提示词会包含这些信息，以鼓励生成与成功特征相似的模式，并避免重复失败的模式。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了现有 LLM 特征工程方法“学习者无关”和“归因无关”的根本性缺陷。
    - 提出了一个更智能、更贴近人类专家工作流程的自动化特征工程范式。
    - 在多个具有挑战性的临床任务上取得了显著且稳定的性能提升，证明了其方法的有效性和鲁棒性。
- **对未来研究/应用的影响**：
    - **研究方面**：为 LLM 在自动化机器学习（AutoML）中的应用开辟了新思路，即如何将模型状态和可解释性信号作为 LLM 的“感知”输入，从而实现更智能的决策。未来工作可以探索更复杂的模型感知信号和搜索策略。
    - **应用方面**：MedFeat 可以直接应用于临床预测模型的开发，帮助医生和数据科学家快速发现新的、有临床意义的预测因子，提升模型性能。其跨队列迁移能力尤其宝贵，允许资源有限的医疗机构直接继承来自大型参考队列的已验证特征，而无需共享患者数据，具有很高的实用价值。
- **局限与开放问题**：
    - **对解释信号的依赖**：如果数据存在强共线性或噪声，特征重要性的排序可能不稳定。作者建议可以通过结合 L1/L2 正则化等预选特征方法来缓解。
    - **计算成本**：迭代式训练和评估模型会带来一定的计算开销。
    - **LLM 的可靠性**：LLM 生成的代码可能包含错误或产生无效特征，需要严格的验证和重试机制。

## 5. 澄清与简化（Clarifications and Simplifications）

- **模型感知（Model-Awareness）**：可以类比为“因材施教”。一个老师（LLM）在给学生（下游模型）出练习题（新特征）时，需要知道这个学生是擅长逻辑推理（线性模型）还是擅长总结归纳（树模型）。对于擅长逻辑推理的学生，老师会出一些需要直觉和发散思维的题目（非线性交互特征）；对于擅长总结归纳的学生，老师会出一些需要逻辑推理的题目（时间趋势特征）。MedFeat 就是让 LLM 扮演这个“因材施教”的老师。
- **特征岛（Feature Islands）**：可以理解为“聚焦探照灯”。整个特征空间是一个黑暗的森林，模型告诉你某些区域（高 SHAP 值特征）可能有宝藏。MedFeat 不是在整个森林里乱找，而是用探照灯（特征岛）聚焦在这些最有希望的区域，进行精细的搜索，这样效率更高。
- **SHAP 值**：可以理解为“模型对每个特征的打分”。这个分数告诉我们，在做出某个预测时，每个特征贡献了多少。MedFeat 利用这个分数来指导 LLM 优先改造那些“贡献大但模型可能还没用好”的特征。
