# Concordia: JIT-Compiled Persistent-Kernel Checkpointing for Fault-Tolerant LLM Inference

- arXiv: https://arxiv.org/abs/2606.23521v1
- arXiv ID: 2606.23521

---

🔍 **核心问题**：长时间运行的LLM推理任务在GPU或通信故障后，会丢失宝贵的GPU驻留状态（如KV缓存、请求调度器状态等），现有恢复机制要么重启整个服务栈，要么需要为每个注意力组件编写应用特定的检查点逻辑，缺乏通用高效的容错方案。

🛠 **方法**：提出Concordia运行时系统，利用设备驻留的持久化内核作为容错基础。它拦截GPU模块加载，支持PTX和SASS级别的指令插桩，在框架和库边界之下插入检查点与暂停钩子；通过JIT编译为每个LLM状态区域生成专用的增量检查点处理器（如KV块扫描器、适配器页扫描器），并热替换到持久化内核的操作表中；持久化内核使用无锁环形缓冲区处理计算、检查点、日志追加和恢复任务，将已提交记录追加到CPU可见的CXL内存或主机DRAM日志中。

✨ **主要贡献**：首次提出基于GPU驻留持久化内核的LLM推理容错框架，无需修改框架代码或库边界即可实现细粒度增量检查点与恢复；通过JIT编译专用处理器和热替换机制，在设备同步点高效执行脏页检测与日志记录，避免主机CPU成为关键路径，显著降低故障恢复开销。
