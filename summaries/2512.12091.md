# GraphPerf-RT: A Graph-Driven Performance Model for Hardware-Aware Scheduling of OpenMP Codes

- arXiv: https://arxiv.org/abs/2512.12091v3
- arXiv ID: 2512.12091

---

## 0. 概览（Concise Summary）

- **主题（Topic）**：异构嵌入式SoC上OpenMP任务并行应用的性能预测与调度优化
- **问题（Problem）**：现有方法无法同时捕捉任务DAG拓扑、代码语义和运行时上下文（DVFS、热状态）的跨层交互，且缺乏校准的不确定性估计，导致调度决策在热约束下不安全
- **方法（Method）**：提出GraphPerf-RT，一种异构图神经网络代理模型，统一编码任务DAG拓扑、CFG代码语义和运行时上下文，通过证据回归提供校准的不确定性量化
- **创新（Innovation）**：首次将任务DAG拓扑、CFG代码语义和运行时上下文（每核DVFS、热状态、利用率）统一到异构图中，并使用Normal-Inverse-Gamma先验的证据回归提供校准的不确定性
- **意义（Significance）**：在三个ARM平台上实现R²=0.81、Spearman ρ=0.95的预测性能，与多智能体模型强化学习集成后，相比无模型基线降低66%执行时间和82%能耗，且零热违规
- **一句话总结**：GraphPerf-RT通过异构图神经网络和证据回归，为异构嵌入式SoC上的OpenMP调度提供快速、准确且具有校准不确定性的性能预测，实现安全高效的任务调度

## 1. 研究动机（Motivation）

**解决的问题**：在异构嵌入式SoC（如Jetson TX2、Orin NX）上，OpenMP任务并行应用的性能预测面临四大挑战：

1. **跨层交互复杂**：性能由应用结构（任务DAG、CFG）、硬件状态（频率、热余量）和调度决策之间的跨层交互共同决定。经典启发式方法将这些因素独立处理，忽略了导致3-5倍执行时间变化的关键耦合效应。

2. **结构信息丢失**：表格模型将任务图展平为聚合统计量，丢弃了依赖结构；现有GNN方法关注静态表示，未纳入运行时上下文。

3. **缺乏不确定性估计**：标准回归缺乏风险感知调度所需的置信度估计，在热约束系统中可能导致过热或硬件损坏。

4. **RL探索风险高**：无模型RL需要大量设备上探索，存在过热风险；模型基RL需要准确的环境模型。

**为什么重要**：自主AI代理（自动驾驶、机器人、边缘AI）必须在严格资源、能量和热约束下做出实时调度决策。准确的性能预测和校准的不确定性对于确保安全运行至关重要。

**挑战**：需要一种方法能够：(1) 统一编码应用结构、硬件状态和调度决策；(2) 提供快速（毫秒级）推理；(3) 输出校准的不确定性估计；(4) 跨基准、输入大小和硬件平台泛化。

## 2. 创新点（Innovation）

**关键创新**：

1. **统一异构图表示**：首次将OpenMP任务DAG拓扑、CFG代码语义和运行时上下文（每核DVFS、热状态、利用率）统一到异构图中，通过类型化边（任务-任务、任务-资源、资源-资源、资源-内存）显式建模跨层性能交互。

2. **证据预测框架**：使用Normal-Inverse-Gamma（NIG）先验的证据回归，在单次前向传播中同时输出点预测和校准的不确定性（包括偶然不确定性和认知不确定性），支持多任务学习。

3. **实用AI代理集成**：与多智能体模型基RL（MAMBRL-D3QN）无缝集成，通过Dyna式合成轨迹实现安全高效的规划，相比无模型基线降低66%执行时间和82%能耗。

4. **可复现评估框架**：完整的数据流水线（OMPi + ALF-llvm + SWEET + 遥测），在三个ARM平台和42个基准测试上进行广泛实验，5种子统计显著性测试。

**与现有方案的不同**：
- 相比ProGraML（静态IR，无运行时上下文）：GraphPerf-RT集成DVFS、热状态和运行时遥测
- 相比HGT（无边缘分解、无热/DVFS集成、无证据头）：GraphPerf-RT具有类型化边缘分解、运行时上下文融合和证据回归
- 相比Ithemal（静态汇编，无并行性和热状态）：GraphPerf-RT建模OpenMP任务DAG和运行时上下文

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

**整体架构**（图3）：输入异构图（任务节点、资源节点、内存节点）→ 类型特定编码器 → 异构GAT层（3-6层）→ 层次池化 → 证据回归头 → 多任务预测+校准不确定性

**异构图表示**（图2）：
- **节点类型**：
  - 任务节点（V_T）：CFG特征（循环数、最大深度、圈复杂度、分支密度）、DAG拓扑度量（深度、到汇点距离、中介中心性）、静态代码统计、运行时快照
  - 资源节点（V_R）：每核DVFS步长、核心掩码位、集群ID、利用率（指数移动平均）、热余量（T_max - T_current）、温度趋势、带宽代理
  - 内存节点（V_M）：缓存层次标识、容量/关联性/行大小、延迟/带宽代理

- **边类型**：
  - E_TT（任务-任务）：优先约束，属性包括依赖类型（spawn/join/data）、关键边标志、跳距离、争用代理
  - E_TR（任务-资源）：调度分配下的任务到核心连接，属性包括亲和强度、迁移开销
  - E_RR（资源-资源）：共享硬件组件（L2缓存、内存控制器）的核心间连接，属性包括共享度、互连延迟
  - E_RM（资源-内存）：核心到缓存级别的连接，属性包括带宽分配

### 3.2 关键算法与数学推导

**类型特定编码器**：
$$h_v^{(0)} = \text{MLP}_{\text{type}(v)}(x_v; \theta_{\text{type}(v)})$$
其中 $\text{type}(v) \in \{T, R, M\}$，每个MLP有2-3层，ReLU激活，dropout p=0.1。

**异构GAT层**：
$$h_v^{(l+1)} = \sigma\left(h_v^{(l)} + \sum_{r \in R} \sum_{u \in N_r(v)} \alpha_{uv,r}^{(l)} W_r^{(l)} h_u^{(l)}\right)$$
其中 $R = \{TT, TR, RR, RM\}$，$N_r(v)$ 是边类型r下的邻居，$\alpha_{uv,r}^{(l)}$ 是注意力系数，$W_r^{(l)}$ 是类型特定变换，$\sigma$ 是ELU。多头注意力（4-8头）捕获多样交互模式。

**图级池化**：
$$h_G = \text{CONCAT}(\text{POOL}_T, \text{POOL}_R, \text{POOL}_M)$$
池化方式可以是均值、最大值或基于注意力的池化，产生固定大小的图表示（256-512维）。

**证据回归**：对于每个度量k，证据头输出NIG参数 $(\gamma_k, \nu_k, \alpha_k, \beta_k)$：
- 预测均值：$\hat{y}_k = \gamma_k$
- 偶然不确定性：$\text{Aleatoric}_k = \beta_k / (\alpha_k - 1)$
- 认知不确定性：$\text{Epistemic}_k = \beta_k / (\nu_k (\alpha_k - 1))$

**损失函数**：NIG负对数边际似然 + 非饱和不确定性正则化 + 排序损失
- 非饱和正则化：防止模型对高误差样本过度自信
- 排序损失：基于下置信界（LCB = $\gamma - k\sigma$）操作，其中 $\sigma = \sqrt{\beta(\nu+1)/(\nu(\alpha-1))}$ 是总预测标准差

### 3.3 模型、训练与数据集

**模型配置**：
- 嵌入维度：d=128
- GAT层数：3-6层
- 注意力头数：4-8头
- 模型大小：12.4MB
- 推理时间：2-7ms（典型任务图8节点、56边）

**训练过程**（三阶段）：
1. **特征学习**：标准多任务回归损失（MSE），AdamW优化器，学习率10⁻³，批次32-128，epochs 50-100
2. **证据训练**：证据损失微调，学习率10⁻⁴，梯度裁剪（范数1.0），epochs 20-50
3. **校准**：后处理校准，缩放预测标准差 $\sigma' = \tau \cdot \sigma$，使PICP>95%

**正则化**：图边dropout（p=0.1-0.2）、小特征噪声（σ=0.05）、设备增强（±5% DVFS表）

**数据集**：
- 三个ARM平台：Jetson TX2（6核，Denver+A57）、RUBIK Pi（8核Cortex-A72）、Jetson Orin NX（8核Cortex-A78AE）
- 42个基准测试：BOTS（12个任务并行应用）+ PolyBench（30个内核）
- 总样本：73,920个，60/20/20训练/验证/测试分割，按（基准、输入、核心掩码、DVFS索引）分层

**基线方法**：线性回归、随机森林、MLP、GCN、HGT、HGT+集成（5模型）、HGT+MC Dropout

### 3.5 技术细节

**数据收集流水线**（附录N.3）：
- OMPi编译OpenMP源码 → ALF-llvm发出LLVM IR和ALF文件 → SWEET生成CFG/调用图 → 后处理映射到OpenMP任务
- 运行时遥测：时间戳、DVFS索引、测量频率、性能计数器（周期、指令、缓存/分支未命中）、能量、热读数

**ALF→DAG流水线**（算法1）：
1. 解析ALF和WCTGW，提取任务和边
2. 折叠小链，保留来源映射
3. 为每个任务提取WCET/BCET、循环、字节、步长代理、分支熵、活变量
4. 添加优先/亲和属性，计算拓扑深度和到汇点距离
5. 添加有向边，标记关键边，计算跳距离
6. 构建资源节点，附加频率、掩码、利用率、热余量
7. 附加设备常数，嵌入为设备上下文
8. 添加任务↔资源和资源↔资源边

**RL集成细节**（附录D）：
- 智能体：Profiler代理（选择核心数和DVFS步长）、热代理（通过温度集群排序核心以避免热点）
- 重放缓冲区：真实（B）和合成（B'）分离
- 规划比率ζ∈{0,5,10,20}控制每真实步的合成轨迹数
- 安全门：仅当GraphPerf-RT证据90/95% PI低于时间和能量阈值时才接受合成样本

## 4. 意义与影响（Significance and Impact）

**解决的问题**：
1. 统一编码应用结构、硬件状态和调度决策的跨层交互
2. 提供校准的不确定性估计，实现风险感知调度
3. 实现快速（2-7ms）推理，支持实时调度
4. 减少RL探索中的过热风险

**贡献**：
- 首个将任务DAG拓扑、CFG代码语义和运行时上下文统一到异构图中的方法
- 证据回归框架提供校准的不确定性分解
- 与多智能体模型基RL集成，实现66%执行时间和82%能耗降低
- 可复现的评估框架

**对未来研究/应用的影响**：
- 为嵌入式AI代理提供安全、不确定性感知调度的基础设施
- 可扩展到能量预测、硬件计数器预测等多目标优化
- 支持Dyna式模型基规划，减少危险设备探索
- 跨平台泛化能力（三个ARM平台验证）

**局限与开放问题**（附录K）：
- 当前聚焦makespan预测，可扩展到能量和硬件计数器
- 需要进一步验证在更复杂异构架构上的泛化能力
- 证据回归的证据收缩问题通过非饱和正则化缓解，但仍有改进空间
- 模型基RL中单智能体性能不如无模型，需要更好的单智能体规划策略

## 5. 澄清与简化（Clarifications and Simplifications）

**类比：智能交通调度系统**

想象一个智能交通调度系统，需要为自动驾驶汽车分配路线和速度：

- **任务节点** = 每辆车的行程（起点、终点、预计时间）
- **资源节点** = 每条道路（当前拥堵程度、限速、温度）
- **内存节点** = 交通枢纽（容量、连接性）
- **任务-任务边** = 车辆之间的依赖关系（如会合点）
- **任务-资源边** = 车辆分配到特定道路
- **资源-资源边** = 道路之间的连接和共享（如交叉口）
- **资源-内存边** = 道路到交通枢纽的连接

传统方法要么忽略道路之间的交互（表格模型），要么只考虑静态地图（静态GNN）。GraphPerf-RT则同时考虑：
- 每辆车的行程细节（CFG特征）
- 每条道路的实时状态（DVFS、温度）
- 道路之间的交互（缓存争用）
- 交通枢纽的容量（内存层次）

**证据回归类比**：就像天气预报不仅预测温度，还给出置信区间（"明天25°C，有80%概率在23-27°C之间"）。GraphPerf-RT不仅预测执行时间，还告诉调度器这个预测有多可靠，从而避免在不确定的情况下做出危险决策。

**异构GAT类比**：就像交通调度员对不同类型的信息使用不同的注意力权重——对紧急车辆（关键任务）给予更多关注，对普通车辆（非关键任务）给予较少关注。同样，GraphPerf-RT对不同类型的节点和边使用不同的注意力机制。
