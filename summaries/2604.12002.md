## 0. 概览（Concise Summary）

- **主题（Topic）**：大语言模型（LLM）在可验证任务（如数学、代码）上的后训练（Post-training）方法。
- **问题（Problem）**：现有的后训练方法存在两难：强化学习（RLVR）使用二元奖励，信号稀疏、样本效率低；知识蒸馏方法需要外部强教师或高质量示范数据，成本高昂或难以获取。核心挑战是：如何在不依赖外部教师或高质量数据的情况下，将稀疏的二元奖励转化为密集的、可学习的信号。
- **方法（Method）**：提出**Self-Distillation Zero (SD-ZERO)** 方法。该方法训练一个单一模型扮演两个角色：**生成器（Generator）** 和**修订者（Reviser）**。首先通过**自我修订训练（SRT）** 阶段让模型学会根据自身回答和二元奖励进行修订；然后在**自我蒸馏（Self-Distillation）** 阶段，让修订者（作为教师）为生成器（作为学生）的在线回答提供逐token的密集监督信号。
- **创新（Innovation）**：核心创新在于**将二元奖励转化为密集的、逐token的自我监督**。具体体现在：1）不需要外部教师或高质量示范；2）通过“生成-修订-蒸馏”的循环，实现了**token级别的自我定位（token-level self-localization）** 和**迭代式的自我进化（iterative self-evolution）**。
- **意义（Significance）**：SD-ZERO 提供了一种样本高效、无需昂贵外部资源的后训练范式，在数学和代码推理任务上显著优于现有强基线方法（如GRPO、RFT、SDFT），为LLM的自我提升开辟了新路径。
- **一句话总结**：本文提出 SD-ZERO，通过让模型自我修订并将修订能力蒸馏回生成过程，成功将稀疏的二元奖励转化为密集的逐token监督，实现了高效、无需外部教师的自我提升。

## 1. 研究动机（Motivation）

当前LLM在可验证任务（如数学、代码）上的后训练方法主要分为两类，但各有局限：

1.  **强化学习（RLVR）**：如GRPO。这类方法仅需一个二元奖励（回答正确/错误），通用性强。然而，**二元奖励信号极其稀疏**，它只告诉模型最终结果的对错，却不提供任何关于中间推理步骤好坏的信息。模型需要通过大量采样和对比来“猜测”正确的推理路径，导致训练样本效率低下，成本高昂。

2.  **知识蒸馏（Distillation）**：如SFT、SDFT。这类方法通过教师模型提供逐token的密集监督信号，样本效率高。但问题在于，它们通常需要一个**外部的、更强的教师模型**（如DeepSeek-R1）或**高质量的人工示范数据**。收集这些数据的成本可能非常高昂，甚至在某些领域无法获得。

**核心矛盾**：RLVR 通用但低效，蒸馏高效但依赖昂贵的外部资源。

**本文试图回答的核心问题**：模型能否**利用自身**的初始尝试（即使是错误的）和对应的稀疏奖励，来为自己提供改进后的密集监督信号？即，能否实现一种**自给自足**的、高效的自我提升方法？

## 2. 创新点（Innovation）

SD-ZERO 的关键创新在于提出了一种全新的自我蒸馏范式，其核心是将二元奖励转化为密集的逐token自我监督。具体创新点如下：

1.  **双角色单一模型架构**：训练一个模型同时扮演**生成器**和**修订者**。生成器负责产生初始回答，修订者则根据生成器的回答和其二元奖励，对其进行修订（纠正错误或改写正确回答）。这避免了对外部教师的需求。

2.  **两阶段训练流程**：
    - **阶段一：自我修订训练（SRT）**：通过让模型自我修订其（主要是错误的）回答，收集成功的修订轨迹进行微调。这一步**激活了模型的自我修订能力**，是后续蒸馏的基础。
    - **阶段二：在线自我蒸馏（Self-Distillation）**：将阶段一训练好的模型作为固定的教师（修订者），对当前模型（学生/生成器）的在线回答进行逐token的KL散度蒸馏。这一步**将修订者的“思考过程”内化到生成器中**，使其能直接生成更优、更简洁的回答。

3.  **关键特性**：
    - **Token级自我定位**：实验发现，修订者虽然只接收二元奖励，但其提供的逐token监督信号能高度集中在回答中导致错误的少数关键token上，实现精准的错误定位和修正引导。
    - **迭代式自我进化**：由于蒸馏过程也提升了模型的修订能力，更新后的模型可以作为更强的教师，通过“教师同步”进行多轮迭代，实现持续的性能提升。

**与现有方案的不同**：
- 相比RLVR，SD-ZERO提供了密集的逐token信号，样本效率更高。
- 相比SDFT等自我蒸馏方法，SD-ZERO**不需要高质量示范**。SDFT需要模型在正确回答上作为教师，而SD-ZERO的教师（修订者）可以**以错误的回答作为输入**，并输出修正后的版本，从而从失败中学习。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

SD-ZERO 的整体架构如图2所示，包含两个阶段：

**阶段一：自我修订训练（SRT）**
- **目的**：训练模型具备根据自身回答和奖励进行修订的能力。
- **流程**：
    1.  **初始生成**：对于问题 $x$，从基础模型 $\pi_\theta$ 中采样一个初始回答 $y_{init}$。
    2.  **验证与提示**：使用二元验证器检查 $y_{init}$ 的正确性，得到奖励 $r \in \{0, 1\}$。根据 $r$ 构建修订提示 $P_r$：
        - 如果 $r=1$（正确）：提示“Let me rephrase the above solution.”（改写）。
        - 如果 $r=0$（错误）：提示“Wait, this response is not correct, let me start over.”（重新开始）。
    3.  **自我修订**：模型根据输入 $x$、初始回答 $y_{init}$ 和提示 $P_r$，生成修订后的回答 $y_{revised}$。
    4.  **数据过滤**：只保留那些修订后回答 $y_{revised}$ 是正确的轨迹 $(x, y_{init}, P_r, y_{revised})$，构成数据集 $D_{REVISION}$。
- **损失函数**：$L_{SRT} = L_{revision} + L_{generation}$，同时训练模型的修订能力和生成能力。

**阶段二：在线自我蒸馏（Self-Distillation）**
- **目的**：将阶段一学到的修订行为内化到生成器中，使其能直接生成更优、更简洁的回答。
- **流程**：
    1.  **初始化**：学生模型 $\theta$ 初始化为阶段一的 SRT 模型参数 $\theta_{SRT}$。教师模型固定为 $\theta_{SRT}$。
    2.  **在线生成**：对于问题 $x$，学生模型 $\pi_\theta$ 生成一个在线回答 $y$。
    3.  **教师监督**：教师模型（修订者）$\pi_{\theta_{SRT}}$ 根据输入 $x$、学生的回答 $y$ 及其奖励 $r$，在每一个token位置 $t$ 产生一个目标分布 $\pi_{\theta_{SRT}}(\cdot | x, y, P_r, y_{<t})$。
    4.  **蒸馏损失**：学生模型被训练去最小化其自身分布与教师分布之间的KL散度。

### 3.2 关键算法与数学推导

**阶段一：自我修订训练（SRT）损失函数**

损失函数由两部分组成：

1.  **修订损失（Revision Loss）**：训练模型在给定上下文（问题、初始回答、奖励提示）时，生成修订后的回答。
    $$L_{revision}(\theta) = \mathbb{E}_{(x, y_{init}, P_r, y_{revised}) \sim D_{REVISION}} \left[ -\sum_{t=1}^{|y'|} \log \pi_\theta(y'_t | x, y_{init}, P_r, y'_{<t}) \right]$$
    其中 $y' = y_{revised}$。

2.  **生成损失（Generation Loss）**：保留模型仅根据问题 $x$ 直接生成正确回答的能力。这里将整个修订轨迹 $[y_{init}, P_r, y_{revised}]$ 视为一个完整的正确回答。
    $$L_{generation}(\theta) = \mathbb{E}_{(x, y_{init}, P_r, y_{revised}) \sim D_{REVISION}} \left[ -\sum_{t=1}^{|y'|} \log \pi_\theta(y'_t | x, y'_{<t}) \right]$$
    其中 $y' = [y_{init}, P_r, y_{revised}]$。

总损失为：
$$L_{SRT}(\theta) = L_{revision}(\theta) + L_{generation}(\theta)$$

**阶段二：在线自我蒸馏损失函数**

该阶段的核心是KL散度最小化。学生模型 $\pi_\theta$ 被训练去匹配教师模型 $\pi_{\theta_{SRT}}$ 的分布。

$$L_{Self-Distillation}(\theta) = \mathbb{E}_{(x,a) \sim D} \mathbb{E}_{y \sim \pi_\theta(\cdot|x)} \left[ \sum_{t=1}^{|y|} D_{KL} \left( \pi_\theta(\cdot | x, y_{<t}) \parallel \pi_{\theta_{SRT}}(\cdot | x, y, P_r, y_{<t}) \right) \right]$$

**Token KL 奖励（Token KL Reward）**：为了分析教师信号，论文定义了每个token $t$ 的KL奖励，近似为生成器和修订者在该token上的log概率差：
$$\text{Token KL Reward at } t = \log \pi_\theta(y_t | x, y_{<t}) - \log \pi_{\theta_{SRT}}(y_t | x, y, P_r, y_{<t})$$

### 3.3 模型、训练与数据集

- **模型**：
    - Qwen3-4B-Instruct
    - Olmo-3-7B-Instruct
- **训练数据**：
    - **数学**：从 OpenR1-Math 数据集中选取 15K 道竞赛和奥林匹克级别的问题。
    - **代码**：从 Codeforces 数据集中选取 7.5K C++ 和 7.5K Python 样本。
- **训练设置**：
    - 采样温度：0.7
    - 训练时token限制：16K
    - 评估时token限制：32K
    - SRT阶段：使用6K个自我修订轨迹进行训练。
    - Self-Distillation阶段：使用剩余的数据进行训练，每个问题只需采样一个回答。

### 3.4 实验设置与结果

- **评估基准**：涵盖数学（AIME24/25, HMMT25, MATH, AMOBench, OpenR1-Math）和代码（Codeforces, LiveCodeBench）共8个基准。
- **基线方法**：
    - **SFT**：在DeepSeek-R1生成的高质量示范上微调。
    - **RFT**：拒绝采样微调，仅使用模型自身生成的正確回答。
    - **GRPO**：基于二元奖励的强化学习方法（使用DAPO变体）。
    - **SDFT**：在线自我蒸馏，但需要高质量示范作为教师输入。
- **主要结果（表1）**：
    - **SRT（阶段一）**：仅用6K数据就显著超越所有基线（SFT, RFT, GRPO, SDFT），平均提升7.8%（Qwen）和9.2%（Olmo）。
    - **SD-ZERO（完整）**：在SRT基础上进一步提升，总提升达到10.5%（Qwen）和10.4%（Olmo），在所有基准上取得最佳平均成绩。
    - **效率**：SD-ZERO在推理时生成的token数量远少于SRT和GRPO，约为SRT的一半（图3），实现了性能与效率的双重提升。

### 3.5 技术细节

- **数据划分**：将15K训练数据分为两部分，6K用于SRT阶段（需要多次采样以获取成功修订），9K用于Self-Distillation阶段（每个问题只需一次采样）。
- **教师同步**：在Self-Distillation阶段，教师模型默认固定为SRT模型。但实验发现，经过一轮蒸馏后，学生模型的修订能力也提升了。此时将教师同步为更新后的学生模型，可以开启新一轮的迭代进化，获得额外3%的性能提升（图5）。
- **Token级自我定位分析**：通过分析Token KL奖励的分布（图4），发现对于错误回答，KL散度高度集中在少数几个关键token上，表明修订者能精准定位错误。对于正确回答，KL分布则更均匀，主要起到保留原有回答的作用。
- **推理行为演化**：通过追踪“Wait”、“let me start over”等自我修订关键词的频率（图6），发现SRT阶段模型表现出显式的自我修订行为（回答变长），而Self-Distillation阶段模型将这些行为内化，回答变短但准确率更高，表现出更主动、更高效的推理。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了后训练中“稀疏奖励 vs. 昂贵监督”的两难问题，提出了一种**自给自足、样本高效**的新范式。
    - 证明了**从自身错误中学习**的有效性，通过“修订-蒸馏”循环，将失败经验转化为可学习的密集信号。
    - 揭示了模型在自我提升过程中的**两种关键能力**：token级自我定位和迭代式自我进化。

- **对未来研究/应用的影响**：
    - **降低门槛**：SD-ZERO 消除了对外部强教师或高质量数据的依赖，使得更多研究者和团队能够进行高效的模型后训练。
    - **新范式**：为“自我改进”类方法提供了新思路，即通过“行为内化”而非简单的“数据增强”来提升模型能力。
    - **应用潜力**：在数学、代码等可验证领域有直接应用价值，其思想有望扩展到其他领域。

- **局限与开放问题**：
    - **对“思考模型”的挑战**：当前方法主要针对生成简洁回答的指令模型。对于产生长链式思考的“思考模型”，其回答中可能包含大量非错误的探索性内容，使得区分“错误”与“探索”变得困难，难以直接应用。
    - **对非可验证领域的扩展**：方法依赖于二元奖励（对/错）。如何将其扩展到没有明确正确答案的领域（如文本生成、创意写作）是一个开放问题。论文提出可以利用元认知信号（如一致性、自我修正）来定义奖励，作为未来方向。

## 5. 澄清与简化（Clarifications and Simplifications）

- **核心思想类比**：可以把模型想象成一个学生。
    - **RLVR**：老师只给学生的试卷打“对”或“错”，不给任何批注。学生需要自己做大量试卷，自己琢磨为什么错，效率很低。
    - **SDFT**：老师直接给出一份标准答案，让学生照着抄。这很高效，但标准答案很贵，不是随时都有。
    - **SD-ZERO**：这个学生很聪明。他先自己做一遍题（**生成器**），然后自己扮演“小老师”（**修订者**），对着自己的答案和红笔批注（二元奖励）进行修改。最后，他把自己当小老师时学到的“批改思路”内化，下次做题时就能直接避免错误，写出更漂亮的答案。整个过程完全靠自己，不需要外部老师。

- **“Token级自我定位”通俗解释**：想象一个学生写数学证明，中间一步用错了公式，导致最终结果错误。传统的二元奖励只会告诉他“你错了”。而SD-ZERO的“修订者”会像一位细心的老师，在学生的证明上圈出“**这里**用错了公式，应该用另一个”，并给出正确的后续步骤。这个“圈出错误位置”的行为，就是token级自我定位。它把“整题错误”这个稀疏信号，转化为了“第X步的Y公式错误”这样的密集、精确的指导。

- **“迭代式自我进化”通俗解释**：学生通过“自己做题 -> 自己批改 -> 内化批改思路”这个过程，解题能力变强了。同时，他“批改”的能力也变强了。现在，他可以用更强的“批改能力”去指导下一轮的“做题”，形成一个“越改越强，越强越能改”的正向循环。这就是迭代式自我进化。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - **自我修正（Self-Correction）**：相关工作如Self-Refine、Reflexion等，通常需要外部反馈或多次迭代。SD-ZERO的不同在于，它**训练**模型进行修正，并将修正能力蒸馏回生成过程，而非在推理时进行昂贵的多轮迭代。
    - **自我训练（Self-Training）**：如STaR、RFT，它们通过过滤自身正确回答来训练。SD-ZERO的创新在于**利用了错误的回答**，通过“修订”这一桥梁，从失败中学习。
    - **自我蒸馏（Self-Distillation）**：如SDFT、OPSD，它们需要高质量示范。SD-ZERO的关键突破是**不需要高质量示范**，教师（修订者）可以处理错误输入。
