## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人操作领域的视觉-语言-动作（VLA）模型。
- **问题（Problem）**：预训练的视觉-语言模型（VLM）的语义空间与机器人控制所需的3D物理空间之间存在结构性不匹配，导致直接的感知-动作映射学习困难且不鲁棒。
- **方法（Method）**：提出**AffordanceVLA**框架，通过引入结构化的**可供性（Affordance）预测**作为任务导向的中间表征，来弥合感知与动作之间的鸿沟。该框架包含三个核心模块：`Which2Act`（目标定位）、`Where2Act`（2D交互区域定位）和 `How2Act`（3D几何推理），并采用混合Transformer（MoT）架构和三阶段渐进式训练策略。
- **创新（Innovation）**：
    1.  首次将**结构化、多粒度的可供性预测**（从粗到细，从2D到3D）作为VLA模型的**内部中间监督信号**，而非外部输入。
    2.  设计了**混合Transformer（MoT）架构**，包含理解、可供性生成和动作三个专用专家，并通过单向注意力机制实现信息的有序流动。
    3.  提出**三阶段渐进式训练策略**和**自动化数据增强管线**，解决了可供性标注稀缺的问题，并实现了从通用语义理解到特定任务控制的平滑过渡。
- **意义（Significance）**：该方法显著提升了VLA模型在模拟和真实环境中的操作成功率、泛化能力和鲁棒性，尤其是在指令跟随和长时序任务上表现出色。它为如何更有效地利用VLM的世界知识进行机器人控制提供了新的范式，证明了结构化中间表征的重要性。
- **一句话总结**：AffordanceVLA通过将“哪个物体、在哪儿、怎么操作”的结构化可供性知识内化到VLA模型中，构建了更精确、鲁棒的感知-动作桥梁，从而在多种机器人操作任务上取得了领先性能。

## 1. 研究动机（Motivation）

- **核心问题**：当前的VLA模型通常尝试将视觉和语言指令直接端到端地映射到机器人动作。然而，VLM的预训练核心是**在语义空间中对齐视觉和语言**，而机器人动作本质上是**3D物理空间中的表征**。这种**结构性不匹配**导致学习一个直接的、鲁棒的映射非常困难。
- **现有方法的不足**：
    - **直接端到端映射**：忽略了任务相关的场景结构，鲁棒性差。
    - **视频预测/视觉预见**：作为中间表征，包含大量冗余信息，且推理过程耗时。
    - **其他视觉表征（如深度图、光流）**：探索和利用仍不充分。
    - **盲目扩大数据**：在不创新模型范式的情况下，无法最大化数据的内在价值，也无法解决根本性的空间鸿沟。
- **为什么重要**：找到一个合适的、任务导向的中间表征来桥接感知和动作，是构建鲁棒、泛化能力强的VLA模型的关键。这直接关系到机器人能否在复杂、非结构化的真实世界中可靠地执行指令。

## 2. 创新点（Innovation）

1.  **结构化可供性作为内部中间表征**：与以往将可供性作为外部输入或稀疏接触点不同，AffordanceVLA将**结构化可供性预测（Which2Act, Where2Act, How2Act）内化到VLA模型内部**，作为联合优化的中间监督信号。这种表征同时具备**空间基础性、语义条件性和动作耦合性**，自然地桥接了视觉、语言和动作。
2.  **混合Transformer（MoT）架构**：设计了三个专用专家：
    - **理解专家（Understanding Expert）**：对齐视觉与语言意图。
    - **可供性生成专家（Affordance Generation Expert）**：预测结构化可供性。
    - **动作专家（Action Expert）**：基于前两者的输出生成动作。
    这种解耦设计通过**单向的“理解-可供性-动作”（UAA）渐进式注意力机制**，防止了信息泄露和表征坍塌，实现了从感知到控制的平滑过渡。
3.  **三阶段渐进式训练策略与数据增强管线**：
    - **阶段I（通用可供性预训练）**：在VQA和交互感知数据上训练可供性生成专家，冻结其他部分。
    - **阶段II（可供性增强的机器人数据协同训练）**：在大规模合成机器人数据上联合训练所有专家，并利用**自动化数据增强管线**（利用LLM和VLM生成可供性标注）解决标注稀缺问题。
    - **阶段III（目标任务后训练）**：在目标数据集上微调，优先优化控制策略。
    这种策略系统地积累了通用世界知识，防止了过拟合，并保护了VLM的指令跟随能力不被动作损失侵蚀。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

AffordanceVLA的核心是一个**混合Transformer（MoT）** 架构，包含三个专家模块，通过UAA注意力机制协同工作。整体流程如图2所示。

1.  **理解专家（$M_{und}$）**：接收当前视觉观测 $O_t$ 和语言指令 $l$，输出指令感知的融合语义表征 $h_{und}^t$。它负责将语言概念精确地锚定到视觉空间。
2.  **可供性生成专家（$M_{gen}$）**：接收 $h_{und}^t$，预测结构化的可供性表征 $\hat{A}_t$。这个表征包含三个并行子模块：
    - `Which2Act`：预测目标物体的视觉潜码（visual latent），实现以物体为中心的定位。
    - `Where2Act`：预测2D可供性热力图，定位精确的交互区域。
    - `How2Act`：预测目标的3D形状和空间布局（10-DoF），提供几何推理。
3.  **动作专家（$M_{act}$）**：接收 $h_{und}^t$、$\hat{A}_t$ 和本体感觉 $s_t$，通过一个生成过程（如扩散模型）解码出平滑的动作序列 $\hat{a}_{t:t+k}$。
4.  **UAA注意力机制**：在专家内部使用双向注意力进行上下文融合；在专家之间，实施严格的**因果注意力**：可供性生成专家只能查询理解专家的输出，动作专家可以查询前两者的输出。这确保了信息的单向流动，防止动作信息污染可供性预测。

### 3.2 关键算法与数学推导

可供性生成专家通过三个子模块预测结构化知识，其损失函数如下：

1.  **Which2Act 损失**：通过MSE损失优化目标视觉潜码 $\hat{z}$ 的重建。
    $$L_{which} = \frac{1}{C \cdot H \cdot W} \sum_{c,h,w} \|\hat{z}_{c,h,w} - z_{q,c,h,w}\|^2$$

2.  **Where2Act 损失**：通过像素级的二元交叉熵（BCE）损失优化2D可供性热力图 $\hat{y}$。
    $$L_{where} = -\frac{1}{H_t W_t} \sum_{i=1}^{H_t W_t} \left[ M_i \log \sigma(\hat{y}_i) + (1-M_i) \log (1-\sigma(\hat{y}_i)) \right]$$
    其中 $\sigma(\cdot)$ 是sigmoid函数，$M$ 是真实掩码。

3.  **How2Act 损失**：包含两部分：
    - **形状生成**：通过扩散模型的噪声预测损失优化3D形状潜码。
        $$L_{shape} = \mathbb{E}_{t \sim U(0,T), \epsilon \sim \mathcal{N}(0,I)} \left[ \|\epsilon - \hat{\epsilon}_\theta(x_t, t, \bar{h}_{shape})\|^2 \right]$$
    - **布局回归**：通过Smooth-L1损失优化10-DoF空间布局向量 $\hat{y}_{layout}$。
        $$L_{layout} = \frac{1}{10} \sum_{j=1}^{10} \text{Smooth}_{L1}(\hat{y}_{layout}^{(j)}, y_{layout}^{(j)})$$

4.  **三阶段训练损失**：
    - **阶段I**：$L_{Stage1} = \lambda_{which} L_{which} + \lambda_{where} L_{where} + \lambda_{shape} L_{shape} + \lambda_{layout} L_{layout}$
    - **阶段II & III**：$L_{Stage2/3} = \lambda_{act} L_{act} + \lambda_{afd} L_{afd}$，其中 $L_{afd}$ 是阶段I中所有可供性损失的总和。在阶段II，$\lambda_{act}=1.0, \lambda_{afd}=0.5$；在阶段III，$\lambda_{afd}=0.15$。

### 3.3 模型、训练与数据集

- **模型架构**：基于MoT架构，视觉编码器使用SigLIP，语言模型使用PaliGemma，动作解码器采用基于扩散的架构（类似π0）。
- **训练策略**：三阶段渐进式训练。
    - **阶段I**：在**通用可供性数据**（AGD20K, RefSpatial, PRISM）上训练，冻结视觉编码器、理解专家和动作专家，仅优化可供性生成专家。
    - **阶段II**：在**大规模合成机器人数据**（InternData-A1）上协同训练，解冻所有模块。为解决可供性标注稀缺，开发了**自动化数据增强管线**：从动作序列中提取关键帧，利用LLM（Claude Opus 4.5）分解指令，VLM（Qwen3-VL）生成检测类别和空间查询，再结合RexOmni、SAM和SAM-3D生成超过10万个可供性标注。
    - **阶段III**：在**目标任务数据**（LIBERO, CALVIN）上后训练，进一步微调以适应特定环境。对于真实世界部署，还会在DROID数据集上进行预适应微调。
- **数据集**：模拟环境使用LIBERO和CALVIN ABC→D基准。真实世界任务为自行设计的多种操作任务。

### 3.4 实验设置与结果

- **模拟实验**：
    - **LIBERO基准**：如表1所示，AffordanceVLA（完整版）平均成功率达到**95.8%**，优于所有对比方法（包括Pi0的94.4%和F1-VLA的95.7%）。即使跳过阶段II，其平均成功率也达到86.2%，证明了MoT架构的有效性。
    - **CALVIN ABC→D基准**：如表2所示，AffordanceVLA（完整版）平均任务链长度为**4.33**，5个连续任务成功率为**75.9%**，与Seer-Large（4.28）等最强模型竞争。从w/o stage II的3.81到完整版的4.33，证明了阶段II的必要性。
- **消融实验**（表3）：
    - **架构与训练策略**：`No-Afd`（无可供性模块）和 `Frozen-Afd`（冻结可供性模块）的性能大幅下降，证明了MoT架构和联合优化的必要性。
    - **可供性表征**：移除`Which2Act`、`Where2Act`或`How2Act`中的任何一个都会导致性能下降，证明了三者协同工作的价值。其中`How2Act`在复杂真实世界任务中作用更显著。
    - **注意力机制**：将双向注意力改为因果注意力的 `Block-wise Tokens` 变体性能显著下降，证明了可供性模块内部信息共享和联合精炼的重要性。
- **数据效率分析**（图3）：AffordanceVLA在仅使用40%微调数据时，性能就超过了使用100%数据的Pi0，展示了其卓越的数据利用效率。
- **真实世界实验**（表5）：
    - **基础任务**：平均成功率88.3%，优于Pi0的70.8%。
    - **复杂任务**：在需要精确指令跟随的`Drawer`和`Toaster`任务中，AffordanceVLA（82.9%）远优于Pi0（44.8%），证明了可供性表征保护了VLM的指令跟随能力。
    - **长时序任务**：在`Pick all the rubbish`任务中，AffordanceVLA的连续执行成功率（46.7%）远高于Pi0（6.7%），且无效动作更少，展示了其涌现出的长时序规划能力。

### 3.5 技术细节

- **Which2Act**：使用冻结的Flux VAE编码器提取目标物体的连续视觉潜码，通过MSE损失让模型学会重建这个潜码，从而迫使模型关注目标物体并过滤背景。
- **Where2Act**：使用轻量级Transformer解码器，将空间位置嵌入作为查询，通过交叉注意力从1D token中提取2D空间信息，再映射为热力图。
- **How2Act**：形状分支使用条件扩散Transformer生成3D体素潜码；布局分支使用MLP直接回归10-DoF向量（旋转、缩放、平移）。
- **自动化数据增强管线**：这是一个关键工程贡献。它利用现成的LLM和VLM模型，从已有的机器人轨迹数据中自动、大规模地生成可供性标注，解决了数据瓶颈问题。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了VLM语义空间与机器人控制物理空间之间的**结构性不匹配**问题。
    - 提出了一种新的范式，即**将结构化的、任务导向的中间表征（可供性）内化到VLA模型中**，而不是作为外部输入或冗余的视觉预测。
    - 通过MoT架构和三阶段训练，实现了**从通用语义理解到特定任务控制的平滑、高效迁移**。
    - 通过自动化数据增强管线，**解决了可供性标注稀缺的瓶颈**，为大规模训练提供了可能。
- **对未来研究/应用的影响**：
    - **研究方向**：为VLA模型的设计提供了新思路，即**中间表征的设计比盲目扩大数据更重要**。未来的工作可以探索其他形式的结构化中间表征。
    - **应用价值**：显著提升了机器人在复杂、非结构化环境中的操作能力，尤其是在**指令跟随、鲁棒性和数据效率**方面，为家用服务、工业自动化等领域的机器人应用铺平了道路。
- **局限与开放问题**：
    - **长时序任务**：在LIBERO-Long任务上，性能提升幅度相对较小，表明模型在**显式长时记忆**方面存在局限。
    - **未来工作**：作者明确提到未来将探索**显式时序建模**，以及扩展到**双臂操作和可变形物体操作**。

## 5. 澄清与简化（Clarifications and Simplifications）

- **可供性（Affordance）的通俗理解**：可以理解为“物体提供的操作可能性”。比如，一个杯子的把手“可供”你抓握，一个微波炉的门“可供”你打开。AffordanceVLA就是让机器人学会识别这些“可能性”。
- **结构化可供性预测的类比**：想象你要去一个陌生的厨房做一道菜。你需要知道：
    1.  **Which2Act**：找出哪个是目标物体（比如“微波炉”）。
    2.  **Where2Act**：确定要操作它的哪个部位（比如“门把手”）。
    3.  **How2Act**：理解它的3D结构（比如门是向外拉的，需要多大的力）。
    AffordanceVLA就是让机器人在执行动作前，先“想”清楚这三个问题。
- **MoT架构的类比**：就像一个高效的团队。
    - **理解专家**是“翻译官”，把人类的语言指令翻译成机器人能理解的视觉特征。
    - **可供性生成专家**是“规划师”，根据翻译结果，画出操作蓝图（哪个物体、哪个点、怎么动）。
    - **动作专家**是“执行者”，根据蓝图，精确地控制手臂和夹爪。
    - **UAA注意力**保证了信息流是单向的：规划师只能听翻译官的，执行者要同时听翻译官和规划师的，但规划师和执行者不能反过来影响翻译官。
- **为什么可供性能保护指令跟随能力**：可以这样理解：在传统的VLA中，动作损失直接“冲刷”VLM的权重，可能会“冲掉”它理解语言的能力。而可供性损失更像一个“锚”，它要求模型必须正确理解语言（如“关闭微波炉” vs “拿起杯子”）才能生成正确的可供性热力图（门把手 vs 杯身）。这个“锚”迫使模型保留并强化其语言理解能力，从而在后续的动作学习中不会“跑偏”。
