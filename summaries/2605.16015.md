## 0. 概览（Concise Summary）

- **主题（Topic）**：基于深度强化学习（DRL）的四旋翼飞行器自适应外环控制。
- **问题（Problem）**：传统的域随机化（DR）方法训练出的鲁棒策略过于保守，在动态扰动下性能不佳，且难以从仿真高效迁移到真实环境（sim-to-real）。
- **方法（Method）**：提出一种三阶段自适应控制架构：1）训练一个能访问真实扰动的“神谕（Oracle）”策略作为性能上限；2）设计一个基于GRU的残差动力学预测器（RDP），仅利用历史状态和电机PWM信号在线估计6维外部扰动；3）引入一个数据高效的线性校准桥和在线推力修正机制，以极少的真实飞行数据弥合仿真与现实的差距。
- **创新（Innovation）**：核心创新在于将RMA（快速运动适应）思想从足式机器人成功迁移到四旋翼飞行器，并显式地估计物理扰动（力和力矩），而非抽象潜变量。同时，提出了一个极其轻量、高效的sim-to-real校准方法。
- **意义（Significance）**：该方法在真实Crazyflie微型四旋翼上，面对质量变化、非对称负载和动态悬挂负载等严重不确定性，显著优于基线方法，实现了精确的轨迹跟踪。这为在复杂、动态环境中部署高可靠性自主无人机提供了有效途径。
- **一句话总结**：本文提出了一种基于强化学习的自适应外环控制架构，通过一个显式估计物理扰动的残差动力学预测器和一个数据高效的线性校准桥，实现了四旋翼在严重动态不确定性下的高性能飞行。

## 1. 研究动机（Motivation）

- **解决的问题**：传统四旋翼控制器（如PID、LQR、MPC）在标称条件下表现良好，但严重依赖精确的数学模型和线性化近似。当遇到未建模的动态（如剧烈气动扰动、执行器磨损、负载变化）时，性能会急剧下降。虽然深度强化学习（DRL）能学习非线性控制策略，但存在严重的“仿真到现实（sim-to-real）”鸿沟。
- **为什么重要**：无人机在基础设施巡检、搜救、复杂环境导航等现实应用中，需要极高的敏捷性、精度和可靠性。能够在动态和不确定环境下保持精确轨迹跟踪是系统安全性和有效性的关键。
- **挑战在哪里**：
    1.  **Sim-to-Real鸿沟**：仿真环境与真实物理世界存在不可避免的差异，导致在仿真中训练的策略直接部署到真实硬件上会失败。
    2.  **域随机化（DR）的局限性**：DR是解决sim-to-real的主流方法，但它迫使策略学习一个在所有随机化环境中表现“最不坏”的保守策略。这种策略在标称条件下牺牲了敏捷性和最优性能，且对动态扰动反应迟缓。
    3.  **动态适应**：核心挑战在于如何让一个基于学习的控制器能够动态地感知并适应其当前所处的具体环境扰动，而不是依赖一个静态的、保守的策略。

## 2. 创新点（Innovation）

本文的关键创新点在于提出了一套完整的、从训练到部署的自适应控制框架，具体包括：

1.  **自适应RL控制器（Oracle Baseline）**：首次训练一个能够访问“神谕”（即真实环境扰动）的RL策略。这为后续的估计器性能设定了一个理论上限，并证明了“如果知道扰动，就能做出最优反应”这一前提。
2.  **残差动力学预测器（RDP）**：核心创新。设计了一个基于GRU的循环神经网络，作为隐式状态估计器。它**不依赖昂贵的力/力矩传感器**，仅通过观测车辆状态和**电机PWM信号**的历史序列，就能在线、实时地预测出6维外部扰动（3个力+3个力矩）。与现有工作（如RMA）使用抽象潜变量不同，RDP显式地估计物理量，更具可解释性。
3.  **数据高效的Sim-to-Real校准桥**：另一个关键创新。作者发现，仿真和真实世界的扰动预测差异主要是一个**仿射变换**（近似线性）。因此，他们提出一个计算量极小的线性回归校准层，仅需**几秒钟**的真实飞行数据（3个简短样本）就能拟合出这个变换，有效消除稳态偏移。这避免了昂贵的、大规模的真实数据重训练。
4.  **在线推力修正机制**：针对电池电压下降等导致的残余推力不匹配问题，引入了一个简单的积分器，根据高度跟踪误差实时累积推力偏置，进一步提升了高度控制的精度。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整个系统采用**级联控制架构**（图4）：
- **内环**：使用四旋翼自带的、高带宽的PID速率控制器，负责快速稳定姿态。
- **外环**：由本文提出的RL策略构成，作为外环控制器，输出**期望的机体角速率**和**总推力**。这些指令被发送给内环PID控制器去跟踪。

核心方法分为三个阶段：
1.  **训练Oracle策略**：在仿真环境中，给RL策略的观测空间额外增加6维的真实扰动向量（$d_t$）。策略学习在“全知”状态下如何最优地响应各种扰动。
2.  **训练残差动力学预测器（RDP）**：使用Oracle策略在随机扰动下生成的约60,000个飞行片段作为数据集。RDP的输入是历史状态和动作序列，输出是预测的6维扰动$\hat{d}_t$。这是一个序列到向量的回归任务。
3.  **Sim-to-Real校准与部署**：将训练好的RDP部署到真实硬件时，其预测存在稳态偏移。通过采集少量真实飞行数据，拟合一个线性回归模型来校准RDP的输出。同时，在线推力修正机制持续补偿推力误差。

### 3.2 关键算法与数学推导

1.  **RL问题形式化**：
    将控制问题建模为马尔可夫决策过程（MDP），目标是最大化期望累积折扣奖励：
    $$J(\theta) = \mathbb{E}_{\pi_\theta} \left[ \sum_{t=0}^{T} \gamma^t r_t \right]$$

2.  **奖励函数**：
    奖励函数旨在鼓励悬停并最小化控制能量：
    $$r_t = c_p \|p_t\|^2 + c_\psi \psi_t^2 + c_v \|v_t\|^2 + c_\omega \|\omega_t\|^2 + c_T (\Delta T_t)^2 + c_{\Delta \omega} \|\Delta \omega_{cmd,t}\|^2 + c_s + c_d \cdot \mathbb{1}_{crash}$$
    其中，各项分别惩罚位置误差、偏航角、线速度、角速度、推力变化率、角速率变化率，并给予存活奖励和碰撞惩罚。

3.  **残差动力学预测器（RDP）**：
    RDP是一个GRU网络，输入特征向量$x_t \in \mathbb{R}^{26}$，包含18维运动学状态、4维上一时刻外环动作、4维电机PWM信号。它处理一个长度为$H=64$的历史窗口，通过两层GRU（隐藏维度64）和一个线性输出头，预测6维扰动$\hat{d}_t$。

4.  **在线推力修正**：
    通过一个积分器累积推力偏置$F_{bias}$，以修正稳态高度误差：
    $$F_{bias} \leftarrow F_{bias} + \eta (z_{des} - z)$$
    其中$\eta$是自适应积分增益，$z_{des}$和$z$分别是期望和当前高度。

### 3.3 模型、训练与数据集

- **模型**：
    - **RL策略网络**：使用PPO算法训练，策略和价值网络均为两层RNN（隐藏层64），处理32步的历史观测窗口。
    - **RDP网络**：两层GRU（隐藏层64），后接线性输出层。
- **训练**：
    - **仿真环境**：基于NVIDIA Isaac Lab，并行运行8192个环境。物理引擎以500Hz运行，RL策略以50Hz运行。
    - **RL训练**：PPO算法，学习率0.001，折扣因子$\gamma=0.99$，GAE参数$\lambda=0.95$，训练约100,000个算法时间步。
    - **RDP训练**：使用Oracle策略在随机扰动下生成的约60,000个飞行片段作为数据集。
- **数据集**：
    - **训练数据**：仿真生成的飞行数据，包含状态、动作、电机PWM和对应的真实扰动。
    - **校准数据**：真实硬件上采集的**极少量**（3个简短样本，几秒钟）飞行数据。

### 3.4 实验设置与结果

- **实验平台**：Bitcraze Crazyflie 2.X微型四旋翼，完全依赖机载IMU和Flow Deck进行状态估计，无外部动捕系统。
- **对比基线**：
    1.  **Base Controller**：在理想无扰动环境下训练。
    2.  **Robust Controller**：使用域随机化（DR）训练。
    3.  **Adaptive + Oracle**：使用真实扰动的理论性能上限。
    4.  **Adaptive + RDP**：本文提出的完整系统。
- **评估指标**：位置跟踪的**均方根误差（RMSE）**。
- **实验场景与结果**：
    1.  **中心负载变化**（表1）：Adaptive+RDP在所有负载下（0%-25%）均表现最佳，RMSE稳定在0.024-0.030m。Robust控制器在标称条件下就出现严重振荡（RMSE=0.111m），被认为不安全而停止测试。Base控制器性能随负载增加而显著下降。
    2.  **非对称负载**（表2）：Adaptive+RDP和Oracle性能接近，远优于Base控制器。RDP成功估计了由偏移重心引起的滚转和俯仰力矩（图7、8）。
    3.  **动态悬挂负载**（表3，图9）：在跟踪“8”字轨迹时，Adaptive+RDP能稳定飞行，RMSE随轨迹速度增加而增大（0.055m到0.141m），但无发散。RDP成功捕捉了悬挂负载的振荡动力学（图10）。

### 3.5 技术细节

- **观测空间**：包含位置误差、旋转矩阵（9维，避免万向锁）、线速度、角速度、上一时刻动作，并堆叠32步历史以处理部分可观测性。
- **动作空间**：4维，包括总推力和三个期望机体角速率。
- **电机PWM信号的重要性**：作者强调，仅使用高层推力/角速率指令不足以唯一确定瞬时扰动，必须包含低层电机PWM信号，因为它隐含了电机延迟、饱和等瞬态信息。
- **Sim-to-Real校准的发现**：线性校准的斜率$m \approx 1.0$，证明仿真和真实世界的动力学流形在结构上是同构的，主要差异在于一个常数仿射平移。这是一个非常重要的发现。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  证明了**主动适应**优于**被动鲁棒**：DR策略的保守性在真实世界中会导致振荡，而主动感知和响应扰动的策略性能更优。
    2.  验证了**隐式扰动估计**的可行性：仅凭标准机载传感器（IMU、光流）和电机信号，就能准确估计复杂的6维外部扰动，无需昂贵的力/力矩传感器。
    3.  提出了**控制相关估计**的概念：RDP不需要完美重建物理量，只要提供一个与趋势一致、控制相关的信号，RL策略就能有效补偿。这降低了估计器的精度要求。
    4.  提供了**数据高效的Sim-to-Real桥梁**：发现仿真与现实之间的差异是近似线性的，从而可以用极少量数据实现高效校准，极大地降低了部署成本。

- **对未来研究/应用的影响**：
    - 为在复杂、动态环境中部署高可靠性无人机提供了新的范式。
    - 其“神谕引导训练+轻量级估计器”的模块化设计思路，可以推广到其他机器人控制问题。
    - 数据高效的校准方法对于资源受限的微型飞行器尤其有价值。

- **局限与开放问题**：
    - 当电机特性因磨损而出现非对称时，仅依赖PWM信号可能不足以准确估计力矩。
    - 当前方法尚未处理剧烈的气动阵风扰动。
    - 需要验证该方法在更大尺度四旋翼和其他构型（如固定翼、VTOL）上的可扩展性。

## 5. 澄清与简化（Clarifications and Simplifications）

- **神谕（Oracle）策略**：可以想象成一个“开挂”的飞行员，他能看到所有看不见的力（比如风、负载），因此总能做出最完美的反应。这个“开挂”的飞行员就是Oracle策略，它告诉我们“如果知道一切，能飞多好”，为我们的实际系统设定了天花板。
- **残差动力学预测器（RDP）**：可以理解为一个“盲人摸象”的专家。他看不见风，但他能通过感受飞机的晃动（历史状态）和听发动机的声音（PWM信号），来“猜”出风是从哪个方向吹来的、有多大。这个“猜测”就是RDP的输出。
- **Sim-to-Real校准桥**：可以想象成一副“有色眼镜”。在仿真世界里，RDP的“视力”是正常的。但到了现实世界，由于光线、空气不同，他看东西会有点“偏色”（稳态偏移）。线性校准桥就是一副很简单的“滤镜”，只需要戴几秒钟，就能把颜色调回正常，让他能像在仿真里一样“看”得清楚。
- **为什么RDP比Oracle还好？**：这听起来反直觉，但可以这样理解：Oracle得到的“真实”扰动在真实世界里可能并不“真实”，因为模型有误差。而RDP是从真实飞行数据中“学”出来的，它产生的信号虽然不完美，但可能更符合真实控制器的“语言”，因此RL策略能更好地理解并利用它。

- **相关工作关联**：本文的核心思想深受**Rapid Motor Adaptation (RMA)**（Ref. 7）的启发，该工作最初在足式机器人上取得了巨大成功。本文成功地将这一思想迁移到了动态性更强的四旋翼飞行器领域。与RMA使用抽象潜变量不同，本文的RDP显式地估计物理量（力和力矩），这是一个重要的区别和改进。
