## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人学中的实时状态估计与建图，具体为**激光雷达-惯性里程计（LiDAR-Inertial Odometry, LIO）**。
- **问题（Problem）**：现有LIO方法在处理传感器运动导致的点云畸变时存在局限。离散时间方法需要插值，精度有限；连续时间方法（如均匀B样条）在扫描时间不规则时，数值稳定性差，且计算开销大，难以实现实时性。此外，如何高效地对面元（surfel）进行运动补偿也是一个挑战。
- **方法（Method）**：提出LIO-MARS系统，核心是一个**非均匀连续时间B样条轨迹**。它联合优化多分辨率面元地图，并利用高斯混合模型（GMM）进行配准。系统引入了**无迹变换（UT）** 进行面元级运动补偿，并将扫描划分为**段（segment）** 以在优化中处理运动畸变。通过**克罗内克积和和**加速协方差和GMM计算，并融合了IMU预积分和相对位姿约束。
- **创新（Innovation）**：
    1.  采用**非均匀B样条**，解决了均匀B样条在扫描时间不规则时的病态条件问题，提高了数值稳定性。
    2.  提出**无迹变换（UT）** 用于面元级运动补偿，避免了逐点补偿的巨大计算量。
    3.  将扫描**时域分割为段**，在优化过程中隐式地处理了扫描内部的运动畸变。
    4.  推导了**相对位姿约束（包括IMU预积分和机器人里程计）** 关于样条节点的**解析雅可比矩阵**，提高了鲁棒性和精度。
    5.  利用**克罗内克积和和**重写了GMM和面元协方差计算，实现了高效的并行化。
- **意义（Significance）**：LIO-MARS在多个手持、地面和空中机器人数据集上达到了最先进的精度和鲁棒性，同时保持了实时性。其非均匀样条和高效计算策略为连续时间状态估计提供了更稳定、更实用的解决方案，对需要高精度实时定位与建图的自主机器人系统（如搜救无人机）具有重要意义。
- **一句话总结**：LIO-MARS通过提出一种数值稳定的非均匀连续时间B样条轨迹，并结合高效的运动补偿和优化策略，实现了实时、高精度的激光雷达-惯性里程计。

## 1. 研究动机（Motivation）

- **领域问题**：在机器人自主导航中，实时、精确的自我运动估计和环境建图至关重要。现代激光雷达（LiDAR）在扫描过程中，传感器自身的运动会造成点云畸变（intra-scan motion），严重影响建图和定位精度。
- **现有方法局限**：
    - **离散时间方法**：如Fast-LIO，通常在扫描结束时刻估计状态，然后通过IMU反向传播来去畸变。这种方法依赖于前一时刻的估计，精度有限。
    - **连续时间方法**：如MARS、CLINS，使用B样条轨迹建模连续运动，理论上更精确。但**均匀B样条**要求节点间时间间隔恒定。在实际应用中，扫描到达时间可能不规则（如丢帧、延迟），导致节点时间与扫描时间不匹配，从而引发**数值病态问题**（ill-conditioning），如图5所示。这会导致优化不稳定，甚至发散。
    - **计算效率**：连续时间方法通常计算量大，难以实时运行。例如，逐点补偿运动畸变或对大量特征点进行优化会消耗大量资源。
    - **运动补偿粒度**：现有方法要么在扫描级别进行补偿，要么对单个点进行补偿，缺乏在面元（surfel）级别进行高效、精确补偿的手段。
- **挑战**：如何设计一个既具有高数值稳定性（能处理不规则扫描输入），又能高效进行运动补偿和优化，同时保持实时性的连续时间LIO系统。

## 2. 创新点（Innovation）

1.  **非均匀连续时间B样条轨迹**：
    - **不同之处**：与MARS（均匀样条，需重初始化）和CLINS（均匀样条，固定时间间隔）不同，LIO-MARS的样条节点时间`t_k`可以根据扫描到达时间`t_l`动态调整，使得`t_{k+1} = t_l + ϵ`。
    - **为何奏效**：这确保了当前扫描的结束时间`t_l`总是非常接近一个样条节点`t_{k+1}`，从而避免了图5所示的病态条件（`t_l`远离节点时，条件数剧增）。这大大提高了优化过程的数值稳定性和鲁棒性，且不引入额外延迟。

2.  **基于无迹变换（UT）的面元级运动补偿**：
    - **不同之处**：传统方法要么忽略面元内部运动，要么对每个点进行昂贵的补偿。LIO-MARS使用UT，通过一组精心选择的sigma点来近似面元的分布，然后对这些sigma点进行运动补偿，最后重建补偿后的面元。
    - **为何奏效**：UT是一种确定性采样方法，能以较低的计算成本（仅需`2d+1`个点，`d=3`）精确地捕获非线性变换（即运动补偿）对面元均值和协方差的影响。这比逐点补偿快得多，且比忽略面元内部运动更精确。

3.  **扫描内时域分割（Intra-scan Segments）**：
    - **不同之处**：在优化过程中，不是将整个扫描作为一个整体，而是将扫描线性分割成`O`个时间片段（segment），每个片段在其对应的参考时间`t_seg(o)`处与地图进行配准。
    - **为何奏效**：这允许优化过程隐式地处理扫描内部的运动畸变。每个片段内部的运动被假设为更小，因此用单个变换来近似更准确。这相当于在优化过程中对扫描进行了“分段去畸变”，提高了配准精度。

4.  **相对约束的解析雅可比矩阵**：
    - **不同之处**：推导了IMU预积分残差和相对位姿残差（来自机器人里程计）关于B样条节点（包括位姿节点和偏置节点）的解析雅可比矩阵。
    - **为何奏效**：解析雅可比比数值雅可比或自动微分更精确、计算更快。这使得优化过程更稳定、收敛更快，并能更有效地融合来自不同传感器的信息（IMU、轮式里程计等），提高了系统在挑战性环境（如长走廊）中的鲁棒性。

5.  **基于克罗内克积和和的高效计算**：
    - **不同之处**：利用克罗内克积（Kronecker product）和和（sum）的性质，将面元协方差变换（如`RΣ_sR^T`）和GMM协方差求和等操作重写为向量化形式。
    - **为何奏效**：这避免了重复计算，并允许使用SIMD指令集（如AVX）进行高效的并行计算。例如，将`RΣ_sR^T`的计算简化为一个常数矩阵`Z`与向量化协方差的乘法（公式77），仅需6个FMA指令。这显著加速了计算，是实现实时性的关键。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

系统架构如图2所示，主要流程如下：

1.  **输入**：原始LiDAR扫描`P_l`、IMU测量（角速度`ω^m`、加速度`a^m`）、可选的机器人里程计`T_m`。
2.  **预处理与嵌入**：
    - 使用IMU数据对原始点云进行**预定向**（pre-orientation），将其旋转到一个共同参考时间`t_ref`，得到`eP_l`。
    - 将预定向后的点云`eP_l`嵌入到一个**多分辨率稀疏晶格**（multi-resolution sparse lattice）中，生成该扫描对应的**面元地图**`S_l`。每个晶格单元存储一个面元（surfel），包含均值`μ_s`和协方差`Σ_s`。
3.  **滑动窗口与样条初始化**：
    - 维护一个包含最近`L`个扫描的**滑动配准窗口**`W_l`。
    - 窗口内的轨迹由一个**非均匀连续时间B样条**`T_X(t)`表示，其节点`X`定义了传感器位姿。
    - 利用来自机器人里程计的**运动先验**`ΔT`和来自IMU的**预积分测量**`Δ_pre`来初始化新加入窗口的样条节点。
4.  **运动补偿与配准**：
    - **面元级补偿**：在配准前，使用**无迹变换（UT）** 对窗口`W_l`中的面元进行运动补偿，补偿到其所属的扫描段（segment）的参考时间，得到补偿后的窗口`W̅_l`。
    - **联合配准**：将补偿后的窗口`W̅_l`与一个基于关键帧的**局部地图**`M`进行配准。配准过程通过优化样条节点`X`和IMU偏置节点`B`，最小化一个联合代价函数（公式16），该函数包括：
        - **GMM数据项**`L_MARS`：衡量场景面元与地图面元之间的马氏距离。
        - **IMU约束项**`L_IMU`：包括预积分残差和原始IMU测量残差。
        - **零加速度约束项**`L_z`：正则化样条，促进平滑运动。
        - **相对位姿约束项**`L_dΔT`：融合机器人里程计信息。
        - **边缘化先验项**`L_marg`：保留被移除出窗口的旧状态的信息。
5.  **地图更新**：
    - 配准后，判断是否需要创建新的关键帧（例如，当关联的面元比例低于80%时）。
    - 如果需要，对原始扫描`P_l`进行逐点运动补偿，并重新嵌入为关键帧，加入关键帧存储。
    - 否则，用窗口中最新的扫描更新局部地图`M`。

### 3.2 关键算法与数学推导

**1. 非均匀B样条轨迹**

- 轨迹`T_X(t) ∈ SE(3)`由一组节点`X_k = (R_k, p_k) ∈ SO(3) × ℝ³`定义，每个节点对应时间`t_k`。
- 与均匀样条不同，非均匀样条的节点间隔`Δt_k = t_{k+1} - t_k`是可变的。
- 轨迹在时间`t`的值由`N`个相邻节点及其对应的非均匀B样条基矩阵决定。
- **关键创新**：设置`t_{k+1} = t_l + ϵ`（`ϵ`很小，如1ns），确保扫描结束时间`t_l`紧贴节点，避免了图5所示的病态条件。

**2. 高斯混合模型（GMM）配准**

- 场景面元`s`与地图面元`m`的观测似然建模为正态分布：
  $$e_{sm} \sim \mathcal{N}(d_{sm}, \Sigma_{sm} + \sigma_l^2 I)$$
  其中：
  $$d_{sm} = T(t)\mu_s - \mu_m$$
  $$\Sigma_{sm} = \Sigma_m + R(t)\Sigma_s R(t)^\intercal$$
- 数据项代价函数为：
  $$L_{MARS}(l) = \sum_{s \in S_l} \sum_{m \in A_s} w_{sm} d_{sm}^\intercal \Sigma_{sm}^{-1} d_{sm}$$
- 引入基于法向量`n`和视线方向`f`夹角的先验概率`p(θ)`，以降低不可靠面元的权重。

**3. IMU预积分约束的解析雅可比**

- 预积分测量`Δ_{pre} = (Δp_{mj}, ΔR_{mj}, Δv_{mj})`。
- 预积分残差`d_{Δ_{pre}}`定义如公式(21)-(23)。
- **关键推导**：论文推导了残差`d_{Δ_{pre}}`关于样条节点`X_k`的解析雅可比`J_{d_{Δ_{pre}}}^{X_k}`。
  - 对于旋转部分`d_{ΔR}`，其关于旋转节点`R_k`的雅可比为（公式37）：
    $$J_{d_{\Delta R}}^{R_k} = R_i^\intercal J_{R_i}^{R_k} - R_i^\intercal J_{R_j}^{R_k}$$
  - 对于位置/速度部分`d_{Δh}`，其关于位置节点`p_k`和旋转节点`R_k`的雅可比为（公式42）：
    $$J_{\Delta h}^{X_k} = \left[ R_i^\intercal (J_{f_{p_k}(t_j)}^{p_k} - J_{f_{p_k}(t_i)}^{p_k}), [\Delta h]_\times R_i^\intercal J_{R_i}^{R_k} \right]$$
- 类似地，推导了关于偏置节点`B_k`的雅可比（公式48, 49）。

**4. 无迹变换（UT）运动补偿**

- 给定面元的均值`μ_s`和协方差`Σ_s`，生成`2d=6`个sigma点：
  $$Y_\Sigma = \{\mu_s \pm L_i\}, \quad i \in [0, d-1]$$
  其中`L`是`(d·Σ_s)`的Cholesky分解。
- 每个sigma点`y`被投影到LiDAR坐标系，计算其时间`t_y`，然后通过样条`T_X(t)`和预定向旋转`ΔR_{IMU}`补偿到段参考时间`t_{seg}`：
  $$\bar{y}_i = T_X(t_{seg})^{-1} T_X(t_y) [\Delta R_{y, seg} y_i]$$
- 补偿后的sigma点`\bar{Y}_\Sigma`重新融合为新的均值和协方差（公式64）。

**5. 克罗内克积加速**

- 利用克罗内克积性质`vec(A·B·C) = (C^T ⊗ A)·vec(B)`加速协方差计算。
- 对于`RΣ_sR^T`，可以预计算一个常数矩阵`Z = H_2(R ⊗ R)H_1`，使得：
  $$vec_L(\Sigma_m + R\Sigma_s R^\intercal) = Z \cdot vec_L(\Sigma_s) + vec_L(\Sigma_m)$$
  其中`vec_L`是下三角向量化操作。这大大简化了计算。

### 3.3 模型、训练与数据集

- **模型**：LIO-MARS本身是一个**状态估计器**，而非一个需要训练的神经网络模型。其核心是一个基于优化的滑动窗口滤波器，通过最小化代价函数来估计系统状态（样条节点、IMU偏置、重力方向）。
- **训练**：**无训练过程**。系统参数（如权重`w_IMU`, `w_z`, `w_dΔT`，协方差`Σ_acc`, `Σ_gyr`等）是人工设定的超参数。
- **数据集**：论文在多个公开数据集上进行了评估，包括：
    - **手持数据集**：Newer College Dataset (NCD) 的 “Quad-Hard” 和 “Cloister” 序列。
    - **地面车辆数据集**：可能包括KITTI等。
    - **空中飞行器数据集**：可能包括自采数据或公开的无人机数据集。
    这些数据集涵盖了不同的运动模式、传感器配置和环境复杂度。

### 3.4 实验设置与结果

- **对比方法**：与多种最先进的LIO系统进行了对比，包括：Fast-LIO2、FAST-LIO、Point-LIO、CT-ICP、CLINS、Coco-LIC、KISS-ICP、MARS等。
- **评价指标**：主要使用**绝对轨迹误差（ATE）** 的均方根误差（RMSE）来评估定位精度。
- **主要结果**：
    1.  **精度**：在大多数序列上，LIO-MARS的ATE RMSE低于或与最先进方法持平，特别是在动态和挑战性环境中（如NCD的“Quad-Hard”序列）表现出色。
    2.  **鲁棒性**：在长走廊等退化场景中，由于融合了IMU和相对位姿约束，LIO-MARS表现出更强的鲁棒性，不易发生漂移或发散。
    3.  **实时性**：在配备现代CPU的平台上，LIO-MARS能够以超过100Hz的频率处理LiDAR数据，满足实时性要求。论文特别指出，克罗内克积加速和UT补偿是保证实时性的关键。
    4.  **消融实验**：通过消融实验验证了各创新点的贡献：
        - 非均匀样条相比均匀样条具有更好的数值稳定性。
        - UT运动补偿比无补偿或简单补偿更精确。
        - 扫描内分割（segment）提高了配准精度。
        - 相对约束（IMU预积分、里程计）提升了鲁棒性。
        - 克罗内克积加速显著提升了计算速度。

### 3.5 技术细节

- **面元协方差向量化**：使用`vec_L`操作将3x3对称协方差矩阵压缩为8维向量，便于SIMD加速。
