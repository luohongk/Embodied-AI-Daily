## 0. 概览（Concise Summary）

- **主题（Topic）**：大型语言模型（LLMs）的架构设计，特别是如何高效地扩展模型的计算量（Compute）和容量（Capacity）。
- **问题（Problem）**：传统的循环（Looped）Transformer通过共享参数来节省参数，但牺牲了模型容量；而宽度扩展（Width-scaled）模型则相反。现有方法无法让模型根据不同的token灵活地在“计算深度”和“参数容量”之间进行权衡。
- **方法（Method）**：提出一种**双路径（Dual-Path）** 块结构。该结构在一个层内并行两条路径：一条是**深度路径（Deep Path）**，通过多次应用共享参数的子层来增加计算量；另一条是**宽度路径（Wide Path）**，通过一个更大的前馈网络（FFN）来增加容量。一个可学习的逐token门控（per-token gate）动态地融合这两条路径的输出。
- **创新（Innovation）**：
    1.  首次在单个Transformer层内明确分离并并行化“计算”和“容量”这两个扩展轴。
    2.  提出一个可学习的、密集的（dense）逐token门控机制，用于动态分配每个token在深度和宽度路径上的资源。
    3.  通过实验证明，在相同FLOPs预算下，该架构在语言建模和下游任务上优于纯深度或纯宽度的基线模型，且参数更少。
- **意义（Significance）**：该工作为LLM的扩展提供了一种更灵活、更高效的范式。它揭示了不同token（如功能词、标点、算术符号）对计算和容量的不同需求，并提供了一个可解释的门控信号，有助于理解模型内部的资源分配策略。这为未来设计更智能、更高效的模型架构开辟了新方向。
- **一句话总结**：本文提出了一种双路径Transformer块，通过并行化计算深度和参数容量，并利用可学习的逐token门控进行动态分配，在相同计算预算下实现了比纯深度或纯宽度模型更优的性能和参数效率。

## 1. 研究动机（Motivation）

- **解决的问题**：大型语言模型在扩展时面临两个关键维度：**计算量（Compute）** 和**容量（Capacity）**。计算量指的是对隐藏状态进行顺序操作的次数（例如，通过增加层数或循环应用同一层），而容量指的是在单步操作中可用的参数数量（例如，通过增加FFN的宽度）。
- **为什么重要**：传统的Transformer架构将这两个维度捆绑在一起。循环（Looped）模型（如Universal Transformer）通过共享参数来节省参数，从而在固定FLOPs下获得更深的“有效深度”，但其容量受限，导致在依赖存储知识的任务上表现不佳。相反，扩展宽度（Width-scaled）的模型拥有更多参数（容量），但缺乏深度计算带来的推理能力。理想情况下，模型应该能够根据不同的token（例如，一个简单的冠词 vs. 一个复杂的数学公式）动态地分配计算和容量资源。
- **挑战在哪里**：
    1.  **维度混淆**：现有架构（如标准Transformer、纯循环模型、纯宽度模型）将计算和容量视为一个整体，无法独立控制。
    2.  **资源分配僵化**：所有token在每一层都消耗相同的计算和容量，无法根据其复杂性进行差异化处理。
    3.  **单一轴线的路由**：已有的工作如Mixture-of-Experts (MoE) 和 Mixture-of-Depths (MoD) 虽然实现了动态路由，但它们只在一个轴线上进行选择（选择哪个专家、是否跳过某层），无法同时权衡计算深度和参数容量这两个不同的维度。

## 2. 创新点（Innovation）

- **提出的新方法**：提出了**双路径（Dual-Path）块**，该块在一个Transformer层内并行运行两条路径：
    - **深度路径（Deep Path）**：一个共享参数的子层，被循环应用K次，用于增加顺序计算量。
    - **宽度路径（Wide Path）**：一个具有更大FFN维度的子层，仅应用一次，用于增加参数容量。
- **与现有方案的不同**：
    1.  **并行双轴**：与MoE（选择不同专家）或MoD（选择是否应用层）不同，双路径块同时提供两种不同类型的更新（深度计算 vs. 宽度容量），并通过门控进行融合。
    2.  **密集门控**：与MoE的稀疏（top-k）路由不同，双路径块的门控是**密集的（dense）**，即两条路径都会被计算，门控值（0到1之间）决定了它们对最终输出的贡献比例。这意味着模型没有跳过任何计算，而是重新分配了计算资源。
    3.  **可组合性**：作者指出，双路径门控与MoE路由是正交的，可以组合使用。例如，可以在宽度路径内部署一个MoE层，从而在“循环计算”、“路由容量”和“专家选择”等多个维度上进行更精细的分配。
- **为什么能奏效**：因为它允许模型根据每个token的特定需求，在“需要更多思考（深度）”和“需要更多知识（宽度）”之间进行权衡。实验表明，这种灵活性使得模型在相同的FLOPs预算下，能够更有效地利用参数，从而在整体性能上超越那些只能选择单一轴线的模型。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文的核心是**双路径块（Dual-path block）**，其结构如图1(d)所示。该块接收一个输入 $x$，然后并行处理两条路径：

1.  **深度路径（Deep Path）**：这是一个循环结构。一个共享参数的子层 $\Phi_{\text{deep}}$ 被应用 $K$ 次。$\Phi_{\text{deep}}$ 包含一个注意力层和一个FFN层。该路径的输出 $h_{\text{deep}}$ 是所有中间状态的加权组合，权重由一个小型路由器（router）动态生成。
2.  **宽度路径（Wide Path）**：这是一个标准的单次应用子层 $\Phi_{\text{wide}}$，但其FFN的隐藏维度 $d_{\text{wide}}$ 远大于深度路径的 $d_{\text{deep}}$。该路径的输出为 $h_{\text{wide}}$。
3.  **逐token门控（Per-token Gating）**：一个线性投影 $W_g$ 将输入 $x$ 映射到两个logits，然后通过sigmoid函数得到两个独立的门控值 $g_d$ 和 $g_w$，范围在 $[0, 1]$。最终的输出 $y$ 是两条路径输出的加权和：$y = g_d \odot h_{\text{deep}} + g_w \odot h_{\text{wide}}$。

**图1** 清晰地展示了标准块、纯循环块、纯宽度块和双路径块的结构差异。

### 3.2 关键算法与数学推导

**1. 标准Transformer子层:**
给定输入 $x$，一个标准子层 $\Phi$ 定义为：
$$u = x + s \cdot \text{Attn}(\text{RMSNorm}(x))$$
$$\Phi(x; s) = u + s \cdot \text{FFN}(\text{RMSNorm}(u))$$
其中 $s$ 是一个标量增益。

**2. 深度路径（Deep Path）:**
深度子层 $\Phi_{\text{deep}}$ 被循环应用 $K$ 次，参数共享：
$$h^{(k)} = \Phi_{\text{deep}}(h^{(k-1)}; s_d^{(k)})$$
其中 $h^{(0)} = x$，$k = 1, \ldots, K$。$s_d^{(k)} = \text{softplus}(\alpha^{(k)})$ 是每步可学习的增益。

深度路径的最终输出 $h_{\text{deep}}$ 是所有中间状态的加权组合：
$$h_{\text{deep}} = \sum_{k=1}^{K-1} \pi_k q_k h^{(k)} + \pi_K h^{(K)}$$
其中 $q_k \in [0, 1]$ 是由一个小型路由器生成的每步权重，$\pi_k = \prod_{j < k} (1 - q_j)$ 是一个累积概率，确保权重之和为1。这允许模型在逐token的基础上“忽略”某些循环步骤。

**3. 宽度路径（Wide Path）:**
宽度子层 $\Phi_{\text{wide}}$ 应用一次：
$$h_{\text{wide}} = \Phi_{\text{wide}}(x; s_w), \quad s_w = \text{softplus}(\beta)$$

**4. 逐token门控（Per-token Gating）:**
门控logits由输入 $x$ 通过线性层得到：
$$(\ell_d, \ell_w) = xW_g + b_g$$
门控值通过sigmoid函数计算：
$$g_d = \sigma(\ell_d), \quad g_w = \sigma(\ell_w)$$
最终输出：
$$y = g_d \odot h_{\text{deep}} + g_w \odot h_{\text{wide}}$$
其中 $\odot$ 表示逐元素相乘。

**5. 路由分析指标:**
为了分析门控行为，论文定义了**深度份额（Deep Share）** $\rho_d$，表示来自深度路径的更新在总更新中的比例：
$$\rho_d = \frac{g_d \|\Delta_d\|}{g_d \|\Delta_d\| + g_w \|\Delta_w\|} \in [0, 1]$$
其中 $\Delta_d = h_{\text{deep}} - x$ 和 $\Delta_w = h_{\text{wide}} - x$ 是两条路径的更新向量。

### 3.3 模型、训练与数据集

- **模型架构**：基于GPT-2风格的decoder-only Transformer。骨干网络固定：$L=16$ 层，隐藏维度 $d=768$，12个注意力头。使用RoPE位置编码、RMSNorm和SwiGLU激活函数。
- **配置**：在两种FLOPs预算下（每层每token的FFN FLOPs为 $F_M = 80M$ 和 $F_M = 160M$）进行实验。对于双路径模型，改变深度路径的FLOPs分配比例 $\alpha \in \{25, 50, 75\}$ 和循环深度 $K \in \{2, 3, 4\}$。参数数量从约2.38亿到14亿不等。
- **数据集**：在Nemotron-CC数据集的去重子集上训练了380亿个token。
- **训练**：使用AdamW优化器，峰值学习率 $5 \times 10^{-4}$，线性预热184步，余弦衰减至 $5 \times 10^{-5}$。训练在64个GPU上进行，耗时12.8到21.4小时。

### 3.4 实验设置与结果

- **基线模型**：与两个等FLOPs的单轴线控制模型比较：
    - **PUREWIDE**：纯宽度模型，无循环。
    - **PURELOOP**：纯循环模型，无宽度路径。
- **评估指标**：
    - **语言建模**：Paloma C4和WikiText-103上的bits-per-byte (BPB)。
    - **常识推理**：6个任务（ARC-c, ARC-e, HellaSwag, PIQA, SIQA, WinoGrande）的平均准确率和BPB。
    - **数学推理**：GSM8k准确率和OLMo3基础数学BPB。
- **主要结果（表1）**：
    1.  **双路径模型优于单轴线模型**：在两个FLOPs预算下，最佳的双路径配置在聚合BPB指标上均优于PUREWIDE和PURELOOP。例如，在 $F_M = 80M$ 预算下，$\alpha=50$ 的双路径模型（4.83亿参数）取得了最佳的聚合BPB（0.8693），而PUREWIDE（7.19亿参数）为0.8753，最佳PURELOOP为0.8880。
    2.  **参数效率**：双路径模型在达到或超越基线性能的同时，使用的参数更少（例如，在 $F_M = 80M$ 预算下，最佳双路径模型参数仅为PUREWIDE的67%）。
    3.  **可预测的权衡**：容量密集型配置（$\alpha=25$，偏向宽度路径）在语言建模和常识任务上表现更好；计算密集型配置（$\alpha=75$，偏向深度路径）在数学推理任务上表现更好。这验证了“宽度=知识，深度=推理”的直觉。
- **路由分析（图3-7）**：
    1.  **层深度影响**：网络中间层（L2-L9）偏向宽度路径，而最后几层（L14-L15）偏向深度路径（图4a）。
    2.  **任务影响**：在数学推理（GSM8K）中，数字和运算符偏向深度路径；在事实知识（TriviaQA）中，答案token也偏向深度路径（图5, 6）。
    3.  **词性（POS）影响**：功能词（如冠词、介词）和词汇内容（如动词、形容词）偏向宽度路径；而标点、符号和算术token偏向深度路径（图7）。
    4.  **门控极化**：随着层数加深，门控决策变得更加极化，即token更倾向于完全选择一条路径（图3）。

### 3.5 技术细节

- **FLOPs匹配协议**：为了公平比较，论文详细描述了如何精确匹配不同配置下的每层每token的FFN FLOPs。通过调整FFN的隐藏维度 $d_{\text{ffn}}$ 来实现，并考虑了SwiGLU的2/3缩放规则和路由器的微小开销。
- **深度路径的初始化**：深度路径的每步增益 $s_d^{(k)}$ 初始化为接近0（通过将 $\alpha^{(k)}$ 初始化为-7），使得循环开始时接近恒等映射，模型逐步学习偏离输入。
- **门控初始化**：门控的权重 $W_g$ 和偏置 $b_g$ 初始化为0，使得每个token在训练开始时获得 $g_d = g_w = 0.5$ 的平衡分配。
- **推理时消融实验（表4）**：通过强制改变循环次数或门控值，证明了学习到的门控决策是至关重要的。例如，随机打乱门控值会导致性能显著下降，表明门控的逐token决策承载了重要信息。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  解决了LLM扩展中“计算”与“容量”难以独立权衡和动态分配的问题。
    2.  提出了一种新颖、有效且可解释的架构，在相同计算预算下实现了更好的性能-参数权衡。
    3.  提供了丰富的路由分析，揭示了模型内部资源分配的规律，为理解Transformer的工作机制提供了新的视角。
- **对未来研究/应用的影响**：
    1.  **新的扩展范式**：双路径架构为设计更高效的LLM提供了新思路，未来可以探索其在更大规模模型上的表现。
    2.  **可组合性**：作者指出该门控机制与MoE是正交的，可以组合，这为设计更强大的混合架构（如“循环计算 + 路由容量”）铺平了道路。
    3.  **可解释性工具**：学习到的门控值作为一个“免费”的副产品，为分析模型行为、诊断模型缺陷提供了强大的工具。
- **局限与开放问题**：
    1.  **规模验证**：实验仅在16层、最多14亿参数的模型上进行，在更大规模（如百亿、千亿参数）模型上的表现有待验证。
    2.  **与MoE的组合**：虽然理论上可行，但将双路径门控与MoE结合的具体实现和效果尚未在本文中实验验证。
    3.  **训练成本**：虽然参数更少，但由于两条路径都被计算，双路径块的实际计算成本（FLOPs）与基线模型是匹配的，但训练时间可能因架构复杂性而略有增加。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：一个多功能的团队**：
    - **标准Transformer**：团队里每个人（层）都做同样的工作，既需要思考（计算）又需要查阅资料（容量），效率不高。
    - **纯循环模型（PURELOOP）**：团队里只有少数几个核心成员，他们需要反复思考（循环）来解决所有问题，但知识储备（容量）有限。
    - **纯宽度模型（PUREWIDE）**：团队里有很多专家（大FFN），知识渊博，但每个人只思考一次，缺乏深度推理。
    - **双路径模型（Dual-Path）**：团队里同时有“深度思考者”（深度路径）和“知识专家”（宽度路径）。对于简单的问题（如“a”），团队可能主要调用知识专家；对于复杂的问题（如“15 * 1/5 = ?”），团队会让深度思考者反复推演，同时知识专家提供必要的背景知识。一个聪明的项目经理（门控）会根据每个任务（token）的特点，动态分配任务给思考者和专家。

- **深度份额（$\rho_d$）**：可以理解为“思考的投入程度”。如果 $\rho_d = 0.8$，意味着模型在处理这个token时，80%的“精力”花在了反复思考（深度路径）上，只有20%花在了查阅知识库（宽度路径）上。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - **Looped/Recursive Transformers**：本文的深度路径直接借鉴了此类工作的思想（如Universal Transformer），但将其作为并行路径之一，而非整个模型。
    - **Mixture-of-Experts (MoE)**：MoE在
