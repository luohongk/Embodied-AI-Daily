## 0. 概览（Concise Summary）

- **主题（Topic）**：大规模短视频搜索中的查询重写（Query Rewriting），特别是利用大型语言模型（LLM）进行上下文感知的查询改写。
- **问题（Problem）**：工业级LLM查询重写器面临训练效率与奖励信号保真度之间的根本矛盾。训练奖励必须反映生产排序器（ranker）如何使用重写结果，但训练过程必须足够廉价以支持数据漂移下的持续部署。此外，缺乏“黄金标准”的重写标签，且离线奖励代理与在线生产环境之间存在“模拟-生产”差距。
- **方法（Method）**：提出了CoRe系统，核心包括：
    1.  **生产对齐的奖励构建**：使用部署中的多模态相关性模型作为奖励源，并采用与生产排序融合代数相匹配的乘法比率形式。
    2.  **半在线混合偏好优化（Semi-online MPO）**：一种高效的训练循环，通过DPO风格的成对目标限制梯度传播，并通过阶段结构减少参数同步，使得在百万级实例规模下进行每周重训练成为可能。
    3.  **自动晋升门控（Auto-promotion Gate）**：基于奖励类指标和稳定性指标，自动检测并防御奖励破解（reward hacking）等不良行为。
    4.  **加法并行路径消费模式**：重写器的输出作为并行信号，在召回、粗排和精排三个阶段被消费，而非替换原有信号，从而限制重写器故障的影响范围。
- **创新（Innovation）**：
    1.  首次提出了一个**生产代数对齐的乘法奖励函数**，有效弥合了训练与部署之间的差距。
    2.  提出了**半在线MPO训练框架**，在工业级规模下实现了高效且稳定的持续学习。
    3.  设计了一个**双族自动晋升门控**，能够自动检测并防御奖励破解，确保了长期训练的稳定性。
    4.  实现了**跨所有排序阶段的加法并行消费模式**，将重写器的风险影响范围降至最低。
- **意义（Significance）**：本文是首个详细记录LLM查询重写器在大型短视频搜索平台上，经过**超过五个月、每周自动重训练和部署**的工业级实践。它证明了通过精心设计的奖励函数、高效的训练算法和稳健的部署架构，LLM重写器可以在生产环境中持续、安全地提升搜索质量，为工业界提供了可复现的范本。
- **一句话总结**：本文提出了CoRe系统，通过生产对齐的奖励、半在线MPO训练和稳健的部署架构，成功实现了一个在工业级视频搜索中每周持续更新并稳定运行超过五个月的LLM查询重写器。

## 1. 研究动机（Motivation）

- **核心矛盾**：在工业级搜索系统中，使用LLM进行查询重写面临一个根本性的张力。一方面，重写器的训练奖励必须**真实反映**生产排序器（ranker）如何消费这个重写结果，即奖励信号需要与最终排序目标高度一致。另一方面，训练过程必须**足够廉价和高效**，以支持在数据分布不断漂移（data drift）的情况下进行频繁（如每周）的重新部署和更新。
- **关键挑战**：
    1.  **缺乏黄金标准**：查询重写没有绝对正确的标签。一个重写的好坏，完全取决于下游排序系统如何利用它以及用户的最终行为。因此，训练信号必须从用户会话的后验行为中诱导出来。
    2.  **模拟-生产差距**：许多现有工作使用离线排序器或检索指标作为奖励代理，但这些代理与生产环境中的复杂排序融合逻辑存在差异，导致训练出的重写器在线上表现不佳。
    3.  **训练效率与规模**：为了捕捉不断演变的用户兴趣，需要大规模的训练数据。全在线策略（如GRPO）在百万级实例规模下计算成本过高，无法满足每周重训练的节奏。
    4.  **长期稳定性**：在持续数月的训练过程中，模型容易出现奖励破解（reward hacking）等退化行为，例如通过生成冗长但无意义的文本来欺骗奖励函数。
- **重要性**：解决这些问题对于将LLM查询重写技术从实验室研究成功落地到工业级生产环境至关重要，能够显著提升大规模搜索系统的相关性和用户体验。

## 2. 创新点（Innovation）

论文提出了四个相互支撑的创新贡献：

1.  **生产对齐的奖励构建（Production-Aligned Reward Construction）**：
    - **新意**：与使用离线排序器作为奖励源不同，CoRe直接使用**部署中的多模态相关性模型**作为奖励源。更重要的是，奖励函数的形式（乘法比率）被设计为与生产排序融合因子（ranking-fusion factor）的代数形式相匹配。
    - **为何奏效**：这种代数一致性确保了重写器被训练去优化一个与线上排序时使用的相同评分函数（$s(\tilde{q}, d)$）的单调函数。这从根本上消除了离线奖励代理带来的“模拟-生产”差距，使得训练目标与线上优化目标完全对齐。

2.  **半在线混合偏好优化（Semi-online MPO）**：
    - **新意**：提出了一种介于完全离线（DPO）和完全在线（GRPO）之间的半在线训练范式。它在一个阶段（phase）内，只进行一次rollout采样，然后使用DPO风格的成对目标（只对top-k/bottom-k的轨迹计算梯度）进行多步梯度更新。同时，采用阶段结构来减少训练器与推理服务器之间的参数同步次数。
    - **为何奏效**：在梯度计算占主导的短文本生成场景下，通过限制梯度传播的轨迹数量，大幅降低了单步训练的计算成本。阶段结构则有效减少了参数同步的开销，并允许将奖励计算调度到非高峰时段，从而使得在百万级实例规模下进行每周重训练成为可能。

3.  **双族自动晋升门控（Two-Family Automated Promotion Gate）**：
    - **新意**：设计了一个包含**奖励类指标**（如$r_{main}$, $S_{pos}$, $r_{rand}$）和**稳定性指标**（如平均长度、截断率、top-10熵）的自动门控系统，用于决定每周训练出的新模型是否晋升到生产环境。
    - **为何奏效**：奖励类指标衡量模型性能的提升，而稳定性指标则能捕捉到奖励函数本身无法察觉的病理行为（如奖励破解）。这种双族设计确保了长期训练的稳健性，防止模型在追求短期奖励提升时发生不可逆的退化。论文中详细记录了稳定性指标成功捕获并防御了一次因“冗长性”导致的奖励破解事件。

4.  **加法并行路径消费模式（Additive Parallel-Path Consumption）**：
    - **新意**：重写器的输出不是替换原有信号，而是作为**新的并行信号**，在召回（recall）、粗排（rawrank）和精排（finerank）三个阶段被同时消费。
    - **为何奏效**：这种设计将重写器故障的影响范围（blast radius）降至最低。如果重写器缓存未命中或服务中断，排序系统可以优雅地回退到基线（无重写器）状态，而不会导致服务中断或排名崩溃。这为在关键生产系统中引入LLM组件提供了必要的安全边界。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

CoRe系统的整体架构如图1所示，包含五个核心模块：

1.  **离线训练循环（Weekly）**：每周从会话日志中提取数据，构建偏好对，并使用半在线MPO算法训练一个候选模型$\theta'$。
2.  **生产对齐的奖励计算**：在训练过程中，计算每个候选重写的奖励。该奖励融合了内容安全分数、行为权重（$w^+, w^-$）和部署中的多模态相关性模型得分$s(\cdot)$。
3.  **自动晋升门控**：在每周的评估阶段，将候选模型$\theta'$与当前部署模型$\theta^*$在留出的评估集上进行比较。门控基于三个奖励类指标和四个稳定性指标的联合决策，决定是否晋升$\theta'$。
4.  **近线服务**：重写器在专用的推理实例上运行。当用户事件发生时，异步生成重写，并将结果缓存在内存KV缓存中。在线排序时进行缓存查找，命中则使用，未命中则回退到原始信号。
5.  **在线排序——加法并行路径消费**：重写器的输出$\tilde{q}$在三个排序阶段被消费：
    - **召回**：新增一个“上下文相关性检索器”，使用$\tilde{q}$进行检索。
    - **粗排**：新增一个基于$s(\tilde{q}, d)$的L0/L1融合因子。
    - **精排**：新增一个“上下文相关性节点”，将$s(\tilde{q}, d)$作为全相关性因子的一部分。

### 3.2 关键算法与数学推导

**1. 生产对齐的奖励函数**

奖励函数是核心，其设计旨在与生产排序融合代数对齐。

- **行为加权平均分数**：
  $$S_{pos}(\tilde{q}) = \frac{\sum_{i \in D^+} w_i^+ s(\tilde{q}, d_i)}{\sum_i w_i^+}, \quad S_{neg}(\tilde{q}) = \frac{\sum_{j \in D^-} w_j^- s(\tilde{q}, d_j)}{\sum_j w_j^-}, \quad S_{rand}(\tilde{q}) = \frac{1}{|D_{rand}|} \sum_{k \in D_{rand}} s(\tilde{q}, d_k)$$
  其中，$D^+$是严格点击正样本池，$D^-$是无操作负样本池，$D_{rand}$是随机文档池。$w_i^+$和$w_j^-$分别是基于用户行为的权重（如严格点击率、无操作率）。

- **基础奖励**：
  $$r_{base}(\tilde{q}) = \text{clip} \left[ \left( \frac{S_{pos}(\tilde{q})}{1 + \gamma S_{neg}(\tilde{q})} \right) \cdot \left( \frac{S_{pos}(\tilde{q})}{S_{rand}(\tilde{q})} \right)^{\alpha_R} \right]$$
  - **乘法比率形式**：这是关键创新。它模拟了生产排序融合的代数形式。第一个比率$\frac{S_{pos}}{1+\gamma S_{neg}}$是精排阶段面对的“硬负样本”对比，第二个比率$\frac{S_{pos}}{S_{rand}}$是召回阶段面对的“易负样本”对比。
  - **$(1+\gamma)$指数**：对正样本项施加轻微强调，鼓励模型提升对高价值文档的相关性，而不仅仅是降低对负样本的相关性。
  - **$\alpha_R$指数**：控制易负样本比率的权重，使其提供召回对齐的信号，但不会主导梯度。

- **最终奖励（含长度和截断惩罚）**：
  $$r(\tilde{q}) = r_{base}(\tilde{q}) \cdot \min\left( \frac{L_{target}}{L}, 1 \right)^{\lambda_L} \cdot \beta^{\mathbb{1}_{cut}}$$
  这是为了应对“冗长性奖励破解”而添加的。$L$是重写长度，$\mathbb{1}_{cut}$指示是否因长度限制而被截断。该惩罚轻微惩罚过长或被截断的重写，防止模型通过增加无意义文本来“刷分”。

**2. 半在线混合偏好优化（MPO）损失函数**

损失函数结合了DPO、BCO和SFT三项，以确保训练的稳定性和有效性。

- **MPO损失**：
  $$\mathcal{L}_{MPO} = w_p \mathcal{L}_{DPO} + w_q \mathcal{L}_{BCO} + w_g \mathcal{L}_{SFT}$$

- **DPO项**：提供排序信号，使偏好重写$\tilde{q}^+$的概率高于非偏好重写$\tilde{q}^-$。
  $$\mathcal{L}_{DPO} = -\log \sigma \left( \beta [r_\theta(\tilde{q}^+) - r_\theta(\tilde{q}^-)] \right)$$
  其中$r_\theta(\tilde{q}) = \log \frac{\pi_\theta(\tilde{q}|q, d_{last})}{\pi_{ref}(\tilde{q}|q, d_{last})}$是策略模型与参考模型的对数概率比。

- **BCO项**：提供绝对校准信号，防止DPO中常见的“共同抑制”失败模式（即偏好和非偏好轨迹的概率同时下降）。
  $$\mathcal{L}_{BCO} = -\log \sigma \left( \beta r_\theta(\tilde{q}^+) - \delta_{BCO} \right) - \log \left( 1 - \sigma \left( \beta r_\theta(\tilde{q}^-) - \delta_{BCO} \right) \right)$$

- **SFT项**：作为正则化器，将策略锚定在之前获得高奖励的重写附近，保持探索能力。
  $$\mathcal{L}_{SFT} = -\log \pi_\theta(\tilde{q}^+ | q, d_{last})$$

### 3.3 模型、训练与数据集

- **模型**：重写器是一个基于 **7B参数Mistral** 骨干网络的生成式语言模型 $\pi_\theta(\tilde{q}|q, d_{last})$。
- **训练数据**：
    - **来源**：从短视频搜索平台的**会话日志**中提取。每个训练实例是一个提示上下文 $(q, d_{last})$，其中 $q$ 是用户原始查询，$d_{last}$ 是用户在搜索前刚刚观看的feed文档。
    - **文档池构建**：对于每个实例，从会话日志中构建三个文档池：
        - $D^+$（严格点击正样本池）：用户点击并产生积极交互的文档。
        - $D^-$（无操作负样本池）：用户看到但未交互的文档（硬负样本）。
        - $D_{rand}$（随机文档池）：从语料库中随机采样的文档（易负样本）。
    - **规模**：初始训练使用四周的数据，约 **240万** 个训练实例。每周持续更新使用一周的数据。
- **训练流程**：
    - **初始训练**：进行四个阶段的半在线MPO训练。
    - **持续更新**：每周从上一周的部署检查点 $\theta^*$ 开始，使用新一周的数据进行一次单阶段、单周期的MPO训练，产生候选模型 $\theta'$。

### 3.5 技术细节

- **偏好对构建**：对于每个训练实例，采样$N$个候选重写。默认分支取奖励最高的top-k和最低的bottom-k重写，形成$k \times k$个偏好对（$k=2$）。如果原始查询$q$的奖励远高于所有重写，则使用原始查询作为唯一偏好项。
