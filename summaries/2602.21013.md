## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人操作中的视觉-语言-动作（VLA）模型，特别是如何赋予其记忆能力以解决非马尔可夫（记忆依赖）任务。
- **问题（Problem）**：当前的VLA模型本质上是“无状态”的，它们仅根据当前视觉输入和语言指令预测动作，缺乏对过去状态和自身行为的记忆，因此在需要空间记忆（如记住物体初始位置）和时间记忆（如记住已完成子任务）的复杂、长时程任务中表现不佳。
- **方法（Method）**：提出一种名为“语言草稿本（Language Scratchpad）”的简单而有效的记忆机制。该机制让VLA在执行每个动作前，先预测一段描述当前意图或环境状态的文本（“想法”），并将这些文本作为后续步骤的输入上下文，形成一个显式的、不断演化的记忆轨迹。
- **创新（Innovation）**：创新性地利用VLA模型固有的语言能力，通过一个外部的、可更新的文本草稿本来显式地编码空间和时间信息，从而将“无状态”的VLA转变为“有状态”的策略，且无需修改模型核心架构。
- **意义（Significance）**：解决了VLA模型在处理现实世界中普遍存在的记忆依赖任务时的关键短板。该方法简单、灵活，与现有VLA架构兼容，并能显著提升性能，为构建更智能、更通用的机器人策略开辟了新路径。
- **一句话总结**：本文通过为VLA模型添加一个可自我更新的语言草稿本，使其能够显式地记忆和追踪任务状态与进度，从而在需要空间和时间记忆的复杂操作任务中取得了显著的性能提升。

## 1. 研究动机（Motivation）

- **解决的问题**：当前主流的VLA模型（如RT-2, OpenVLA）在处理非马尔可夫任务时存在根本性缺陷。这些模型是“反应式”或“无状态”的，即 `p(at | ot, l)`，动作仅依赖于当前观测 `ot` 和指令 `l`。然而，许多现实世界的操作任务，如“交换两个物体的位置”、“先触摸A再拾取B”，都需要策略记住过去的动作或环境状态（例如，物体A的初始位置、自己是否已经完成了“触摸”这个子任务）。
- **为什么重要**：记忆是智能体在复杂、长时程任务中做出正确决策的基础。如果机器人无法记住自己做过什么或物体在哪里，就无法完成多步骤、有状态依赖的任务。这严重限制了VLA模型在真实世界中的应用潜力。
- **挑战在哪里**：
    1.  **空间记忆缺失**：模型需要记住物体在场景中的初始位置，并在物体被移动后仍能回忆起该位置。
    2.  **时间记忆缺失**：模型需要记住自己已经完成了哪些子任务，以及接下来应该做什么。例如，在“交换”任务中，模型需要区分自己正处于“拿起物体A”还是“将物体A放到B的初始位置”的阶段。
    3.  **现有方法的局限**：简单的循环网络（RNN）虽然具有内在记忆，但在长时程任务中训练困难（梯度消失/爆炸、GPU内存限制），且难以保留任务早期的关键信息。而基于固定大小上下文窗口的Transformer模型，其记忆能力受限于窗口大小。

## 2. 创新点（Innovation）

- **关键创新**：提出了**语言草稿本（Language Scratchpad）** 机制，这是一种为VLA模型赋予显式记忆的简单而有效的方法。
- **与现有方案的不同**：
    - 不同于修改模型架构（如添加记忆网络或使用循环结构），该方法**利用VLM本身的语言生成能力**，通过一个外部的、可读写的文本缓冲区来实现记忆。
    - 不同于简单的“历史帧堆叠”，该方法提供的是**语义化的、结构化的记忆**（如“红色三角形在[-16, 2, 2]位置”），而非原始的像素信息，这使得记忆更高效、更易于推理。
    - 不同于需要额外训练的记忆模块，该方法**与现有VLA模型高度兼容**，只需在训练和推理时调整输入输出格式即可。
- **为什么能奏效**：
    1.  **显式记忆**：草稿本提供了一个显式的存储空间，模型可以随时写入和读取关键信息，避免了信息在隐式状态中衰减或丢失。
    2.  **语义表达**：语言是一种高度灵活和丰富的表达方式，可以轻松编码空间坐标（如“物体在[10, 20, 30]”）、任务状态（如“已完成子任务1”）和行动计划（如“计划：先触摸，再重置”）。
    3.  **自我监督**：模型通过预测“想法”（描述）来生成草稿本内容，这个过程可以看作是模型对自身行为的“自我解释”和“计划”，有助于模型更好地理解和执行任务。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

论文提出的“草稿本增强型VLA”架构如图1所示。其核心思想是在标准的VLA模型基础上，增加一个**语言草稿本（Scratchpad）** `S_t`。

- **输入**：在每一步 `t`，模型的输入不再是 `(o_t, l)`，而是 `(o_t, S_t, l)`。其中 `o_t` 是当前观测，`l` 是初始语言指令，`S_t` 是到当前步为止累积的草稿本内容。
- **输出**：模型输出动作 `a_t` 和一个新的语言描述 `d_t`。这个 `d_t` 可以是“想法”（如“正在拾取红色三角形”）或“状态更新”（如“已完成”）。
- **更新机制**：草稿本 `S_t` 会随着任务的进行而动态更新。更新逻辑如算法1所示。当模型预测出一个特殊的更新标记（如 `<done>` 标记）时，当前的描述 `d_t` 就会被追加到草稿本中，形成新的 `S_{t+1}`。否则，草稿本保持不变。
- **草稿本结构**：论文将草稿本设计为三个部分（如图1所示）：
    - **Grounding（定位）**：记录任务的初始化条件，如物体和机械臂的初始位置（**空间记忆**）。
    - **Plan（计划）**：列出完成高级指令所需的子任务序列。
    - **Act（动作）**：追踪已经完成的子任务，记录当前进度（**时间记忆**）。

### 3.2 关键算法与数学推导

**算法1：草稿本更新逻辑**

该算法描述了模型如何与草稿本交互。

```pseudocode
Algorithm 1 Scratchpad Update Logic
Require: p: The policy. o_t: The observation at timestep t. S_t: The scratchpad at timestep t. l: The high-level language instruction or goal.
1: function FORWARDPASS(p, o_t, S_t, l)
2:   (a_t, d_t) ← p(o_t, S_t, l)  ▷ Generate action and description
3:   if d_t ∈ d_update then        ▷ Check if the description triggers an update
4:     S_{t+1} ← S_t ∪ d_t        ▷ Update scratchpad by adding the description
5:   else
6:     S_{t+1} ← S_t              ▷ Keep scratchpad unchanged
7:   end if
8:   return (a_t, S_{t+1})        ▷ Return the action and the new scratchpad
9: end function
```

**数学形式化**

- **标准VLA模型**：$p(a_t | o_t, l)$
- **草稿本增强VLA模型**：$p(a_t, d_t | o_t, S_t, l)$
    - 其中，$S_t = \{d_1, d_2, ..., d_n\}$ 是到时间步 $t$ 为止所有被记录下来的描述 $d$ 的集合。
    - 模型不仅预测动作 $a_t$，还预测一个描述 $d_t$。这个 $d_t$ 可能被用于更新草稿本。

### 3.3 模型、训练与数据集

- **模型架构**：
    - **T-VLA（Transformer VLA）**：基于PaliGemma-2（3B参数）VLM。输入是单帧图像、指令和线性化的草稿本。动作被离散化为256个bin，作为词汇表中的token。
    - **R-VLA（Recurrent VLA）**：基于Mamba（130M参数）语言模型和ViT视觉编码器。它将整个轨迹（包括观测、指令、草稿本、动作）序列化为一个长文本序列，进行下一个token预测。动作通过一个MLP头从`<action>` token的嵌入中回归得到。
- **训练**：
    - **T-VLA**：全参数微调，学习率 $2 \times 10^{-5}$，batch size 8。
    - **R-VLA**：全参数微调，学习率 $10^{-5}$，batch size 4。
    - 优化器均为Adam。
- **数据集**：
    - **ClevrSkills-Mem**：从ClevrSkills基准中划分出的5个记忆依赖任务，包含空间和时间记忆的不同组合。使用环境自带的oracle求解器生成包含子任务分割的轨迹，用于构建草稿本。
    - **MemoryBench**：使用其公开的100条Put-Block-Back任务轨迹。
    - **真实世界任务**：自收集的200条Pick-Place-Restore任务轨迹，通过遥操作UFACTORY xArm 6机械臂完成。

### 3.4 实验设置与结果

- **实验设置**：
    - 动作空间：7维向量（6自由度末端执行器位姿增量 + 1维夹爪状态）。
    - 图像：统一缩放至224x224。
    - 评估：ClevrSkills-Mem上对每个任务进行50次未见过的初始位置rollout；MemoryBench上使用25条测试轨迹；真实世界任务上进行20次随机初始位置rollout。

- **主要结果**：
    1.  **ClevrSkills-Mem（图3）**：
        - **T-VLA**：在所有任务上表现挣扎，成功率极低。
        - **T-VLA + Scratchpad**：性能大幅提升，平均成功率提升**48.8%**。在Touch-Reset-Pick、Swap和Place-Next-to-Restore任务上提升显著（68%-72%）。但在需要细粒度时间记忆的Rotate-Restore任务上仍失败。
        - **R-VLA**：由于内在记忆，性能优于T-VLA。
        - **R-VLA + Scratchpad**：性能进一步提升，平均提升**11%**。说明即使对有记忆的模型，显式的草稿本也有帮助。
    2.  **MemoryBench（表I）**：
        - **T-VLA**：在Put-Block-Back任务上成功率为0%。
        - **T-VLA + Scratchpad**：成功率达到40%。在放宽评估标准（“sim-eval”，即模拟按压按钮）后，成功率可达100%，证明模型的空间和时间记忆能力完美，失败主要源于精确操作（按压小按钮）的困难。
    3.  **真实世界任务（表II）**：
        - **OpenVLA**：成功率为0%，无法区分“放置”和“恢复”这两个相反的子任务。
        - **OpenVLA + Scratchpad**：成功率达到**65%**，平均子任务完成率为2.4/3，证明了其在真实世界中的有效性。

### 3.5 技术细节

- **特殊Token**：使用 `<plan>...</plan>` 包裹定位和计划部分；使用 `<think>` 标记模型预测的“想法”；使用 `<act>` 标记动作的开始；使用 `<done>` 标记子任务的完成，触发草稿本更新。
- **R-VLA的动作预测**：由于Mamba模型在离散化动作上表现不佳，论文采用了一个MLP作为动作头。模型预测一个 `<action>` token，其对应的嵌入向量被送入MLP，回归出7维连续动作值。
- **数据生成**：利用ClevrSkills环境中的oracle求解器，可以自动获取子任务分割和物体位置信息，从而自动生成包含 `Grounding`、`Plan` 和 `Act` 部分的草稿本数据，无需人工标注。

## 4. 意义与影响（Significance and Impact）

- **解决了什么问题**：解决了当前VLA模型在处理需要空间和时间记忆的非马尔可夫操作任务时的核心缺陷。通过一个简单的语言草稿本，将“无状态”模型转变为“有状态”模型。
- **贡献是什么**：
    1.  提出了一种新颖、简单、通用的语言草稿本记忆机制。
    2.  创建了ClevrSkills-Mem基准，用于系统评估机器人策略的记忆能力。
    3.  在仿真和真实世界任务上验证了该方法对非循环和循环VLA模型均有显著提升。
- **对未来研究/应用的影响**：
    - 该方法为增强VLA模型的记忆能力提供了一个轻量级、即插即用的解决方案，有望被广泛应用于各种机器人操作任务。
    - 它展示了利用语言作为通用记忆介质的潜力，可以启发未来将更复杂的推理、规划和记忆机制（如Chain-of-Thought）与VLA模型结合。
    - 该方法与最新的推理模型（如Embodied Chain-of-Thought）兼容，可以协同工作。
- **作者提到的局限与开放问题**：
    - 该方法在需要**细粒度、低层级时间记忆**的任务（如Rotate-Restore）上效果不佳，因为语言草稿本难以精确追踪连续的微小状态变化（如旋转角度）。
    - 草稿本的设计（如结构、内容）目前是人为设定的，未来可以探索如何让模型**自动学习**生成更有效的草稿本。
    - 真实世界实验的数据量（200条轨迹）相对较小，限制了模型的泛化能力（如恢复位置精度不高）。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比：游戏中的“任务日志”**：可以将语言草稿本想象成一个角色扮演游戏（RPG）中的任务日志。当你接到一个复杂任务（如“去A地取剑，然后去B地杀龙”），日志会记录下任务目标和步骤。当你完成“取剑”后，日志会更新为“已完成：取剑，下一步：去B地杀龙”。机器人策略就像玩家，它不再需要记住所有事情，只需随时查看日志（草稿本）就能知道“我在哪”、“我做了什么”、“下一步该做什么”。这个日志就是显式的、结构化的记忆。
- **为什么“无状态”模型会失败**：想象一个机器人要执行“交换两个杯子”的任务。它先拿起左边的红杯。此时，它的“大脑”里只有“拿起红杯”这个动作的记忆。当它放下红杯后，它的下一个动作取决于当前视觉输入。如果它看到右边有个蓝杯，它可能会直接去拿蓝杯，而忘记了它的真正目标是“把红杯放到蓝杯的位置，再把蓝杯放到红杯的位置”。因为它没有记录“红杯的初始位置”和“我已经完成了第一步”这些信息。草稿本就像一个便利贴，帮它记下了这些关键信息。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - 与**SAM2Act**相比，SAM2Act使用了一个专门的记忆库和注意力机制，而本文的方法更简单，直接利用语言。本文在MemoryBench上的实验表明，其方法在记忆能力上可以媲美甚至超越这种专门设计的方法（在sim-eval下达到100%）。
    - 与**Embodied Chain-of-Thought**（如Robotic Control via Embodied Chain-of-Thought Reasoning）相比，本文的草稿本不仅包含推理过程，还显式地存储了空间状态和任务进度，更侧重于记忆而非推理。
    - 与**循环策略**（如RoboMamba, ClevrSkills）相比，本文证明了即使对于有内在记忆的循环网络，一个显式的、外部的语言记忆仍然能带来显著的性能提升，尤其是在长时程任务中。
