## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人学习中的视觉世界模型（Visual World Models），特别是应用于受隐藏环境漂移（如水流、风力）影响的物体（如水面航行器）的运动预测与规划。
- **问题（Problem）**：现有的大多数动作条件世界模型假设运动主要由即时控制决定，但在现实世界中（如水面航行器），物体因惯性（动量）而持续运动，并受到不可观测的环境漂移（如水流）影响。从单帧图像中无法推断这些隐藏状态，导致长时域预测不准确。
- **方法（Method）**：提出 FlowMo-WM，一个端到端可训练的视觉世界模型。它通过两个时间分支将图像-动作历史分解为：一个短时历史隐状态（用于总结物体中心运动，如动量）和一个长时历史上下文（用于总结缓慢变化的外源影响，如环境漂移）。在隐空间推演中，使用一个“零上下文残差转移”机制，将动作条件的基础动力学与上下文相关的漂移效应分离开。
- **创新（Innovation）**：1) 提出了一种新的视觉世界模型公式，从历史数据中推断物体动量和环境输运，无需直接监督流场。2) 设计了短/长时历史分解与零上下文残差转移架构，有效分离了可控动力学与不可控漂移。3) 在包含多种隐藏流场、干扰和随机化动力学的水面航行器模拟环境中，展示了预测和规划性能的提升。
- **意义（Significance）**：这项工作强调了在开放物理环境中，世界模型不仅需要建模受控动作，还必须考虑物体的持续运动和不可观测环境的影响。它为在受水流、风力等干扰的机器人（如水面/空中机器人）上应用世界模型提供了新的思路和有效方法。
- **一句话总结**：FlowMo-WM 通过将图像-动作历史分解为短时动量状态和长时漂移上下文，并利用零上下文残差转移，显著提升了在隐藏环境漂移影响下的长时域预测与规划精度。

## 1. 研究动机（Motivation）

- **问题与空白**：在机器人学习领域，世界模型（World Models）通过学习从视觉观测和动作中预测未来状态，使智能体能够进行规划和决策。然而，许多现有的动作条件世界模型（如Dreamer、TD-MPC）在评估时，其环境设定中运动主要由即时动作主导。但在许多真实世界场景中，例如水面航行器（ASV）、空中机器人（UAV）或任何漂浮物体，其运动不仅受当前推力控制，还受到**物体自身动量（惯性）** 和**隐藏的环境漂移（如水流、风力）** 的共同影响。这些关键状态（速度、环境流场）无法从单张图像中直接观测，导致预测问题本质上是部分可观测的（Partially Observable）。
- **重要性**：解决这个问题对于将世界模型成功应用于真实世界的机器人至关重要。例如，水面航行器在规划路径时，如果无法准确预测水流对其未来位置的影响，规划出的轨迹可能完全无效，甚至导致任务失败。长时域预测的微小误差会因动量和漂移的存在而迅速累积放大。
- **挑战**：挑战在于如何仅从**干净的图像和动作历史**中，端到端地、无监督地推断出物体的动量状态和隐藏的环境漂移。模型不能依赖任何特权信息，如速度传感器、流场图或仿真器内部状态。这要求模型具备从历史观测序列中提取和分离不同时间尺度动态信息的能力。

## 2. 创新点（Innovation）

- **关键创新**：论文的核心创新在于提出了一种**时间分解（Temporal Factorization）** 和**零上下文残差转移（Zero-Context Residual Transition）** 的架构。
- **与现有方案的不同**：
    1.  **时间分解**：不同于大多数使用单一循环状态（如RSSM）或仅依赖当前帧的模型，FlowMo-WM 显式地将图像-动作历史分解为两个部分：
        - **短时历史状态（$z_t$）**：从最近几帧（K=8）中提取，用于捕捉与物体中心运动相关的快速变化状态，如位姿、速度、执行器延迟等。
        - **长时历史上下文（$c_t$）**：从更长但稀疏的历史（32步，步长4）中提取，用于捕捉缓慢变化的外源影响，即环境漂移。
    2.  **零上下文残差转移**：在隐空间的状态转移中，模型不是简单地将 $z_t$、$c_t$ 和动作 $a_t$ 一起输入一个网络。而是设计了一个残差结构：
        $$z_{t+1} = z_t + F_\theta(z_t, a_t) + R_\theta(z_t, a_t, c_t) - R_\theta(z_t, a_t, 0)$$
        其中 $F_\theta$ 建模基础的动作条件动力学（无漂移时的运动），$R_\theta$ 建模上下文相关的漂移效应。当上下文 $c_t$ 为0时，残差项 $R_\theta(z_t, a_t, c_t) - R_\theta(z_t, a_t, 0)$ 恰好为0，使得模型退化为纯基础动力学。这提供了一个清晰的消融实验手段，并强制模型将漂移效应作为对基础动力学的修正来学习。
- **为什么能奏效**：这种分解和残差结构非常符合物理直觉。物体的运动可以看作是“自身控制”和“环境外力”的叠加。短时状态负责“自身控制”部分，长时上下文负责“环境外力”部分。零上下文残差确保了模型在无漂移时也能正常工作，并且使得上下文的作用更加明确和可解释。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

FlowMo-WM 的整体架构如图1所示，主要包括以下部分：

1.  **共享视觉编码器（Shared Visual Encoder）**：接收一张160x160的俯视RGB图像，并输出一个图像嵌入。编码器使用CoordConv（坐标卷积）将像素坐标信息融入特征，然后通过4层卷积、自适应池化和线性投影得到嵌入向量。
2.  **短时历史分支（Short-History Branch）**：使用一个GRU网络处理最近 $K_z=8$ 步的图像嵌入和动作序列，输出物体运动状态 $z_t \in \mathbb{R}^{112}$。
3.  **长时历史分支（Long-History Branch）**：使用另一个GRU网络处理一个32步长但步长为4的历史序列（即覆盖128步的历史），输出环境上下文 $c_t \in \mathbb{R}^{8}$。
4.  **零上下文残差转移（Zero-Context Residual Transition）**：根据公式(1)进行隐状态推演。$F_\theta$ 和 $R_\theta$ 是两个小型MLP网络。
5.  **位姿预测头（Pose Prediction Head）**：一个MLP，将隐状态 $z_t$ 映射为归一化的未来位姿 $\hat{y}_t = (\hat{x}, \hat{y}, \cos\hat{\theta}, \sin\hat{\theta})$。

### 3.2 关键算法与数学推导

核心是隐空间的状态转移公式：

$$z_{t+1} = z_t + F_\theta(z_t, a_t) + R_\theta(z_t, a_t, c_t) - R_\theta(z_t, a_t, 0)$$

- **$z_t$**：当前时刻的物体运动状态。
- **$a_t$**：当前时刻执行的动作。
- **$c_t$**：当前时刻的环境上下文。
- **$F_\theta(z_t, a_t)$**：基础动力学函数，预测在没有环境漂移时，状态 $z_t$ 在动作 $a_t$ 下的变化量。
- **$R_\theta(z_t, a_t, c_t)$**：漂移残差函数，预测在给定上下文 $c_t$ 时，状态 $z_t$ 在动作 $a_t$ 下的额外变化量。
- **$R_\theta(z_t, a_t, 0)$**：当上下文为零时的漂移残差。减去这一项确保了当 $c_t=0$ 时，整个残差项为零，模型退化为纯基础动力学。

训练损失函数是一个多步位姿推演损失：

$$\mathcal{L} = \sum_{h=1}^{H} \left( \lambda_p \|\hat{p}_{t+h} - p_{t+h}\|_2^2 + \lambda_\theta \|\hat{r}_{t+h} - r_{t+h}\|_2^2 \right)$$

- $H=60$：预测时域长度。
- $\hat{p}_{t+h}$ 和 $p_{t+h}$：预测和真实的位置 $(x, y)$。
- $\hat{r}_{t+h}$ 和 $r_{t+h}$：预测和真实的朝向表示 $(\cos\theta, \sin\theta)$。
- $\lambda_p=1, \lambda_\theta=2$：位置和朝向损失的权重。

### 3.3 模型、训练与数据集

- **模型**：FlowMo-WM 总参数量为 663,964。对比模型包括 LeWorldModel-style (664,612)、PlaNet/RSSM-style (664,644) 和 TD-MPC2-style (667,780)，参数量相近，确保公平比较。
- **训练**：使用 AdamW 优化器，学习率 $3\times10^{-4}$，batch size 256，训练 20,000 步。训练数据来自一个2D水面航行器模拟器。
- **数据集**：
    - **模拟器**：2D水面航行器模拟器，包含10种不同的静态流场（无流、均匀流、涡流、双环流等），以及随机化的船体形态（双推进器船、三角推进器船）、物理参数（质量、惯性、阻力等）。
    - **训练集**：2,400 个episode，每个episode 300步，共 720,000 个转移。
    - **测试集**：480 个episode，共 144,000 个转移。所有测试数据与训练数据完全独立（不同初始条件、参数、动作序列和流场实例）。
    - **输入**：模型仅接收 160x160 的干净俯视RGB图像和动作历史。图像中不包含流场箭头、目标点、速度向量等任何辅助信息。

### 3.4 实验设置与结果

- **实验设置**：
    - **预测任务**：开环预测。给定历史图像-动作和未来60步的真实动作，预测未来60步的位姿。主要指标是位置误差。
    - **规划任务**：采样式规划。模型作为模拟器，对512个候选动作序列进行45步推演，根据预测结果评分，选择最优序列执行。任务包括：到达目标、定点保持、方形航点、Z字形航点。
    - **消融实验**：在预测时，将上下文 $c_t$ 置零或随机打乱，观察预测误差的变化。
    - **探针分析**：训练后，使用线性探针（Linear Probes）从 $z_t$、$c_t$ 等隐变量中解码特权信息（如动量、局部流场），分析模型学到了什么。
- **主要结果**：
    1.  **预测精度**（图4A）：FlowMo-WM 在所有时域上均优于对比模型。在60步时，位置误差为0.344，而最好的对比模型（PlaNet/RSSM）为0.426，误差降低了约19%。
    2.  **规划成功率**（图4B, 4C, 表1）：FlowMo-WM 在规划任务中取得了最高的成功率（98.5%），显著高于其他学习模型（91.0%-93.1%）。
    3.  **上下文消融**（第5节）：将上下文 $c_t$ 置零后，60步位置误差从0.344急剧上升到0.840；打乱上下文后，误差进一步上升到1.100。这强烈证明了长时历史上下文对于稳定预测至关重要。
    4.  **探针分析**（第5节）：动量可以从 $[z_t, c_t]$ 中以较高的 $R^2=0.667$ 线性解码，而局部流场的线性 $R^2$ 接近0。这表明上下文包含的是对预测有用的非线性编码信息，而非简单的物理量存储。

### 3.5 技术细节

- **CoordConv**：在输入图像上叠加归一化的x和y坐标图，帮助卷积网络感知空间位置。
- **GRU**：门控循环单元，用于处理序列数据，提取时间特征。
- **位姿表示**：使用 $(\cos\theta, \sin\theta)$ 表示朝向，避免了角度周期性带来的问题。
- **归一化**：位置坐标从 $[0,10]$ 归一化到 $[0,1]$ 或类似范围，以利于训练。
- **零上下文残差**：设计 $R_\theta(z_t, a_t, 0)$ 项是为了使零上下文时的残差项严格为零，这比简单地不输入上下文更干净、更可解释。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了现有视觉世界模型在受隐藏环境漂移影响场景下预测不准的核心问题。
    - 提出了一种新颖、有效且可解释的架构（时间分解 + 零上下文残差），为处理部分可观测的、多时间尺度的动力学问题提供了新范式。
    - 通过详尽的实验（预测、规划、消融、探针）验证了方法的有效性，并深入分析了模型内部表征。
- **对未来研究/应用的影响**：
    - **机器人领域**：为水面航行器、水下机器人、空中机器人等受环境干扰的机器人应用世界模型进行规划和控制铺平了道路。
    - **世界模型设计**：启发未来的世界模型设计应考虑显式地建模不同时间尺度的动态因素，而不仅仅是增加循环网络的容量。
    - **无监督表征学习**：展示了如何从视觉-动作历史中无监督地学习出具有物理意义（如动量）的隐变量。
- **局限与开放问题**：
    - **现实世界泛化**：当前实验基于简化的2D仿真和干净图像。扩展到真实视频需要处理视角变化、遮挡、传感器噪声等挑战。
    - **极端流场**：当环境漂移强度超过机器人控制能力时，模型仍会失败。对训练分布外的流场泛化能力有限。
    - **上下文更新**：当前上下文在开环推演中是固定的。未来工作可以探索在闭环执行中在线更新上下文，以应对动态变化的环境。
    - **更丰富的探针**：可以使用更复杂的探针（如非线性探针）或基于真实流场标签的探针，来更精确地量化上下文编码了多少物理漂移信息。

## 5. 澄清与简化（Clarifications and Simplifications）

- **类比理解“时间分解”**：想象你在一个移动的火车上（环境漂移）走路（自身动作）。你的总运动是“走路”和“火车移动”的叠加。短时历史状态 $z_t$ 就像你最近几步的步态和方向，它决定了你相对于火车的运动。长时历史上下文 $c_t$ 就像你通过观察窗外景物（历史图像）推断出的火车速度和方向。FlowMo-WM 就是学会了如何从你的“走路记录”和“窗外景物变化”中，分离出你自身的运动（$F_\theta$）和火车的运动（$R_\theta$）。
- **类比理解“零上下文残差”**：假设你有一个预测“总位移”的函数。零上下文残差的设计相当于把这个函数拆成了两部分：一个“基础位移预测器”（$F_\theta$），它假设火车是静止的；和一个“火车位移修正器”（$R_\theta$）。当你知道火车在动（$c_t$ 非零）时，就加上修正。当你不知道火车是否在动（$c_t=0$）时，修正器自动输出0，预测就退回到“火车静止”的假设。这保证了模型在无漂移环境下也能正常工作，并且修正项的作用非常明确。
- **为什么用“位姿预测”而不是“图像预测”**：预测未来图像（如Dreamer）计算量大且包含大量与任务无关的细节（如背景纹理）。FlowMo-WM 直接预测任务相关的位姿（位置和朝向），这使得模型更高效、更专注于核心动力学，并且训练目标（位姿误差）与最终任务（规划）更一致。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - 与 **ContextWM** [18] 类似，都试图从历史中推断上下文。但 FlowMo-WM 的上下文专门用于建模缓慢变化的漂移，并通过零上下文残差与基础动力学分离，结构更简洁、目标更明确。
    - 与 **LeWorldModel** [24] 类似，都是端到端训练的隐空间预测模型。但 FlowMo-WM 引入了时间分解来处理部分可观测性，而 LeWorldModel 主要关注单帧到单帧的预测稳定性。
    - 与 **PlaNet/RSSM** [2, 3, 4] 相比，RSSM 使用一个单一的循环状态来编码所有历史信息，而 FlowMo-WM 显式地将历史分解为不同时间尺度的两个部分，更有利于学习不同动态。
