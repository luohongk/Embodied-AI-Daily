# The Poisoned Chalice of LLM Evaluation Report

- arXiv: https://arxiv.org/abs/2607.07481v1
- arXiv ID: 2607.07481

---

🔍 **核心问题**：大型语言模型在软件工程评估中可能因训练数据泄露导致性能虚高，但多数模型训练语料不公开，难以直接检测数据污染。

🛠 **方法**：举办首届“LLM评估毒杯”竞赛，将污染检测建模为源代码上的白盒成员推断任务，提供数据集、目标模型和基线攻击方法，并在未公开模型和数据集上进行最终评估。

✨ **主要贡献**：提出一种无需完整训练语料即可检测模型数据暴露的实用方法，并通过竞赛设计推动泛化性强的检测技术，促进软件工程领域可信的LLM评估。
