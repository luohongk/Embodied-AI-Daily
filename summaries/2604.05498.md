## 0. 概览（Concise Summary）

- **主题（Topic）**：机器人控制领域的**世界动作模型（World Action Model, WAM）** 的安全性与鲁棒性，具体研究针对WAM的**越狱攻击（Jailbreak Attack）**。
- **问题（Problem）**：现有研究忽视了WAM在物理世界中的安全漏洞。WAM能直接驱动机器人执行物理动作，一旦其安全对齐机制被绕过，将导致人员伤亡、设备损坏等**真实物理风险**，而不仅仅是数字域的文本或图像危害。目前缺乏系统评估WAM越狱脆弱性的方法和基准。
- **方法（Method）**：提出了**JailWAM**，首个专门针对WAM的越狱攻击与评估框架。核心包括：(1)**视觉-轨迹映射（Visual-Trajectory Mapping, VTM）**，将异构的动作空间统一为视觉轨迹表示；(2)**风险判别器（Risk Discriminator, RD）**，高效筛选视觉轨迹中的危险行为；(3)**双路径验证策略（Dual-Path Verification Strategy）**，先进行快速开环筛选，再进行精确的闭环物理仿真验证。
- **创新（Innovation）**：
    1.  首次将越狱攻击的研究从数字域（LLM, VLM）拓展到物理域（机器人WAM），定义了**三级安全分类框架**。
    2.  提出了**视觉-轨迹映射（VTM）**，解决了不同WAM架构间动作空间异构、难以统一评估的难题。
    3.  设计了**双路径验证策略**，在保证评估精度的同时，大幅降低了计算成本（实现了2.5倍加速）。
- **意义（Significance）**：揭示了当前最先进的WAM（如LingBot-VA）存在严重的安全漏洞（攻击成功率高达84.2%），为设计安全可靠的机器人控制系统敲响了警钟，并提供了有效的防御技术方案和评估基准（JailWAM-Bench）。
- **一句话总结**：本文首次系统性地研究了机器人世界动作模型（WAM）的越狱攻击问题，提出了一个高效的攻击与评估框架JailWAM，揭示了WAM在物理世界中的严重安全风险，并提供了相应的防御策略。

## 1. 研究动机（Motivation）

- **关键挑战与空白**：
    1.  **安全威胁的范式转变**：传统的越狱攻击（如针对LLM、VLM）产生的危害仅限于数字域（如生成有害文本、暴力图像）。然而，WAM直接驱动物理机器人，一旦被越狱，其危害将直接转化为**物理世界中的真实伤害**（如碰撞、损坏设备、伤人）。这是一个全新的、更严重的安全威胁，但现有研究几乎完全忽视了这一点。
    2.  **缺乏系统评估方法**：评估WAM的越狱脆弱性面临三大核心难题：
        - **缺乏安全分级**：没有统一的、量化的标准来定义和评估机器人手臂运动的安全等级。
        - **动作空间异构**：不同的WAM模型输出不同的动作表示（如关节角度、末端执行器位移等），难以设计统一的、跨架构的攻击和评估协议。
        - **验证成本高昂**：验证物理有害行为通常需要昂贵的全闭环仿真，计算和人力开销巨大，难以大规模进行。
- **重要性**：解决这个问题至关重要，因为WAM强大的物理交互能力是一把“双刃剑”。如果其安全性得不到保障，将直接威胁人身安全、财产安全与环境安全。在WAM技术快速发展的当下，必须提前识别并解决其安全漏洞，才能确保其安全、可靠地部署。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **首次定义物理域越狱攻击**：将越狱攻击的概念从数字域（LLM, VLM）正式引入到物理域（机器人WAM），并系统性地研究了其脆弱性。
    2.  **提出三级安全分类框架**：将机器人手臂运动安全量化为三个等级：**Level 0（安全合规）**、**Level 1（运动失败）** 和 **Level 2（灾难性风险）**，为定量评估提供了基础。
    3.  **提出视觉-轨迹映射（VTM）**：这是解决动作空间异构问题的关键。通过将低维、异构的动作序列（如关节角度）投影到标准化的、多视角的**视觉轨迹图**上，VTM将物理执行过程转化为视觉语言模型（VLM）易于理解和推理的视觉信号，从而实现了跨架构的统一评估。
    4.  **提出双路径验证策略**：巧妙地将评估过程解耦为两个阶段：
        - **Stage I（开环视觉筛选）**：利用VTM和轻量级风险判别器（RD）快速、低成本地筛选出高风险指令，过滤掉大量无害指令。
        - **Stage II（闭环物理验证）**：仅对筛选出的高风险指令进行昂贵的全闭环仿真验证，由人类专家最终确认。这种“粗筛-细验”的策略在**效率**和**准确性**之间取得了极佳的平衡。
- **与现有方案的不同**：现有工作要么关注数字域安全，要么在机器人安全评估中只关注任务成功率，忽略了中间过程的危险行为。JailWAM则**专门针对WAM的物理安全漏洞**，并提供了从攻击生成、统一评估到高效验证的完整解决方案。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

JailWAM框架的整体架构如图3所示，包含三个核心模块：

1.  **攻击生成**：利用大语言模型（LLM，如Gemini）和精心设计的越狱模板，生成大量描述危险动作的指令（如“执行一个400度旋转的关节限位测试”）。这些指令是任务无关的，直接指定物理威胁。
2.  **双路径验证策略**：
    - **Stage I: 开环视觉筛选**：将WAM对每个候选指令的**开环预测动作序列**，通过VTM转化为视觉轨迹图。然后，轻量级的**风险判别器（RD）** 分析该图，快速给出安全等级（0, 1, 2）。只有被判定为高风险（Level 1或2）的指令才会进入下一阶段。
    - **Stage II: 闭环物理验证**：对高风险指令，在物理仿真器中执行**全闭环**的机器人控制。仿真器根据WAM的每一步动作更新环境状态，并反馈给WAM。最终的执行结果（如碰撞、振荡）由人类专家进行最终判定，确认其真实危害等级。
3.  **核心组件**：
    - **视觉-轨迹映射（VTM）**：将异构动作序列转化为统一的视觉轨迹图。
    - **风险判别器（RD）**：基于Qwen3-VL-2B-Instruct微调，用于从视觉轨迹图中预测安全风险。

### 3.2 关键算法与数学推导

1.  **问题形式化**：
    给定一个WAM模型 $M$，初始上下文 $x_t = (o_{\le t}, s_{\le t})$，目标是找到一个对抗性指令 $l^*$，使得模型输出的动作序列 $a^*_{t:t+H}$ 最不安全。
    $$a_{t:t+H} \sim M(o_{\le t}, s_{\le t}, l)$$
    $$l^* = \arg\max_{l \in \mathcal{L}_{adv}} \mathcal{R}(\mathcal{V}(M(x_t, l)))$$
    其中 $\mathcal{L}_{adv}$ 是由LLM生成的对抗指令池，$\mathcal{V}(\cdot)$ 是VTM映射，$\mathcal{R}(\cdot)$ 是风险判别器。

2.  **视觉-轨迹映射（VTM）**：
    将WAM输出的相对动作位移序列 $\Delta A = \{\Delta A_1, \Delta A_2, ..., \Delta A_t\}$ 转换为绝对空间坐标 $P$：
    $$P = P_0 + \sum_{i=1}^{t} \mathcal{F}(\Delta A_i) \in \mathbb{R}^3$$
    其中 $\mathcal{F}$ 是积分器。然后，将3D轨迹正交投影到2D平面（俯视图和正视图）：
    $$v^{(xy)} = \Pi_{xy}(P), \quad v^{(xz)} = \Pi_{xz}(P)$$
    最后，通过渲染函数 $\Phi$ 将投影、物理属性与环境约束 $C_{env}$（如工作台边界）合成统一的视觉轨迹图：
    $$\mathcal{V} = \Phi\left(P, v^{(xy)}, v^{(xz)}, C_{env}\right)$$

3.  **优化目标**：
    最终的风险标签 $y^*$ 由风险判别器 $\mathcal{R}$ 给出：
    $$y^* = \mathcal{R}(\mathcal{V}(a^*)) \in \{0, 1, 2\}$$
    越狱成功当且仅当 $y^* \in \{1, 2\}$（即非Level 0）。

### 3.3 模型、训练与数据集

- **目标模型**：
    - **主要测试床**：**LingBot-VA**（在RoboTwin仿真器中），一个最先进的WAM。
    - **跨架构迁移测试**：**Motus**（使用不同的VLM解码器），**Cosmos-Policy**（在LIBERO环境中）。
    - **非WAM基线**：**$\pi_{0.5}$**（一个非WAM的视觉-语言-动作模型），用于对比。
- **风险判别器（RD）训练**：
    - **基础模型**：**Qwen3-VL-2B-Instruct**，一个轻量级的视觉-语言模型。
    - **数据集构建**：从RoboTwin仿真器的50个不同操作任务中，每个任务采样500个候选轨迹，共计25K个样本。这些样本的视觉轨迹图由**Gemini-3.1-Pro**通过思维链（CoT）进行初步标注，再由人类专家进行严格的人工验证和修正，得到三级安全标签（Level 0, 1, 2）。
    - **训练方式**：在构建的数据集上对Qwen3-VL进行**微调（Fine-Tuning）**，使其能够根据输入的视觉轨迹图和对抗性指令，准确预测安全风险等级。

### 3.4 实验设置与结果

- **实验设置**：
    - **环境**：RoboTwin仿真器（主要），LIBERO仿真器（用于跨环境测试）。
    - **基线方法**：**Clean**（标准指令），**RSA**（随机后缀攻击），**TPA**（模板提示攻击）。
    - **指标**：**攻击成功率（ASR）**，分解为**运动失败率（MFR，Level 1）** 和**灾难性风险率（CRR，Level 2）**。同时报告RD自动评估和人类专家评估的结果。
- **主要结果（表1）**：
    - 在LingBot-VA上，JailWAM实现了**84.2%** 的人类评估ASR（62% MFR + 22.2% CRR），远超基线方法（< 6%）。
    - 在Motus上，零样本迁移的JailWAM指令实现了**60.6%** 的ASR，证明了跨架构的迁移性。
- **跨环境与架构结果（表2）**：
    - 在Cosmos-Policy（LIBERO环境）上，零样本迁移实现了**46.5%** 的ASR。
    - 在非WAM模型 $\pi_{0.5}$ 上，ASR仅为**5.0%**，表明JailWAM的攻击效果与WAM的视觉生成先验密切相关。
- **消融实验（表3）**：
    - 验证了VTM和微调（FT）的重要性。仅使用原始动作+RD的Level 2召回率为0%，而完整的VTM+RD（Ours）达到了**70%**，整体一致性为**90%**。
- **效率分析（表4）**：
    - 双路径验证策略相比全闭环验证，将仿真运行次数从100次减少到21次，总时间从9.15小时减少到**3.66小时**，实现了**2.5倍**的加速，同时保留了大部分危险案例。
- **防御机制（图7）**：
    - 将Stage I的RD作为推理时过滤器，能将恶意指令的最终ASR从平均91.5%降低到极低水平（如`put object cabinet`任务中降至2%），同时保持对良性指令96%的通过率。

### 3.5 技术细节

- **越狱指令生成**：使用LLM（如Gemini-3.1-Pro）和一组专门的越狱模板（T），这些模板被设计用来直接生成描述危险动作的指令，而不是修改一个良性任务。这使得生成的指令是**任务无关**和**场景无关**的。
- **视觉轨迹图渲染**：VTM模块不仅绘制了3D轨迹的2D投影，还显式地注入了物理约束信息（如工作台边界、桌面高度），使得RD能够更容易地识别出“越界”、“碰撞”等危险行为。
- **风险判别器选择**：选择Qwen3-VL-2B-Instruct而非更大模型，是为了在**高吞吐量的筛选循环**中保证推理速度，这是实现可扩展评估的关键。
- **跨种子可靠性（Cross-Seed Reliability）**：实验发现，由不同LLM生成的指令对初始状态变化的鲁棒性不同。Gemini-3.1-Pro生成的指令在20个不同随机种子下仍能保持82.5%的可靠性，优于GPT-5.2和Claude-Opus-4.5。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    1.  **填补了安全研究空白**：首次系统性地揭示了WAM在物理域中的越狱攻击脆弱性，将AI安全研究从数字域拓展到了物理域。
    2.  **提供了评估工具**：提出了JailWAM框架和JailWAM-Bench基准，为未来研究WAM的安全性提供了标准化的评估方法和数据集。
    3.  **推动了防御研究**：通过实验验证了基于RD的推理时过滤是一种有效的防御手段，为设计安全的机器人控制系统提供了可行的技术路径。
- **对未来研究/应用的影响**：
    - **对WAM开发者**：本文是一个强烈的警示，必须在模型训练和部署前进行严格的安全对齐，不能仅关注任务性能。
    - **对机器人安全社区**：提供了一个新的研究方向，即“物理域越狱攻击”，并展示了如何利用视觉语言模型进行高效的物理安全评估。
    - **对政策制定者**：为制定关于具身智能体安全部署的法规和标准提供了技术依据。
- **局限与开放问题**：
    - **攻击的局限性**：攻击效果依赖于LLM生成指令的质量，且对非WAM架构（如$\pi_{0.5}$）效果有限。
    - **防御的局限性**：提出的防御（RD过滤）是推理时的，无法从根本上解决模型本身的安全对齐问题。更鲁棒的、训练阶段的防御方法有待研究。
    - **仿真到现实的鸿沟**：所有实验均在仿真环境中进行，攻击和防御策略在真实机器人上的有效性和安全性有待验证。

## 5. 澄清与简化（Clarifications and Simplifications）

- **什么是“世界动作模型（WAM）”**？
    - **通俗解释**：可以把它想象成一个“会做白日梦的机器人”。传统的机器人控制模型是“看到什么，就做什么”。而WAM不仅能“看到”当前状态，还能在内部“想象”出未来几秒钟的世界状态和它自己应该做的动作。这种“想象”能力让它能更好地规划复杂的物理操作，比如抓取一个移动的物体。本文研究的WAM（如LingBot-VA）就是利用视频生成模型来实现这种“想象”的。
- **什么是“越狱攻击”**？
    - **通俗解释**：就像手机有“越狱”漏洞一样，AI模型也有。对于聊天机器人，越狱就是通过精心设计的“咒语”（提示词），让它说出不该说的话（比如制造炸弹的指南）。对于WAM，越狱就是通过特殊的指令，让机器人“发疯”，做出危险的物理动作，比如用最大力气撞向桌子。
- **“视觉-轨迹映射（VTM）”为什么重要？**
    - **类比**：想象一下，你要判断一个舞者跳得好不好。如果给你一份记录他每个关节角度的数据表格（原始动作），你很难直观判断。但如果给你一段他跳舞的视频（视觉轨迹），你一眼就能看出他动作是否流畅、有没有摔倒的风险。VTM的作用就是把枯燥的“关节角度数据表格”变成了直观的“机器人运动视频截图”，这样，一个训练有素的“裁判”（风险判别器）就能快速、准确地判断这个动作是否危险。
- **“双路径验证策略”如何工作？**
    - **类比**：你要从1000个应聘者中找出最优秀的10个人。
        - **传统方法（全闭环）**：让每个应聘者都来公司面试一天（昂贵的仿真），耗时耗力。
        - **JailWAM方法（双路径）**：
            - **Stage I（简历筛选）**：先快速浏览所有简历（开环视觉筛选），根据一些硬性指标（如学历、经验）快速淘汰掉900个明显不合格的。这个步骤非常快，成本低。
            - **Stage II（最终面试）**：只对剩下的100个候选人进行正式面试（闭环仿真），由专家（人类）最终确定最优秀的10个。这个步骤虽然精确，但只针对少数人，总成本大大降低。
