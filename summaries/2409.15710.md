## 0. 概览（Concise Summary）

- **主题（Topic）**：双足机器人运动控制的参数自动调优与仿真到现实（Sim-to-Real）迁移。
- **问题（Problem）**：基于优化的双足运动控制器（如模型预测控制 MPC）通常需要大量、繁琐且依赖专家经验的手动参数调整。同时，在利用可微编程进行自动调参时，面临高保真度模型不可微、低保真度模型精度不足的两难困境。
- **方法（Method）**：提出一个结合 **DiffTune**（一种基于可微编程的模型自动调优方法）和 **GRFM-Net**（地面反作用力与力矩网络）的框架。使用低保真但可微的单刚体模型（SRBM）作为基础，通过 GRFM-Net 学习并补偿 MPC 指令与实际控制效果之间的差异，从而在保持可微性的同时提升模型保真度。
- **创新（Innovation）**：
    1.  首次将 **DiffTune** 应用于双足机器人运动 MPC 的参数自动调优，利用一阶梯度信息实现比贝叶斯优化更高效的参数学习。
    2.  提出 **GRFM-Net**，一种数据驱动的方法，用于弥合可微低保真模型与真实物理系统之间的“仿真到现实”差距，使得在仿真中学习的参数能有效迁移到真实硬件上。
- **意义（Significance）**：显著减少了双足机器人控制器参数调优所需的人工成本和专家依赖，提高了参数优化的效率和效果。通过 GRFM-Net 有效解决了 Sim-to-Real 迁移难题，为更复杂、更鲁棒的机器人控制策略的自动化开发铺平了道路。
- **一句话总结**：本文通过将可微自动调优方法 DiffTune 与数据驱动的保真度增强网络 GRFM-Net 相结合，高效地自动优化了双足机器人 MPC 控制器的参数，并成功实现了从仿真到真实硬件的有效迁移。

## 1. 研究动机（Motivation）

- **问题与空白**：双足机器人的运动控制，特别是基于优化的方法如模型预测控制（MPC），虽然能生成复杂的运动，但其性能高度依赖于大量控制参数（如成本函数中的权重矩阵 Q 和 R）的精细调整。这种调整过程通常需要专家经验，耗时且不直观，成为了该领域发展的一个瓶颈。
- **重要性**：解决参数自动调优问题，可以降低双足机器人控制算法的开发门槛，加速新算法的迭代，并使非专家用户也能部署高性能的机器人系统。
- **挑战**：
    1.  **参数空间维度高**：MPC 的参数（如 Q, R 矩阵）维度较高，传统的无模型优化方法（如贝叶斯优化）在高维空间中效率低下，搜索空间呈指数级增长。
    2.  **可微性与保真度的权衡**：基于梯度的自动调优方法（如 DiffTune）需要可微的动力学模型。高保真度的全阶动力学模型（包含接触、关节摩擦等）虽然精确，但通常不可微或难以求导。而低保真度的简化模型（如单刚体模型 SRBM）虽然可微，但精度不足，会导致“仿真到现实”差距，即仿真中学习到的最优参数在真实机器人上表现不佳。

## 2. 创新点（Innovation）

- **关键创新**：
    1.  **应用 DiffTune 于双足 MPC 调参**：首次将基于可微编程的模型自动调优方法 **DiffTune** 应用于双足机器人的线性 MPC 控制器参数学习。DiffTune 利用一阶梯度信息（通过反向传播计算损失函数对参数的梯度）来高效地更新参数，相比仅使用零阶信息的贝叶斯优化，收敛速度更快，更适用于高维参数空间。
    2.  **提出 GRFM-Net 弥合 Sim-to-Real 差距**：为了解决可微性与保真度的矛盾，提出了 **GRFM-Net**。这是一个数据驱动的神经网络，用于学习并补偿 MPC 输出的地面反作用力/力矩指令（$u$）与机器人实际感受到的控制效果（$\bar{u}$）之间的差异。这个差异源于执行器动力学、地面接触、机械传动等复杂且难以建模的因素。
- **与现有方案的不同**：
    - 与贝叶斯优化相比，DiffTune 利用梯度信息，参数搜索更高效。
    - 与直接使用全阶动力学模型进行可微编程相比，GRFM-Net 避免了处理不可微的接触模型和复杂的逆运动学求解，同时通过数据学习提升了低保真 SRBM 的精度。
- **为何能奏效**：GRFM-Net 作为一个可微的“校正器”，被嵌入到可微的 SRBM 仿真器中。这使得 DiffTune 可以在一个更接近真实物理系统的“增强仿真器”中进行参数优化。学习到的参数因此更能适应真实硬件的特性，从而有效缩小了 Sim-to-Real 差距。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

整体框架如图 1 所示，包含两个主要阶段：

1.  **GRFM-Net 训练阶段**：
    - 在高保真度仿真器（HFS）中，使用不同的参考轨迹让机器人行走。
    - 收集数据对：MPC 输出的原始 GRFM 指令（$u$）和 HFS 中传感器读取的真实 GRFM（$\bar{u}$）。
    - 使用这些数据训练 GRFM-Net，使其学会从 $u$ 预测 $\bar{u}$。

2.  **DiffTune 参数优化阶段**：
    - 将训练好的 GRFM-Net 嵌入到可微的 SRBM 仿真器中，形成“增强仿真器”。
    - 在增强仿真器中，使用 DiffTune 进行闭环参数学习。对于给定的参数 $\theta$，仿真器运行 MPC 控制器，产生轨迹，并计算损失函数 $L(\theta)$。
    - 通过自动微分计算损失对参数的梯度 $\nabla_\theta L$，并使用梯度下降法更新参数 $\theta$。
    - 学习到的最优参数 $\theta^*$ 最终部署到真实 HECTOR 机器人上。

### 3.2 关键算法与数学推导

**1. 单刚体模型 (SRBM) 动力学**：
MPC 使用线性化的 SRBM，其连续时间动力学方程为：
$$m(\ddot{p}_c + g) = \sum_{i=0}^{1} F_i$$
$$\frac{d}{dt}(G_I \omega) = \sum_{i=0}^{1} \{ r_{f_i} \times F_i + M_i \}$$
其中 $m$ 是质量，$p_c$ 是质心位置，$g$ 是重力，$G_I$ 是世界坐标系下的转动惯量，$\omega$ 是角速度，$F_i$ 和 $M_i$ 是脚 $i$ 上的地面反作用力和力矩，$r_{f_i}$ 是从质心到脚 $i$ 的向量。

**2. MPC 优化问题**：
MPC 在每个时间步求解一个凸二次规划问题：
$$\text{minimize}_{x, u} \sum_{k=0}^{N-1} \left( \|x_k - x_k^{ref}\|_Q^2 + \|u_k\|_R^2 \right)$$
约束条件包括：
- **线性化动力学**：$x_{k+1} = \hat{A}_k x_k + \hat{B}_k u_k$
- **摩擦锥**：$-\mu F_{z,i,k} \leq F_{x,i,k} \leq \mu F_{z,i,k}$ 等
- **力限制**：$0 \leq F_{z,i,k} \leq F_{max}$
- **线足约束**：$-l_h F_{z,i,k} \leq M_{y,i,k} \leq l_t F_{z,i,k}$
其中 $x = [e; p_c; \omega; \dot{p}_c; g]$ 是状态，$u = [F_0; F_1; M_0; M_1]$ 是控制输入，$Q$ 和 $R$ 是待学习的权重矩阵。

**3. DiffTune 梯度计算**：
DiffTune 通过敏感性传播计算损失 $L(\theta)$ 对参数 $\theta$ 的梯度：
$$\nabla_\theta L = \sum_{j=1}^{T} \frac{\partial L}{\partial x_j} \frac{\partial x_j}{\partial \theta} + \sum_{j=0}^{T-1} \frac{\partial L}{\partial u_j} \frac{\partial u_j}{\partial \theta}$$
其中状态敏感性 $\frac{\partial x_j}{\partial \theta}$ 和控制敏感性 $\frac{\partial u_j}{\partial \theta}$ 通过以下递推关系计算：
$$\frac{\partial x_{j+1}}{\partial \theta} = (\nabla_{x_j} f + \nabla_{u_j} f \nabla_{x_j} h) \frac{\partial x_j}{\partial \theta} + \nabla_{u_j} f \nabla_\theta h$$
$$\frac{\partial u_j}{\partial \theta} = \nabla_{x_j} h \frac{\partial x_j}{\partial \theta} + \nabla_\theta h$$
其中 $f$ 是动力学模型，$h$ 是控制器策略（即 MPC 求解器）。

**4. GRFM-Net 增强的动力学**：
传统的可微仿真器使用名义动力学 $x_{j+1} = f(x_j) + g(x_j) u_j$。GRFM-Net 通过一个神经网络 $\hat{\phi}$ 来修正控制输入，形成增强动力学：
$$\bar{x}_{j+1} = f(\bar{x}_j) + g(\bar{x}_j) \hat{\phi}(u_{j-w:j})$$
其中 $\hat{\phi}$ 学习从历史控制指令序列 $u_{j-w:j}$ 到实际控制效果 $\bar{u}_j$ 的映射。

### 3.3 模型、训练与数据集

- **GRFM-Net**：两个独立的多层感知机（MLP），分别用于预测力和力矩。每个网络有 18 个输入（历史长度 $w=3$，即当前及前两步的 6 维 GRFM 指令），3 个隐藏层（每层 64 个神经元），6 个输出。使用 Softsign 激活函数和层归一化。
- **训练**：使用 Adam 优化器，学习率 0.001，L2 正则化权重 0.001。训练 200 个 epoch 后收敛。力和力矩网络的最终 MSE 损失分别为 12.78 和 0.55。
- **数据集**：从高保真度仿真器（HFS，基于 MATLAB/Simulink 的 Simscape Multibody）中收集。HFS 模拟了关节阻尼、连杆摩擦和弹性地面接触等特性。在多种参考轨迹下收集 MPC 指令和 HFS 传感器读数的数据对。

### 3.4 实验设置与结果

- **机器人平台**：HECTOR V2 双足机器人。
- **测试轨迹**：三种难度递增的轨迹：直线、C 形曲线、S 形曲线。
- **对比基线**：
    1.  名义未调优参数
    2.  专家手动调优参数
    3.  DiffTune 无 GRFM-Net（消融实验）
- **实验结果**：
    - **保真度验证**：图 2 展示了 GRFM-Net 的输出比 MPC 原始指令更接近 HFS 传感器读数，证明了其提升模型保真度的有效性。
    - **硬件实验**：表 I 和图 3 展示了关键结果。
        - **DiffTune + GRFM-Net** 在所有三个轨迹上均取得了最低的总损失 $L$。
        - 与专家调优参数相比，总损失分别降低了 **19.4%**（直线）、**40.5%**（C 形曲线）和 **17.0%**（S 形曲线）。
        - 与名义未调优参数相比，总损失降低高达 **68.7%**。
        - **DiffTune 无 GRFM-Net** 在复杂轨迹（C 形和 S 形曲线）上表现极差，总损失甚至远高于名义参数，这凸显了 GRFM-Net 在弥合 Sim-to-Real 差距中的关键作用。

### 3.5 技术细节

- **损失函数设计**：$L = \alpha_1 L_{Eul} + \alpha_2 L_{pos} + L_{ctrl}$，其中 $L_{Eul}$ 和 $L_{pos}$ 分别惩罚姿态角和质心位置的跟踪误差，$L_{ctrl}$ 惩罚控制输入的平滑度。超参数 $\alpha_1$ 和 $\alpha_2$ 用于平衡不同目标的量级。
- **DiffTune 设置**：学习率 $\beta_Q = 0.05$，$\beta_R = 0.08$。参数 $\theta$ 为 MPC 成本函数中的对角权重矩阵 $Q$ 和 $R$。每次迭代运行 10 次。
- **控制流程**：MPC 以 1000 Hz 的频率运行。其输出的 GRFM 指令通过力-力矩映射（公式 5）转换为关节扭矩，用于支撑腿。摆动腿则使用基于逆运动学的 PD 控制器。

## 4. 意义与影响（Significance and Impact）

- **解决的问题与贡献**：
    - 解决了双足机器人控制器参数依赖专家手动调优的痛点，提出了一种高效、自动化的解决方案。
    - 提出了一种通用的、数据驱动的 Sim-to-Real 迁移方法（GRFM-Net），该方法不依赖于对物理过程的精确建模，而是通过学习指令与效果之间的残差来提升仿真保真度。
    - 成功将基于梯度的自动调优方法（DiffTune）应用于一个具有挑战性的、欠驱动的双足机器人系统，验证了其在高维参数空间中的有效性。

- **对未来研究/应用的影响**：
    - 为机器人控制器的自动化设计和部署提供了新范式，有望加速人形机器人在工业、服务、救援等领域的应用。
    - GRFM-Net 的思想可以推广到其他类型的机器人（如四足机器人、机械臂），用于补偿任何形式的模型-现实不匹配。
    - 为更复杂的控制策略（如全阶动力学 MPC、非线性 MPC）的自动调优奠定了基础。

- **局限与开放问题**：
    - **鲁棒性**：作者指出，当前学习到的参数在部署时对外部干扰的鲁棒性有待提高。
    - **在线调优**：目前是离线学习，未来工作将探索如何实现在线参数调优，使机器人能适应环境变化。
    - **泛化性**：当前是针对单一轨迹进行调优，未来需要开发能跨多种轨迹自动调优的通用管道。
    - **数据依赖**：GRFM-Net 的训练依赖于高保真度仿真器，对于没有精确仿真器的平台，如何获取高质量数据是一个挑战。

## 5. 澄清与简化（Clarifications and Simplifications）

- **DiffTune 是什么？**：可以把它想象成一个“自动驾驶教练”。教练（DiffTune）看着学员（MPC 控制器）在模拟器（SRBM + GRFM-Net）里开车。每次学员开完一圈，教练就能根据行驶轨迹与理想轨迹的偏差（损失函数），精确地指出学员的哪个“旋钮”（参数 $\theta$）应该往哪个方向拧多少，才能开得更好。这个“指出方向”的能力来自于数学上的“梯度”，而 DiffTune 通过自动微分高效地计算了这个梯度。

- **GRFM-Net 的作用是什么？**：模拟器（SRBM）是一个简化的“卡丁车模型”，它忽略了轮胎打滑、悬挂系统等细节。如果直接用这个模型训练学员，学员学到的驾驶技巧在真车上可能不适用。GRFM-Net 就像一个“经验丰富的机械师”，它通过观察真车（高保真仿真器）在各种情况下的表现，学习了一个“校正表”。当学员在模拟器中给出一个方向盘指令时，机械师（GRFM-Net）会告诉模拟器：“这个指令在真车上实际会产生这样的效果”。这样，模拟器就变得更像真车了，学员学到的技巧也就能直接用在真车上。

- **为什么不用真机器人数据训练 GRFM-Net？**：因为真机器人脚上没有足够精确的力传感器来测量真实的 GRFM。所以作者退而求其次，使用了一个非常精确的仿真器（HFS）来生成“伪真实”数据。这就像用顶级赛车游戏模拟器来训练赛车手，虽然不如真车，但比简陋的卡丁车模型好得多。

## 6. 补充说明（Additional Notes）

- **相关工作关联**：
    - 与 **贝叶斯优化（BO）** 相比，本文的 DiffTune 方法利用梯度信息，在高维参数空间中效率更高。BO 是零阶方法，适用于黑盒优化，但维度灾难问题严重。
    - 与 **执行器网络（Actuator Network, AN）** 相比，GRFM-Net 的输入输出链条更长。AN 只建模电机指令到扭矩的短链映射，而 GRFM-Net 建模从 MPC 的 GRFM 指令到实际 GRFM 的映射，这包含了接触交互、机械系统行为和执行器动力学等更复杂的环节。
