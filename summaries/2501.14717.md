# What Really Matters for Table LLMs? A Meta-Evaluation of Model and Data Effects

- arXiv: https://arxiv.org/abs/2501.14717v2
- arXiv ID: 2501.14717

---

## 0. 概览（Concise Summary）

- **主题（Topic）**：表格理解（Table Understanding）领域的大语言模型（LLM）指令微调（Instruction Tuning）研究。
- **问题（Problem）**：当前表格LLM研究中存在“选择悖论”（Paradox of Choice），即由于基础模型（Base Model）和训练数据集（Training Data）的多样性，难以归因性能提升的具体来源，导致结果的可复现性和可比性不足。
- **方法（Method）**：通过控制变量实验，在三个不同的7B规模基础模型（Mistral, OLMo, Phi）上，使用四个现有表格指令微调数据集（TableLlama, TableLLM, TableBenchLLM, TableGPT）进行复制，共得到12个模型，并在16个表格基准上进行评估。
- **创新（Innovation）**：首次通过大规模、跨模型、跨数据集的复制实验，定量地解耦了基础模型选择和训练数据对表格LLM性能的影响，并引入Shapley-R²归因分析量化其贡献。
- **意义（Significance）**：揭示了基础模型的选择是决定表格LLM性能的主导因素（解释了81.6%的性能方差），而训练数据的影响相对较小（13.8%）。这挑战了现有研究中将性能提升主要归功于新训练数据的普遍假设，强调了标准化、跨模型评估框架的重要性。
- **一句话总结**：本文通过大规模复制实验，首次定量证明在表格LLM指令微调中，基础模型的选择比训练数据对性能的影响更为关键。

## 1. 研究动机（Motivation）

- **领域问题**：在LLM时代，表格理解研究面临“选择悖论”。研究者面临众多基础模型（如Llama, Mistral, Phi）和多样化的表格指令微调数据集（如TableLlama, TableLLM, TableGPT）。这些表格LLM在基础模型、训练数据和评估基准上各不相同，导致**性能提升难以归因**。例如，一个模型表现更好，是因为它使用了更好的基础模型，还是因为其训练数据更优质？这引发了关于结果可复现性和可比性的担忧。
- **重要性**：如果不能厘清性能提升的来源，研究者可能会将基础模型本身的优势错误地归功于其提出的新训练数据或方法，从而误导领域发展方向。这对于构建可靠、科学的表格理解研究至关重要。
- **挑战**：现有研究（如Zhang et al., 2024a,b; Zha et al., 2023）通常固定使用单个基础模型，并将性能进步归因于新的指令数据。这种实验设计无法区分基础模型和训练数据各自的贡献，存在潜在的误导性。因此，需要进行**控制变量实验**来解耦这两个因素。

## 2. 创新点（Innovation）

- **大规模、跨模型复制实验**：首次对四个现有表格LLM（TableLlama, TableLLM, TableBenchLLM, TableGPT）在三个不同的7B规模基础模型（Mistral, OLMo, Phi）上进行复制，共产生12个可直接比较的模型。这为解耦基础模型和训练数据的影响提供了统一的实验平台。
- **定量归因分析**：引入**Shapley-R²分解**方法，首次定量地分离了基础模型选择和训练数据对下游任务性能的贡献。结果显示，基础模型选择解释了81.6%的性能方差，而训练数据仅解释了13.8%。这为从业者长期以来的直觉提供了量化、可复现的证据。
- **统一的评估网格**：在16个涵盖多种表格任务和泛化场景的基准上对所有12个模型进行统一评估，提供了一个一致的比较基础，避免了以往研究中因评估基准不同而导致的比较困难。
- **新的SOTA**：在复制过程中，在HiTab数据集上取得了新的最优性能（State-of-the-Art, SOTA）。

## 3. 主要内容（Main Content）【重点】

### 3.1 设计架构与方法

本文的核心方法是**控制变量实验**。作者构建了一个“3（基础模型）× 4（训练数据集）= 12（微调模型）”的网格，系统地研究两个关键因素对表格LLM性能的影响。

- **基础模型（Base Models）**：选择了三个在2023-2024年发布的、不同来源的7B规模指令微调模型：
    - Mistral-7B-Instruct-v0.3
    - OLMo 7B Instruct
    - Phi 3 Small Instruct (7B)
- **训练数据集（Training Datasets）**：复制了四个现有表格LLM的指令微调阶段，使用其公开的训练数据：
    - TableLlama
    - TableLLM
    - TableBenchLLM
    - TableGPT
- **评估基准（Evaluation Benchmarks）**：选择了16个表格基准，分为**域内（In-Domain, ID）** 和**域外（Out-of-Domain, OOD）** 两类，以评估模型的泛化能力。包括真实世界数据集（如FeTaQA, HiTab, WikiTQ）和合成数据集。

通过这种设计，作者可以回答三个核心研究问题（RQ）：
- **RQ1**: 基础模型的选择对性能有多大影响？
- **RQ2**: 训练数据对性能有多大影响？
- **RQ3**: 微调后的模型在域外任务上表现如何？

### 3.2 关键算法与数学推导

本文的核心分析方法是**Shapley-R²分解**，用于量化不同因素对模型性能方差的贡献。

- **Shapley值**：源自合作博弈论，用于公平地分配多个参与者（此处为“基础模型”和“训练集”）共同创造的“收益”（此处为模型性能的R²值）的贡献。它考虑了所有可能的参与者组合，并计算每个参与者的平均边际贡献。
- **R²分解**：将线性回归模型的决定系数（R²）分解为各个预测变量（此处为“基础模型”和“训练集”）的贡献。Shapley-R²分解结合了这两种思想，将R²值归因于不同的预测变量。

具体来说，作者构建了一个线性回归模型，以模型在16个基准上的性能为因变量，以“基础模型”和“训练集”为两个分类自变量。然后，使用Shapley-R²分解来计算每个自变量对总R²的贡献。

**公式表示**：
设 $Y$ 为模型性能，$X_{model}$ 和 $X_{data}$ 分别代表基础模型和训练集。线性回归模型为：
$$Y = \beta_0 + \beta_1 X_{model} + \beta_2 X_{data} + \epsilon$$
其中 $\epsilon$ 是误差项。总R²衡量了模型解释的方差比例。Shapley-R²分解将 $R^2$ 分解为：
$$R^2 = \phi_{model} + \phi_{data} + \phi_{interaction}$$
其中 $\phi_{model}$ 和 $\phi_{data}$ 分别是基础模型和训练集的Shapley-R²贡献，$\phi_{interaction}$ 是它们的交互作用贡献。本文的结果显示，$\phi_{model} = 0.816$，$\phi_{data} = 0.138$，交互作用贡献很小。

### 3.3 模型、训练与数据集

- **模型**：三个7B基础模型（Mistral, OLMo, Phi）和12个在其上微调的表格LLM。
- **训练**：对每个基础模型，分别使用四个表格指令微调数据集进行全参数微调（Full Fine-tuning），复制原始论文的指令微调阶段。
- **数据集**：
    - **训练数据集**：TableLlama, TableLLM, TableBenchLLM, TableGPT。这些数据集包含多种表格任务，如表格问答、事实验证、文本生成等。
    - **评估数据集**：共16个，包括8个真实数据集（如FeTaQA, HiTab, TabMWP, TAT-QA, WikiTQ, TabFact, ToTTo, Beer等）和8个合成数据集（如DeepM, DI, ED, C, CF, CTA, TabBeval等），覆盖了表格问答、事实验证、文本生成、模式推理等多种任务。

### 3.4 实验设置与结果

- **实验设置**：
    - 在三个基础模型上，分别用四个训练数据集进行微调，得到12个模型。
    - 在16个评估基准上评估这12个模型，并与未微调的基础模型（N/A）进行比较。
    - 使用Shapley-R²分解分析性能方差来源。
    - 计算微调后模型与基础模型在OOD任务上性能的皮尔逊相关系数（Pearson r）。

- **主要结果**：
    1.  **基础模型选择是主导因素（RQ1）**：
        - **Shapley-R²分解**（图4）显示，基础模型选择解释了**81.6%** 的性能方差，而训练数据仅解释了**13.8%**。
        - **强相关性**（图5）：微调后模型在OOD任务上的性能与基础模型本身的性能之间存在强线性相关（Pearson r 通常在0.7到0.9之间）。例如，Phi模型本身在TabMWP上就优于Mistral和OLMo，微调后依然保持优势。
        - **结论**：基础模型的内在能力是决定微调后模型性能的最关键因素。

    2.  **训练数据的影响（RQ2）**：
        - **域内性能显著提升**：当评估任务包含在训练数据中时，微调后模型性能相比基础模型有显著提升（例如，Mistral在FeTaQA上从20.0提升到38.7）。
        - **特定训练数据表现更优**：TableLLM的训练数据在不同基础模型上，尤其是在表格问答任务上，普遍取得了最佳或具有竞争力的OOD性能。作者认为这可能是因为TableLLM在构建数据时使用了GPT-3.5等LLM来增强推理过程。
        - **结论**：训练数据对域内性能提升显著，但其对OOD泛化的影响有限且依赖于数据质量。

    3.  **OOD泛化仍是挑战（RQ3）**：
        - **性能差距巨大**：模型在OOD任务上的最佳性能远低于其在域内任务上的性能。例如，Phi模型在包含HiTab的训练集上达到63.6，而最佳OOD性能仅为45.3。
        - **性能可能下降**：指令微调有时会导致OOD性能下降。例如，Mistral在WikiTQ上从27.9降至25.5（使用TableGPT数据），表明存在过拟合或灾难性遗忘的风险。
        - **错误分析**（图7，表3）：模型最常见的错误是**接地错误（Grounding Error）**（占67%），即无法将问题与表格中的正确信息关联起来，尤其是在处理长表格或长提示时。其次是**数学推理错误（Math Reasoning Error）**（14%）。这表明指令微调本身不足以培养鲁棒的表格接地和数值推理能力。

### 3.5 技术细节

- **复制实现**：作者声称其复制模型取得了与原始论文相当或更好的域内性能（图3）。
- **Shapley-R²分解**：这是一种基于博弈论的归因方法，比简单的方差分析更公平，因为它考虑了所有可能的因素组合。具体实现细节在论文Section F.1中。
- **错误分析**：作者从Phi模型（使用TableLLM数据训练）的预测结果中随机抽取了1000个样本进行人工错误分析，将错误分为五类：接地错误、数学推理错误、未遵循指令、幻觉和常识错误。

## 4. 意义与影响（Significance and Impact）

- **解决了什么问题**：
    - 解决了表格LLM研究中“选择悖论”带来的**性能归因困难**问题。
    - 挑战了现有研究中普遍存在的、将性能提升主要归功于新训练数据的**隐含假设**。
    - 揭示了**基础模型选择**在表格指令微调中的主导作用，为领域提供了第一个量化的、可复现的证据。

- **贡献**：
    - 提供了第一个大规模、跨模型、跨数据集的表格LLM复制研究。
    - 建立了一个包含12个模型和16个基准的统一评估网格，为未来研究提供了标准化比较的基础。
    - 通过Shapley-R²分解，首次定量分离了基础模型和训练数据的贡献。
    - 指出了表格LLM在**泛化**和**推理**（特别是接地和数值推理）方面的持续弱点。

- **对未来研究/应用的影响**：
    - **研究范式转变**：呼吁研究者从“设计更好的训练数据”转向“选择更强的基座模型”，并建立更系统、跨模型的评估框架。
    - **评估实践**：强调**扩展评估广度**（跨模型、跨数据集）与扩展模型规模同等重要。未来的研究应进行更广泛的标准化比较。
    - **模型开发**：提示开发者在进行表格指令微调时，应优先考虑选择性能更强的基座模型，并注意微调可能带来的OOD性能下降风险。

- **局限与开放问题**：
    - 仅研究了三个7B规模的基础模型，结果在更大模型或不同架构模型上的泛化性有待验证。
    - 无法对闭源商业模型（如GPT-4）进行此类控制实验。
    - 未能穷尽所有可能的基础模型和训练数据集。

## 5. 澄清与简化（Clarifications and Simplifications）

- **“选择悖论”类比**：想象你要做一道菜（训练一个表格LLM）。你有很多种锅（基础模型，如Mistral, Phi）和很多种食材（训练数据，如TableLlama, TableLLM）。以前的研究总是说“因为我用了新的食材，所以菜更好吃了”。但本文发现，其实**锅的好坏**（基础模型的能力）对菜的味道影响更大，占了80%以上。食材（训练数据）虽然也重要，但远没有锅的影响大。所以，如果你想做一道好菜，首先得选一口好锅。

- **Shapley-R²分解类比**：假设一个篮球队赢球了，功劳归谁？是得分后卫（基础模型）还是教练的战术（训练数据）？Shapley-R²分解就像一个公平的统计方法，它会计算：如果把得分后卫换掉，球队赢球概率下降多少？如果把教练换掉，赢球概率下降多少？通过比较所有可能的“换人”组合，最终算出得分后卫和教练对赢球的贡献分别是多少。本文就是用这个方法算出“基础模型”贡献了81.6%的赢球概率。

- **接地错误（Grounding Error）**：这是最常见的错误。可以理解为模型“看”了表格，但“对不上号”。比如，问题问“哪个国家的金牌最多？”，表格里有一列“国家”和一列“金牌数”。模型可能知道要回答“美国”，但它错误地把“金牌数”列里的一个数字当成了国家名，或者根本没找到“金牌数”这一列。这就像让一个学生去图书馆找一本书，他进了图书馆（看到了表格），但不知道按什么索引去找（无法将问题与表格结构关联起来）。
